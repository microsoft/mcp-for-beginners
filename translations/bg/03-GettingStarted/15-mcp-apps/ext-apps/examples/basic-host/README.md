# Пример: Основен хост

Примерна имплементация, показваща как да се изгради MCP хост приложение, което се свързва с MCP сървъри и рендерира UI на инструменти в защитена пясъчник среда.

Този основен хост може също да се използва за тестване на MCP приложения по време на локална разработка.

## Основни файлове

- [`index.html`](../../../../../../../03-GettingStarted/15-mcp-apps/ext-apps/examples/basic-host/index.html) / [`src/index.tsx`](../../../../../../../03-GettingStarted/15-mcp-apps/ext-apps/examples/basic-host/src/index.tsx) - React UI хост с избор на инструмент, въвеждане на параметри и управление на iframe
- [`sandbox.html`](../../../../../../../03-GettingStarted/15-mcp-apps/ext-apps/examples/basic-host/sandbox.html) / [`src/sandbox.ts`](../../../../../../../03-GettingStarted/15-mcp-apps/ext-apps/examples/basic-host/src/sandbox.ts) - Външен iframe прокси с валидиране на сигурността и двупосочно предаване на съобщения
- [`src/implementation.ts`](../../../../../../../03-GettingStarted/15-mcp-apps/ext-apps/examples/basic-host/src/implementation.ts) - Основна логика: връзка със сървър, извикване на инструменти и настройка на AppBridge

## Първи стъпки

```bash
npm install
npm run start
# Отворете http://localhost:8080
```

По подразбиране хост приложението ще се опита да се свърже с MCP сървър на `http://localhost:3001/mcp`. Можете да конфигурирате това поведение чрез задаване на променливата на средата `SERVERS` със JSON масив от URL адреси на сървъри:

```bash
SERVERS='["http://localhost:1234/mcp", "http://localhost:5678/mcp"]' npm run start
```

## Архитектура

Този пример използва модел с двойни iframe елементи за сигурна изолация на UI:

```
Host (port 8080)
  └── Outer iframe (port 8081) - sandbox proxy
        └── Inner iframe (srcdoc) - untrusted tool UI
```

**Защо два iframe елемента?**

- Външният iframe работи на отделен произход (порт 8081), което предотвратява директен достъп до хоста
- Вътрешният iframe получава HTML чрез `srcdoc` и е ограничен от sandbox атрибути
- Съобщенията преминават през външния iframe, който ги валидира и ги препраща двупосочно

Тази архитектура гарантира, че дори и кодът на UI на инструмента да е злонамерен, той не може да осъществи достъп до DOM-а на хост приложението, бисквитките или JavaScript контекста.
