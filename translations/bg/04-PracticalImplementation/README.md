# Практическа реализация

[![Как да създаваме, тестваме и внедряваме MCP приложения с реални инструменти и работни процеси](../../../translated_images/bg/05.64bea204e25ca891.webp)](https://youtu.be/vCN9-mKBDfQ)

_(Кликнете върху изображението по-горе, за да гледате видео на този урок)_

Практическата реализация е мястото, където силата на Model Context Protocol (MCP) става осезаема. Докато разбирането на теорията и архитектурата зад MCP е важно, истинската стойност се появява, когато приложите тези концепции за изграждане, тестване и внедряване на решения, които решават реални проблеми. Тази глава създава мост между концептуалните знания и практическата разработка, като ви насочва в процеса на оживяване на базирани на MCP приложения.

Независимо дали разработвате интелигентни асистенти, интегрирате AI в бизнес работни процеси или създавате персонализирани инструменти за обработка на данни, MCP предоставя гъвкава основа. Неговият езиково-агностичен дизайн и официални SDK за популярни програмни езици го правят достъпен за широк диапазон от разработчици. Чрез използване на тези SDK можете бързо да прототипирате, итератерате и мащабирате решенията си на различни платформи и среди.

В следващите секции ще намерите практически примери, примерен код и стратегии за внедряване, които демонстрират как да приложите MCP в C#, Java със Spring, TypeScript, JavaScript и Python. Ще научите също как да дебъгвате и тествате MCP сървъри, да управлявате API и да внедрявате решения в облака чрез Azure. Тези практични ресурси са създадени, за да ускорят вашето обучение и да ви помогнат уверено да изградите надеждни, готови за продукция MCP приложения.

## Преглед

Този урок се фокусира върху практическите аспекти на реализация на MCP чрез няколко програмни езика. Ще изследваме как да използваме MCP SDK в C#, Java със Spring, TypeScript, JavaScript и Python за изграждане на стабилни приложения, дебъг и тестване на MCP сървъри и създаване на многократно използваеми ресурси, подсказки и инструменти.

## Основни функции на сървъра

MCP сървърите могат да реализират всякаква комбинация от следните функции:

### Ресурси

Ресурсите предоставят контекст и данни за използване от потребителя или AI модела:

- Релационни хранилища с документи
- Бази със знания
- Структурирани източници на данни
- Файлови системи

### Подсказки

Подсказките са шаблонни съобщения и работни процеси за потребителите:

- Предефинирани шаблони за разговори
- Водени модели на взаимодействие
- Специализирани диалогови структури

### Инструменти

Инструментите са функции, които AI моделът изпълнява:

- Помощни средства за обработка на данни
- Външни интеграции с API
- Изчислителни възможности
- Функционалности за търсене

## Внедряване на Отдалечен MCP сървър в Azure

Нека видим дали можем да внедрим споменатия по-рано пример:

1. Клонирайте репозиторито

    ```bash
    git clone https://github.com/Azure-Samples/remote-mcp-apim-functions-python.git
    cd remote-mcp-apim-functions-python
    ```

1. Регистрирайте ресурсния доставчик `Microsoft.App`.

   - Ако използвате Azure CLI, изпълнете `az provider register --namespace Microsoft.App --wait`.
   - Ако използвате Azure PowerShell, изпълнете `Register-AzResourceProvider -ProviderNamespace Microsoft.App`. След това изчакайте и изпълнете `(Get-AzResourceProvider -ProviderNamespace Microsoft.App).RegistrationState`, за да проверите дали регистрацията е завършена.

1. Изпълнете тази [azd](https://aka.ms/azd) команда, за да осигурите API Management услугата, function app (с код) и всички други необходими Azure ресурси

    ```shell
    azd up
    ```

    Тази команда трябва да внедри всички облачни ресурси в Azure

### Тествайте вашия сървър с MCP Inspector

1. В **нова терминална сесия**, инсталирайте и стартирайте MCP Inspector

    ```shell
    npx @modelcontextprotocol/inspector
    ```

    Трябва да видите интерфейс, подобен на:

    ![Connect to Node inspector](../../../translated_images/bg/connect.141db0b2bd05f096.webp)

1. Натиснете CTRL и кликнете, за да заредите MCP Inspector уеб приложението от URL-а, показан от приложението (например [http://127.0.0.1:6274/#resources](http://127.0.0.1:6274/#resources))
1. Задайте типа транспорт на `SSE`
1. Посочете URL-а на работещия API Management SSE крайна точка, показана след `azd up` и натиснете **Connect**:

    ```shell
    https://<apim-servicename-from-azd-output>.azure-api.net/mcp/sse
    ```

1. **Списък с инструменти**. Кликнете върху инструмент и **Run Tool**.  

Ако всички стъпки са успешни, вече ще сте свързани със MCP сървъра и ще можете да извикате инструмент.

## MCP сървъри за Azure

[Remote-mcp-functions](https://github.com/Azure-Samples/remote-mcp-functions-dotnet): Този набор от репозитории са бързи шаблони за старт за изграждане и внедряване на персонализирани отдалечени MCP (Model Context Protocol) сървъри, използвайки Azure Functions с Python, C# .NET или Node/TypeScript.

Примерите предоставят пълно решение, което позволява на разработчиците да:

- Строят и пускат локално: Разработват и дебъгват MCP сървър на локална машина
- Внедряват в Azure: Лесно внедряване в облака с проста команда azd up
- Свързват се от клиенти: Свързване към MCP сървъра от различни клиенти, включително режим Copilot агент на VS Code и MCP Inspector инструмент

### Основни функции

- Сигурност по дизайн: MCP сървърът е защитен чрез ключове и HTTPS
- Опции за удостоверяване: Поддържа OAuth чрез вграден auth и/или API Management
- Изолация на мрежата: Позволява изолация на мрежата чрез Azure Virtual Networks (VNET)
- Архитектура без сървър: Използва Azure Functions за мащабируема, събитийно-ориентирана работа
- Локална разработка: Цялостна поддръжка за локална разработка и дебъг
- Лесно внедряване: Оптимизиран процес за внедряване в Azure

Репозиторият включва всички необходими конфигурационни файлове, изходен код и инфраструктурни дефиниции за бърз старт с продукционна MCP реализация.

## Ключови изводи

- MCP SDK предоставя езиково-специфични инструменти за изграждане на стабилни MCP решения
- Процесът на дебъг и тестване е критичен за надеждни MCP приложения
- Многократно използваеми шаблони за подсказки осигуряват последователни AI взаимодействия
- Добре проектирани работни потоци могат да оркестрират сложни задачи с множество инструменти
- Реализацията на MCP решения изисква внимание към сигурността, производителността и обработката на грешки

## Упражнение

Проектирайте практичен MCP работен процес, който адресира реален проблем във вашата област:

1. Идентифицирайте 3-4 инструмента, които биха били полезни за решаване на проблема
2. Създайте диаграма на работния процес, показваща как тези инструменти взаимодействат
3. Реализирайте основна версия на един от инструментите, използвайки предпочитания език
4. Създайте шаблон за подсказка, който да помогне на модела ефективно да използва вашия инструмент
