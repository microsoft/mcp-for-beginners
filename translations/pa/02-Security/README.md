# MCP ਸੁਰੱਖਿਆ: ਏਆਈ ਸਿਸਟਮਾਂ ਲਈ ਵਿਸਤ੍ਰਿਤ ਸੁਰੱਖਿਆ

[![MCP Security Best Practices](../../../translated_images/pa/03.175aed6dedae133f9d41e49cefd0f0a9a39c3317e1eaa7ef7182696af7534308.png)](https://youtu.be/88No8pw706o)

_(ਇਸ ਪਾਠ ਦਾ ਵੀਡੀਓ ਦੇਖਣ ਲਈ ਉਪਰ ਦਿੱਤੀ ਤਸਵੀਰ 'ਤੇ ਕਲਿੱਕ ਕਰੋ)_

ਸੁਰੱਖਿਆ ਏਆਈ ਸਿਸਟਮ ਡਿਜ਼ਾਈਨ ਦਾ ਮੂਲ ਹੈ, ਇਸ ਲਈ ਅਸੀਂ ਇਸਨੂੰ ਆਪਣਾ ਦੂਜਾ ਭਾਗ ਬਣਾਉਂਦੇ ਹਾਂ। ਇਹ ਮਾਈਕ੍ਰੋਸਾਫਟ ਦੇ [ਸੁਰੱਖਿਅਤ ਭਵਿੱਖ ਪਹਿਲ](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) ਤੋਂ ਲਿਆ ਗਿਆ **ਸੁਰੱਖਿਅਤ ਡਿਜ਼ਾਈਨ ਦੁਆਰਾ** ਸਿਧਾਂਤ ਨਾਲ ਮੇਲ ਖਾਂਦਾ ਹੈ।

ਮਾਡਲ ਕਾਂਟੈਕਸਟ ਪ੍ਰੋਟੋਕੋਲ (MCP) ਏਆਈ-ਚਲਿਤ ਐਪਲੀਕੇਸ਼ਨਾਂ ਲਈ ਸ਼ਕਤੀਸ਼ਾਲੀ ਨਵੀਆਂ ਸਮਰੱਥਾਵਾਂ ਲਿਆਉਂਦਾ ਹੈ, ਜਦਕਿ ਇਹ ਪਰੰਪਰਾਗਤ ਸਾਫਟਵੇਅਰ ਖਤਰਿਆਂ ਤੋਂ ਅੱਗੇ ਵਧ ਕੇ ਵਿਲੱਖਣ ਸੁਰੱਖਿਆ ਚੁਣੌਤੀਆਂ ਪੇਸ਼ ਕਰਦਾ ਹੈ। MCP ਸਿਸਟਮਾਂ ਨੂੰ ਸਥਾਪਿਤ ਸੁਰੱਖਿਆ ਚਿੰਤਾਵਾਂ (ਸੁਰੱਖਿਅਤ ਕੋਡਿੰਗ, ਘੱਟੋ-ਘੱਟ ਅਧਿਕਾਰ, ਸਪਲਾਈ ਚੇਨ ਸੁਰੱਖਿਆ) ਅਤੇ ਨਵੇਂ ਏਆਈ-ਵਿਸ਼ੇਸ਼ ਖਤਰਿਆਂ ਦਾ ਸਾਹਮਣਾ ਕਰਨਾ ਪੈਂਦਾ ਹੈ ਜਿਵੇਂ ਕਿ ਪ੍ਰਾਂਪਟ ਇੰਜੈਕਸ਼ਨ, ਟੂਲ ਜਹਿਰੀਲਾ ਬਣਾਉਣਾ, ਸੈਸ਼ਨ ਹਾਈਜੈਕਿੰਗ, ਗੁੰਝਲਦਾਰ ਡਿਪਟੀ ਹਮਲੇ, ਟੋਕਨ ਪਾਸਥਰੂ ਕਮਜ਼ੋਰੀਆਂ, ਅਤੇ ਗਤੀਸ਼ੀਲ ਸਮਰੱਥਾ ਸੋਧ।

ਇਹ ਪਾਠ MCP ਲਾਗੂ ਕਰਨ ਵਿੱਚ ਸਭ ਤੋਂ ਮਹੱਤਵਪੂਰਨ ਸੁਰੱਖਿਆ ਖਤਰਿਆਂ ਦੀ ਜਾਂਚ ਕਰਦਾ ਹੈ—ਜਿਸ ਵਿੱਚ ਪ੍ਰਮਾਣਿਕਤਾ, ਅਧਿਕਾਰ, ਅਤਿਰਿਕਤ ਅਧਿਕਾਰ, ਅਪਰੋਕਸ਼ ਪ੍ਰਾਂਪਟ ਇੰਜੈਕਸ਼ਨ, ਸੈਸ਼ਨ ਸੁਰੱਖਿਆ, ਗੁੰਝਲਦਾਰ ਡਿਪਟੀ ਸਮੱਸਿਆਵਾਂ, ਟੋਕਨ ਪ੍ਰਬੰਧਨ, ਅਤੇ ਸਪਲਾਈ ਚੇਨ ਕਮਜ਼ੋਰੀਆਂ ਸ਼ਾਮਲ ਹਨ। ਤੁਸੀਂ ਐਕਸ਼ਨਯੋਗ ਨਿਯੰਤਰਣ ਅਤੇ ਸਰਵੋਤਮ ਅਭਿਆਸ ਸਿੱਖੋਗੇ ਜੋ ਇਹਨਾਂ ਖਤਰਿਆਂ ਨੂੰ ਘਟਾਉਂਦੇ ਹਨ ਅਤੇ Microsoft ਦੇ ਹੱਲਾਂ ਜਿਵੇਂ ਕਿ ਪ੍ਰਾਂਪਟ ਸ਼ੀਲਡਜ਼, Azure ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ, ਅਤੇ GitHub ਐਡਵਾਂਸਡ ਸੁਰੱਖਿਆ ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਆਪਣੇ MCP ਤਾਇਨਾਤੀ ਨੂੰ ਮਜ਼ਬੂਤ ਕਰਦੇ ਹਨ।

## ਸਿੱਖਣ ਦੇ ਲਕੜੇ

ਇਸ ਪਾਠ ਦੇ ਅੰਤ ਤੱਕ, ਤੁਸੀਂ ਸਮਰੱਥ ਹੋਵੋਗੇ:

- **MCP-ਵਿਸ਼ੇਸ਼ ਖਤਰਿਆਂ ਦੀ ਪਹਿਚਾਣ ਕਰੋ**: MCP ਸਿਸਟਮਾਂ ਵਿੱਚ ਵਿਲੱਖਣ ਸੁਰੱਖਿਆ ਖਤਰਿਆਂ ਨੂੰ ਪਛਾਣੋ ਜਿਵੇਂ ਕਿ ਪ੍ਰਾਂਪਟ ਇੰਜੈਕਸ਼ਨ, ਟੂਲ ਜਹਿਰੀਲਾ ਬਣਾਉਣਾ, ਅਤਿਰਿਕਤ ਅਧਿਕਾਰ, ਸੈਸ਼ਨ ਹਾਈਜੈਕਿੰਗ, ਗੁੰਝਲਦਾਰ ਡਿਪਟੀ ਸਮੱਸਿਆਵਾਂ, ਟੋਕਨ ਪਾਸਥਰੂ ਕਮਜ਼ੋਰੀਆਂ, ਅਤੇ ਸਪਲਾਈ ਚੇਨ ਖਤਰਿਆਂ
- **ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣ ਲਾਗੂ ਕਰੋ**: ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਰੋਕਥਾਮ ਜਿਵੇਂ ਕਿ ਮਜ਼ਬੂਤ ਪ੍ਰਮਾਣਿਕਤਾ, ਘੱਟੋ-ਘੱਟ ਅਧਿਕਾਰ ਪਹੁੰਚ, ਸੁਰੱਖਿਅਤ ਟੋਕਨ ਪ੍ਰਬੰਧਨ, ਸੈਸ਼ਨ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣ, ਅਤੇ ਸਪਲਾਈ ਚੇਨ ਪੁਸ਼ਟੀਕਰਨ ਲਾਗੂ ਕਰੋ
- **Microsoft ਸੁਰੱਖਿਆ ਹੱਲਾਂ ਦੀ ਵਰਤੋਂ ਕਰੋ**: MCP ਵਰਕਲੋਡ ਸੁਰੱਖਿਆ ਲਈ Microsoft ਪ੍ਰਾਂਪਟ ਸ਼ੀਲਡਜ਼, Azure ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ, ਅਤੇ GitHub ਐਡਵਾਂਸਡ ਸੁਰੱਖਿਆ ਨੂੰ ਸਮਝੋ ਅਤੇ ਤਾਇਨਾਤ ਕਰੋ
- **ਟੂਲ ਸੁਰੱਖਿਆ ਦੀ ਪੁਸ਼ਟੀ ਕਰੋ**: ਟੂਲ ਮੈਟਾਡੇਟਾ ਦੀ ਪੁਸ਼ਟੀ, ਗਤੀਸ਼ੀਲ ਬਦਲਾਵਾਂ ਦੀ ਨਿਗਰਾਨੀ, ਅਤੇ ਅਪਰੋਕਸ਼ ਪ੍ਰਾਂਪਟ ਇੰਜੈਕਸ਼ਨ ਹਮਲਿਆਂ ਤੋਂ ਬਚਾਅ ਦੀ ਮਹੱਤਤਾ ਨੂੰ ਸਮਝੋ
- **ਸਰਵੋਤਮ ਅਭਿਆਸਾਂ ਨੂੰ ਜੋੜੋ**: ਸਥਾਪਿਤ ਸੁਰੱਖਿਆ ਮੂਲ (ਸੁਰੱਖਿਅਤ ਕੋਡਿੰਗ, ਸਰਵਰ ਹਾਰਡਨਿੰਗ, ਜ਼ੀਰੋ ਟਰੱਸਟ) ਨੂੰ MCP-ਵਿਸ਼ੇਸ਼ ਨਿਯੰਤਰਣ ਨਾਲ ਮਿਲਾ ਕੇ ਵਿਸਤ੍ਰਿਤ ਸੁਰੱਖਿਆ ਪ੍ਰਦਾਨ ਕਰੋ

# MCP ਸੁਰੱਖਿਆ ਆਰਕੀਟੈਕਚਰ ਅਤੇ ਨਿਯੰਤਰਣ

ਆਧੁਨਿਕ MCP ਲਾਗੂ ਕਰਨ ਲਈ ਪਰਤਦਾਰ ਸੁਰੱਖਿਆ ਪਹੁੰਚਾਂ ਦੀ ਲੋੜ ਹੁੰਦੀ ਹੈ ਜੋ ਪਰੰਪਰਾਗਤ ਸਾਫਟਵੇਅਰ ਸੁਰੱਖਿਆ ਅਤੇ ਏਆਈ-ਵਿਸ਼ੇਸ਼ ਖਤਰਿਆਂ ਦੋਹਾਂ ਨੂੰ ਸੰਬੋਧਨ ਕਰਦੀਆਂ ਹਨ। ਤੇਜ਼ੀ ਨਾਲ ਵਿਕਸਤ ਹੋ ਰਹੀ MCP ਵਿਸ਼ੇਸ਼ਤਾ ਆਪਣੇ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣਾਂ ਨੂੰ ਮਜ਼ਬੂਤ ਕਰ ਰਹੀ ਹੈ, ਜਿਸ ਨਾਲ ਇਹ ਉਦਯੋਗ ਸੁਰੱਖਿਆ ਆਰਕੀਟੈਕਚਰਾਂ ਅਤੇ ਸਥਾਪਿਤ ਸਰਵੋਤਮ ਅਭਿਆਸਾਂ ਨਾਲ ਬਿਹਤਰ ਇੰਟਿਗ੍ਰੇਸ਼ਨ ਕਰ ਸਕਦੀ ਹੈ।

[Microsoft Digital Defense Report](https://aka.ms/mddr) ਤੋਂ ਖੋਜ ਦਰਸਾਉਂਦੀ ਹੈ ਕਿ **98% ਰਿਪੋਰਟ ਕੀਤੀਆਂ ਚੋਰੀਆਂ ਮਜ਼ਬੂਤ ਸੁਰੱਖਿਆ ਸਫਾਈ ਨਾਲ ਰੋਕੀਆਂ ਜਾ ਸਕਦੀਆਂ ਹਨ**। ਸਭ ਤੋਂ ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਸੁਰੱਖਿਆ ਰਣਨੀਤੀ ਮੂਲ ਸੁਰੱਖਿਆ ਅਭਿਆਸਾਂ ਨੂੰ MCP-ਵਿਸ਼ੇਸ਼ ਨਿਯੰਤਰਣਾਂ ਨਾਲ ਜੋੜਦੀ ਹੈ—ਸਾਬਤ ਬੇਸਲਾਈਨ ਸੁਰੱਖਿਆ ਉਪਾਅ ਕੁੱਲ ਸੁਰੱਖਿਆ ਖਤਰੇ ਨੂੰ ਘਟਾਉਣ ਵਿੱਚ ਸਭ ਤੋਂ ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਰਹਿੰਦੇ ਹਨ।

## ਮੌਜੂਦਾ ਸੁਰੱਖਿਆ ਦ੍ਰਿਸ਼

> **ਨੋਟ:** ਇਹ ਜਾਣਕਾਰੀ MCP ਸੁਰੱਖਿਆ ਮਿਆਰਾਂ ਨੂੰ **18 ਦਸੰਬਰ, 2025** ਤੱਕ ਦਰਸਾਉਂਦੀ ਹੈ। MCP ਪ੍ਰੋਟੋਕੋਲ ਤੇਜ਼ੀ ਨਾਲ ਵਿਕਸਤ ਹੋ ਰਿਹਾ ਹੈ, ਅਤੇ ਭਵਿੱਖ ਵਿੱਚ ਨਵੇਂ ਪ੍ਰਮਾਣਿਕਤਾ ਪੈਟਰਨ ਅਤੇ ਵਧੇਰੇ ਨਿਯੰਤਰਣ ਲਾਗੂ ਕੀਤੇ ਜਾ ਸਕਦੇ ਹਨ। ਹਮੇਸ਼ਾ ਮੌਜੂਦਾ [MCP ਵਿਸ਼ੇਸ਼ਤਾ](https://spec.modelcontextprotocol.io/), [MCP GitHub ਰਿਪੋਜ਼ਟਰੀ](https://github.com/modelcontextprotocol), ਅਤੇ [ਸੁਰੱਖਿਆ ਸਰਵੋਤਮ ਅਭਿਆਸ ਦਸਤਾਵੇਜ਼](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) ਨੂੰ ਵੇਖੋ।

### MCP ਪ੍ਰਮਾਣਿਕਤਾ ਦਾ ਵਿਕਾਸ

MCP ਵਿਸ਼ੇਸ਼ਤਾ ਨੇ ਪ੍ਰਮਾਣਿਕਤਾ ਅਤੇ ਅਧਿਕਾਰ ਦੇ ਤਰੀਕੇ ਵਿੱਚ ਕਾਫੀ ਤਬਦੀਲੀ ਕੀਤੀ ਹੈ:

- **ਮੂਲ ਤਰੀਕਾ**: ਸ਼ੁਰੂਆਤੀ ਵਿਸ਼ੇਸ਼ਤਾਵਾਂ ਵਿੱਚ ਵਿਕਾਸਕਾਰਾਂ ਨੂੰ ਕਸਟਮ ਪ੍ਰਮਾਣਿਕਤਾ ਸਰਵਰ ਬਣਾਉਣੇ ਪੈਂਦੇ ਸਨ, ਜਦਕਿ MCP ਸਰਵਰ OAuth 2.0 ਅਧਿਕਾਰ ਸਰਵਰ ਵਜੋਂ ਸਿੱਧਾ ਯੂਜ਼ਰ ਪ੍ਰਮਾਣਿਕਤਾ ਸੰਭਾਲਦੇ ਸਨ
- **ਮੌਜੂਦਾ ਮਿਆਰ (2025-11-25)**: ਅਪਡੇਟ ਕੀਤੀ ਵਿਸ਼ੇਸ਼ਤਾ MCP ਸਰਵਰਾਂ ਨੂੰ ਬਾਹਰੀ ਪਹਚਾਣ ਪ੍ਰਦਾਤਾਵਾਂ (ਜਿਵੇਂ Microsoft Entra ID) ਨੂੰ ਪ੍ਰਮਾਣਿਕਤਾ ਸੌਂਪਣ ਦੀ ਆਗਿਆ ਦਿੰਦੀ ਹੈ, ਜਿਸ ਨਾਲ ਸੁਰੱਖਿਆ ਸਥਿਤੀ ਸੁਧਰਦੀ ਹੈ ਅਤੇ ਲਾਗੂ ਕਰਨ ਦੀ ਜਟਿਲਤਾ ਘਟਦੀ ਹੈ
- **ਟ੍ਰਾਂਸਪੋਰਟ ਲੇਅਰ ਸੁਰੱਖਿਆ**: ਸਥਾਨਕ (STDIO) ਅਤੇ ਰਿਮੋਟ (Streamable HTTP) ਕਨੈਕਸ਼ਨਾਂ ਲਈ ਸਹੀ ਪ੍ਰਮਾਣਿਕਤਾ ਪੈਟਰਨਾਂ ਨਾਲ ਸੁਰੱਖਿਅਤ ਟ੍ਰਾਂਸਪੋਰਟ ਮਕੈਨਿਜ਼ਮਾਂ ਲਈ ਵਧੀਆ ਸਹਿਯੋਗ

## ਪ੍ਰਮਾਣਿਕਤਾ ਅਤੇ ਅਧਿਕਾਰ ਸੁਰੱਖਿਆ

### ਮੌਜੂਦਾ ਸੁਰੱਖਿਆ ਚੁਣੌਤੀਆਂ

ਆਧੁਨਿਕ MCP ਲਾਗੂ ਕਰਨ ਵਿੱਚ ਕਈ ਪ੍ਰਮਾਣਿਕਤਾ ਅਤੇ ਅਧਿਕਾਰ ਚੁਣੌਤੀਆਂ ਹਨ:

### ਖਤਰੇ ਅਤੇ ਹਮਲੇ ਦੇ ਰਸਤੇ

- **ਗਲਤ ਅਧਿਕਾਰ ਲਾਜਿਕ**: MCP ਸਰਵਰਾਂ ਵਿੱਚ ਖਰਾਬ ਅਧਿਕਾਰ ਲਾਗੂ ਕਰਨ ਨਾਲ ਸੰਵੇਦਨਸ਼ੀਲ ਡਾਟਾ ਖੁਲ ਸਕਦਾ ਹੈ ਅਤੇ ਗਲਤ ਤਰੀਕੇ ਨਾਲ ਪਹੁੰਚ ਨਿਯੰਤਰਣ ਲਾਗੂ ਹੋ ਸਕਦੇ ਹਨ
- **OAuth ਟੋਕਨ ਚੋਰੀ**: ਸਥਾਨਕ MCP ਸਰਵਰ ਟੋਕਨ ਚੋਰੀ ਨਾਲ ਹਮਲਾਵਰ ਸਰਵਰਾਂ ਦੀ ਨਕਲ ਕਰਕੇ ਡਾਊਨਸਟਰੀਮ ਸੇਵਾਵਾਂ ਤੱਕ ਪਹੁੰਚ ਪ੍ਰਾਪਤ ਕਰ ਸਕਦੇ ਹਨ
- **ਟੋਕਨ ਪਾਸਥਰੂ ਕਮਜ਼ੋਰੀਆਂ**: ਗਲਤ ਟੋਕਨ ਸੰਭਾਲ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣਾਂ ਨੂੰ ਬਾਈਪਾਸ ਅਤੇ ਜਵਾਬਦੇਹੀ ਵਿੱਚ ਖਾਮੀਆਂ ਪੈਦਾ ਕਰਦੀ ਹੈ
- **ਅਤਿਰਿਕਤ ਅਧਿਕਾਰ**: MCP ਸਰਵਰਾਂ ਨੂੰ ਜ਼ਰੂਰੀ ਘੱਟੋ-ਘੱਟ ਅਧਿਕਾਰ ਨਾ ਦੇਣ ਨਾਲ ਹਮਲਿਆਂ ਲਈ ਮੌਕਾ ਵਧਦਾ ਹੈ

#### ਟੋਕਨ ਪਾਸਥਰੂ: ਇੱਕ ਗੰਭੀਰ ਵਿਰੋਧੀ ਪੈਟਰਨ

**ਟੋਕਨ ਪਾਸਥਰੂ ਮੌਜੂਦਾ MCP ਅਧਿਕਾਰ ਵਿਸ਼ੇਸ਼ਤਾ ਵਿੱਚ ਸਖ਼ਤ ਤੌਰ 'ਤੇ ਮਨਾਹੀ ਹੈ** ਕਿਉਂਕਿ ਇਸ ਦੇ ਗੰਭੀਰ ਸੁਰੱਖਿਆ ਪ੍ਰਭਾਵ ਹਨ:

##### ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣਾਂ ਦੀ ਚਾਲਾਕੀ ਨਾਲ ਚਾਲਾਕੀ
- MCP ਸਰਵਰ ਅਤੇ ਡਾਊਨਸਟਰੀਮ API ਮਹੱਤਵਪੂਰਨ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣ (ਰੇਟ ਸੀਮਾ, ਬੇਨਤੀ ਪੁਸ਼ਟੀ, ਟ੍ਰੈਫਿਕ ਨਿਗਰਾਨੀ) ਲਾਗੂ ਕਰਦੇ ਹਨ ਜੋ ਸਹੀ ਟੋਕਨ ਪੁਸ਼ਟੀ 'ਤੇ ਨਿਰਭਰ ਕਰਦੇ ਹਨ
- ਸਿੱਧਾ ਕਲਾਇੰਟ-ਤੋਂ-API ਟੋਕਨ ਵਰਤੋਂ ਇਹਨਾਂ ਜਰੂਰੀ ਸੁਰੱਖਿਆ ਉਪਾਅ ਨੂੰ ਬਾਈਪਾਸ ਕਰ ਦਿੰਦੀ ਹੈ, ਜਿਸ ਨਾਲ ਸੁਰੱਖਿਆ ਆਰਕੀਟੈਕਚਰ ਖਰਾਬ ਹੁੰਦਾ ਹੈ

##### ਜਵਾਬਦੇਹੀ ਅਤੇ ਆਡਿਟ ਚੁਣੌਤੀਆਂ  
- MCP ਸਰਵਰ ਉੱਪਰ-ਜਾਰੀ ਕੀਤੇ ਟੋਕਨਾਂ ਵਾਲੇ ਕਲਾਇੰਟਾਂ ਵਿੱਚ ਫਰਕ ਨਹੀਂ ਕਰ ਸਕਦੇ, ਜਿਸ ਨਾਲ ਆਡਿਟ ਟਰੇਲ ਟੁੱਟ ਜਾਂਦਾ ਹੈ
- ਡਾਊਨਸਟਰੀਮ ਸਰੋਤ ਸਰਵਰ ਲਾਗ ਹਮਲਾਵਰਾਂ ਦੀ ਥਾਂ MCP ਸਰਵਰ ਮੱਧਸਥਤਾ ਨੂੰ ਗਲਤ ਤਰੀਕੇ ਨਾਲ ਦਰਸਾਉਂਦੇ ਹਨ
- ਘਟਨਾ ਜਾਂਚ ਅਤੇ ਅਨੁਕੂਲਤਾ ਆਡਿਟਿੰਗ ਬਹੁਤ ਮੁਸ਼ਕਲ ਹੋ ਜਾਂਦੀ ਹੈ

##### ਡਾਟਾ ਚੋਰੀ ਦੇ ਖਤਰੇ
- ਅਣਪੁਸ਼ਟੀਕ੍ਰਿਤ ਟੋਕਨ ਦਾਅਵੇ ਚੋਰੀ ਹੋਏ ਟੋਕਨਾਂ ਨਾਲ ਬਦਮਾਸ਼ਾਂ ਨੂੰ MCP ਸਰਵਰਾਂ ਨੂੰ ਡਾਟਾ ਚੋਰੀ ਲਈ ਪ੍ਰਾਕਸੀ ਵਜੋਂ ਵਰਤਣ ਦੀ ਆਗਿਆ ਦਿੰਦੇ ਹਨ
- ਭਰੋਸਾ ਸੀਮਾ ਉਲੰਘਣਾ ਅਣਅਧਿਕ੍ਰਿਤ ਪਹੁੰਚ ਪੈਟਰਨ ਬਣਾਉਂਦੀ ਹੈ ਜੋ ਨਿਰਧਾਰਿਤ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣਾਂ ਨੂੰ ਬਾਈਪਾਸ ਕਰਦੀ ਹੈ

##### ਬਹੁ-ਸੇਵਾ ਹਮਲੇ ਦੇ ਰਸਤੇ
- ਕਈ ਸੇਵਾਵਾਂ ਵੱਲੋਂ ਸਵੀਕਾਰ ਕੀਤੇ ਗਏ ਟੋਕਨ ਹਮਲਾਵਰਾਂ ਨੂੰ ਜੁੜੇ ਸਿਸਟਮਾਂ ਵਿੱਚ ਪਾਸੇ-ਪਾਸੇ ਹਿਲਚਲ ਕਰਨ ਦੀ ਆਗਿਆ ਦਿੰਦੇ ਹਨ
- ਜਦੋਂ ਟੋਕਨ ਮੂਲ ਦੀ ਪੁਸ਼ਟੀ ਨਹੀਂ ਹੋ ਸਕਦੀ, ਤਾਂ ਸੇਵਾਵਾਂ ਵਿਚਕਾਰ ਭਰੋਸਾ ਧਾਰਣਾ ਉਲੰਘਿਆ ਜਾ ਸਕਦਾ ਹੈ

### ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣ ਅਤੇ ਰੋਕਥਾਮ

**ਮਹੱਤਵਪੂਰਨ ਸੁਰੱਖਿਆ ਲੋੜਾਂ:**

> **ਜ਼ਰੂਰੀ**: MCP ਸਰਵਰਾਂ ਨੂੰ **ਕਦੇ ਵੀ** ਉਹ ਟੋਕਨ ਸਵੀਕਾਰ ਨਹੀਂ ਕਰਨੇ ਚਾਹੀਦੇ ਜੋ ਖਾਸ ਤੌਰ 'ਤੇ MCP ਸਰਵਰ ਲਈ ਜਾਰੀ ਨਹੀਂ ਕੀਤੇ ਗਏ

  - **ਪਹਚਾਣ ਇੰਟਿਗ੍ਰੇਸ਼ਨ**: [Microsoft Entra ID ਨਾਲ MCP ਸਰਵਰ ਪ੍ਰਮਾਣਿਕਤਾ](https://den.dev/blog/mcp-server-auth-entra-id-session/)

  - ਸਹੀ ਟੋਕਨ ਘੁੰਮਾਅ ਅਤੇ ਮਿਆਦ ਖਤਮ ਕਰਨ ਦੀ ਨੀਤੀ ਲਾਗੂ ਕਰੋ
  - ਟੋਕਨ ਦੁਹਰਾਈ ਹਮਲਿਆਂ ਅਤੇ ਅਣਅਧਿਕ੍ਰਿਤ ਵਰਤੋਂ ਨੂੰ ਰੋਕੋ

- **ਸੁਰੱਖਿਅਤ ਟੋਕਨ ਸਟੋਰੇਜ**: ਟੋਕਨਾਂ ਨੂੰ ਰੈਸਟ ਅਤੇ ਟ੍ਰਾਂਜ਼ਿਟ ਦੋਹਾਂ ਵਿੱਚ ਇੰਕ੍ਰਿਪਸ਼ਨ ਨਾਲ ਸੁਰੱਖਿਅਤ ਰੱਖੋ
  - **ਸਰਵੋਤਮ ਅਭਿਆਸ**: [ਸੁਰੱਖਿਅਤ ਟੋਕਨ ਸਟੋਰੇਜ ਅਤੇ ਇੰਕ੍ਰਿਪਸ਼ਨ ਮਾਰਗਦਰਸ਼ਨ](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

  - ਵਿਆਪਕ ਜਾਂ ਬੇਲੋੜੇ ਅਧਿਕਾਰਾਂ ਤੋਂ ਬਚੋ ਜੋ ਹਮਲੇ ਦੇ ਮੌਕੇ ਵਧਾਉਂਦੇ ਹਨ

  - ਅਸਧਾਰਣ ਅਧਿਕਾਰ ਵਰਤੋਂ ਦੇ ਪੈਟਰਨਾਂ ਦੀ ਨਿਗਰਾਨੀ ਕਰੋ

## ਏਆਈ-ਵਿਸ਼ੇਸ਼ ਸੁਰੱਖਿਆ ਖਤਰੇ

### ਪ੍ਰਾਂਪਟ ਇੰਜੈਕਸ਼ਨ ਅਤੇ ਟੂਲ ਮੈਨਿਪੂਲੇਸ਼ਨ ਹਮਲੇ

ਆਧੁਨਿਕ MCP ਲਾਗੂ ਕਰਨ ਵਿੱਚ ਸੁਖੜੇ ਏਆਈ-ਵਿਸ਼ੇਸ਼ ਹਮਲੇ ਦੇ ਰਸਤੇ ਹਨ ਜੋ ਪਰੰਪਰਾਗਤ ਸੁਰੱਖਿਆ ਉਪਾਅ ਪੂਰੀ ਤਰ੍ਹਾਂ ਸੰਬੋਧਨ ਨਹੀਂ ਕਰ ਸਕਦੇ:

#### **ਅਪਰੋਕਸ਼ ਪ੍ਰਾਂਪਟ ਇੰਜੈਕਸ਼ਨ (ਕ੍ਰਾਸ-ਡੋਮੇਨ ਪ੍ਰਾਂਪਟ ਇੰਜੈਕਸ਼ਨ)**

**ਅਪਰੋਕਸ਼ ਪ੍ਰਾਂਪਟ ਇੰਜੈਕਸ਼ਨ** MCP-ਸਮਰਥਿਤ ਏਆਈ ਸਿਸਟਮਾਂ ਵਿੱਚ ਸਭ ਤੋਂ ਗੰਭੀਰ ਕਮਜ਼ੋਰੀਆਂ ਵਿੱਚੋਂ ਇੱਕ ਹੈ। ਹਮਲਾਵਰ ਬਾਹਰੀ ਸਮੱਗਰੀ—ਦਸਤਾਵੇਜ਼, ਵੈੱਬ ਪੰਨੇ, ਈਮੇਲ ਜਾਂ ਡਾਟਾ ਸਰੋਤਾਂ—ਵਿੱਚ ਖ਼ਤਰਨਾਕ ਹੁਕਮ ਛੁਪਾ ਦਿੰਦੇ ਹਨ, ਜੋ ਏਆਈ ਸਿਸਟਮ ਬਾਅਦ ਵਿੱਚ ਵੈਧ ਹੁਕਮਾਂ ਵਾਂਗ ਪ੍ਰਕਿਰਿਆ ਕਰਦਾ ਹੈ।

**ਹਮਲੇ ਦੇ ਦ੍ਰਿਸ਼:**
- **ਦਸਤਾਵੇਜ਼-ਆਧਾਰਿਤ ਇੰਜੈਕਸ਼ਨ**: ਪ੍ਰਕਿਰਿਆ ਕੀਤੇ ਦਸਤਾਵੇਜ਼ਾਂ ਵਿੱਚ ਛੁਪੇ ਖ਼ਤਰਨਾਕ ਹੁਕਮ ਜੋ ਅਣਚਾਹੇ ਏਆਈ ਕਾਰਜਾਂ ਨੂੰ ਚਾਲੂ ਕਰਦੇ ਹਨ
- **ਵੈੱਬ ਸਮੱਗਰੀ ਦਾ ਦੁਰਪਯੋਗ**: ਖ਼ਤਰਨਾਕ ਪ੍ਰਾਂਪਟਾਂ ਵਾਲੇ ਵੈੱਬ ਪੰਨੇ ਜੋ ਸਕ੍ਰੈਪ ਕੀਤੇ ਜਾਣ 'ਤੇ ਏਆਈ ਵਿਹਾਰ ਨੂੰ ਮੈਨਿਪੂਲੇਟ ਕਰਦੇ ਹਨ
- **ਈਮੇਲ-ਆਧਾਰਿਤ ਹਮਲੇ**: ਈਮੇਲਾਂ ਵਿੱਚ ਖ਼ਤਰਨਾਕ ਪ੍ਰਾਂਪਟ ਜੋ ਏਆਈ ਸਹਾਇਕਾਂ ਨੂੰ ਜਾਣਕਾਰੀ ਲੀਕ ਕਰਨ ਜਾਂ ਅਣਅਧਿਕ੍ਰਿਤ ਕਾਰਵਾਈ ਕਰਨ ਲਈ ਮਜਬੂਰ ਕਰਦੇ ਹਨ
- **ਡਾਟਾ ਸਰੋਤ ਪ੍ਰਦੂਸ਼ਣ**: ਖ਼ਤਰਨਾਕ ਸਮੱਗਰੀ ਵਾਲੇ ਡਾਟਾਬੇਸ ਜਾਂ API ਜੋ ਏਆਈ ਸਿਸਟਮਾਂ ਨੂੰ ਪ੍ਰਦਾਨ ਕੀਤੇ ਜਾਂਦੇ ਹਨ

**ਅਸਲੀ ਪ੍ਰਭਾਵ**: ਇਹ ਹਮਲੇ ਡਾਟਾ ਚੋਰੀ, ਗੋਪਨੀਯਤਾ ਉਲੰਘਣਾ, ਨੁਕਸਾਨਦਾਇਕ ਸਮੱਗਰੀ ਬਣਾਉਣ, ਅਤੇ ਯੂਜ਼ਰ ਇੰਟਰੈਕਸ਼ਨਾਂ ਦੀ ਮੈਨਿਪੂਲੇਸ਼ਨ ਦਾ ਕਾਰਨ ਬਣ ਸਕਦੇ ਹਨ। ਵਿਸਥਾਰ ਲਈ ਵੇਖੋ [Prompt Injection in MCP (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/)।

![Prompt Injection Attack Diagram](../../../translated_images/pa/prompt-injection.ed9fbfde297ca877c15bc6daa808681cd3c3dc7bf27bbbda342ef1ba5fc4f52d.png)

#### **ਟੂਲ ਜਹਿਰੀਲਾ ਬਣਾਉਣਾ ਹਮਲੇ**

**ਟੂਲ ਜਹਿਰੀਲਾ ਬਣਾਉਣਾ** MCP ਟੂਲਾਂ ਦੀ ਮੈਟਾਡੇਟਾ ਨੂੰ ਨਿਸ਼ਾਨਾ ਬਣਾਉਂਦਾ ਹੈ, ਜੋ LLMs ਟੂਲ ਵੇਰਵੇ ਅਤੇ ਪੈਰਾਮੀਟਰਾਂ ਨੂੰ ਸਮਝ ਕੇ ਕਾਰਜ ਕਰਨ ਦੇ ਤਰੀਕੇ ਨੂੰ ਸ਼ੋਸ਼ਣ ਕਰਦਾ ਹੈ।

**ਹਮਲੇ ਦੇ ਤਰੀਕੇ:**
- **ਮੈਟਾਡੇਟਾ ਮੈਨਿਪੂਲੇਸ਼ਨ**: ਹਮਲਾਵਰ ਟੂਲ ਵੇਰਵੇ, ਪੈਰਾਮੀਟਰ ਪਰਿਭਾਸ਼ਾ, ਜਾਂ ਵਰਤੋਂ ਦੇ ਉਦਾਹਰਣਾਂ ਵਿੱਚ ਖ਼ਤਰਨਾਕ ਹੁਕਮ ਸ਼ਾਮਲ ਕਰਦੇ ਹਨ
- **ਅਦ੍ਰਿਸ਼ਟ ਹੁਕਮ**: ਟੂਲ ਮੈਟਾਡੇਟਾ ਵਿੱਚ ਛੁਪੇ ਪ੍ਰਾਂਪਟ ਜੋ ਏਆਈ ਮਾਡਲ ਦੁਆਰਾ ਪ੍ਰਕਿਰਿਆ ਕੀਤੇ ਜਾਂਦੇ ਹਨ ਪਰ ਮਨੁੱਖੀ ਵਰਤੋਂਕਾਰਾਂ ਲਈ ਅਦ੍ਰਿਸ਼ਟ ਹੁੰਦੇ ਹਨ
- **ਗਤੀਸ਼ੀਲ ਟੂਲ ਸੋਧ ("ਰੱਗ ਪੁੱਲ")**: ਵਰਤੋਂਕਾਰਾਂ ਦੁਆਰਾ ਮਨਜ਼ੂਰ ਕੀਤੇ ਟੂਲਾਂ ਨੂੰ ਬਾਅਦ ਵਿੱਚ ਬਿਨਾਂ ਜਾਣੂ ਕਰਵਾਏ ਖ਼ਤਰਨਾਕ ਕਾਰਜ ਕਰਨ ਲਈ ਸੋਧਿਆ ਜਾਂਦਾ ਹੈ
- **ਪੈਰਾਮੀਟਰ ਇੰਜੈਕਸ਼ਨ**: ਟੂਲ ਪੈਰਾਮੀਟਰ ਸਕੀਮਾਂ ਵਿੱਚ ਖ਼ਤਰਨਾਕ ਸਮੱਗਰੀ ਜੋ ਮਾਡਲ ਵਿਹਾਰ ਨੂੰ ਪ੍ਰਭਾਵਿਤ ਕਰਦੀ ਹੈ

## ਉੱਨਤ MCP ਸੁਰੱਖਿਆ ਖਤਰੇ

### ਸੈਸ਼ਨ ਹਾਈਜੈਕਿੰਗ ਕਮਜ਼ੋਰੀਆਂ

**ਸੈਸ਼ਨ ਹਾਈਜੈਕਿੰਗ** ਸਥਿਤੀ ਵਾਲੇ MCP ਲਾਗੂ ਕਰਨ ਵਿੱਚ ਇੱਕ ਗੰਭੀਰ ਹਮਲਾ ਹੈ ਜਿੱਥੇ ਅਣਅਧਿਕ੍ਰਿਤ ਧਿਰਾਂ ਵਾਸਤਵਿਕ ਸੈਸ਼ਨ ਪਹਚਾਣ ਪੱਤਰ ਪ੍ਰਾਪਤ ਕਰਕੇ ਕਲਾਇੰਟਾਂ ਦੀ ਨਕਲ ਕਰਦੀਆਂ ਹਨ ਅਤੇ ਅਣਅਧਿਕ੍ਰਿਤ ਕਾਰਵਾਈਆਂ ਕਰਦੀਆਂ ਹਨ।

#### **ਹਮਲੇ ਦੇ ਦ੍ਰਿਸ਼ ਅਤੇ ਖਤਰੇ**

- **ਸੈਸ਼ਨ ਹਾਈਜੈਕ ਪ੍ਰਾਂਪਟ ਇੰਜੈਕਸ਼ਨ**: ਚੋਰੀ ਹੋਏ ਸੈਸ਼ਨ ID ਵਾਲੇ ਹਮਲਾਵਰ ਸੈਸ਼ਨ ਸਾਂਝੇ ਕਰਨ ਵਾਲੇ ਸਰਵਰਾਂ ਵਿੱਚ ਖ਼ਤਰਨਾਕ ਘਟਨਾਵਾਂ ਇੰਜੈਕਟ ਕਰ ਸਕਦੇ ਹਨ, ਜੋ ਨੁਕਸਾਨਦਾਇਕ ਕਾਰਵਾਈਆਂ ਜਾਂ ਸੰਵੇਦਨਸ਼ੀਲ ਡਾਟਾ ਤੱਕ ਪਹੁੰਚ ਦਾ ਕਾਰਨ ਬਣ ਸਕਦੇ ਹਨ
- **ਸਿੱਧੀ ਨਕਲ**: ਚੋਰੀ ਹੋਏ ਸੈਸ਼ਨ ID ਨਾਲ ਹਮਲਾਵਰ ਸਿੱਧਾ MCP ਸਰਵਰ ਕਾਲ ਕਰ ਸਕਦੇ ਹਨ ਜੋ ਪ੍ਰਮਾਣਿਕਤਾ ਨੂੰ ਬਾਈਪਾਸ ਕਰਦਾ ਹੈ ਅਤੇ ਹਮਲਾਵਰਾਂ ਨੂੰ ਵਾਸਤਵਿਕ ਯੂਜ਼ਰ ਵਾਂਗ ਵਰਤਦਾ ਹੈ
- **ਖ਼ਰਾਬ ਰੀਜ਼ਿਊਮੇਬਲ ਸਟ੍ਰੀਮ**: ਹਮਲਾਵਰ ਬੇਨਤੀਆਂ ਨੂੰ ਅਕਸਰ ਰੋਕ ਸਕਦੇ ਹਨ, ਜਿਸ ਨਾਲ ਵਾਸਤਵਿਕ ਕਲਾਇੰਟਾਂ ਨੂੰ ਸੰਭਾਵਿਤ ਖ਼ਤਰਨਾਕ ਸਮੱਗਰੀ ਨਾਲ ਰੀਜ਼ਿਊਮ ਕਰਨਾ ਪੈਂਦਾ ਹੈ

#### **ਸੈਸ਼ਨ ਪ੍ਰਬੰਧਨ ਲਈ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣ**

**ਮਹੱਤਵਪੂਰਨ ਲੋੜਾਂ:**
- **ਅਧਿਕਾਰ ਪ੍ਰਮਾਣਿਕਤਾ ਦੀ ਜਾਂਚ**: MCP ਸਰਵਰ ਜੋ ਅਧਿਕਾਰ ਲਾਗੂ ਕਰਦੇ ਹਨ ਉਹ ਸਾਰੇ ਆਉਣ ਵਾਲੇ ਬੇਨਤੀਆਂ ਦੀ ਜਾਂਚ **ਜ਼ਰੂਰ** ਕਰਨ ਅਤੇ ਪ੍ਰਮਾਣਿਕਤਾ ਲਈ ਸੈਸ਼ਨਾਂ 'ਤੇ ਭਰੋਸਾ ਨਹੀਂ ਕਰਨਾ ਚਾਹੀਦਾ

- **ਉਪਭੋਗਤਾ-ਵਿਸ਼ੇਸ਼ ਬਾਈਂਡਿੰਗ**: ਸੈਸ਼ਨ ID ਨੂੰ ਉਪਭੋਗਤਾ-ਵਿਸ਼ੇਸ਼ ਜਾਣਕਾਰੀ ਨਾਲ ਜੁੜੋ ਜਿਵੇਂ ਕਿ `<user_id>:<session_id>` ਫਾਰਮੈਟ ਵਰਤ ਕੇ ਤਾਂ ਜੋ ਵੱਖ-ਵੱਖ ਉਪਭੋਗਤਾਵਾਂ ਦੇ ਸੈਸ਼ਨਾਂ ਦੀ ਗਲਤ ਵਰਤੋਂ ਰੋਕੀ ਜਾ ਸਕੇ
- **ਸੈਸ਼ਨ ਜੀਵਨਚੱਕਰ ਪ੍ਰਬੰਧਨ**: ਸਹੀ ਸਮਾਪਤੀ, ਘੁੰਮਾਅ ਅਤੇ ਅਵੈਧਤਾ ਲਾਗੂ ਕਰੋ ਤਾਂ ਜੋ ਕਮਜ਼ੋਰੀਆਂ ਦੇ ਖਿੜਕੀਆਂ ਸੀਮਿਤ ਰਹਿਣ
- **ਟ੍ਰਾਂਸਪੋਰਟ ਸੁਰੱਖਿਆ**: ਸਾਰੇ ਸੰਚਾਰ ਲਈ ਜ਼ਰੂਰੀ HTTPS ਵਰਤੋ ਤਾਂ ਜੋ ਸੈਸ਼ਨ ID ਦੀ ਚੋਰੀ ਰੋਕੀ ਜਾ ਸਕੇ

### ਗੁੰਝਲਦਾਰ ਡਿਪਟੀ ਸਮੱਸਿਆ

**ਗੁੰਝਲਦਾਰ ਡਿਪਟੀ ਸਮੱਸਿਆ** ਉਸ ਵੇਲੇ ਹੁੰਦੀ ਹੈ ਜਦੋਂ MCP ਸਰਵਰ ਗਾਹਕਾਂ ਅਤੇ ਤੀਜੀ ਪੱਖ ਦੀਆਂ ਸੇਵਾਵਾਂ ਵਿਚਕਾਰ ਪ੍ਰਮਾਣਿਕਤਾ ਪ੍ਰਾਕਸੀ ਵਜੋਂ ਕੰਮ ਕਰਦੇ ਹਨ, ਜਿਸ ਨਾਲ ਸਥਿਰ ਕਲਾਇੰਟ ID ਦੀ ਦੁਰਵਰਤੋਂ ਰਾਹੀਂ ਅਧਿਕਾਰ ਬਾਈਪਾਸ ਦੇ ਮੌਕੇ ਬਣਦੇ ਹਨ।

#### **ਹਮਲਾ ਮਕੈਨਿਕਸ ਅਤੇ ਖਤਰੇ**

- **ਕੁਕੀ-ਆਧਾਰਿਤ ਸਹਿਮਤੀ ਬਾਈਪਾਸ**: ਪਿਛਲੀ ਉਪਭੋਗਤਾ ਪ੍ਰਮਾਣਿਕਤਾ ਸਹਿਮਤੀ ਕੁਕੀਜ਼ ਬਣਾਉਂਦੀ ਹੈ ਜਿਨ੍ਹਾਂ ਨੂੰ ਹਮਲਾਵਰ ਮਾਲਿਸ਼ੀਅਸ ਅਧਿਕਾਰ ਬੇਨਤੀਆਂ ਨਾਲ ਬਣਾਈਆਂ ਗਈਆਂ ਰੀਡਾਇਰੈਕਟ URI ਰਾਹੀਂ ਦੁਰਵਰਤੋਂ ਕਰਦੇ ਹਨ
- **ਅਧਿਕਾਰ ਕੋਡ ਚੋਰੀ**: ਮੌਜੂਦਾ ਸਹਿਮਤੀ ਕੁਕੀਜ਼ ਕਾਰਨ ਅਧਿਕਾਰ ਸਰਵਰ ਸਹਿਮਤੀ ਸਕ੍ਰੀਨਾਂ ਨੂੰ ਛੱਡ ਸਕਦੇ ਹਨ ਅਤੇ ਕੋਡ ਹਮਲਾਵਰ-ਨਿਯੰਤਰਿਤ ਐਂਡਪੌਇੰਟਸ ਨੂੰ ਭੇਜ ਸਕਦੇ ਹਨ  
- **ਬਿਨਾਂ ਅਧਿਕਾਰ ਵਾਲੀ API ਪਹੁੰਚ**: ਚੋਰੀ ਹੋਏ ਅਧਿਕਾਰ ਕੋਡ ਟੋਕਨ ਬਦਲਾਅ ਅਤੇ ਉਪਭੋਗਤਾ ਨਕਲ ਕਰਨ ਦੀ ਆਗਿਆ ਦਿੰਦੇ ਹਨ ਬਿਨਾਂ ਸਪਸ਼ਟ ਮਨਜ਼ੂਰੀ ਦੇ

#### **ਰੋਕਥਾਮ ਰਣਨੀਤੀਆਂ**

**ਜ਼ਰੂਰੀ ਨਿਯੰਤਰਣ:**
- **ਸਪਸ਼ਟ ਸਹਿਮਤੀ ਦੀ ਲੋੜ**: MCP ਪ੍ਰਾਕਸੀ ਸਰਵਰ ਜੋ ਸਥਿਰ ਕਲਾਇੰਟ ID ਵਰਤਦੇ ਹਨ ਉਹ ਹਰ ਗਤੀਸ਼ੀਲ ਰਜਿਸਟਰਡ ਕਲਾਇੰਟ ਲਈ ਉਪਭੋਗਤਾ ਸਹਿਮਤੀ ਲੈਣ **ਜ਼ਰੂਰ** ਕਰਨੀ ਚਾਹੀਦੀ ਹੈ
- **OAuth 2.1 ਸੁਰੱਖਿਆ ਲਾਗੂ ਕਰਨਾ**: ਸਾਰੇ ਅਧਿਕਾਰ ਬੇਨਤੀਆਂ ਲਈ ਮੌਜੂਦਾ OAuth ਸੁਰੱਖਿਆ ਸਰਵੋਤਮ ਅਭਿਆਸਾਂ ਦੀ ਪਾਲਣਾ ਕਰੋ ਜਿਸ ਵਿੱਚ PKCE (ਪ੍ਰੂਫ ਕੀ ਫਾਰ ਕੋਡ ਐਕਸਚੇਂਜ) ਸ਼ਾਮਲ ਹੈ
- **ਕਲਾਇੰਟ ਦੀ ਸਖ਼ਤ ਜਾਂਚ**: ਰੀਡਾਇਰੈਕਟ URI ਅਤੇ ਕਲਾਇੰਟ ਪਹਿਚਾਣਕਾਰਾਂ ਦੀ ਕੜੀ ਜਾਂਚ ਲਾਗੂ ਕਰੋ ਤਾਂ ਜੋ ਦੁਰਵਰਤੋਂ ਰੋਕੀ ਜਾ ਸਕੇ

### ਟੋਕਨ ਪਾਸਥਰੂ ਕਮਜ਼ੋਰੀਆਂ  

**ਟੋਕਨ ਪਾਸਥਰੂ** ਇੱਕ ਸਪਸ਼ਟ ਵਿਰੋਧੀ-ਪੈਟਰਨ ਹੈ ਜਿੱਥੇ MCP ਸਰਵਰ ਗਾਹਕ ਟੋਕਨਾਂ ਨੂੰ ਬਿਨਾਂ ਸਹੀ ਜਾਂਚ ਦੇ ਸਵੀਕਾਰ ਕਰਦੇ ਹਨ ਅਤੇ ਉਨ੍ਹਾਂ ਨੂੰ ਡਾਊਨਸਟਰੀਮ API ਨੂੰ ਅੱਗੇ ਭੇਜਦੇ ਹਨ, ਜੋ MCP ਅਧਿਕਾਰ ਵਿਸ਼ੇਸ਼ਤਾਵਾਂ ਦਾ ਉਲੰਘਣ ਹੈ।

#### **ਸੁਰੱਖਿਆ ਪ੍ਰਭਾਵ**

- **ਨਿਯੰਤਰਣ ਚਾਲਾਕੀ**: ਸਿੱਧਾ ਗਾਹਕ-ਤੋਂ-API ਟੋਕਨ ਵਰਤੋਂ ਮਹੱਤਵਪੂਰਨ ਦਰ-ਸੀਮਾ, ਜਾਂਚ ਅਤੇ ਨਿਗਰਾਨੀ ਨਿਯੰਤਰਣਾਂ ਨੂੰ ਬਾਈਪਾਸ ਕਰਦੀ ਹੈ
- **ਆਡਿਟ ਟ੍ਰੇਲ ਖਰਾਬੀ**: ਅੱਪਸਟਰੀਮ ਜਾਰੀ ਕੀਤੇ ਟੋਕਨ ਗਾਹਕ ਪਛਾਣ ਅਸੰਭਵ ਬਣਾਉਂਦੇ ਹਨ, ਜਿਸ ਨਾਲ ਘਟਨਾ ਜਾਂਚ ਸਮਰੱਥਾ ਟੁੱਟ ਜਾਂਦੀ ਹੈ
- **ਪ੍ਰਾਕਸੀ-ਆਧਾਰਿਤ ਡਾਟਾ ਚੋਰੀ**: ਅਣਜਾਂਚੇ ਟੋਕਨਾਂ ਨਾਲ ਮਾਲਿਸ਼ੀਅਸ ਕਿਰਦਾਰ ਸਰਵਰਾਂ ਨੂੰ ਬਿਨਾਂ ਅਧਿਕਾਰ ਡਾਟਾ ਪਹੁੰਚ ਲਈ ਪ੍ਰਾਕਸੀ ਵਜੋਂ ਵਰਤ ਸਕਦੇ ਹਨ
- **ਭਰੋਸਾ ਸੀਮਾ ਉਲੰਘਣਾ**: ਜਦੋਂ ਟੋਕਨ ਮੂਲ ਦੀ ਪੁਸ਼ਟੀ ਨਹੀਂ ਹੋ ਸਕਦੀ ਤਾਂ ਡਾਊਨਸਟਰੀਮ ਸੇਵਾਵਾਂ ਦੇ ਭਰੋਸੇ ਦੇ ਅਨੁਮਾਨ ਟੁੱਟ ਸਕਦੇ ਹਨ
- **ਬਹੁ-ਸੇਵਾ ਹਮਲਾ ਵਧਾਉਣਾ**: ਕਈ ਸੇਵਾਵਾਂ 'ਤੇ ਸਵੀਕਾਰ ਕੀਤੇ ਗਏ ਸਮਝੌਤਾ ਟੋਕਨ ਲੈਟਰਲ ਮੂਵਮੈਂਟ ਦੀ ਆਗਿਆ ਦਿੰਦੇ ਹਨ

#### **ਲੋੜੀਂਦੇ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣ**

**ਬਿਨਾਂ ਸਮਝੌਤੇ ਦੇ ਲੋੜਾਂ:**
- **ਟੋਕਨ ਜਾਂਚ**: MCP ਸਰਵਰ ਉਹ ਟੋਕਨ ਕਦੇ ਵੀ ਸਵੀਕਾਰ ਨਾ ਕਰਨ ਜੋ ਖਾਸ ਤੌਰ 'ਤੇ MCP ਸਰਵਰ ਲਈ ਜਾਰੀ ਨਾ ਕੀਤੇ ਗਏ ਹੋਣ
- **ਦਰਸ਼ਕ ਜਾਂਚ**: ਹਮੇਸ਼ਾ ਟੋਕਨ ਦਰਸ਼ਕ ਦਾਅਵੇ ਦੀ ਜਾਂਚ ਕਰੋ ਕਿ ਉਹ MCP ਸਰਵਰ ਦੀ ਪਹਿਚਾਣ ਨਾਲ ਮੇਲ ਖਾਂਦੇ ਹਨ
- **ਸਹੀ ਟੋਕਨ ਜੀਵਨਚੱਕਰ**: ਛੋਟੇ ਸਮੇਂ ਵਾਲੇ ਐਕਸੈਸ ਟੋਕਨਾਂ ਨੂੰ ਸੁਰੱਖਿਅਤ ਘੁੰਮਾਅ ਅਭਿਆਸਾਂ ਨਾਲ ਲਾਗੂ ਕਰੋ


## AI ਸਿਸਟਮਾਂ ਲਈ ਸਪਲਾਈ ਚੇਨ ਸੁਰੱਖਿਆ

ਸਪਲਾਈ ਚੇਨ ਸੁਰੱਖਿਆ ਰਵਾਇਤੀ ਸਾਫਟਵੇਅਰ ਨਿਰਭਰਤਾਵਾਂ ਤੋਂ ਅੱਗੇ ਵਧ ਕੇ ਪੂਰੇ AI ਪਰਿਸਰ ਨੂੰ ਸ਼ਾਮਲ ਕਰਦੀ ਹੈ। ਆਧੁਨਿਕ MCP ਲਾਗੂ ਕਰਨ ਵਾਲੇ ਸਾਰੇ AI-ਸੰਬੰਧਿਤ ਘਟਕਾਂ ਦੀ ਕੜੀ ਜਾਂਚ ਅਤੇ ਨਿਗਰਾਨੀ ਕਰਨੀ ਚਾਹੀਦੀ ਹੈ, ਕਿਉਂਕਿ ਹਰ ਇੱਕ ਸੰਭਾਵਿਤ ਕਮਜ਼ੋਰੀਆਂ ਲਿਆਉਂਦਾ ਹੈ ਜੋ ਸਿਸਟਮ ਦੀ ਅਖੰਡਤਾ ਨੂੰ ਖਤਰੇ ਵਿੱਚ ਪਾ ਸਕਦਾ ਹੈ।

### ਵਧੇਰੇ AI ਸਪਲਾਈ ਚੇਨ ਘਟਕ

**ਰਵਾਇਤੀ ਸਾਫਟਵੇਅਰ ਨਿਰਭਰਤਾਵਾਂ:**
- ਖੁੱਲ੍ਹੇ ਸਰੋਤ ਲਾਇਬ੍ਰੇਰੀਆਂ ਅਤੇ ਫਰੇਮਵਰਕ
- ਕੰਟੇਨਰ ਇਮੇਜ ਅਤੇ ਬੇਸ ਸਿਸਟਮ  
- ਵਿਕਾਸ ਟੂਲ ਅਤੇ ਬਿਲਡ ਪਾਈਪਲਾਈਨ
- ਢਾਂਚਾਗਤ ਘਟਕ ਅਤੇ ਸੇਵਾਵਾਂ

**AI-ਵਿਸ਼ੇਸ਼ ਸਪਲਾਈ ਚੇਨ ਤੱਤ:**
- **ਫਾਊਂਡੇਸ਼ਨ ਮਾਡਲ**: ਵੱਖ-ਵੱਖ ਪ੍ਰਦਾਤਾਵਾਂ ਤੋਂ ਪ੍ਰੀ-ਟ੍ਰੇਨਡ ਮਾਡਲ ਜਿਨ੍ਹਾਂ ਦੀ ਮੂਲਤਾ ਦੀ ਜਾਂਚ ਲੋੜੀਂਦੀ ਹੈ
- **ਐਂਬੈਡਿੰਗ ਸੇਵਾਵਾਂ**: ਬਾਹਰੀ ਵੈਕਟਰਾਈਜ਼ੇਸ਼ਨ ਅਤੇ ਸੈਮਾਂਟਿਕ ਖੋਜ ਸੇਵਾਵਾਂ
- **ਸੰਦਰਭ ਪ੍ਰਦਾਤਾ**: ਡਾਟਾ ਸਰੋਤ, ਗਿਆਨ ਅਧਾਰ ਅਤੇ ਦਸਤਾਵੇਜ਼ ਰਿਪੋਜ਼ਿਟਰੀਜ਼  
- **ਤੀਜੀ ਪੱਖ API**: ਬਾਹਰੀ AI ਸੇਵਾਵਾਂ, ML ਪਾਈਪਲਾਈਨ ਅਤੇ ਡਾਟਾ ਪ੍ਰੋਸੈਸਿੰਗ ਐਂਡਪੌਇੰਟ
- **ਮਾਡਲ ਆਰਟੀਫੈਕਟ**: ਵਜ਼ਨ, ਸੰਰਚਨਾ ਅਤੇ ਫਾਈਨ-ਟਿਊਨਡ ਮਾਡਲ ਵੈਰੀਅੰਟ
- **ਟ੍ਰੇਨਿੰਗ ਡਾਟਾ ਸਰੋਤ**: ਮਾਡਲ ਟ੍ਰੇਨਿੰਗ ਅਤੇ ਫਾਈਨ-ਟਿਊਨਿੰਗ ਲਈ ਵਰਤੇ ਗਏ ਡਾਟਾਸੈੱਟ

### ਵਿਸਤ੍ਰਿਤ ਸਪਲਾਈ ਚੇਨ ਸੁਰੱਖਿਆ ਰਣਨੀਤੀ

#### **ਘਟਕ ਜਾਂਚ ਅਤੇ ਭਰੋਸਾ**
- **ਮੂਲਤਾ ਦੀ ਜਾਂਚ**: ਸਾਰੇ AI ਘਟਕਾਂ ਦੇ ਮੂਲ, ਲਾਇਸੈਂਸ ਅਤੇ ਅਖੰਡਤਾ ਦੀ ਜਾਂਚ ਇੰਟਿਗ੍ਰੇਸ਼ਨ ਤੋਂ ਪਹਿਲਾਂ ਕਰੋ
- **ਸੁਰੱਖਿਆ ਮੁਲਾਂਕਣ**: ਮਾਡਲ, ਡਾਟਾ ਸਰੋਤ ਅਤੇ AI ਸੇਵਾਵਾਂ ਲਈ ਕਮਜ਼ੋਰੀ ਸਕੈਨ ਅਤੇ ਸੁਰੱਖਿਆ ਸਮੀਖਿਆ ਕਰੋ
- **ਖਿਆਤੀ ਵਿਸ਼ਲੇਸ਼ਣ**: AI ਸੇਵਾ ਪ੍ਰਦਾਤਾਵਾਂ ਦੇ ਸੁਰੱਖਿਆ ਟ੍ਰੈਕ ਰਿਕਾਰਡ ਅਤੇ ਅਭਿਆਸਾਂ ਦਾ ਮੁਲਾਂਕਣ ਕਰੋ
- **ਅਨੁਕੂਲਤਾ ਜਾਂਚ**: ਯਕੀਨੀ ਬਣਾਓ ਕਿ ਸਾਰੇ ਘਟਕ ਸੰਗਠਨਾਤਮਕ ਸੁਰੱਖਿਆ ਅਤੇ ਨਿਯਮਕ ਲੋੜਾਂ ਨੂੰ ਪੂਰਾ ਕਰਦੇ ਹਨ

#### **ਸੁਰੱਖਿਅਤ ਡਿਪਲੋਇਮੈਂਟ ਪਾਈਪਲਾਈਨ**  
- **ਆਟੋਮੇਟਿਡ CI/CD ਸੁਰੱਖਿਆ**: ਆਟੋਮੇਟਿਡ ਡਿਪਲੋਇਮੈਂਟ ਪਾਈਪਲਾਈਨਾਂ ਵਿੱਚ ਸੁਰੱਖਿਆ ਸਕੈਨਿੰਗ ਸ਼ਾਮਲ ਕਰੋ
- **ਆਰਟੀਫੈਕਟ ਅਖੰਡਤਾ**: ਸਾਰੇ ਡਿਪਲੋਇਡ ਆਰਟੀਫੈਕਟ (ਕੋਡ, ਮਾਡਲ, ਸੰਰਚਨਾ) ਲਈ ਕ੍ਰਿਪਟੋਗ੍ਰਾਫਿਕ ਜਾਂਚ ਲਾਗੂ ਕਰੋ
- **ਚਰਣਵਾਰ ਡਿਪਲੋਇਮੈਂਟ**: ਹਰ ਚਰਣ 'ਤੇ ਸੁਰੱਖਿਆ ਜਾਂਚ ਨਾਲ ਪ੍ਰਗਟਿਸੀਲ ਡਿਪਲੋਇਮੈਂਟ ਰਣਨੀਤੀਆਂ ਵਰਤੋ
- **ਭਰੋਸੇਯੋਗ ਆਰਟੀਫੈਕਟ ਰਿਪੋਜ਼ਿਟਰੀਜ਼**: ਸਿਰਫ਼ ਜਾਂਚੇ ਹੋਏ, ਸੁਰੱਖਿਅਤ ਆਰਟੀਫੈਕਟ ਰਜਿਸਟਰੀਜ਼ ਅਤੇ ਰਿਪੋਜ਼ਿਟਰੀਜ਼ ਤੋਂ ਡਿਪਲੋਇ ਕਰੋ

#### **ਲਗਾਤਾਰ ਨਿਗਰਾਨੀ ਅਤੇ ਪ੍ਰਤੀਕਿਰਿਆ**
- **ਨਿਰਭਰਤਾ ਸਕੈਨਿੰਗ**: ਸਾਰੇ ਸਾਫਟਵੇਅਰ ਅਤੇ AI ਘਟਕ ਨਿਰਭਰਤਾਵਾਂ ਲਈ ਲਗਾਤਾਰ ਕਮਜ਼ੋਰੀ ਨਿਗਰਾਨੀ
- **ਮਾਡਲ ਨਿਗਰਾਨੀ**: ਮਾਡਲ ਦੇ ਵਿਹਾਰ, ਪ੍ਰਦਰਸ਼ਨ ਡ੍ਰਿਫਟ ਅਤੇ ਸੁਰੱਖਿਆ ਅਸਧਾਰਨਤਾਵਾਂ ਦੀ ਲਗਾਤਾਰ ਮੁਲਾਂਕਣ
- **ਸੇਵਾ ਸਿਹਤ ਟ੍ਰੈਕਿੰਗ**: ਬਾਹਰੀ AI ਸੇਵਾਵਾਂ ਦੀ ਉਪਲਬਧਤਾ, ਸੁਰੱਖਿਆ ਘਟਨਾਵਾਂ ਅਤੇ ਨੀਤੀ ਬਦਲਾਅ ਦੀ ਨਿਗਰਾਨੀ
- **ਖ਼ਤਰਾ ਬੁੱਧੀ ਇੰਟਿਗ੍ਰੇਸ਼ਨ**: AI ਅਤੇ ML ਸੁਰੱਖਿਆ ਖਤਰਿਆਂ ਲਈ ਖ਼ਤਰਾ ਫੀਡ ਸ਼ਾਮਲ ਕਰੋ

#### **ਪਹੁੰਚ ਨਿਯੰਤਰਣ ਅਤੇ ਘੱਟੋ-ਘੱਟ ਅਧਿਕਾਰ**
- **ਘਟਕ-ਸਤਰ ਅਧਿਕਾਰ**: ਕਾਰੋਬਾਰੀ ਜ਼ਰੂਰਤ ਅਨੁਸਾਰ ਮਾਡਲ, ਡਾਟਾ ਅਤੇ ਸੇਵਾਵਾਂ ਦੀ ਪਹੁੰਚ ਸੀਮਿਤ ਕਰੋ
- **ਸੇਵਾ ਖਾਤਾ ਪ੍ਰਬੰਧਨ**: ਘੱਟੋ-ਘੱਟ ਲੋੜੀਂਦੇ ਅਧਿਕਾਰਾਂ ਵਾਲੇ ਸਮਰਪਿਤ ਸੇਵਾ ਖਾਤੇ ਲਾਗੂ ਕਰੋ
- **ਨੈੱਟਵਰਕ ਵੰਡ**: AI ਘਟਕਾਂ ਨੂੰ ਅਲੱਗ ਕਰੋ ਅਤੇ ਸੇਵਾਵਾਂ ਵਿਚਕਾਰ ਨੈੱਟਵਰਕ ਪਹੁੰਚ ਸੀਮਿਤ ਕਰੋ
- **API ਗੇਟਵੇ ਨਿਯੰਤਰਣ**: ਬਾਹਰੀ AI ਸੇਵਾਵਾਂ ਲਈ ਕੇਂਦਰੀਕ੍ਰਿਤ API ਗੇਟਵੇਜ਼ ਵਰਤ ਕੇ ਪਹੁੰਚ ਨਿਯੰਤਰਿਤ ਅਤੇ ਨਿਗਰਾਨੀ ਕਰੋ

#### **ਘਟਨਾ ਪ੍ਰਤੀਕਿਰਿਆ ਅਤੇ ਪੁਨਰ ਪ੍ਰਾਪਤੀ**
- **ਤੇਜ਼ ਪ੍ਰਤੀਕਿਰਿਆ ਪ੍ਰਕਿਰਿਆਵਾਂ**: ਸਮਝੌਤਾ ਕੀਤੇ AI ਘਟਕਾਂ ਨੂੰ ਪੈਚ ਕਰਨ ਜਾਂ ਬਦਲਣ ਲਈ ਸਥਾਪਿਤ ਪ੍ਰਕਿਰਿਆਵਾਂ
- **ਪ੍ਰਮਾਣ ਪੱਤਰ ਘੁੰਮਾਅ**: ਰਾਜ, API ਕੁੰਜੀਆਂ ਅਤੇ ਸੇਵਾ ਪ੍ਰਮਾਣ ਪੱਤਰਾਂ ਨੂੰ ਘੁੰਮਾਉਣ ਲਈ ਆਟੋਮੇਟਿਡ ਪ੍ਰਣਾਲੀਆਂ
- **ਰੋਲਬੈਕ ਸਮਰੱਥਾ**: AI ਘਟਕਾਂ ਦੇ ਪਹਿਲਾਂ ਜਾਣੇ-ਪਹਚਾਣੇ-ਚੰਗੇ ਵਰਜਨਾਂ 'ਤੇ ਤੇਜ਼ੀ ਨਾਲ ਵਾਪਸ ਜਾਣ ਦੀ ਸਮਰੱਥਾ
- **ਸਪਲਾਈ ਚੇਨ ਬ੍ਰੀਚ ਪੁਨਰ ਪ੍ਰਾਪਤੀ**: ਅੱਪਸਟਰੀਮ AI ਸੇਵਾ ਸਮਝੌਤਿਆਂ ਲਈ ਖ਼ਾਸ ਪ੍ਰਤੀਕਿਰਿਆ ਪ੍ਰਕਿਰਿਆਵਾਂ

### ਮਾਈਕ੍ਰੋਸਾਫਟ ਸੁਰੱਖਿਆ ਟੂਲ ਅਤੇ ਇੰਟਿਗ੍ਰੇਸ਼ਨ

**GitHub Advanced Security** ਵਿਆਪਕ ਸਪਲਾਈ ਚੇਨ ਸੁਰੱਖਿਆ ਪ੍ਰਦਾਨ ਕਰਦਾ ਹੈ ਜਿਸ ਵਿੱਚ ਸ਼ਾਮਲ ਹਨ:
- **ਸੀਕ੍ਰੇਟ ਸਕੈਨਿੰਗ**: ਰਿਪੋਜ਼ਿਟਰੀਜ਼ ਵਿੱਚ ਪ੍ਰਮਾਣ ਪੱਤਰ, API ਕੁੰਜੀਆਂ ਅਤੇ ਟੋਕਨਾਂ ਦੀ ਆਟੋਮੇਟਿਡ ਪਹਿਚਾਣ
- **ਨਿਰਭਰਤਾ ਸਕੈਨਿੰਗ**: ਖੁੱਲ੍ਹੇ ਸਰੋਤ ਨਿਰਭਰਤਾਵਾਂ ਅਤੇ ਲਾਇਬ੍ਰੇਰੀਆਂ ਲਈ ਕਮਜ਼ੋਰੀ ਮੁਲਾਂਕਣ
- **CodeQL ਵਿਸ਼ਲੇਸ਼ਣ**: ਸੁਰੱਖਿਆ ਕਮਜ਼ੋਰੀਆਂ ਅਤੇ ਕੋਡਿੰਗ ਸਮੱਸਿਆਵਾਂ ਲਈ ਸਥਿਰ ਕੋਡ ਵਿਸ਼ਲੇਸ਼ਣ
- **ਸਪਲਾਈ ਚੇਨ ਇਨਸਾਈਟਸ**: ਨਿਰਭਰਤਾ ਸਿਹਤ ਅਤੇ ਸੁਰੱਖਿਆ ਸਥਿਤੀ ਵਿੱਚ ਦ੍ਰਿਸ਼ਟੀ

**Azure DevOps ਅਤੇ Azure Repos ਇੰਟਿਗ੍ਰੇਸ਼ਨ:**
- ਮਾਈਕ੍ਰੋਸਾਫਟ ਵਿਕਾਸ ਪਲੇਟਫਾਰਮਾਂ 'ਤੇ ਸੁਰੱਖਿਆ ਸਕੈਨਿੰਗ ਦਾ ਸਹਿਜ ਇੰਟਿਗ੍ਰੇਸ਼ਨ
- AI ਵਰਕਲੋਡ ਲਈ Azure ਪਾਈਪਲਾਈਨਾਂ ਵਿੱਚ ਆਟੋਮੇਟਿਡ ਸੁਰੱਖਿਆ ਜਾਂਚ
- ਸੁਰੱਖਿਅਤ AI ਘਟਕ ਡਿਪਲੋਇਮੈਂਟ ਲਈ ਨੀਤੀ ਲਾਗੂ ਕਰਨਾ

**ਮਾਈਕ੍ਰੋਸਾਫਟ ਅੰਦਰੂਨੀ ਅਭਿਆਸ:**
ਮਾਈਕ੍ਰੋਸਾਫਟ ਸਾਰੇ ਉਤਪਾਦਾਂ ਵਿੱਚ ਵਿਆਪਕ ਸਪਲਾਈ ਚੇਨ ਸੁਰੱਖਿਆ ਅਭਿਆਸ ਲਾਗੂ ਕਰਦਾ ਹੈ। [The Journey to Secure the Software Supply Chain at Microsoft](https://devblogs.microsoft.com/engineering-at-microsoft/the-journey-to-secure-the-software-supply-chain-at-microsoft/) ਵਿੱਚ ਪ੍ਰਮਾਣਿਤ ਤਰੀਕਿਆਂ ਬਾਰੇ ਜਾਣੋ।


## ਫਾਊਂਡੇਸ਼ਨ ਸੁਰੱਖਿਆ ਸਰਵੋਤਮ ਅਭਿਆਸ

MCP ਲਾਗੂ ਕਰਨ ਵਾਲੇ ਤੁਹਾਡੇ ਸੰਗਠਨ ਦੀ ਮੌਜੂਦਾ ਸੁਰੱਖਿਆ ਸਥਿਤੀ ਨੂੰ ਵਿਰਾਸਤ ਵਿੱਚ ਲੈਂਦੇ ਹਨ ਅਤੇ ਉਸ ਨੂੰ ਮਜ਼ਬੂਤ ਕਰਦੇ ਹਨ। ਮੂਲ ਸੁਰੱਖਿਆ ਅਭਿਆਸਾਂ ਨੂੰ ਮਜ਼ਬੂਤ ਕਰਨਾ AI ਸਿਸਟਮਾਂ ਅਤੇ MCP ਡਿਪਲੋਇਮੈਂਟ ਦੀ ਕੁੱਲ ਸੁਰੱਖਿਆ ਨੂੰ ਬਹੁਤ ਬਹਿਤਰ ਬਣਾਉਂਦਾ ਹੈ।

### ਮੁੱਖ ਸੁਰੱਖਿਆ ਮੂਲ

#### **ਸੁਰੱਖਿਅਤ ਵਿਕਾਸ ਅਭਿਆਸ**
- **OWASP ਅਨੁਕੂਲਤਾ**: [OWASP Top 10](https://owasp.org/www-project-top-ten/) ਵੈੱਬ ਐਪਲੀਕੇਸ਼ਨ ਕਮਜ਼ੋਰੀਆਂ ਤੋਂ ਸੁਰੱਖਿਆ
- **AI-ਵਿਸ਼ੇਸ਼ ਸੁਰੱਖਿਆ**: [OWASP Top 10 for LLMs](https://genai.owasp.org/download/43299/?tmstv=1731900559) ਲਈ ਨਿਯੰਤਰਣ ਲਾਗੂ ਕਰੋ
- **ਸੁਰੱਖਿਅਤ ਸੀਕ੍ਰੇਟ ਪ੍ਰਬੰਧਨ**: ਟੋਕਨ, API ਕੁੰਜੀਆਂ ਅਤੇ ਸੰਵੇਦਨਸ਼ੀਲ ਸੰਰਚਨਾ ਡਾਟਾ ਲਈ ਸਮਰਪਿਤ ਵੌਲਟ ਵਰਤੋ
- **ਐਂਡ-ਟੂ-ਐਂਡ ਇਨਕ੍ਰਿਪਸ਼ਨ**: ਸਾਰੇ ਐਪਲੀਕੇਸ਼ਨ ਘਟਕਾਂ ਅਤੇ ਡਾਟਾ ਫਲੋਜ਼ ਵਿੱਚ ਸੁਰੱਖਿਅਤ ਸੰਚਾਰ ਲਾਗੂ ਕਰੋ
- **ਇਨਪੁੱਟ ਵੈਧਤਾ**: ਸਾਰੇ ਉਪਭੋਗਤਾ ਇਨਪੁੱਟ, API ਪੈਰਾਮੀਟਰ ਅਤੇ ਡਾਟਾ ਸਰੋਤਾਂ ਦੀ ਕੜੀ ਜਾਂਚ

#### **ਇੰਫਰਾਸਟਰੱਕਚਰ ਮਜ਼ਬੂਤੀ**
- **ਮਲਟੀ-ਫੈਕਟਰ ਪ੍ਰਮਾਣਿਕਤਾ**: ਸਾਰੇ ਪ੍ਰਸ਼ਾਸਕੀ ਅਤੇ ਸੇਵਾ ਖਾਤਿਆਂ ਲਈ ਜ਼ਰੂਰੀ MFA
- **ਪੈਚ ਪ੍ਰਬੰਧਨ**: ਓਪਰੇਟਿੰਗ ਸਿਸਟਮ, ਫਰੇਮਵਰਕ ਅਤੇ ਨਿਰਭਰਤਾਵਾਂ ਲਈ ਆਟੋਮੇਟਿਡ, ਸਮੇਂ-ਸਿਰ ਪੈਚਿੰਗ  
- **ਪਹਿਚਾਣ ਪ੍ਰਦਾਤਾ ਇੰਟਿਗ੍ਰੇਸ਼ਨ**: ਐਂਟਰਪ੍ਰਾਈਜ਼ ਪਹਿਚਾਣ ਪ੍ਰਦਾਤਾਵਾਂ (Microsoft Entra ID, Active Directory) ਰਾਹੀਂ ਕੇਂਦਰੀਕ੍ਰਿਤ ਪਹਿਚਾਣ ਪ੍ਰਬੰਧਨ
- **ਨੈੱਟਵਰਕ ਵੰਡ**: MCP ਘਟਕਾਂ ਦੀ ਤਰਕਸ਼ੀਲ ਅਲੱਗ-ਅਲੱਗਤਾ ਤਾਂ ਜੋ ਲੈਟਰਲ ਮੂਵਮੈਂਟ ਸੀਮਿਤ ਰਹੇ
- **ਘੱਟੋ-ਘੱਟ ਅਧਿਕਾਰ ਦਾ ਸਿਧਾਂਤ**: ਸਾਰੇ ਸਿਸਟਮ ਘਟਕਾਂ ਅਤੇ ਖਾਤਿਆਂ ਲਈ ਘੱਟੋ-ਘੱਟ ਲੋੜੀਂਦੇ ਅਧਿਕਾਰ

#### **ਸੁਰੱਖਿਆ ਨਿਗਰਾਨੀ ਅਤੇ ਪਛਾਣ**
- **ਵਿਆਪਕ ਲੌਗਿੰਗ**: AI ਐਪਲੀਕੇਸ਼ਨ ਗਤੀਵਿਧੀਆਂ ਸਮੇਤ MCP ਕਲਾਇੰਟ-ਸਰਵਰ ਇੰਟਰੈਕਸ਼ਨਾਂ ਦੀ ਵਿਸਥਾਰਪੂਰਵਕ ਲੌਗਿੰਗ
- **SIEM ਇੰਟਿਗ੍ਰੇਸ਼ਨ**: ਵਿਸ਼ੇਸ਼ ਘਟਨਾਵਾਂ ਦੀ ਪਛਾਣ ਲਈ ਕੇਂਦਰੀਕ੍ਰਿਤ ਸੁਰੱਖਿਆ ਜਾਣਕਾਰੀ ਅਤੇ ਘਟਨਾ ਪ੍ਰਬੰਧਨ
- **ਵਿਹਾਰਕ ਵਿਸ਼ਲੇਸ਼ਣ**: ਸਿਸਟਮ ਅਤੇ ਉਪਭੋਗਤਾ ਵਿਹਾਰ ਵਿੱਚ ਅਸਧਾਰਨ ਪੈਟਰਨ ਪਛਾਣਣ ਲਈ AI-ਚਾਲਿਤ ਨਿਗਰਾਨੀ
- **ਖ਼ਤਰਾ ਬੁੱਧੀ**: ਬਾਹਰੀ ਖ਼ਤਰਾ ਫੀਡ ਅਤੇ ਸਮਝੌਤਾ ਸੂਚਕਾਂਕਾਂ (IOCs) ਦਾ ਇੰਟਿਗ੍ਰੇਸ਼ਨ
- **ਘਟਨਾ ਪ੍ਰਤੀਕਿਰਿਆ**: ਸੁਰੱਖਿਆ ਘਟਨਾ ਦੀ ਪਛਾਣ, ਪ੍ਰਤੀਕਿਰਿਆ ਅਤੇ ਪੁਨਰ ਪ੍ਰਾਪਤੀ ਲਈ ਵਧੀਆ ਪਰਿਭਾਸ਼ਿਤ ਪ੍ਰਕਿਰਿਆਵਾਂ

#### **ਜ਼ੀਰੋ ਟਰੱਸਟ ਆਰਕੀਟੈਕਚਰ**
- **ਕਦੇ ਭਰੋਸਾ ਨਾ ਕਰੋ, ਹਮੇਸ਼ਾ ਜਾਂਚੋ**: ਉਪਭੋਗਤਾਵਾਂ, ਡਿਵਾਈਸਾਂ ਅਤੇ ਨੈੱਟਵਰਕ ਕਨੈਕਸ਼ਨਾਂ ਦੀ ਲਗਾਤਾਰ ਜਾਂਚ
- **ਮਾਈਕ੍ਰੋ-ਸੈਗਮੈਂਟੇਸ਼ਨ**: ਵਿਅਕਤੀਗਤ ਵਰਕਲੋਡ ਅਤੇ ਸੇਵਾਵਾਂ ਨੂੰ ਅਲੱਗ ਕਰਨ ਵਾਲੇ ਨੈੱਟਵਰਕ ਨਿਯੰਤਰਣ
- **ਪਹਿਚਾਣ-ਕੇਂਦਰਿਤ ਸੁਰੱਖਿਆ**: ਨੈੱਟਵਰਕ ਸਥਿਤੀ ਦੀ ਬਜਾਏ ਪੁਸ਼ਟੀ ਕੀਤੀਆਂ ਪਹਿਚਾਣਾਂ 'ਤੇ ਆਧਾਰਿਤ ਸੁਰੱਖਿਆ ਨੀਤੀਆਂ
- **ਲਗਾਤਾਰ ਖ਼ਤਰਾ ਮੁਲਾਂਕਣ**: ਮੌਜੂਦਾ ਸੰਦਰਭ ਅਤੇ ਵਿਹਾਰ ਦੇ ਆਧਾਰ 'ਤੇ ਗਤੀਸ਼ੀਲ ਸੁਰੱਖਿਆ ਸਥਿਤੀ ਦਾ ਮੁਲਾਂਕਣ
- **ਸ਼ਰਤੀ ਪਹੁੰਚ**: ਖ਼ਤਰਾ ਕਾਰਕਾਂ, ਸਥਾਨ ਅਤੇ ਡਿਵਾਈਸ ਭਰੋਸੇ ਦੇ ਆਧਾਰ 'ਤੇ ਅਨੁਕੂਲ ਪਹੁੰਚ ਨਿਯੰਤਰਣ

### ਐਂਟਰਪ੍ਰਾਈਜ਼ ਇੰਟਿਗ੍ਰੇਸ਼ਨ ਪੈਟਰਨ

#### **ਮਾਈਕ੍ਰੋਸਾਫਟ ਸੁਰੱਖਿਆ ਪਰਿਸਰ ਇੰਟਿਗ੍ਰੇਸ਼ਨ**
- **Microsoft Defender for Cloud**: ਵਿਆਪਕ ਕਲਾਉਡ ਸੁਰੱਖਿਆ ਸਥਿਤੀ ਪ੍ਰਬੰਧਨ
- **Azure Sentinel**: AI ਵਰਕਲੋਡ ਸੁਰੱਖਿਆ ਲਈ ਕਲਾਉਡ-ਮੂਲ SIEM ਅਤੇ SOAR ਸਮਰੱਥਾ
- **Microsoft Entra ID**: ਸ਼ਰਤੀ ਪਹੁੰਚ ਨੀਤੀਆਂ ਨਾਲ ਐਂਟਰਪ੍ਰਾਈਜ਼ ਪਹਿਚਾਣ ਅਤੇ ਪਹੁੰਚ ਪ੍ਰਬੰਧਨ
- **Azure Key Vault**: ਹਾਰਡਵੇਅਰ ਸੁਰੱਖਿਆ ਮਾਡਿਊਲ (HSM) ਸਮਰਥਿਤ ਕੇਂਦਰੀਕ੍ਰਿਤ ਸੀਕ੍ਰੇਟ ਪ੍ਰਬੰਧਨ
- **Microsoft Purview**: AI ਡਾਟਾ ਸਰੋਤਾਂ ਅਤੇ ਵਰਕਫਲੋਜ਼ ਲਈ ਡਾਟਾ ਗਵਰਨੈਂਸ ਅਤੇ ਅਨੁਕੂਲਤਾ

#### **ਅਨੁਕੂਲਤਾ ਅਤੇ ਸ਼ਾਸਨ**
- **ਨਿਯਮਕ ਅਨੁਕੂਲਤਾ**: MCP ਲਾਗੂ ਕਰਨ ਵਾਲੇ ਉਦਯੋਗ-ਵਿਸ਼ੇਸ਼ ਅਨੁਕੂਲਤਾ ਲੋੜਾਂ (GDPR, HIPAA, SOC 2) ਨੂੰ ਪੂਰਾ ਕਰਨ
- **ਡਾਟਾ ਵਰਗੀਕਰਨ**: AI ਸਿਸਟਮਾਂ ਦੁਆਰਾ ਪ੍ਰੋਸੈਸ ਕੀਤੇ ਸੰਵੇਦਨਸ਼ੀਲ ਡਾਟਾ ਦੀ ਠੀਕ ਵਰਗੀਕਰਨ ਅਤੇ ਸੰਭਾਲ
- **ਆਡਿਟ ਟ੍ਰੇਲ**: ਨਿਯਮਕ ਅਨੁਕੂਲਤਾ ਅਤੇ ਫੋਰੈਂਸਿਕ ਜਾਂਚ ਲਈ ਵਿਆਪਕ ਲੌਗਿੰਗ
- **ਪ੍ਰਾਈਵੇਸੀ ਨਿਯੰਤਰਣ**: AI ਸਿਸਟਮ ਆਰਕੀਟੈਕਚਰ ਵਿੱਚ ਪ੍ਰਾਈਵੇਸੀ-ਬਾਈ-ਡਿਜ਼ਾਈਨ ਸਿਧਾਂਤਾਂ ਦੀ ਲਾਗੂਆਤ
- **ਬਦਲਾਅ ਪ੍ਰਬੰਧਨ**: AI ਸਿਸਟਮ ਬਦਲਾਅ ਦੀ ਸੁਰੱਖਿਆ ਸਮੀਖਿਆ ਲਈ ਰਸਮੀ ਪ੍ਰਕਿਰਿਆਵਾਂ

ਇਹ ਮੂਲ ਅਭਿਆਸ ਇੱਕ ਮਜ਼ਬੂਤ ਸੁਰੱਖਿਆ ਬੇਸਲਾਈਨ ਬਣਾਉਂਦੇ ਹਨ ਜੋ MCP-ਵਿਸ਼ੇਸ਼ ਸੁਰੱਖਿਆ ਨਿਯੰਤਰਣਾਂ ਦੀ ਪ੍ਰਭਾਵਸ਼ੀਲਤਾ ਨੂੰ ਵਧਾਉਂਦੇ ਹਨ ਅਤੇ AI-ਚਲਿਤ ਐਪਲੀਕੇਸ਼ਨਾਂ ਲਈ ਵਿਆਪਕ ਸੁਰੱਖਿਆ ਪ੍ਰਦਾਨ ਕਰਦੇ ਹਨ।

## ਮੁੱਖ ਸੁਰੱਖਿਆ ਸਿੱਖਣੀਆਂ

- **ਪੜਤਾਲੀ ਸੁਰੱਖਿਆ ਦ੍ਰਿਸ਼ਟੀਕੋਣ**: ਮੂਲ ਸੁਰੱਖਿਆ ਅਭਿਆਸਾਂ (ਸੁਰੱਖਿਅਤ ਕੋਡਿੰਗ, ਘੱਟੋ-ਘੱਟ ਅਧਿਕਾਰ, ਸਪਲਾਈ ਚੇਨ ਜਾਂਚ, ਲਗਾਤਾਰ ਨਿਗਰਾਨੀ) ਨੂੰ AI-ਵਿਸ਼ੇਸ਼ ਨਿਯੰਤਰਣਾਂ ਨਾਲ ਜੋੜ ਕੇ ਵਿਆਪਕ ਸੁਰੱਖਿਆ ਪ੍ਰਦਾਨ ਕਰੋ

- **AI-ਵਿਸ਼ੇਸ਼ ਖ਼ਤਰਾ ਦ੍ਰਿਸ਼ਟੀਕੋਣ**: MCP ਸਿਸਟਮਾਂ ਨੂੰ ਵਿਲੱਖਣ ਖਤਰਿਆਂ ਦਾ ਸਾਹਮਣਾ ਹੈ ਜਿਵੇਂ ਪ੍ਰਾਂਪਟ ਇੰਜੈਕਸ਼ਨ, ਟੂਲ ਜਹਿਰਲਾਪਣ, ਸੈਸ਼ਨ ਹਾਈਜੈਕਿੰਗ, ਗੁੰਝਲਦਾਰ ਡਿਪਟੀ ਸਮੱਸਿਆਵਾਂ, ਟੋਕਨ ਪਾਸਥਰੂ ਕਮਜ਼ੋਰੀਆਂ ਅਤੇ ਅਤਿਰਿਕਤ ਅਧਿਕਾਰ ਜੋ ਵਿਸ਼ੇਸ਼ ਰੋਕਥਾਮ ਮੰਗਦੇ ਹਨ

- **ਪ੍ਰਮਾਣਿਕਤਾ ਅਤੇ ਅਧਿਕਾਰਤਾ ਵਿੱਚ ਸ਼੍ਰੇਸ਼ਠਤਾ**: ਬਾਹਰੀ ਪਹਿਚਾਣ ਪ੍ਰਦਾਤਾਵਾਂ (Microsoft Entra ID) ਦੀ ਵਰਤੋਂ ਨਾਲ ਮਜ਼ਬੂਤ ਪ੍ਰਮਾਣਿਕਤਾ ਲਾਗੂ ਕਰੋ, ਸਹੀ ਟੋਕਨ ਜਾਂਚ ਲਾਗੂ ਕਰੋ ਅਤੇ ਕਦੇ ਵੀ ਉਹ ਟੋਕਨ ਨਾ ਸਵੀਕਾਰੋ ਜੋ ਖਾਸ ਤੌਰ 'ਤੇ ਤੁਹਾਡੇ MCP ਸਰਵਰ ਲਈ ਜਾਰੀ ਨਾ ਕੀਤੇ ਗਏ ਹੋਣ

- **AI ਹਮਲੇ ਰੋਕਥਾਮ**: Microsoft Prompt Shields ਅਤੇ Azure Content Safety ਨੂੰ ਅਪਰੋਕਸ਼ ਪ੍ਰਾਂਪਟ ਇੰਜੈਕਸ਼ਨ ਅਤੇ ਟੂਲ ਜਹਿਰਲਾਪਣ ਹਮਲਿਆਂ ਤੋਂ ਬਚਾਅ ਲਈ ਤੈਨਾਤ ਕਰੋ, ਜਦਕਿ ਟੂਲ ਮੈਟਾਡੇਟਾ ਦੀ ਜਾਂਚ ਅਤੇ ਗਤੀਸ਼ੀਲ ਬਦਲਾਅ ਦੀ ਨਿਗਰਾਨੀ ਕਰੋ

- **ਸੈਸ਼ਨ ਅਤੇ ਟ੍ਰਾਂਸਪੋਰਟ ਸੁਰੱਖਿਆ**: ਕ੍ਰਿਪਟੋਗ੍ਰਾਫਿਕ ਤੌਰ 'ਤੇ ਸੁਰੱਖਿਅਤ, ਗੈਰ-ਨਿਰਧਾਰਿਤ ਸੈਸ਼ਨ ID ਵਰਤੋ ਜੋ ਉਪਭੋਗਤਾ ਪਹਿਚਾਣ ਨਾਲ ਜੁੜੇ ਹੋਣ, ਸਹੀ ਸੈਸ਼ਨ ਜੀਵਨਚੱਕਰ ਪ੍ਰਬੰਧਨ ਲਾਗੂ ਕਰੋ ਅਤੇ ਕਦੇ ਵੀ ਪ੍ਰਮਾਣਿਕਤਾ ਲਈ ਸੈਸ਼ਨਾਂ ਦੀ ਵਰਤੋਂ ਨਾ ਕਰੋ

- **OAuth ਸੁਰੱਖਿਆ ਸਰਵੋਤਮ ਅਭਿਆਸ**: ਗਤੀਸ਼ੀਲ ਰਜਿਸਟਰਡ ਕਲਾਇੰਟਾਂ ਲਈ ਸਪਸ਼ਟ ਉਪਭੋਗਤਾ ਸਹਿਮਤੀ, PKCE ਸਮੇਤ ਠੀਕ OAuth 2.1 ਲਾਗੂਆਤ ਅਤੇ ਸਖ਼ਤ ਰੀਡਾਇਰੈਕਟ URI ਜਾਂਚ ਰਾਹੀਂ ਗੁੰਝਲਦਾਰ ਡਿਪਟੀ ਹਮਲਿਆਂ ਨੂੰ ਰੋਕੋ  

- **ਟੋਕਨ ਸੁਰੱਖਿਆ ਸਿਧਾਂਤ**: ਟੋਕਨ ਪਾਸਥਰੂ ਵਿਰੋਧੀ-ਪੈਟਰਨ ਤੋਂ ਬਚੋ, ਟੋਕਨ ਦਰਸ਼ਕ ਦਾਅਵਿਆਂ ਦੀ ਜਾਂਚ ਕਰੋ, ਛੋਟੇ ਸਮੇਂ ਵਾਲੇ ਟੋਕਨ ਸੁਰੱਖਿਅਤ ਘੁੰਮਾਅ ਨਾਲ ਲਾਗੂ ਕਰੋ ਅਤੇ ਸਾਫ਼ ਭਰੋਸਾ ਸੀਮਾਵਾਂ ਬਣਾਈ ਰੱਖੋ

- **ਵਿਆਪਕ ਸਪਲਾਈ ਚੇਨ ਸੁਰੱਖਿਆ**: ਸਾਰੇ AI ਪਰਿਸਰ ਘਟਕਾਂ (ਮਾਡਲ, ਐਂਬੈਡਿੰਗ, ਸੰਦਰਭ ਪ੍ਰਦਾਤਾ, ਬਾਹਰੀ API) ਨੂੰ ਰਵਾਇਤੀ ਸਾਫਟਵੇਅਰ ਨਿਰਭਰਤਾਵਾਂ ਵਰਗੇ ਹੀ ਸਖ਼ਤ ਸੁਰੱਖਿਆ ਨਾਲ ਸੰਭਾਲੋ

- **ਲਗਾਤਾਰ ਵਿਕਾਸ**: ਤੇਜ਼ੀ ਨਾਲ ਬਦਲ ਰਹੀਆਂ MCP ਵਿਸ਼ੇਸ਼ਤਾਵਾਂ ਨਾਲ ਅਪ-ਟੂ-ਡੇਟ ਰਹੋ, ਸੁਰੱਖਿਆ ਕਮਿਊਨਿਟੀ ਮਿਆਰਾਂ ਵਿੱਚ ਯੋਗਦਾਨ ਪਾਓ ਅਤੇ ਪ੍ਰੋਟੋਕੋਲ ਦੇ ਵਿਕਾਸ ਦੇ ਨਾਲ ਅਨੁਕੂਲ ਸੁਰੱਖਿਆ ਸਥਿਤੀ ਬਣਾਈ ਰੱਖੋ

- **ਮਾਈਕ੍ਰੋਸਾਫਟ ਸੁਰੱਖਿਆ ਇੰਟਿਗ੍ਰੇਸ਼ਨ**: MCP ਡਿਪਲੋਇਮੈਂਟ ਸੁਰੱਖਿਆ ਵਧਾਉਣ ਲਈ Microsoft ਦੇ ਵਿਆਪਕ ਸੁਰੱਖਿਆ ਪਰਿਸਰ (Prompt Shields, Azure Content Safety, GitHub Advanced Security, Entra ID) ਦਾ ਲਾਭ ਉਠਾਓ
