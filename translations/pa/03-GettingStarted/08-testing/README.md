## ਟੈਸਟਿੰਗ ਅਤੇ ਡੀਬੱਗਿੰਗ

ਤੁਹਾਡੇ MCP ਸਰਵਰ ਦੀ ਟੈਸਟਿੰਗ ਸ਼ੁਰੂ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ, ਮੌਜੂਦਾ ਉਪਲੱਬਧ ਟੂਲਾਂ ਅਤੇ ਡੀਬੱਗਿੰਗ ਲਈ ਸਰਵੋਤਮ ਅਭਿਆਸਾਂ ਨੂੰ ਸਮਝਣਾ ਮਹੱਤਵਪੂਰਨ ਹੈ। ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਟੈਸਟਿੰਗ ਇਹ ਯਕੀਨੀ ਬਣਾਉਂਦੀ ਹੈ ਕਿ ਤੁਹਾਡਾ ਸਰਵਰ ਉਮੀਦਅਨੁਸਾਰ ਕੰਮ ਕਰ ਰਿਹਾ ਹੈ ਅਤੇ ਇਹ ਤੁਹਾਨੂੰ ਜਲਦ ਹੀ ਸਮੱਸਿਆਵਾਂ ਦੀ ਪਛਾਣ ਅਤੇ ਸਮਾਧਾਨ ਕਰਨ ਵਿੱਚ ਮਦਦ ਕਰਦੀ ਹੈ। ਹੇਠਾਂ ਦਿੱਤੇ ਸੈਕਸ਼ਨ ਵਿੱਚ ਤੁਹਾਡੀ MCP ਅਮਲਦਾਰਤਾ ਨੂੰ ਸਹੀ ਢੰਗ ਨਾਲ ਜਾਂਚਣ ਲਈ ਸੁਝਾਏ ਗਏ ਤਰੀਕੇ ਬਿਆਨ ਕੀਤੇ ਗਏ ਹਨ।

## ਝਲਕ

ਇਹ ਪਾਠ ਇਹ ਕਿਵੇਂ ਚੁਣਨਾ ਹੈ ਕਿ ਕਿਹੜਾ ਟੈਸਟਿੰਗ ਅਭਿਗਮ ਸਹੀ ਹੈ ਅਤੇ ਸਭ ਤੋਂ ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਟੈਸਟਿੰਗ ਟੂਲ ਕਿਹੜਾ ਹੈ, ਤੇ ਧਿਆਨ ਦੇਂਦਾ ਹੈ।

## ਸਿੱਖਣ ਦੇ ਉਦੇਸ਼

ਇਸ ਪਾਠ ਦੇ ਅੰਤ ਤੱਕ, ਤੁਸੀਂ ਸਮਰੱਥ ਹੋਵੋਗੇ:

- ਟੈਸਟਿੰਗ ਲਈ ਵੱਖ-ਵੱਖ ਅਭਿਗਮਾਂ ਨੂੰ ਵਰਣਨ ਕਰਨ ਵਿੱਚ।
- ਆਪਣੇ ਕੋਡ ਦੀ ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਟੈਸਟਿੰਗ ਲਈ ਵੱਖ-ਵੱਖ ਟੂਲਾਂ ਦਾ ਉਪਯੋਗ ਕਰਨ ਵਿੱਚ।

## MCP ਸਰਵਰ ਦੀ ਟੈਸਟਿੰਗ

MCP ਤੁਹਾਡੇ ਲਈ ਟੈਸਟ ਅਤੇ ਡੀਬੱਗ ਕਰਨ ਵਾਲੇ ਟੂਲ ਪ੍ਰਦਾਨ ਕਰਦਾ ਹੈ:

- **MCP ਇੰਸਪੈਕਟਰ**: ਇਕ ਕਮਾਂਡ ਲਾਈਨ ਟੂਲ ਜੋ CLI ਟੂਲ ਵੱਜੋਂ ਅਤੇ ਵਿਜ਼ੂਅਲ ਟੂਲ ਵੱਜੋਂ ਚਲਾਇਆ ਜਾ ਸਕਦਾ ਹੈ।
- **ਹੱਥੀ ਟੈਸਟਿੰਗ**: ਤੁਸੀਂ curl ਵਰਗੇ ਟੂਲ ਦਾ ਉਪਯੋਗ ਕਰ ਸਕਦੇ ਹੋ ਜਿਹੜਾ ਵੈੱਬ ਰਿਕਵੈਸਟ ਚਲਾ ਸਕਦਾ ਹੈ, ਪਰ ਕੋਈ ਵੀ HTTP ਚਲਾਉਣ ਵਾਲਾ ਟੂਲ ਇਸ ਕੰਮ ਲਈ ਯੋਗ ਹੈ।
- **ਯੂਨਿਟ ਟੈਸਟਿੰਗ**: ਤुਹਾਡੇ ਮਨਪਸੰਦ ਟੈਸਟਿੰਗ ਫ੍ਰੇਮਵਰਕ ਨਾਲ ਇਸਦੀ ਖਾਸੀਅਤਾਂ ਦੀ ਜਾਂਚ ਕੀਤੀ ਜਾ ਸਕਦੀ ਹੈ ਜਿਸ ਨਾਲ ਸਰਵਰ ਅਤੇ ਕਲਾਇੰਟ ਦੋਹਾਂ ਦੀ ਪੁਸ਼ਟੀ ਹੋ ਸਕਦੀ ਹੈ।

### MCP ਇੰਸਪੈਕਟਰ ਦੀ ਵਰਤੋਂ

ਅਸੀਂ ਇਸ ਟੂਲ ਦੀ ਵਰਤੋਂ ਪਹਿਲਾਂ ਦੇ ਪਾਠਾਂ ਵਿੱਚ ਵਿਆਖਿਆ ਕੀਤੀ ਹੈ, ਪਰ ਆਓ ਇਸ ਬਾਰੇ ਥੋੜ੍ਹਾ ਉੱਚ ਪੁੱਛ ਦੇਖੀਏ। ਇਹ Node.js ਵਿੱਚ ਬਣਾਇਆ ਗਿਆ ਇੱਕ ਟੂਲ ਹੈ ਜਿਸਨੂੰ ਤੁਸੀਂ `npx` ਚਲਾਕੀ ਨਾਲ ਵਰਤ ਸਕਦੇ ਹੋ ਜੋ ਅਸਥਾਈ ਤੌਰ 'ਤੇ ਟੂਲ ਡਾਊਨਲੋਡ ਕਰਦਾ ਹੈ, ਇੰਸਟਾਲ ਕਰਦਾ ਹੈ ਅਤੇ ਤੁਹਾਡੇ ਬਿਨਤੀ ਨੂੰ ਚਲਾਉਣ ਤੋਂ ਬਾਅਦ ਆਪੋ-ਆਪ ਸਾਫ਼ ਕਰ ਲੈਂਦਾ ਹੈ।

[MCP ਇੰਸਪੈਕਟਰ](https://github.com/modelcontextprotocol/inspector) ਤੁਹਾਡੀ ਮਦਦ ਕਰਦਾ ਹੈ:

- **ਸਰਵਰ ਸਮਰੱਥਾਵਾਂ ਦੀ ਖੋਜ**: ਉਪਲੱਬਧ ਸਰੋਤ, ਟੂਲ ਅਤੇ ਪ੍ਰੰਪਟਾਂ ਨੂੰ ਆਪਣੇ ਆਪ ਪਤਾ ਲਗਾਉਣਾ
- **ਟੂਲ ਚਲਾਉਣ ਦੀ ਜਾਂਚ**: ਵੱਖ-ਵੱਖ ਪੈਰਾਮੀਟਰਾਂ ਨਾਲ ਕੋਸ਼ਿਸ਼ ਕਰੋ ਅਤੇ ਤਤਕਾਲ ਜਵਾਬ ਵੇਖੋ
- **ਸਰਵਰ ਮੈਟਾਡੇਟਾ ਵੇਖਣਾ**: ਸਰਵਰ ਜਾਣਕਾਰੀ, ਸਕੀਮਾ ਅਤੇ ਸੰਰਚਨਾਵਾਂ ਦੀ ਜਾਂਚ ਕਰੋ

ਇਕ ਆਮ ਚਲਾਉਣਾ ਇਸ ਤਰ੍ਹਾਂ ਦਿੱਸਦਾ ਹੈ:

```bash
npx @modelcontextprotocol/inspector node build/index.js
```

ਉਪਰੋਕਤ ਕਮਾਂਡ ਇੱਕ MCP ਅਤੇ ਇਸਦੀ ਵਿਜ਼ੂਅਲ ਇੰਟਰਫੇਸ ਸ਼ੁਰੂ ਕਰਦੀ ਹੈ ਅਤੇ ਤੁਹਾਡੇ ਬ੍ਰਾਊਜ਼ਰ ਵਿੱਚ ਸਥਾਨਕ ਵੈੱਬ ਇੰਟਰਫੇਸ ਖੋਲਦੀ ਹੈ। ਤੁਸੀਂ ਇੱਕ ਡੈਸ਼ਬੋਰਡ ਵੇਖ ਸਕਦੇ ਹੋ ਜਿਹੜਾ ਤੁਹਾਡੇ ਰਜਿਸਟਰਡ MCP ਸਰਵਰਾਂ, ਉਨ੍ਹਾਂ ਦੇ ਉਪਲੱਬਧ ਟੂਲਾਂ, ਸਰੋਤਾਂ ਅਤੇ ਪ੍ਰੰਪਟਾਂ ਨੂੰ ਦਰਸਾਉਂਦਾ ਹੈ। ਇਹ ਇੰਟਰਫੇਸ ਤੁਹਾਨੂੰ ਇੰਟਰਐਕਟਿਵ ਤਰੀਕੇ ਨਾਲ ਟੂਲ ਚਲਾਉਣ ਵਾਲੀ ਜਾਂਚ ਕਰਨ, ਸਰਵਰ ਮੈਟਾਡੇਟਾ ਨੂੰ ਜਾਂਚਣ ਅਤੇ ਤਤਕਾਲ ਜਵਾਬਾਂ ਨੂੰ ਵੇਖਣ ਦੀ ਸੁਵਿਧਾ ਦਿੰਦਾ ਹੈ, ਸੇ MCP ਸਰਵਰਾਂ ਦੀ ਪੁਸ਼ਟੀ ਅਤੇ ਡੀਬੱਗਿੰਗ ਕਰਨਾ ਸੌਖਾ ਬਣਾਉਂਦਾ ਹੈ।

ਇਸ ਤਰ੍ਹਾਂ ਦਿੱਸਦਾ ਹੈ: ![Inspector](../../../../translated_images/pa/connect.141db0b2bd05f096.webp)

ਤੁਸੀਂ ਇਸ ਟੂਲ ਨੂੰ CLI ਮੋਡ ਵਿੱਚ ਵੀ ਚਲਾ ਸਕਦੇ ਹੋ ਜਿਸ ਵਿੱਚ ਤੁਸੀਂ `--cli` ਐਟਰੀਬਿਊਟ ਸ਼ਾਮਲ ਕਰਦੇ ਹੋ। ਇੱਥੇ CLI ਮੋਡ ਵਿੱਚ ਟੂਲ ਚਲਾਉਣ ਦੀ ਉਦਾਹਰਨ ਹੈ ਜੋ ਸਰਵਰ ਤੇ ਸਾਰੇ ਟੂਲਾਂ ਦੀ ਸੂਚੀ ਦਿਖਾਂਦਾ ਹੈ:

```sh
npx @modelcontextprotocol/inspector --cli node build/index.js --method tools/list
```

### ਹੱਥੀ ਟੈਸਟਿੰਗ

ਸਰਵਰ ਸਮਰੱਥਾਵਾਂ ਦੀ ਜਾਂਚ ਕਰਨ ਲਈ ਇੰਸਪੈਕਟਰ ਟੂਲ ਚਲਾਉਣ ਤੋਂ ਇਲਾਵਾ, ਇੱਕ ਹੋਰ ਮਿਲਦਾ-ਜੁਲਦਾ ਤਰੀਕਾ ਹੈ HTTP ਚਲਾਉਣ ਵਾਲੇ ਕਲਾਇਂਟ ਜੋ ਕਿ curl ਵਰਗਾ ਹੋ ਸਕਦਾ ਹੈ।

curl ਨਾਲ, ਤੁਸੀਂ MCP ਸਰਵਰਾਂ ਨੂੰ ਸਿੱਧਾ HTTP ਰਿਕਵੈਸਟਾਂ ਰਾਹੀਂ ਜਾਂਚ ਸਕਦੇ ਹੋ:

```bash
# ਉਦਾਹਰਨ: ਟੈਸਟ ਸਰਵਰ ਮੈਟਾਡੇਟਾ
curl http://localhost:3000/v1/metadata

# ਉਦਾਹਰਨ: ਇੱਕ ਟੂਲ ਚਲਾਓ
curl -X POST http://localhost:3000/v1/tools/execute \
  -H "Content-Type: application/json" \
  -d '{"name": "calculator", "parameters": {"expression": "2+2"}}'
```

ਜਿਵੇਂ ਤੁਸੀਂ curl ਦੀ ਉਪਯੋਗਤਾ ਤੋਂ ਵੇਖ ਸਕਦੇ ਹੋ, ਤੁਸੀਂ ਇੱਕ POST ਬੇਨਤੀ ਭੇਜ ਕੇ ਟੂਲ ਨੂੰ ਇਸਦੇ ਨਾਮ ਅਤੇ ਪੈਰਾਮੀਟਰਾਂ ਸਮੇਤ payload ਦੇ ਕੇ ਕਾਲ ਕਰਦੇ ਹੋ। ਉਹ ਅਭਿਆਸ ਵਰਤੋ ਜੋ ਤੁਹਾਡੇ ਲਈ ਸਭ ਤੋਂ ਵਧੀਆ ਹੋਵੇ। CLI ਟੂਲ ਸਧਾਰਣ ਤੌਰ 'ਤੇ ਤੇਜ਼ ਹੁੰਦੇ ਹਨ ਅਤੇ ਝਲਕ ਉਪਯੋਗਿਤਾ ਲਈ ਸਕ੍ਰਿਪਟਿੰਗ ਮੇਲ ਖਾਂਦੇ ਹਨ ਜੋ CI/CD ਵਾਤਾਵਰਣ ਵਿੱਚ ਲਾਭਦਾਇਕ ਹੋ ਸਕਦਾ ਹੈ।

### ਯੂਨਿਟ ਟੈਸਟਿੰਗ

ਆਪਣੇ ਟੂਲਾਂ ਅਤੇ ਸਰੋਤਾਂ ਲਈ ਯੂਨਿਟ ਟੈਸਟ ਬਣਾਓ ਤਾਂ ਜੋ ਇਹ ਯਕੀਨੀ ਬਣਾ ਸਕੋ ਕਿ ਉਹ ਉਮੀਦ ਅਨੁਸਾਰ ਕੰਮ ਕਰਦੇ ਹਨ। ਇੱਥੇ ਕੁਝ ਉਦਾਹਰਨ ਹਨ ਜਿਹੜੇ ਟੈਸਟਿੰਗ ਕੋਡ ਦਿਖਾਉਂਦੇ ਹਨ।

```python
import pytest

from mcp.server.fastmcp import FastMCP
from mcp.shared.memory import (
    create_connected_server_and_client_session as create_session,
)

# ਪੂਰੇ ਮੋਡੀਊਲ ਨੂੰ ਅਸਿੰਕ ਟੈਸਟਾਂ ਲਈ ਨਿਸ਼ਾਨ ਲਗਾਓ
pytestmark = pytest.mark.anyio


async def test_list_tools_cursor_parameter():
    """Test that the cursor parameter is accepted for list_tools.

    Note: FastMCP doesn't currently implement pagination, so this test
    only verifies that the cursor parameter is accepted by the client.
    """

 server = FastMCP("test")

    # ਕੁਝ ਟੈਸਟ ਟੂਲ ਬਣਾਓ
    @server.tool(name="test_tool_1")
    async def test_tool_1() -> str:
        """First test tool"""
        return "Result 1"

    @server.tool(name="test_tool_2")
    async def test_tool_2() -> str:
        """Second test tool"""
        return "Result 2"

    async with create_session(server._mcp_server) as client_session:
        # ਕਰਸਰ ਪੈਰਾਮੀਟਰ ਦੇ ਬਿਨਾਂ ਟੈਸਟ ਕਰੋ (ਛੱਡ ਦਿੱਤਾ ਗਿਆ)
        result1 = await client_session.list_tools()
        assert len(result1.tools) == 2

        # ਕਰਸਰ=None ਦੇ ਨਾਲ ਟੈਸਟ ਕਰੋ
        result2 = await client_session.list_tools(cursor=None)
        assert len(result2.tools) == 2

        # ਕਰਸਰ ਨੂੰ ਸਟਰਿੰਗ ਵਜੋਂ ਟੈਸਟ ਕਰੋ
        result3 = await client_session.list_tools(cursor="some_cursor_value")
        assert len(result3.tools) == 2

        # ਖਾਲੀ ਸਟਰਿੰਗ ਕਰਸਰ ਦੇ ਨਾਲ ਟੈਸਟ ਕਰੋ
        result4 = await client_session.list_tools(cursor="")
        assert len(result4.tools) == 2
    
```

ਉਪਰੋਕਤ ਕੋਡ ਹੇਠ ਲਿਖਿਆ ਕੰਮ ਕਰਦਾ ਹੈ:

- pytest ਫ੍ਰੇਮਵਰਕ ਦੀ ਵਰਤੋਂ ਕਰਦਾ ਹੈ ਜੋ ਤੁਹਾਡੀ ਟੈਸਟਾਂ ਨੂੰ ਫੰਕਸ਼ਨਾਂ ਵਜੋਂ ਬਣਾਉਂਦਾ ਹੈ ਅਤੇ assert ਬਿਆਨਾਂ ਦੀ ਵਰਤੋਂ ਕਰਦਾ ਹੈ।
- ਦੋ ਵੱਖ-ਵੱਖ ਟੂਲਾਂ ਵਾਲਾ MCP ਸਰਵਰ ਬਣਾਉਂਦਾ ਹੈ।
- `assert` ਬਿਆਨ ਦੀ ਵਰਤੋਂ ਕਰਦਾ ਹੈ ਤਾਂ ਜੋ ਨਿਸ਼ਚਿਤ ਸ਼ਰਤਾਂ ਦੀ ਪੁਸ਼ਟੀ ਕਰ ਸਕੇ।

[ਪੂਰੇ ਫਾਈਲ ਨੂੰ ਇੱਥੇ ਵੇਖੋ](https://github.com/modelcontextprotocol/python-sdk/blob/main/tests/client/test_list_methods_cursor.py)

ਉਪਰੋਕਤ ਫਾਈਲ ਦੇ ਆਧਾਰ 'ਤੇ, ਤੁਸੀਂ ਆਪਣੇ ਸਰਵਰ ਦੀ ਜਾਂਚ ਕਰ ਸਕਦੇ ਹੋ ਤਾਂ ਜੋ ਯਕੀਨ ਹੋ ਸਕੇ ਕਿ ਸਮਰੱਥਾਵਾਂ ਉਨ੍ਹਾਂ ਮਿਤੀ ਅਨੁਸਾਰ ਬਣਾਈਆਂ ਗਈਆਂ ਹਨ।

ਸਾਰੇ ਪ੍ਰਮੁੱਖ SDKs ਵਿੱਚ ਇੰਨਾ ਹੀ ਟੈਸਟਿੰਗ ਹਿੱਸੇ ਹੁੰਦੇ ਹਨ ਤਾਂ ਜੋ ਤੁਸੀਂ ਆਪਣੇ ਚੁਣੇ ਹੋਏ Runtime ਦੇ ਲੋਕ-ਚਲਿਤ ਕਰ ਸਕੋ।
