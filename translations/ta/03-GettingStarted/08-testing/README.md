## சோதனை மற்றும் பிழைத்திருத்தம்

உங்கள் MCP சேவையகத்தை சோதிக்க ஆரம்பிப்பதற்கு முன், கிடைக்கும் கருவிகள் மற்றும் பிழைத்திருத்த சிறந்த நடைமுறைகளைக் குறித்து புரிந்து கொள்வது முக்கியம். கற்றல் சோதனை உங்கள் சேவையகம் எதிர்பார்த்தபடி பணிபுரியும் என்பதைக் உறுதி செய்யும் மற்றும் பிரச்சனைகளை விரைவாக கண்டறிந்து தீர்க்க உதவும். கீழ்க்கண்ட பகுதி உங்கள் MCP செயல்பாட்டை சரிபார்க்க பரிந்துரைக்கப்பட்ட அணுகுமுறைகளை விவரிக்கிறது.

## கண்ணோட்டம்

இந்த பாடத்தில் சரியான சோதனைப் பயன்முறையை மற்றும் மிகச் சிறந்த சோதனை கருவியை எப்படி தேர்ந்தெடுக்க வேண்டும் என்பதை உள்ளடக்கியது.

## கற்றலுக்கான குறிக்கோள்கள்

இந்த பாடத்தை முடித்ததும், நீங்கள்:

- பல்வேறு சோதனை முறைகளைக் குறிப்பிடக்கூடும்.
- உங்கள் குறியீட்டை பிரயோஜனபடுத்தி சோதிக்க பல்வேறு கருவிகளை பயன்படுத்தக்கூடும்.


## MCP சேவையகங்களை சோதித்தல்

MCP உங்களுடைய சேவையகங்களை சோதிக்க மற்றும் பிழைத்திருத்த உதவ கருவிகளை வழங்குகிறது:

- **MCP இன்ஸ்பெக்டர்**: ஒரு கமாண்ட் லைன் கருவி, CLI கருவியாய் மற்றும் ஒரு பார்வை கருவியாய் இயங்கக்கூடியது.
- **கைமுறை சோதனை**: உங்கள் விருப்பமான கருவி, உதாரணமாக curl பயன்படுத்தியும் வலை கோரிக்கைகளை இயக்கலாம், ஆனால் HTTP இயங்கக்கூடிய எந்த கருவியும் பணி செய்யும்.
- **அலகு சோதனை**: சேவையக மற்றும் கிளையண்ட் இரண்டின் அம்சங்களை சோதிக்க விரும்பும் சோதனை கட்டமைப்பை பயன்படுத்த முடியும்.

### MCP இன்ஸ்பெக்டர் பயன்பாடு

இந்த கருவியின் பயன்பாட்டைப் பற்றி முந்தைய பாடங்களில் விவரிக்கப்பட்டுள்ளது, ஆனால் இப்போது அதன் சில மேல்நிலை அம்சங்களைப் பற்றி பேசுவோம். இது Node.js இல் உருவாக்கப்பட்டு, `npx` நிரலை அழைத்து பயன்படுத்தப்படுகின்றது, இது தற்காலிகமாக கருவியை பதிவிறக்கம் செய்து நிறுவி, உங்கள் கோரிக்கையை நிறைவேற்றியதும் தானாகவே இறுக்கிறது.

[MCP இன்ஸ்பெக்டர்](https://github.com/modelcontextprotocol/inspector) நீங்கள் செய்ய உதவுகிறது:

- **சேவையக திறன்களை கண்டுபிடித்தல்**: கிடைக்கும் வளங்கள், கருவிகள் மற்றும் முன்மொழிவுகளை தானாக கண்டறிதல்
- **கருவி செயல்பாட்டை சோதித்தல்**: வேறுபட்ட அளவுருக்களை முயற்சி செய்து நேரடி பதில்களை பார்வையிடுதல்
- **சேவையக தகவல்களை காண்பித்தல்**: சேவையகத் தகவல், வடிவமைப்புகள் மற்றும் உள்ளமைவுகளை ஆய்வு செய்தல்

கருவியின் ஒரு சாதாரண இயக்கம் இதுபோல் தெரியலாம்:

```bash
npx @modelcontextprotocol/inspector node build/index.js
```

மேலே உள்ள கட்டளை MCP மற்றும் அதனுடைய பார்வைப் படிவை தொடங்கி உலாவியில் உள்ளூர் வலை இடைமுகத்தை தொடங்குகிறது. பங்கேற்கப்பட்ட MCP சேவையகங்கள், அவற்றின் கிடைக்கும் கருவிகள், வளங்கள் மற்றும் முன்மொழிவுகள் ஆகியவற்றை குறியீட்டு இடைமுகம் காண்பிக்கும். இந்த இடைமுகம் கருவி செயல்பாட்டை பகிர்வான முறையில் சோதிக்க, சேவையக தகவல்களை ஆய்வு செய்ய, நேரடி பதில்களை பார்க்க அனுமதித்து, உங்கள் MCP சேவையக செயலாக்கங்களை சரிபார்க்க மற்றும் பிழைத்திருத்த எளிதாக்குகிறது.

இதுபோல் தோன்றும்: ![Inspector](../../../../translated_images/ta/connect.141db0b2bd05f096.webp)

இந்த கருவியை CLI முறையிலும் இயக்கலாம், அதற்காக `--cli` பண்பைச் சேர்க்க வேண்டும். "CLI" முறையில் கருவியை இயக்கி சேவையகத்தில் அனைத்து கருவிகளையும் பட்டியலிடும் உதாரணம் இங்கே:

```sh
npx @modelcontextprotocol/inspector --cli node build/index.js --method tools/list
```

### கைமுறை சோதனை

சேவையக திறன்களை சோதிக்க இன்ஸ்பெக்டர் கருவியை இயக்குவதற்கு பின், மற்றொரு உங்கள் அணுகுமுறை HTTP பயன்படுத்தக்கூடிய கிளையண்டை இயக்குவது, உதாரணமாக curl பயன்படுத்தும்.

curl மூலம், நீங்கள் MCP சேவையகங்களை நேரடியாக HTTP கோரிக்கை மூலமாக சோதிக்கலாம்:

```bash
# உதாரணம்: சேவை கருவி வடிவமைப்பு தரவு
curl http://localhost:3000/v1/metadata

# உதாரணம்: ஒரு கருவி செயல்படுத்தவும்
curl -X POST http://localhost:3000/v1/tools/execute \
  -H "Content-Type: application/json" \
  -d '{"name": "calculator", "parameters": {"expression": "2+2"}}'
```

மேலே curl பயன்பாட்டை பார்த்தீர்கள் போல, ஒரு கருவியை அதன் பெயர் மற்றும் அளவுருக்களின் தொகுப்புடன் நடத்திய ஒரு POST கோரிக்கையை பயன்படுத்தி இயக்கலாம். உங்களுக்கு ஏற்ற அணுகுமுறையை பயன்படுத்துங்கள். CLI கருவிகள் பொதுவாக விரைவாகவும், ஸ்கிரிப்ட் எழுதக்கூடியவையும் ஆகும், இதில் CI/CD சூழல் மிகவும் உதவியாக இருக்கும்.

### அலகு சோதனை

உங்கள் கருவிகளுக்கும் வளங்களுக்கும் அலகு சோதனைகள் உருவாக்கி அவை எதிர்பார்த்தபடி வேலை செய்கின்றனவா என்று உறுதி செய்யுங்கள். கீழே சில சோதனை குறியீட்டின் உதாரணம் உள்ளது.

```python
import pytest

from mcp.server.fastmcp import FastMCP
from mcp.shared.memory import (
    create_connected_server_and_client_session as create_session,
)

# முழு மொட்யூலை அசிங்க் சோதனைகளுக்குப் பயன்படுத்த குறி இடு
pytestmark = pytest.mark.anyio


async def test_list_tools_cursor_parameter():
    """Test that the cursor parameter is accepted for list_tools.

    Note: FastMCP doesn't currently implement pagination, so this test
    only verifies that the cursor parameter is accepted by the client.
    """

 server = FastMCP("test")

    # சில சோதனை கருவிகள் உருவாக்கு
    @server.tool(name="test_tool_1")
    async def test_tool_1() -> str:
        """First test tool"""
        return "Result 1"

    @server.tool(name="test_tool_2")
    async def test_tool_2() -> str:
        """Second test tool"""
        return "Result 2"

    async with create_session(server._mcp_server) as client_session:
        # கர்சர் அளவுருவினை இல்லாமல் சோதனை செய் (விடுபடாதது)
        result1 = await client_session.list_tools()
        assert len(result1.tools) == 2

        # கர்சர்=None என்றுடன் சோதனை செய்
        result2 = await client_session.list_tools(cursor=None)
        assert len(result2.tools) == 2

        # கர்சரை எழுத்துரு என கொண்டு சோதனை செய்
        result3 = await client_session.list_tools(cursor="some_cursor_value")
        assert len(result3.tools) == 2

        # காலியான எழுதுரு கர்சருடன் சோதனை செய்
        result4 = await client_session.list_tools(cursor="")
        assert len(result4.tools) == 2
    
```

மேலே உள்ள குறியீடு பின்வருமாறு செய்கிறது:

- pytest கட்டமைப்பை பயன்படுத்தி, உங்கள் சோதனைகளை செயல்பாடுகளாக உருவாக்கி `assert` உரைகூறுகளை பயன்படுத்துகிறது.
- இரண்டு வெவ்வேறு கருவிகளுடன் MCP சேவையகத்தை உருவாக்குகிறது.
- குறிப்பிட்ட நிபந்தனைகள் பூர்த்தி செய்யப்படுகிறதா என்று `assert` மூலம் சரிபார்க்கிறது.

[முழு கோப்பை இங்கே பார்க்கவும்](https://github.com/modelcontextprotocol/python-sdk/blob/main/tests/client/test_list_methods_cursor.py)

மேலே உள்ள கோப்பின் அடிப்படையில், உங்கள் சேவையகத்தை சோதித்து திறன்கள் சரியாக உருவாக்கப்பட்டுள்ளதா என்பதை உறுதி செய்யலாம்.

முக்கியமான அனைத்து SDKகளுக்கும் இதுபோன்ற சோதனை பகுதிகள் உள்ளன, ஆகையால் நீங்கள் உங்கள் தேர்ந்தெடுத்த இயக்ககத்திற்கே பொருத்தமாக மாற்றிக்கொள்ளலாம்.
