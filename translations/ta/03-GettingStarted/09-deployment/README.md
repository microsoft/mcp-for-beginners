# MCP சர்வர்களை வெளியிடுதல்

உங்கள் MCP சர்வர் வெளியிடப்படுவது, அதன் கருவிகள் மற்றும் வளங்களை உங்கள் உள்ளூராட்சி சுற்றுப்புறத்தைத் தாண்டி பிறர் அணுகக் கூடியதாகக் செய்கிறது. அளவீடு, நம்பகத்தன்மை மற்றும் பராமரிப்பு எளிமை பற்றிய உங்கள் தேவைகளின் அடிப்படையில் பல வெளியீட்டு யோசனைகள் உள்ளன. கீழே, நீங்கள் MCP சர்வர்களை உள்ளூரில், கொண்டெய்நர்கள் மற்றும் மேகத்தில் வெளியிடுவதற்கான வழிகாட்டுதல்கள் காணலாம்.

## மேற்பார்வை

இந்த பாடம் உங்கள் MCP சர்வர் செயலியை எப்படி வெளியிடுவது என்பதைக் covers செய்யும்.

## கற்றல் நோக்கங்கள்

இந்த பாட முடிவில், நீங்கள் திறந்திருப்பீர்கள்:

- வேறு வெவ்வேறு வெளியீட்டு அணுகுமுறைகளை மதிப்பீடு செய்ய.
- உங்கள் செயலியை வெளியிட.

## உள்ளூர் மேம்பாடு மற்றும் வெளியீடு

உங்கள் சர்வர் பயனர்களின் கணினியில் இயக்கப்படுவதை நோக்கமாகக் கொண்டிருந்தால், நீங்கள் பின்வரும் படிகளை பின்பற்றலாம்:

1. **சர்வரை பதிவிறக்குக**. நீங்கள் சர்வரை எழுதவில்லை என்றால், முதலில் அதை உங்கள் கணினிக்கு பதிவிறக்குக.  
1. **சர்வர் செயலியை துவக்குக**: உங்கள் MCP சர்வர் செயலியை இயக்குக

SSE க்கான (stdio வகை சர்வருக்கு தேவையில்லை)

1. **நெட்வொர்க் அமைக்குக**: சர்வர் எதிர்பார்க்கப்படும் போர்ட்டில் அணுகக்கூடியதாக இருப்பதை உறுதிசெய்க  
1. **கிளையண்டுகளை இணைக்குக**: `http://localhost:3000` போன்ற உள்ளூர் இணைப்பு URL களை பயன்படுத்துக

## மேக வெளியீடு

MCP சர்வர்கள் பல மேக தளங்களில் வெளியிடப்படலாம்:

- **சர்வர் இல்லா செயல்பாடுகள்**: MCP சர்வர்களை செர்வர் இல்லா செயல்பாடுகளாக வெளியிடுங்கள்  
- **கொண்டெய்னர் சேவைகள்**: Azure Container Apps, AWS ECS, அல்லது Google Cloud Run போன்ற சேவைகளைப் பயன்படுத்துக  
- **குபெர்னிட்டீஸ்**: உயர் கிடைக்கத்தன்மைக்காக MCP சர்வர்களை குபெர்னிட்டீஸ் குவளைப்புகளிலும் பராமரிக்கவும் வெளியிடவும் செய்க

### உதாரணம்: Azure Container Apps

Azure Container Apps MCP சர்வர்கள் வெளியிடுவதை ஆதரிக்கிறது. இது இன்னும் பணியில் உள்ளது மற்றும் தற்போது SSE சர்வர்களை மட்டுமே ஆதரிக்கிறது.

இதன்படி நீங்கள் செய்வது:

1. ஒரு ரெபோவை க்ளோன் செய்க:

  ```sh
  git clone https://github.com/anthonychu/azure-container-apps-mcp-sample.git
  ```

1. உள்ளூரில் சோதனை செய்து இயக்குக:

  ```sh
  uv venv
  uv sync

  # லினக்ஸ்/மெக்ஓஎஸ்
  export API_KEYS=<AN_API_KEY>
  # விண்டோஸ்
  set API_KEYS=<AN_API_KEY>

  uv run fastapi dev main.py
  ```

1. உள்ளூரிலும் சோதிக்க, *.vscode* கோப்பகத்தில் *mcp.json* கோப்பை உருவாக்கி கீழ்க்கண்ட உள்ளடக்கத்தைச் சேர்க்கவும்:

  ```json
  {
      "inputs": [
          {
              "type": "promptString",
              "id": "weather-api-key",
              "description": "Weather API Key",
              "password": true
          }
      ],
      "servers": {
          "weather-sse": {
              "type": "sse",
              "url": "http://localhost:8000/sse",
              "headers": {
                  "x-api-key": "${input:weather-api-key}"
              }
          }
      }
  }
  ```

  SSE சர்வர் துவங்கப்பட்டவுடன், JSON கோப்பில் பிளே ஐகானைக் கிளிக் செய்யலாம், இப்போது இந்த சர்வரின் கருவிகள் GitHub Copilot மூலம் கண்டறிகின்றன, கருவி ஐகானைப் பார்க்கவும்.

1. வெளியிட, கீழ்காணும் கட்டளையை இயக்குக:

  ```sh
  az containerapp up -g <RESOURCE_GROUP_NAME> -n weather-mcp --environment mcp -l westus --env-vars API_KEYS=<AN_API_KEY> --source .
  ```

இங்கு உள்ளது, உள்ளூரில் வெளியிடவும், Azureக்கு வெளியிடும் இந்த படிகள் மூலம்.
