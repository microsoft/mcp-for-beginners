# MCP பாதுகாப்பு: AI அமைப்புகளுக்கான விரிவான பாதுகாப்பு

[![MCP Security Best Practices](../../../translated_images/ta/03.175aed6dedae133f9d41e49cefd0f0a9a39c3317e1eaa7ef7182696af7534308.png)](https://youtu.be/88No8pw706o)

_(இந்த பாடத்தின் வீடியோவை பார்க்க மேலே உள்ள படத்தை கிளிக் செய்யவும்)_

பாதுகாப்பு AI அமைப்பு வடிவமைப்பின் அடிப்படையாகும், அதனால் இதை நாங்கள் இரண்டாவது பிரிவாக முன்னுரிமை அளிக்கிறோம். இது Microsoft-ன் [Secure Future Initiative](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) இல் உள்ள **Secure by Design** கொள்கையுடன் ஒத்துப்போகிறது.

Model Context Protocol (MCP) AI இயக்கப்படும் பயன்பாடுகளுக்கு சக்திவாய்ந்த புதிய திறன்களை கொண்டு வருகிறது, அதே சமயம் பாரம்பரிய மென்பொருள் அபாயங்களைத் தாண்டி தனித்துவமான பாதுகாப்பு சவால்களை அறிமுகப்படுத்துகிறது. MCP அமைப்புகள் நிலையான பாதுகாப்பு கவலைகளையும் (பாதுகாப்பான குறியீடு, குறைந்த உரிமைகள், வழங்கல் சங்கிலி பாதுகாப்பு) மற்றும் புதிய AI-சார்ந்த அச்சுறுத்தல்களையும் எதிர்கொள்கின்றன, அதாவது prompt injection, tool poisoning, session hijacking, confused deputy attacks, token passthrough vulnerabilities மற்றும் dynamic capability modification.

இந்த பாடம் MCP அமலாக்கங்களில் மிக முக்கியமான பாதுகாப்பு அபாயங்களை ஆராய்கிறது—அங்குள்ள authentication, authorization, அதிக உரிமைகள், மறைமுக prompt injection, session பாதுகாப்பு, confused deputy பிரச்சினைகள், token மேலாண்மை மற்றும் வழங்கல் சங்கிலி பாதிப்புகள் ஆகியவை அடங்கும். நீங்கள் Microsoft தீர்வுகள் போன்ற Prompt Shields, Azure Content Safety மற்றும் GitHub Advanced Security-ஐ பயன்படுத்தி உங்கள் MCP அமலாக்கத்தை வலுப்படுத்தும் செயல்படுத்தக்கூடிய கட்டுப்பாடுகள் மற்றும் சிறந்த நடைமுறைகளை கற்றுக்கொள்வீர்கள்.

## கற்றல் நோக்கங்கள்

இந்த பாடத்தின் முடிவில், நீங்கள்:

- **MCP-சார்ந்த அச்சுறுத்தல்களை அடையாளம் காண்பது**: prompt injection, tool poisoning, அதிக உரிமைகள், session hijacking, confused deputy பிரச்சினைகள், token passthrough பாதிப்புகள் மற்றும் வழங்கல் சங்கிலி அபாயங்கள் போன்ற MCP அமைப்புகளில் தனித்துவமான பாதுகாப்பு அபாயங்களை அறிதல்
- **பாதுகாப்பு கட்டுப்பாடுகளை பயன்படுத்துதல்**: வலுவான authentication, குறைந்த உரிமை அணுகல், பாதுகாப்பான token மேலாண்மை, session பாதுகாப்பு கட்டுப்பாடுகள் மற்றும் வழங்கல் சங்கிலி சரிபார்ப்பு போன்ற பயனுள்ள தடுப்புகளை அமல்படுத்துதல்
- **Microsoft பாதுகாப்பு தீர்வுகளை பயன்படுத்துதல்**: MCP பணிச்சுமை பாதுகாப்புக்காக Microsoft Prompt Shields, Azure Content Safety மற்றும் GitHub Advanced Security-ஐ புரிந்து கொண்டு பயன்படுத்துதல்
- **கருவி பாதுகாப்பை சரிபார்த்தல்**: கருவி மெட்டாடேட்டா சரிபார்ப்பு, இயக்க மாற்றங்களை கண்காணித்தல் மற்றும் மறைமுக prompt injection தாக்குதல்களுக்கு எதிராக பாதுகாப்பு முக்கியத்துவத்தை அறிதல்
- **சிறந்த நடைமுறைகளை ஒருங்கிணைத்தல்**: நிலையான பாதுகாப்பு அடிப்படைகள் (பாதுகாப்பான குறியீடு, சர்வர் கடுமைப்படுத்தல், பூஜ்ஜிய நம்பிக்கை) மற்றும் MCP-சார்ந்த கட்டுப்பாடுகளை ஒருங்கிணைத்து விரிவான பாதுகாப்பை உருவாக்குதல்

# MCP பாதுகாப்பு கட்டமைப்பு மற்றும் கட்டுப்பாடுகள்

நவீன MCP அமலாக்கங்கள் பாரம்பரிய மென்பொருள் பாதுகாப்பும் AI-சார்ந்த அச்சுறுத்தல்களும் இரண்டையும் கையாளும் அடுக்கு பாதுகாப்பு அணுகுமுறைகளை தேவைப்படுத்துகின்றன. விரைவாக வளர்ந்து வரும் MCP விவரக்குறிப்பு அதன் பாதுகாப்பு கட்டுப்பாடுகளை மேம்படுத்தி வருகிறது, இது நிறுவன பாதுகாப்பு கட்டமைப்புகளுடன் மற்றும் நிலையான சிறந்த நடைமுறைகளுடன் சிறந்த ஒருங்கிணைப்பை வழங்குகிறது.

[Microsoft Digital Defense Report](https://aka.ms/mddr) ஆய்வுகள் **98% புகாரான மீறல்கள் வலுவான பாதுகாப்பு சீர்திருத்தத்தால் தடுப்படையுமென்று** காட்டுகின்றன. மிகச் சிறந்த பாதுகாப்பு தந்திரம் அடிப்படையான பாதுகாப்பு நடைமுறைகளையும் MCP-சார்ந்த கட்டுப்பாடுகளையும் இணைக்கும் முறையாகும்—சான்றளிக்கப்பட்ட அடிப்படை பாதுகாப்பு நடவடிக்கைகள் மொத்த பாதுகாப்பு அபாயத்தை குறைப்பதில் மிகுந்த தாக்கம் செலுத்துகின்றன.

## தற்போதைய பாதுகாப்பு சூழல்

> **குறிப்பு:** இந்த தகவல் **டிசம்பர் 18, 2025** அன்று உள்ள MCP பாதுகாப்பு தரநிலைகளை பிரதிபலிக்கிறது. MCP நெறிமுறை விரைவாக வளர்ந்து கொண்டிருக்கிறது, எதிர்கால அமலாக்கங்கள் புதிய authentication மாதிரிகள் மற்றும் மேம்பட்ட கட்டுப்பாடுகளை அறிமுகப்படுத்தலாம். எப்போதும் தற்போதைய [MCP Specification](https://spec.modelcontextprotocol.io/), [MCP GitHub repository](https://github.com/modelcontextprotocol) மற்றும் [பாதுகாப்பு சிறந்த நடைமுறைகள் ஆவணம்](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) ஆகியவற்றை பார்க்கவும்.

### MCP Authentication வளர்ச்சி

MCP விவரக்குறிப்பு authentication மற்றும் authorization அணுகுமுறையில் முக்கிய மாற்றங்களை கொண்டுள்ளது:

- **மூல அணுகுமுறை**: ஆரம்ப விவரக்குறிப்புகள் உருவாக்குநர்களை தனிப்பட்ட authentication சர்வர்களை உருவாக்க வேண்டியிருந்தது, MCP சர்வர்கள் OAuth 2.0 Authorization Servers ஆக செயல்பட்டு பயனர் authentication-ஐ நேரடியாக நிர்வகித்தன
- **தற்போதைய தரநிலை (2025-11-25)**: புதுப்பிக்கப்பட்ட விவரக்குறிப்பு MCP சர்வர்களுக்கு வெளிப்புற அடையாள வழங்குநர்களுக்கு (Microsoft Entra ID போன்றவை) authentication-ஐ ஒப்படைக்க அனுமதிக்கிறது, இது பாதுகாப்பு நிலையை மேம்படுத்தி செயலாக்க சிக்கலை குறைக்கிறது
- **பரிமாற்ற அடுக்கு பாதுகாப்பு**: உள்ளூர் (STDIO) மற்றும் தொலைநிலை (Streamable HTTP) இணைப்புகளுக்கான சரியான authentication மாதிரிகளுடன் பாதுகாப்பான பரிமாற்ற முறைகளுக்கு மேம்பட்ட ஆதரவு

## Authentication மற்றும் Authorization பாதுகாப்பு

### தற்போதைய பாதுகாப்பு சவால்கள்

நவீன MCP அமலாக்கங்கள் பல authentication மற்றும் authorization சவால்களை எதிர்கொள்கின்றன:

### அபாயங்கள் மற்றும் தாக்குதல்த் திசைகள்

- **தவறான Authorization தர்க்கம்**: MCP சர்வர்களில் தவறான authorization அமலாக்கம் நுண்ணறிவு தரவுகளை வெளிப்படுத்தி தவறான அணுகல் கட்டுப்பாடுகளை பயன்படுத்தக்கூடும்
- **OAuth Token களவாடம்**: உள்ளூர் MCP சர்வர் token திருட்டு தாக்குதலாளர்களுக்கு சர்வர்களை போல நடந்து கீழ் சேவைகளுக்கு அணுக அனுமதிக்கிறது
- **Token Passthrough பாதிப்புகள்**: தவறான token கையாளுதல் பாதுகாப்பு கட்டுப்பாடுகளை தவிர்க்கவும் பொறுப்புத்தன்மை இடைவெளிகளை உருவாக்கவும் செய்கிறது
- **அதிக உரிமைகள்**: அதிக உரிமைகள் கொண்ட MCP சர்வர்கள் குறைந்த உரிமை கொள்கைகளை மீறி தாக்குதலுக்கு வாய்ப்பளிக்கின்றன

#### Token Passthrough: ஒரு முக்கிய எதிர்மறை மாதிரி

தற்போதைய MCP authorization விவரக்குறிப்பில் **token passthrough தெளிவாகத் தடைசெய்யப்பட்டுள்ளது** கடுமையான பாதுகாப்பு விளைவுகளுக்காக:

##### பாதுகாப்பு கட்டுப்பாடு தவிர்க்கல்
- MCP சர்வர்கள் மற்றும் கீழ் API-கள் முக்கிய பாதுகாப்பு கட்டுப்பாடுகளை (விகித வரம்பு, கோரிக்கை சரிபார்ப்பு, போக்குவரத்து கண்காணிப்பு) சரியான token சரிபார்ப்பின் அடிப்படையில் செயல்படுத்துகின்றன
- நேரடி கிளையன்ட்-க்கு-API token பயன்பாடு இந்த பாதுகாப்புகளை தவிர்க்கிறது, பாதுகாப்பு கட்டமைப்பை பாதிக்கிறது

##### பொறுப்புத்தன்மை மற்றும் கணக்கெடுப்பு சவால்கள்  
- MCP சர்வர்கள் மேல்நிலை token-ஐ பயன்படுத்தும் கிளையன்டுகளை வேறுபடுத்த முடியாது, கணக்கெடுப்பு பாதைகள் உடைக்கப்படுகின்றன
- கீழ் வள சேவையக பதிவுகள் தவறான கோரிக்கை மூலங்களை காட்டுகின்றன, உண்மையான MCP சர்வர் இடைநிலையினை காட்டாது
- சம்பவ விசாரணை மற்றும் ஒத்துழைப்பு கணக்கெடுப்பு மிகவும் கடினமாகிறது

##### தரவு திருட்டு அபாயங்கள்
- சரிபார்க்கப்படாத token கோரிக்கைகள் திருடப்பட்ட token-களை கொண்ட தீய நோக்கமுள்ளவர்கள் MCP சர்வர்களை தரவு திருட்டுக்கான பிரதிநிதிகளாக பயன்படுத்த அனுமதிக்கின்றன
- நம்பிக்கை எல்லை மீறல்கள் அனுமதிக்கப்படாத அணுகல் முறைகளை உருவாக்குகின்றன

##### பல சேவை தாக்குதல் திசைகள்
- பல சேவைகள் ஏற்றுக்கொள்ளும் களவாடப்பட்ட token-கள் இணைக்கப்பட்ட அமைப்புகளில் பக்கவழி நகர்வை ஏற்படுத்துகின்றன
- token மூலங்களை சரிபார்க்க முடியாதபோது சேவைகளுக்கு இடையேயான நம்பிக்கை கோளாறுகள் ஏற்படலாம்

### பாதுகாப்பு கட்டுப்பாடுகள் மற்றும் தடுப்புகள்

**முக்கிய பாதுகாப்பு தேவைகள்:**

> **கட்டாயம்**: MCP சர்வர்கள் **MCP சர்வருக்காக தெளிவாக வழங்கப்படாத எந்த token-ஐயும் ஏற்கக்கூடாது**

#### Authentication மற்றும் Authorization கட்டுப்பாடுகள்

- **கடுமையான Authorization ஆய்வு**: MCP சர்வர் authorization தர்க்கத்தை முழுமையாக ஆய்வு செய்து, மட்டுமே நோக்கப்பட்ட பயனர்கள் மற்றும் கிளையன்டுகள் நுண்ணறிவு வளங்களுக்கு அணுக முடியும் என்பதை உறுதி செய்யவும்
  - **அமல்படுத்தல் வழிகாட்டி**: [Azure API Management as Authentication Gateway for MCP Servers](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)
  - **அடையாள ஒருங்கிணைப்பு**: [Microsoft Entra ID-ஐ MCP Server Authentication-க்கு பயன்படுத்துதல்](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **பாதுகாப்பான Token மேலாண்மை**: [Microsoft-ன் token சரிபார்ப்பு மற்றும் ஆயுள் கால சிறந்த நடைமுறைகள்](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens) ஐ செயல்படுத்தவும்
  - token பார்வையாளர் கோரிக்கைகள் MCP சர்வர் அடையாளத்துடன் பொருந்துவதை சரிபார்க்கவும்
  - சரியான token மாற்றம் மற்றும் காலாவதி கொள்கைகளை அமல்படுத்தவும்
  - token மீண்டும் பயன்படுத்தல் தாக்குதல்களை மற்றும் அனுமதியற்ற பயன்பாட்டை தடுப்பது

- **பாதுகாப்பான Token சேமிப்பு**: ஓய்விலும் பரிமாற்றத்திலும் குறியாக்கத்துடன் token சேமிப்பை பாதுகாக்கவும்
  - **சிறந்த நடைமுறைகள்**: [Secure Token Storage and Encryption Guidelines](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### அணுகல் கட்டுப்பாடு அமல்படுத்தல்

- **குறைந்த உரிமை கொள்கை**: MCP சர்வர்களுக்கு தேவையான குறைந்தபட்ச உரிமைகள் மட்டுமே வழங்கவும்
  - உரிமை மதிப்பாய்வுகள் மற்றும் புதுப்பிப்புகளை முறையாகச் செய்யவும்
  - **Microsoft ஆவணம்**: [Secure Least-Privileged Access](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **பங்கு அடிப்படையிலான அணுகல் கட்டுப்பாடு (RBAC)**: நுணுக்கமான பங்கு ஒதுக்கீடுகளை செயல்படுத்தவும்
  - குறிப்பிட்ட வளங்கள் மற்றும் செயல்களுக்கு பங்குகளை கட்டுப்படுத்தவும்
  - பரவலான அல்லது தேவையற்ற உரிமைகளைத் தவிர்க்கவும்

- **தொடர்ச்சியான உரிமை கண்காணிப்பு**: தொடர்ச்சியான அணுகல் கணக்கெடுப்பு மற்றும் கண்காணிப்பை செயல்படுத்தவும்
  - உரிமை பயன்பாட்டு முறைமைகளை சந்தேகத்துடன் கண்காணிக்கவும்
  - அதிகமான அல்லது பயன்படுத்தப்படாத உரிமைகளை உடனடியாக சரிசெய்யவும்

## AI-சார்ந்த பாதுகாப்பு அச்சுறுத்தல்கள்

### Prompt Injection மற்றும் கருவி மாற்றுதல் தாக்குதல்கள்

நவீன MCP அமலாக்கங்கள் பாரம்பரிய பாதுகாப்பு நடவடிக்கைகள் முழுமையாக கையாள முடியாத நுணுக்கமான AI-சார்ந்த தாக்குதல் திசைகளை எதிர்கொள்கின்றன:

#### **மறைமுக Prompt Injection (Cross-Domain Prompt Injection)**

**மறைமுக Prompt Injection** MCP-ஐ இயக்கும் AI அமைப்புகளில் மிக முக்கியமான பாதிப்புகளை ஏற்படுத்தும். தாக்குதலாளர்கள் தீய நோக்கமுள்ள கட்டளைகளை வெளிப்புற உள்ளடக்கங்களில் (ஆவணங்கள், வலைப்பக்கங்கள், மின்னஞ்சல்கள் அல்லது தரவு மூலங்கள்) மறைத்து வைக்கின்றனர், பின்னர் AI அமைப்புகள் அவற்றை சட்டபூர்வமான கட்டளைகளாக செயல்படுத்துகின்றன.

**தாக்குதல் சூழல்கள்:**
- **ஆவண அடிப்படையிலான Injection**: செயலாக்கப்பட்ட ஆவணங்களில் மறைக்கப்பட்ட தீய கட்டளைகள் AI-ஐ எதிர்பாராத செயல்களை செய்ய தூண்டுகின்றன
- **வலை உள்ளடக்க பயன்பாடு**: பாதிக்கப்பட்ட வலைப்பக்கங்களில் உள்ள prompt-கள் AI நடத்தை மாற்றுகின்றன
- **மின்னஞ்சல் அடிப்படையிலான தாக்குதல்கள்**: மின்னஞ்சல்களில் உள்ள தீய prompt-கள் AI உதவியாளர்களை தகவல் வெளியிடவோ அல்லது அனுமதியற்ற செயல்களை செய்யவோ தூண்டுகின்றன
- **தரவு மூலக் கலப்பு**: பாதிக்கப்பட்ட தரவுத்தளங்கள் அல்லது API-கள் AI அமைப்புகளுக்கு பாதிக்கப்பட்ட உள்ளடக்கங்களை வழங்குகின்றன

**உண்மையான தாக்கம்**: இத்தாக்குதல்கள் தரவு திருட்டு, தனியுரிமை மீறல், தீய உள்ளடக்க உருவாக்கம் மற்றும் பயனர் தொடர்புகளை மாற்றுதல் போன்ற விளைவுகளை ஏற்படுத்தலாம். விரிவான பகுப்பாய்வுக்கு [Prompt Injection in MCP (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/) பார்க்கவும்.

![Prompt Injection Attack Diagram](../../../translated_images/ta/prompt-injection.ed9fbfde297ca877c15bc6daa808681cd3c3dc7bf27bbbda342ef1ba5fc4f52d.png)

#### **கருவி விஷப்பொருள் தாக்குதல்கள்**

**கருவி விஷப்பொருள்** MCP கருவிகளின் மெட்டாடேட்டாவை குறிவைத்து, LLM-கள் கருவி விளக்கங்கள் மற்றும் அளவுரு விவரங்களை எப்படி புரிகின்றன என்பதைக் கையாள்கிறது.

**தாக்குதல் முறைகள்:**
- **மெட்டாடேட்டா மாற்றுதல்**: தாக்குதலாளர்கள் கருவி விளக்கங்கள், அளவுரு வரையறைகள் அல்லது பயன்பாட்டு உதாரணங்களில் தீய கட்டளைகளை ஊற்றுகின்றனர்
- **காணாத கட்டளைகள்**: கருவி மெட்டாடேட்டாவில் மறைக்கப்பட்ட prompt-கள் AI மாதிரிகள் செயல்படுத்தும், ஆனால் மனித பயனர்களுக்கு தெரியாதவை
- **இயக்க கருவி மாற்றுதல் ("Rug Pulls")**: பயனர்கள் ஒப்புதல் அளித்த கருவிகள் பின்னர் தீய செயல்களை செய்ய மாற்றப்படுகின்றன
- **அளவுரு ஊற்றுதல்**: கருவி அளவுரு திட்டங்களில் தீய உள்ளடக்கம் ஊற்றப்படுகின்றது, இது மாதிரி நடத்தை பாதிக்கிறது

**ஓர் சேவையக அபாயங்கள்**: தொலை MCP சர்வர்கள் கருவி வரையறைகளை பயனர் ஒப்புதலுக்குப் பிறகு புதுப்பிக்க முடியும், இதனால் முன்பு பாதுகாப்பான கருவிகள் தீயவையாக மாறும் சூழல்கள் உருவாகின்றன. விரிவான பகுப்பாய்வுக்கு [Tool Poisoning Attacks (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks) பார்க்கவும்.

![Tool Injection Attack Diagram](../../../translated_images/ta/tool-injection.3b0b4a6b24de6befe7d3afdeae44138ef005881aebcfc84c6f61369ce31e3640.png)

#### **மேலும் AI தாக்குதல் திசைகள்**

- **Cross-Domain Prompt Injection (XPIA)**: பல தள உள்ளடக்கங்களை பயன்படுத்தி பாதுகாப்பு கட்டுப்பாடுகளை தவிர்க்கும் நுணுக்கமான தாக்குதல்கள்
- **Dynamic Capability Modification**: கருவி திறன்களில் நேரடி மாற்றங்கள், ஆரம்ப பாதுகாப்பு மதிப்பீடுகளை தவிர்க்கும்
- **Context Window Poisoning**: பெரிய context ஜன்னல்களை மாற்றி தீய கட்டளைகளை மறைக்கும் தாக்குதல்கள்
- **Model Confusion Attacks**: மாதிரி வரம்புகளை பயன்படுத்தி எதிர்பாராத அல்லது பாதுகாப்பற்ற நடத்தை உருவாக்குதல்

### AI பாதுகாப்பு அபாய விளைவுகள்

**உயர் தாக்கம் விளைவுகள்:**
- **தரவு திருட்டு**: அனுமதியற்ற அணுகல் மற்றும் நுண்ணறிவு அல்லது தனிப்பட்ட தரவுகளை திருடல்
- **தனியுரிமை மீறல்கள்**: தனிப்பட்ட அடையாள தகவல்கள் (PII) மற்றும் ரகசிய வணிக தரவுகள் வெளிப்படுதல்  
- **அமைப்பு மாற்றுதல்**: முக்கிய அமைப்புகள் மற்றும் பணிச்சூழல்களில் எதிர்பாராத மாற்றங்கள்
- **அங்கீகார குறியீடு திருட்டு**: authentication token-கள் மற்றும் சேவை அங்கீகாரங்கள் களவாடப்படுதல்
- **பக்கவழி நகர்வு**: பாதிக்கப்பட்ட AI அமைப்புகளை பயன்படுத்தி பரவலான நெட்வொர்க் தாக்குதல்கள்

### Microsoft AI பாதுகாப்பு தீர்வுகள்

#### **AI Prompt Shields: Injection தாக்குதல்களுக்கு மேம்பட்ட பாதுகாப்பு**

Microsoft **AI Prompt Shields** நேரடி மற்றும் மறைமுக prompt injection தாக்குதல்களுக்கு பல பாதுகாப்பு அடுக்குகளின் மூலம் விரிவான பாதுகாப்பை வழங்குகின்றன:

##### **முக்கிய பாதுகாப்பு இயந்திரங்கள்:**

1. **மேம்பட்ட கண்டறிதல் மற்றும் வடிகட்டி**
   - இயந்திரக் கற்றல் மற்றும் NLP தொழில்நுட்பங்கள் வெளிப்புற உள்ளடக்கங்களில் தீய கட்டளைகளை கண்டறிகின்றன
   - ஆவணங்கள், வலைப்பக்கங்கள், மின்னஞ்சல்கள் மற்றும் தரவு மூலங்களை நேரடி பகுப்பாய்வு
   - சட்டபூர்வ மற்றும் தீய prompt மாதிரிகளின் சூழல் புரிதல்

2. **ஒளிர்வு தொழில்நுட்பங்கள்**  
   - நம்பகமான அமைப்பு கட்டளைகள் மற்றும் பாதிக்கப்பட்ட வெளிப்புற உள்ளீடுகளை வேறுபடுத்துகிறது
   - மாதிரி தொடர்புடைய உரை மாற்ற முறைகள், தீய உள்ளடக்கத்தை தனிமைப்படுத்துதல்
   - AI அமைப்புகள் சரியான கட்டளை வரிசையை பராமரிக்கவும் ஊற்றப்பட்ட கட்டளைகளை புறக்கணிக்கவும் உதவுகிறது

3. **வெளிப்படையான எல்லை மற்றும் தரவு குறி அமைப்புகள்**
   - நம்பகமான அமைப்பு செய்திகளுக்கும் வெளிப்புற உள்ளீடு உரைக்கும் இடையே தெளிவான எல்லை வரையறை
   - நம்பகமான மற்றும் நம்பாத தரவு மூலங்களுக்கிடையேயான எல்லைகளை சிறப்பு குறிகள் மூலம் காட்டுதல்
   - கட்டளை குழப்பம் மற்றும் அனுமதியற்ற கட்டளை செயல்பாட்டை தடுக்கும் தெளிவான பிரிவு

4. **தொடர்ச்சியான அச்சுறுத்தல் நுண்ணறிவு**

   - வளர்ந்து வரும் அச்சுறுத்தல்களுக்கு எதிராக பாதுகாப்பு மாதிரிகள் அடிக்கடி புதுப்பிக்கப்படுகின்றன

## மேம்பட்ட MCP பாதுகாப்பு அச்சுறுத்தல்கள்

### Session Hijacking பாதிப்புகள்

**Session hijacking** என்பது நிலையான MCP அமலாக்கங்களில் முக்கிய தாக்குதல் திசையாகும், இதில் அனுமதியற்ற தரப்புகள் செல்லுபடியான session அடையாளங்களைப் பெற்று, கிளையன்டுகளை போல நடந்து அனுமதியற்ற செயல்களைச் செய்கின்றன.

#### **தாக்குதல் சூழல்கள் மற்றும் அபாயங்கள்**

- **Session Hijack Prompt Injection**: திருடப்பட்ட session ID-களை கொண்ட தாக்குதலாளர்கள் session நிலையை பகிரும் சர்வர்களில் தீய நிகழ்வுகளை ஊற்றுகின்றனர், இது தீய செயல்களை தூண்டவோ நுண்ணறிவு தரவுகளை அணுகவோ செய்யலாம்
- **நேரடி போலி நடிப்பு**: திருடப்பட்ட session ID-கள் authentication-ஐ தவிர்த்து நேரடி MCP சர்வர் அழைப்புகளைச் செய்ய அனுமதிக்கின்றன, தாக்குதலாளர்களை செல்லுபடியான பயனர்களாக கருதுகின்றன
- **தகர்க்கக்கூடிய மீண்டும் தொடக்கக்கூடிய ஸ்ட்ரீம்கள்**: தாக்குதலாளர்கள் கோரிக்கைகளை முன்கூட்டியே நிறுத்தி, செல்லுபடியான கிளையன்டுகள் தீய உள்ளடக்கத்துடன் மீண்டும் தொடங்குவதை ஏற்படுத்தலாம்

#### **Session மேலாண்மைக்கான பாதுகாப்பு கட்டுப்பாடுகள்**

**முக்கிய தேவைகள்:**
- **அங்கீகார சரிபார்ப்பு**: அங்கீகாரத்தை செயல்படுத்தும் MCP சேவையகங்கள் அனைத்து உள்ளீட்டு கோரிக்கைகளையும் சரிபார்க்க **கட்டாயம்** மற்றும் அங்கீகாரத்திற்காக அமர்வுகளை நம்பக்கூடாது  
- **பாதுகாப்பான அமர்வு உருவாக்கம்**: பாதுகாப்பான சீரற்ற அமர்வு ஐடிகளைக் குறியாக்க ரகசியமான, பாதுகாப்பான சீரற்ற எண் உருவாக்கிகளுடன் உருவாக்கவும்  
- **பயனர்-சார்ந்த பிணைப்பு**: அமர்வு ஐடிகளை `<user_id>:<session_id>` போன்ற வடிவங்களில் பயனர்-சார்ந்த தகவலுடன் பிணைக்கவும், இது பயனர் இடையே அமர்வு தவறான பயன்பாட்டை தடுக்கும்  
- **அமர்வு ஆயுள் மேலாண்மை**: பாதிப்பு சாளரங்களை குறைக்க சரியான காலாவதி, சுழற்சி மற்றும் செல்லாததாக்கல் செயல்படுத்தவும்  
- **பரிமாற்ற பாதுகாப்பு**: அமர்வு ஐடி பிடிப்பைத் தடுக்கும் வகையில் அனைத்து தொடர்புகளுக்கும் HTTPS கட்டாயம்  

### குழப்பமான துணை பிரச்சனை

**குழப்பமான துணை பிரச்சனை** என்பது MCP சேவையகங்கள் கிளையன்ட்களுக்கும் மூன்றாம் தரப்பு சேவைகளுக்கும் இடையில் அங்கீகார பிரதிநிதிகளாக செயல்படும் போது ஏற்படுகிறது, இது நிலையான கிளையன்ட் ஐடி தவறான பயன்பாட்டின் மூலம் அங்கீகாரத்தை தவிர்க்க வாய்ப்புகளை உருவாக்குகிறது.

#### **தாக்குதல் இயந்திரங்கள் மற்றும் அபாயங்கள்**

- **குக்கீ அடிப்படையிலான ஒப்புதல் தவிர்ப்பு**: முந்தைய பயனர் அங்கீகாரம் ஒப்புதல் குக்கீகளை உருவாக்குகிறது, தாக்குதலாளர்கள் தீய அங்கீகார கோரிக்கைகள் மற்றும் உருவாக்கப்பட்ட மறுவழி URIகளை பயன்படுத்தி அவற்றை தவறாக பயன்படுத்துகிறார்கள்  
- **அங்கீகார குறியீடு திருட்டு**: உள்ள ஒப்புதல் குக்கீகள் அங்கீகார சேவையகங்களை ஒப்புதல் திரைகளை தவிர்க்கச் செய்து குறியீடுகளை தாக்குதலாளரின் கட்டுப்பாட்டில் உள்ள இடங்களுக்கு மறுவழி செய்கின்றன  
- **அங்கீகாரம் இல்லாத API அணுகல்**: திருடப்பட்ட அங்கீகார குறியீடுகள் டோக்கன் பரிமாற்றத்தையும் பயனர் போல நடிப்பையும் அனுமதிக்கின்றன  

#### **தடுக்குமுறை நடவடிக்கைகள்**

**கட்டாய கட்டுப்பாடுகள்:**
- **தெளிவான ஒப்புதல் தேவைகள்**: நிலையான கிளையன்ட் ஐடிகளைப் பயன்படுத்தும் MCP பிரதிநிதி சேவையகங்கள் ஒவ்வொரு இயக்கத்தில் பதிவு செய்யப்பட்ட கிளையன்டுக்கும் பயனர் ஒப்புதலை பெற **கட்டாயம்**  
- **OAuth 2.1 பாதுகாப்பு செயல்பாடு**: அனைத்து அங்கீகார கோரிக்கைகளுக்கும் PKCE (குறியீடு பரிமாற்றத்திற்கான சான்று விசை) உட்பட தற்போதைய OAuth பாதுகாப்பு சிறந்த நடைமுறைகளை பின்பற்றவும்  
- **கடுமையான கிளையன்ட் சரிபார்ப்பு**: தவறான பயன்பாட்டைத் தடுக்கும் வகையில் மறுவழி URIகளும் கிளையன்ட் அடையாளங்களும் கடுமையாக சரிபார்க்கப்பட வேண்டும்  

### டோக்கன் பாஸ்த்ரூ பாதிப்புகள்  

**டோக்கன் பாஸ்த்ரூ** என்பது MCP சேவையகங்கள் சரியான சரிபார்ப்பு இல்லாமல் கிளையன்ட் டோக்கன்களை ஏற்றுக்கொண்டு அவற்றை கீழ்தர APIகளுக்கு அனுப்பும் ஒரு தெளிவான எதிர்மறை மாதிரியாகும், இது MCP அங்கீகார விவரக்குறிப்புகளை மீறுகிறது.

#### **பாதுகாப்பு விளைவுகள்**

- **கட்டுப்பாடு தவிர்ப்பு**: நேரடி கிளையன்ட்-க்கு-API டோக்கன் பயன்பாடு முக்கியமான வீதக் கட்டுப்பாடு, சரிபார்ப்பு மற்றும் கண்காணிப்பு கட்டுப்பாடுகளை தவிர்க்கிறது  
- **ஆடிட் தடம் கெடுதல்**: மேல்நிலை வழங்கப்பட்ட டோக்கன்கள் கிளையன்ட் அடையாளத்தை அறிய முடியாமல் செய்து சம்பவ விசாரணை திறன்களை உடைக்கின்றன  
- **பிரதிநிதி அடிப்படையிலான தரவு திருட்டு**: சரிபார்க்கப்படாத டோக்கன்கள் தீய நோக்கமுள்ளவர்கள் சேவையகங்களை அனுமதியில்லாத தரவு அணுகலுக்கான பிரதிநிதிகளாக பயன்படுத்த அனுமதிக்கின்றன  
- **நம்பிக்கை எல்லை மீறல்கள்**: டோக்கன் தோற்றம் சரிபார்க்க முடியாத போது கீழ்தர சேவைகளின் நம்பிக்கை கருதுகோள்கள் மீறப்படலாம்  
- **பல சேவை தாக்குதல் விரிவாக்கம்**: பல சேவைகளில் ஏற்றுக்கொள்ளப்பட்ட கெடுபிடிக்கப்பட்ட டோக்கன்கள் பக்கவழி நகர்வை அனுமதிக்கின்றன  

#### **தேவையான பாதுகாப்பு கட்டுப்பாடுகள்**

**மாற்றமுடியாத தேவைகள்:**
- **டோக்கன் சரிபார்ப்பு**: MCP சேவையகங்கள் MCP சேவையகத்துக்காக தெளிவாக வழங்கப்படாத டோக்கன்களை ஏற்றக்கூடாது  
- **கேட்கும் நபர் சரிபார்ப்பு**: டோக்கன் கேட்கும் நபர் கோரிக்கைகள் MCP சேவையகத்தின் அடையாளத்துடன் பொருந்துவதை எப்போதும் சரிபார்க்கவும்  
- **சரியான டோக்கன் ஆயுள்**: குறுகிய ஆயுள் கொண்ட அணுகல் டோக்கன்களை பாதுகாப்பான சுழற்சி நடைமுறைகளுடன் செயல்படுத்தவும்  


## AI அமைப்புகளுக்கான வழங்கல் சங்கிலி பாதுகாப்பு

வழங்கல் சங்கிலி பாதுகாப்பு பாரம்பரிய மென்பொருள் சார்புகளுக்கு அப்பால் விரிவடைந்து முழு AI சூழலை உள்ளடக்கியுள்ளது. நவீன MCP செயல்பாடுகள் அனைத்து AI தொடர்புடைய கூறுகளையும் கடுமையாக சரிபார்க்கவும் கண்காணிக்கவும் வேண்டும், ஏனெனில் ஒவ்வொன்றும் அமைப்பு ஒருங்கிணைப்பை பாதிக்கும் அபாயங்களை கொண்டிருக்கலாம்.

### விரிவடைந்த AI வழங்கல் சங்கிலி கூறுகள்

**பாரம்பரிய மென்பொருள் சார்புகள்:**
- திறந்த மூல நூலகங்கள் மற்றும் கட்டமைப்புகள்  
- கன்டெய்னர் படங்கள் மற்றும் அடிப்படை அமைப்புகள்  
- மேம்பாட்டு கருவிகள் மற்றும் கட்டுமான குழாய்கள்  
- அடித்தள கூறுகள் மற்றும் சேவைகள்  

**AI-சார்ந்த வழங்கல் சங்கிலி கூறுகள்:**
- **அடித்தள மாதிரிகள்**: பல வழங்குநர்களிடமிருந்து முன் பயிற்சி பெற்ற மாதிரிகள், மூலத்தன்மை சரிபார்ப்பை தேவைப்படுத்தும்  
- **எம்பெட்டிங் சேவைகள்**: வெளிப்புற வெக்டரைசேஷன் மற்றும் அர்த்தவியல் தேடல் சேவைகள்  
- **சூழல் வழங்குநர்கள்**: தரவு மூலங்கள், அறிவுத்தளங்கள் மற்றும் ஆவணக் களஞ்சியங்கள்  
- **மூன்றாம் தரப்பு APIகள்**: வெளிப்புற AI சேவைகள், இயந்திரக் கற்றல் குழாய்கள் மற்றும் தரவு செயலாக்க இடைமுகங்கள்  
- **மாதிரி பொருட்கள்**: எடை, கட்டமைப்புகள் மற்றும் நுட்பமாக திருத்தப்பட்ட மாதிரி வகைகள்  
- **பயிற்சி தரவு மூலங்கள்**: மாதிரி பயிற்சி மற்றும் நுட்ப திருத்தத்திற்கு பயன்படுத்தப்படும் தரவுத்தொகுப்புகள்  

### விரிவான வழங்கல் சங்கிலி பாதுகாப்பு திட்டம்

#### **கூறு சரிபார்ப்பு மற்றும் நம்பிக்கை**
- **மூலத்தன்மை சரிபார்ப்பு**: அனைத்து AI கூறுகளின் மூலத்தை, உரிமத்தை மற்றும் ஒருங்கிணைப்புக்கு முன் ஒருங்கிணைப்பை சரிபார்க்கவும்  
- **பாதுகாப்பு மதிப்பீடு**: மாதிரிகள், தரவு மூலங்கள் மற்றும் AI சேவைகளுக்கான பாதிப்பு ஸ்கேன் மற்றும் பாதுகாப்பு மதிப்பீடுகளை நடத்தவும்  
- **புகழ் பகுப்பாய்வு**: AI சேவை வழங்குநர்களின் பாதுகாப்பு வரலாறு மற்றும் நடைமுறைகளை மதிப்பாய்வு செய்யவும்  
- **கட்டுப்பாடு சரிபார்ப்பு**: அனைத்து கூறுகளும் நிறுவன பாதுகாப்பு மற்றும் ஒழுங்குமுறை தேவைகளை பூர்த்தி செய்கின்றன என்பதை உறுதி செய்யவும்  

#### **பாதுகாப்பான வெளியீட்டு குழாய்கள்**  
- **தானியங்கி CI/CD பாதுகாப்பு**: தானியங்கி வெளியீட்டு குழாய்களில் முழுமையான பாதுகாப்பு ஸ்கேனிங் ஒருங்கிணைப்பு  
- **பொருள் ஒருமைத்தன்மை**: அனைத்து வெளியிடப்பட்ட பொருட்களுக்கும் (கோடு, மாதிரிகள், கட்டமைப்புகள்) குறியாக்க சரிபார்ப்பை செயல்படுத்தவும்  
- **படிநிலை வெளியீடு**: ஒவ்வொரு படியிலும் பாதுகாப்பு சரிபார்ப்புடன் முன்னேற்ற வெளியீட்டு நடைமுறைகள்  
- **நம்பகமான பொருள் களஞ்சியங்கள்**: சரிபார்க்கப்பட்ட, பாதுகாப்பான பொருள் பதிவகங்களிலிருந்து மட்டுமே வெளியீடு  

#### **தொடர்ச்சியான கண்காணிப்பு மற்றும் பதில்**
- **சார்பு ஸ்கேனிங்**: அனைத்து மென்பொருள் மற்றும் AI கூறு சார்புகளுக்கான தொடர்ச்சியான பாதிப்பு கண்காணிப்பு  
- **மாதிரி கண்காணிப்பு**: மாதிரி நடத்தை, செயல்திறன் மாற்றம் மற்றும் பாதுகாப்பு விசித்திரங்களை தொடர்ச்சியாக மதிப்பாய்வு  
- **சேவை ஆரோக்கிய கண்காணிப்பு**: வெளிப்புற AI சேவைகளின் கிடைக்கும் நிலை, பாதுகாப்பு சம்பவங்கள் மற்றும் கொள்கை மாற்றங்களை கண்காணிக்கவும்  
- **பிரதிகாரம் நுண்ணறிவு ஒருங்கிணைப்பு**: AI மற்றும் இயந்திரக் கற்றல் பாதுகாப்பு அபாயங்களுக்கு குறிப்பிட்ட பிரதி தகவல் ஊட்டங்களை இணைத்தல்  

#### **அணுகல் கட்டுப்பாடு மற்றும் குறைந்த உரிமைகள்**
- **கூறு நிலை அனுமதிகள்**: வணிக தேவையின் அடிப்படையில் மாதிரிகள், தரவு மற்றும் சேவைகளுக்கு அணுகலை கட்டுப்படுத்தவும்  
- **சேவை கணக்கு மேலாண்மை**: குறைந்தபட்ச தேவையான அனுமதிகளுடன் தனித்த சேவை கணக்குகளை செயல்படுத்தவும்  
- **பிணைய பிரிவாக்கம்**: AI கூறுகளை தனிமைப்படுத்தி சேவைகளுக்கு இடையேயான பிணைய அணுகலைக் குறைக்கவும்  
- **API வாயில்கள் கட்டுப்பாடுகள்**: வெளிப்புற AI சேவைகளுக்கு அணுகலை கட்டுப்படுத்தவும் கண்காணிக்கவும் மையப்படுத்தப்பட்ட API வாயில்களை பயன்படுத்தவும்  

#### **சம்பவ பதில் மற்றும் மீட்பு**
- **விரைவு பதில் நடைமுறைகள்**: கெடுபிடிக்கப்பட்ட AI கூறுகளை பாச்சிங் அல்லது மாற்றுவதற்கான நிறுவப்பட்ட செயல்முறைகள்  
- **அடையாள மாற்றம்**: ரகசியங்கள், API விசைகள் மற்றும் சேவை அடையாளங்களை தானியங்கி முறையில் மாற்றும் அமைப்புகள்  
- **மீட்டெடுப்பு திறன்கள்**: AI கூறுகளின் முன் அறியப்பட்ட நல்ல பதிப்புகளுக்கு விரைவில் திரும்பும் திறன்  
- **வழங்கல் சங்கிலி மீறல் மீட்பு**: மேல்நிலை AI சேவை கெடுபிடிப்புகளுக்கு பதிலளிக்கும் குறிப்பிட்ட செயல்முறைகள்

- **நடத்தை பகுப்பாய்வு**: அமைப்பு மற்றும் பயனர் நடத்தை பற்றிய விசித்திர மாதிரிகளை கண்டறிய AI இயக்கப்பட்ட கண்காணிப்பு  
- **பிரதிகாரம் நுண்ணறிவு**: வெளிப்புற அபாய ஊட்டங்கள் மற்றும் கெடுபிடிப்பு குறியீடுகளின் ஒருங்கிணைப்பு  
- **சம்பவ பதில்**: பாதுகாப்பு சம்பவ கண்டறிதல், பதில் மற்றும் மீட்பு செயல்முறைகள்  

#### **சூனிய நம்பிக்கை கட்டமைப்பு**
- **எப்போதும் சரிபார்க்கவும், ஒருபோதும் நம்பாதே**: பயனர்கள், சாதனங்கள் மற்றும் பிணைய இணைப்புகளின் தொடர்ச்சியான சரிபார்ப்பு  
- **மைக்ரோ பிரிவாக்கம்**: தனித்தனியான பணிகள் மற்றும் சேவைகளை தனிமைப்படுத்தும் நுணுக்கமான பிணைய கட்டுப்பாடுகள்  
- **அடையாள மைய பாதுகாப்பு**: பிணைய இடம் அல்லாமல் சரிபார்க்கப்பட்ட அடையாளங்களின் அடிப்படையில் பாதுகாப்பு கொள்கைகள்  
- **தொடர்ச்சியான அபாய மதிப்பீடு**: தற்போதைய சூழல் மற்றும் நடத்தை அடிப்படையில் இயக்கக்கூடிய பாதுகாப்பு நிலை மதிப்பீடு  
- **நிபந்தனை அடிப்படையிலான அணுகல்**: அபாய காரணிகள், இடம் மற்றும் சாதன நம்பிக்கையின் அடிப்படையில் தழுவும் அணுகல் கட்டுப்பாடுகள்  

### நிறுவன ஒருங்கிணைப்பு மாதிரிகள்

#### **Microsoft பாதுகாப்பு சூழல் ஒருங்கிணைப்பு**
- **Microsoft Defender for Cloud**: விரிவான மேக பாதுகாப்பு நிலை மேலாண்மை  
- **Azure Sentinel**: AI பணிகளுக்கான மேக-உருவ SIEM மற்றும் SOAR திறன்கள்  
- **Microsoft Entra ID**: நிபந்தனை அடிப்படையிலான அணுகல் கொள்கைகளுடன் நிறுவன அடையாள மற்றும் அணுகல் மேலாண்மை  
- **Azure Key Vault**: ஹார்ட்வேர் பாதுகாப்பு மாட்யூல் (HSM) ஆதரவு கொண்ட மையப்படுத்தப்பட்ட ரகசிய மேலாண்மை  
- **Microsoft Purview**: AI தரவு மூலங்கள் மற்றும் பணிவழிகளுக்கான தரவு ஆட்சி மற்றும் ஒழுங்குமுறை  

#### **ஒழுங்குமுறை மற்றும் ஆட்சி**
- **ஒழுங்குமுறை ஒத்துழைப்பு**: MCP செயல்பாடுகள் தொழில் சார்ந்த ஒழுங்குமுறை தேவைகளை (GDPR, HIPAA, SOC 2) பூர்த்தி செய்கின்றன என்பதை உறுதி செய்யவும்  
- **தரவு வகைப்பாடு**: AI அமைப்புகள் செயலாக்கும் நுண்ணறிவு தரவுகளின் சரியான வகைப்பாடு மற்றும் கையாளல்  
- **ஆடிட் தடங்கள்**: ஒழுங்குமுறை ஒத்துழைப்பு மற்றும் குற்றவியல் விசாரணைக்கான விரிவான பதிவு  
- **தனியுரிமை கட்டுப்பாடுகள்**: AI அமைப்பு கட்டமைப்பில் தனியுரிமை-ஆல்-வடிவமைப்பு கொள்கைகளின் செயல்பாடு  
- **மாற்ற மேலாண்மை**: AI அமைப்பு மாற்றங்களுக்கான பாதுகாப்பு மதிப்பீடுகளுக்கான அதிகாரபூர்வ செயல்முறைகள்  

இந்த அடித்தள நடைமுறைகள் MCP-சார்ந்த பாதுகாப்பு கட்டுப்பாடுகளின் செயல்திறனை மேம்படுத்தும் வலுவான பாதுகாப்பு அடித்தளத்தை உருவாக்கி AI இயக்கப்பட்ட பயன்பாடுகளுக்கு விரிவான பாதுகாப்பை வழங்குகின்றன.

## முக்கிய பாதுகாப்பு எடுத்துக்காட்டுகள்

- **படிநிலை பாதுகாப்பு அணுகல்**: அடித்தள பாதுகாப்பு நடைமுறைகள் (பாதுகாப்பான குறியீடு, குறைந்த உரிமை, வழங்கல் சங்கிலி சரிபார்ப்பு, தொடர்ச்சியான கண்காணிப்பு) மற்றும் AI-சார்ந்த கட்டுப்பாடுகளை இணைத்து விரிவான பாதுகாப்பு  
- **AI-சார்ந்த அபாய நிலம்**: MCP அமைப்புகள் தனித்துவமான அபாயங்களை எதிர்கொள்கின்றன, இதில் ப்ராம்ட் ஊட்டுதல், கருவி விஷப்பொருள் தாக்குதல், அமர்வு கவர்ச்சி, குழப்பமான துணை பிரச்சனை, டோக்கன் பாஸ்த்ரூ பாதிப்புகள் மற்றும் அதிக உரிமைகள் உள்ளன, அவற்றுக்கு சிறப்பு தடுப்புகள் தேவை  
- **அங்கீகாரம் மற்றும் அங்கீகார சிறந்த நடைமுறைகள்**: வெளிப்புற அடையாள வழங்குநர்களை (Microsoft Entra ID) பயன்படுத்தி வலுவான அங்கீகாரத்தை செயல்படுத்தவும், சரியான டோக்கன் சரிபார்ப்பை கடைபிடிக்கவும், MCP சேவையகத்துக்காக தெளிவாக வழங்கப்படாத டோக்கன்களை ஒருபோதும் ஏற்கக்கூடாது  
- **AI தாக்குதல் தடுப்பு**: Microsoft Prompt Shields மற்றும் Azure Content Safety ஐ பயன்படுத்தி நேரடி அல்லாத ப்ராம்ட் ஊட்டுதல் மற்றும் கருவி விஷப்பொருள் தாக்குதல்களை எதிர்க்கவும், கருவி மெட்டாடேட்டாவை சரிபார்க்கவும் மற்றும் இயக்க மாற்றங்களை கண்காணிக்கவும்  
- **அமர்வு மற்றும் பரிமாற்ற பாதுகாப்பு**: பயனர் அடையாளங்களுக்கு பிணைக்கப்பட்ட குறியாக்க ரகசியமான, சீரற்ற அமர்வு ஐடிகளைப் பயன்படுத்தவும், சரியான அமர்வு ஆயுள் மேலாண்மையை செயல்படுத்தவும், அங்கீகாரத்திற்காக அமர்வுகளை ஒருபோதும் பயன்படுத்தக்கூடாது  
- **OAuth பாதுகாப்பு சிறந்த நடைமுறைகள்**: இயக்கத்தில் பதிவு செய்யப்பட்ட கிளையன்டுகளுக்கான தெளிவான பயனர் ஒப்புதலை வழங்கி குழப்பமான துணை தாக்குதல்களைத் தடுக்கும், PKCE உடன் சரியான OAuth 2.1 செயல்பாடு மற்றும் கடுமையான மறுவழி URI சரிபார்ப்பு  
- **டோக்கன் பாதுகாப்பு 원칙ங்கள்**: டோக்கன் பாஸ்த்ரூ எதிர்மறை மாதிரிகளைத் தவிர்க்கவும், டோக்கன் கேட்கும் நபர் கோரிக்கைகளை சரிபார்க்கவும், பாதுகாப்பான சுழற்சியுடன் குறுகிய ஆயுள் டோக்கன்களை செயல்படுத்தவும் மற்றும் தெளிவான நம்பிக்கை எல்லைகளை பராமரிக்கவும்  
- **விரிவான வழங்கல் சங்கிலி பாதுகாப்பு**: அனைத்து AI சூழல் கூறுகளையும் (மாதிரிகள், எம்பெட்டிங்கள், சூழல் வழங்குநர்கள், வெளிப்புற APIகள்) பாரம்பரிய மென்பொருள் சார்புகளுக்கு சமமான பாதுகாப்பு கடுமையுடன் கையாளவும்  
- **தொடர்ச்சியான முன்னேற்றம்**: விரைவாக மாறும் MCP விவரக்குறிப்புகளுடன் புதுப்பித்து, பாதுகாப்பு சமூக தரநிலைகளுக்கு பங்களித்து, நெறிமுறை வளர்ச்சியுடன் பாதுகாப்பு நிலைகளை தக்கவைத்துக் கொள்ளவும்  
- **Microsoft பாதுகாப்பு ஒருங்கிணைப்பு**: Microsoft இன் விரிவான பாதுகாப்பு சூழலை (Prompt Shields, Azure Content Safety, GitHub Advanced Security, Entra ID) பயன்படுத்தி MCP வெளியீட்டின் பாதுகாப்பை மேம்படுத்தவும்

விரிவான பாதுகாப்பு வழிகாட்டலுக்கு, இந்த பிரிவில் உள்ள சிறப்பு ஆவணங்களை பார்க்கவும்:

- **[MCP பாதுகாப்பு சிறந்த நடைமுறைகள் 2025](./mcp-security-best-practices-2025.md)** - MCP செயலாக்கங்களுக்கான முழுமையான பாதுகாப்பு சிறந்த நடைமுறைகள்
- **[Azure உள்ளடக்க பாதுகாப்பு செயலாக்கம்](./azure-content-safety-implementation.md)** - Azure உள்ளடக்க பாதுகாப்பு ஒருங்கிணைப்புக்கான நடைமுறை உதாரணங்கள்  
- **[MCP பாதுகாப்பு கட்டுப்பாடுகள் 2025](./mcp-security-controls-2025.md)** - MCP செயலாக்கங்களுக்கான சமீபத்திய பாதுகாப்பு கட்டுப்பாடுகள் மற்றும் தொழில்நுட்பங்கள்
- **[MCP சிறந்த நடைமுறைகள் விரைவு குறிப்பு](./mcp-best-practices.md)** - அவசியமான MCP பாதுகாப்பு நடைமுறைகளுக்கான விரைவு குறிப்பு வழிகாட்டி
