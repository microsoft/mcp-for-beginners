# परिवर्तन सूची: MCP फॉर बिगिनर्स पाठ्यक्रम

यह दस्तावेज़ मॉडल कॉन्टेक्स्ट प्रोटोकॉल (MCP) फॉर बिगिनर्स पाठ्यक्रम में किए गए सभी महत्वपूर्ण परिवर्तनों का रिकॉर्ड है। परिवर्तन उल्टे कालानुक्रमिक क्रम में दर्ज किए गए हैं (नवीनतम परिवर्तन पहले)।

## 18 दिसंबर, 2025

### सुरक्षा दस्तावेज़ीकरण अपडेट - MCP विनिर्देशन 2025-11-25

#### MCP सुरक्षा सर्वोत्तम प्रथाएँ (02-Security/mcp-best-practices.md) - विनिर्देशन संस्करण अपडेट
- **प्रोटोकॉल संस्करण अपडेट**: नवीनतम MCP विनिर्देशन 2025-11-25 (25 नवंबर, 2025 को जारी) का संदर्भ दिया गया
  - सभी विनिर्देशन संस्करण संदर्भ 2025-06-18 से 2025-11-25 में अपडेट किए गए
  - दस्तावेज़ तिथि संदर्भ अगस्त 18, 2025 से दिसंबर 18, 2025 में अपडेट किए गए
  - सभी विनिर्देशन URL वर्तमान दस्तावेज़ीकरण की ओर इंगित करते हैं, सत्यापित किया गया
- **सामग्री सत्यापन**: नवीनतम मानकों के अनुसार सुरक्षा सर्वोत्तम प्रथाओं का व्यापक सत्यापन
  - **Microsoft सुरक्षा समाधान**: Prompt Shields (पूर्व में "Jailbreak risk detection"), Azure Content Safety, Microsoft Entra ID, और Azure Key Vault के लिए वर्तमान शब्दावली और लिंक सत्यापित किए गए
  - **OAuth 2.1 सुरक्षा**: नवीनतम OAuth सुरक्षा सर्वोत्तम प्रथाओं के साथ संरेखण की पुष्टि की गई
  - **OWASP मानक**: LLMs के लिए OWASP टॉप 10 संदर्भ वर्तमान हैं, सत्यापित किया गया
  - **Azure सेवाएँ**: सभी Microsoft Azure दस्तावेज़ीकरण लिंक और सर्वोत्तम प्रथाएँ सत्यापित की गईं
- **मानक संरेखण**: सभी संदर्भित सुरक्षा मानक वर्तमान हैं, पुष्टि की गई
  - NIST AI जोखिम प्रबंधन फ्रेमवर्क
  - ISO 27001:2022
  - OAuth 2.1 सुरक्षा सर्वोत्तम प्रथाएँ
  - Azure सुरक्षा और अनुपालन फ्रेमवर्क
- **कार्यान्वयन संसाधन**: सभी कार्यान्वयन गाइड लिंक और संसाधन सत्यापित किए गए
  - Azure API प्रबंधन प्रमाणीकरण पैटर्न
  - Microsoft Entra ID एकीकरण गाइड
  - Azure Key Vault सीक्रेट्स प्रबंधन
  - DevSecOps पाइपलाइंस और निगरानी समाधान

### दस्तावेज़ीकरण गुणवत्ता आश्वासन
- **विनिर्देशन अनुपालन**: सुनिश्चित किया गया कि सभी अनिवार्य MCP सुरक्षा आवश्यकताएँ (MUST/MUST NOT) नवीनतम विनिर्देशन के अनुरूप हों
- **संसाधन नवीनता**: Microsoft दस्तावेज़ीकरण, सुरक्षा मानकों, और कार्यान्वयन गाइड के सभी बाहरी लिंक सत्यापित किए गए
- **सर्वोत्तम प्रथाओं कवरेज**: प्रमाणीकरण, प्राधिकरण, AI-विशिष्ट खतरों, आपूर्ति श्रृंखला सुरक्षा, और एंटरप्राइज़ पैटर्न का व्यापक कवरेज सुनिश्चित किया गया

## 6 अक्टूबर, 2025

### शुरुआत अनुभाग विस्तार – उन्नत सर्वर उपयोग और सरल प्रमाणीकरण

#### उन्नत सर्वर उपयोग (03-GettingStarted/10-advanced)
- **नया अध्याय जोड़ा गया**: उन्नत MCP सर्वर उपयोग के लिए व्यापक गाइड प्रस्तुत किया गया, जिसमें नियमित और निम्न-स्तरीय सर्वर आर्किटेक्चर दोनों शामिल हैं।
  - **नियमित बनाम निम्न-स्तरीय सर्वर**: दोनों दृष्टिकोणों के लिए Python और TypeScript में विस्तृत तुलना और कोड उदाहरण।
  - **हैंडलर-आधारित डिज़ाइन**: स्केलेबल, लचीले सर्वर कार्यान्वयन के लिए टूल/संसाधन/प्रॉम्प्ट प्रबंधन का हैंडलर-आधारित स्पष्टीकरण।
  - **व्यावहारिक पैटर्न**: वास्तविक दुनिया के परिदृश्य जहां निम्न-स्तरीय सर्वर पैटर्न उन्नत सुविधाओं और आर्किटेक्चर के लिए लाभकारी हैं।

#### सरल प्रमाणीकरण (03-GettingStarted/11-simple-auth)
- **नया अध्याय जोड़ा गया**: MCP सर्वरों में सरल प्रमाणीकरण लागू करने के लिए चरण-दर-चरण गाइड।
  - **प्रमाणीकरण अवधारणाएँ**: प्रमाणीकरण बनाम प्राधिकरण और क्रेडेंशियल हैंडलिंग का स्पष्ट स्पष्टीकरण।
  - **मूलभूत प्रमाणीकरण कार्यान्वयन**: Python (Starlette) और TypeScript (Express) में मिडलवेयर-आधारित प्रमाणीकरण पैटर्न, कोड नमूनों के साथ।
  - **उन्नत सुरक्षा की ओर प्रगति**: सरल प्रमाणीकरण से शुरू करने और OAuth 2.1 तथा RBAC तक बढ़ने के लिए मार्गदर्शन, उन्नत सुरक्षा मॉड्यूल के संदर्भ के साथ।

ये जोड़ MCP सर्वर कार्यान्वयन को अधिक मजबूत, सुरक्षित, और लचीला बनाने के लिए व्यावहारिक, हाथों-हाथ मार्गदर्शन प्रदान करते हैं, मौलिक अवधारणाओं को उन्नत उत्पादन पैटर्न के साथ जोड़ते हैं।

  - होस्ट अब स्पष्ट रूप से AI अनुप्रयोग के रूप में परिभाषित, जो कई MCP क्लाइंट कनेक्शनों का समन्वय करता है
  - क्लाइंट को प्रोटोकॉल कनेक्टर के रूप में वर्णित, जो एक-से-एक सर्वर संबंध बनाए रखता है
  - सर्वर को स्थानीय बनाम दूरस्थ तैनाती परिदृश्यों के साथ उन्नत किया गया
- **प्रिमिटिव पुनर्गठन**: सर्वर और क्लाइंट प्रिमिटिव का पूर्ण पुनर्गठन
  - सर्वर प्रिमिटिव: संसाधन (डेटा स्रोत), प्रॉम्प्ट (टेम्पलेट), टूल (निष्पादन योग्य फ़ंक्शन) विस्तृत व्याख्याओं और उदाहरणों के साथ
  - क्लाइंट प्रिमिटिव: सैंपलिंग (LLM पूर्णताएँ), एलिसिटेशन (उपयोगकर्ता इनपुट), लॉगिंग (डिबगिंग/निगरानी)
  - वर्तमान डिस्कवरी (`*/list`), पुनःप्राप्ति (`*/get`), और निष्पादन (`*/call`) विधि पैटर्न के साथ अद्यतन
- **प्रोटोकॉल वास्तुकला**: दो-स्तरीय वास्तुकला मॉडल प्रस्तुत किया गया
  - डेटा लेयर: JSON-RPC 2.0 आधार, जीवनचक्र प्रबंधन और प्रिमिटिव के साथ
  - ट्रांसपोर्ट लेयर: STDIO (स्थानीय) और Streamable HTTP के साथ SSE (दूरस्थ) ट्रांसपोर्ट तंत्र
- **सुरक्षा फ्रेमवर्क**: स्पष्ट उपयोगकर्ता सहमति, डेटा गोपनीयता संरक्षण, टूल निष्पादन सुरक्षा, और ट्रांसपोर्ट लेयर सुरक्षा सहित व्यापक सुरक्षा सिद्धांत
- **संचार पैटर्न**: प्रारंभ, खोज, निष्पादन, और सूचना प्रवाह दिखाने के लिए प्रोटोकॉल संदेशों को अद्यतन किया गया
- **कोड उदाहरण**: वर्तमान MCP SDK पैटर्न को प्रतिबिंबित करने के लिए बहुभाषी उदाहरण (.NET, Java, Python, JavaScript) ताज़ा किए गए

  - Azure Content Safety एकीकरण पैटर्न
