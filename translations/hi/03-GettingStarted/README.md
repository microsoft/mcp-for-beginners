इस सेक्शन में कई पाठ शामिल हैं:

- **1 आपका पहला सर्वर**, इस पहले पाठ में, आप सीखेंगे कि अपने पहले सर्वर को कैसे बनाएं और इंस्पेक्टर टूल के साथ इसका निरीक्षण कैसे करें, जो आपके सर्वर का परीक्षण और डिबग करने का एक मूल्यवान तरीका है, [पाठ के लिए](01-first-server/README.md)

- **2 क्लाइंट**, इस पाठ में, आप सीखेंगे कि एक क्लाइंट कैसे लिखा जाए जो आपके सर्वर से जुड़ सके, [पाठ के लिए](02-client/README.md)

- **3 LLM के साथ क्लाइंट**, क्लाइंट लिखने का एक और बेहतर तरीका है कि इसमें एक LLM जोड़ा जाए ताकि यह "मोलभाव" कर सके कि सर्वर के साथ क्या करना है, [पाठ के लिए](03-llm-client/README.md)

- **4 Visual Studio Code में GitHub Copilot Agent मोड का उपयोग करते हुए सर्वर का उपभोग**। यहाँ हम MCP सर्वर को Visual Studio Code के भीतर चलाने पर ध्यान दे रहे हैं, [पाठ के लिए](04-vscode/README.md)

- **5 stdio ट्रांसपोर्ट सर्वर** stdio ट्रांसपोर्ट स्थानीय MCP सर्वर-क्लाइंट संचार के लिए अनुशंसित मानक है, जो सुरक्षित subprocess-आधारित संचार के साथ अंतर्निहित प्रक्रिया पृथक्करण प्रदान करता है, [पाठ के लिए](05-stdio-server/README.md)

- **6 MCP के साथ HTTP स्ट्रीमिंग (स्ट्रीम करने योग्य HTTP)**। आधुनिक HTTP स्ट्रीमिंग ट्रांसपोर्ट के बारे में सीखें (यह दूरस्थ MCP सर्वरों के लिए अनुशंसित तरीका है [MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/basic/transports/#streamable-http) के अनुसार), प्रगति सूचनाएं, और स्ट्रीम करने योग्य HTTP का उपयोग करके स्केलेबल, वास्तविक समय MCP सर्वर और क्लाइंट कैसे इम्प्लीमेंट करें। [पाठ के लिए](06-http-streaming/README.md)

- **7 VSCode के लिए AI टूलकिट का उपयोग** ताकि आप अपने MCP क्लाइंट और सर्वरों का उपभोग और परीक्षण कर सकें, [पाठ के लिए](07-aitk/README.md)

- **8 परीक्षण**। यहाँ हम विशेष रूप से इस पर ध्यान देंगे कि हम अपने सर्वर और क्लाइंट को विभिन्न तरीकों से कैसे परीक्षण कर सकते हैं, [पाठ के लिए](08-testing/README.md)

- **9 परिनियोजन**। यह अध्याय आपकी MCP समाधानों के विभिन्न परिनियोजन तरीकों पर नजर डालेगा, [पाठ के लिए](09-deployment/README.md)

- **10 उन्नत सर्वर उपयोग**। यह अध्याय उन्नत सर्वर उपयोग को कवर करता है, [पाठ के लिए](./10-advanced/README.md)

- **11 प्रमाणीकरण**। यह अध्याय सरल प्रमाणीकरण जोड़ने के तरीके को कवर करता है, बेसिक ऑथ से लेकर JWT और RBAC के उपयोग तक। आपको यहां से शुरू करने और फिर अध्याय 5 के उन्नत विषयों को देखने और अध्याय 2 में सिफारिशों के माध्यम से अतिरिक्त सुरक्षा मजबूत करने के लिए प्रोत्साहित किया जाता है, [पाठ के लिए](./11-simple-auth/README.md)

- **12 MCP होस्ट्स**। लोकप्रिय MCP होस्ट क्लाइंट जैसे Claude Desktop, Cursor, Cline, और Windsurf को कॉन्फ़िगर और उपयोग करें। ट्रांसपोर्ट प्रकार और समस्या निवारण सीखें, [पाठ के लिए](./12-mcp-hosts/README.md)

- **13 MCP इंस्पेक्टर**। MCP इंस्पेक्टर टूल का उपयोग करके अपने MCP सर्वरों का इंटरैक्टिव रूप से डिबग और परीक्षण करें। टूल, रिसोर्स, और प्रोटोकॉल संदेशों के समस्या निवारण सीखें, [पाठ के लिए](./13-mcp-inspector/README.md)

- **14 सैंपलिंग**। MCP सर्वर बनाएं जो MCP क्लाइंट्स के साथ LLM संबंधित कार्यों पर सहयोग करते हैं। [पाठ के लिए](./14-sampling/README.md)

- **15 MCP ऐप्स**। MCP सर्वर बनाएं जो UI निर्देशों के साथ भी उत्तर दें, [पाठ के लिए](./15-mcp-apps/README.md)

Model Context Protocol (MCP) एक खुला प्रोटोकॉल है जो यह मानकीकृत करता है कि एप्लिकेशन LLMs को संदर्भ कैसे प्रदान करें। MCP को AI एप्लिकेशन के लिए USB-C पोर्ट की तरह सोचें - यह AI मॉडल को विभिन्न डेटा स्रोतों और उपकरणों से जोड़ने का एक मानकीकृत तरीका प्रदान करता है।

## सीखने के उद्देश्य

इस पाठ के अंत तक, आप सक्षम होंगे:

- C#, Java, Python, TypeScript, और JavaScript में MCP के लिए विकास वातावरण स्थापित करना
- कस्टम फीचर्स (रिसोर्सेस, प्रॉम्प्ट्स, और टूल्स) के साथ बेसिक MCP सर्वर बनाना और परिनियोजित करना
- होस्ट एप्लिकेशन बनाना जो MCP सर्वरों से जुड़ते हैं
- MCP इम्प्लीमेंटेशन का परीक्षण और डिबग करना
- सामान्य सेटअप चुनौतियों और उनके समाधानों को समझना
- अपने MCP इम्प्लीमेंटेशन को लोकप्रिय LLM सेवाओं से जोड़ना

## अपना MCP वातावरण सेट करना

MCP के साथ काम शुरू करने से पहले, यह जरूरी है कि आप अपना विकास वातावरण तैयार करें और बुनियादी कार्यप्रवाह को समझें। यह अनुभाग आपको प्रारंभिक सेटअप चरणों के माध्यम से मार्गदर्शन करेगा ताकि MCP के साथ एक सुलभ शुरुआत सुनिश्चित हो सके।

### पूर्व आवश्यकताएं

MCP विकास में गोता लगाने से पहले यह सुनिश्चित करें कि आपके पास है:

- **विकास वातावरण**: आपकी चुनी हुई भाषा (C#, Java, Python, TypeScript, या JavaScript) के लिए
- **IDE/एडिटर**: Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm, या कोई भी आधुनिक कोड एडिटर
- **पैकेज प्रबंधक**: NuGet, Maven/Gradle, pip, या npm/yarn
- **API कुंजी**: किसी भी AI सेवा के लिए जिन्हें आप अपने होस्ट एप्लिकेशन में उपयोग करने की योजना बनाते हैं

### आधिकारिक SDKs

आगामी अध्यायों में आप Python, TypeScript, Java और .NET का उपयोग करते हुए बनाए गए समाधान देखेंगे। यहां सभी आधिकारिक समर्थित SDKs हैं।

MCP कई भाषाओं के लिए आधिकारिक SDKs प्रदान करता है (जो [MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/) के अनुरूप हैं):
- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk) - माइक्रोसॉफ्ट के सहयोग से मेन्टेन्ड
- [Java SDK](https://github.com/modelcontextprotocol/java-sdk) - Spring AI के सहयोग से मेन्टेन्ड
- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) - आधिकारिक TypeScript इम्प्लीमेंटेशन
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk) - आधिकारिक Python इम्प्लीमेंटेशन (FastMCP)
- [Kotlin SDK](https://github.com/modelcontextprotocol/kotlin-sdk) - आधिकारिक Kotlin इम्प्लीमेंटेशन
- [Swift SDK](https://github.com/modelcontextprotocol/swift-sdk) - Loopwork AI के सहयोग से मेन्टेन्ड
- [Rust SDK](https://github.com/modelcontextprotocol/rust-sdk) - आधिकारिक Rust इम्प्लीमेंटेशन
- [Go SDK](https://github.com/modelcontextprotocol/go-sdk) - आधिकारिक Go इम्प्लीमेंटेशन

## मुख्य बिंदु

- भाषा-विशिष्ट SDKs के साथ MCP विकास वातावरण स्थापित करना सरल है
- MCP सर्वर बनाना टूल्स बनाना और उन्हें स्पष्ट स्कीमा के साथ रजिस्टर करना शामिल है
- MCP क्लाइंट सर्वर और मॉडल से जुड़कर विस्तारित क्षमताओं का लाभ उठाते हैं
- परीक्षण और डिबगिंग भरोसेमंद MCP इम्प्लीमेंटेशन के लिए आवश्यक हैं
- परिनियोजन विकल्प स्थानीय विकास से लेकर क्लाउड-आधारित समाधानों तक रहते हैं
