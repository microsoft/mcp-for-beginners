## परीक्षण और डीबगिंग

अपने MCP सर्वर का परीक्षण शुरू करने से पहले, उपलब्ध टूल और डीबगिंग के लिए सर्वोत्तम प्रथाओं को समझना महत्वपूर्ण है। प्रभावी परीक्षण यह सुनिश्चित करता है कि आपका सर्वर अपेक्षित व्यवहार करता है और आपको जल्दी से समस्याओं की पहचान और समाधान करने में मदद करता है। निम्न खंड में आपके MCP कार्यान्वयन को मान्य करने के लिए अनुशंसित दृष्टिकोणों का वर्णन किया गया है।

## अवलोकन

यह पाठ सही परीक्षण दृष्टिकोण चुनने और सबसे प्रभावी परीक्षण उपकरण का उपयोग करने के बारे में है।

## सीखने के उद्देश्य

इस पाठ के अंत तक, आप सक्षम होंगे:

- परीक्षण के विभिन्न दृष्टिकोणों का वर्णन करने के लिए।
- अपने कोड का प्रभावी परीक्षण करने के लिए विभिन्न उपकरणों का उपयोग करने के लिए।

## MCP सर्वरों का परीक्षण

MCP आपके सर्वरों का परीक्षण और डीबग करने में आपकी मदद करने के लिए उपकरण प्रदान करता है:

- **MCP इंस्पेक्टर**: एक कमांड लाइन टूल जो CLI उपकरण के रूप में और एक दृश्य उपकरण के रूप में चलाया जा सकता है।
- **मैनुअल परीक्षण**: आप curl जैसे टूल का उपयोग करके वेब अनुरोध चला सकते हैं, लेकिन कोई भी HTTP चला सकने वाला टूल काम करेगा।
- **यूनिट परीक्षण**: यह संभव है कि आप अपनी पसंद के परीक्षण फ्रेमवर्क का उपयोग करके सर्वर और क्लाइंट दोनों की विशेषताओं का परीक्षण करें।

### MCP इंस्पेक्टर का उपयोग

हमने इस उपकरण के उपयोग का वर्णन पिछले पाठों में किया है लेकिन आइए इसे एक उच्च स्तरीय रूप से थोड़ा चर्चा करें। यह Node.js में बनाया गया एक उपकरण है और आप इसे `npx` निष्पाद्य को कॉल करके उपयोग कर सकते हैं, जो टूल को अस्थाई रूप से डाउनलोड और इंस्टॉल करेगा और आपकी अनुरोध प्रक्रिया पूरी होने के बाद खुद को साफ़ कर देगा।

[MCP इंस्पेक्टर](https://github.com/modelcontextprotocol/inspector) आपकी मदद करता है:

- **सर्वर क्षमताओं की खोज**: उपलब्ध संसाधनों, टूल्स, और प्रम्प्ट्स का स्वतः पता लगाना
- **टूल निष्पादन का परीक्षण**: विभिन्न पैरामीटर आज़माएं और रीयल-टाइम में प्रतिक्रियाएं देखें
- **सर्वर मेटाडेटा देखें**: सर्वर जानकारी, स्कीमास, और कॉन्फ़िगरेशन की जांच करें

टूल का एक सामान्य चलन इस प्रकार दिखता है:

```bash
npx @modelcontextprotocol/inspector node build/index.js
```

ऊपर दिया गया कमांड एक MCP और इसके दृश्य इंटरफेस को शुरू करता है और आपके ब्राउज़र में एक स्थानीय वेब इंटरफ़ेस लॉन्च करता है। आप अपने पंजीकृत MCP सर्वरों, उनके उपलब्ध टूल्स, संसाधनों, और प्रम्प्ट्स को दिखाते हुए एक डैशबोर्ड देख सकते हैं। यह इंटरफ़ेस आपको इंटरैक्टिव रूप से टूल निष्पादन का परीक्षण करने, सर्वर मेटाडेटा का निरीक्षण करने, और रीयल-टाइम प्रतिक्रियाएँ देखने की अनुमति देता है, जिससे आपके MCP सर्वर कार्यान्वयन को मान्य करना और डीबग करना आसान हो जाता है।

यह इस प्रकार दिख सकता है: ![Inspector](../../../../translated_images/hi/connect.141db0b2bd05f096.webp)

आप इस टूल को CLI मोड में भी चला सकते हैं जिसमें आप `--cli` विकल्प जोड़ते हैं। यहाँ "CLI" मोड में टूल को चलाने का एक उदाहरण दिया गया है जो सर्वर पर सभी टूल्स की सूची देता है:

```sh
npx @modelcontextprotocol/inspector --cli node build/index.js --method tools/list
```

### मैनुअल परीक्षण

सर्वर क्षमताओं का परीक्षण करने के लिए इंस्पेक्टर टूल चलाने के अलावा, एक समान दृष्टिकोण HTTP का उपयोग करने में सक्षम क्लाइंट चलाना है जैसे कि curl।

curl के साथ, आप सीधे HTTP अनुरोधों का उपयोग करके MCP सर्वरों का परीक्षण कर सकते हैं:

```bash
# उदाहरण: टेस्ट सर्वर मेटाडेटा
curl http://localhost:3000/v1/metadata

# उदाहरण: एक टूल निष्पादित करें
curl -X POST http://localhost:3000/v1/tools/execute \
  -H "Content-Type: application/json" \
  -d '{"name": "calculator", "parameters": {"expression": "2+2"}}'
```

जैसा कि आप ऊपर curl के उपयोग से देख सकते हैं, आप उपकरण को उसके नाम और पैरामीटर वाले पेलोड का उपयोग करके कॉल करने के लिए POST अनुरोध का उपयोग करते हैं। वह तरीका उपयोग करें जो आपको सबसे अच्छा लगे। CLI टूलों का उपयोग सामान्यतः तेज होता है और उन्हें स्क्रिप्ट किया जा सकता है जो CI/CD वातावरण में उपयोगी हो सकता है।

### यूनिट परीक्षण

अपने टूल्स और संसाधनों के लिए यूनिट टेस्ट बनाएँ ताकि यह सुनिश्चित किया जा सके कि वे अपेक्षित रूप से काम करें। यहां कुछ उदाहरण परीक्षण कोड दिया गया है।

```python
import pytest

from mcp.server.fastmcp import FastMCP
from mcp.shared.memory import (
    create_connected_server_and_client_session as create_session,
)

# पूरे मॉड्यूल को एसिंक्रोनस परीक्षणों के लिए चिह्नित करें
pytestmark = pytest.mark.anyio


async def test_list_tools_cursor_parameter():
    """Test that the cursor parameter is accepted for list_tools.

    Note: FastMCP doesn't currently implement pagination, so this test
    only verifies that the cursor parameter is accepted by the client.
    """

 server = FastMCP("test")

    # कुछ परीक्षण उपकरण बनाएं
    @server.tool(name="test_tool_1")
    async def test_tool_1() -> str:
        """First test tool"""
        return "Result 1"

    @server.tool(name="test_tool_2")
    async def test_tool_2() -> str:
        """Second test tool"""
        return "Result 2"

    async with create_session(server._mcp_server) as client_session:
        # कर्सर पैरामीटर के बिना परीक्षण करें (छोड़ा गया)
        result1 = await client_session.list_tools()
        assert len(result1.tools) == 2

        # कर्सर=None के साथ परीक्षण करें
        result2 = await client_session.list_tools(cursor=None)
        assert len(result2.tools) == 2

        # कर्सर को स्ट्रिंग के रूप में लेकर परीक्षण करें
        result3 = await client_session.list_tools(cursor="some_cursor_value")
        assert len(result3.tools) == 2

        # खाली स्ट्रिंग कर्सर के साथ परीक्षण करें
        result4 = await client_session.list_tools(cursor="")
        assert len(result4.tools) == 2
    
```

पिछला कोड निम्नलिखित करता है:

- pytest फ्रेमवर्क का उपयोग करता है जो आपको फ़ंक्शन्स के रूप में परीक्षण बनाने और assert स्टेटमेंट का उपयोग करने की अनुमति देता है।
- दो विभिन्न टूल्स के साथ एक MCP सर्वर बनाता है।
- कुछ शर्तों की जांच करने के लिए `assert` स्टेटमेंट का उपयोग करता है।

[पूर्ण फाइल यहाँ देखें](https://github.com/modelcontextprotocol/python-sdk/blob/main/tests/client/test_list_methods_cursor.py)

ऊपर दी गई फाइल के आधार पर, आप अपने सर्वर का परीक्षण कर सकते हैं ताकि क्षमताओं के सही ढंग से बनाए जाने की पुष्टि हो सके।

सभी प्रमुख SDK में इसी तरह के परीक्षण अनुभाग होते हैं ताकि आप अपने चुने हुए रनटाइम के अनुसार समायोजन कर सकें।
