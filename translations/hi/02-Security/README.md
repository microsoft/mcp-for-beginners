# MCP सुरक्षा: एआई सिस्टम के लिए व्यापक सुरक्षा

[![MCP Security Best Practices](../../../translated_images/hi/03.175aed6dedae133f9d41e49cefd0f0a9a39c3317e1eaa7ef7182696af7534308.png)](https://youtu.be/88No8pw706o)

_(इस पाठ का वीडियो देखने के लिए ऊपर की छवि पर क्लिक करें)_

सुरक्षा एआई सिस्टम डिज़ाइन का मूलभूत हिस्सा है, इसलिए हम इसे अपने दूसरे अनुभाग के रूप में प्राथमिकता देते हैं। यह Microsoft के [Secure Future Initiative](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) के **Secure by Design** सिद्धांत के अनुरूप है।

मॉडल कॉन्टेक्स्ट प्रोटोकॉल (MCP) एआई-संचालित अनुप्रयोगों में शक्तिशाली नई क्षमताएँ लाता है, साथ ही पारंपरिक सॉफ़्टवेयर जोखिमों से परे अद्वितीय सुरक्षा चुनौतियाँ प्रस्तुत करता है। MCP सिस्टमों को स्थापित सुरक्षा चिंताओं (सुरक्षित कोडिंग, न्यूनतम विशेषाधिकार, आपूर्ति श्रृंखला सुरक्षा) के साथ-साथ नए एआई-विशिष्ट खतरों का सामना करना पड़ता है, जिनमें प्रॉम्प्ट इंजेक्शन, टूल पॉइज़निंग, सेशन हाईजैकिंग, कन्फ्यूज्ड डिप्टी हमले, टोकन पासथ्रू कमजोरियां, और गतिशील क्षमता संशोधन शामिल हैं।

यह पाठ MCP कार्यान्वयन में सबसे महत्वपूर्ण सुरक्षा जोखिमों का अन्वेषण करता है—जिसमें प्रमाणीकरण, प्राधिकरण, अत्यधिक अनुमतियाँ, अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन, सेशन सुरक्षा, कन्फ्यूज्ड डिप्टी समस्याएँ, टोकन प्रबंधन, और आपूर्ति श्रृंखला कमजोरियाँ शामिल हैं। आप इन जोखिमों को कम करने के लिए क्रियान्वयन योग्य नियंत्रण और सर्वोत्तम प्रथाएँ सीखेंगे, साथ ही Microsoft समाधानों जैसे Prompt Shields, Azure Content Safety, और GitHub Advanced Security का उपयोग करके अपने MCP परिनियोजन को मजबूत करेंगे।

## सीखने के उद्देश्य

इस पाठ के अंत तक, आप सक्षम होंगे:

- **MCP-विशिष्ट खतरों की पहचान करें**: MCP सिस्टमों में अद्वितीय सुरक्षा जोखिमों को पहचानें, जिनमें प्रॉम्प्ट इंजेक्शन, टूल पॉइज़निंग, अत्यधिक अनुमतियाँ, सेशन हाईजैकिंग, कन्फ्यूज्ड डिप्टी समस्याएँ, टोकन पासथ्रू कमजोरियां, और आपूर्ति श्रृंखला जोखिम शामिल हैं
- **सुरक्षा नियंत्रण लागू करें**: मजबूत प्रमाणीकरण, न्यूनतम विशेषाधिकार पहुँच, सुरक्षित टोकन प्रबंधन, सेशन सुरक्षा नियंत्रण, और आपूर्ति श्रृंखला सत्यापन सहित प्रभावी निवारण लागू करें
- **Microsoft सुरक्षा समाधानों का लाभ उठाएं**: MCP वर्कलोड सुरक्षा के लिए Microsoft Prompt Shields, Azure Content Safety, और GitHub Advanced Security को समझें और तैनात करें
- **टूल सुरक्षा का सत्यापन करें**: टूल मेटाडेटा सत्यापन के महत्व को पहचानें, गतिशील परिवर्तनों की निगरानी करें, और अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन हमलों से रक्षा करें
- **सर्वोत्तम प्रथाओं को एकीकृत करें**: स्थापित सुरक्षा मूल सिद्धांतों (सुरक्षित कोडिंग, सर्वर हार्डनिंग, ज़ीरो ट्रस्ट) को MCP-विशिष्ट नियंत्रणों के साथ संयोजित करें ताकि व्यापक सुरक्षा सुनिश्चित हो सके

# MCP सुरक्षा वास्तुकला और नियंत्रण

आधुनिक MCP कार्यान्वयन परतदार सुरक्षा दृष्टिकोणों की मांग करते हैं जो पारंपरिक सॉफ़्टवेयर सुरक्षा और एआई-विशिष्ट खतरों दोनों को संबोधित करते हैं। तेजी से विकसित हो रही MCP विनिर्देश अपनी सुरक्षा नियंत्रणों को परिपक्व कर रही है, जिससे उद्यम सुरक्षा वास्तुकला और स्थापित सर्वोत्तम प्रथाओं के साथ बेहतर एकीकरण संभव हो रहा है।

[Microsoft Digital Defense Report](https://aka.ms/mddr) के अनुसंधान से पता चलता है कि **98% रिपोर्ट किए गए उल्लंघन मजबूत सुरक्षा स्वच्छता द्वारा रोके जा सकते हैं**। सबसे प्रभावी सुरक्षा रणनीति मौलिक सुरक्षा प्रथाओं को MCP-विशिष्ट नियंत्रणों के साथ संयोजित करती है—सिद्ध बेसलाइन सुरक्षा उपाय कुल सुरक्षा जोखिम को कम करने में सबसे प्रभावशाली बने रहते हैं।

## वर्तमान सुरक्षा परिदृश्य

> **नोट:** यह जानकारी MCP सुरक्षा मानकों को **18 दिसंबर, 2025** तक दर्शाती है। MCP प्रोटोकॉल तेजी से विकसित हो रहा है, और भविष्य के कार्यान्वयन नए प्रमाणीकरण पैटर्न और उन्नत नियंत्रण पेश कर सकते हैं। नवीनतम मार्गदर्शन के लिए हमेशा वर्तमान [MCP विनिर्देश](https://spec.modelcontextprotocol.io/), [MCP GitHub रिपॉजिटरी](https://github.com/modelcontextprotocol), और [सुरक्षा सर्वोत्तम प्रथाएँ दस्तावेज़](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) देखें।

### MCP प्रमाणीकरण का विकास

MCP विनिर्देश ने प्रमाणीकरण और प्राधिकरण के दृष्टिकोण में महत्वपूर्ण विकास किया है:

- **मूल दृष्टिकोण**: प्रारंभिक विनिर्देशों में डेवलपर्स को कस्टम प्रमाणीकरण सर्वर लागू करने की आवश्यकता थी, जिसमें MCP सर्वर OAuth 2.0 प्राधिकरण सर्वर के रूप में कार्य करते थे जो सीधे उपयोगकर्ता प्रमाणीकरण प्रबंधित करते थे
- **वर्तमान मानक (2025-11-25)**: अपडेटेड विनिर्देश MCP सर्वरों को बाहरी पहचान प्रदाताओं (जैसे Microsoft Entra ID) को प्रमाणीकरण सौंपने की अनुमति देता है, जिससे सुरक्षा स्थिति में सुधार और कार्यान्वयन जटिलता में कमी आती है
- **ट्रांसपोर्ट लेयर सुरक्षा**: स्थानीय (STDIO) और दूरस्थ (Streamable HTTP) कनेक्शनों के लिए उचित प्रमाणीकरण पैटर्न के साथ सुरक्षित ट्रांसपोर्ट तंत्र के लिए उन्नत समर्थन

## प्रमाणीकरण और प्राधिकरण सुरक्षा

### वर्तमान सुरक्षा चुनौतियाँ

आधुनिक MCP कार्यान्वयन कई प्रमाणीकरण और प्राधिकरण चुनौतियों का सामना करते हैं:

### जोखिम और खतरे

- **गलत कॉन्फ़िगर किया गया प्राधिकरण लॉजिक**: MCP सर्वरों में दोषपूर्ण प्राधिकरण कार्यान्वयन संवेदनशील डेटा को उजागर कर सकता है और गलत तरीके से पहुँच नियंत्रण लागू कर सकता है
- **OAuth टोकन समझौता**: स्थानीय MCP सर्वर टोकन चोरी हमलावरों को सर्वरों का प्रतिरूपण करने और डाउनस्ट्रीम सेवाओं तक पहुँचने में सक्षम बनाती है
- **टोकन पासथ्रू कमजोरियां**: अनुचित टोकन हैंडलिंग सुरक्षा नियंत्रणों को बायपास और जवाबदेही में अंतराल पैदा करती है
- **अत्यधिक अनुमतियाँ**: अधिक विशेषाधिकार प्राप्त MCP सर्वर न्यूनतम विशेषाधिकार सिद्धांत का उल्लंघन करते हैं और हमले की सतह बढ़ाते हैं

#### टोकन पासथ्रू: एक गंभीर एंटी-पैटर्न

**टोकन पासथ्रू वर्तमान MCP प्राधिकरण विनिर्देश में स्पष्ट रूप से निषिद्ध है** क्योंकि इसके गंभीर सुरक्षा परिणाम हैं:

##### सुरक्षा नियंत्रण परिपथ
- MCP सर्वर और डाउनस्ट्रीम API महत्वपूर्ण सुरक्षा नियंत्रण (रेट लिमिटिंग, अनुरोध सत्यापन, ट्रैफ़िक निगरानी) लागू करते हैं जो उचित टोकन सत्यापन पर निर्भर करते हैं
- सीधे क्लाइंट-से-API टोकन उपयोग इन आवश्यक सुरक्षा उपायों को बायपास करता है, जिससे सुरक्षा वास्तुकला कमजोर होती है

##### जवाबदेही और ऑडिट चुनौतियाँ  
- MCP सर्वर यह नहीं पहचान सकते कि क्लाइंट्स अपस्ट्रीम-इश्यूड टोकन का उपयोग कर रहे हैं, जिससे ऑडिट ट्रेल टूट जाती है
- डाउनस्ट्रीम संसाधन सर्वर लॉग अनुरोध के वास्तविक MCP सर्वर मध्यस्थों के बजाय भ्रामक अनुरोध मूल दिखाते हैं
- घटना जांच और अनुपालन ऑडिटिंग काफी कठिन हो जाती है

##### डेटा चोरी जोखिम
- अप्रमाणित टोकन दावे चोरी किए गए टोकन के साथ दुर्भावनापूर्ण अभिनेताओं को MCP सर्वरों का उपयोग डेटा चोरी के लिए प्रॉक्सी के रूप में करने की अनुमति देते हैं
- ट्रस्ट सीमा उल्लंघन अनधिकृत पहुँच पैटर्न की अनुमति देते हैं जो इच्छित सुरक्षा नियंत्रणों को बायपास करते हैं

##### बहु-सेवा हमला वेक्टर
- समझौता किए गए टोकन कई सेवाओं द्वारा स्वीकार किए जाने पर जुड़े सिस्टमों में पार्श्व आंदोलन सक्षम करते हैं
- जब टोकन मूल सत्यापित नहीं हो पाते तो सेवाओं के बीच ट्रस्ट मान्यताएँ उल्लंघन हो सकती हैं

### सुरक्षा नियंत्रण और निवारण

**महत्वपूर्ण सुरक्षा आवश्यकताएँ:**

> **अनिवार्य**: MCP सर्वर **किसी भी ऐसे टोकन को स्वीकार नहीं कर सकते जो स्पष्ट रूप से MCP सर्वर के लिए जारी नहीं किए गए हों**

#### प्रमाणीकरण और प्राधिकरण नियंत्रण

- **कठोर प्राधिकरण समीक्षा**: MCP सर्वर प्राधिकरण लॉजिक का व्यापक ऑडिट करें ताकि केवल इच्छित उपयोगकर्ता और क्लाइंट संवेदनशील संसाधनों तक पहुँच सकें
  - **कार्यान्वयन मार्गदर्शिका**: [Azure API Management as Authentication Gateway for MCP Servers](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)
  - **पहचान एकीकरण**: [MCP सर्वर प्रमाणीकरण के लिए Microsoft Entra ID का उपयोग](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **सुरक्षित टोकन प्रबंधन**: [Microsoft के टोकन सत्यापन और जीवनचक्र सर्वोत्तम प्रथाओं](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens) को लागू करें
  - टोकन ऑडियंस दावों का MCP सर्वर पहचान से मेल होना सत्यापित करें
  - उचित टोकन रोटेशन और समाप्ति नीतियाँ लागू करें
  - टोकन रिप्ले हमलों और अनधिकृत उपयोग को रोकें

- **सुरक्षित टोकन भंडारण**: टोकन भंडारण को एन्क्रिप्शन के साथ सुरक्षित करें, चाहे वह स्थिर हो या ट्रांज़िट में
  - **सर्वोत्तम प्रथाएँ**: [सुरक्षित टोकन भंडारण और एन्क्रिप्शन दिशानिर्देश](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### पहुँच नियंत्रण कार्यान्वयन

- **न्यूनतम विशेषाधिकार सिद्धांत**: MCP सर्वरों को केवल आवश्यक न्यूनतम अनुमतियाँ प्रदान करें
  - नियमित अनुमति समीक्षा और अद्यतन ताकि विशेषाधिकार वृद्धि रोकी जा सके
  - **Microsoft दस्तावेज़ीकरण**: [सुरक्षित न्यूनतम विशेषाधिकार पहुँच](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **भूमिका-आधारित पहुँच नियंत्रण (RBAC)**: सूक्ष्म भूमिका असाइनमेंट लागू करें
  - भूमिकाओं को विशिष्ट संसाधनों और क्रियाओं तक सीमित करें
  - व्यापक या अनावश्यक अनुमतियों से बचें जो हमले की सतह बढ़ाती हैं

- **निरंतर अनुमति निगरानी**: निरंतर पहुँच ऑडिटिंग और निगरानी लागू करें
  - अनुमतियों के उपयोग पैटर्न की असामान्यताओं के लिए निगरानी करें
  - अत्यधिक या अप्रयुक्त विशेषाधिकारों को शीघ्रता से ठीक करें

## एआई-विशिष्ट सुरक्षा खतरे

### प्रॉम्प्ट इंजेक्शन और टूल मैनिपुलेशन हमले

आधुनिक MCP कार्यान्वयन जटिल एआई-विशिष्ट हमले वेक्टरों का सामना करते हैं जिन्हें पारंपरिक सुरक्षा उपाय पूरी तरह से संबोधित नहीं कर पाते:

#### **अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन (क्रॉस-डोमेन प्रॉम्प्ट इंजेक्शन)**

**अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन** MCP-सक्षम एआई सिस्टमों में सबसे गंभीर कमजोरियों में से एक है। हमलावर बाहरी सामग्री—दस्तावेज़, वेब पेज, ईमेल, या डेटा स्रोतों—में दुर्भावनापूर्ण निर्देश छिपाते हैं जिन्हें एआई सिस्टम बाद में वैध आदेशों के रूप में संसाधित करता है।

**हमला परिदृश्य:**
- **दस्तावेज़-आधारित इंजेक्शन**: संसाधित दस्तावेज़ों में छिपे दुर्भावनापूर्ण निर्देश जो अनपेक्षित एआई क्रियाएँ ट्रिगर करते हैं
- **वेब सामग्री शोषण**: समझौता किए गए वेब पेज जिनमें एम्बेडेड प्रॉम्प्ट होते हैं जो स्क्रैपिंग के दौरान एआई व्यवहार को प्रभावित करते हैं
- **ईमेल-आधारित हमले**: ईमेल में दुर्भावनापूर्ण प्रॉम्प्ट जो एआई सहायकों को जानकारी लीक करने या अनधिकृत क्रियाएँ करने के लिए प्रेरित करते हैं
- **डेटा स्रोत संदूषण**: समझौता किए गए डेटाबेस या API जो दूषित सामग्री एआई सिस्टमों को प्रदान करते हैं

**वास्तविक प्रभाव**: ये हमले डेटा चोरी, गोपनीयता उल्लंघन, हानिकारक सामग्री निर्माण, और उपयोगकर्ता इंटरैक्शन में हेरफेर का कारण बन सकते हैं। विस्तृत विश्लेषण के लिए देखें [Prompt Injection in MCP (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/)।

![Prompt Injection Attack Diagram](../../../translated_images/hi/prompt-injection.ed9fbfde297ca877c15bc6daa808681cd3c3dc7bf27bbbda342ef1ba5fc4f52d.png)

#### **टूल पॉइज़निंग हमले**

**टूल पॉइज़निंग** MCP टूल्स को परिभाषित करने वाले मेटाडेटा को लक्षित करता है, जो LLMs द्वारा टूल विवरण और पैरामीटर की व्याख्या के तरीके का शोषण करता है ताकि निष्पादन निर्णय लिए जा सकें।

**हमला तंत्र:**
- **मेटाडेटा हेरफेर**: हमलावर टूल विवरण, पैरामीटर परिभाषाएँ, या उपयोग उदाहरणों में दुर्भावनापूर्ण निर्देश इंजेक्ट करते हैं
- **अदृश्य निर्देश**: टूल मेटाडेटा में छिपे प्रॉम्प्ट जो एआई मॉडल द्वारा संसाधित होते हैं लेकिन मानव उपयोगकर्ताओं को दिखाई नहीं देते
- **गतिशील टूल संशोधन ("रग पुल्स")**: उपयोगकर्ताओं द्वारा अनुमोदित टूल बाद में बिना उपयोगकर्ता की जानकारी के दुर्भावनापूर्ण क्रियाएँ करने के लिए संशोधित किए जाते हैं
- **पैरामीटर इंजेक्शन**: टूल पैरामीटर स्कीमाओं में दुर्भावनापूर्ण सामग्री जो मॉडल व्यवहार को प्रभावित करती है

**होस्टेड सर्वर जोखिम**: दूरस्थ MCP सर्वर उच्च जोखिम प्रस्तुत करते हैं क्योंकि टूल परिभाषाएँ प्रारंभिक उपयोगकर्ता अनुमोदन के बाद अपडेट की जा सकती हैं, जिससे पहले सुरक्षित टूल दुर्भावनापूर्ण बन सकते हैं। व्यापक विश्लेषण के लिए देखें [Tool Poisoning Attacks (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks)।

![Tool Injection Attack Diagram](../../../translated_images/hi/tool-injection.3b0b4a6b24de6befe7d3afdeae44138ef005881aebcfc84c6f61369ce31e3640.png)

#### **अतिरिक्त एआई हमला वेक्टर**

- **क्रॉस-डोमेन प्रॉम्प्ट इंजेक्शन (XPIA)**: जटिल हमले जो कई डोमेनों की सामग्री का उपयोग करके सुरक्षा नियंत्रणों को बायपास करते हैं
- **गतिशील क्षमता संशोधन**: टूल क्षमताओं में वास्तविक समय में परिवर्तन जो प्रारंभिक सुरक्षा आकलनों से बच जाते हैं
- **संदर्भ विंडो पॉइज़निंग**: बड़े संदर्भ विंडो को हेरफेर करने वाले हमले जो दुर्भावनापूर्ण निर्देश छिपाते हैं
- **मॉडल भ्रम हमले**: मॉडल सीमाओं का शोषण कर अप्रत्याशित या असुरक्षित व्यवहार उत्पन्न करना

### एआई सुरक्षा जोखिम प्रभाव

**उच्च प्रभाव परिणाम:**
- **डेटा चोरी**: संवेदनशील उद्यम या व्यक्तिगत डेटा की अनधिकृत पहुँच और चोरी
- **गोपनीयता उल्लंघन**: व्यक्तिगत पहचान योग्य जानकारी (PII) और गोपनीय व्यावसायिक डेटा का खुलासा  
- **सिस्टम हेरफेर**: महत्वपूर्ण सिस्टम और वर्कफ़्लो में अनपेक्षित संशोधन
- **प्रमाणपत्र चोरी**: प्रमाणीकरण टोकन और सेवा प्रमाणपत्रों का समझौता
- **पार्श्व आंदोलन**: व्यापक नेटवर्क हमलों के लिए समझौता किए गए एआई सिस्टमों का उपयोग

### Microsoft AI सुरक्षा समाधान

#### **AI Prompt Shields: इंजेक्शन हमलों के खिलाफ उन्नत सुरक्षा**

Microsoft **AI Prompt Shields** प्रत्यक्ष और अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन हमलों के खिलाफ कई सुरक्षा परतों के माध्यम से व्यापक रक्षा प्रदान करते हैं:

##### **मुख्य सुरक्षा तंत्र:**

1. **उन्नत पहचान और फ़िल्टरिंग**
   - मशीन लर्निंग एल्गोरिदम और NLP तकनीकें बाहरी सामग्री में दुर्भावनापूर्ण निर्देशों का पता लगाती हैं
   - दस्तावेज़ों, वेब पेजों, ईमेल, और डेटा स्रोतों का वास्तविक समय विश्लेषण
   - वैध और दुर्भावनापूर्ण प्रॉम्प्ट पैटर्न की संदर्भगत समझ

2. **स्पॉटलाइटिंग तकनीकें**  
   - विश्वसनीय सिस्टम निर्देशों और संभावित समझौता किए गए बाहरी इनपुट के बीच अंतर करती हैं
   - टेक्स्ट ट्रांसफॉर्मेशन विधियाँ जो मॉडल प्रासंगिकता बढ़ाती हैं और दुर्भावनापूर्ण सामग्री को अलग करती हैं
   - एआई सिस्टमों को उचित निर्देश पदानुक्रम बनाए रखने और इंजेक्ट किए गए आदेशों को अनदेखा करने में मदद करती हैं

3. **डेलिमीटर और डाटामार्किंग सिस्टम**
   - विश्वसनीय सिस्टम संदेशों और बाहरी इनपुट टेक्स्ट के बीच स्पष्ट सीमा परिभाषा
   - विशेष मार्कर जो विश्वसनीय और अविश्वसनीय डेटा स्रोतों के बीच सीमाओं को उजागर करते हैं
   - स्पष्ट पृथक्करण जो निर्देश भ्रम और अनधिकृत आदेश निष्पादन को रोकता है

4. **निरंतर खतरा खुफिया**
   - Microsoft लगातार उभरते हमले पैटर्न की निगरानी करता है और सुरक्षा उपाय अपडेट करता है
   - नए इंजेक्शन तकनीकों और हमले वेक्टरों के लिए सक्रिय खतरा खोज
   - विकसित होते खतरों के खिलाफ प्रभावशीलता बनाए रखने के लिए नियमित सुरक्षा मॉडल अपडेट

5. **Azure Content Safety एकीकरण**
   - व्यापक Azure AI Content Safety सूट का हिस्सा
   - जेलब्रेक प्रयासों, हानिकारक सामग्री, और सुरक्षा नीति उल्लंघनों के लिए अतिरिक्त पहचान
   - AI अनुप्रयोग घटकों में एकीकृत सुरक्षा नियंत्रण

## उन्नत MCP सुरक्षा खतरे

### सेशन हाईजैकिंग कमजोरियां

**सेशन हाईजैकिंग** राज्यपूर्ण MCP कार्यान्वयन में एक गंभीर हमला वेक्टर है जहाँ अनधिकृत पक्ष वैध सेशन पहचानकर्ताओं को प्राप्त कर उनका दुरुपयोग करके क्लाइंट का प्रतिरूपण करते हैं और अनधिकृत क्रियाएँ करते हैं।

#### **हमला परिदृश्य और जोखिम**

- **सेशन हाईजैक प्रॉम्प्ट इंजेक्शन**: चोरी किए गए सेशन आईडी वाले हमलावर सेशन स्थिति साझा करने वाले सर्वरों में दुर्भावनापूर्ण घटनाएँ इंजेक्ट करते हैं, जो संभावित रूप से हानिकारक क्रियाएँ ट्रिगर कर सकते हैं या संवेदनशील डेटा तक पहुँच सकते हैं
- **प्रत्यक्ष प्रतिरूपण**: चोरी किए गए सेशन आईडी सीधे MCP सर्वर कॉल सक्षम करते हैं जो प्रमाणीकरण को बायपास करते हैं, हमलावरों को वैध उपयोगकर्ता के रूप में मानते हैं
- **समझौता किए गए पुनः आरंभ योग्य स्ट्रीम**: हमलावर अनुरोधों को समय से पहले समाप्त कर सकते हैं, जिससे वैध क्लाइंट संभावित रूप से दुर्भावनापूर्ण सामग्री के साथ पुनः आरंभ करते हैं

#### **सेशन प्रबंधन के लिए सुरक्षा नियंत्रण**

**महत्वपूर्ण आवश्यकताएँ:**
- **प्राधिकरण सत्यापन**: MCP सर्वर जो प्राधिकरण लागू करते हैं, उन्हें सभी इनबाउंड अनुरोधों का सत्यापन करना **अनिवार्य** है और प्रमाणीकरण के लिए सत्रों पर निर्भर नहीं होना चाहिए  
- **सुरक्षित सत्र निर्माण**: क्रिप्टोग्राफिक रूप से सुरक्षित, गैर-निर्धारित सत्र आईडी का उपयोग करें जो सुरक्षित रैंडम नंबर जनरेटर से उत्पन्न हों  
- **उपयोगकर्ता-विशिष्ट बाइंडिंग**: सत्र आईडी को उपयोगकर्ता-विशिष्ट जानकारी से जोड़ें जैसे `<user_id>:<session_id>` प्रारूप में ताकि क्रॉस-यूजर सत्र दुरुपयोग से बचा जा सके  
- **सत्र जीवनचक्र प्रबंधन**: उचित समाप्ति, घुमाव, और अमान्यकरण लागू करें ताकि कमजोरियों की खिड़कियां सीमित रहें  
- **परिवहन सुरक्षा**: सभी संचार के लिए अनिवार्य HTTPS ताकि सत्र आईडी की चोरी से बचा जा सके  

### भ्रमित डिप्टी समस्या

**भ्रमित डिप्टी समस्या** तब होती है जब MCP सर्वर क्लाइंट और तृतीय-पक्ष सेवाओं के बीच प्रमाणीकरण प्रॉक्सी के रूप में कार्य करते हैं, जिससे स्थैतिक क्लाइंट आईडी के दुरुपयोग के माध्यम से प्राधिकरण बाईपास के अवसर बनते हैं।

#### **हमले की प्रक्रिया और जोखिम**

- **कुकी-आधारित सहमति बाईपास**: पूर्व उपयोगकर्ता प्रमाणीकरण सहमति कुकीज़ बनाता है जिन्हें हमलावर रिडायरेक्ट URI के साथ दुर्भावनापूर्ण प्राधिकरण अनुरोधों के माध्यम से दुरुपयोग करते हैं  
- **प्राधिकरण कोड चोरी**: मौजूदा सहमति कुकीज़ प्राधिकरण सर्वरों को सहमति स्क्रीन छोड़ने पर मजबूर कर सकती हैं, जिससे कोड हमलावर नियंत्रित एंडपॉइंट्स पर रीडायरेक्ट हो जाते हैं  
- **अनधिकृत API पहुंच**: चोरी हुए प्राधिकरण कोड टोकन विनिमय और उपयोगकर्ता की नकल करने की अनुमति देते हैं बिना स्पष्ट अनुमोदन के  

#### **रोकथाम रणनीतियाँ**

**अनिवार्य नियंत्रण:**
- **स्पष्ट सहमति आवश्यकताएं**: स्थैतिक क्लाइंट आईडी वाले MCP प्रॉक्सी सर्वर को प्रत्येक गतिशील रूप से पंजीकृत क्लाइंट के लिए उपयोगकर्ता की सहमति प्राप्त करनी **चाहिए**  
- **OAuth 2.1 सुरक्षा कार्यान्वयन**: सभी प्राधिकरण अनुरोधों के लिए PKCE (प्रूफ की फॉर कोड एक्सचेंज) सहित वर्तमान OAuth सुरक्षा सर्वोत्तम प्रथाओं का पालन करें  
- **कठोर क्लाइंट सत्यापन**: दुरुपयोग रोकने के लिए रिडायरेक्ट URI और क्लाइंट पहचानकर्ताओं का सख्त सत्यापन लागू करें  

### टोकन पासथ्रू कमजोरियां  

**टोकन पासथ्रू** एक स्पष्ट एंटी-पैटर्न है जहां MCP सर्वर उचित सत्यापन के बिना क्लाइंट टोकन स्वीकार करते हैं और उन्हें डाउनस्ट्रीम API को अग्रेषित करते हैं, जो MCP प्राधिकरण विनिर्देशों का उल्लंघन है।

#### **सुरक्षा प्रभाव**

- **नियंत्रण बाईपास**: सीधे क्लाइंट-से-API टोकन उपयोग महत्वपूर्ण दर सीमांकन, सत्यापन, और निगरानी नियंत्रणों को बाईपास करता है  
- **ऑडिट ट्रेल भ्रष्टाचार**: अपस्ट्रीम जारी किए गए टोकन क्लाइंट पहचान को असंभव बनाते हैं, जिससे घटना जांच क्षमताएं टूट जाती हैं  
- **प्रॉक्सी-आधारित डेटा चोरी**: अप्रमाणित टोकन दुर्भावनापूर्ण अभिनेताओं को सर्वरों का उपयोग अनधिकृत डेटा पहुंच के लिए प्रॉक्सी के रूप में करने की अनुमति देते हैं  
- **ट्रस्ट सीमा उल्लंघन**: जब टोकन के स्रोत सत्यापित नहीं हो पाते, तो डाउनस्ट्रीम सेवाओं के ट्रस्ट मान्यताएं उल्लंघित हो सकती हैं  
- **मल्टी-सर्विस हमला विस्तार**: कई सेवाओं में स्वीकार किए गए समझौता किए गए टोकन पार्श्व आंदोलन सक्षम करते हैं  

#### **आवश्यक सुरक्षा नियंत्रण**

**अपरिहार्य आवश्यकताएं:**
- **टोकन सत्यापन**: MCP सर्वर को ऐसे टोकन स्वीकार नहीं करने चाहिए जो स्पष्ट रूप से MCP सर्वर के लिए जारी नहीं किए गए हों  
- **दर्शक सत्यापन**: हमेशा टोकन दर्शक दावों का सत्यापन करें कि वे MCP सर्वर की पहचान से मेल खाते हों  
- **उचित टोकन जीवनचक्र**: सुरक्षित घुमाव के साथ अल्पकालिक एक्सेस टोकन लागू करें  

## AI सिस्टम के लिए सप्लाई चेन सुरक्षा

सप्लाई चेन सुरक्षा पारंपरिक सॉफ़्टवेयर निर्भरताओं से आगे बढ़कर पूरे AI पारिस्थितिकी तंत्र को शामिल करती है। आधुनिक MCP कार्यान्वयन को सभी AI-संबंधित घटकों का कड़ाई से सत्यापन और निगरानी करनी चाहिए, क्योंकि प्रत्येक संभावित कमजोरियां प्रस्तुत करता है जो सिस्टम की अखंडता को खतरे में डाल सकती हैं।

### विस्तारित AI सप्लाई चेन घटक

**पारंपरिक सॉफ़्टवेयर निर्भरताएं:**
- ओपन-सोर्स लाइब्रेरी और फ्रेमवर्क  
- कंटेनर इमेज और बेस सिस्टम  
- विकास उपकरण और बिल्ड पाइपलाइंस  
- अवसंरचना घटक और सेवाएं  

**AI-विशिष्ट सप्लाई चेन तत्व:**
- **फाउंडेशन मॉडल**: विभिन्न प्रदाताओं से पूर्व-प्रशिक्षित मॉडल जिनके लिए उत्पत्ति सत्यापन आवश्यक है  
- **एम्बेडिंग सेवाएं**: बाहरी वेक्टराइजेशन और सेमांटिक खोज सेवाएं  
- **संदर्भ प्रदाता**: डेटा स्रोत, ज्ञान आधार, और दस्तावेज़ भंडार  
- **तृतीय-पक्ष API**: बाहरी AI सेवाएं, ML पाइपलाइंस, और डेटा प्रोसेसिंग एंडपॉइंट्स  
- **मॉडल आर्टिफैक्ट्स**: वज़न, विन्यास, और फाइन-ट्यून किए गए मॉडल वेरिएंट  
- **प्रशिक्षण डेटा स्रोत**: मॉडल प्रशिक्षण और फाइन-ट्यूनिंग के लिए उपयोग किए गए डेटासेट  

### व्यापक सप्लाई चेन सुरक्षा रणनीति

#### **घटक सत्यापन और विश्वास**
- **उत्पत्ति सत्यापन**: सभी AI घटकों के स्रोत, लाइसेंसिंग, और अखंडता का एकीकरण से पहले सत्यापन करें  
- **सुरक्षा मूल्यांकन**: मॉडल, डेटा स्रोत, और AI सेवाओं के लिए कमजोरियों के स्कैन और सुरक्षा समीक्षा करें  
- **प्रतिष्ठा विश्लेषण**: AI सेवा प्रदाताओं के सुरक्षा ट्रैक रिकॉर्ड और प्रथाओं का मूल्यांकन करें  
- **अनुपालन सत्यापन**: सुनिश्चित करें कि सभी घटक संगठनात्मक सुरक्षा और नियामक आवश्यकताओं को पूरा करते हैं  

#### **सुरक्षित तैनाती पाइपलाइंस**  
- **स्वचालित CI/CD सुरक्षा**: स्वचालित तैनाती पाइपलाइनों में सुरक्षा स्कैनिंग को एकीकृत करें  
- **आर्टिफैक्ट अखंडता**: सभी तैनात आर्टिफैक्ट्स (कोड, मॉडल, विन्यास) के लिए क्रिप्टोग्राफिक सत्यापन लागू करें  
- **चरणबद्ध तैनाती**: प्रत्येक चरण में सुरक्षा सत्यापन के साथ प्रगतिशील तैनाती रणनीतियों का उपयोग करें  
- **विश्वसनीय आर्टिफैक्ट रिपॉजिटरी**: केवल सत्यापित, सुरक्षित आर्टिफैक्ट रजिस्ट्री और रिपॉजिटरी से तैनात करें  

#### **निरंतर निगरानी और प्रतिक्रिया**
- **निर्भरता स्कैनिंग**: सभी सॉफ़्टवेयर और AI घटक निर्भरताओं के लिए चल रही कमजोरियों की निगरानी  
- **मॉडल निगरानी**: मॉडल व्यवहार, प्रदर्शन विचलन, और सुरक्षा असामान्यताओं का निरंतर मूल्यांकन  
- **सेवा स्वास्थ्य ट्रैकिंग**: बाहरी AI सेवाओं की उपलब्धता, सुरक्षा घटनाओं, और नीति परिवर्तनों की निगरानी  
- **खतरा खुफिया एकीकरण**: AI और ML सुरक्षा जोखिमों के लिए विशिष्ट खतरा फ़ीड्स को शामिल करें  

#### **पहुँच नियंत्रण और न्यूनतम विशेषाधिकार**
- **घटक-स्तरीय अनुमतियां**: व्यवसाय आवश्यकताओं के आधार पर मॉडल, डेटा, और सेवाओं तक पहुंच सीमित करें  
- **सेवा खाता प्रबंधन**: न्यूनतम आवश्यक अनुमतियों के साथ समर्पित सेवा खाते लागू करें  
- **नेटवर्क विभाजन**: AI घटकों को अलग करें और सेवाओं के बीच नेटवर्क पहुंच सीमित करें  
- **API गेटवे नियंत्रण**: बाहरी AI सेवाओं तक पहुंच को नियंत्रित और निगरानी के लिए केंद्रीकृत API गेटवे का उपयोग करें  

#### **घटना प्रतिक्रिया और पुनर्प्राप्ति**
- **त्वरित प्रतिक्रिया प्रक्रियाएं**: समझौता किए गए AI घटकों के लिए पैचिंग या प्रतिस्थापन की स्थापित प्रक्रियाएं  
- **प्रमाणपत्र घुमाव**: रहस्यों, API कुंजी, और सेवा प्रमाणपत्रों के घुमाव के लिए स्वचालित प्रणालियां  
- **रोलबैक क्षमताएं**: AI घटकों के पूर्व ज्ञात-सही संस्करणों पर जल्दी वापस जाने की क्षमता  
- **सप्लाई चेन उल्लंघन पुनर्प्राप्ति**: अपस्ट्रीम AI सेवा समझौते के लिए विशिष्ट प्रतिक्रिया प्रक्रियाएं  

### Microsoft सुरक्षा उपकरण और एकीकरण

**GitHub Advanced Security** व्यापक सप्लाई चेन सुरक्षा प्रदान करता है जिसमें शामिल हैं:  
- **सीक्रेट स्कैनिंग**: रिपॉजिटरी में क्रेडेंशियल, API कुंजी, और टोकन का स्वचालित पता लगाना  
- **निर्भरता स्कैनिंग**: ओपन-सोर्स निर्भरताओं और लाइब्रेरीज़ के लिए कमजोरियों का आकलन  
- **CodeQL विश्लेषण**: सुरक्षा कमजोरियों और कोडिंग मुद्दों के लिए स्थैतिक कोड विश्लेषण  
- **सप्लाई चेन इनसाइट्स**: निर्भरता स्वास्थ्य और सुरक्षा स्थिति की दृश्यता  

**Azure DevOps और Azure Repos एकीकरण:**  
- Microsoft विकास प्लेटफार्मों में निर्बाध सुरक्षा स्कैनिंग एकीकरण  
- AI वर्कलोड के लिए Azure पाइपलाइनों में स्वचालित सुरक्षा जांच  
- सुरक्षित AI घटक तैनाती के लिए नीति प्रवर्तन  

**Microsoft आंतरिक प्रथाएं:**  
Microsoft सभी उत्पादों में व्यापक सप्लाई चेन सुरक्षा प्रथाओं को लागू करता है। [The Journey to Secure the Software Supply Chain at Microsoft](https://devblogs.microsoft.com/engineering-at-microsoft/the-journey-to-secure-the-software-supply-chain-at-microsoft/) में सिद्ध दृष्टिकोणों के बारे में जानें।  

## फाउंडेशन सुरक्षा सर्वोत्तम प्रथाएं

MCP कार्यान्वयन आपकी संगठन की मौजूदा सुरक्षा स्थिति को अपनाते और उस पर निर्माण करते हैं। मौलिक सुरक्षा प्रथाओं को मजबूत करना AI सिस्टम और MCP तैनाती की समग्र सुरक्षा को काफी बढ़ाता है।

### कोर सुरक्षा मूल बातें

#### **सुरक्षित विकास प्रथाएं**
- **OWASP अनुपालन**: [OWASP टॉप 10](https://owasp.org/www-project-top-ten/) वेब एप्लिकेशन कमजोरियों से सुरक्षा  
- **AI-विशिष्ट सुरक्षा**: [LLM के लिए OWASP टॉप 10](https://genai.owasp.org/download/43299/?tmstv=1731900559) के लिए नियंत्रण लागू करें  
- **सुरक्षित रहस्य प्रबंधन**: टोकन, API कुंजी, और संवेदनशील विन्यास डेटा के लिए समर्पित वॉल्ट का उपयोग करें  
- **एंड-टू-एंड एन्क्रिप्शन**: सभी एप्लिकेशन घटकों और डेटा प्रवाहों में सुरक्षित संचार लागू करें  
- **इनपुट सत्यापन**: सभी उपयोगकर्ता इनपुट, API पैरामीटर, और डेटा स्रोतों का कठोर सत्यापन  

#### **इन्फ्रास्ट्रक्चर हार्डनिंग**
- **मल्टी-फैक्टर प्रमाणीकरण**: सभी प्रशासनिक और सेवा खातों के लिए अनिवार्य MFA  
- **पैच प्रबंधन**: ऑपरेटिंग सिस्टम, फ्रेमवर्क, और निर्भरताओं के लिए स्वचालित, समय पर पैचिंग  
- **पहचान प्रदाता एकीकरण**: एंटरप्राइज़ पहचान प्रदाताओं (Microsoft Entra ID, Active Directory) के माध्यम से केंद्रीकृत पहचान प्रबंधन  
- **नेटवर्क विभाजन**: MCP घटकों का तार्किक पृथक्करण ताकि पार्श्व आंदोलन की संभावना सीमित हो  
- **न्यूनतम विशेषाधिकार सिद्धांत**: सभी सिस्टम घटकों और खातों के लिए न्यूनतम आवश्यक अनुमतियां  

#### **सुरक्षा निगरानी और पता लगाना**
- **व्यापक लॉगिंग**: AI एप्लिकेशन गतिविधियों सहित MCP क्लाइंट-सर्वर इंटरैक्शन का विस्तृत लॉगिंग  
- **SIEM एकीकरण**: विसंगति पता लगाने के लिए केंद्रीकृत सुरक्षा सूचना और घटना प्रबंधन  
- **व्यवहार विश्लेषण**: सिस्टम और उपयोगकर्ता व्यवहार में असामान्य पैटर्न का पता लगाने के लिए AI-संचालित निगरानी  
- **खतरा खुफिया**: बाहरी खतरा फ़ीड्स और समझौता संकेतकों (IOCs) का एकीकरण  
- **घटना प्रतिक्रिया**: सुरक्षा घटना पता लगाने, प्रतिक्रिया, और पुनर्प्राप्ति के लिए सुव्यवस्थित प्रक्रियाएं  

#### **शून्य ट्रस्ट आर्किटेक्चर**
- **कभी भरोसा न करें, हमेशा सत्यापित करें**: उपयोगकर्ताओं, उपकरणों, और नेटवर्क कनेक्शनों का निरंतर सत्यापन  
- **माइक्रो-सेगमेंटेशन**: व्यक्तिगत वर्कलोड और सेवाओं को अलग करने वाले सूक्ष्म नेटवर्क नियंत्रण  
- **पहचान-केंद्रित सुरक्षा**: नेटवर्क स्थान के बजाय सत्यापित पहचान पर आधारित सुरक्षा नीतियां  
- **निरंतर जोखिम मूल्यांकन**: वर्तमान संदर्भ और व्यवहार के आधार पर गतिशील सुरक्षा स्थिति मूल्यांकन  
- **सशर्त पहुंच**: जोखिम कारकों, स्थान, और उपकरण भरोसे के आधार पर अनुकूलित पहुंच नियंत्रण  

### एंटरप्राइज़ एकीकरण पैटर्न

#### **Microsoft सुरक्षा पारिस्थितिकी तंत्र एकीकरण**
- **Microsoft Defender for Cloud**: व्यापक क्लाउड सुरक्षा स्थिति प्रबंधन  
- **Azure Sentinel**: AI वर्कलोड सुरक्षा के लिए क्लाउड-नेटिव SIEM और SOAR क्षमताएं  
- **Microsoft Entra ID**: सशर्त पहुंच नीतियों के साथ एंटरप्राइज़ पहचान और पहुंच प्रबंधन  
- **Azure Key Vault**: हार्डवेयर सुरक्षा मॉड्यूल (HSM) समर्थन के साथ केंद्रीकृत रहस्य प्रबंधन  
- **Microsoft Purview**: AI डेटा स्रोतों और वर्कफ़्लोज़ के लिए डेटा गवर्नेंस और अनुपालन  

#### **अनुपालन और शासन**
- **नियामक संरेखण**: सुनिश्चित करें कि MCP कार्यान्वयन उद्योग-विशिष्ट अनुपालन आवश्यकताओं (GDPR, HIPAA, SOC 2) को पूरा करते हैं  
- **डेटा वर्गीकरण**: AI सिस्टम द्वारा संसाधित संवेदनशील डेटा का उचित वर्गीकरण और प्रबंधन  
- **ऑडिट ट्रेल्स**: नियामक अनुपालन और फोरेंसिक जांच के लिए व्यापक लॉगिंग  
- **गोपनीयता नियंत्रण**: AI सिस्टम आर्किटेक्चर में प्राइवेसी-बाय-डिज़ाइन सिद्धांतों का कार्यान्वयन  
- **परिवर्तन प्रबंधन**: AI सिस्टम संशोधनों की सुरक्षा समीक्षा के लिए औपचारिक प्रक्रियाएं  

ये मौलिक प्रथाएं एक मजबूत सुरक्षा आधार बनाती हैं जो MCP-विशिष्ट सुरक्षा नियंत्रणों की प्रभावशीलता को बढ़ाती हैं और AI-संचालित अनुप्रयोगों के लिए व्यापक सुरक्षा प्रदान करती हैं।  

## प्रमुख सुरक्षा निष्कर्ष

- **परतदार सुरक्षा दृष्टिकोण**: मौलिक सुरक्षा प्रथाओं (सुरक्षित कोडिंग, न्यूनतम विशेषाधिकार, सप्लाई चेन सत्यापन, निरंतर निगरानी) को AI-विशिष्ट नियंत्रणों के साथ संयोजित करें ताकि व्यापक सुरक्षा सुनिश्चित हो  
- **AI-विशिष्ट खतरा परिदृश्य**: MCP सिस्टमों को अद्वितीय जोखिमों का सामना करना पड़ता है जैसे प्रॉम्प्ट इंजेक्शन, टूल पॉइज़निंग, सत्र हाईजैकिंग, भ्रमित डिप्टी समस्याएं, टोकन पासथ्रू कमजोरियां, और अत्यधिक अनुमतियां जिनके लिए विशेष रोकथाम आवश्यक है  
- **प्रमाणीकरण और प्राधिकरण उत्कृष्टता**: बाहरी पहचान प्रदाताओं (Microsoft Entra ID) का उपयोग करके मजबूत प्रमाणीकरण लागू करें, उचित टोकन सत्यापन लागू करें, और कभी भी ऐसे टोकन स्वीकार न करें जो स्पष्ट रूप से आपके MCP सर्वर के लिए जारी न किए गए हों  
- **AI हमले रोकथाम**: Microsoft Prompt Shields और Azure Content Safety तैनात करें ताकि अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन और टूल पॉइज़निंग हमलों से बचाव हो, साथ ही टूल मेटाडेटा का सत्यापन और गतिशील परिवर्तनों की निगरानी करें  
- **सत्र और परिवहन सुरक्षा**: क्रिप्टोग्राफिक रूप से सुरक्षित, गैर-निर्धारित सत्र आईडी का उपयोग करें जो उपयोगकर्ता पहचान से बंधे हों, उचित सत्र जीवनचक्र प्रबंधन लागू करें, और प्रमाणीकरण के लिए सत्रों का उपयोग कभी न करें  
- **OAuth सुरक्षा सर्वोत्तम प्रथाएं**: गतिशील रूप से पंजीकृत क्लाइंट के लिए स्पष्ट उपयोगकर्ता सहमति, PKCE के साथ उचित OAuth 2.1 कार्यान्वयन, और सख्त रिडायरेक्ट URI सत्यापन के माध्यम से भ्रमित डिप्टी हमलों को रोकें  
- **टोकन सुरक्षा सिद्धांत**: टोकन पासथ्रू एंटी-पैटर्न से बचें, टोकन दर्शक दावों का सत्यापन करें, सुरक्षित घुमाव के साथ अल्पकालिक टोकन लागू करें, और स्पष्ट ट्रस्ट सीमाएं बनाए रखें  
- **व्यापक सप्लाई चेन सुरक्षा**: सभी AI पारिस्थितिकी तंत्र घटकों (मॉडल, एम्बेडिंग, संदर्भ प्रदाता, बाहरी API) को पारंपरिक सॉफ़्टवेयर निर्भरताओं के समान सुरक्षा कठोरता के साथ देखें  
- **निरंतर विकास**: तेजी से विकसित हो रहे MCP विनिर्देशों के साथ अद्यतित रहें, सुरक्षा समुदाय मानकों में योगदान दें, और प्रोटोकॉल के परिपक्व होने के साथ अनुकूल सुरक्षा स्थिति बनाए रखें  
- **Microsoft सुरक्षा एकीकरण**: Microsoft के व्यापक सुरक्षा पारिस्थितिकी तंत्र (Prompt Shields, Azure Content Safety, GitHub Advanced Security, Entra ID) का उपयोग करके MCP तैनाती सुरक्षा बढ़ाएं
