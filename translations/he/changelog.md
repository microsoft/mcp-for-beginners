- **עדכון גרסת פרוטוקול**: עודכן להפנות למפרט MCP העדכני ביותר 2025-11-25 (שוחרר ב-25 בנובמבר 2025)

  - עודכנו הפניות לתאריכי המסמך מ-18 באוגוסט 2025 ל-18 בדצמבר 2025
  - אושרו שכל כתובות ה-URL של המפרט מפנות לתיעוד הנוכחי
- **אימות תוכן**: אימות מקיף של שיטות אבטחה מומלצות מול התקנים העדכניים ביותר
  - **פתרונות אבטחה של מיקרוסופט**: אושרו מונחים וקישורים עדכניים ל-Prompt Shields (לשעבר "זיהוי סיכון לפריצת מערכת"), Azure Content Safety, Microsoft Entra ID, ו-Azure Key Vault
  - **אבטחת OAuth 2.1**: אושרה התאמה לשיטות האבטחה המומלצות האחרונות של OAuth

  - **שירותי Azure**: אושרו כל הקישורים לתיעוד מיקרוסופט Azure ושיטות עבודה מומלצות
- **התאמת תקנים**: כל תקני האבטחה המופנים אושרו כנכונים
  - מסגרת ניהול סיכוני AI של NIST
  - ISO 27001:2022
  - שיטות אבטחה מומלצות ל-OAuth 2.1
  - מסגרות אבטחה וציות של Azure
- **משאבי יישום**: אושרו כל הקישורים למדריכי יישום ומשאבים
  - דפוסי אימות ניהול API של Azure
  - מדריכי אינטגרציה של Microsoft Entra ID
  - ניהול סודות ב-Azure Key Vault
  - צינורות DevSecOps ופתרונות ניטור

## 6 באוקטובר 2025

### הרחבת פרק התחלה – שימוש מתקדם בשרת ואימות פשוט

#### שימוש מתקדם בשרת (03-GettingStarted/10-advanced)
- **פרק חדש נוסף**: הוצג מדריך מקיף לשימוש מתקדם בשרת MCP, הכולל ארכיטקטורות שרת רגילות ונמוכות-רמה.
  - **שרת רגיל לעומת שרת נמוך-רמה**: השוואה מפורטת ודוגמאות קוד בפייתון ו-TypeScript לשתי הגישות.
  - **עיצוב מבוסס מטפל**: הסבר על ניהול כלים/משאבים/פרומפטים מבוסס מטפל ליישומי שרת גמישים וניתנים להרחבה.
  - **דפוסים מעשיים**: תרחישים מהעולם האמיתי בהם דפוסי שרת נמוך-רמה מועילים לתכונות וארכיטקטורה מתקדמות.

#### אימות פשוט (03-GettingStarted/11-simple-auth)
- **פרק חדש נוסף**: מדריך שלב-אחר-שלב ליישום אימות פשוט בשרתי MCP.
  - **מושגי אימות**: הסבר ברור על אימות לעומת הרשאה, וטיפול באישורים.
  - **יישום אימות בסיסי**: דפוסי אימות מבוססי middleware בפייתון (Starlette) ו-TypeScript (Express), עם דוגמאות קוד.
  - **התקדמות לאבטחה מתקדמת**: הנחיות להתחלה עם אימות פשוט והתקדמות ל-OAuth 2.1 ו-RBAC, עם הפניות למודולי אבטחה מתקדמים.

תוספות אלו מספקות הדרכה מעשית וידנית לבניית יישומי שרת MCP חזקים, מאובטחים וגמישים יותר, המחברים בין מושגים בסיסיים לדפוסי ייצור מתקדמים.

  - שרתים משופרים עם תרחישי פריסה מקומית מול מרוחקת

- **ארכיטקטורת פרוטוקול**: הצגת מודל ארכיטקטורה דו-שכבתי  
  - שכבת נתונים: בסיס JSON-RPC 2.0 עם ניהול מחזור חיים ופרימיטיבים  
  - שכבת תעבורה: STDIO (מקומי) ו-HTTP סטרימבילי עם SSE (מרוחק)
