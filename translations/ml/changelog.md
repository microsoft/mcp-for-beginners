# മാറ്റങ്ങൾ: MCP for Beginners പാഠ്യപദ്ധതി

ഈ രേഖ മോഡൽ കോൺടെക്സ്റ്റ് പ്രോട്ടോക്കോൾ (MCP) for Beginners പാഠ്യപദ്ധതിയിൽ നടത്തിയ എല്ലാ പ്രധാന മാറ്റങ്ങളും രേഖപ്പെടുത്തുന്നതിന് ഉപയോഗിക്കുന്നു. മാറ്റങ്ങൾ ക്രമവിരുദ്ധമായ ക്രമത്തിൽ രേഖപ്പെടുത്തിയിരിക്കുന്നു (പുതിയ മാറ്റങ്ങൾ ആദ്യം).

## ഡിസംബർ 18, 2025

### സുരക്ഷാ ഡോക്യുമെന്റേഷൻ അപ്ഡേറ്റ് - MCP സ്പെസിഫിക്കേഷൻ 2025-11-25

#### MCP Security Best Practices (02-Security/mcp-best-practices.md) - സ്പെസിഫിക്കേഷൻ പതിപ്പ് അപ്ഡേറ്റ്
- **പ്രോട്ടോക്കോൾ പതിപ്പ് അപ്ഡേറ്റ്**: ഏറ്റവും പുതിയ MCP സ്പെസിഫിക്കേഷൻ 2025-11-25 (നവംബർ 25, 2025 റിലീസ്) ഉദ്ധരിച്ച് അപ്ഡേറ്റ് ചെയ്തു
  - എല്ലാ സ്പെസിഫിക്കേഷൻ പതിപ്പ് റഫറൻസുകളും 2025-06-18 മുതൽ 2025-11-25 ആയി അപ്ഡേറ്റ് ചെയ്തു
  - ഡോക്യുമെന്റ് തീയതി റഫറൻസുകൾ ഓഗസ്റ്റ് 18, 2025 മുതൽ ഡിസംബർ 18, 2025 ആയി അപ്ഡേറ്റ് ചെയ്തു
  - എല്ലാ സ്പെസിഫിക്കേഷൻ URLs നിലവിലെ ഡോക്യുമെന്റേഷനിലേക്ക് പോയെന്ന് സ്ഥിരീകരിച്ചു
- **ഉള്ളടക്ക പരിശോധന**: ഏറ്റവും പുതിയ സ്റ്റാൻഡേർഡുകൾക്ക് അനുസൃതമായി സുരക്ഷാ മികച്ച പ്രാക്ടീസുകളുടെ സമഗ്ര പരിശോധന
  - **Microsoft Security Solutions**: Prompt Shields (മുൻപ് "Jailbreak risk detection"), Azure Content Safety, Microsoft Entra ID, Azure Key Vault എന്നിവയുടെ നിലവിലെ പദങ്ങൾക്കും ലിങ്കുകൾക്കും സ്ഥിരീകരണം
  - **OAuth 2.1 Security**: ഏറ്റവും പുതിയ OAuth സുരക്ഷാ മികച്ച പ്രാക്ടീസുകളുമായി പൊരുത്തം ഉറപ്പാക്കി
  - **OWASP Standards**: LLMs-നുള്ള OWASP Top 10 റഫറൻസുകൾ ഇപ്പോഴും നിലവിലുള്ളവയാണെന്ന് സ്ഥിരീകരിച്ചു
  - **Azure Services**: എല്ലാ Microsoft Azure ഡോക്യുമെന്റേഷൻ ലിങ്കുകളും മികച്ച പ്രാക്ടീസുകളും പരിശോധിച്ചു
- **സ്റ്റാൻഡേർഡുകൾ പൊരുത്തം**: എല്ലാ റഫറൻസുചെയ്ത സുരക്ഷാ സ്റ്റാൻഡേർഡുകളും നിലവിലുള്ളവയാണെന്ന് സ്ഥിരീകരിച്ചു
  - NIST AI Risk Management Framework
  - ISO 27001:2022
  - OAuth 2.1 Security Best Practices
  - Azure security and compliance frameworks
- **ഇംപ്ലിമെന്റേഷൻ റിസോഴ്‌സുകൾ**: എല്ലാ ഇംപ്ലിമെന്റേഷൻ ഗൈഡ് ലിങ്കുകളും റിസോഴ്‌സുകളും പരിശോധിച്ചു
  - Azure API Management authentication patterns
  - Microsoft Entra ID integration guides
  - Azure Key Vault secrets management
  - DevSecOps പൈപ്പ്ലൈനുകളും മോണിറ്ററിംഗ് സൊല്യൂഷനുകളും

### ഡോക്യുമെന്റേഷൻ ഗുണനിലവാര ഉറപ്പ്
- **സ്പെസിഫിക്കേഷൻ പാലനം**: എല്ലാ നിർബന്ധമായ MCP സുരക്ഷാ ആവശ്യകതകളും (MUST/MUST NOT) ഏറ്റവും പുതിയ സ്പെസിഫിക്കേഷനുമായി പൊരുത്തം ഉറപ്പാക്കി
- **റിസോഴ്‌സ് പുതുമ**: Microsoft ഡോക്യുമെന്റേഷൻ, സുരക്ഷാ സ്റ്റാൻഡേർഡുകൾ, ഇംപ്ലിമെന്റേഷൻ ഗൈഡുകൾ എന്നിവയിലേക്കുള്ള എല്ലാ ബാഹ്യ ലിങ്കുകളും പരിശോധിച്ചു
- **മികച്ച പ്രാക്ടീസുകൾ ഉൾക്കൊള്ളൽ**: ഓത്തന്റിക്കേഷൻ, ഓത്തറൈസേഷൻ, AI-നിഷ്ഠമായ ഭീഷണികൾ, സപ്ലൈ ചെയിൻ സുരക്ഷ, എന്റർപ്രൈസ് പാറ്റേണുകൾ എന്നിവയുടെ സമഗ്ര ഉൾക്കൊള്ളൽ സ്ഥിരീകരിച്ചു

## ഒക്ടോബർ 6, 2025

### Getting Started വിഭാഗം വിപുലീകരണം – അഡ്വാൻസ്ഡ് സർവർ ഉപയോഗവും ലളിതമായ ഓത്തന്റിക്കേഷനും

#### Advanced Server Usage (03-GettingStarted/10-advanced)
- **പുതിയ അധ്യായം ചേർത്തു**: അഡ്വാൻസ്ഡ് MCP സർവർ ഉപയോഗത്തിന് സമഗ്ര ഗൈഡ് അവതരിപ്പിച്ചു, സാധാരണയും ലോ-ലെവൽ സർവർ ആർക്കിടെക്ചറുകളും ഉൾക്കൊള്ളുന്നു.
  - **സാധാരണ vs ലോ-ലെവൽ സർവർ**: രണ്ട് സമീപനങ്ങൾക്കും Python, TypeScript കോഡ് ഉദാഹരണങ്ങളോടെ വിശദീകരണം.
  - **ഹാൻഡ്ലർ അടിസ്ഥാന ഡിസൈൻ**: സ്കേലബിൾ, ഫ്ലെക്സിബിൾ സർവർ ഇംപ്ലിമെന്റേഷനുകൾക്കായി ഹാൻഡ്ലർ അടിസ്ഥാന ടൂൾ/റിസോഴ്‌സ്/പ്രോംപ്റ്റ് മാനേജ്മെന്റ് വിശദീകരണം.
  - **പ്രായോഗിക പാറ്റേണുകൾ**: അഡ്വാൻസ്ഡ് ഫീച്ചറുകൾക്കും ആർക്കിടെക്ചറിനും ലോ-ലെവൽ സർവർ പാറ്റേണുകൾ എങ്ങനെ പ്രയോജനപ്പെടുന്നു എന്നുള്ള യഥാർത്ഥ ലോക സാഹചര്യങ്ങൾ.

#### Simple Authentication (03-GettingStarted/11-simple-auth)
- **പുതിയ അധ്യായം ചേർത്തു**: MCP സർവറുകളിൽ ലളിതമായ ഓത്തന്റിക്കേഷൻ നടപ്പിലാക്കാനുള്ള ഘട്ടം ഘട്ടമായുള്ള ഗൈഡ്.
  - **ഓത്ത് ആശയങ്ങൾ**: ഓത്തന്റിക്കേഷൻ vs ഓത്തറൈസേഷൻ, ക്രെഡൻഷ്യൽ കൈകാര്യം എന്നിവയുടെ വ്യക്തമായ വിശദീകരണം.
  - **ബേസിക് ഓത്ത് ഇംപ്ലിമെന്റേഷൻ**: Python (Starlette)യും TypeScript (Express)ഉം ഉപയോഗിച്ച് മിഡിൽവെയർ അടിസ്ഥാന ഓത്തന്റിക്കേഷൻ പാറ്റേണുകൾ, കോഡ് സാമ്പിളുകളോടെ.
  - **അഡ്വാൻസ്ഡ് സുരക്ഷയിലേക്ക് പുരോഗതി**: ലളിതമായ ഓത്ത് ആരംഭിച്ച് OAuth 2.1, RBAC എന്നിവയിലേക്ക് മുന്നോട്ട് പോകാനുള്ള മാർഗ്ഗനിർദ്ദേശം, അഡ്വാൻസ്ഡ് സുരക്ഷാ മോഡ്യൂളുകൾക്ക് റഫറൻസുകൾ ഉൾപ്പെടെ.

ഈ ചേർക്കലുകൾ കൂടുതൽ ശക്തമായ, സുരക്ഷിതമായ, ഫ്ലെക്സിബിൾ MCP സർവർ ഇംപ്ലിമെന്റേഷനുകൾ നിർമ്മിക്കാൻ പ്രായോഗിക, കൈകൊണ്ടുള്ള മാർഗ്ഗനിർദ്ദേശം നൽകുന്നു, അടിസ്ഥാന ആശയങ്ങളെ അഡ്വാൻസ്ഡ് പ്രൊഡക്ഷൻ പാറ്റേണുകളുമായി ബന്ധിപ്പിക്കുന്നു.

  - **പൂർത്തീകരണ ട്രാക്കിംഗ്**: "After completing this module..." "After completing this lab..." ആയി അപ്ഡേറ്റ് ചെയ്തു

    - ഇൻപുട്ട് കാര്യക്ഷമത, പ്രകടനം, ഗുണമേന്മ, ഉപയോക്തൃ അനുഭവ പരിഗണനകൾ

    - പരാജയ വിശകലനവും മെച്ചപ്പെടുത്തൽ രീതി

#### കോർ ആശയങ്ങൾ (01-CoreConcepts/) - പൂർണ്ണ ആധുനികീകരണം
- **പ്രോട്ടോക്കോൾ പതിപ്പ് അപ്ഡേറ്റ്**: MCP Specification 2025-06-18-നു അനുസൃതമായി തീയതി അടിസ്ഥാനത്തിലുള്ള പതിപ്പുമായി (YYYY-MM-DD ഫോർമാറ്റ്)
- **ആർക്കിടെക്ചർ മെച്ചപ്പെടുത്തൽ**: ഹോസ്റ്റുകൾ, ക്ലയന്റുകൾ, സർവറുകൾ എന്നിവയുടെ വിവരണങ്ങൾ MCP ആർക്കിടെക്ചർ പാറ്റേണുകൾ അനുസരിച്ച് മെച്ചപ്പെടുത്തി
  - ഹോസ്റ്റുകൾ ഇപ്പോൾ MCP ക്ലയന്റ് കണക്ഷനുകൾ ഏകോപിപ്പിക്കുന്ന AI ആപ്ലിക്കേഷനുകളായി വ്യക്തമായി നിർവചിച്ചു
  - ക്ലയന്റുകൾ പ്രോട്ടോക്കോൾ കണക്ടറുകളായി ഒറ്റ-ഓരോ സർവർ ബന്ധം നിലനിർത്തുന്നതായി വിവരണം
  - സർവറുകൾ ലൊക്കൽ vs. റിമോട്ട് ഡിപ്ലോയ്മെന്റ് സീനാരിയോകളോടെ മെച്ചപ്പെടുത്തി
- **പ്രിമിറ്റീവ് പുനർഘടന**: സർവർ, ക്ലയന്റ് പ്രിമിറ്റീവുകളുടെ പൂർണ്ണ പുനർഘടന
  - സർവർ പ്രിമിറ്റീവുകൾ: റിസോഴ്‌സുകൾ (ഡാറ്റാ സോഴ്‌സുകൾ), പ്രോംപ്റ്റുകൾ (ടെംപ്ലേറ്റുകൾ), ടൂളുകൾ (എക്സിക്യൂട്ടബിൾ ഫംഗ്ഷനുകൾ) വിശദീകരണങ്ങളോടും ഉദാഹരണങ്ങളോടും
  - ക്ലയന്റ് പ്രിമിറ്റീവുകൾ: സാമ്പ്ലിംഗ് (LLM പൂർത്തീകരണങ്ങൾ), എലിസിറ്റേഷൻ (ഉപയോക്തൃ ഇൻപുട്ട്), ലോഗിംഗ് (ഡീബഗ്ഗിംഗ്/നിരീക്ഷണം)
  - നിലവിലെ ഡിസ്കവറി (`*/list`), റിട്രീവൽ (`*/get`), എക്സിക്യൂഷൻ (`*/call`) മെത്തഡ് പാറ്റേണുകൾ ഉപയോഗിച്ച് അപ്ഡേറ്റ് ചെയ്തു
- **പ്രോട്ടോക്കോൾ ആർക്കിടെക്ചർ**: രണ്ട് ലെയർ ആർക്കിടെക്ചർ മോഡൽ അവതരിപ്പിച്ചു
  - ഡാറ്റാ ലെയർ: JSON-RPC 2.0 അടിസ്ഥാനവും ലൈഫ്‌സൈക്കിൾ മാനേജ്മെന്റും പ്രിമിറ്റീവുകളും
  - ട്രാൻസ്പോർട്ട് ലെയർ: STDIO (ലൊക്കൽ) & Streamable HTTP SSE (റിമോട്ട്) ട്രാൻസ്പോർട്ട് മെക്കാനിസങ്ങൾ
- **സെക്യൂരിറ്റി ഫ്രെയിംവർക്ക്**: വ്യക്തമായ ഉപയോക്തൃ സമ്മതം, ഡാറ്റാ സ്വകാര്യത സംരക്ഷണം, ടൂൾ എക്സിക്യൂഷൻ സുരക്ഷ, ട്രാൻസ്പോർട്ട് ലെയർ സെക്യൂരിറ്റി എന്നിവ ഉൾപ്പെടുന്ന സമഗ്ര സെക്യൂരിറ്റി സിദ്ധാന്തങ്ങൾ
- **കമ്മ്യൂണിക്കേഷൻ പാറ്റേണുകൾ**: പ്രോട്ടോക്കോൾ സന്ദേശങ്ങൾ ഇൻഷിയലൈസേഷൻ, ഡിസ്കവറി, എക്സിക്യൂഷൻ, നോട്ടിഫിക്കേഷൻ ഫ്ലോകൾ കാണിക്കുന്നു
- **കോഡ് ഉദാഹരണങ്ങൾ**: MCP SDK പാറ്റേണുകൾ പ്രതിഫലിപ്പിക്കുന്ന മൾട്ടി-ഭാഷാ (.NET, ജാവ, Python, ജാവാസ്ക്രിപ്റ്റ്) ഉദാഹരണങ്ങൾ പുതുക്കി

#### സെക്യൂരിറ്റി (02-Security/) - സമഗ്ര സെക്യൂരിറ്റി പുനർഘടന

  - GitHub Advanced Security സപ്ലൈ ചെയിൻ സംരക്ഷണത്തിനായി
