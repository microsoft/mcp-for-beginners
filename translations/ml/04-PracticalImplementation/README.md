# പ്രായോഗിക നടപ്പാക്കല്‍

[![വാസ്തവത്തിലുള്ള ടൂളുകളും വര്‍ക്ക്ഫ്ലോകളുമായ MCP ആപ്പുകള്‍ എങ്ങനെ നിര്‍മിച്ച്, പരിശോധിച്ചും, ഡിപ്ലോ ചെയ്യാമെന്ന് കാണിക്കുന്ന വീഡിയോ](../../../translated_images/ml/05.64bea204e25ca891.webp)](https://youtu.be/vCN9-mKBDfQ)

_(ഈ പാഠത്തിന്റെ വീഡിയോ കാണുന്നതിന് മുകളില്‍ കാണുന്ന ചിത്രം ക്ലിക്ക് ചെയ്യുക)_

പ്രായോഗിക നടപ്പാക്കല്‍ MCP (Model Context Protocol) യുടെ ശക്തി പ്രത്യക്ഷമാകുന്ന ഭാഗമാണ്. MCP-യുടെ സിദ്ധാന്തവും ആര്‍ക്കിടെക്ചറും മനസ്സിലാക്കുന്നത് പ്രധാനമാണ്, പക്ഷേ ആരായിപ്പകര്‍ത്തലുകള്‍ പ്രായോഗികമായി പ്രയോഗിച്ച് വൈദ്യുതിക്കുന്നതില്‍ യഥാര്‍ത്ഥ മൂല്യം എത്തും. ഈ അധ്യായം ആശയവത്കരണ അറിവും കൈമാറ്റ നടപ്പാക്കലും തമ്മിലുള്ള ഗാപ് നിറയ്ക്കുന്നു, MCP-അധിഷ്ഠിത അപ്ലിക്കേഷനുകള്‍ നിര്‍മ്മിക്കുന്ന പ്രക്രിയയിലൂടെ നിങ്ങളെ നയിക്കുന്നു.

നിങ്ങള്‍ ബുദ്ധിമുട്ടുള്ള അസിസ്റ്റന്റ്മാരെ വികസിപ്പിക്കുകയോ, AI-യെ ബിസിനസ് വര്‍ക്ക്ഫ്ലോകളിലേക്ക് സംയോജിപ്പിക്കുകയോ, ഡേറ്റാ പ്രോസസ്സിങ്ങിന് കസ്റ്റം ടൂളുകള്‍ നിര്‍മിക്കുകയോ ചെയ്താലും MCP انعത്തിന്റെ ഫ്ലെക്സ്‌ബിള്‍ അടിസ്ഥാനം നല്‍കുന്നു. ഇതിന്റെ ഭാഷ-അസംബന്ധിത ഡിസൈന്‍, ജനപ്രിയപ്രോഗ്രാമിംഗ് ഭാഷകളുടെ ഔദ്യോഗിക SDKകള്‍, വിപുലമായ ഡെവലപ്പര്‍മാര്‍ക്ക് ഇടയിലാണ് ഈ പ്ലാറ്റ്ഫോം ലഭ്യമാകുന്നത്. ഈ SDK-കളുടെ സഹായത്തോടെ, വിവിധ പ്ലാറ്റ്ഫോമുകളിലെയും പരിസരങ്ങളിലെയും നിങ്ങളുടെ പരിഹാരങ്ങള്‍ വേഗത്തില്‍ പ്രോട്ടോട്ടൈപ്പ് ചെയ്യാനും പര്യാലോചന നടത്താനും വ്യാപിപ്പിക്കാനും കഴിയും.

ഇതിനെ തുടര്‍ന്നുള്ള വകുപ്പുകളില്‍, MCP-യുടെ C#, Java with Spring, TypeScript, JavaScript, Python എന്നിവയിലുള്ള പ്രായോഗിക ഉദാഹരണങ്ങള്‍, സാമ്പിള്‍ കോഡുകളും ഡിപ്ലോയ്‌മെന്റ് തന്ത്രങ്ങളും കാണും. MCP സെര്‍വറുകള്‍ ഡീബഗ് ചെയ്യാനും പരീക്ഷിക്കാനും, APIകളെ മാനേജ് ചെയ്യാനും, Azure ഉപയോഗിച്ച് ക്ലൗഡിലേക്ക് സൊല്യൂഷനുകള്‍ ഡിപ്ലോയ് ചെയ്യാനും നിങ്ങള്‍ പഠിക്കും. ഈ കൈകൊണ്ട് സഹായിക്കുന്ന വിഭവങ്ങള്‍ നിങ്ങളുടെ പഠനമേന്മ നടത്താനും ഉറച്ച, പ്രൊഡക്ഷന്‍ തയ്യാറുള്ള MCP ആപ്പ്ലിക്കേഷനുകള്‍ പ്രതിഷ്ഠിക്കാന്‍ സഹായിക്കും.

## അവലോകനം

ഈ പാഠം MCP നടപ്പാക്കലിന്റെ പ്രായോഗിക ഭാഗങ്ങളില്‍ ശ്രദ്ധ കേന്ദ്രീകരിക്കുന്നു, വ്യത്യസ്ത പ്രോഗ്രാമിംഗ് ഭാഷകളില്‍ MCP SDKകള്‍ ഉപയോഗിച്ച് കഠിനമായ ആപ്പ്ലിക്കേഷനുകള്‍ നിര്‍മ്മിക്കല്‍, MCP സെര്‍വറുകള്‍ പരിശോധനയും ഡീബഗിംഗും, റിയൂസബിൾ റിസോഴ്‌സസുകള്‍, പ്രോംപ്റ്റുകള്‍, ടൂളുകള്‍ സൃഷ്ടിക്കല്‍ തുടങ്ങിയവ പഠിക്കും.

## പഠന ലക്ഷ്യങ്ങള്‍

ഈ പാഠം തീരുമ്പോള്‍ നിങ്ങള്‍ക്ക് സാധിക്കേണ്ട കാര്യങ്ങള്‍:

- വിവിധ പ്രോഗ്രാമിംഗ് ഭാഷകളില്‍ ഔദ്യോഗിക SDKകള്‍ ഉപയോഗിച്ച് MCP പരിഹാരങ്ങള്‍ നടപ്പാക്കുക
- MCP സെര്‍വറുകള്‍ കാര്യക്ഷമമായി ഡീബഗ് ചെയ്ത് പരീക്ഷിക്കുക
- സെര്‍വര്‍ സവിശേഷതകള്‍ (റിസോഴ്‌സുകള്‍, പ്രോംപ്റ്റുകള്‍, ടൂളുകള്‍) സൃഷ്ടിക്കുകയും ഉപയോഗിക്കുകയും ചെയ്യുക
- സങ്കീര്‍ണ്ണമായ ടാസ്കുകള്‍ക്കായുള്ള MCP വര്‍ക്ക്ഫ്ലോകള്‍ രൂപകല്‍പ്പന ചെയ്യുക
- പ്രകടനക്ഷമതയുടെയും വിശ്വസ്തതയുടെയും അടിസ്ഥാനത്തില്‍ MCP നടപ്പാക്കലുകള്‍ മെച്ചപ്പെടുത്തുക

## പ്രധാനം സെര്‍വര്‍ സവിശേഷതകള്‍

MCP സെര്‍വറുകള്‍ താഴെയുള്ള സവിശേഷതകളിലെ ഏതെങ്കിലും സംയോജനം നടപ്പാക്കാം:

### റിസോഴ്‌സുകള്‍

വ്യവഹാര സംവിധാനത്തിനും AI മodel-ഉം ഉപയോഗിക്കുന്ന പിന്തുണയും ഡാറ്റയും നല്‍കുന്നു:

- ഡോക്യുമെന്റ് സ്റ്റോറുകള്‍
- നോളജ് ബേസുകള്‍
- ഘടിത ഡാറ്റ സ്രോതസ്സുകള്‍
- ഫയല്‍ സിസ്റ്റങ്ങള്‍

### പ്രോംപ്റ്റുകള്‍

ഉപഭോക്താക്കള്‍ക്കായുള്ള ടെംപ്ലേറ്റുചെയ്ത സന്ദേശങ്ങളും വര്‍ക്ക്ഫ്ലോകളും:

- മുന്‍കൂര്‍ നിര്‍ണയിച്ച സംഭാഷണ ടെംപ്ലേറ്റുകള്‍
- നയിക്കുന്ന ഇടപാടക വഴികള്‍
- പ്രത്യേക സംഭാഷണ ഘടനകള്‍

### ടൂളുകള്‍

AI മോഡലിന് നിര്‍വഹിക്കാൻ ഫംഗ്ഷനുകള്‍:

- ഡാറ്റ പ്രോസസ്സിംഗ് ഉപകരണങ്ങള്‍
- ബാഹ്യ API ഇന്റഗ്രേഷനുകൾ
- കണക്കുകള്‍ നടത്താനുള്ള കഴിവുകള്‍
- തിരയല്‍ സവിശേഷതകള്‍

## സാമ്പിൾ ഇറക്കങ്ങൾ: C# നടപ്പാക്കല്‍

സര്‍ക്കുലറായ C# SDK-ല്‍ MCP-യുടെ വ്യത്യസ്ത വശങ്ങളെ പ്രകടിപ്പിക്കുന്ന നിരവധി ഉദാഹരണങ്ങള്‍ ഉള്‍പ്പെടുന്നു:

- **അടിസ്ഥാനം MCP ക്ലയന്റ്**: MCP ക്ലയന്റ് ഉണ്ടാക്കാനും ടൂൾ കോളുകളും എങ്ങനെ ചെയ്യാമെന്ന ലളിതമായ ഉദാഹരണം
- **അടിസ്ഥാന MCP സെര്‍വര്‍**: അടിസ്ഥാന ടൂൾ രജിസ്ട്രേഷനോടെ ലഘൂകരിച്ച സെര്‍വർ നടപ്പാക്കല്‍
- **അഡ്വാന്‍സ്ഡ് MCP സെര്‍വര്‍**: ടൂൾ രജിസ്ട്രേഷൻ, ഓതന്റിക്കേഷൻ, തെറ്റ് കൈകാര്യം എന്നിവയുള്ള പൂർണ്ണ സവിശേഷത നല്‍കുന്ന സെര്‍വര്‍
- **ASP.NET ഇന്റഗ്രേഷൻ**: ASP.NET കോറിനോടുള്ള ഇന്റഗ്രേഷൻ ഉദാഹരണങ്ങൾ
- **ടൂൾ നടപ്പാക്കല്‍ മാതൃകകള്‍**: വ്യത്യസ്ത സങ്കീർണ്ണതകളുള്ള ടൂളുകളെ നടപ്പാക്കാനുള്ള വിവിധ മാതൃകകള്‍

C# MCP SDK ഇപ്പോൾ പ്രീവ്യൂ अवस्थामा ആണ്, APIകൾ മാറാൻ സാധ്യത ഉണ്ട്. SDK വികസിക്കുമ്പോൾ ഞങ്ങള്‍ ഈ ബ്ലോഗ് തുടർച്ചയായി പുതുക്കും.

- മേൽനോട്ടം

## Azure-ലേക്ക് റിമോട്ട് MCP സെര്‍വര്‍ ഡിപ്ലോയ് ചെയ്യുക

ആദ്യം പറഞ്ഞ സാമ്പിള്‍ ഡിപ്ലോയ് ചെയ്യാമോ എന്ന് നോക്കാം:

1. റെപ്പോ ക്ലോണ്‍ ചെയ്യുക

    ```bash
    git clone https://github.com/Azure-Samples/remote-mcp-apim-functions-python.git
    cd remote-mcp-apim-functions-python
    ```

2. `Microsoft.App` റിലേറ്റഡ് റിസോഴ്‌സ് പ്രൊവൈഡറെ രജിസ്റ്റര്‍ ചെയ്യുക.

   - Azure CLI ഉപയോക്താക്കളായാല്‍, `az provider register --namespace Microsoft.App --wait` നേടുക.
   - Azure PowerShell ഉപയോഗിക്കുന്നവരായാല്‍, `Register-AzResourceProvider -ProviderNamespace Microsoft.App` കമാന്‍ഡ് നടത്തുക. കുറച്ചു സമയം കഴിഞ്ഞ് `(Get-AzResourceProvider -ProviderNamespace Microsoft.App).RegistrationState` വഴി രജിസ്‌ട്രേഷൻ പൂർത്തിയായെന്ന് ഉറപ്പാക്കുക.

3. [azd](https://aka.ms/azd) കമാൻഡ് ഉപയോഗിച്ച് API മാനേജ്‌മെന്റ് സര്‍വീസ്, ഫംഗ്ഷന്‍ ആപ്പ് (കോഡിനൊപ്പം) മറ്റ് ആവശ്യമായ ക്ലൗഡ് റിസോഴ്‌സുകളുടെയും പ്രൊവിഷണിംഗ് നടത്തുക

    ```shell
    azd up
    ```
  
    ഈ കമാൻഡ് ആസ്യൂറിലെ ക്ലൗഡ് റിസോഴ്‌സുകൾ എല്ലാം ഡിപ്ലോയ് ചെയ്യും

### MCP ഇൻസ്പെക്ടറുമായി നിങ്ങളുടെ സെര്‍വർ പരിശോധിക്കല്‍

1. **പുതിയ ടെർമിനൽ വിൻഡോയിൽ**, MCP ഇൻസ്പെക്ടർ ഇൻസ്റ്റാൾ ചെയ്ത് റൺ ചെയ്യുക

    ```shell
    npx @modelcontextprotocol/inspector
    ```
  
    നിങ്ങൾക്കു ഇത്തരത്തിലുള്ള ഇന്റർഫേസ് കാണാം:

    ![Connect to Node inspector](../../../translated_images/ml/connect.141db0b2bd05f096.webp)

2. മുന്നോട്ട് കാണിച്ച URL (ഉദാ: [http://127.0.0.1:6274/#resources](http://127.0.0.1:6274/#resources)) ക്ലിക്ക് ചെയ്ത് MCP ഇൻസ്പെക്ടർ വെബ് ആപ്പ് ലോഡ് ചെയ്യുക
3. ട്രാന്‍സ്‌പോര്‍ട്ട് തരം `SSE` ആയി സെറ്റ് ചെയ്യുക
4. നിങ്ങളുടെ റണ്ണിംഗ് API മാനേജ്‌മെന്റ് SSE এন্ড്പോയിന്റ്(url), `azd up` പകരം കാണിച്ച പോലെ, സെറ്റ് ചെയ്ത് **Connect** അമര്‍ത്തുക

    ```shell
    https://<apim-servicename-from-azd-output>.azure-api.net/mcp/sse
    ```

5. **ടൂളുകള്‍ ലിസ്റ്റ് ചെയ്യുക**. ഒരു ടൂളില്‍ ക്ലിക്ക് ചെയ്ത് **Run Tool** തിരഞ്ഞെടുക്കുക.

എല്ലാ ഘട്ടങ്ങളും സഫലമായിരുന്നുവെങ്കിൽ, MCP സെര്‍വറുമായി ബന്ധിപ്പിക്കപ്പെട്ട് ഒരു ടൂൾ കോളും സാധിച്ചിട്ടുണ്ട്.

## Azure-യ്ക്ക് MCP സെര്‍വറുകള്‍

- Azure-ലേക്ക് ഡിപ്ലോയ് ചെയ്യുക: എളുപ്പം ക്ലൗഡിലേക്ക് azd up കമാൻഡ് കൊണ്ട് ഡിപ്ലോയ് ചെയ്യുക
- ക്ലയന്റ്‌മാരില്‍നിന്ന് കണക്ട് ചെയ്യുക: VS Code നിന്ന് MCP സെര്‍വര്‍ കണക്ട് ചെയ്യുക, Copilot ഏജന്റ് മോഡ്, MCP ഇൻസ്പെക്ടര്‍ ടൂള്‍ ഉള്‍പ്പെടെ

### പ്രധാന സവിശേഷതകള്‍

- ഡിസൈനിലുള്ള സുരക്ഷ: MCP സെര്‍വര്‍ കീകളും HTTPS ഉപയോഗിച്ച് സുരക്ഷിതം
- ഓതന്റിക്കേഷൻ ഓപ്ഷനുകൾ: ഒAuth പിന്തുണ ഉള്ള ബില്‍ട്ട്-ഇൻ ഓത്ത്/അഥവാ API മാനേജ്മെന്റ്
- നെറ്റ്‌വര്‍ക്ക് ഐസൊലേഷന്‍: Azure Virtual Networks (VNET) മുഖേന നെറ്റ്‌വര്‍ക്ക് ഐസൊലേഷന്‍ അനുവദിക്കുന്നു
- സെര്‍വര്‍ലെസ് ആര്‍ക്കിടെക്ചര്‍: അധിഷ്ഠിത ഫംഗ്ഷനുകൾ ഉപയോഗിച്ച് സ്കെയിലബിൾ, ഇവന്റ് ഡ്രിവൻ നടപ്പാക്കല്‍
- ലൊക്കല്‍ ഡെവലപ്പ്മെന്റ്: സമഗ്രമായ ലൊക്കല്‍ ഡെവലപ്പ്മെന്റ്, ഡീബഗിങ്ങ് പിന്തുണ
- ലളിതമായ ഡിപ്ലോയ്‌മെന്റ്: Azure-ല്‍ streamlined ഡിപ്ലോയ്മെന്റ് പ്രക്രിയ

റെപ്പോസിറ്ററിയിൽ എല്ലാ ആവശ്യമുള്ള കോൺഫിഗറേഷൻ ഫയലുകളും ഉറവിട കോഡും ഇൻഫ്രാസ്ട്രക്ചർ നിർവചനങ്ങളും ഉൾക്കൊള്ളുന്നു, അതിനാൽ പ്രൊഡക്ഷന്‍ MCP സെര്‍വർ നടപ്പാക്കല്‍ വേഗത്തിൽ തുടങ്ങാനാകും.

- MCP നടപ്പാക്കല്‍ സുരക്ഷ, പ്രകടനം, തെറ്റ് കൈകാര്യം എന്നിവ പരിഗണിച്ച് ചെയ്യേണ്ടതാണ്

## പരിപാടി

നിങ്ങളുടെ മേഖലയില്‍ യഥാര്‍ത്ഥ പ്രശ്‌നം പരിഹരിക്കുന്ന ഒരു പ്രായോഗിക MCP വര്‍ക്ക്ഫ്ലോ രൂപകല്‍പ്പന ചെയ്യുക:

1. ഈ പ്രശ്‌നത്തിന് സഹായകമായ 3-4 ടൂളുകള്‍ കണ്ടെത്തുക
2. ഈ ടൂളുകള്‍ എങ്ങനെയാണ് ഇടപെട്ടുകൊണ്ടിരിക്കുന്നതെന്ന് ഒരു വര്‍ക്ക്ഫ്ലോ ചാര്‍ട്ട് രൂപകല്‍പ്പന ചെയ്യുക
3. നിങ്ങളുടെ ഇഷ്ടഭാഷ ഉപയോഗിച്ച് ഒരു ടൂളിന്റെ ലളിതമായ ഒരു പതിപ്പ് നടപ്പാക്കുക
4. മോഡല്‍ നിങ്ങളുടെ ടൂൾ ഫലപ്രദമായി ഉപയോഗിക്കാനായി സഹായകമായ ഒരു പ്രോംപ്റ്റ് ടെംപ്ലേറ്റ് സൃഷ്ടിക്കുക
