# താത്പര്യപരമായ നടപ്പാക്കല്‍

[![നിങ്ങളുടെ MCP ആപ്ലിക്കേഷനുകള്‍ യഥാര്‍ത്ഥ ഉപകരണങ്ങളും പ്രവൃത്തിശൈലികളും ഉപയോഗിച്ച് എങ്ങനെ നിര്‍മ്മിക്കാം, പരീക്ഷിക്കാം, വിന്യസിക്കാം](../../../translated_images/ml/05.64bea204e25ca891.webp)](https://youtu.be/vCN9-mKBDfQ)

_(ഈ പാഠത്തിന്റെ വീഡിയോ കാണാന്‍ മുകളില്‍ ചിത്രത്തെ ക്ലിക്ക് ചെയ്യുക)_

Model Context Protocol (MCP) യുടെ ശക്തി സാക്ഷാത്കരിക്കുന്നിടമാണ് താത്പര്യപരമായ നടപ്പാക്കലെന്നത്. MCPയുടെ സിദ്ധാന്തവും ഭൗതികഘടനയും മനസ്സിലാക്കുന്നത് അത്യന്താപേക്ഷിതമാണെങ്കിലും, യാഥാര്‍ത്ഥ പ്രശ്നങ്ങള്‍ക്ക് പരിഹാരങ്ങളുണ്ടാക്കാന്‍ ഈ ആശയങ്ങള്‍ പ്രയോഗിക്കുമ്പോഴാണ് യഥാര്‍ത്ഥ മൂല്യം തെളിയുന്നത്. ഈ അദ്ധ്യായം സങ്കല്‍പ്പാതരമായ അറിവിനും പ്രായോഗിക വികസനത്തിനുമിടയില്‍ പാലം പണിയുന്നു, MCP അടിസ്ഥാനമാക്കിയുള്ള ആപ്ലിക്കേഷനുകള്‍ സജീവമാക്കാനുള്ള പ്രക്രിയയിലൂടെ നിങ്ങള്‍ നയിക്കുന്നു.

ബുദ്ധിവൃത്തികളുള്ള സഹായികളെ വളരെക്കാലം നിര്‍മിക്കുകയോ, എഐയെ ബിസിനസ് പ്രവൃത്തിശൈലികളുമായി സംയോജിപ്പിക്കുകയോ, ഡേറ്റാ പ്രോസസ്സിംഗിനായി ഇഷ്ടാനുസൃത ഉപകരണങ്ങള്‍ നിര്‍മിക്കുകയോ ചെയ്യുമ്പോഴും MCP ഒരു അന്വയശീലമായ അടിസ്ഥാനം നല്‍കുന്നു. അതിന്റെ ഭാഷാ-അഗ്നോസ്റ്റിക് ഡിസൈനും പ്രചാരത്തിലുള്ള പ്രോഗ്രാമിംഗ് ഭാഷകളുടെ ഔദ്യോഗിക SDKകളും വിപുലമായ ഡെവലപ്പര്‍മാര്‍ക്കുള്ള പ്രവേശനം സൌകര്യമാക്കുന്നു. ഈ SDKകള്‍ ഉപയോഗിച്ച് നിങ്ങള്‍ വേഗത്തില്‍ പ്രോട്ടോടൈപ്പ് നിര്‍മ്മിക്കാനും, ആവര്‍ത്തിക്കാനും, വ്യത്യസ്ത പ്ലാറ്റ്ഫോമുകളിലെയും പരിസ്ഥിതികളിലെയും ത്തുമായി നിങ്ങളുടെ പരിഹാരങ്ങള്‍ വലുതാക്കി എത്തിക്കാം.

ബാലപ്രത്യേക പരിച്ഛേദങ്ങളില്‍ നിന്ന്, C#, ജാവാ സ്പ്രിംഗ് ഉപയോഗിച്ച്, ടൈപ്പ്സ്ക്രിപ്റ്റ്, ജാവാസ്ക്രിപ്റ്റ്, പൈത്തണ്‍ എന്നിവയില്‍ MCP എങ്ങനെ നടപ്പാക്കാമെന്ന് കാണിക്കുന്ന പ്രായോഗിക ഉദാഹരണങ്ങള്‍, സാമ്പിൾ കോഡ്, വിന്യസന തന്ത്രങ്ങള്‍ നിങ്ങള്‍ കണ്ടെത്തും. MCP സേവര്‍മാര്‍ എങ്ങനെ ഡീബഗ് ചെയ്യുകയും തിയടുകയും ചെയ്യാമെന്നും, APIകളെ എങ്ങനെ കൈകാര്യം ചെയ്യാമെന്നും, ആസ്യൂറില്‍ ക്ലൗഡിലേക്ക് പരിഹാരങ്ങള്‍ വിന്യസിക്കാമെന്നും നിങ്ങള്‍ പഠിക്കും. ഈ പ്രായോഗിക വിഭവങ്ങള്‍ നിങ്ങളുടെ പഠനത്തെ വേഗത്തിലാക്കുകയും വിശ്വസനീയമായ, പ്രൊഡക്ഷന്‍-സജ്ജമായ MCP ആപ്ലിക്കേഷനുകള്‍ സ്വയം നിര്‍മിക്കാനുള്ള ആത്മവിശ്വാസം വര്‍ദ്ധിപ്പിക്കാനും രൂപകല്പന ചെയ്തതാണ്.

## അവലോകനം

ഈ പാഠം, MCP നടപ്പാക്കലിന്റെ പ്രായോഗിക വശങ്ങളെ വിവിധ പ്രോഗ്രാമിംഗ് ഭാഷകളിലൂടെ ശ്രദ്ധിക്കുന്നതാണ്. C#, ജാവാ സ്പ്രിംഗ് ഉപയോഗിച്ച്, ടൈപ്പ്സ്ക്രിപ്റ്റ്, ജാവാസ്ക്രിപ്റ്റ്, പൈത്തണ്‍ SDKകള്‍ ഉപയോഗിച്ച് കരുത്തുറ്റ ആപ്ലിക്കേഷനുകള്‍ നിർമ്മിക്കുന്നതിനും MCP സേവര്‍മാര്‍ ഡീബഗ് ചെയ്ത് പരീക്ഷിക്കുന്നതിനും, പ്രയോജനപ്പെടുത്താവുന്ന വിഭവങ്ങള്‍, പ്രോംപ്റ്റുകൾ, ഉപകരണങ്ങള്‍ സൃഷ്ടിക്കുന്നതിനും നാം അന്വേഷിക്കും.

## പഠന ലക്ഷ്യങ്ങള്‍

ഈ പാഠം പൂര്‍ത്തിയാകുമ്പോള്‍ നിങ്ങള്‍ക്ക് സാധിക്കാനാകും:

- വിവിധ പ്രോഗ്രാമിംഗ് ഭാഷകളിലെ ഔദ്യോഗിക SDKകള്‍ ഉപയോഗിച്ച് MCP പരിഹാരങ്ങള്‍ നടപ്പാക്കുക
- MCP സേവര്‍മാര്‍ സമഗ്രമായി ഡീബഗ് ചെയ്തു പരീക്ഷിക്കുക
- സേര്‍വര്‍ ഫീച്ചറുകള്‍ (വിഭവങ്ങള്‍, പ്രോംപ്റ്റുകള്‍, ഉപകരണങ്ങള്‍) സൃഷ്ടിച്ച് ഉപയോഗിക്കുക
- സങ്കീര്‍ണ്ണ പ്രവൃത്തികള്‍ക്കായി ഫലപ്രദമായ MCP പ്രവൃത്തി പ്രവാഹങ്ങള്‍ രൂപകല്‍പന ചെയ്യുക
- പ്രകടനവും വിശ്വസനീയതയും ഉപയുക്തമാക്കാന്‍ MCP നടപ്പാക്കലുകള്‍ മെച്ചപ്പെടുത്തുക

- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk)
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk)

- [Go SDK](https://github.com/modelcontextprotocol/go-sdk)

## പ്രധാന സേര്‍വര്‍ ഫീച്ചറുകള്‍

MCP സേര്‍വറുകള്‍ ഈ ഫീച്ചറുകളുടെ ഏതെങ്കിലും സംയോജനം നടപ്പിലാക്കാം:

### വിഭവങ്ങള്‍

ഉപയോക്താവിനോ AI മോഡലിനോ ഉപയോഗിക്കാന്‍ ഉള്ള പ്രেক্ষിതവും ഡാറ്റയും വിഭവങ്ങള്‍ നല്‍കുന്നു:

- രേഖാ സംഭാരശേഖരങ്ങള്‍
- അറിവ് അടിസ്ഥാനങ്ങള്‍
- ഘടിത ഡാറ്റ സ്രോതസ്സുകള്‍
- ഫയല്‍ സിസ്റ്റങ്ങള്‍

### പ്രോംപ്റ്റുകള്‍

ഉപയോക്താക്കള്‍ക്കുള്ള ടെംപ്ലേറ്റു സന്ദേശങ്ങളുടെയും പ്രവൃത്തി പ്രവാഹങ്ങളുടെയും സങ്കല്പമാണ്:

- മുന്‍നിര്‍ദേശിച്ച സംഭാഷണ ടെംപ്ലേറ്റുകള്‍
- മാര്‍ഗ്ഗനിര്‍ദേശ ചാറ്റ് മാതൃകകള്‍
- പ്രത്യേക രേഖാമൊഴി ഘടനകള്‍

### ഉപകരണങ്ങള്‍

AI മോഡല്‍ പ്രവര്‍ത്തിപ്പിക്കാന്‍ സഹായിക്കുന്ന ഫംഗ്ഷനുകള്‍:

- ഡാറ്റ പ്രോസസ്സിംഗ് ഉപകരണങ്ങള്‍
- ബാഹ്യ API സംയോജിപ്പിക്കല്‍
- കംപ്യൂട്ടേഷണല്‍ കഴിവുകള്‍
- തിരയല്‍ സവിശേഷതകള്‍

## MCP C# നടപ്പാക്കല്‍: സാമ്പിൾ

ഓദ്യോഗിക C# SDK റിപോസിറ്ററിയില്‍ MCPയുടെ വ്യത്യസ്ത വശങ്ങള്‍ കാണിക്കുന്ന നിരവധി സാമ്പിളുകള്‍ ഉണ്ട്:

- **അടിസ്ഥാന MCP ക്ലയന്റ്**: എങ്ങനെ MCP ക്ലയന്റ് സൃഷ്ടിച്ച് ഉപകരണങ്ങള്‍ വിളിക്കാമെന്ന് കാണിക്കുന്ന ലളിത ഉദാഹരണം
- **അടിസ്ഥാന MCP സേര്‍വര്‍**: അടിസ്ഥാന ഉപകരണ രജിസ്ട്രേഷന്‍ ഉള്ള കുറഞ്ഞ സേര്‍വര്‍ നടപ്പാക്കല്‍
- **വർദ്ധിച്ച MCP സേര്‍വര്‍**: ഉപകരണ രജിസ്ട്രേഷന്‍, അംഗീകാരം, പിശക് കൈകാര്യം ചെയ്യല്‍ എന്നിവയുമായി സമഗ്ര സേര്‍വര്‍
- **ASP.NET സംയോജനം**: ASP.NET കോരെ സംയോജിപ്പിച്ചുള്ള ഉദാഹരണങ്ങള്‍
- **ഉപകരണ നടപ്പാക്കല്‍ മാതൃകകള്‍**: വ്യത്യസ്ത സങ്കീര്‍ണ്ണതയുള്ള ഉപകരണങ്ങള്‍ നടപ്പാക്കാനുള്ള വിവിധ മാതൃകകള്‍

MCP C# SDK പ്രിവ്യൂ നിലയിലാണു; APIകളില്‍ മാറ്റം വരാം. SDK വികസനമുണ്ടാകുന്നതിനൊപ്പം ഈ ബ്ലോഗ് സജീവമായി പുതുക്കുന്നതാണ്.

### പ്രധാന സവിശേഷതകള്‍

- [C# MCP Nuget ModelContextProtocol](https://www.nuget.org/packages/ModelContextProtocol)
- നിങ്ങളുടെ ആദ്യ MCP സേര്‍വര്‍ നിര്‍മിക്കുന്നു: [https://devblogs.microsoft.com/dotnet/build-a-model-context-protocol-mcp-server-in-csharp/](https://devblogs.microsoft.com/dotnet/build-a-model-context-protocol-mcp-server-in-csharp/)

പുരന്ന C# നടപ്പാക്കല്‍ സാമ്പിളുകള്‍ കാണാന്‍, [ഓദ്യോഗിക C# SDK സാമ്പിള്‍ റിപോസിറ്ററി](https://github.com/modelcontextprotocol/csharp-sdk) സന്ദര്‍ശിക്കുക.

## API മാനേജ്മെന്റ്

MCP സേര്‍വറുകള്‍ സൗരക്ഷിതമാക്കാന്‍ Azure API Management ഉത്തമ പരിഹാരമാണ്. ആശയം, നിങ്ങളുടെ MCP സേര്‍വറിനു മുന്‍പില്‍ Azure API Management സ്ഥാപിച്ച് താഴെപ്പറയുന്ന സവിശേഷതകള്‍ കൈകാര്യം ചെയ്യുക എന്നതാണ്:

- നിരക്ക് നിയന്ത്രണം
- ടോക്കണ്‍ മാനേജ്മെന്റ്
- മേല്‍നോട്ടം
- ലോഡ് ബാലന്‍സിംഗ്
- സുരക്ഷ

### Azure സാമ്പിൾ

ഈ Azure സാമ്പിൾ MCP സേര്‍വര്‍ സൃഷ്ടിക്കുകയും Azure API Management ഉപയോഗിച്ച് അത് സുരക്ഷിതമാക്കുകയും ചെയ്യുന്നു: [creating an MCP Server and securing it with Azure API Management](https://github.com/Azure-Samples/remote-mcp-apim-functions-python).

## ജലം MCP സേര്‍വര്‍ Azure-ലേക്ക് വിന്യസിക്കല്‍

മുന്‍പ് പരാമര്‍ശിച്ച സാമ്പിള്‍ Azure-ലേക്ക് വിന്യസിക്കാമോ നോക്കാം:

1. റിപൊ ക്ലോണ്‍ ചെയ്യുക

    ```bash
    git clone https://github.com/Azure-Samples/remote-mcp-apim-functions-python.git
    cd remote-mcp-apim-functions-python
    ```

1. `Microsoft.App` വിഭവം പ്രൊവൈഡറെ രജിസ്റ്റര്‍ ചെയ്യുക.

   - Azure CLI ഉപയോഗിക്കുകയാണെങ്കില്‍ `az provider register --namespace Microsoft.App --wait` ഓടിക്കുക.
   - Azure PowerShell ഉപയോഗിക്കുകയാണെങ്കില്‍ `Register-AzResourceProvider -ProviderNamespace Microsoft.App` ഓടിക്കുക. പകുതിയില്‍ `(Get-AzResourceProvider -ProviderNamespace Microsoft.App).RegistrationState` ഉപയോഗിച്ച് രജിസ്ട്രേഷന്‍ പൂര്‍ത്തിയാണോ എന്ന് പരിശോധിക്കുക.

1. ഈ [azd](https://aka.ms/azd) കമാന്‍ഡ് ഓടിക്കുക. ഇത് API മാനേജ്മെന്റ് സർവീസ്, function app (കോഡോടുകൂടി), മറ്റ് ആവശ്യമായ Azure വിഭവങ്ങള്‍ ഒരുക്കും

    ```shell
    azd up
    ```

    ഈ കമാന്‍ഡുകൾ സകല ക്ലൗഡ് വിഭവങ്ങളും Azure-ല്‍ വിന്യസിക്കും

### MCP ഇൻസ്പക്ടറിനൊപ്പം സെര്‍വര്‍ പരീക്ഷണം

1. **പുതിയ ടേര്‍ഷറിന് വിന്‍ഡോവില്‍** MCP ഇൻസ്പക്ടർ ഇൻസ്റ്റാൾ ചെയ്ത് ഓടിക്കുക

    ```shell
    npx @modelcontextprotocol/inspector
    ```

    ഇതിനകം ഇന്റ്റര്‍ഫേസ് ഇങ്ങനെ കാണാം:

    ![Connect to Node inspector](../../../translated_images/ml/connect.141db0b2bd05f096.webp)

1. ആപ്പ് പ്രദര്‍ശിപ്പിക്കുന്ന URL (ഉദാ: [http://127.0.0.1:6274/#resources](http://127.0.0.1:6274/#resources)) CONTROL ക്ലിക്ക് ചെയ്ത് MCP ഇൻസ്പക്ടർ വെബ് ആപ്പ് ലോഡ് ചെയ്യുക
1. ട്രാൻസ്പോർട് തരം `SSE` ആക്കി സെറ്റ് ചെയ്യുക
1. നിങ്ങളുടെ പ്രവർത്തിക്കുന്ന API Management SSE എന്റ്പോയിന്റ് `azd up` കഴിഞ്ഞ് പ്രദര്‍ശിപ്പിക്കുന്നതിന് അനുസൃതമായി URL സെറ്റ് ചെയ്ത് **Connect** ക്ലിക്ക് ചെയ്യുക:

    ```shell
    https://<apim-servicename-from-azd-output>.azure-api.net/mcp/sse
    ```

1. **ടൂൾ ലിസ്റ്റ് ചെയ്യുക**. ഒരു ഉപകരണം തിരഞ്ഞെടുക്കുക, **Run Tool** ക്ലിക്ക് ചെയ്യുക.

എല്ലാ ഘട്ടങ്ങളും ശരിയായി പൂരിപ്പിച്ചാൽ, ഇപ്പോൾ MCP സേര്‍വറുമായി ബന്ധിപ്പിച്ചോളും ഉപകരണം വിളിച്ചോളും കഴിഞ്ഞിരിക്കും.

## Azure-ലുള്ള MCP സേര്‍വറുകള്‍

[Remote-mcp-functions](https://github.com/Azure-Samples/remote-mcp-functions-dotnet): Python, C# .NET, Node/TypeScript ഉപയോഗിച്ച് Azure Functions വഴി ഇഷ്ടാനുസൃത റിമോട്ട് MCP (Model Context Protocol) സേര്‍വറുകള്‍ നിര്‍മ്മിക്കാനും വിന്യസിക്കാനുമായുളള ത്വരിത തുടക്ക ടെംപ്ലേറ്റ് സെറ്റ്.

സാമ്പിളുകള്‍ ഡെവലപ്പര്‍മാര്‍ക്ക് ഈ കഴിവുകള്‍ നല്‍കുന്നു:

- പ്രാദേശിക അടിസ്ഥാനത്തില്‍ സേര്‍വര്‍ നിര്‍മ്മിച്ച് ഓടിക്കുക: പ്രാദേശിക മെഷീന്‍ വഴി MCP സേര്‍വർ ഡെവലപ്പ്മെന്റ് നും ഡീബഗ് ചെയ്യല്‍
- Azure-ല്‍ വിന്യസിക്കുക: ലളിതമായ azd up കമാന്‍ഡ് ഉപയോഗിച്ച് എളുപ്പം ക്ലൗഡിലേക്ക് വിന്യസിക്കുക
- ക്ലയന്റുകളില്‍ നിന്നു ബന്ധിപ്പിക്കുക: VS Code-ന്റെ Copilot ഏജന്റ്മോഡ് ഉള്‍പ്പെടെയുള്ള വിവിധ ക്ലയന്റുകളില്‍നിന്നു MCP സേര്‍വറുമായി കണക്ട് ചെയ്യുക

- MCP SDKകള്‍ കരുത്തുറ്റ MCP പരിഹാരങ്ങള്‍ നടപ്പാക്കാനുള്ള ഭാഷാനുസൃത ഉപകരണങ്ങള്‍ നല്‍കുന്നു
- ഡീബഗും പരീക്ഷണവും MCP ആപ്ലിക്കേഷനുകളുടെ വിശ്വസനീയതയ്ക്ക് അനിവാര്യമാണ്
- പുനര്‍പ്രയോജനപ്പെടാനുളള പ്രോംപ്റ്റ് ടെംപ്ലേറ്റുകള്‍ സ്ഥിരതയുള്ള AI ഇടപെടലുകള്‍ സാധ്യമാക്കും
- നന്നായി രൂപകല്‍പന ചെയ്ത പ്രവൃത്തി പ്രവാഹങ്ങള്‍ നിരവധി ഉപകരണങ്ങള്‍ ഉപയോഗിച്ച് സങ്കീർണ്ണ പ്രവൃത്തികള്‍ ഏകോപിപ്പിക്കും
- MCP പരിഹാരങ്ങള്‍ നടപ്പാക്കുമ്പോള്‍ സുരക്ഷ, പ്രകടനം, പിശക് കൈകാര്യം ചെയ്യല്‍ മുന്‍കൂട്ടി പരിഗണിക്കണം

## അഭ്യാസം

നിങ്ങളുടെ മേഖലയില്‍ യഥാര്‍ത്ഥ പ്രശ്നം പരിഹരിക്കുന്ന പ്രായോഗിക MCP പ്രവൃത്തി പ്രവാഹം രൂപകല്‍പന ചെയ്യുക:

1. ഈ പ്രശ്നം പരിഹരിക്കാന്‍ സഹായകമായ 3-4 ഉപകരണങ്ങള്‍ തിരഞ്ഞടുക്കുക
2. ഈ ഉപകരണങ്ങള്‍ എങ്ങെത്തന് ഉപയോഗിക്കുന്നുവെന്ന് കാണിക്കുന്ന പ്രവൃത്തി പ്രവാഹ എട്ട മാതൃക വരയ്ക്കുക
3. നിങ്ങളുടെ ഇഷ്ട ഭാഷയില്‍ ഒരു ഉപകരണത്തിന്റെ അടിസ്ഥാന പതിപ്പ് നടപ്പിലാക്കുക
4. മോഡലിന് നിങ്ങളുടെ ഉപകരണം ഫലപ്രദമായി ഉപയോഗിക്കാന്‍ സഹായിക്കുന്ന പ്രോംപ്റ്റ് ടെംപ്ലേറ്റ് സൃഷ്ടിക്കുക
