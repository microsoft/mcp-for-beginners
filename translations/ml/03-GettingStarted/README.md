## ആരംഭിക്കുന്നത്  

[![നിങ്ങളുടെ ആദ്യ MCP സെർവർ നിർമ്മിക്കൂ](../../../translated_images/ml/04.0ea920069efd979a.webp)](https://youtu.be/sNDZO9N4m9Y)

_(ഈ പാഠത്തിന്റെ വീഡിയോ കാണാൻ മുകളിൽ ചിത്രത്തെ ക്ലിക്ക് ചെയ്യുക)_

ഈ വിഭാഗം നിരവധി പാഠങ്ങളായി 구성ിച്ചിരിക്കുന്നു:

- **1 നിങ്ങളുടെ ആദ്യ സെർവർ**, ഈ ആദ്യ പാഠത്തിൽ, നിങ്ങൾ എങ്ങനെ നിങ്ങളുടെ ആദ്യ സെർവർ സൃഷ്ടിച്ച് ഇൻസ്പെക്ടർ ടൂൾ ഉപയോഗിച്ച് പരിശോധിക്കാമെന്ന് പഠിക്കും, ഇത് നിങ്ങളുടെ സെർവർ പരിശോധനക്കും ഡീബഗിംഗിനും മൂല്യമുള്ള മാർഗമാണ്, [പാഠത്തിലേക്ക്](01-first-server/README.md)

- **2 ക്ലയന്റ്**, ഈ പാഠത്തിൽ, നിങ്ങൾ എങ്ങനെ ഒരു സെർവറിന് കണക്‌ട് ചെയ്യാൻ കഴിയുന്ന ക്ലയന്റ് എഴുതാമെന്ന് പഠിക്കാം, [പാഠത്തിലേക്ക്](02-client/README.md)

- **3 LLM ներառിച്ച ക്ലയന്റ്**, ഒരു LLM ചേർത്തുകൊണ്ട് ക്ലയന്റ് എഴുതുന്നത് മെച്ചപ്പെട്ട മാർഗമാണ്, ഇതുവഴി സെർവറുമായി "നിയമനം" നടത്താൻ കഴിയും, [പാഠത്തിലേക്ക്](03-llm-client/README.md)

- **4 Visual Studio Code-ൽ GitHub Copilot ഏജൻറ് മോഡിൽ സെർവർ ഉപയോഗിക്കുന്നത്**. ഇവിടെ, Visual Studio Code-ൽ നിന്നുള്ള MCP സെർവർ പ്രവർത്തനമെന്ന് കാണിക്കുന്നു, [പാഠത്തിലേക്ക്](04-vscode/README.md)

- **5 stdio ട്രാൻസ്പോർട്ട് സെർവർ** stdio transport ആണ് പ്രാദേശിക MCP സെർവർ-ക്ലയന്റ് ആശയവിനിമയത്തിനുള്ള ശിപാർശ ചെയ്ത സ്റ്റാൻഡേർഡ്, ശക്തമായ ഉപപ്രക്രിയ അടിസ്ഥാന ആശയവിനിമയം ഉൾക്കൊള്ളുന്നു [പാഠത്തിലേക്ക്](05-stdio-server/README.md)

- **6 MCP യുമായി HTTP സ്ട്രീമിംഗ് (Streamable HTTP)**. ആധുനിക HTTP സ്ട്രീം ട്രാൻസ്പോർട്ട് വിശദീകരിക്കുകയും (സൂചിപ്പിച്ച പോലെ [MCP സ്പെസിഫിക്കേഷൻ 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/basic/transports/#streamable-http)), പുരോഗതി വിവരം നൽകലും, Streamable HTTP ഉപയോഗിച്ച് ഉയര്‍ന്ന തോതിൽ MCP സെർവർകളും ക്ലയന്റുകളും എങ്ങിനെ നടപ്പിലാക്കാമെന്നുമെല്ലാം പഠിക്കുക. [പാഠത്തിലേക്ക്](06-http-streaming/README.md)

- **7 VSCode-നുള്ള AI കിറ്റ് ഉപയോഗിക്കൽ** നിങ്ങളുടെ MCP ക്ലയന്റുകളും സെർവർകളും ഉപയോഗിച്ച് പരീക്ഷണങ്ങളും ഘടിപ്പിക്കാനും [പാഠത്തിലേക്ക്](07-aitk/README.md)

- **8 പരിശോധന**. ഇവിടെ പ്രത്യേകിച്ച് സെർവർ, ക്ലയന്റുകൾ വ്യത്യസ്ത വഴികളിൽ എങ്ങനെ പരീക്ഷിക്കാമെന്നു കാണിക്കും, [പാഠത്തിലേക്ക്](08-testing/README.md)

- **9 വിന്യസനം**. നിങ്ങളുടെ MCP പരിഹാരങ്ങൾ വിന്യസിക്കുന്ന വ്യത്യസ്ത മാർഗങ്ങൾ പരിശോധിക്കുന്നു, [പാഠത്തിലേക്ക്](09-deployment/README.md)

- **10 ആഡ്‍വാൻസ്ഡ് സെർവർ ഉപയോഗം**. ഈ അദ്ധ്യായം ആഡ്‍വാൻസ്ഡ് സെർവർ ഉപയോഗത്തെ ഉൾക്കൊള്ളുന്നു, [പാഠത്തിലേക്ക്](./10-advanced/README.md)

- **11 അഥ്**. ബേസിക് അഥിൽ നിന്ന് JWT, RBAC വരെ ലളിതമായ അഥ് എങ്ങനെ ചേർക്കാമെന്ന് പഠിക്കുക. ഇവിടെ ആരംഭിച്ച് പിന്നീട് അദ്ധ്യായം 5-ൽ ആഡ്‍വാൻസ്ഡ് വിഷയങ്ങൾ കൂടി നോക്കി 2-ാം അദ്ധ്യായത്തിലെ നിർദ്ദേശങ്ങൾ പാലിച്ച് അധിക സുരക്ഷ ലഭിക്കാം, [പാഠത്തിലേക്ക്](./11-simple-auth/README.md)

- **12 MCP ഹോസ്റ്റ്‌സ്**. പ്രചാരത്തിലുള്ള MCP ഹോസ്റ്റ് ക്ലയന്റുകൾ നിയന്ത്രിക്കുകയും ഉപയോഗിക്കുകയും ചെയ്യുക, ഉദാഹരണത്തിന് Claude Desktop, Cursor, Cline, Windsurf. ട്രാൻസ്പോർട്ട് തരം, പ്രശ്നപരിഹാരങ്ങൾ പഠിക്കുക, [പാഠത്തിലേക്ക്](./12-mcp-hosts/README.md)

- **13 MCP ഇൻസ്പെക്ടർ**. MCP ഇൻസ്പെക്ടർ ഉപകരണം ഉപയോഗിച്ച് MCP സെർവർകൾ ഡീബഗ് ചെയ്ത് പരീക്ഷിക്കുക. ടൂൾസ്, സ്രോതസ്സ്, പ്രോട്ടോകോൾ സന്ദേശങ്ങൾ സംശോധിപ്പിക്കാനും പഠിക്കുക, [പാഠത്തിലേക്ക്](./13-mcp-inspector/README.md)

- **14 സാമ്പ്ലിംഗ്**. LLM ബന്ധപ്പെട്ട പ്രവർത്തനങ്ങളിൽ MCC ക്ലയന്റുകളുമായി സഹകരിക്കുന്ന MCP സെർവർ നിർമിക്കുക. [പാഠത്തിലേക്ക്](./14-sampling/README.md)

- **15 MCP ആപ്ലിക്കേഷനുകൾ**. UI നിർദേശങ്ങളോടുകൂടിയ MCP സെർവർ നിർമ്മിക്കുക, [പാഠത്തിലേക്ക്](./15-mcp-apps/README.md)

Model Context Protocol (MCP) ഒരു തുറന്ന പ്രോട്ടോകോൾ ആണ്, ഇത് എങ്ങനെ അപ്ലിക്കേഷനുകൾ LLM-കൾക്ക് സൂചിപ്പിക്കുകയുള്ളൂ എന്ന് സ്റ്റാൻഡർഡൈസ് ചെയ്യുന്നു. MCP-നെ AI അപ്ലിക്കേഷനുകൾക്കുള്ള USB-C പോർട്ട്‌ ആണെന്ന് കരുതുക - അത് വിവിധ ഡാറ്റ ഉറവിടങ്ങൾക്കും ഉപകരണങ്ങൾക്കുമായി AI മോഡലുകൾ കണക്റ്റ് ചെയ്യാനുള്ള സ്റ്റാൻഡേർഡ് വഴി നൽകുന്നു.

## പഠന ലക്ഷ്യങ്ങൾ

ഈ പാഠം അവസാനം, നിങ്ങൾക്ക് സാധിക്കുമെന്ന്:

- C#, Java, Python, TypeScript, JavaScript എന്നിവയിൽ MCP വികസനങ്ങൾ സജ്ജമാക്കുക
- ഇഷ്ടാനുസൃത ഫീച്ചറുകളുള്ള (ഉപകരണങ്ങൾ, പ്രൊംപ്റ്റുകൾ, ടൂളുകൾ) അടിസ്ഥാന MCP സെർവർ നിർമ്മിച്ച് വിന്യസിക്കുക
- MCP സെർവർ‌കൾക്ക് കണക്ട് ചെയ്യുന്ന ഹോസ്റ്റ് അപ്ലിക്കേഷനുകൾ സൃഷ്ടിക്കുക
- MCP നടപ്പാക്കലുകൾ പരീക്ഷിക്കുകയും ഡീബഗ് ചെയ്യുകയും ചെയ്യുക
- സാധാരണ രൂപീകരണ പ്രയാസങ്ങളും അവയുടെ പരിഹാരങ്ങളും മനസ്സിലാക്കുക
- Your MCP നടപ്പാക്കലുകൾ പ്രചാരത്തിലുള്ള LLM സേവനങ്ങളുമായി കണക്റ്റ് ചെയ്യുക

## നിങ്ങളുടെ MCP പരിസ്ഥിതി സജ്ജമാക്കൽ

MCP-യുമായി പ്രവർത്തനം തുടങ്ങുന്നതിന് മുൻപ്, നിങ്ങളുടെ ഡെവലപ്മെന്റ് പരിസ്ഥിതി തയ്യാറാക്കിയല്ലോ എന്ന് ഉറപ്പാക്കുക, അടിസ്ഥാന പ്രവൃത്തി പ്രവണതകൾ മനസ്സിലാക്കുക അത്യാവശ്യമാണ്. ഇത് MCP-നൊപ്പം സുഖകരമായി ആരംഭിക്കാനുള്ള പ്രാഥമിക ക്രമീകരണങ്ങൾ 안내 ചെയ്യും.

### മുൻപരിചയങ്ങൾ

MCP വികസനത്തിലേക്ക് കടക്കുന്നതിനു മുൻപ്, നിങ്ങൾക്കുണ്ടെന്ന് ഉറപ്പാക്കുക:

- **ഡെവലപ്മെന്റ് പരിസ്ഥിതി**: നിങ്ങൾ തിരഞ്ഞെടുത്ത ഭാഷയ്ക്ക് (C#, Java, Python, TypeScript, JavaScript)
- **ഐഡിയ/എഡിറ്റർ**: Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm, അല്ലെങ്കിൽ ഏതെങ്കിലും ആധുനിക കോഡ് എഡിറ്റർ
- **പാക്കേജ് മാനേജർമാർ**: NuGet, Maven/Gradle, pip, npm/yarn
- **API കീകൾ**: നിങ്ങളുടെ ഹോസ്റ്റ് അപ്ലിക്കേഷനുകളിൽ ഉപയോഗിക്കാൻ ആഗ്രഹിക്കുന്ന ഏതെങ്കിലും AI സേവനങ്ങൾക്കായി

### ഔദ്യോഗിക SDK കളി

വരാനിരിക്കുന്ന അദ്ധ്യായങ്ങളിൽ Python, TypeScript, Java, .NET അധിഷ്ഠിത പരിഹാരങ്ങൾ കാണും. ഔദ്യോഗികമായി പിന്തുണയ്ക്കുന്ന SDK കളിതന്നെ ഇവയാണു.

MCP വിവിധ ഭാഷകൾക്ക് ഔദ്യോഗിക SDKകൾ നൽകുന്നു ([MCP സ്പെസിഫിക്കേഷൻ 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/) അനുസരിച്ച്):
- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk) - Microsoft-നുമായി സഹകരണത്തിൽ പരിപാലിക്കുന്നു
- [Java SDK](https://github.com/modelcontextprotocol/java-sdk) - Spring AI-യുമായി ചേർന്നാണ് പരിപാലനം
- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) - ഔദ്യോഗിക TypeScript നടപ്പാക്കൽ
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk) - ഔദ്യോഗിക Python നടപ്പാക്കൽ (FastMCP)
- [Kotlin SDK](https://github.com/modelcontextprotocol/kotlin-sdk) - ഔദ്യോഗിക Kotlin നടപ്പാക്കൽ
- [Swift SDK](https://github.com/modelcontextprotocol/swift-sdk) - Loopwork AI-യുമായി ചേർന്ന് പരിപാലിക്കുന്നു
- [Rust SDK](https://github.com/modelcontextprotocol/rust-sdk) - ഔദ്യോഗിക Rust നടപ്പാക്കൽ
- [Go SDK](https://github.com/modelcontextprotocol/go-sdk) - ഔദ്യോഗിക Go നടപ്പാക്കൽ

## പ്രധാനപ്പെട്ട സൂത്രവാക്യങ്ങൾ

- ഭാഷാഭേദമുള്ള SDKകൾ ഉപയോഗിച്ച് MCP ഡെവലപ്മെന്റ് പരിസ്ഥിതി സജ്ജീകരണം എളുപ്പമാണ്
- MCP സെർവർ നിർമിക്കുന്നതിൽ ടൂളുകൾ സൃഷ്ടിച്ച് രജിസ്റ്റർ ചെയ്യുകയും സ്പഷ്ടമായ സ്കീമകൾ നൽകുകയും ചെയ്യണം
- MCP ക്ലയന്റുകൾ സെർവർകളുമായി, മോഡലുകളുമായും കണക്റ്റ് ചെയ്ത് കൂടുതൽ ശേഷികൾ പ്രയോജനപ്പെടുത്തുന്നു
- MCP നടപ്പാക്കലുകൾ വിശ്വസനീയമാക്കാൻ പരിശോധനയും ഡീബഗിംഗും അനിവാര്യമാണ്
- വിന്യസനം പ്രാദേശിക വികസന മുതൽ ക്ലൗഡ് അടിസ്ഥാന പരിഹാരങ്ങൾ വരെ വൈവിധ്യമാർന്ന സംവിധാനങ്ങൾ കാണുന്നു
