# MCP സെർവ്വറുകൾ വിന്യസിക്കൽ

താങ്കളുടെ MCP സെർവർ വിന്യസിക്കുന്നത്, അതിന്റെ ഉപകരണങ്ങളും വിഭവങ്ങളും നിങ്ങളുടെ പ്രാദേശിക പരിസരത്തിന് പുറംവശത്ത് മറ്റുള്ളവർക്ക് ഉപയോഗിക്കാൻ അനുവദിക്കുന്നു. സ്കെയിലബിലിറ്റി, വിശ്വാസ്യത, തന്ന 관리 എളുപ്പം എന്നിങ്ങനെയുള്ള നിങ്ങളുടെ ആവശ്യകതകൾ ആശ്രയിച്ച് പല വിന്യസിക്കൽ നയങ്ങളും പരിഗണിക്കാവുന്നതാണ്. താഴെ MCP സെർവർ കമ്പ്യൂട്ടറിൽ, കണ്ടെയ്‌നറുകളിൽ, ക്ലൗഡിൽ വിന്യസിക്കാൻ മാർഗനിർദേശം നിങ്ങൾക്കു ലഭിക്കും.

## അവലോകനം

ഈ പാഠത്തിൽ നിങ്ങൾക്ക് MCP സെർവർ ആപ്പ് എങ്ങനെ വിന്യസിക്കാമെന്ന് പഠിക്കും.

## പഠന ലക്ഷ്യങ്ങൾ

ഈ പാഠത്തിന്റെ അവസാനം, നിങ്ങൾക്ക് കഴിയും:

- വിവിധ വിന്യസിക്കൽ സമീപനങ്ങളെ വിലയിരുത്തുക.
- നിങ്ങളുടെ ആപ്പ് വിന്യസിക്കുക.

## പ്രാദേശിക വികസനവും വിന്യസിക്കൽവും

നിങ്ങളുടെ സെർവർ ഉപയോക്താക്കളുടെ കമ്പ്യൂട്ടറിൽ പ്രവർത്തിക്കുന്നതിന് വേണ്ടിയുള്ളതാണ് എങ്കിൽ താഴെയുള്ള ചുവടുകൾ പിന്തുടരാം:

1. **സെർവർ ഡൗൺലോഡ് ചെയ്യുക**. നിങ്ങൾ സെർവർ എഴുതിയില്ലെങ്കിൽ, ആദ്യം അത് നിങ്ങളുടെ യന്ത്രത്തിലേക്ക് ഡൗൺലോഡ് ചെയ്യുക.  
1. **സെർവർ പ്രക്രിയ ആരംഭിക്കുക**: നിങ്ങളുടെ MCP സെർവർ ആപ്പ് റൺ ചെയ്യുക

SSEയ്ക്കായി (stdio തരം സെർവറിന് വേണ്ടതല്ല)

1. **നെറ്റ്വർക്ക് ക്രമീകരിക്കുക**: സെർവർ പ്രതീക്ഷിക്കുന്ന പോർട്ടിൽ ആക്സസ് ചെയ്യാവുന്നുവെന്ന് ഉറപ്പാക്കുക  
1. **ക്ലയന്റുകൾ ബന്ധിപ്പിക്കുക**: `http://localhost:3000` പോലുള്ള പ്രാദേശിക കണക്ഷൻ URLs ഉപയോഗിക്കുക

## ക്ലൗഡ് വിന്യസിക്കൽ

MCP സെർവർ വിവിധ ക്ലൗഡ് പ്ലാറ്റ്‌ഫോമുകളിൽ വിന്യസിക്കാം:

- **സെർവറെസ് ഫങ്ഷൻസ്**: ലോൺ MCP സെർവറുകളെ സെർവറെസ് ഫങ്ഷൻസ് ആക്കി വിന്യസിക്കുക  
- **കണ്ടെയ്‌നർ സർവീസുകൾ**: Azure Container Apps, AWS ECS, Google Cloud Run പോലുള്ള സർവീസുകൾ ഉപയോഗിക്കുക  
- **കുബർണീറ്റിസ്**: ഉയർന്ന ലഭ്യതക്കായി MCP സെർവർ കുബർണീറ്റിസ് ക്ലസ്റ്ററുകളിൽ വിന്യസിച്ച് മാനേജ് ചെയ്യുക

### ഉദാഹരണം: Azure Container Apps

Azure Container Apps MCP സെർവർ വിന്യസിക്കൽ പിന്തുണയ്ക്കുന്നു. ഇത് ഇപ്പോഴും പുരോഗമന ഘട്ടത്തിലാണ്, വെറുതെ SSE സെർവറുകൾ മാത്രമാണ് ഇത് നിലവിൽ പിന്തുണയ്ക്കുന്നത്.

ഇങ്ങനെ ചെയ്യാം:

1. ഒരു റെപ്പോ ക്ലോൺ ചെയ്യുക:

  ```sh
  git clone https://github.com/anthonychu/azure-container-apps-mcp-sample.git
  ```

1. കാര്യങ്ങൾ പരീക്ഷിക്കാൻ പ്രാദേശികമായി റൺ ചെയ്യുക:

  ```sh
  uv venv
  uv sync

  # ലിനക്സും മാകോഎസും
  export API_KEYS=<AN_API_KEY>
  # വിൻഡോസ്
  set API_KEYS=<AN_API_KEY>

  uv run fastapi dev main.py
  ```

1. പ്രാദേശികമായി പരീക്ഷിക്കാൻ, *.vscode* ഡയറക്ടറിയിൽ *mcp.json* ഫയൽ സൃഷ്ടിച്ച് താഴെയുള്ള ഉള്ളടക്കം ചേർക്കുക:

  ```json
  {
      "inputs": [
          {
              "type": "promptString",
              "id": "weather-api-key",
              "description": "Weather API Key",
              "password": true
          }
      ],
      "servers": {
          "weather-sse": {
              "type": "sse",
              "url": "http://localhost:8000/sse",
              "headers": {
                  "x-api-key": "${input:weather-api-key}"
              }
          }
      }
  }
  ```

  SSE സെർവർ ആരംഭിച്ചതിനു ശേഷം, JSON ഫയലിൽ പ്ലേ ഐക്കണിൽ ക്ലിക്കുചെയ്യുക, സെർവറിലെ ഉപകരണങ്ങൾ GitHub Copilot വഴി സ്ഥിരീകരിക്കുന്നതായി കാണാം, ടൂൾ ഐക്കൺ കാണുക.

1. വിന്യസിക്കാൻ, താഴെയുള്ള കമാൻഡ് റൺ ചെയ്യുക:

  ```sh
  az containerapp up -g <RESOURCE_GROUP_NAME> -n weather-mcp --environment mcp -l westus --env-vars API_KEYS=<AN_API_KEY> --source .
  ```

ഇത് കൊണ്ട്, പ്രാദേശികമായി വിന്യസിക്കുക, Azure-ിൽ ഈ ഘട്ടങ്ങൾ വഴി വിന്യസിക്കുക.
