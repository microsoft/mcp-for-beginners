# MCP Безбедност: Свеобухватна заштита за AI системе

[![MCP Security Best Practices](../../../translated_images/sr/03.175aed6dedae133f9d41e49cefd0f0a9a39c3317e1eaa7ef7182696af7534308.png)](https://youtu.be/88No8pw706o)

_(Кликните на слику изнад да бисте погледали видео о овој лекцији)_

Безбедност је основа дизајна AI система, због чега јој придајемо приоритет као другој секцији. Ово је у складу са Microsoft-овим принципом **Secure by Design** из [Secure Future Initiative](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/).

Протокол контекста модела (MCP) доноси моћне нове могућности AI-погонским апликацијама, истовремено уводећи јединствене безбедносне изазове који превазилазе традиционалне ризике софтвера. MCP системи се суочавају како са устаљеним безбедносним питањима (сигурно програмирање, најмање привилегије, безбедност ланца снабдевања), тако и са новим претњама специфичним за AI, укључујући убризгавање упита, тровање алата, отмицу сесије, нападе збуњеног посредника, рањивости у прослеђивању токена и динамичку модификацију могућности.

Ова лекција истражује најкритичније безбедносне ризике у MCP имплементацијама — обухватајући аутентификацију, ауторизацију, прекомерне дозволе, индиректно убризгавање упита, безбедност сесије, проблеме збуњеног посредника, управљање токенима и рањивости ланца снабдевања. Научићете применљиве контроле и најбоље праксе за ублажавање ових ризика уз коришћење Microsoft решења као што су Prompt Shields, Azure Content Safety и GitHub Advanced Security за јачање ваше MCP имплементације.

## Циљеви учења

До краја ове лекције моћи ћете да:

- **Препознате претње специфичне за MCP**: Уочите јединствене безбедносне ризике у MCP системима укључујући убризгавање упита, тровање алата, прекомерне дозволе, отмицу сесије, проблеме збуњеног посредника, рањивости у прослеђивању токена и ризике ланца снабдевања
- **Примените безбедносне контроле**: Спроведете ефикасне мере ублажавања укључујући робусну аутентификацију, приступ најмањих привилегија, сигурно управљање токенима, контроле безбедности сесије и верификацију ланца снабдевања
- **Искористите Microsoft безбедносна решења**: Разумете и примените Microsoft Prompt Shields, Azure Content Safety и GitHub Advanced Security за заштиту MCP радних оптерећења
- **Валидирате безбедност алата**: Препознате значај валидације метаподатака алата, праћења динамичких промена и одбране од индиректних напада убризгавања упита
- **Интегришете најбоље праксе**: Комбинујете устаљене безбедносне основе (сигурно програмирање, ојачавање сервера, zero trust) са MCP-специфичним контролама за свеобухватну заштиту

# MCP Безбедносна архитектура и контроле

Модерне MCP имплементације захтевају слојевите приступе безбедности који обухватају како традиционалну безбедност софтвера тако и претње специфичне за AI. Брзо еволуирајућа MCP спецификација наставља да унапређује своје безбедносне контроле, омогућавајући бољу интеграцију са безбедносним архитектурама предузећа и устаљеним најбољим праксама.

Истраживање из [Microsoft Digital Defense Report](https://aka.ms/mddr) показује да би **98% пријављених пропуста било спречено уз робусну безбедносну хигијену**. Најефикаснија стратегија заштите комбинује основне безбедносне праксе са MCP-специфичним контролама — доказане базне мере безбедности остају најзначајније у смањењу укупног безбедносног ризика.

## Тренутни безбедносни пејзаж

> **Напомена:** Ове информације одражавају MCP безбедносне стандарде стања на **18. децембар 2025.** MCP протокол се брзо развија, а будуће имплементације могу увести нове обрасце аутентификације и побољшане контроле. Увек се консултујте са актуелном [MCP спецификацијом](https://spec.modelcontextprotocol.io/), [MCP GitHub репозиторијумом](https://github.com/modelcontextprotocol) и [документацијом најбољих безбедносних пракси](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) за најновија упутства.

### Еволуција MCP аутентификације

MCP спецификација је значајно еволуирала у приступу аутентификацији и ауторизацији:

- **Првобитни приступ**: Ране спецификације су захтевале од програмера да имплементирају прилагођене сервере за аутентификацију, при чему су MCP сервери деловали као OAuth 2.0 Authorization Servers који директно управљају аутентификацијом корисника
- **Тренутни стандард (2025-11-25)**: Ажурирана спецификација омогућава MCP серверима да делегирају аутентификацију спољним провајдерима идентитета (као што је Microsoft Entra ID), побољшавајући безбедносни положај и смањујући сложеност имплементације
- **Безбедност транспортног слоја**: Побољшана подршка за сигурне транспортне механизме са одговарајућим обрасцима аутентификације за локалне (STDIO) и удаљене (Streamable HTTP) везе

## Безбедност аутентификације и ауторизације

### Тренутни безбедносни изазови

Модерне MCP имплементације се суочавају са неколико изазова у аутентификацији и ауторизацији:

### Ризици и претње

- **Погрешно конфигурисана логика ауторизације**: Недостаци у имплементацији ауторизације на MCP серверима могу открити осетљиве податке и неправилно применити контроле приступа
- **Компромитовање OAuth токена**: Крађа локалног MCP серверског токена омогућава нападачима да се лажно представљају као сервери и приступају сервисима низводно
- **Ранљивости у прослеђивању токена**: Неправилно руковање токенима ствара заобилазне путеве безбедносних контрола и празнине у одговорности
- **Прекомерне дозволе**: MCP сервери са превише привилегија крше принцип најмањих привилегија и проширују површине напада

#### Прослеђивање токена: Критичан анти-образац

**Прослеђивање токена је изричито забрањено** у тренутној MCP спецификацији ауторизације због озбиљних безбедносних последица:

##### Заобилажење безбедносних контрола
- MCP сервери и низводни API-ји спроводе критичне безбедносне контроле (ограничење брзине, валидација захтева, праћење саобраћаја) које зависе од исправне валидације токена
- Директна употреба токена клијента према API-ју заобилази ове основне заштите, поткопавајући безбедносну архитектуру

##### Изазови у одговорности и ревизији  
- MCP сервери не могу разликовати клијенте који користе токене издате горе, што ремети трагове ревизије
- Логови низводних сервера ресурса приказују погрешне изворе захтева уместо стварних MCP серверских посредника
- Истраживање инцидената и ревизија усклађености постају знатно тежи

##### Ризици извлачења података
- Невалидиране тврдње токена омогућавају злонамерним актерима са украденим токенима да користе MCP сервере као проксије за извлачење података
- Кршења граница поверења дозвољавају неовлашћене обрасце приступа који заобилазе намењене безбедносне контроле

##### Вектори напада више сервиса
- Компромитовани токени прихваћени од стране више сервиса омогућавају латерално кретање кроз повезане системе
- Претпоставке поверења између сервиса могу бити нарушене када се порекло токена не може проверити

### Безбедносне контроле и мере ублажавања

**Критични безбедносни захтеви:**

> **ОБАВЕЗНО**: MCP сервери **НЕ СМЕЈУ** прихватати било које токене који нису изричито издати за MCP сервер

#### Контроле аутентификације и ауторизације

- **Строга ревизија ауторизације**: Спроведите свеобухватне ревизије логике ауторизације MCP сервера како бисте осигурали да само намењени корисници и клијенти могу приступити осетљивим ресурсима
  - **Водич за имплементацију**: [Azure API Management као аутентификациони пролаз за MCP сервере](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)
  - **Интеграција идентитета**: [Коришћење Microsoft Entra ID за аутентификацију MCP сервера](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **Сигурно управљање токенима**: Примените [Microsoft-ове најбоље праксе за валидацију и животни циклус токена](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens)
  - Валидација тврдњи о публици токена да одговарају идентитету MCP сервера
  - Имплементација правилне ротације и политика истека токена
  - Превенција напада репродукције токена и неовлашћене употребе

- **Заштићено складиштење токена**: Сигурно складиштење токена са енкрипцијом у миру и током преноса
  - **Најбоље праксе**: [Упутства за сигурно складиштење и енкрипцију токена](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### Имплементација контроле приступа

- **Принцип најмањих привилегија**: Доделите MCP серверима само минималне дозволе потребне за намењену функционалност
  - Редовне ревизије и ажурирања дозвола како би се спречило ширење привилегија
  - **Microsoft документација**: [Сигуран приступ са најмање привилегија](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **Контрола приступа заснована на улогама (RBAC)**: Имплементирајте фино подешене доделе улога
  - Строга ограничења улога на специфичне ресурсе и акције
  - Избегавајте широке или непотребне дозволе које проширују површине напада

- **Континуирано праћење дозвола**: Спроведите сталну ревизију и праћење приступа
  - Пратите обрасце коришћења дозвола ради откривања аномалија
  - Брзо отклањајте прекомерне или неискоришћене привилегије

## Претње специфичне за AI

### Убризгавање упита и напади манипулације алатима

Модерне MCP имплементације се суочавају са софистицираним AI-специфичним векторима напада које традиционалне безбедносне мере не могу у потпуности адресирати:

#### **Индиректно убризгавање упита (Cross-Domain Prompt Injection)**

**Индиректно убризгавање упита** представља једну од најкритичнијих рањивости у AI системима са омогућеним MCP-ом. Нападачи убацују злонамерна упутства у спољни садржај — документе, веб странице, е-поруке или изворе података — које AI системи касније обрађују као легитимне команде.

**Сценарији напада:**
- **Убризгавање у документима**: Злонамерна упутства сакривена у обрађеним документима која покрећу нежељене AI акције
- **Експлоатација веб садржаја**: Компромитоване веб странице са уграђеним упитима који манипулишу понашањем AI када се скрејпују
- **Напади путем е-поште**: Злонамерни упити у е-порукама који узрокују да AI асистенти цуре информације или извршавају неовлашћене радње
- **Загађење извора података**: Компромитоване базе података или API-ји који служе заражени садржај AI системима

**Утицај у стварном свету**: Ови напади могу довести до извлачења података, кршења приватности, генерисања штетног садржаја и манипулације корисничким интеракцијама. За детаљну анализу погледајте [Prompt Injection у MCP (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/).

![Prompt Injection Attack Diagram](../../../translated_images/sr/prompt-injection.ed9fbfde297ca877c15bc6daa808681cd3c3dc7bf27bbbda342ef1ba5fc4f52d.png)

#### **Напади тровања алата**

**Тровање алата** циља метаподатке који дефинишу MCP алате, експлоатишући начин на који LLM модели тумаче описе алата и параметре за доношење одлука о извршењу.

**Механизми напада:**
- **Манипулација метаподацима**: Нападачи убацују злонамерна упутства у описе алата, дефиниције параметара или примере коришћења
- **Невидљива упутства**: Скривени упити у метаподацима алата које AI модели обрађују, али су невидљиви људским корисницима
- **Динамичка модификација алата („Rug Pulls“) **: Алати одобрени од стране корисника касније се мењају да извршавају злонамерне радње без знања корисника
- **Убризгавање параметара**: Злонамерни садржај уграђен у шеме параметара алата који утичу на понашање модела

**Ризици хостованих сервера**: Удаљени MCP сервери представљају повећане ризике јер дефиниције алата могу бити ажуриране након почетног одобрења корисника, стварајући сценарије у којима претходно безбедни алати постају злонамерни. За свеобухватну анализу погледајте [Tool Poisoning Attacks (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks).

![Tool Injection Attack Diagram](../../../translated_images/sr/tool-injection.3b0b4a6b24de6befe7d3afdeae44138ef005881aebcfc84c6f61369ce31e3640.png)

#### **Додатни AI вектори напада**

- **Cross-Domain Prompt Injection (XPIA)**: Софистицирани напади који користе садржај из више домена за заобилажење безбедносних контрола
- **Динамичка модификација могућности**: Промене могућности алата у реалном времену које измичу почетним безбедносним проценама
- **Тровање контекст прозора**: Напади који манипулишу великим контекст прозорима да сакрију злонамерна упутства
- **Напади конфузије модела**: Експлоатација ограничења модела за стварање непредвидивог или небезбедног понашања

### Утицај AI безбедносних ризика

**Последице великог утицаја:**
- **Извлачење података**: Неовлашћен приступ и крађа осетљивих података предузећа или личних података
- **Кршњење приватности**: Излагање лично идентификационих информација (PII) и поверљивих пословних података  
- **Манипулација системом**: Нежељене измене критичних система и радних токова
- **Крађа акредитива**: Компромитовање аутентификационих токена и сервисних акредитива
- **Латерално кретање**: Коришћење компромитованих AI система као полазних тачака за шире мрежне нападе

### Microsoft AI безбедносна решења

#### **AI Prompt Shields: Напредна заштита од напада убризгавања упита**

Microsoft **AI Prompt Shields** пружају свеобухватну одбрану од директних и индиректних напада убризгавања упита кроз више безбедносних слојева:

##### **Основни механизми заштите:**

1. **Напредна детекција и филтрирање**
   - Алгоритми машинског учења и NLP технике детектују злонамерна упутства у спољном садржају
   - Анализа у реалном времену докумената, веб страница, е-порука и извора података за уграђене претње
   - Контекстуално разумевање легитимних у односу на злонамерне обрасце упита

2. **Технике истакнућа**  
   - Разликује између поузданих системских упутстава и потенцијално компромитованих спољних уноса
   - Методи трансформације текста који побољшавају релевантност модела уз изолацију злонамерног садржаја
   - Помажу AI системима да одрже правилну хијерархију упутстава и игноришу убризгане команде

3. **Системи за раздвајање и означавање података**
   - Јасно дефинисање граница између поузданих системских порука и спољног уноса текста
   - Специјални маркери који истичу границе између поузданих и непоузданих извора података
   - Јасна подела спречава конфузију упутстава и неовлашћено извршавање команди

4. **Континуирана обавештеност о претњама**

## Напредне MCP безбедносне претње

### Ранљивости отмице сесије

**Отмица сесије** представља критичан вектор напада у стањивим MCP имплементацијама где неовлашћене стране добијају и злоупотребљавају легитимне идентификаторе сесије да би се лажно представљале као клијенти и извршавале неовлашћене радње.

#### **Сценарији напада и ризици**

- **Отмица сесије и убризгавање упита**: Нападачи са украденим ID-јевима сесије убацују злонамерне догађаје у сервере који деле стање сесије, потенцијално покрећући штетне акције или приступајући осетљивим подацима
- **Директна имперсонација**: Украдени ID-јеви сесије омогућавају директне позиве MCP сервера који заобилазе аутентификацију, третирајући нападаче као легитимне кориснике
- **Компромитовани настављиви токови**: Нападачи могу прекинути захтеве пре времена, узрокујући да легитимни клијенти наставе са потенцијално злонамерним садржајем

#### **Безбедносне контроле за управљање сесијама**

**Критични захтеви:**
- **Верификација ауторизације**: MCP сервери који имплементирају ауторизацију **МОРАЈУ** да верификују СВЕ долазне захтеве и **НЕ СМЕЈУ** да се ослањају на сесије за аутентификацију  
- **Сигурна генерација сесија**: Користите криптографски сигурне, недетерминистичке ИД-ове сесија генерисане помоћу сигурних генератора случајних бројева  
- **Веза са корисником**: Вежите ИД-ове сесија за корисничке информације користећи формате као што је `<user_id>:<session_id>` да бисте спречили злоупотребу сесија између корисника  
- **Управљање животним циклусом сесије**: Имплементирајте правилно истицање, ротацију и неважење сесија како бисте ограничили прозоре рањивости  
- **Безбедност транспорта**: Обавезан HTTPS за сву комуникацију како би се спречило пресретање ИД-ова сесија  

### Проблем збуњеног заменика

Проблем **збуњеног заменика** јавља се када MCP сервери делују као прокси за аутентификацију између клијената и услуга трећих страна, стварајући могућности за заобилажење ауторизације кроз експлоатацију статичких ИД-ова клијената.

#### **Механизам напада и ризици**

- **Заобилажење пристанка засновано на колачићима**: Претходна аутентификација корисника ствара колачиће пристанка које нападачи злоупотребљавају путем злонамерних захтева за ауторизацију са израђеним URI-јевима за преусмеравање  
- **Крађа кода за ауторизацију**: Постојећи колачићи пристанка могу узроковати да сервери за ауторизацију прескоче екране пристанка и преусмере кодове на крајње тачке под контролом нападача  
- **Неовлашћен приступ API-ју**: Украдени кодови за ауторизацију омогућавају размену токена и имитацију корисника без изричитог одобрења  

#### **Стратегије ублажавања**

**Обавезне контроле:**  
- **Изричити захтеви за пристанак**: MCP прокси сервери који користе статичке ИД-ове клијената **МОРАЈУ** да добију кориснички пристанак за сваког динамички регистрованог клијента  
- **Имплементација безбедности OAuth 2.1**: Пратите актуелне најбоље праксе безбедности OAuth укључујући PKCE (Proof Key for Code Exchange) за све захтеве за ауторизацију  
- **Строга валидација клијената**: Имплементирајте ригорозну валидацију URI-јева за преусмеравање и идентификатора клијената како бисте спречили злоупотребу  

### Ранљивости у прослеђивању токена  

**Прослеђивање токена** представља експлицитни анти-патерн где MCP сервери прихватају токене клијената без одговарајуће валидације и прослеђују их даљим API-јевима, кршећи MCP спецификације ауторизације.

#### **Безбедносне импликације**

- **Заобилажење контроле**: Директна употреба токена клијента према API-ју заобилази критичне контроле ограничења брзине, валидације и надзора  
- **Корупција трага ревизије**: Токени издати горе чине немогућим идентификацију клијента, што онемогућава истрагу инцидената  
- **Извлачење података преко проксија**: Невалидирани токени омогућавају злонамерним актерима да користе сервере као проксије за неовлашћени приступ подацима  
- **Прекид граница поверења**: Услуге ниже у ланцу могу прекршити претпоставке о поверењу када се не може проверити порекло токена  
- **Ширење напада на више услуга**: Компромитовани токени прихваћени у више услуга омогућавају бочно кретање  

#### **Потребне безбедносне контроле**

**Неопходни захтеви:**  
- **Валидација токена**: MCP сервери **НЕ СМЕЈУ** прихватати токене који нису изричито издати за MCP сервер  
- **Верификација публике**: Увек валидајте да захтеви публике токена одговарају идентитету MCP сервера  
- **Правилан животни циклус токена**: Имплементирајте краткотрајне приступне токене са сигурним праксама ротације  

## Безбедност ланца снабдевања за AI системе

Безбедност ланца снабдевања је еволуирала изван традиционалних софтверских зависности и обухвата цео AI екосистем. Савремене MCP имплементације морају ригорозно верификовати и пратити све AI компоненте, јер свака уводи потенцијалне рањивости које могу угрозити интегритет система.

### Проширене компоненте AI ланца снабдевања

**Традиционалне софтверске зависности:**  
- Отворене библиотеке и оквири  
- Контејнер слике и базични системи  
- Развојни алати и процеси изградње  
- Инфраструктурне компоненте и услуге  

**AI-специфични елементи ланца снабдевања:**  
- **Основни модели**: Предтренирани модели од различитих провајдера који захтевају верификацију порекла  
- **Услуге уграђивања**: Спољне услуге векторизације и семантичке претраге  
- **Провајдери контекста**: Извори података, базе знања и складишта докумената  
- **API-ји трећих страна**: Спољне AI услуге, ML процеси и крајње тачке за обраду података  
- **Артефакти модела**: Тежине, конфигурације и фино подешене варијанте модела  
- **Извори података за тренинг**: Скуп података коришћен за тренирање и фино подешавање модела  

### Комплетна стратегија безбедности ланца снабдевања

#### **Верификација компоненти и поверење**  
- **Валидација порекла**: Верификујте порекло, лиценце и интегритет свих AI компоненти пре интеграције  
- **Безбедносна процена**: Спроводите скенирање рањивости и безбедносне прегледе модела, извора података и AI услуга  
- **Анализа репутације**: Процените безбедносну историју и праксе провајдера AI услуга  
- **Верификација усаглашености**: Осигурајте да све компоненте испуњавају организационе безбедносне и регулаторне захтеве  

#### **Сигурни процеси деплоја**  
- **Аутоматизовано CI/CD скенирање**: Интегришите безбедносно скенирање у све аутоматизоване процесе деплоја  
- **Интегритет артефаката**: Имплементирајте криптографску верификацију свих деплојованих артефаката (код, модели, конфигурације)  
- **Фазни деплој**: Користите прогресивне стратегије деплоја са безбедносном валидацијом у свакој фази  
- **Поверени регистри артефаката**: Деплојујте само из верификованих, сигурних регистара и складишта артефаката  

#### **Континуирани надзор и одговор**  
- **Скенирање зависности**: Континуирано праћење рањивости за све софтверске и AI компоненте  
- **Надзор модела**: Континуирана процена понашања модела, одступања у перформансама и безбедносних аномалија  
- **Праћење здравља услуга**: Надгледање спољних AI услуга за доступност, безбедносне инциденте и промене политика  
- **Интеграција обавештајних података о претњама**: Укључивање извора претњи специфичних за AI и ML безбедносне ризике  

#### **Контрола приступа и најмања привилегија**  
- **Дозволе на нивоу компоненти**: Ограничите приступ моделима, подацима и услугама на основу пословне потребе  
- **Управљање сервисним налозима**: Имплементирајте посебне сервисне налоге са минималним потребним дозволама  
- **Сегментација мреже**: Изолујте AI компоненте и ограничите мрежни приступ између услуга  
- **Контроле API гатеваја**: Користите централизоване API гатеваје за контролу и надзор приступа спољним AI услугама  

#### **Одговор на инциденте и опоравак**  
- **Процедуре брзог одговора**: Успостављени процеси за закрпе или замену компромитованих AI компоненти  
- **Ротација акредитива**: Аутоматизовани системи за ротацију тајни, API кључева и сервисних акредитива  
- **Могућности враћања**: Способност брзог враћања на претходне познато добре верзије AI компоненти  
- **Опоравак од прекршаја у ланцу снабдевања**: Специфичне процедуре за реаговање на компромитовање AI услуга горе у ланцу

## Најбоље праксе основне безбедности

MCP имплементације наслеђују и надограђују постојећи безбедносни положај ваше организације. Јачање основних безбедносних пракси значајно побољшава укупну безбедност AI система и MCP деплоја.

### Основе безбедности

#### **Праксе безбедног развоја**  
- **Усклађеност са OWASP-ом**: Заштита од [OWASP Top 10](https://owasp.org/www-project-top-ten/) рањивости веб апликација  
- **AI-специфичне заштите**: Имплементација контрола за [OWASP Top 10 за LLM](https://genai.owasp.org/download/43299/?tmstv=1731900559)  
- **Сигурно управљање тајнама**: Користите посебне складишта за токене, API кључеве и осетљиве конфигурационе податке  
- **Енд-то-енд енкрипција**: Имплементирајте сигурну комуникацију кроз све компоненте апликације и токове података  
- **Валидација уноса**: Ригорозна валидација свих корисничких уноса, API параметара и извора података  

#### **Ојачавање инфраструктуре**  
- **Мултифакторска аутентификација**: Обавезна MFA за све административне и сервисне налоге  
- **Управљање закрпама**: Аутоматизовано и благовремено закрпљивање оперативних система, оквира и зависности  
- **Интеграција провајдера идентитета**: Централизовано управљање идентитетом преко корпоративних провајдера (Microsoft Entra ID, Active Directory)  
- **Сегментација мреже**: Логичка изолација MCP компоненти ради ограничења потенцијалног бочног кретања  
- **Принцип најмањих привилегија**: Минималне потребне дозволе за све системске компоненте и налоге  

#### **Надзор и детекција безбедности**  
- **Свеобухватно логовање**: Детаљно логовање активности AI апликација, укључујући интеракције MCP клијент-сервер  
- **Интеграција са SIEM-ом**: Централизовано управљање безбедносним информацијама и догађајима за детекцију аномалија  
- **Бихејвиорална аналитика**: AI-подржано праћење за откривање необичних образаца у понашању система и корисника  
- **Обавештајне информације о претњама**: Интеграција спољних извора претњи и индикатора компромитовања (IOC)  
- **Одговор на инциденте**: Добро дефинисане процедуре за детекцију, одговор и опоравак од безбедносних инцидената  

#### **Архитектура нултог поверења**  
- **Никада не веруј, увек верификуј**: Континуирана верификација корисника, уређаја и мрежних веза  
- **Микро-сегментација**: Грануларне мрежне контроле које изолују појединачна радна оптерећења и услуге  
- **Безбедност усредсређена на идентитет**: Безбедносне политике засноване на верификованим идентитетима, а не на мрежној локацији  
- **Континуирана процена ризика**: Динамичка процена безбедносног положаја на основу тренутног контекста и понашања  
- **Условни приступ**: Контроле приступа које се прилагођавају на основу фактора ризика, локације и поверења у уређај  

### Обрасци интеграције у предузећу

#### **Интеграција Microsoft безбедносног екосистема**  
- **Microsoft Defender for Cloud**: Свеобухватно управљање безбедносним положајем у облаку  
- **Azure Sentinel**: Облачно родни SIEM и SOAR могућности за заштиту AI радних оптерећења  
- **Microsoft Entra ID**: Управљање идентитетом и приступом у предузећу са политикама условног приступа  
- **Azure Key Vault**: Централизовано управљање тајнама са подршком за хардверски безбедносни модул (HSM)  
- **Microsoft Purview**: Управљање подацима и усаглашеност за AI изворе података и токове рада  

#### **Усаглашеност и управљање**  
- **Регулаторна усаглашеност**: Осигурајте да MCP имплементације испуњавају индустријске захтеве (GDPR, HIPAA, SOC 2)  
- **Класификација података**: Правилна категоризација и руковање осетљивим подацима које обрађују AI системи  
- **Трагови ревизије**: Свеобухватно логовање за регулаторну усаглашеност и форензичку истрагу  
- **Контроле приватности**: Имплементација принципа приватности по дизајну у архитектури AI система  
- **Управљање променама**: Формални процеси за безбедносне прегледе измена AI система  

Ове основне праксе стварају робусну безбедносну основу која побољшава ефикасност MCP-специфичних безбедносних контрола и пружа свеобухватну заштиту AI покретаних апликација.

## Кључне безбедносне поруке

- **Слојевити приступ безбедности**: Комбинујте основне безбедносне праксе (сигуран код, најмање привилегије, верификацију ланца снабдевања, континуирани надзор) са AI-специфичним контролама за свеобухватну заштиту  

- **AI-специфични безбедносни ризици**: MCP системи се суочавају са јединственим ризицима укључујући убризгавање упита, тровање алата, отимање сесија, проблем збуњеног заменика, рањивости у прослеђивању токена и прекомерне дозволе које захтевају специјализоване мере  

- **Изврсност у аутентификацији и ауторизацији**: Имплементирајте робусну аутентификацију користећи спољне провајдере идентитета (Microsoft Entra ID), спроводите правилну валидацију токена и никада не прихватајте токене који нису изричито издати за ваш MCP сервер  

- **Превенција AI напада**: Распоредите Microsoft Prompt Shields и Azure Content Safety за одбрану од индиректних напада убризгавања упита и тровања алата, уз валидацију метаподатака алата и праћење динамичких промена  

- **Безбедност сесија и транспорта**: Користите криптографски сигурне, недетерминистичке ИД-ове сесија везане за корисничке идентитете, имплементирајте правилно управљање животним циклусом сесија и никада не користите сесије за аутентификацију  

- **Најбоље праксе OAuth безбедности**: Спречите нападе збуњеног заменика кроз изричити кориснички пристанак за динамички регистроване клијенте, правилну имплементацију OAuth 2.1 са PKCE и строгу валидацију URI-јева за преусмеравање  

- **Принципи безбедности токена**: Избегавајте анти-патерн прослеђивања токена, валидајте захтеве публике токена, имплементирајте краткотрајне токене са сигурном ротацијом и одржавајте јасне границе поверења  

- **Свеобухватна безбедност ланца снабдевања**: Третирајте све AI екосистемске компоненте (моделе, уграђивања, провајдере контекста, спољне API-је) са истом безбедносном строгошћу као традиционалне софтверске зависности  

- **Континуирана еволуција**: Будите у току са брзо еволуирајућим MCP спецификацијама, доприноси безбедносним стандардима заједнице и одржавајте адаптивне безбедносне положаје како протокол сазрева  

- **Интеграција Microsoft безбедности**: Искористите свеобухватни Microsoft безбедносни екосистем (Prompt Shields, Azure Content Safety, GitHub Advanced Security, Entra ID) за побољшану заштиту MCP деплоја
