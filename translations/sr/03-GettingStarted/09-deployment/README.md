# Деплоyмент MCP сервера

Деплоyмент вашег MCP сервера омогућава другима да приступе његовим алатима и ресурсима изван вашег локалног окружења. Постоје различите стратегије деплоyмента које треба узети у обзир, у зависности од ваших захтева за скалабилношћу, поузданошћу и лакоћом управљања. Испод ћете пронаћи упутства за деплоyмент MCP сервера локално, у контејнерима и у облак.

## Преглед

Ова лекција покрива како да деплоyујете вашу MCP Server апликацију.

## Циљеви учења

До краја ове лекције, моћи ћете да:

- Процените различите приступе деплоyменту.
- Деплоyујете вашу апликацију.

## Локални развој и деплоyмент

Ако је ваш сервер намењен да се користи на корисничким машинама, можете пратити следеће кораке:

1. **Преузмите сервер**. Уколико нисте ви написали сервер, прво га преузмите на своју машину.  
1. **Покрените серверски процес**: Покрените вашу MCP сервер апликацију.

За SSE (није потребно за stdio тип сервера)

1. **Конфигуришите мрежу**: Осигурајте да је сервер доступан на жељеном порту.  
1. **Повежите клијенте**: Користите локалне URL адресе као што је `http://localhost:3000`

## Деплоyмент у облак

MCP сервери могу бити деплоyовани на различите облачне платформе:

- **Безсерверске функције**: Деплоyујте лагане MCP сервере као безсерверске функције  
- **Сервиси за контејнере**: Користите сервисе као што су Azure Container Apps, AWS ECS, или Google Cloud Run  
- **Kubernetes**: Деплоyујте и управљајте MCP серверима у Kubernetes кластерима за високу доступност

### Пример: Azure Container Apps

Azure Container Apps подржава деплоyмент MCP сервера. Још увек је у развоју и тренутно подржава SSE сервере.

Ево како то можете урадити:

1. Клонирајте репозиторијум:

  ```sh
  git clone https://github.com/anthonychu/azure-container-apps-mcp-sample.git
  ```

1. Покрените га локално да бисте тестирали:

  ```sh
  uv venv
  uv sync

  # линукс/macOS
  export API_KEYS=<AN_API_KEY>
  # виндовс
  set API_KEYS=<AN_API_KEY>

  uv run fastapi dev main.py
  ```

1. Да бисте га пробали локално, направите *mcp.json* фајл у *.vscode* директоријуму и додајте следећи садржај:

  ```json
  {
      "inputs": [
          {
              "type": "promptString",
              "id": "weather-api-key",
              "description": "Weather API Key",
              "password": true
          }
      ],
      "servers": {
          "weather-sse": {
              "type": "sse",
              "url": "http://localhost:8000/sse",
              "headers": {
                  "x-api-key": "${input:weather-api-key}"
              }
          }
      }
  }
  ```

  Када SSE сервер буде покренут, моћи ћете да кликнете на иконицу за пуштање у JSON фајлу, сада би требало да се алати на серверу препознају од стране GitHub Copilot-а, видите икону алата.

1. Да бисте деплоyовали, покрените следећу команду:

  ```sh
  az containerapp up -g <RESOURCE_GROUP_NAME> -n weather-mcp --environment mcp -l westus --env-vars API_KEYS=<AN_API_KEY> --source .
  ```

Ето га, деплоyујте га локално или на Azure пратећи ове кораке.
