## Тестирање и отклањање грешака

Пре него што почнете са тестирањем вашег MCP сервера, важно је разумети доступне алате и најбоље праксе за отклањање грешака. Ефикасно тестирање осигурава да се ваш сервер понаша онако како се очекује и помаже вам да брзо идентификујете и решите проблеме. Следећи одељак описује препоручене приступе за валидацију ваше MCP имплементације.

## Преглед

Ова лекција покрива како изабрати прави приступ тестирању и најефикаснији алат за тестирање.

## Циљеви учења

До краја ове лекције моћи ћете да:

- Опишете различите приступе за тестирање.
- Користите различите алате за ефикасно тестирање вашег кода.

## Тестирање MCP сервера

MCP пружа алате који вам помажу да тестирате и отклањате грешке на вашим серверима:

- **MCP Инспектор**: алат са командне линије који се може покретати као CLI алат и као визуелни алат.
- **Ручно тестирање**: Можете користити алат попут curl за покретање веб захтева, али било који алат способан за HTTP ће одговарати.
- **Јединично тестирање**: Могуће је користити ваш омиљени тестирачки оквир за тестирање функција како сервера тако и клијента.

### Коришћење MCP инспектора

Описали смо коришћење овог алата у претходним лекцијама, али хајде да га укратко представимо. То је алат изграђен у Node.js и можете га користити позивањем извршног програма `npx` који ће привремено преузети и инсталирати сам алат и очистити се након што се заврши извршавање вашег захтева.

[MCP инспектор](https://github.com/modelcontextprotocol/inspector) вам помаже да:

- **Откријете могућности сервера**: Аутоматски детектује расположиве ресурсе, алате и упите
- **Тестирате извршење алата**: Искушајте различите параметре и гледајте одговоре у реалном времену
- **Погледате метаподатке сервера**: Испитујте информације о серверу, шеме и конфигурације

Типичан покретачки пример алата изгледа овако:

```bash
npx @modelcontextprotocol/inspector node build/index.js
```

Горе наведена команда покреће MCP и његов визуелни интерфејс и отвара локални веб интерфејс у вашем прегледачу. Можете очекивати да видите контролну таблу која приказује ваше регистроване MCP сервере, њихове расположиве алате, ресурсе и упите. Интерфејс вам омогућава интерактивно тестирање извршења алата, преглед метаподатака сервера и посматрање одговора у реалном времену, што олакшава ваљидирање и отклањање грешака у вашим MCP сервер имплементацијама.

Ево како то може изгледати: ![Inspector](../../../../translated_images/sr/connect.141db0b2bd05f096.webp) 

Такође можете покренути овај алат у CLI режиму у којем случају додајете опцију `--cli`. Ево примера покретања алата у "CLI" режиму који приказује све алате на серверу:

```sh
npx @modelcontextprotocol/inspector --cli node build/index.js --method tools/list
```

### Ручно тестирање

Поред покретања инспектора за тестирање могућности сервера, сличан приступ је коришћење клијента способног за коришћење HTTP као што је на пример curl.

Помоћу curl можете директно тестирати MCP сервере коришћењем HTTP захтева:

```bash
# Пример: Метаподаци тест сервера
curl http://localhost:3000/v1/metadata

# Пример: Покрени алатку
curl -X POST http://localhost:3000/v1/tools/execute \
  -H "Content-Type: application/json" \
  -d '{"name": "calculator", "parameters": {"expression": "2+2"}}'
```

Као што можете видети из горњег примера коришћења curl, користите POST захтев за позивање алата коришћењем података са именом алата и његовим параметрима. Користите приступ који вам највише одговара. CLI алати обично су бржи за коришћење и погодни за скриптовање што може бити корисно у CI/CD окружењу.

### Јединично тестирање

Направите јединичне тестове за ваше алате и ресурсе како бисте осигурали да раде како се очекује. Ево примера кода за тестирање.

```python
import pytest

from mcp.server.fastmcp import FastMCP
from mcp.shared.memory import (
    create_connected_server_and_client_session as create_session,
)

# Означи цео модул за асинхроне тестове
pytestmark = pytest.mark.anyio


async def test_list_tools_cursor_parameter():
    """Test that the cursor parameter is accepted for list_tools.

    Note: FastMCP doesn't currently implement pagination, so this test
    only verifies that the cursor parameter is accepted by the client.
    """

 server = FastMCP("test")

    # Направи неколико алата за тестирање
    @server.tool(name="test_tool_1")
    async def test_tool_1() -> str:
        """First test tool"""
        return "Result 1"

    @server.tool(name="test_tool_2")
    async def test_tool_2() -> str:
        """Second test tool"""
        return "Result 2"

    async with create_session(server._mcp_server) as client_session:
        # Тестирај без параметра курсора (прескочено)
        result1 = await client_session.list_tools()
        assert len(result1.tools) == 2

        # Тестирај са курсором=None
        result2 = await client_session.list_tools(cursor=None)
        assert len(result2.tools) == 2

        # Тестирај са курсором као стринг
        result3 = await client_session.list_tools(cursor="some_cursor_value")
        assert len(result3.tools) == 2

        # Тестирај са празним стринг курсором
        result4 = await client_session.list_tools(cursor="")
        assert len(result4.tools) == 2
    
```

Претходни код ради следеће:

- Користи pytest оквир који вам омогућава да креирате тестове као функције и користите assert изјаве.
- Креира MCP сервер са два различита алата.
- Користи `assert` изјаву да провери да су одређени услови испуњени.

Погледајте [целу датотеку овде](https://github.com/modelcontextprotocol/python-sdk/blob/main/tests/client/test_list_methods_cursor.py)

Важно је да, с обзиром на ову датотеку, можете тестирате свој сервер да бисте били сигурни да се могућности креирају како треба.

Сви главни SDK имају сличне тест секције па их можете прилагодити вашем изабраном окружењу.
