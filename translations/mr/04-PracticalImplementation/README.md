# व्यावहारिक अंमलबजावणी

[![खऱ्या साधनांशी आणि कार्यप्रवाहांसह MCP अॅप्स कसे तयार करावे, चाचणी घ्यावी आणि तैनात करावी](../../../translated_images/mr/05.64bea204e25ca891.webp)](https://youtu.be/vCN9-mKBDfQ)

_(या धड्याचा व्हिडिओ पाहण्यासाठी वर दिलेल्या प्रतिमेवर क्लिक करा)_

व्यावहारिक अंमलबजावणी म्हणजे Model Context Protocol (MCP) ची ताकद प्रत्यक्षात कशी दिसून येते. MCP मागील सिद्धांत आणि आर्किटेक्चर समजून घेणे महत्त्वाचे असले तरी, खरी किंमत ती संकल्पना वापरून वास्तविक समस्या सोडविण्यासाठी सोल्यूशन्स तयार, चाचणी आणि तैनात करताना दिसून येते. हा अध्याय संकल्पनात्मक ज्ञान आणि प्रत्यक्ष विकास यांच्यामधील अंतर भरण्यास मदत करतो, तुम्हाला MCP-आधारित अॅप्लिकेशन्स जीवनात आणण्याच्या प्रक्रियेत मार्गदर्शन करतो.

तुम्ही बुद्धिमत्ता सहाय्यक विकसित करत असाल, व्यावसायिक कार्यप्रवाहांमध्ये AI एकात्मिक करत असाल किंवा डेटा प्रक्रिया करण्यासाठी सानुकूल साधने तयार करत असाल, MCP एक लवचिक पाया पुरवतो. त्याची भाषा-स्वतंत्र रचना आणि लोकप्रिय प्रोग्रामिंग भाषांसाठी अधिकृत SDKs विविध प्रकारच्या विकासकांसाठी प्रवेशयोग्य बनवतात. या SDKs चा उपयोग करून तुम्ही जलद साचे तयार करू शकता, पुनरावृत्ती करू शकता आणि वेगवेगळ्या प्लॅटफॉर्म्स आणि वातावरणांमध्ये तुमचे सोल्यूशन्स स्केल करू शकता.

खालील विभागांमध्ये तुम्हाला व्यावहारिक उदाहरणे, नमुना कोड आणि तैनात करण्याच्या धोरणांबाबत माहिती मिळेल, ज्यातून C#, Java with Spring, TypeScript, JavaScript, आणि Python मधील MCP चा कसा अंमलबजावणी करायचा हे शिकता येईल. तुम्हाला तुमचे MCP सर्व्हर डिबग कसे करायचे, चाचणी कशी घ्यायची, APIs व्यवस्थापित कसे करायचे आणि Azure वापरून क्लाउडमध्ये सोल्यूशन्स कसे तैनात करायचे हेही शिकवले जाईल. हे व्यावहारिक साधने तुमचे शिक्षण वेगाने वाढवण्यासाठी आणि आत्मविश्वासाने मजबूत, उत्पादनासाठी तयार MCP अॅप्लिकेशन्स तयार करण्यासाठी डिझाइन केलेले आहेत.

## आढावा

हा धडा विविध प्रोग्रामिंग भाषांमधील MCP अंमलबजावणीतल्या व्यावहारिक पैलूंवर लक्ष केंद्रित करतो. आपण C#, Java with Spring, TypeScript, JavaScript, आणि Python मधील MCP SDKs चा उपयोग करून मजबूत अॅप्लिकेशन्स तयार करणे, MCP सर्व्हर डिबग आणि चाचणी घेणे, आणि पुनर्वापरयोग्य संसाधने, प्रॉम्प्ट्स आणि साधने तयार करणे शिकू.

## शिकण्याचे उद्दिष्टे

या धड्याच्या शेवटी, तुम्ही खालील गोष्टी करू शकाल:

- विविध प्रोग्रामिंग भाषांमध्ये अधिकृत SDKs वापरून MCP सोल्यूशन्स अंमलात आणा
- MCP सर्व्हर्स धडकशी व चाचणी घ्या
- सर्व्हर फिचर्स तयार करा आणि वापरा (संसाधने, प्रॉम्प्ट्स, आणि साधने)
- गुंतागुंतीच्या कार्यांसाठी प्रभावी MCP कार्यप्रवाह डिझाइन करा
- कामगिरी आणि विश्वसनीयतेसाठी MCP अंमलबजावणी ऑप्टिमाइझ करा

## मुख्य सर्व्हर फिचर्स

MCP सर्व्हर्स खालील फिचर्सचे कोणतेही संयोजन अंमलात आणू शकतात:

### संसाधने

संसाधने वापरकर्त्यासाठी किंवा AI मॉडेलसाठी संदर्भ आणि डेटा पुरवतात:

- दस्तऐवज संचय
- ज्ञान आधार
- संरचित डेटा स्रोत
- फाईल सिस्टीम

### प्रॉम्प्ट्स

प्रॉम्प्ट्स वापरकर्त्यांसाठी साच्यांतर्गत संदेश आणि कार्यप्रवाह असतात:

- पूर्वनिर्धारित संभाषण साचे
- मार्गदर्शित संवाद पद्धती
- विशेष संवाद संरचना

### साधने

साधने AI मॉडेलद्वारे चालविण्यासाठी फंक्शन असतात:

- डेटा प्रक्रिया उपयुक्तता
- बाह्य API एकत्रीकरण
- गणनात्मक क्षमता
- शोध कार्यप्रणाली

## नमुना अंमलबजावणी: C# अंमलबजावणी

अधिकृत C# SDK रीपॉजिटरीमध्ये MCP च्या विविध पैलूंना दाखवणाऱ्या अनेक नमुना अंमलबजावण्या आहेत:

- **मूलभूत MCP क्लायंट**: MCP क्लायंट तयार करण्याचे आणि साधने कॉल करण्याचे सोपे उदाहरण
- **मूलभूत MCP सर्व्हर**: बेसिक साधन नोंदणीसह अत्यल्प सर्व्हर अंमलबजावणी
- **प्रगत MCP सर्व्हर**: साधन नोंदणी, प्रमाणीकरण, आणि त्रुटी हाताळणीसह पूर्ण वैशिष्ट्यांसह सर्व्हर
- **ASP.NET समाकलन**: ASP.NET Core सह समाकलन दाखवणारे उदाहरणे
- **साधन अंमलबजावणी पॅटर्न्स**: वेगवेगळ्या जटिलतेची साधने अंमलात आणण्यासाठी विविध नमुने

MCP C# SDK अजून पूर्वावलोकनात आहे आणि API मध्ये बदल होऊ शकतात. आम्ही SDK विकसित होत असल्याने हा ब्लॉग अद्ययावत करत राहू.

## Azure वर रिमोट MCP सर्व्हर तैनात करा

आता बघू की आपण आधी नमूद केलेला नमुना तैनात करू शकतो का:

1. रेपो क्लोन करा

    ```bash
    git clone https://github.com/Azure-Samples/remote-mcp-apim-functions-python.git
    cd remote-mcp-apim-functions-python
    ```

2. `Microsoft.App` संसाधन प्रदाता नोंदणी करा.

   - जर तुम्ही Azure CLI वापरत असाल, तर `az provider register --namespace Microsoft.App --wait` चालवा.
   - जर तुम्ही Azure PowerShell वापरत असाल, तर `Register-AzResourceProvider -ProviderNamespace Microsoft.App` चालवा. नंतर काही वेळाने `(Get-AzResourceProvider -ProviderNamespace Microsoft.App).RegistrationState` वापरून नोंदणी पूर्ण झाली आहे का ते तपासा.

3. हा [azd](https://aka.ms/azd) कमांड चालवा ज्यामुळे API व्यवस्थापन सेवा, function app (कोडसह) आणि आवश्यकतेनुसार इतर Azure संसाधने पुरवली जातील.

    ```shell
    azd up
    ```

    हा आज्ञा सर्व क्लाउड संसाधने Azure वर तैनात करेल.

### MCP Inspector सह तुमचा सर्व्हर चाचणी

1. एका **नवीन टर्मिनल विंडो** मध्ये MCP Inspector इन्स्टॉल आणि चालवा

    ```shell
    npx @modelcontextprotocol/inspector
    ```

    तुम्हाला खालीलप्रमाणे इंटरफेस दिसायला हवा:

    ![Connect to Node inspector](../../../translated_images/mr/connect.141db0b2bd05f096.webp)

2. MCP Inspector वेब ऍप URL वापरून (उदा. [http://127.0.0.1:6274/#resources](http://127.0.0.1:6274/#resources)) CTRL+क्लिक करा
3. ट्रान्सपोर्ट प्रकार `SSE` सेट करा
4. `azd up` नंतर दाखवलेला API Management SSE एंडपॉइंट URL सेट करा आणि **Connect** करा:

    ```shell
    https://<apim-servicename-from-azd-output>.azure-api.net/mcp/sse
    ```

5. **List Tools** वर क्लिक करा. साधनावर क्लिक करा आणि **Run Tool** करा.

जर सर्व पावले यशस्वी झाली असतील, तर तुम्ही आता MCP सर्व्हरशी जोडलात आणि साधन यशस्वीरित्या कॉल करू शकलात.

## Azure साठी MCP सर्व्हर्स

[Remote-mcp-functions](https://github.com/Azure-Samples/remote-mcp-functions-dotnet): या रेंजमध्ये Azure Functions वापरून Python, C# .NET किंवा Node/TypeScript मध्ये कस्टम रिमोट MCP सर्व्हर तयार करण्यासाठी जलद प्रारंभ टेम्प्लेट आहे.

नमुने पूर्ण सोल्यूशन पुरवतात ज्याद्वारे विकासक करू शकतात:

- स्थानिकपणे तयार आणि चालवा: स्थानिक संगणकावर MCP सर्व्हर विकसित आणि डिबग करा
- Azure वर तैनात करा: सुलभपणे क्लाउडमध्ये `azd up` आदेशासह तैनात करा
- क्लायंट्सकडून कनेक्ट करा: विविध क्लायंट्सद्वारे MCP सर्व्हरशी कनेक्ट व्हा, जसे VS Code चा Copilot एजंट मोड आणि MCP Inspector साधन

### मुख्य वैशिष्ट्ये

- डिझाइननुसार सुरक्षा: MCP सर्व्हर कीज आणि HTTPS वापरून सुरक्षित केला जातो
- प्रमाणीकरण पर्याय: अंगभूत प्रमाणीकरण आणि/किंवा API Management वापरून OAuth समर्थन
- नेटवर्क आयसोलेशन: Azure Virtual Networks (VNET) वापरून नेटवर्क पृथक्करण संसाधित करण्याची सुविधा
- सर्व्हरलेस आर्किटेक्चर: Azure Functions वापरून प्रमाण स्केलेबल, इव्हेंट-चालित कार्य
- स्थानिक विकास: सर्वांगीण स्थानिक विकास आणि डिबगिंग समर्थन
- सोपी तैनाती: Azure साठी सुलभ तैनाती प्रक्रिया

या रिपॉझिटरीमध्ये सर्व आवश्यक कॉन्फिगरेशन फाइल्स, स्त्रोत कोड आणि संरचनात्मक तपशील आहेत जे तुम्हाला उत्पादनासाठी तयार MCP सर्व्हर अंमलबजावणी लवकर सुरू करण्यास मदत करतात.

- [Azure Remote MCP Functions Python](https://github.com/Azure-Samples/remote-mcp-functions-python) - Python सह Azure Functions वापरून MCP चे नमुना अंमलबजावणी

- [Azure Remote MCP Functions .NET](https://github.com/Azure-Samples/remote-mcp-functions-dotnet) - C# .NET सह Azure Functions वापरून MCP चे नमुना अंमलबजावणी

- [Azure Remote MCP Functions Node/Typescript](https://github.com/Azure-Samples/remote-mcp-functions-typescript) - Node/TypeScript सह Azure Functions वापरून MCP चे नमुना अंमलबजावणी.

## मुख्य मुद्दे

- MCP SDKs भाषानुसार विशिष्ट साधने पुरवतात ज्यामुळे मजबूत MCP सोल्यूशन्स तयार करता येतात
- डिबगिंग आणि चाचणी प्रक्रिया विश्वासार्ह MCP अॅप्ससाठी अत्यंत महत्त्वाची आहे
- पुनर्वापरयोग्य प्रॉम्प्ट साचे सतत AI संवाद देण्यास मदत करतात
- चांगल्या प्रकारे डिझाइन केलेले कार्यप्रवाह अनेक उपकरणांचा वापर करून गुंतागुंतीच्या कार्यांचे आयोजन करू शकतात
- MCP सोल्यूशन्स तयार करताना सुरक्षा, कार्यक्षमता, आणि त्रुटी हाताळणी यांचा विचार करणे आवश्यक आहे

## सराव

तुमच्या क्षेत्रातील खऱ्या समस्येवर उपाय करणारा व्यावहारिक MCP कार्यप्रवाह डिझाइन करा:

1. या समस्येचे समाधान करण्यासाठी 3-4 उपयुक्त साधने ओळखा
2. ह्या साधनांच्या परस्परसंबंध दाखवणारा कार्यप्रवाह आलेख तयार करा
3. तुमच्या पसंतीच्या भाषेत एक मूलभूत साधन अंमलात आणा
4. मॉडेलला तुमचे साधन प्रभावीपणे वापरण्यास मदत करणारा प्रॉम्प्ट साचा तयार करा
