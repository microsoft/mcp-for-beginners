# MCP सुरक्षा: AI प्रणालींसाठी सर्वसमावेशक संरक्षण

[![MCP Security Best Practices](../../../translated_images/mr/03.175aed6dedae133f9d41e49cefd0f0a9a39c3317e1eaa7ef7182696af7534308.png)](https://youtu.be/88No8pw706o)

_(या धड्याचा व्हिडिओ पाहण्यासाठी वरील प्रतिमा क्लिक करा)_

सुरक्षा ही AI प्रणाली डिझाइनसाठी मूलभूत आहे, म्हणूनच आम्ही ती आमच्या दुसऱ्या विभागात प्राधान्य देतो. हे Microsoft च्या [Secure Future Initiative](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) मधील **Secure by Design** तत्त्वाशी सुसंगत आहे.

मॉडेल कॉन्टेक्स्ट प्रोटोकॉल (MCP) AI-चालित अनुप्रयोगांसाठी शक्तिशाली नवीन क्षमता आणतो, तर पारंपरिक सॉफ्टवेअर धोके ओलांडणाऱ्या अनोख्या सुरक्षा आव्हानांना देखील जन्म देतो. MCP प्रणालींना स्थापित सुरक्षा चिंता (सुरक्षित कोडिंग, किमान विशेषाधिकार, पुरवठा साखळी सुरक्षा) तसेच नवीन AI-विशिष्ट धोके जसे की प्रॉम्प्ट इंजेक्शन, टूल विषबाधा, सत्र हायजॅकिंग, कन्फ्यूज्ड डेप्युटी हल्ले, टोकन पासथ्रू असुरक्षितता, आणि डायनॅमिक क्षमता बदल यांचा सामना करावा लागतो.

हा धडा MCP अंमलबजावणीतल्या सर्वात महत्त्वाच्या सुरक्षा धोके तपासतो—प्रमाणीकरण, प्राधिकरण, जास्त परवानग्या, अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन, सत्र सुरक्षा, कन्फ्यूज्ड डेप्युटी समस्या, टोकन व्यवस्थापन, आणि पुरवठा साखळी असुरक्षितता यांचा समावेश आहे. तुम्हाला या धोके कमी करण्यासाठी कृतीक्षम नियंत्रण आणि सर्वोत्तम पद्धती शिकविल्या जातील, तसेच Microsoft च्या Prompt Shields, Azure Content Safety, आणि GitHub Advanced Security सारख्या उपायांचा वापर करून तुमच्या MCP तैनातीला मजबूत करण्याचे मार्गदर्शन मिळेल.

## शिकण्याचे उद्दिष्टे

या धड्याच्या शेवटी, तुम्ही सक्षम असाल:

- **MCP-विशिष्ट धोके ओळखणे**: MCP प्रणालींमधील अनोख्या सुरक्षा धोके ओळखणे जसे की प्रॉम्प्ट इंजेक्शन, टूल विषबाधा, जास्त परवानग्या, सत्र हायजॅकिंग, कन्फ्यूज्ड डेप्युटी समस्या, टोकन पासथ्रू असुरक्षितता, आणि पुरवठा साखळी धोके
- **सुरक्षा नियंत्रण लागू करणे**: मजबूत प्रमाणीकरण, किमान विशेषाधिकार प्रवेश, सुरक्षित टोकन व्यवस्थापन, सत्र सुरक्षा नियंत्रण, आणि पुरवठा साखळी पडताळणी यांसह प्रभावी प्रतिबंधात्मक उपाय राबविणे
- **Microsoft सुरक्षा उपायांचा वापर करणे**: MCP कार्यभार संरक्षणासाठी Microsoft Prompt Shields, Azure Content Safety, आणि GitHub Advanced Security समजून घेणे आणि तैनात करणे
- **टूल सुरक्षा सत्यापित करणे**: टूल मेटाडेटा सत्यापनाचे महत्त्व, डायनॅमिक बदलांसाठी देखरेख, आणि अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन हल्ल्यांपासून संरक्षण ओळखणे
- **सर्वोत्तम पद्धती एकत्र करणे**: स्थापित सुरक्षा मूलतत्त्वे (सुरक्षित कोडिंग, सर्व्हर हार्डनिंग, झिरो ट्रस्ट) MCP-विशिष्ट नियंत्रणांसह एकत्र करून सर्वसमावेशक संरक्षण मिळविणे

# MCP सुरक्षा आर्किटेक्चर आणि नियंत्रण

आधुनिक MCP अंमलबजावण्या पारंपरिक सॉफ्टवेअर सुरक्षा आणि AI-विशिष्ट धोके दोन्ही हाताळणाऱ्या स्तरित सुरक्षा दृष्टिकोनांची गरज असते. जलद विकसित होणारा MCP तपशील त्याच्या सुरक्षा नियंत्रणांना अधिक परिपक्व करत आहे, ज्यामुळे एंटरप्राइझ सुरक्षा आर्किटेक्चर आणि स्थापित सर्वोत्तम पद्धतींसह चांगले एकत्रीकरण शक्य होते.

[Microsoft Digital Defense Report](https://aka.ms/mddr) मधील संशोधन दर्शविते की **98% नोंदवलेल्या उल्लंघनांना मजबूत सुरक्षा स्वच्छतेने प्रतिबंध करता येऊ शकतो**. सर्वात प्रभावी संरक्षण धोरण म्हणजे मूलभूत सुरक्षा पद्धती MCP-विशिष्ट नियंत्रणांसह एकत्र करणे—प्रमाणित सुरक्षा उपाय हे एकूण सुरक्षा धोका कमी करण्यासाठी सर्वाधिक प्रभावी ठरतात.

## सध्याचा सुरक्षा परिदृश्य

> **टीप:** ही माहिती **18 डिसेंबर 2025** पर्यंतच्या MCP सुरक्षा मानकांचे प्रतिबिंब आहे. MCP प्रोटोकॉल जलद विकसित होत आहे, आणि भविष्यातील अंमलबजावण्या नवीन प्रमाणीकरण नमुने आणि सुधारित नियंत्रण आणू शकतात. सद्यस्थितीत मार्गदर्शनासाठी सदैव [MCP Specification](https://spec.modelcontextprotocol.io/), [MCP GitHub repository](https://github.com/modelcontextprotocol), आणि [सुरक्षा सर्वोत्तम पद्धती दस्तऐवज](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) पहा.

### MCP प्रमाणीकरणाचा विकास

MCP तपशील प्रमाणीकरण आणि प्राधिकरणाच्या दृष्टिकोनातून लक्षणीय बदल झाला आहे:

- **मूळ दृष्टिकोन**: सुरुवातीच्या तपशीलांमध्ये विकसकांनी कस्टम प्रमाणीकरण सर्व्हर तयार करणे आवश्यक होते, ज्यात MCP सर्व्हर OAuth 2.0 प्राधिकरण सर्व्हर म्हणून वापरले जात होते जे वापरकर्त्यांचे प्रमाणीकरण थेट व्यवस्थापित करतात
- **सध्याचा मानक (2025-11-25)**: अद्ययावत तपशील MCP सर्व्हरना बाह्य ओळख प्रदात्यांना (जसे Microsoft Entra ID) प्रमाणीकरण सोपवण्याची परवानगी देतो, ज्यामुळे सुरक्षा स्थिती सुधारते आणि अंमलबजावणीची गुंतागुंत कमी होते
- **ट्रान्सपोर्ट लेयर सुरक्षा**: स्थानिक (STDIO) आणि दूरस्थ (Streamable HTTP) कनेक्शनसाठी योग्य प्रमाणीकरण नमुन्यांसह सुरक्षित ट्रान्सपोर्ट यंत्रणांसाठी सुधारित समर्थन

## प्रमाणीकरण आणि प्राधिकरण सुरक्षा

### सध्याच्या सुरक्षा आव्हाने

आधुनिक MCP अंमलबजावण्या अनेक प्रमाणीकरण आणि प्राधिकरण आव्हानांना सामोरे जात आहेत:

### धोके आणि हल्ल्याचे मार्ग

- **चुकीचे प्राधिकरण लॉजिक**: MCP सर्व्हरमधील दोषपूर्ण प्राधिकरण अंमलबजावणी संवेदनशील डेटा उघड करू शकते आणि चुकीच्या प्रकारे प्रवेश नियंत्रण लागू करू शकते
- **OAuth टोकन चोरी**: स्थानिक MCP सर्व्हर टोकन चोरीमुळे हल्लेखोरांना सर्व्हरची नक्कल करण्याची आणि डाउनस्ट्रीम सेवा वापरण्याची संधी मिळते
- **टोकन पासथ्रू असुरक्षितता**: चुकीच्या टोकन हाताळणीमुळे सुरक्षा नियंत्रण बायपास आणि जबाबदारीतील अंतर निर्माण होतात
- **जास्त परवानग्या**: जास्त विशेषाधिकार असलेले MCP सर्व्हर किमान विशेषाधिकार तत्त्वांचे उल्लंघन करतात आणि हल्ल्याचा पृष्ठभाग वाढवतात

#### टोकन पासथ्रू: एक गंभीर विरोधी नमुना

सध्याच्या MCP प्राधिकरण तपशीलात **टोकन पासथ्रू स्पष्टपणे निषिद्ध आहे** कारण त्याचे गंभीर सुरक्षा परिणाम होतात:

##### सुरक्षा नियंत्रण फसवणे
- MCP सर्व्हर आणि डाउनस्ट्रीम API महत्त्वाचे सुरक्षा नियंत्रण (दर मर्यादा, विनंती पडताळणी, ट्रॅफिक मॉनिटरिंग) अंमलात आणतात जे योग्य टोकन पडताळणीवर अवलंबून असतात
- थेट क्लायंट-टू-API टोकन वापर या आवश्यक संरक्षणांना बायपास करते, ज्यामुळे सुरक्षा आर्किटेक्चर कमजोर होते

##### जबाबदारी आणि लेखापरीक्षण आव्हाने  
- MCP सर्व्हर अपस्ट्रीम-इश्यूड टोकन वापरणाऱ्या क्लायंट्समध्ये फरक करू शकत नाहीत, ज्यामुळे लेखापरीक्षण मार्ग तुटतात
- डाउनस्ट्रीम रिसोर्स सर्व्हर लॉग्स खोट्या विनंती मूळ दाखवतात, खऱ्या MCP सर्व्हर मध्यस्थाऐवजी
- घटनेची चौकशी आणि अनुपालन लेखापरीक्षण खूप कठीण होते

##### डेटा चोरी धोके
- अवैध टोकन दाव्यांमुळे चोरी केलेल्या टोकनसह हल्लेखोर MCP सर्व्हरचा वापर डेटा चोरीसाठी प्रॉक्सी म्हणून करू शकतात
- विश्वास मर्यादा उल्लंघनामुळे अनधिकृत प्रवेश नमुने तयार होतात जे नियोजित सुरक्षा नियंत्रणांना बायपास करतात

##### बहु-सेवा हल्ल्याचे मार्ग
- अनेक सेवा स्वीकारलेल्या तुटलेल्या टोकनमुळे जोडलेल्या प्रणालींमध्ये आडव्या हालचाली शक्य होतात
- टोकन मूळ तपासता न आल्यास सेवा दरम्यान विश्वास गृहितकांचे उल्लंघन होऊ शकते

### सुरक्षा नियंत्रण आणि प्रतिबंध

**महत्त्वाच्या सुरक्षा आवश्यकता:**

> **अनिवार्य:** MCP सर्व्हर **कधीही** असे टोकन स्वीकारू नयेत जे स्पष्टपणे MCP सर्व्हरसाठी जारी केलेले नाहीत

#### प्रमाणीकरण आणि प्राधिकरण नियंत्रण

- **कठोर प्राधिकरण पुनरावलोकन**: MCP सर्व्हर प्राधिकरण लॉजिकचे सखोल लेखापरीक्षण करा जेणेकरून फक्त इच्छित वापरकर्ते आणि क्लायंट्स संवेदनशील संसाधनांपर्यंत प्रवेश करू शकतील
  - **अंमलबजावणी मार्गदर्शक**: [Azure API Management as Authentication Gateway for MCP Servers](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)
  - **ओळख एकत्रीकरण**: [Using Microsoft Entra ID for MCP Server Authentication](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **सुरक्षित टोकन व्यवस्थापन**: [Microsoft च्या टोकन पडताळणी आणि जीवनचक्र सर्वोत्तम पद्धती](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens) लागू करा
  - टोकन प्रेक्षक दावे MCP सर्व्हर ओळखीशी जुळतात का ते पडताळा
  - योग्य टोकन रोटेशन आणि कालबाह्यता धोरणे राबवा
  - टोकन पुनरावृत्ती हल्ले आणि अनधिकृत वापर प्रतिबंधित करा

- **संरक्षित टोकन संचयन**: विश्रांती आणि ट्रान्सिटमध्ये एन्क्रिप्शनसह टोकन संचयन सुरक्षित करा
  - **सर्वोत्तम पद्धती**: [Secure Token Storage and Encryption Guidelines](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### प्रवेश नियंत्रण अंमलबजावणी

- **किमान विशेषाधिकार तत्त्व**: MCP सर्व्हरना फक्त आवश्यक किमान परवानग्या द्या
  - विशेषाधिकार वाढ टाळण्यासाठी नियमित परवानगी पुनरावलोकने आणि अद्यतने करा
  - **Microsoft दस्तऐवज**: [Secure Least-Privileged Access](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **भूमिका-आधारित प्रवेश नियंत्रण (RBAC)**: सूक्ष्म भूमिका नियुक्ती करा
  - भूमिका विशिष्ट संसाधने आणि क्रियांपुरती मर्यादित ठेवा
  - हल्ल्याचा पृष्ठभाग वाढवणाऱ्या व्यापक किंवा अनावश्यक परवानग्या टाळा

- **सतत परवानगी निरीक्षण**: सतत प्रवेश लेखापरीक्षण आणि देखरेख राबवा
  - परवानगी वापर नमुन्यांवर लक्ष ठेवा
  - जास्त किंवा न वापरल्या जाणाऱ्या विशेषाधिकारांचे त्वरीत निराकरण करा

## AI-विशिष्ट सुरक्षा धोके

### प्रॉम्प्ट इंजेक्शन आणि टूल मॅनिप्युलेशन हल्ले

आधुनिक MCP अंमलबजावण्या अशा प्रगत AI-विशिष्ट हल्ल्यांना सामोरे जात आहेत ज्यांना पारंपरिक सुरक्षा उपाय पूर्णपणे हाताळू शकत नाहीत:

#### **अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन (क्रॉस-डोमेन प्रॉम्प्ट इंजेक्शन)**

**अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन** हे MCP-सक्षम AI प्रणालींमधील सर्वात गंभीर असुरक्षिततेपैकी एक आहे. हल्लेखोर बाह्य सामग्री—दस्तऐवज, वेब पृष्ठे, ईमेल, किंवा डेटा स्रोतांमध्ये—दुर्भावनायुक्त सूचना लपवतात, ज्यांना AI प्रणाली नंतर वैध आदेश म्हणून प्रक्रिया करते.

**हल्ल्याचे प्रकार:**
- **दस्तऐवज-आधारित इंजेक्शन**: प्रक्रिया केलेल्या दस्तऐवजांमध्ये लपविलेल्या दुर्भावनायुक्त सूचना ज्यामुळे अनपेक्षित AI क्रिया होतात
- **वेब सामग्री शोषण**: संक्रमित वेब पृष्ठे ज्यात एम्बेड केलेले प्रॉम्प्ट्स असतात जे AI वर्तनाला प्रभावित करतात
- **ईमेल-आधारित हल्ले**: ईमेलमधील दुर्भावनायुक्त प्रॉम्प्ट्स ज्यामुळे AI सहाय्यक माहिती लीक करतात किंवा अनधिकृत क्रिया करतात
- **डेटा स्रोत दूषित करणे**: संक्रमित डेटाबेस किंवा API जे AI प्रणालींना दूषित सामग्री पुरवतात

**वास्तविक परिणाम**: हे हल्ले डेटा चोरी, गोपनीयता उल्लंघन, हानिकारक सामग्री निर्मिती, आणि वापरकर्ता संवादांचे मनिप्युलेशन करू शकतात. सविस्तर विश्लेषणासाठी पहा [Prompt Injection in MCP (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/)।

![Prompt Injection Attack Diagram](../../../translated_images/mr/prompt-injection.ed9fbfde297ca877c15bc6daa808681cd3c3dc7bf27bbbda342ef1ba5fc4f52d.png)

#### **टूल विषबाधा हल्ले**

**टूल विषबाधा** MCP टूल्सची व्याख्या करणाऱ्या मेटाडेटावर हल्ला करतो, ज्यामुळे LLMs टूल वर्णने आणि पॅरामीटर्स कसे समजतात आणि अंमलबजावणी निर्णय घेतात याचा गैरफायदा घेतला जातो.

**हल्ल्याचे प्रकार:**
- **मेटाडेटा फेरफार**: हल्लेखोर टूल वर्णने, पॅरामीटर व्याख्या, किंवा वापर उदाहरणांमध्ये दुर्भावनायुक्त सूचना इंजेक्ट करतात
- **अदृश्य सूचना**: टूल मेटाडेटामध्ये लपविलेल्या प्रॉम्प्ट्स जे AI मॉडेल्सद्वारे प्रक्रिया होतात पण मानवी वापरकर्त्यांना दिसत नाहीत
- **डायनॅमिक टूल बदल ("रग पुल्स")**: वापरकर्त्यांनी मंजूर केलेले टूल नंतर दुर्भावनायुक्त क्रिया करण्यासाठी बदलले जातात, वापरकर्त्यांना न कळता
- **पॅरामीटर इंजेक्शन**: टूल पॅरामीटर स्कीमामध्ये दुर्भावनायुक्त सामग्री जी मॉडेल वर्तनावर परिणाम करते

**होस्टेड सर्व्हर धोके**: दूरस्थ MCP सर्व्हरमध्ये टूल व्याख्या वापरकर्त्यांच्या सुरुवातीच्या मंजुरीनंतर अद्यतनित केल्या जाऊ शकतात, ज्यामुळे पूर्वी सुरक्षित टूल्स दुर्भावनायुक्त होऊ शकतात. सविस्तर विश्लेषणासाठी पहा [Tool Poisoning Attacks (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks)।

![Tool Injection Attack Diagram](../../../translated_images/mr/tool-injection.3b0b4a6b24de6befe7d3afdeae44138ef005881aebcfc84c6f61369ce31e3640.png)

#### **अतिरिक्त AI हल्ल्याचे मार्ग**

- **क्रॉस-डोमेन प्रॉम्प्ट इंजेक्शन (XPIA)**: अनेक डोमेनमधील सामग्रीचा वापर करून सुरक्षा नियंत्रण बायपास करणारे प्रगत हल्ले
- **डायनॅमिक क्षमता बदल**: सुरुवातीच्या सुरक्षा मूल्यमापनांना वगळून टूल क्षमतांमध्ये रिअल-टाइम बदल
- **कॉन्टेक्स्ट विंडो विषबाधा**: मोठ्या कॉन्टेक्स्ट विंडोमध्ये दुर्भावनायुक्त सूचना लपविणारे हल्ले
- **मॉडेल कन्फ्युजन हल्ले**: मॉडेल मर्यादा वापरून अनपेक्षित किंवा असुरक्षित वर्तन तयार करणे

### AI सुरक्षा धोका परिणाम

**उच्च परिणाम:**
- **डेटा चोरी**: संवेदनशील एंटरप्राइझ किंवा वैयक्तिक डेटाचा अनधिकृत प्रवेश आणि चोरी
- **गोपनीयता उल्लंघन**: वैयक्तिक ओळखण्यायोग्य माहिती (PII) आणि गोपनीय व्यावसायिक डेटा उघड होणे  
- **प्रणाली मनिप्युलेशन**: महत्त्वाच्या प्रणाली आणि कार्यप्रवाहांमध्ये अनपेक्षित बदल
- **प्रमाणीकरण टोकन आणि सेवा क्रेडेन्शियल चोरी**
- **आडवी हालचाल**: व्यापक नेटवर्क हल्ल्यांसाठी तुटलेल्या AI प्रणालींचा वापर

### Microsoft AI सुरक्षा उपाय

#### **AI Prompt Shields: इंजेक्शन हल्ल्यांविरुद्ध प्रगत संरक्षण**

Microsoft **AI Prompt Shields** थेट आणि अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन हल्ल्यांविरुद्ध बहुस्तरीय सुरक्षा प्रदान करतात:

##### **मुख्य संरक्षण यंत्रणा:**

1. **प्रगत शोध आणि फिल्टरिंग**
   - मशीन लर्निंग अल्गोरिदम आणि NLP तंत्रज्ञान बाह्य सामग्रीतील दुर्भावनायुक्त सूचना शोधतात
   - दस्तऐवज, वेब पृष्ठे, ईमेल, आणि डेटा स्रोतांमध्ये एम्बेड केलेल्या धोक्यांचे रिअल-टाइम विश्लेषण
   - वैध आणि दुर्भावनायुक्त प्रॉम्प्ट नमुन्यांचे संदर्भात्मक समज

2. **स्पॉटलाइटिंग तंत्र**
   - विश्वासार्ह प्रणाली सूचना आणि संभाव्य संक्रमित बाह्य इनपुट यामध्ये फरक करतो
   - मजकूर रूपांतरण पद्धती ज्यामुळे मॉडेलची सुसंगतता वाढते आणि दुर्भावनायुक्त सामग्री वेगळी केली जाते
   - AI प्रणालींना योग्य सूचना श्रेणी राखण्यास आणि इंजेक्ट केलेल्या आदेशांना दुर्लक्षित करण्यास मदत

3. **डिलिमीटर आणि डेटामार्किंग सिस्टम**
   - विश्वासार्ह प्रणाली संदेश आणि बाह्य इनपुट मजकूर यामध्ये स्पष्ट सीमा ठरवणे
   - विश्वासार्ह आणि अविश्वसनीय डेटा स्रोतांमधील सीमारेषा दर्शविणारे विशेष चिन्हे
   - स्पष्ट विभाजन ज्यामुळे सूचना गोंधळ आणि अनधिकृत आदेश अंमलबजावणी टाळली जाते

4. **सतत धोका बुद्धिमत्ता**
   - Microsoft सतत नवीन हल्ल्यांच्या नमुन्यांवर लक्ष ठेवतो आणि संरक्षण अद्यतनित करतो
   - नवीन इंजेक्शन तंत्र आणि हल्ल्यांच्या मार्गांसाठी सक्रिय शोध
   - विकसित होत असलेल्या धोक्यांविरुद्ध प्रभावीपणा राखण्यासाठी नियमित सुरक्षा मॉडेल अद्यतने

5. **Azure Content Safety एकत्रीकरण**
   - व्यापक Azure AI Content Safety सूटचा भाग
   - जेलब्रेक प्रयत्न, हानिकारक सामग्री, आणि सुरक्षा धोरण उल्लंघनांसाठी अतिरिक्त शोध
   - AI अनुप्रयोग घटकांमध्ये एकसंध सुरक्षा नियंत्रण

## प्रगत MCP सुरक्षा धोके

### सत्र हायजॅकिंग असुरक्षितता

**सत्र हायजॅकिंग** ही राज्यसंपन्न MCP अंमलबजावणीत एक गंभीर हल्ला मार्ग आहे जिथे अनधिकृत पक्ष वैध सत्र आयडेंटिफायर्स मिळवून क्लायंटची नक्कल करतात आणि अनधिकृत क्रिया करतात.

#### **हल्ल्याचे प्रकार आणि धोके**

- **सत्र हायजॅक प्रॉम्प्ट इंजेक्शन**: चोरी केलेल्या सत्र आयडीसह हल्लेखोर सत्र स्थिती सामायिक करणाऱ्या सर्व्हरमध्ये दुर्भावनायुक्त घटना इंजेक्ट करतात, ज्यामुळे हानिकारक क्रिया होऊ शकतात किंवा संवेदनशील डेटा मिळू शकतो
- **थेट नक्कल**: चोरी केलेल्या सत्र आयडीमुळे प्रमाणीकरण बायपास करून थेट MCP सर्व्हर कॉल करता येतात, ज्यामुळे हल्लेखोर वैध वापरकर्ते म्हणून वागू शकतात
- **तुटलेले पुनरारंभयोग्य स्ट्रीम्स**: हल्लेखोर विनंत्या वेळेपूर्वी थांबवू शकतात, ज्यामुळे वैध क्लायंट्स संभाव्य दुर्भावनायुक्त सामग्रीसह पुनरारंभ करतात

#### **सत्र व्यवस्थापनासाठी सुरक्षा नियंत्रण**

**महत्त्वाच्या आवश्यकता:**
- **प्राधिकरण पडताळणी**: प्राधिकरण अंमलात आणणाऱ्या MCP सर्व्हरने सर्व इनबाउंड विनंत्या पडताळल्या पाहिजेत आणि प्रमाणीकरणासाठी सत्रांवर अवलंबून राहू नये
- **सुरक्षित सत्र निर्मिती**: क्रिप्टोग्राफिकदृष्ट्या सुरक्षित, अनिश्चित सत्र आयडी वापरा जे सुरक्षित यादृच्छिक संख्या जनकांसह तयार केले जातात
- **वापरकर्ता-विशिष्ट बाइंडिंग**: सत्र आयडी वापरकर्ता-विशिष्ट माहितीसह `<user_id>:<session_id>` सारख्या स्वरूपात बांधा जेणेकरून क्रॉस-युजर सत्राचा गैरवापर टाळता येईल
- **सत्र जीवनचक्र व्यवस्थापन**: संवेदनशीलता मर्यादित करण्यासाठी योग्य कालबाह्यता, फेरफार आणि अमान्यकरण अंमलात आणा
- **वाहतूक सुरक्षा**: सत्र आयडीच्या अडथळ्यापासून बचाव करण्यासाठी सर्व संवादासाठी HTTPS अनिवार्य आहे

### गोंधळलेला डिप्युटी समस्या

**गोंधळलेला डिप्युटी समस्या** तेव्हा उद्भवते जेव्हा MCP सर्व्हर क्लायंट आणि तृतीय-पक्ष सेवा यांच्यात प्रमाणीकरण प्रॉक्सी म्हणून कार्य करतात, ज्यामुळे स्थिर क्लायंट आयडीचा गैरवापर करून प्राधिकरण बायपास होण्याची संधी निर्माण होते.

#### **हल्ल्याची यंत्रणा आणि धोके**

- **कुकी-आधारित संमती बायपास**: पूर्वीच्या वापरकर्त्याच्या प्रमाणीकरणामुळे संमती कुकी तयार होतात ज्याचा हल्लेखोर दुष्ट प्राधिकरण विनंत्यांसह तयार केलेल्या रीडायरेक्ट URI द्वारे गैरवापर करतात
- **प्राधिकरण कोड चोरी**: विद्यमान संमती कुकीमुळे प्राधिकरण सर्व्हर संमती स्क्रीन वगळू शकतात आणि कोड हल्लेखोरांच्या नियंत्रित टप्प्यांकडे पुनर्निर्देशित करतात  
- **अनधिकृत API प्रवेश**: चोरी केलेले प्राधिकरण कोड टोकन विनिमय आणि वापरकर्ता छद्मवेशासाठी वापरले जातात, ज्यासाठी स्पष्ट मंजुरी आवश्यक नाही

#### **तडजोडी धोरणे**

**अनिवार्य नियंत्रण:**
- **स्पष्ट संमती आवश्यकता**: स्थिर क्लायंट आयडी वापरणाऱ्या MCP प्रॉक्सी सर्व्हरने प्रत्येक गतिशील नोंदणीकृत क्लायंटसाठी वापरकर्त्याची संमती मिळवणे आवश्यक आहे
- **OAuth 2.1 सुरक्षा अंमलबजावणी**: सर्व प्राधिकरण विनंत्यांसाठी PKCE (प्रूफ की फॉर कोड एक्सचेंज) सह सध्याच्या OAuth सुरक्षा सर्वोत्तम पद्धतींचे पालन करा
- **कडक क्लायंट पडताळणी**: गैरवापर टाळण्यासाठी रीडायरेक्ट URI आणि क्लायंट ओळखपत्रांची काटेकोर पडताळणी करा

### टोकन पासथ्रू असुरक्षितता  

**टोकन पासथ्रू** हा एक स्पष्ट अँटी-पॅटर्न आहे जिथे MCP सर्व्हर योग्य पडताळणी न करता क्लायंट टोकन स्वीकारतात आणि त्यांना डाउनस्ट्रीम API कडे पुढे पाठवतात, ज्यामुळे MCP प्राधिकरण तपशीलांचे उल्लंघन होते.

#### **सुरक्षा परिणाम**

- **नियंत्रण टाळणे**: थेट क्लायंट-टू-API टोकन वापर महत्त्वाच्या दर मर्यादा, पडताळणी आणि निरीक्षण नियंत्रणांना बायपास करते
- **ऑडिट ट्रेल भ्रष्टाचार**: अपस्ट्रीम-इश्यूड टोकनमुळे क्लायंट ओळखणे अशक्य होते, ज्यामुळे घटनेची चौकशी करणे कठीण होते
- **प्रॉक्सी-आधारित डेटा चोरी**: अमान्य टोकन हल्लेखोरांना सर्व्हर प्रॉक्सी म्हणून वापरून अनधिकृत डेटा प्रवेश करण्यास सक्षम करतात
- **विश्वास सीमा उल्लंघने**: टोकन मूळ तपासता न आल्यास डाउनस्ट्रीम सेवांच्या विश्वास गृहितकांचे उल्लंघन होऊ शकते
- **बहु-सेवा हल्ल्याचा विस्तार**: अनेक सेवांमध्ये स्वीकारलेले बिघडलेले टोकन पार्श्वभूमीवर हालचाल करण्यास अनुमती देतात

#### **आवश्यक सुरक्षा नियंत्रण**

**अविचलनीय आवश्यकता:**
- **टोकन पडताळणी**: MCP सर्व्हरने फक्त त्यांच्यासाठी स्पष्टपणे जारी केलेले टोकन स्वीकारावे, अन्यथा नाही
- **प्रेक्षक पडताळणी**: टोकन प्रेक्षक दावे नेहमी MCP सर्व्हरच्या ओळखीशी जुळतात का ते तपासा
- **योग्य टोकन जीवनचक्र**: सुरक्षित फेरफारासह अल्पकालीन प्रवेश टोकन अंमलात आणा


## AI प्रणालींसाठी पुरवठा साखळी सुरक्षा

पुरवठा साखळी सुरक्षा पारंपरिक सॉफ्टवेअर अवलंबित्वांपेक्षा पुढे जाऊन संपूर्ण AI परिसंस्थेचा समावेश करते. आधुनिक MCP अंमलबजावण्या सर्व AI-संबंधित घटकांची काटेकोरपणे पडताळणी आणि निरीक्षण करणे आवश्यक आहे, कारण प्रत्येक घटक संभाव्य असुरक्षितता आणतो ज्यामुळे प्रणालीची अखंडता धोक्यात येऊ शकते.

### विस्तारित AI पुरवठा साखळी घटक

**पारंपरिक सॉफ्टवेअर अवलंबित्वे:**
- मुक्त स्रोत लायब्ररी आणि फ्रेमवर्क
- कंटेनर प्रतिमा आणि बेस सिस्टम  
- विकास साधने आणि बिल्ड पाइपलाइन
- पायाभूत सुविधा घटक आणि सेवा

**AI-विशिष्ट पुरवठा साखळी घटक:**
- **फाउंडेशन मॉडेल्स**: विविध पुरवठादारांकडून पूर्व-प्रशिक्षित मॉडेल्स ज्यांची उत्पत्ती पडताळणी आवश्यक आहे
- **एम्बेडिंग सेवा**: बाह्य व्हेक्टरायझेशन आणि अर्थपूर्ण शोध सेवा
- **संदर्भ पुरवठादार**: डेटा स्रोत, ज्ञान आधार, आणि दस्तऐवज संच  
- **तृतीय-पक्ष API**: बाह्य AI सेवा, ML पाइपलाइन, आणि डेटा प्रक्रिया टप्पे
- **मॉडेल कलाकृती**: वजन, संरचना, आणि सूक्ष्मसुधारित मॉडेल प्रकार
- **प्रशिक्षण डेटा स्रोत**: मॉडेल प्रशिक्षण आणि सूक्ष्मसुधारणेसाठी वापरलेले डेटासेट

### सर्वसमावेशक पुरवठा साखळी सुरक्षा धोरण

#### **घटक पडताळणी आणि विश्वास**
- **उत्पत्ती पडताळणी**: एकत्रीकरणापूर्वी सर्व AI घटकांची उत्पत्ती, परवाना, आणि अखंडता तपासा
- **सुरक्षा मूल्यांकन**: मॉडेल्स, डेटा स्रोत, आणि AI सेवांसाठी असुरक्षितता स्कॅन आणि सुरक्षा पुनरावलोकने करा
- **प्रतिष्ठा विश्लेषण**: AI सेवा पुरवठादारांच्या सुरक्षा इतिहास आणि पद्धतींचे मूल्यांकन करा
- **अनुपालन पडताळणी**: सर्व घटक संस्थात्मक सुरक्षा आणि नियामक आवश्यकता पूर्ण करतात याची खात्री करा

#### **सुरक्षित तैनाती पाइपलाइन**  
- **स्वयंचलित CI/CD सुरक्षा**: स्वयंचलित तैनाती पाइपलाइनमध्ये सुरक्षा स्कॅनिंग समाकलित करा
- **कलाकृती अखंडता**: सर्व तैनात कलाकृतींसाठी (कोड, मॉडेल्स, संरचना) क्रिप्टोग्राफिक पडताळणी अंमलात आणा
- **टप्प्याटप्प्याने तैनाती**: प्रत्येक टप्प्यावर सुरक्षा पडताळणीसह प्रगत तैनाती धोरणे वापरा
- **विश्वसनीय कलाकृती संग्रहालये**: फक्त पडताळलेले, सुरक्षित कलाकृती संग्रहालय आणि रेपॉजिटरीजमधून तैनात करा

#### **सतत निरीक्षण आणि प्रतिसाद**
- **अवलंबित्व स्कॅनिंग**: सर्व सॉफ्टवेअर आणि AI घटकांच्या अवलंबित्वांसाठी सतत असुरक्षितता निरीक्षण
- **मॉडेल निरीक्षण**: मॉडेल वर्तन, कार्यप्रदर्शनातील बदल, आणि सुरक्षा अनियमिततांचे सतत मूल्यांकन
- **सेवा आरोग्य ट्रॅकिंग**: बाह्य AI सेवांच्या उपलब्धता, सुरक्षा घटना, आणि धोरण बदलांचे निरीक्षण
- **धोका बुद्धिमत्ता समाकलन**: AI आणि ML सुरक्षा धोके यांसाठी विशिष्ट धोका फीड्स समाविष्ट करा

#### **प्रवेश नियंत्रण आणि किमान विशेषाधिकार**
- **घटक-स्तरीय परवानग्या**: व्यवसाय गरजेनुसार मॉडेल्स, डेटा, आणि सेवांवर प्रवेश मर्यादित करा
- **सेवा खाते व्यवस्थापन**: आवश्यक किमान परवानग्यांसह समर्पित सेवा खाती अंमलात आणा
- **नेटवर्क विभागणी**: AI घटक वेगळे करा आणि सेवांमधील नेटवर्क प्रवेश मर्यादित करा
- **API गेटवे नियंत्रण**: बाह्य AI सेवांवर प्रवेश नियंत्रित आणि निरीक्षण करण्यासाठी केंद्रीकृत API गेटवे वापरा

#### **घटना प्रतिसाद आणि पुनर्प्राप्ती**
- **त्वरित प्रतिसाद प्रक्रिया**: बिघडलेल्या AI घटकांचे पॅचिंग किंवा बदल करण्यासाठी स्थापित प्रक्रिया
- **प्रमाणपत्र फेरफार**: रहस्ये, API कीज, आणि सेवा प्रमाणपत्रे फेरफार करण्यासाठी स्वयंचलित प्रणाली
- **रोलबॅक क्षमता**: AI घटकांच्या पूर्वीच्या ज्ञात-चांगल्या आवृत्त्यांकडे जलद परत जाण्याची क्षमता
- **पुरवठा साखळी भंग पुनर्प्राप्ती**: अपस्ट्रीम AI सेवा बिघडल्यास प्रतिसादासाठी विशिष्ट प्रक्रिया

### मायक्रोसॉफ्ट सुरक्षा साधने आणि समाकलन

**GitHub Advanced Security** पुरवठा साखळी संरक्षणासाठी समग्र सुविधा देते ज्यात समाविष्ट आहे:
- **रहस्य स्कॅनिंग**: रेपॉजिटरीजमधील प्रमाणपत्रे, API कीज, आणि टोकनचे स्वयंचलित शोध
- **अवलंबित्व स्कॅनिंग**: मुक्त स्रोत अवलंबित्वे आणि लायब्ररीसाठी असुरक्षितता मूल्यांकन
- **CodeQL विश्लेषण**: सुरक्षा असुरक्षितता आणि कोडिंग समस्या यांसाठी स्थिर कोड विश्लेषण
- **पुरवठा साखळी अंतर्दृष्टी**: अवलंबित्व आरोग्य आणि सुरक्षा स्थितीवर दृष्टी

**Azure DevOps & Azure Repos समाकलन:**
- मायक्रोसॉफ्ट विकास प्लॅटफॉर्मवर सुरक्षेचे अखंड स्कॅनिंग समाकलन
- AI वर्कलोडसाठी Azure पाइपलाइनमध्ये स्वयंचलित सुरक्षा तपासणी
- सुरक्षित AI घटक तैनातीसाठी धोरण अंमलबजावणी

**मायक्रोसॉफ्ट अंतर्गत पद्धती:**
मायक्रोसॉफ्ट सर्व उत्पादनांमध्ये विस्तृत पुरवठा साखळी सुरक्षा पद्धती अंमलात आणते. [The Journey to Secure the Software Supply Chain at Microsoft](https://devblogs.microsoft.com/engineering-at-microsoft/the-journey-to-secure-the-software-supply-chain-at-microsoft/) येथे सिद्ध पद्धतींबद्दल जाणून घ्या.


## फाउंडेशन सुरक्षा सर्वोत्तम पद्धती

MCP अंमलबजावण्या आपल्या संस्थेच्या विद्यमान सुरक्षा स्थितीवर आधारित असतात आणि त्यावर बांधल्या जातात. मूलभूत सुरक्षा पद्धती मजबूत केल्याने AI प्रणाली आणि MCP तैनातींची एकूण सुरक्षा लक्षणीयरीत्या वाढते.

### मुख्य सुरक्षा मूलतत्त्वे

#### **सुरक्षित विकास पद्धती**
- **OWASP अनुपालन**: [OWASP Top 10](https://owasp.org/www-project-top-ten/) वेब अनुप्रयोग असुरक्षिततेपासून संरक्षण करा
- **AI-विशिष्ट संरक्षण**: [OWASP Top 10 for LLMs](https://genai.owasp.org/download/43299/?tmstv=1731900559) साठी नियंत्रण अंमलात आणा
- **सुरक्षित रहस्य व्यवस्थापन**: टोकन, API कीज, आणि संवेदनशील संरचना डेटासाठी समर्पित वॉल्ट वापरा
- **एंड-टू-एंड एन्क्रिप्शन**: सर्व अनुप्रयोग घटक आणि डेटा प्रवाहांमध्ये सुरक्षित संवाद अंमलात आणा
- **इनपुट पडताळणी**: सर्व वापरकर्ता इनपुट, API पॅरामीटर्स, आणि डेटा स्रोतांची काटेकोर पडताळणी करा

#### **पायाभूत सुविधा कडक करणे**
- **मल्टी-फॅक्टर प्रमाणीकरण**: सर्व प्रशासकीय आणि सेवा खात्यांसाठी अनिवार्य MFA
- **पॅच व्यवस्थापन**: ऑपरेटिंग सिस्टम, फ्रेमवर्क, आणि अवलंबित्वांसाठी स्वयंचलित, वेळेवर पॅचिंग  
- **ओळख पुरवठादार समाकलन**: एंटरप्राइझ ओळख पुरवठादारांद्वारे केंद्रीकृत ओळख व्यवस्थापन (Microsoft Entra ID, Active Directory)
- **नेटवर्क विभागणी**: MCP घटकांचे तार्किक वेगळेपण जेणेकरून पार्श्वभूमी हालचाली मर्यादित होतील
- **किमान विशेषाधिकार तत्त्व**: सर्व प्रणाली घटक आणि खात्यांसाठी आवश्यक किमान परवानग्या

#### **सुरक्षा निरीक्षण आणि शोध**
- **सर्वसमावेशक लॉगिंग**: AI अनुप्रयोग क्रियाकलापांचे तपशीलवार लॉगिंग, ज्यात MCP क्लायंट-सर्व्हर संवाद समाविष्ट आहे
- **SIEM समाकलन**: अनियमितता शोधण्यासाठी केंद्रीकृत सुरक्षा माहिती आणि घटना व्यवस्थापन
- **वर्तन विश्लेषण**: प्रणाली आणि वापरकर्ता वर्तनातील असामान्य नमुने शोधण्यासाठी AI-चालित निरीक्षण
- **धोका बुद्धिमत्ता**: बाह्य धोका फीड्स आणि तक्रार निर्देशक (IOCs) समाकलित करा
- **घटना प्रतिसाद**: सुरक्षा घटना शोध, प्रतिसाद, आणि पुनर्प्राप्तीसाठी सुव्यवस्थित प्रक्रिया

#### **झिरो ट्रस्ट आर्किटेक्चर**
- **कधीही विश्वास ठेवू नका, नेहमी पडताळणी करा**: वापरकर्ते, उपकरणे, आणि नेटवर्क कनेक्शनचे सतत पडताळणी
- **मायक्रो-सेगमेंटेशन**: वैयक्तिक वर्कलोड आणि सेवांचे सूक्ष्म नेटवर्क नियंत्रण
- **ओळख-केंद्रित सुरक्षा**: नेटवर्क स्थानाऐवजी पडताळलेल्या ओळखीवर आधारित सुरक्षा धोरणे
- **सतत धोका मूल्यांकन**: वर्तमान संदर्भ आणि वर्तनावर आधारित गतिशील सुरक्षा स्थितीचे मूल्यांकन
- **सशर्त प्रवेश**: धोका घटक, स्थान, आणि उपकरण विश्वासावर आधारित प्रवेश नियंत्रण

### एंटरप्राइझ समाकलन नमुने

#### **मायक्रोसॉफ्ट सुरक्षा परिसंस्था समाकलन**
- **Microsoft Defender for Cloud**: सर्वसमावेशक क्लाउड सुरक्षा स्थिती व्यवस्थापन
- **Azure Sentinel**: AI वर्कलोड संरक्षणासाठी क्लाउड-नेटिव्ह SIEM आणि SOAR क्षमता
- **Microsoft Entra ID**: सशर्त प्रवेश धोरणांसह एंटरप्राइझ ओळख आणि प्रवेश व्यवस्थापन
- **Azure Key Vault**: हार्डवेअर सुरक्षा मॉड्यूल (HSM) समर्थनासह केंद्रीकृत रहस्य व्यवस्थापन
- **Microsoft Purview**: AI डेटा स्रोत आणि कार्यप्रवाहांसाठी डेटा शासन आणि अनुपालन

#### **अनुपालन आणि शासन**
- **नियामक संरेखन**: MCP अंमलबजावण्या उद्योग-विशिष्ट अनुपालन आवश्यकता पूर्ण करतात याची खात्री करा (GDPR, HIPAA, SOC 2)
- **डेटा वर्गीकरण**: AI प्रणालीद्वारे प्रक्रिया केलेल्या संवेदनशील डेटाचे योग्य वर्गीकरण आणि हाताळणी
- **ऑडिट ट्रेल्स**: नियामक अनुपालन आणि फॉरेन्सिक तपासणीसाठी सर्वसमावेशक लॉगिंग
- **गोपनीयता नियंत्रण**: AI प्रणाली आर्किटेक्चरमध्ये प्रायव्हसी-बाय-डिझाइन तत्त्वांची अंमलबजावणी
- **बदल व्यवस्थापन**: AI प्रणाली बदलांसाठी सुरक्षा पुनरावलोकनांची औपचारिक प्रक्रिया

हे मूलभूत पद्धती एक मजबूत सुरक्षा आधार तयार करतात जे MCP-विशिष्ट सुरक्षा नियंत्रणांची कार्यक्षमता वाढवतात आणि AI-चालित अनुप्रयोगांसाठी सर्वसमावेशक संरक्षण प्रदान करतात.

## मुख्य सुरक्षा मुद्दे

- **स्तरीय सुरक्षा दृष्टिकोन**: मूलभूत सुरक्षा पद्धती (सुरक्षित कोडिंग, किमान विशेषाधिकार, पुरवठा साखळी पडताळणी, सतत निरीक्षण) AI-विशिष्ट नियंत्रणांसह एकत्र करा जेणेकरून सर्वसमावेशक संरक्षण मिळेल

- **AI-विशिष्ट धोका परिदृश्य**: MCP प्रणालींना प्रॉम्प्ट इंजेक्शन, टूल विषबाधा, सत्र हायजॅकिंग, गोंधळलेला डिप्युटी समस्या, टोकन पासथ्रू असुरक्षितता, आणि अतिरेकी परवानग्यांसारख्या अनन्य धोके भेडसावतात ज्यासाठी विशेष तडजोडी आवश्यक आहेत

- **प्रमाणीकरण आणि प्राधिकरण उत्कृष्टता**: बाह्य ओळख पुरवठादार (Microsoft Entra ID) वापरून मजबूत प्रमाणीकरण अंमलात आणा, योग्य टोकन पडताळणी करा, आणि आपल्या MCP सर्व्हरसाठी स्पष्टपणे जारी न केलेले टोकन कधीही स्वीकारू नका

- **AI हल्ला प्रतिबंध**: अप्रत्यक्ष प्रॉम्प्ट इंजेक्शन आणि टूल विषबाधा हल्ल्यांपासून संरक्षणासाठी Microsoft Prompt Shields आणि Azure Content Safety तैनात करा, टूल मेटाडेटा पडताळा आणि गतिशील बदलांसाठी निरीक्षण करा

- **सत्र आणि वाहतूक सुरक्षा**: क्रिप्टोग्राफिकदृष्ट्या सुरक्षित, अनिश्चित सत्र आयडी वापरा जे वापरकर्ता ओळखीशी बांधलेले असतील, योग्य सत्र जीवनचक्र व्यवस्थापन करा, आणि प्रमाणीकरणासाठी कधीही सत्रांचा वापर करू नका

- **OAuth सुरक्षा सर्वोत्तम पद्धती**: गतिशील नोंदणीकृत क्लायंटसाठी स्पष्ट वापरकर्ता संमती, PKCE सह योग्य OAuth 2.1 अंमलबजावणी, आणि कडक रीडायरेक्ट URI पडताळणीद्वारे गोंधळलेला डिप्युटी हल्ला टाळा  

- **टोकन सुरक्षा तत्त्वे**: टोकन पासथ्रू अँटी-पॅटर्न टाळा, टोकन प्रेक्षक दावे पडताळा, सुरक्षित फेरफारासह अल्पकालीन टोकन अंमलात आणा, आणि स्पष्ट विश्वास सीमा राखा

- **सर्वसमावेशक पुरवठा साखळी सुरक्षा**: सर्व AI परिसंस्था घटक (मॉडेल्स, एम्बेडिंग्ज, संदर्भ पुरवठादार, बाह्य API) पारंपरिक सॉफ्टवेअर अवलंबित्वांप्रमाणेच काटेकोरपणे सुरक्षित करा

- **सतत प्रगती**: जलद विकसित होणाऱ्या MCP तपशीलांसह अद्ययावत रहा, सुरक्षा समुदाय मानकांमध्ये योगदान द्या, आणि प्रोटोकॉल प्रगल्भ होत असताना अनुकूल सुरक्षा स्थिती राखा

- **मायक्रोसॉफ्ट सुरक्षा समाकलन**: Microsoft च्या सर्वसमावेशक सुरक्षा परिसंस्थेचा (Prompt Shields, Azure Content Safety, GitHub Advanced Security, Entra ID) वापर करून MCP तैनातीचे संरक्षण वाढवा
