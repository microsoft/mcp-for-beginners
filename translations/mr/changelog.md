# बदलनोंची नोंद: MCP for Beginners अभ्यासक्रम

हा दस्तऐवज Model Context Protocol (MCP) for Beginners अभ्यासक्रमात केलेल्या सर्व महत्त्वाच्या बदलांची नोंद म्हणून काम करतो. बदल उलट कालक्रमानुसार (नवीनतम बदल आधी) नोंदवले आहेत.

  - सर्व स्पेसिफिकेशन URL वर्तमान दस्तऐवजाकडे निर्देशित असल्याची पुष्टी

- **स्पेसिफिकेशन अनुपालन**: सर्व अनिवार्य MCP सुरक्षा आवश्यकता (MUST/MUST NOT) नवीनतम स्पेसिफिकेशनशी सुसंगत असल्याची खात्री
- **संसाधन वर्तमानता**: Microsoft दस्तऐवज, सुरक्षा मानके, आणि अंमलबजावणी मार्गदर्शकांसाठी सर्व बाह्य दुवे तपासले
- **सर्वोत्तम पद्धतींचे कव्हरेज**: प्रमाणीकरण, प्राधिकरण, AI-विशिष्ट धोके, पुरवठा साखळी सुरक्षा, आणि एंटरप्राइझ नमुन्यांचे व्यापक कव्हरेज पुष्टी केली

## ऑक्टोबर 6, 2025

### Getting Started विभाग विस्तार – प्रगत सर्व्हर वापर & सोपी प्रमाणीकरण

#### प्रगत सर्व्हर वापर (03-GettingStarted/10-advanced)
- **नवीन प्रकरण जोडले**: प्रगत MCP सर्व्हर वापरासाठी व्यापक मार्गदर्शक सादर केला, ज्यात नियमित आणि लो-लेव्हल सर्व्हर आर्किटेक्चर दोन्हींचा समावेश आहे.
  - **नियमित विरुद्ध लो-लेव्हल सर्व्हर**: दोन्ही दृष्टिकोनांसाठी Python आणि TypeScript मध्ये तपशीलवार तुलना आणि कोड उदाहरणे.
  - **हँडलर-आधारित डिझाइन**: स्केलेबल, लवचिक सर्व्हर अंमलबजावणीसाठी टूल/संसाधन/प्रॉम्प्ट व्यवस्थापनाचे हँडलर-आधारित स्पष्टीकरण.
  - **व्यावहारिक नमुने**: प्रगत वैशिष्ट्ये आणि आर्किटेक्चरसाठी लो-लेव्हल सर्व्हर नमुन्यांचा वापर करणारे वास्तविक जीवनातील परिस्थिती.

#### सोपी प्रमाणीकरण (03-GettingStarted/11-simple-auth)
- **नवीन प्रकरण जोडले**: MCP सर्व्हरमध्ये सोपी प्रमाणीकरण कशी अंमलात आणायची यासाठी टप्प्याटप्प्याने मार्गदर्शक.
  - **प्रमाणीकरण संकल्पना**: प्रमाणीकरण विरुद्ध प्राधिकरण आणि क्रेडेन्शियल हाताळणी यांचे स्पष्ट स्पष्टीकरण.
  - **मूलभूत प्रमाणीकरण अंमलबजावणी**: Python (Starlette) आणि TypeScript (Express) मध्ये मिडलवेअर-आधारित प्रमाणीकरण नमुने, कोड उदाहरणांसह.
  - **प्रगत सुरक्षा कडे प्रगती**: सोप्या प्रमाणीकरणापासून OAuth 2.1 आणि RBAC कडे कसे पुढे जायचे यासाठी मार्गदर्शन, प्रगत सुरक्षा मॉड्यूल संदर्भांसह.

हे जोडणी प्रायोगिक, हाताळणी मार्गदर्शन प्रदान करतात जे अधिक मजबूत, सुरक्षित, आणि लवचिक MCP सर्व्हर अंमलबजावणी तयार करण्यासाठी, मूलभूत संकल्पना आणि प्रगत उत्पादन नमुन्यांमध्ये पूल तयार करतात.

- **प्रिमिटिव्ह पुनर्रचना**: सर्व्हर आणि क्लायंट प्रिमिटिव्हचे पूर्णपणे पुनर्रचित
  - सर्व्हर प्रिमिटिव्ह: संसाधने (डेटा स्रोत), प्रॉम्प्ट (टेम्पलेट), टूल्स (कार्यकारी फंक्शन्स) तपशीलवार स्पष्टीकरणे आणि उदाहरणांसह
  - क्लायंट प्रिमिटिव्ह: सॅम्पलिंग (LLM पूर्णता), एलिसिटेशन (वापरकर्ता इनपुट), लॉगिंग (डिबगिंग/निरीक्षण)
  - सध्याच्या शोध (`*/list`), पुनर्प्राप्ती (`*/get`), आणि अंमलबजावणी (`*/call`) पद्धतींचे अद्यतन
- **प्रोटोकॉल आर्किटेक्चर**: दोन-स्तरीय आर्किटेक्चर मॉडेल सादर
  - डेटा स्तर: JSON-RPC 2.0 पाया, जीवनचक्र व्यवस्थापन आणि प्रिमिटिव्हसह
  - वाहतूक स्तर: STDIO (स्थानिक) आणि Streamable HTTP सह SSE (दूरस्थ) वाहतूक यंत्रणा
- **सुरक्षा फ्रेमवर्क**: स्पष्ट वापरकर्ता संमती, डेटा गोपनीयता संरक्षण, टूल अंमलबजावणी सुरक्षा, आणि वाहतूक स्तर सुरक्षा यांसह व्यापक सुरक्षा तत्त्वे
- **संवाद नमुने**: प्रारंभ, शोध, अंमलबजावणी, आणि सूचना प्रवाह दर्शविणारे प्रोटोकॉल संदेश अद्यतनित
- **कोड उदाहरणे**: सध्याच्या MCP SDK नमुन्यांचे प्रतिबिंबित करणारी बहुभाषिक उदाहरणे (.NET, Java, Python, JavaScript) ताजेतवाने

#### सुरक्षा (02-Security/) - व्यापक सुरक्षा पुनर्रचना  
- **मानके संरेखन**: MCP Specification 2025-06-18 सुरक्षा आवश्यकता पूर्णपणे संरेखित
- **प्रमाणीकरण विकास**: कस्टम OAuth सर्व्हरपासून बाह्य ओळख प्रदाता प्रतिनिधित्व (Microsoft Entra ID) पर्यंत विकास दस्तऐवजित
- **AI-विशिष्ट धोका विश्लेषण**: आधुनिक AI हल्ला व्हेक्टरचे सुधारित कव्हरेज
  - वास्तविक उदाहरणांसह तपशीलवार प्रॉम्प्ट इंजेक्शन हल्ला परिस्थिती
  - टूल विषबाधा यंत्रणा आणि "रग पुल" हल्ला नमुने
  - संदर्भ विंडो विषबाधा आणि मॉडेल गोंधळ हल्ले
- **Microsoft AI सुरक्षा उपाय**: Microsoft सुरक्षा परिसंस्थेचे व्यापक कव्हरेज
  - प्रगत शोध, स्पॉटलाइटिंग, आणि डिलिमीटर तंत्रांसह AI Prompt Shields
  - Azure Content Safety एकत्रिकरण नमुने
  - पुरवठा साखळी संरक्षणासाठी GitHub Advanced Security
- **प्रगत धोका प्रतिबंध**: तपशीलवार सुरक्षा नियंत्रण
  - MCP-विशिष्ट हल्ला परिस्थिती आणि क्रिप्टोग्राफिक सत्र ID आवश्यकता असलेल्या सत्र हायजॅकिंगसाठी
  - स्पष्ट संमती आवश्यकता असलेल्या MCP प्रॉक्सी परिस्थितीतील Confused Deputy समस्या
  - अनिवार्य पडताळणी नियंत्रणांसह टोकन पासथ्रू असुरक्षितता
- **पुरवठा साखळी सुरक्षा**: फाउंडेशन मॉडेल्स, एम्बेडिंग सेवा, संदर्भ प्रदाते, आणि तृतीय-पक्ष API समाविष्ट करून AI पुरवठा साखळी कव्हरेज विस्तारित
- **फाउंडेशन सुरक्षा**: शून्य विश्वास आर्किटेक्चर आणि Microsoft सुरक्षा परिसंस्था यांसह एंटरप्राइझ सुरक्षा नमुन्यांचे सुधारित एकत्रिकरण
- **संसाधन संघटन**: प्रकारानुसार (अधिकृत दस्तऐवज, मानके, संशोधन, Microsoft उपाय, अंमलबजावणी मार्गदर्शक) वर्गीकृत व्यापक संसाधन दुवे

### दस्तऐवज गुणवत्ता सुधारणा
- **संरचित शिक्षण उद्दिष्टे**: विशिष्ट, क्रियाशील परिणामांसह सुधारित शिक्षण उद्दिष्टे
- **क्रॉस-रेफरन्सेस**: संबंधित सुरक्षा आणि मुख्य संकल्पना विषयांमध्ये दुवे जोडले
- **सध्याची माहिती**: सर्व तारीख संदर्भ आणि निर्दिष्टिकरण दुवे सध्याच्या मानकांनुसार अद्ययावत
- **अंमलबजावणी मार्गदर्शन**: दोन्ही विभागांमध्ये विशिष्ट, क्रियाशील अंमलबजावणी मार्गदर्शक जोडले
