- **موارد التنفيذ**: التحقق من جميع روابط وأدلة التنفيذ والموارد

#### استخدام الخادم المتقدم (03-GettingStarted/10-advanced)
- **إضافة فصل جديد**: تقديم دليل شامل لاستخدام خادم MCP المتقدم، يغطي كل من البنى الخدمية العادية والمنخفضة المستوى.
  - **الخادم العادي مقابل المنخفض المستوى**: مقارنة مفصلة وأمثلة برمجية في Python وTypeScript لكلا النهجين.
  - **تصميم قائم على المعالجات**: شرح إدارة الأدوات/الموارد/المطالبات القائمة على المعالجات لتنفيذات خادم مرنة وقابلة للتوسع.
  - **أنماط عملية**: سيناريوهات واقعية حيث تكون أنماط الخادم المنخفض المستوى مفيدة للميزات المتقدمة والهندسة المعمارية.

#### المصادقة البسيطة (03-GettingStarted/11-simple-auth)
- **إضافة فصل جديد**: دليل خطوة بخطوة لتنفيذ المصادقة البسيطة في خوادم MCP.
  - **مفاهيم المصادقة**: شرح واضح للمصادقة مقابل التفويض، والتعامل مع بيانات الاعتماد.
  - **تنفيذ المصادقة الأساسية**: أنماط مصادقة تعتمد على الوسيط في Python (Starlette) وTypeScript (Express)، مع عينات كود.
  - **التقدم نحو الأمان المتقدم**: إرشادات للبدء بالمصادقة البسيطة والتقدم إلى OAuth 2.1 وRBAC، مع مراجع لوحدات الأمان المتقدمة.

- **تحديث إصدار البروتوكول**: تحديث للإشارة إلى MCP Specification 2025-06-18 مع ترقيم تاريخي (صيغة YYYY-MM-DD)  
- **تحسين البنية المعمارية**: وصف محسن للمضيفين والعملاء والخوادم لتعكس أنماط بنية MCP الحالية  
  - تم تعريف المضيفين بوضوح كتطبيقات ذكاء اصطناعي تنسق اتصالات عملاء MCP المتعددة  
  - وصف العملاء كموصلات بروتوكول تحافظ على علاقات واحد لواحد مع الخادم  
  - تحسين الخوادم مع سيناريوهات النشر المحلية والبعيدة  
- **إعادة هيكلة البدائيات**: إعادة هيكلة كاملة لبدائيات الخادم والعميل  
  - بدائيات الخادم: الموارد (مصادر البيانات)، المطالبات (القوالب)، الأدوات (الدوال القابلة للتنفيذ) مع شروحات وأمثلة مفصلة  
  - بدائيات العميل: العينات (اكتمالات LLM)، الاستنباط (إدخال المستخدم)، التسجيل (تصحيح الأخطاء/المراقبة)  
  - تحديث أنماط طرق الاكتشاف (`*/list`)، الاسترجاع (`*/get`)، والتنفيذ (`*/call`) الحالية  
- **بنية البروتوكول**: تقديم نموذج بنية ذو طبقتين  
  - طبقة البيانات: أساس JSON-RPC 2.0 مع إدارة دورة الحياة والبدائيات  
  - طبقة النقل: STDIO (محلي) وHTTP قابل للبث مع SSE (نقل بعيد)  
- **إطار الأمان**: مبادئ أمان شاملة تشمل موافقة المستخدم الصريحة، حماية خصوصية البيانات، أمان تنفيذ الأدوات، وأمان طبقة النقل  
- **أنماط الاتصال**: تحديث رسائل البروتوكول لعرض تدفقات التهيئة، الاكتشاف، التنفيذ، والإشعارات  
- **أمثلة برمجية**: تحديث أمثلة متعددة اللغات (.NET، Java، Python، JavaScript) لتعكس أنماط SDK الحالية  

#### الأمان (02-Security/) - إعادة هيكلة أمنية شاملة  
- **توافق المعايير**: توافق كامل مع متطلبات أمان MCP Specification 2025-06-18  
- **تطور المصادقة**: توثيق التطور من خوادم OAuth مخصصة إلى تفويض مزود هوية خارجي (Microsoft Entra ID)  
- **تحليل التهديدات الخاصة بالذكاء الاصطناعي**: تغطية محسنة لمتجهات هجوم الذكاء الاصطناعي الحديثة  
  - سيناريوهات هجوم حقن المطالبات مع أمثلة من العالم الحقيقي  
  - آليات تسمم الأدوات وأنماط هجوم "سحب السجادة"  
  - تسمم نافذة السياق وهجمات تشويش النموذج  
- **حلول أمان الذكاء الاصطناعي من Microsoft**: تغطية شاملة لنظام أمان Microsoft  
  - AI Prompt Shields مع تقنيات الكشف المتقدمة، التمييز، وتقنيات الفواصل  
  - أنماط تكامل Azure Content Safety  
  - GitHub Advanced Security لحماية سلسلة التوريد  
- **تخفيف التهديدات المتقدمة**: ضوابط أمان مفصلة لـ  
  - اختطاف الجلسة مع سيناريوهات هجوم MCP ومتطلبات معرف الجلسة التشفيري  
  - مشاكل الوكيل المربك في سيناريوهات وكيل MCP مع متطلبات موافقة صريحة  
  - ثغرات تمرير الرموز مع ضوابط تحقق إلزامية  
- **أمان سلسلة التوريد**: توسيع تغطية سلسلة توريد الذكاء الاصطناعي بما في ذلك النماذج الأساسية، خدمات التضمين، مزودي السياق، وواجهات برمجة التطبيقات الخارجية  
- **أمان الأساس**: تكامل محسّن مع أنماط أمان المؤسسات بما في ذلك بنية الثقة الصفرية ونظام أمان Microsoft  
- **تنظيم الموارد**: تصنيف شامل لروابط الموارد حسب النوع (الوثائق الرسمية، المعايير، الأبحاث، حلول Microsoft، أدلة التنفيذ)
