# پیاده‌سازی عملی

[![چگونه برنامه‌های MCP را با ابزارها و جریان‌های کاری واقعی بسازیم، آزمایش کنیم و مستقر کنیم](../../../translated_images/fa/05.64bea204e25ca891.webp)](https://youtu.be/vCN9-mKBDfQ)

_(برای مشاهده ویدیوی این درس روی تصویر بالا کلیک کنید)_

پیاده‌سازی عملی جایی است که قدرت پروتکل مدل کانتکست (MCP) ملموس می‌شود. در حالی که درک نظریه و معماری پشت MCP مهم است، ارزش واقعی زمانی ظاهر می‌شود که این مفاهیم را برای ساخت، آزمایش و استقرار راه‌حل‌هایی به کار ببرید که مشکلات دنیای واقعی را حل می‌کنند. این فصل پل بین دانش مفهومی و توسعه عملی است و شما را در فرآیند به‌کارگیری برنامه‌های مبتنی بر MCP راهنمایی می‌کند.

چه شما در حال توسعه دستیاران هوشمند باشید، چه یکپارچه‌سازی هوش مصنوعی در جریان‌های کاری کسب‌وکار، یا ساخت ابزارهای سفارشی برای پردازش داده‌ها، MCP پایه‌ای انعطاف‌پذیر فراهم می‌کند. طراحی مستقل از زبان و SDKهای رسمی آن برای زبان‌های برنامه‌نویسی محبوب، آن را برای طیف گسترده‌ای از توسعه‌دهندگان در دسترس قرار می‌دهد. با بهره‌گیری از این SDKها می‌توانید سریع نمونه‌سازی، تکرار و مقیاس‌بندی راه‌حل‌های خود را در پلتفرم‌ها و محیط‌های مختلف انجام دهید.

در بخش‌های بعدی، نمونه‌های عملی، کد نمونه و استراتژی‌های استقرار را خواهید دید که نحوه پیاده‌سازی MCP در C#، جاوا با Spring، تایپ‌اسکریپت، جاوااسکریپت، و پایتون را نشان می‌دهد. همچنین خواهید آموخت چگونه سرورهای MCP خود را اشکال‌زدایی و آزمایش کنید، APIها را مدیریت کنید و راه‌حل‌ها را با استفاده از Azure به فضای ابری استقرار دهید. این منابع عملی برای تسریع یادگیری شما و کمک به ساخت برنامه‌های MCP قوی و آماده تولید طراحی شده‌اند.

## مرور کلی

این درس بر جنبه‌های عملی پیاده‌سازی MCP در زبان‌های برنامه‌نویسی مختلف تمرکز دارد. نحوه استفاده از SDKهای MCP در C#، جاوا با Spring، تایپ‌اسکریپت، جاوااسکریپت و پایتون برای ساخت برنامه‌های قوی، اشکال‌زدایی و آزمایش سرورهای MCP و خلق منابع قابل استفاده مجدد، پرامپت‌ها و ابزارها را بررسی می‌کنیم.

## اهداف یادگیری

در پایان این درس، شما قادر خواهید بود:

- پیاده‌سازی راه‌حل‌های MCP با استفاده از SDKهای رسمی در زبان‌های مختلف برنامه‌نویسی
- اشکال‌زدایی و آزمایش سیستماتیک سرورهای MCP
- ایجاد و استفاده از ویژگی‌های سرور (منابع، پرامپت‌ها و ابزارها)
- طراحی جریان‌های کاری موثر MCP برای وظایف پیچیده
- بهینه‌سازی پیاده‌سازی‌های MCP برای عملکرد و قابلیت اطمینان

## ویژگی‌های اصلی سرور

سرورهای MCP می‌توانند هر ترکیبی از این ویژگی‌ها را پیاده‌سازی کنند:

### منابع

منابع زمینه و داده را برای استفاده کاربر یا مدل هوش مصنوعی فراهم می‌کنند:

- مخازن مستندات
- پایگاه‌های دانش
- منابع داده ساختاریافته
- سیستم‌های فایل

### پرامپت‌ها

پرامپت‌ها پیام‌ها و جریان‌های کاری قالب‌بندی‌شده برای کاربران هستند:

- قالب‌های گفتگو از پیش تعریف‌شده
- الگوهای تعامل هدایت‌شده
- ساختارهای گفتگو تخصصی

### ابزارها

ابزارها توابعی هستند که مدل هوش مصنوعی می‌تواند اجرا کند:

- ابزارهای پردازش داده
- یکپارچگی با API خارجی
- قابلیت‌های محاسباتی
- قابلیت جستجو

## نمونه پیاده‌سازی: پیاده‌سازی سی‌شارپ

مخزن SDK رسمی سی‌شارپ شامل چند نمونه پیاده‌سازی است که جنبه‌های مختلف MCP را نشان می‌دهند:

- **کلاینت MCP پایه**: مثال ساده از چگونگی ایجاد کلاینت MCP و فراخوانی ابزارها
- **سرور MCP پایه**: پیاده‌سازی حداقلی سرور با ثبت ابزار پایه
- **سرور پیشرفته MCP**: سرور کامل با ثبت ابزار، احراز هویت و مدیریت خطا
- **یکپارچه‌سازی با ASP.NET**: مثال‌هایی از ادغام با ASP.NET Core
- **الگوهای پیاده‌سازی ابزار**: الگوهای مختلف برای پیاده‌سازی ابزارها با سطوح مختلف پیچیدگی

SDK سی‌شارپ MCP در مرحله پیش‌نمایش است و APIها ممکن است تغییر کنند. ما این وبلاگ را به‌طور مداوم به‌روزرسانی خواهیم کرد.

## استقرار سرور MCP راه دور در Azure

بیایید ببینیم آیا می‌توانیم نمونه‌ای که قبلاً اشاره کردیم را مستقر کنیم:

1. مخزن را کلون کنید

    ```bash
    git clone https://github.com/Azure-Samples/remote-mcp-apim-functions-python.git
    cd remote-mcp-apim-functions-python
    ```

1. ارائه‌دهنده منابع `Microsoft.App` را ثبت کنید.

   - اگر از Azure CLI استفاده می‌کنید، دستور `az provider register --namespace Microsoft.App --wait` را اجرا کنید.
   - اگر از Azure PowerShell استفاده می‌کنید، دستور `Register-AzResourceProvider -ProviderNamespace Microsoft.App` را اجرا کنید. سپس پس از مدتی با دستور `(Get-AzResourceProvider -ProviderNamespace Microsoft.App).RegistrationState` بررسی کنید که ثبت کامل شده است یا خیر.

1. این دستور [azd](https://aka.ms/azd) را اجرا کنید تا سرویس مدیریت API، اپلیکیشن فانکشن (به همراه کد) و سایر منابع Azure مورد نیاز تأمین شوند

    ```shell
    azd up
    ```

    این دستورات باید تمام منابع ابری را در Azure مستقر کنند.

### آزمایش سرور خود با MCP Inspector

1. در یک **پنجره ترمینال جدید**، MCP Inspector را نصب و اجرا کنید

    ```shell
    npx @modelcontextprotocol/inspector
    ```

    باید مشابه این رابط کاربری را ببینید:

    ![اتصال به Node inspector](../../../translated_images/fa/connect.141db0b2bd05f096.webp)

1. با کلید CTRL روی URL نمایش داده شده توسط برنامه کلیک کنید تا برنامه وب MCP Inspector باز شود (مثلاً [http://127.0.0.1:6274/#resources](http://127.0.0.1:6274/#resources))
1. نوع انتقال را روی `SSE` تنظیم کنید
1. URL نقطه انتهایی SSE مدیریت API در حال اجرای خود را که پس از اجرای `azd up` نمایش داده شده است تنظیم کرده و **اتصال** را بزنید:

    ```shell
    https://<apim-servicename-from-azd-output>.azure-api.net/mcp/sse
    ```

1. **لیست ابزارها**. روی یک ابزار کلیک کرده و **اجرای ابزار** را بزنید.

اگر همه مراحل به درستی انجام شده باشد، اکنون باید به سرور MCP متصل شده باشید و توانسته باشید یک ابزار را فراخوانی کنید.

## سرورهای MCP برای Azure

[Remote-mcp-functions](https://github.com/Azure-Samples/remote-mcp-functions-dotnet): این مجموعه مخازن قالب شروع سریعی برای ساخت و استقرار سرورهای سفارشی MCP راه دور (Model Context Protocol) با استفاده از Azure Functions با پایتون، سی‌شارپ .NET یا Node/TypeScript است.

نمونه‌ها یک راه‌حل کامل فراهم می‌کنند که به توسعه‌دهندگان اجازه می‌دهد:

- ساخت و اجرای محلی: توسعه و اشکال‌زدایی سرور MCP روی ماشین محلی
- استقرار در Azure: به راحتی با یک فرمان ساده azd up روی ابر مستقر شوند
- اتصال از کلاینت‌ها: اتصال به سرور MCP از کلاینت‌های مختلف مانند حالت عامل Copilot در VS Code و ابزار MCP Inspector

### ویژگی‌های کلیدی

- امنیت ذاتی: سرور MCP به وسیله کلیدها و HTTPS تأمین امنیت شده است
- گزینه‌های احراز هویت: پشتیبانی از OAuth با استفاده از احراز هویت داخلی و/یا مدیریت API
- ایزولاسیون شبکه: امکان ایزولاسیون شبکه با استفاده از شبکه‌های مجازی Azure (VNET)
- معماری بدون سرور: استفاده از Azure Functions برای اجرای مقیاس‌پذیر و مبتنی بر رویداد
- توسعه محلی: پشتیبانی کامل برای توسعه و اشکال‌زدایی محلی
- استقرار ساده: فرآیند استقرار ساده و مستقیم روی Azure

مخزن شامل همه فایل‌های پیکربندی لازم، کد منبع و تعاریف زیرساخت است تا سریعاً با پیاده‌سازی سرور MCP آماده تولید شروع کنید.

- [Azure Remote MCP Functions Python](https://github.com/Azure-Samples/remote-mcp-functions-python) - نمونه پیاده‌سازی MCP با استفاده از Azure Functions و پایتون

- [Azure Remote MCP Functions .NET](https://github.com/Azure-Samples/remote-mcp-functions-dotnet) - نمونه پیاده‌سازی MCP با استفاده از Azure Functions و سی‌شارپ .NET

- [Azure Remote MCP Functions Node/Typescript](https://github.com/Azure-Samples/remote-mcp-functions-typescript) - نمونه پیاده‌سازی MCP با استفاده از Azure Functions و Node/TypeScript.

## نکات کلیدی

- SDKهای MCP ابزارهای خاص زبان برای پیاده‌سازی راه‌حل‌های قدرتمند MCP فراهم می‌کنند
- فرآیند اشکال‌زدایی و آزمایش برای برنامه‌های MCP قابل اطمینان حیاتی است
- قالب‌های پرامپت قابل استفاده مجدد تعاملات یکسان با هوش مصنوعی را ممکن می‌سازند
- جریان‌های کاری خوب طراحی شده می‌توانند وظایف پیچیده را با استفاده از چندین ابزار هماهنگ کنند
- پیاده‌سازی راه‌حل‌های MCP نیازمند توجه به امنیت، عملکرد و مدیریت خطا است

## تمرین

یک جریان کاری عملی MCP طراحی کنید که مشکلی واقعی در حوزه خودتان را حل کند:

1. سه تا چهار ابزار که برای حل این مشکل مفید هستند را شناسایی کنید
2. یک نمودار جریان کاری ایجاد کنید که نشان دهد چگونه این ابزارها با هم تعامل دارند
3. نسخه پایه‌ای از یکی از ابزارها را با زبان مورد علاقه خود پیاده‌سازی کنید
4. یک قالب پرامپت بسازید که به مدل کمک کند ابزار شما را به‌طور مؤثر استفاده کند
