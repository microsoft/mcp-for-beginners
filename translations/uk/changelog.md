- **Додано нову главу**: Введено комплексний посібник з розширеного використання MCP сервера, що охоплює як звичайну, так і низькорівневу серверну архітектуру.
  - **Звичайний vs. низькорівневий сервер**: Детальне порівняння та приклади коду на Python і TypeScript для обох підходів.
  - **Дизайн на основі обробників**: Пояснення управління інструментами/ресурсами/підказками на основі обробників для масштабованих, гнучких серверних реалізацій.
  - **Практичні шаблони**: Реальні сценарії, де шаблони низькорівневого сервера корисні для розширених функцій та архітектури.

#### Проста автентифікація (03-GettingStarted/11-simple-auth)
- **Додано нову главу**: Покроковий посібник з реалізації простої автентифікації в MCP серверах.
  - **Концепції автентифікації**: Чітке пояснення різниці між автентифікацією та авторизацією, а також обробки облікових даних.
  - **Реалізація базової автентифікації**: Шаблони автентифікації на основі middleware у Python (Starlette) та TypeScript (Express) з прикладами коду.
  - **Прогрес до розширеної безпеки**: Рекомендації щодо початку з простої автентифікації та переходу до OAuth 2.1 і RBAC з посиланнями на розширені модулі безпеки.

Ці доповнення надають практичні, практичні рекомендації для створення більш надійних, безпечних і гнучких реалізацій MCP серверів, поєднуючи базові концепції з розширеними виробничими шаблонами.

#### Основні концепції (01-CoreConcepts/) - Повна модернізація
- **Оновлення версії протоколу**: Оновлено посилання на поточну MCP Specification 2025-06-18 з версіонуванням за датою (формат РРРР-ММ-ДД)
- **Уточнення архітектури**: Покращені описи хостів, клієнтів та серверів для відображення поточних патернів архітектури MCP
  - Хости тепер чітко визначені як AI-додатки, що координують кілька підключень клієнтів MCP
  - Клієнти описані як протокольні конектори, що підтримують відносини один-до-одного з серверами
  - Сервери покращені з урахуванням локального та віддаленого розгортання
- **Перебудова примітивів**: Повне оновлення примітивів сервера та клієнта
  - Примітиви сервера: Ресурси (джерела даних), Підказки (шаблони), Інструменти (виконувані функції) з детальними поясненнями та прикладами
  - Примітиви клієнта: Вибірка (LLM completions), Збір (ввід користувача), Логування (налагодження/моніторинг)
  - Оновлено з урахуванням поточних патернів методів discovery (`*/list`), отримання (`*/get`) та виконання (`*/call`)
- **Архітектура протоколу**: Запроваджено двошарову модель архітектури
  - Шар даних: основа JSON-RPC 2.0 з управлінням життєвим циклом та примітивами
  - Транспортний шар: STDIO (локальний) та Streamable HTTP з SSE (віддалений) транспортні механізми
- **Фреймворк безпеки**: Комплексні принципи безпеки, включно з явною згодою користувача, захистом приватності даних, безпекою виконання інструментів та безпекою транспортного шару
- **Патерни комунікації**: Оновлені повідомлення протоколу для демонстрації ініціалізації, виявлення, виконання та потоків сповіщень
- **Приклади коду**: Оновлені багатомовні приклади (.NET, Java, Python, JavaScript) для відображення поточних патернів MCP SDK

#### Безпека (02-Security/) - Комплексне оновлення безпеки  
- **Відповідність стандартам**: Повна відповідність вимогам безпеки MCP Specification 2025-06-18
- **Еволюція автентифікації**: Документовано еволюцію від власних OAuth серверів до делегації зовнішнього провайдера ідентичності (Microsoft Entra ID)

  - Детальні сценарії атак ін’єкції підказок з реальними прикладами
  - Механізми отруєння інструментів та патерни атак "rug pull"
  - Отруєння контекстного вікна та атаки на плутанину моделей
- **Рішення безпеки Microsoft AI**: Комплексне охоплення екосистеми безпеки Microsoft
  - AI Prompt Shields з розширеним виявленням, підсвічуванням та техніками розділення
  - Патерни інтеграції Azure Content Safety
  - GitHub Advanced Security для захисту ланцюга постачання
- **Розширене пом’якшення загроз**: Детальні контролі безпеки для
  - Викрадення сесій з MCP-специфічними сценаріями атак та криптографічними вимогами до ID сесії
  - Проблеми Confused Deputy у сценаріях проксі MCP з явними вимогами згоди
  - Вразливості passthrough токенів з обов’язковими контролями валідації
- **Безпека ланцюга постачання**: Розширене охоплення AI ланцюга постачання, включно з фундаментальними моделями, сервісами embeddings, провайдерами контексту та сторонніми API
- **Безпека основи**: Покращена інтеграція з корпоративними патернами безпеки, включно з архітектурою нульової довіри та екосистемою безпеки Microsoft
- **Організація ресурсів**: Категоризовані комплексні посилання на ресурси за типом (Офіційна документація, Стандарти, Дослідження, Рішення Microsoft, Посібники з впровадження)
