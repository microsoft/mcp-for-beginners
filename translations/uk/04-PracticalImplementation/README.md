# Практична реалізація

[![How to Build, Test, and Deploy MCP Apps with Real Tools and Workflows](../../../translated_images/uk/05.64bea204e25ca891.webp)](https://youtu.be/vCN9-mKBDfQ)

_(Натисніть на зображення вище, щоб переглянути відео цього уроку)_

Практична реалізація – це той момент, коли міць Протоколу Контексту Моделі (MCP) стає відчутною. Хоча розуміння теорії та архітектури MCP важливе, справжня цінність виникає, коли ви застосовуєте ці концепції для створення, тестування та розгортання рішень, які вирішують реальні проблеми. Цей розділ долає розрив між концептуальними знаннями та практичною розробкою, керуючи вами через процес втілення додатків на основі MCP.

Чи розробляєте ви інтелектуальних помічників, інтегруєте ШІ у бізнес-процеси, чи створюєте спеціальні інструменти для обробки даних, MCP надає гнучку основу. Його незалежний від мови дизайн та офіційні SDK для популярних мов програмування роблять його доступним для широкого кола розробників. Використовуючи ці SDK, ви можете швидко прототипувати, ітерувати та масштабувати свої рішення на різних платформах і середовищах.

У наступних розділах ви знайдете практичні приклади, зразки коду та стратегії розгортання, які демонструють, як реалізувати MCP на C#, Java з Spring, TypeScript, JavaScript та Python. Ви також навчитеся налагоджувати та тестувати сервери MCP, керувати API і розгортати рішення у хмарі за допомогою Azure. Ці практичні ресурси створені для прискорення вашого навчання та допоможуть вам впевнено створювати надійні, готові до продуктивного використання застосунки MCP.

## Огляд

Цей урок зосереджений на практичних аспектах реалізації MCP у кількох мовах програмування. Ми розглянемо, як використовувати MCP SDK на C#, Java з Spring, TypeScript, JavaScript та Python для створення надійних додатків, налагодження та тестування серверів MCP, а також створення повторно використовуваних ресурсів, підказок і інструментів.

## Цілі навчання

До кінця уроку ви зможете:

- Реалізувати рішення MCP за допомогою офіційних SDK на різних мовах програмування
- Систематично налагоджувати та тестувати сервери MCP
- Створювати та використовувати серверні функції (Ресурси, Підказки та Інструменти)
- Проєктувати ефективні процеси MCP для складних завдань
- Оптимізувати реалізації MCP для продуктивності та надійності

## Основні функції сервера

Сервери MCP можуть реалізовувати будь-яку комбінацію цих функцій:

### Ресурси

Ресурси надають контекст і дані для користувача або моделі ШІ:

- Репозиторії документів
- Бази знань
- Структуровані джерела даних
- Файлові системи

### Підказки

Підказки — це шаблонні повідомлення та робочі процеси для користувачів:

- Попередньо визначені шаблони розмов
- Направлені патерни взаємодії
- Спеціалізовані структури діалогів

### Інструменти

Інструменти — це функції для виконання моделлю ШІ:

- Утиліти для обробки даних
- Інтеграції зовнішніх API
- Обчислювальні можливості
- Функції пошуку

## Приклади реалізації: Реалізація на C#

Офіційний репозиторій SDK на C# містить кілька прикладів реалізацій, що демонструють різні аспекти MCP:

- **Базовий MCP клієнт**: Простий приклад створення MCP клієнта та виклику інструментів
- **Базовий MCP сервер**: Мінімальна реалізація сервера з базовою реєстрацією інструментів
- **Розширений MCP сервер**: Повнофункціональний сервер із реєстрацією інструментів, автентифікацією та обробкою помилок
- **Інтеграція з ASP.NET**: Приклади інтеграції з ASP.NET Core
- **Патерни реалізації інструментів**: Різні патерни для реалізації інструментів різної складності

MCP C# SDK знаходиться у прев’ю і API можуть змінюватись. Ми постійно оновлюватимемо цей блог разом із розвитком SDK.

### Основні функції

- [C# MCP Nuget ModelContextProtocol](https://www.nuget.org/packages/ModelContextProtocol)
- Створення вашого [першого MCP сервера](https://devblogs.microsoft.com/dotnet/build-a-model-context-protocol-mcp-server-in-csharp/).

Для повних зразків реалізації на C# відвідайте [офіційний репозиторій зразків C# SDK](https://github.com/modelcontextprotocol/csharp-sdk)

## Розгортання віддаленого MCP сервера в Azure

Перевіримо, чи можемо ми розгорнути згаданий раніше зразок:

1. Клонуйте репозиторій

    ```bash
    git clone https://github.com/Azure-Samples/remote-mcp-apim-functions-python.git
    cd remote-mcp-apim-functions-python
    ```

1. Зареєструйте провайдера ресурсів `Microsoft.App`.

   - Якщо ви використовуєте Azure CLI, виконайте команду `az provider register --namespace Microsoft.App --wait`.
   - Якщо ви використовуєте Azure PowerShell, виконайте `Register-AzResourceProvider -ProviderNamespace Microsoft.App`. Потім через деякий час перевірте стан реєстрації командою `(Get-AzResourceProvider -ProviderNamespace Microsoft.App).RegistrationState`.

1. Виконайте цю команду [azd](https://aka.ms/azd), щоб розгорнути сервіс управління API, функціональний додаток (із кодом) та всі інші необхідні ресурси Azure

    ```shell
    azd up
    ```

    Ця команда має розгорнути всі хмарні ресурси в Azure

### Тестування сервера за допомогою MCP Inspector

1. Відкрийте **нове вікно терміналу**, інсталюйте та запустіть MCP Inspector

    ```shell
    npx @modelcontextprotocol/inspector
    ```

    Ви маєте побачити інтерфейс, подібний до:

    ![Connect to Node inspector](../../../translated_images/uk/connect.141db0b2bd05f096.webp)

1. CTRL-клікніть, щоб відкрити веб-застосунок MCP Inspector за URL, який відобразить додаток (наприклад, [http://127.0.0.1:6274/#resources](http://127.0.0.1:6274/#resources))
1. Виберіть тип транспорту `SSE`
1. Введіть URL свого запущеного API Management SSE кінцевого пункту, який відображається після виконання `azd up`, та **Підключіться**:

    ```shell
    https://<apim-servicename-from-azd-output>.azure-api.net/mcp/sse
    ```

1. **Список інструментів**. Клікніть на інструмент і натисніть **Запустити інструмент**.  

Якщо всі кроки виконані правильно, ви маєте бути підключені до MCP сервера і могли викликати інструмент.

## MCP сервери для Azure

[Remote-mcp-functions](https://github.com/Azure-Samples/remote-mcp-functions-dotnet): Ця серія репозиторіїв є шаблонами швидкого старту для побудови та розгортання користувацьких віддалених MCP (Протокол Контексту Моделі) серверів за допомогою Azure Functions з Python, C# .NET або Node/TypeScript.

Зразки надають комплексне рішення, що дозволяє розробникам:

- Створювати та запускати локально: Розробляти і налагоджувати MCP сервер на локальній машині
- Розгортати в Azure: Легко розгортати у хмарі за допомогою простої команди azd up
- Підключатися з клієнтів: Підключатися до MCP сервера з різних клієнтів, включно з режимом агента Copilot у VS Code та інструментом MCP Inspector

### Основні функції

- Безпека за замовчуванням: MCP сервер захищений за допомогою ключів та HTTPS
- Варіанти аутентифікації: Підтримує OAuth із вбудованою автентифікацією та/або API Management
- Ізоляція мережі: Дозволяє мережеву ізоляцію за допомогою Azure Virtual Networks (VNET)
- Безсерверна архітектура: Використовує Azure Functions для масштабованого, подієво орієнтованого виконання
- Локальна розробка: Повна підтримка локальної розробки та налагодження
- Просте розгортання: Спрощений процес розгортання в Azure

## Основні висновки

- SDK MCP надають мовно-специфічні інструменти для реалізації надійних рішень MCP
- Процес налагодження і тестування критичний для надійних застосунків MCP
- Повторно використовувані шаблони підказок забезпечують послідовну взаємодію з ШІ
- Добре спроєктовані робочі процеси можуть оркеструвати складні завдання за допомогою кількох інструментів
- Реалізація рішень MCP вимагає врахування безпеки, продуктивності та обробки помилок

## Вправа

Спроєктуйте практичний робочий процес MCP, який розв’язує реальну проблему у вашій сфері:

1. Визначте 3-4 інструменти, які були б корисними для розв’язання цієї проблеми
2. Створіть діаграму робочого процесу, що показує, як ці інструменти взаємодіють
3. Реалізуйте базову версію одного з інструментів, використовуючи вашу улюблену мову програмування
4. Створіть шаблон підказки, що допоможе моделі ефективно використовувати ваш інструмент
