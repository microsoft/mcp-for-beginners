# ప్రాక్టికల్ అమలు

[![వాస్తవ సాంకేతికాలతో, పనివిధానాలతో MCP యాప్స్‌ను ఎలా నిర్మించాలి, పరీక్షించాలి, మరియు అమలు చేయాలి](../../../translated_images/te/05.64bea204e25ca891.webp)](https://youtu.be/vCN9-mKBDfQ)

_(ఈ పాఠం వీడియోను వీక్షించడానికి పై చిత్రం పై క్లిక్ చేయండి)_

ప్రాక్టికల్ అమలు అనగా మోడల్ కాన్టెక్ట్ ప్రోటోకాల్ (MCP) శక్తిని స్పష్టంగా ఒకటిగా చూడటం. MCP వెనుక ఉన్న సిద్ధాంతం మరియు నమూనా పిలవడాన్ని అర్థం చేసుకోవడం ముఖ్యమైన విషయం అయినా, నిజమైన విలువ మీరు ఈ భావనలను ఉపయోగించి నిజజీవిత సమస్యలను పరిష్కరించే పరిష్కారాలను నిర్మించడం, పరీక్షించడం, మరియు అమలు చేయడంలో వస్తుంది. ఈ అధ్యాయం సైద్ధాంతిక జ్ఞానం మరియు హ్యాండ్స్-ఆన్ డెవలప్‌మెంట్ మధ్య లోకాన్ని తగ్గిస్తుంది, MCP ఆధారిత అప్లికేషన్లను ప్రాణాంతకం చేయడానికి ప్రాసెస్ ద్వారా మీరు నడిపిస్తుంది.

మీరు తెలివైన సహాయకులను అభివృద్ధి చేస్తున్నారా, వ్యాపార పని ప్రవాహాల్లో AIని అనుసంధానం చేస్తున్నారా, లేక డేటా ప్రాసెసింగ్ కోసం కస్టమ్ సాధనాలను నిర్మిస్తున్నారా అనేదానిపై సంబంధం లేకుండా, MCP ఒక అనుకూలమైన మౌలికాన్ని అందిస్తుంది. దాని భాష-ఏజన్ల డిజైన్ మరియు ప్రముఖ ప్రోగ్రామింగ్ భాషల కోసం అధికారిక SDKలు విస్తృత పరిధిలో ఉన్న డెవలపర్లకు అందుబాటులో ఉంటాయి. ఈ SDKలను ఉపయోగించి, మీరు త్వరగా ప్రోటోటైప్లు చేయవచ్చు, పునర్‌సమీక్షించవచ్చు, మరియు విభిన్న ప్లాట్‌ఫారాల ద్వారా మీ పరిష్కారాలను పెంచవచ్చు.

తర్వాతి విభాగాలలో, మీరు ప్రాక్టికల్ ఉదాహరణలు, నమూనా కోడ్, మరియు అమలు వ్యూహాలను కనుగొంటారు, ఇవి C#, Java విత్ Spring, TypeScript, JavaScript, మరియు Pythonలో MCPని ఎలా అమలు చేయాలో చూపిస్తాయి. మీరు మీ MCP సర్వర్లను డీబగ్ చేయడం మరియు పరీక్షించడం, APIలను నిర్వహించడం, మరియు Azure ఉపయోగించి క్లౌడ్‌కు పరిష్కారాలను అమలు చేయడం ఎలా చేయాలో కూడా నేర్చుకుంటారు. ఈ హ్యాండ్స్-ఆన్ వనరులు మీ నేర్చుకున్నత విధానాన్ని వేగవంతం చేసి, మీరు నమ్మకంగా దృఢత, ప్రొడక్షన్-రెడీ MCP అప్లికేషన్లను నిర్మించడంలో సహాయపడతాయి.

## అవలోకనం

ఈ పాఠం ఒకటి MCP అమలులో ప్రాక్టికల్ అంశాలపై దృష్టి సారిస్తుంది, పలు ప్రోగ్రామింగ్ భాషలలో. మనం C#, Java విత్ Spring, TypeScript, JavaScript, మరియు Python లో MCP SDKలను ఉపయోగించి బలమైన అప్లికేషన్లను ఎలా నిర్మించాలో, MCP సర్వర్లను డీబగ్ మరియు పరీక్షించాలో, మరియు పునర్వినియోగ వనరులు, ప్రాంప్ట్లు, మరియు సాధనాలు ఎలా సృష్టించాలో పరిశీలిస్తాము.

## నేర్చుకునే లక్ష్యాలు

ఈ పాఠం ముగించగానే, మీరు చేయగలుగుతారు:

- వివిధ ప్రోగ్రామింగ్ భాషలలో అధికారిక SDKలు ఉపయోగించి MCP పరిష్కారాలను అమలు చేయడం
- MCP సర్వర్లను యథావిధిగా డీబగ్ చేసి, పరీక్షించడం
- సర్వర్ বৈశిష్ట్యాలు సృష్టించడం మరియు ఉపయోగించడం (వనరులు, ప్రాంప్ట్లు, సాధనలు)
- సంక్లిష్ట పనులకు సమర్థవంతమైన MCP పని ప్రవాహాలను రూపకల్పన చేయడం
- పనితన మరియు విశ్వసనీయత కోసం MCP అమలును ఆప్టిమైజ్ చేయడం

## కోర్ సర్వర్ বৈశిష్ట్యాలు

MCP సర్వర్లు ఈ বৈశిష్ట్యాల ఏయైనా సంయోజనాన్ని అమలు చేయవచ్చు:

### వనరులు

వనరులు వినియోగదారుని లేదా AI నమూనాకు ఉపయోగించడానికి సందర్భం మరియు డేటాను అందించును:

- డాక్యుమెంట్ రిపాజిటరీలు
- జ్ఞానాభివృద్ధి కేంద్రాలు
- నిర్మాణాత్మక డేటా మూలాలు
- ఫైల్ సిస్టమ్స్

### ప్రాంప్ట్లు

ప్రాంప్ట్లు వినియోగదారులకు నిర్వచించిన సందేశాలు మరియు పని ప్రవాహాల టెంప్లెట్లు:

- ముందుగా నిర్వచిత సంభాషణ నమూనాలు
- మార్గనిర్దేశక పరస్పర క్రియాకలాప నమూనాలు
- విశిష్ట డయలాగ్ నిర్మాణాలు

### సాధనాలు

సాధనాలు AI నమూనా అమలు చేసే కార్యాలు:

- డేటా ప్రాసెసింగ్ ఉపకరణాలు
- బయటి API అనుసంధానాలు
- గణనీయ శక్తులు
- శోధన సామర్ధ్యం

## నమూనా అమలు: C# అమలు

అధికారిక C# SDK రీపొసిటరీలో MCP వివిధ అంశాలను చూపించే అనేక నమూనాలు ఉన్నాయి:

- **ప్రాథమిక MCP క్లయింట్**: MCP క్లయింట్‌ను సృష్టించి సాధనాలను పిలవడం ఎలా అనేది చూపించే సాదాసీదా ఉదాహరణ
- **ప్రాథమిక MCP సర్వర్**: ప్రాథమిక సాధన రిజిస్ట్రేషన్‌తో కనిష్ట సర్వర్ అమలు
- **అత్యధిక MCP సర్వర్**: సాధన రిజిస్ట్రేషన్, ప్రమాణీకరణ, మరియు లోప నిర్వహణతో పూర్తి స్వరూప సర్వర్
- **ASP.NET అనుసంధానం**: ASP.NET కోర్‌తో అనుసంధానాన్ని చూపించే ఉదాహరణలు
- **సాధన అమలు నమూనాలు**: వివిధ క్లిష్టత స్థాయిలతో సాధనాలను అమలు చేయడానికి నమూనాలు

MCP C# SDK ప్రివ్యూ స్థితిలో ఉంది మరియు APIs మారవచ్చు. SDK అభివృద్ధి ఫాలో అవుతూ మనం ఈ బ్లాగ్‌ను నిరంతరం నవీకరిస్తాము.

### ముఖ్య বৈశిష్ట్యాలు

- [C# MCP Nuget ModelContextProtocol](https://www.nuget.org/packages/ModelContextProtocol)
- మీ [మొదటి MCP సర్వర్](https://devblogs.microsoft.com/dotnet/build-a-model-context-protocol-mcp-server-in-csharp/) ను నిర్మించడం.

సంపూర్ణ C# అమలు నమూనాల కోసం, అధికారిక C# SDK నమూనాలు రీపొసిటరీని చూడండి: [official C# SDK samples repository](https://github.com/modelcontextprotocol/csharp-sdk)

## API నిర్వహణ

Azure API నిర్వహణ MCP సర్వర్లను ఎలా భద్రపరచాలో ఒక గొప్ప సమాధానం. MCP సర్వర్ ముందు ఒక Azure API నిర్వహణ సర్వీస్‌ను ఉంచి మీరు కోరుకోగల ఫీచర్లను నిర్వహించబడతాయి:

- రేటు పరిమితి
- టోకెన్ నిర్వహణ
- పర్యవేక్షణ
- లోడ్ బ్యాలెన్సింగ్
- భద్రత

### Azure నమూనా

కింది Azure నమూనా ఇదే చేస్తుంది, అంటే [MCP సర్వర్ సృష్టించి Azure API నిర్వహణతో భద్రపరచడం](https://github.com/Azure-Samples/remote-mcp-apim-functions-python).

క్రింద చిత్రంలో అధికారం ప్రవాహం ఎలా జరుగుతుందో చూడండి:

![APIM-MCP](https://github.com/Azure-Samples/remote-mcp-apim-functions-python/blob/main/mcp-client-authorization.gif?raw=true)

ముందు ఉన్న చిత్రంలో, ఈ క్రింది చర్యలు జరుగుతాయి:

- Microsoft Entra ఉపయోగించి ప్రమాణీకరణ/అధికారం జరుగుతుంది.
- Azure API నిర్వహణ గేట్వేగా పనిచేస్తుంది, మరియు పాలసీలను ఉపయోగించి రవాణాను నిర్వహిస్తుంది మరియు పర్యవేక్షిస్తుంది.
- Azure మానిటర్ అన్ని అభ్యర్థనలను దరఖాస్తు చేస్తుంది తదుపరి విశ్లేషణకు.

## రిమోట్ MCP సర్వర్‌ను Azureలో అమలు చేయండి

ముందుగా పేర్కొన్న నమూనాను అమలు చేయగలమా చూద్దాం:

1. రీపొసిటరీని క్లోన్ చేయండి

    ```bash
    git clone https://github.com/Azure-Samples/remote-mcp-apim-functions-python.git
    cd remote-mcp-apim-functions-python
    ```

2. `Microsoft.App` వనరు ప్రొవైడర్‌ని నమోదు చేయండి.

   - మీరు Azure CLI వాడితే, `az provider register --namespace Microsoft.App --wait` ను నడపండి.
   - మీరు Azure PowerShell వాడితే, `Register-AzResourceProvider -ProviderNamespace Microsoft.App` ను నడపండి. కొంత సమయం తర్వాత `(Get-AzResourceProvider -ProviderNamespace Microsoft.App).RegistrationState` ను ఉపయోగించి నమోదు పూర్తయిందా అని తనిఖీ చేయండి.

3. ఈ [azd](https://aka.ms/azd) ఆదేశం అమలు చేయండి, ఇది API నిర్వహణ సేవ, ఫంక్షన్ యాప్ (కోడ్‌తో), మరియు ఇతర అవసరమైన Azure వనరులను ప్రొవిజన్ చేస్తుంది.

    ```shell
    azd up
    ```

    ఈ ఆదేశం Azureలో సమస్త క్లౌడ్ వనరులను అమలు చేయాలి.

### MCP ఇన్‌స్పెక్టర్‌తో మీ సర్వర్‌ను పరీక్షించడం

1. **కొత్త టెర్మినల్ విండో లో**, MCP ఇన్‌స్పెక్టర్ ఇన్స్టాల్ చేసి నడిపించండి

    ```shell
    npx @modelcontextprotocol/inspector
    ```

    మీరు ఇలాంటి ఇంటర్ఫేస్ చూడగలుగుతారు:

    ![Connect to Node inspector](../../../translated_images/te/connect.141db0b2bd05f096.webp)

2. MCP ఇన్‌స్పెక్టర్ వెబ్ యాప్‌ను URL నుండి CTRL క్లిక్ చేసి ఓపెన్ చేయండి (ఉదాహరణకి: [http://127.0.0.1:6274/#resources](http://127.0.0.1:6274/#resources))
3. ట్రాన్స్పోర్ట్ రకం `SSE` గా సెట్ చేయండి
4. `azd up` తర్వాత చూపే మీ API నిర్వహణ SSE ఎండ్‌పాయింట్ URLను సెట్ చేసి **Connect** చేసుకోండి:

    ```shell
    https://<apim-servicename-from-azd-output>.azure-api.net/mcp/sse
    ```

5. **సాధనాల జాబితా చూపించండి**. ఒక సాధనాన్ని క్లిక్ చేసి **రన్ టూల్** చేయండి.

అన్ని దశలు సరిగా జరిగితే, మీరు MCP సర్వర్‌తో కనెక్ట్ అయ్యారు మరియు ఒక సాధనాన్ని పిలవగలిగారు.

## Azure కోసం MCP సర్వర్లు

[Remote-mcp-functions](https://github.com/Azure-Samples/remote-mcp-functions-dotnet): ఈ రీపొసిటరీలు Azure Functions ఉపయోగించి Python, C# .NET లేదా Node/TypeScriptతో కస్టమ్ రిమోట్ MCP (Model Context Protocol) సర్వర్లను నిర్మించడానికి మరియు అమలు చేయడానికి క్విక్వ్ స్టార్ట్ టెంప్లేట్.

నమూనాలు డెవలపర్లకి పూర్తి పరిష్కారాన్ని అందిస్తాయి, వీటితో:

- స్థానికంగా నిర్మించండి మరియు నడపండి: స్థానిక యంత్రంలో MCP సర్వర్ డెవలప్ చేసి డీబగ్ చేయండి
- Azureలో అమలు చేయండి: సులభంగా క్లౌడ్‌లో azd up ఆదేశంతో అమలు చేయండి
- క్లయింట్ల నుంచి కనెక్ట్ అవ్వండి: వివిధ క్లయింట్ల నుండి MCP సర్వర్‌తో కనెక్ట్ అవ్వండి, వీటిలో VS కోడ్ యొక్క Copilot ఏజెంట్ మోడ్ మరియు MCP ఇన్‌స్పెక్టర్ టూల్ ఉన్నాయి

### ముఖ్య বৈశిష్ట్యాలు

- డిజైన్లో భద్రత: MCP సర్వర్ కీలు మరియు HTTPS ఉపయోగించి భద్రపరచబడుతుంది
- ప్రమాణీకరణ ఎంపికలు: ఆటోమేటిక్ OAuth, మరియు/లేదా API నిర్వహణ ఉపయోగించు మద్దతు
- నెట్‌వర్క్ వేరుపాట్లు: Azure వర్చువల్ నెట్‌వర్క్‌లు (VNET) ఉపయోగించి నెట్‌వర్క్ వేరుపాట్లను అనుమతిస్తుంది
- సర్వర్‌లెస్ వాస్తవికరణ: స్కేల్ అయ్యే, ఈవెంట్ ఆధారిత ఎగ్జిక్యుషన్ కోసం Azure Functions ఉపయోగిస్తుంది
- స్థానిక అభివృద్ధి: సమగ్ర స్థానిక అభివృద్ధి మరియు డీబగ్గింగ్ మద్దతు
- సాదాసీదా అమలు: Azureకి సులభమైన డిప్లాయ్‌మెంట్ ప్రాసెస్

ఈ రీపొసిటరీ సమస్త అవసరంల వనరులు, మూల కోడ్, మరియు మౌలిక సదుపాయ నిర్వచనాలను ఉపయోగించి ప్రొడక్షన్-రెడీ MCP సర్వర్ అమలుకు త్వరగా ప్రారంభించేలా చేస్తుంది.

- [Azure Remote MCP Functions Python](https://github.com/Azure-Samples/remote-mcp-functions-python) - Python తో Azure Functions ఉపయోగించి MCP అమలు నమూనా

- [Azure Remote MCP Functions .NET](https://github.com/Azure-Samples/remote-mcp-functions-dotnet) - C# .NET తో Azure Functions ఉపయోగించి MCP అమలు నమూనా

- [Azure Remote MCP Functions Node/Typescript](https://github.com/Azure-Samples/remote-mcp-functions-typescript) - Node/TypeScript తో Azure Functions ఉపయోగించి MCP అమలు నమూనా

## ముఖ్య ముచ్చట్లు

- MCP SDKలు భాషా-విశిష్ట సాధనాలను సరఫరా చేస్తాయి, గట్టి MCP పరిష్కారాలను అమలు చేయడానికి
- డీబగ్గింగ్ మరియు పరీక్షించే ప్రక్రియ విశ్వసనీయ MCP అప్లికేషన్ల కోసం అత్యవసరం
- పునర్వినియోగ ప్రాంప్ట్ టెంప్లేట్లు అనుకూల AI పరస్పర చర్యలకు సహాయపడతాయి
- బాగా రూపకల్పన చేసిన పని ప్రవాహాలు అనేక సాధనాలను ఉపయోగించి సంక్లిష్ట పనులను సమన్వయించగలవు
- MCP పరిష్కారాలను అమలు చేయడంలో భద్రత, పనితనం, లోప నిర్వహణ విషయాలను పరిగణలోకి తీసుకోవాలి

## వ్యాయామం

మీ డొమైన్‌లో నిజజీవిత సమస్యను పరిష్కరించే ఒక ప్రాక్టికల్ MCP పని ప్రవాహాన్ని రూపకల్పన చేయండి:

1. ఈ సమస్య పరిష్కారం కోసం ఉపయోగకరమైన 3-4 సాధనాలను గుర్తించండి
2. ఈ సాధనాలు ఎలా పరస్పర చర్యలు చూపుతాయో పని ప్రవాహ చార్ట్ సృష్టించండి
3. మీ ఇష్ట భాషలో ఒక సాధన యొక్క ప్రాథమిక వెర్షన్‌ను అమలు చేయండి
4. నమూనాకు మీ సాధనను సమర్థవంతంగా ఉపయోగించడానికి సహాయపడే ప్రాంప్ట్ టెంప్లేట్ సృష్టించండి
