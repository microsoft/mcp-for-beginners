# MCP ಸರ್ವರ್ ಗಳು ನಿಯೋಜಿಸುವುದು

ನಿಮ್ಮ MCP ಸರ್ವರ್ ಅನ್ನು ನಿಯೋಜಿಸುವುದರಿಂದ ಇತರರು ಅದರ ಸಾಧನಗಳು ಮತ್ತು ಸಂಪನ್ಮೂಲಗಳನ್ನು ನಿಮ್ಮ ಸ್ಥಳೀಯ ಪರಿಸರ ಮೀರಿ ಪ್ರವೇಶಿಸಬಹುದು. ನಿಮ್ಮ ಪ್ರಮಾಣದಾಯಕತೆ, ವಿಶ್ವಾಸಾರ್ಹತೆ ಮತ್ತು ನಿರ್ವಹಣೆಯ ಸುಲಭತೆಯ ಅವಶ್ಯಕತೆಗಳನ್ನು ಆಧರಿಸಿ ಹಲವಾರು ನಿಯೋಜನ ತಂತ್ರಗಳನ್ನು ಪರಿಗಣಿಸಬಹುದು. ಕೆಳಗೆ ನೀವು MCP ಸರ್ವರ್ ಗಳನ್ನು ಸ್ಥಳೀಯವಾಗಿ, ಕಾಂಟೈನರ್ ಗಳಲ್ಲಿ ಮತ್ತು ಕ್ಲೌಡ್ ಗೆ ನಿಯೋಜಿಸುವಿಕೆಗಾಗಿ ಮಾರ್ಗದರ್ಶನ ಕಂಡುಹಿಡಿಯುವಿರಿ.

## ಸಾರಾಂಶ

ಈ ಪಾಠವು ನಿಮ್ಮ MCP ಸರ್ವರ್ ಆಪ್ ಅನ್ನು ಹೇಗೆ ನಿಯೋಜಿಸುವುದನ್ನು ಒಳಗೊಂಡಿದೆ.

## ಅಧ್ಯಯನ ಉದ್ದೇಶಗಳು

ಈ ಪಾಠದ ಅಂತ್ಯದವರೆಗೆ, ನೀವು ಸಾಧ್ಯವಾಗುವುದು:

- ವಿಭಿನ್ನ ನಿಯೋಜನೆ ವಿಧಾನಗಳನ್ನು ಮೌಲ್ಯಮಾಪನ ಮಾಡುವುದು.
- ನಿಮ್ಮ ಆಪ್ ಅನ್ನು ನಿಯೋಜಿಸುವುದು.

## ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿ ಮತ್ತು ನಿಯೋಜನೆ

ನಿಮ್ಮ ಸರ್ವರ್ ಬಳಕೆದಾರರ ಯಂತ್ರದಲ್ಲಿ ಕಾರ್ಯನಿರ್ವಹಿಸುವ ಉದ್ದೇಶವಿದ್ದರೆ, ನೀವು ಕೆಳಗಿನ ಹಂತಗಳನ್ನು ಅನುಸರಿಸಬಹುದು:

1. **ಸರ್ವರ್ ಅನ್ನು ಡೌನ್‌ಲೋಡ್ ಮಾಡಿ**. ನೀವು ಸರ್ವರ್ ರಚಿಸದಿದ್ದರೆ, ಮೊದಲು ಅದನ್ನು ನಿಮ್ಮ ಯಂತ್ರಕ್ಕೆ ಡೌನ್‌ಲೋಡ್ ಮಾಡಿಕೊಳ್ಳಿ.
1. **ಸರ್ವರ್ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಪ್ರಾರಂಭಿಸಿ**: ನಿಮ್ಮ MCP ಸರ್ವರ್ ಆಪ್ ಅನ್ನು ರನ್ ಮಾಡಿ

SSE ಗೆ (stdio ಬಗೆಯ ಸರ್ವರ್ ಗಾಗಿ ಅಗತ್ಯವಿಲ್ಲ)

1. **ನೆಟ್‌ವರ್ಕಿಂಗ್ ಅನ್ನು ಸಂರಚಿಸಿ**: ಸರ್ವರ್ ನಿರೀಕ್ಷಿತ ಪೋರ್ಟ್ನಲ್ಲಿ ಲಭ್ಯವಾಗುವಂತೆ ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ
1. **ಗ್ರಾಹಕರನ್ನು ಸಂಪರ್ಕಿಸಿ**: `http://localhost:3000`ಂತಹ ಸ್ಥಳೀಯ ಸಂಪರ್ಕ URL ಗಳು ಬಳಸಿಕೊಳ್ಳಿ

## ಕ್ಲೌಡ್ ನಿಯೋಜನೆ

MCP ಸರ್ವರ್ ಗಳನ್ನು ವಿಭಿನ್ನ ಕ್ಲೌಡ್ ವೇದಿಕೆಗಳಿಗೆ ನಿಯೋಜಿಸಬಹುದು:

- **ಸರ್ವರ್‌ಲೆಸ್ ಫಂಕ್ಷನ್ಸ್**: MCP ಸರ್ವರ್ ಗಳನ್ನು ಲಘುಪ್ರಶಸ್ತ ಸರ್ವರ್‌ಲೆಸ್ ಫಂಕ್ಷನ್ಸ್ ಆಗಿ ನಿಯೋಜಿಸಿ
- **ಕಾಂಟೈನರ್ ಸೇವೆಗಳು**: Azure Container Apps, AWS ECS, ಅಥವಾ Google Cloud Run ಮುಂತಾದ ಸೇವೆಗಳ ಬಳಕೆ
- **ಕೂಬರ್ನೇಟಿಸ್**: ಕುಬರ್ನೇಟಿಸ್ ಕ್ಲಸ್ಟರ್ ಗಳಲ್ಲಿ MCP ಸರ್ವರ್ ಗಳನ್ನು ನಿಯೋಜಿಸಿ ಮತ್ತು ನಿರ್ವಹಿಸಿ, ಹೆಚ್ಚಿನ ಲಭ್ಯತೆಗಾಗಿ

### ಉದಾಹರಣೆ: Azure Container Apps

Azure Container Apps MCP ಸರ್ವರ್ ಗಳ ನಿಯೋಜನೆಯನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ. ಇದು ಇನ್ನೂ ಕೆಲಸದಲ್ಲಿ ಇದೆ ಮತ್ತು ಪ್ರಸ್ತುತ SSE ಸರ್ವರ್ ಗಳನ್ನು ಮಾತ್ರ ಬೆಂಬಲಿಸುತ್ತದೆ.

ನೀವು ಹೇಗೆ ಮುಂದಾಗಬಹುದು:

1. ರೆಪೋ ಒಂದನ್ನು ಕ್ಲೋನ್ ಮಾಡಿ:

  ```sh
  git clone https://github.com/anthonychu/azure-container-apps-mcp-sample.git
  ```

1. ಸ್ಥಳೀಯವಾಗಿ ಅದನ್ನು ರನ್ ಮಾಡಿ ಪರೀಕ್ಷೆ ಮಾಡಿ:

  ```sh
  uv venv
  uv sync

  # ಲೀನಕ್ಸ/ಮ್ಯಾಕ್‌ಒಎಸ್
  export API_KEYS=<AN_API_KEY>
  # ವಿಂಡೋಸ್
  set API_KEYS=<AN_API_KEY>

  uv run fastapi dev main.py
  ```

1. ಅದನ್ನು ಸ್ಥಳೀಯವಾಗಿ ಪ್ರಯತ್ನಿಸಲು, *.vscode* ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ *mcp.json* ಫೈಲ್ ರಚಿಸಿ ಮತ್ತು ಕೆಳಗಿನ ವಿಷಯವನ್ನು ಸೇರಿಸಿ:

  ```json
  {
      "inputs": [
          {
              "type": "promptString",
              "id": "weather-api-key",
              "description": "Weather API Key",
              "password": true
          }
      ],
      "servers": {
          "weather-sse": {
              "type": "sse",
              "url": "http://localhost:8000/sse",
              "headers": {
                  "x-api-key": "${input:weather-api-key}"
              }
          }
      }
  }
  ```

  SSE ಸರ್ವರ್ ಪ್ರಾರಂಭವಾದ ಮೇಲೆ, JSON ಫೈಲ್ ನಲ್ಲಿ ಪ್ಲೇ ಐಕಾನ್ ಕ್ಲಿಕ್ ಮಾಡಬಹುದು, ನೀವು ಈಗ GitHub Copilot ನಲ್ಲಿ ಸರ್ವರ್ ಮೇಲಿನ ಸಾಧನಗಳನ್ನು ಕಾಣಬಹುದು, ಸಾಧನ ಐಕಾನ್ ನೋಡಿ.

1. ನಿಯೋಜಿಸಲು, ಈ ಕೆಳಗಿನ ಆಜ್ಞೆಯನ್ನು ರನ್ ಮಾಡಿ:

  ```sh
  az containerapp up -g <RESOURCE_GROUP_NAME> -n weather-mcp --environment mcp -l westus --env-vars API_KEYS=<AN_API_KEY> --source .
  ```

ಇದೇ, ಇದನ್ನು ಸ್ಥಳೀಯವಾಗಿ ನಿಯೋಜಿಸಿ, ಈ ಹಂತಗಳ ಮೂಲಕ ಅದನ್ನು Azure ಗೆ ನಿಯೋಜಿಸಿ.
