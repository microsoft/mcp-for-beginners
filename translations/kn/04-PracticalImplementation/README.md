# ಪ್ರಾಯೋಗಿಕ ಅನುಷ್ಠಾನ

[![ವಾಸ್ತವಿಕ ಸಾಧನಗಳು ಮತ್ತು ವರ್ಕ್ಫ್ಲೋಗಳೊಂದಿಗೆ MCP ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಹೇಗೆ ನಿರ್ಮಿಸಬೇಕು, ಪರೀಕ್ಷಿಸಬೇಕು ಮತ್ತು ನಿಯೋಜಿಸಬೇಕು](../../../translated_images/kn/05.64bea204e25ca891.webp)](https://youtu.be/vCN9-mKBDfQ)

_(ಈ ಪಾಠದ ವೀಡಿಯೋವನ್ನು ವೀಕ್ಷಿಸಲು ಮೇಲೆ ಚಿತ್ರದ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ)_

ಪ್ರಾಯೋಗಿಕ ಅನುಷ್ಠಾನವು ಮಾದರಿ ಸಂದರ್ಭ ಪ್ರೋಟೋಕಾಲ್ (MCP) ಶಕ್ತಿಯನ್ನು ಸ್ಪಷ್ಟವಾಗಿ ತೋರಿಸುತ್ತದೆ. MCP ಗೆ ಸಂಬಂಧಿಸಿದ ಸಿದ್ಧಾಂತ ಮತ್ತು ವಾಸ್ತುಶಿಲ್ಪವನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು ಮುಖ್ಯವಾಗಿರಬಹುದು, ಆದರೆ ನಿಜವಾದ ಮೌಲ್ಯವು ಈ ಸಂಕ್ಷಿಪ್ತಿಗಳನ್ನು ನೈಜ ಜಗತ್ತಿನ ಸಮಸ್ಯೆಗಳನ್ನು ಪರಿಹರಿಸಲು ನಿರ್ಮಿಸಲು, ಪರೀಕ್ಷಿಸಲು ಮತ್ತು ನಿಯೋಜಿಸಲು ಅನ್ವಯಿಸಿದ್ದಾಗ ಬರುತ್ತದೆ. ಈ ಅಧ್ಯಾಯವು ಕಲ್ಪನಾತ್ಮಕ ಜ್ಞಾನ ಮತ್ತು ಕೈಯಾರೆ ಅಭಿವೃದ್ಧಿಗೊಳಿಸುವಿಕೆಯ ನಡುವೆ ಸೇತುವೆಯನ್ನು ಒದಗಿಸುತ್ತದೆ, MCP ಆಧಾರಿತ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಬದುಕಿಗೆ ತರುವ ಪ್ರಕ್ರಿಯೆಯಲ್ಲಿ ನಿಮಗೆ ಮಾರ್ಗದರ್ಶನ ಮಾಡುತ್ತದೆ.

ನೀವು ಬುದ್ಧಿವಂತ ಸಹಾಯಕರನ್ನು ಅಭಿವೃದ್ಧಿಪಡಿಸುತ್ತಿದ್ದಿರ ಅಥವಾ ವ್ಯಾಪಾರದ ವರ್ತಿ ವ್ಯವಸ್ಥೆಗಳೊಳಗೆ AI ನ್ನು ಸಂಯೋಜಿಸುತ್ತಿದ್ದಿರ ಅಥವಾ ಡೇಟಾ ಪ್ರಕ್ರಿಯೆಯಿಗಾಗಿ ಕಸ್ಟಮ್ ಟೂಲ್‌ಗಳನ್ನು ನಿರ್ಮಿಸುತ್ತಿದ್ದಿರ MCP ಗಳು ದೊರಕುವ ನೈಸರ್ಗಿಕ ಆಧಾರವನ್ನು ಒದಗಿಸುತ್ತವೆ. ಇದರ ಭಾಷೆ-ನಿರಪೇಕ್ಷ ವಿನ್ಯಾಸ ಮತ್ತು ಜನಪ್ರಿಯ ಪ್ರೋಗ್ರಾಮಿಂಗ್ ಭಾಷೆಗಳಿಗೆ ಅಧಿಕೃತ SDK ಗಳು ವ್ಯಾಪಕ ಸಂಖ್ಯೆಯ ಅಭಿವೃದ್ಧಿಪಡಿಸುವವರಿಗೆ ಜನಪ್ರಿಯವಾಗಲು ಸಹಾಯಕ. ಈ SDK ಗಳನ್ನು ಬಳಸಿ ನೀವು ವೇಗವಾಗಿ ನಕ್ಷೆ ನಿರ್ಮಿಸಬಹುದು, ಪನ್ಚಾವಣೆಯೊಂದಿಗೆ ಪ್ರಯೋಗ ಮಾಡಿ ಮತ್ತು ವಿವಿಧ ವೇದಿಕೆಗಳು ಮತ್ತು ಪರಿಸರಗಳಲ್ಲಿ ನಿಮ್ಮ ಪರಿಹಾರಗಳನ್ನು ವಿಸ್ತರಿಸಬಹುದು.

ಕೆಳಗಿನ ವಿಭಾಗಗಳಲ್ಲಿ, ನೀವು MCP ಅನ್ನು C#, Java with Spring, TypeScript, JavaScript ಮತ್ತು Python ನಲ್ಲಿ ಹೇಗೆ ಅನುಷ್ಠಾನಗೊಳಿಸಬಹುದು ಎಂಬುದನ್ನು ತೋರಿಸುವ ಪ್ರಾಯೋಗಿಕ ಉದಾಹರಣೆಗಳು, ಮಾದರಿ ಕೋಡ್ ಮತ್ತು ನಿಯೋಜನೆ ತಂತ್ರಗಳನ್ನು ಕಾಣುವಿರಿ. ನೀವು MCP ಸರ್ವರ್‌ಗಳನ್ನು ಹೇಗೆ ಡಿಬಗ್ ಮತ್ತು ಪರೀಕ್ಷಿಸಬೇಕೆಂದು, API ಗಳನ್ನು ನಿರ್ವಹಿಸುವುದು ಮತ್ತು Azure ಬಳಸಿ ಕ್ಲೌಡ್‌ಗೆ ಪರಿಹಾರಗಳನ್ನು ನಿಯೋಜಿಸುವ ವಿಧಾನಗಳನ್ನು ಕಲಿಯುವಿರಿ. ಈ ಕೈಯಾರೆ ಸಂಪನ್ಮೂಲಗಳು ನಿಮ್ಮ ಕಲಿಕೆಯ ವೇಗವನ್ನು ಹೆಚ್ಚಿಸಲು ಮತ್ತು ಭರವಸೆಪೂರ್ವಕವಾಗಿ ಬಲಿಷ್ಠ, ಉತ್ಪಾದನೆ-ಸಿದ್ಧ MCP ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿರ್ಮಿಸಲು ಸಹಾಯಕವಾಗಿವೆ.

## ಅವಲೋಕನ

ಈ ಪಾಠವು ವಿವಿಧ ಪ್ರೋಗ್ರಾಮಿಂಗ್ ಭಾಷೆಗಳ ಮೂಲಕ MCP ಅನುಷ್ಠಾನದ ಪ್ರಾಯೋಗಿಕ ಅಂಶಗಳ ಮೇಲೆ ಕೇಂದ್ರೀಕರಿಸಿದೆ. ನಾವು C#, Java with Spring, TypeScript, JavaScript ಮತ್ತು Python ನಲ್ಲಿ MCP SDK ಗಳನ್ನು ಉಪಯೋಗಿಸಿ ಬಲಿಷ್ಠ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿರ್ಮಿಸುವುದು, MCP ಸರ್ವರ್‌ಗಳನ್ನು ಡಿಬಗ್ ಮತ್ತು ಪರೀಕ್ಷಿಸುವುದು, ಪುನಃಬಳಕೆ ಮಾಡಬಹುದಾದ ಸಂಪನ್ಮೂಲಗಳು, ಪ್ರಾಂಪ್ಟ್‌ಗಳು ಮತ್ತು ಉಪಕರಣಗಳನ್ನು ರಚಿಸುವುದನ್ನು ಅನ್ವೇಷಿಸುವೆವು.

## ಕಲಿಕೆ ಉದ್ದೇಶಗಳು

ಈ ಪಾಠದ ಅಂತ್ಯಕ್ಕೆ, ನೀವು ಕೆಳಗಿನ ಕೆಲಸಗಳನ್ನು ಮಾಡಬಲ್ಲಿರಿ:

- ವಿವಿಧ ಪ್ರೋಗ್ರಾಮಿಂಗ್ ಭಾಷೆಗಳಲ್ಲಿ ಅಧಿಕೃತ SDK ಗಳನ್ನು ಬಳಸಿ MCP ಪರಿಹಾರಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸುವುದು
- MCP ಸರ್ವರ್‌ಗಳನ್ನು ಕ್ರಮಬದ್ಧವಾಗಿ ಡಿಬಗ್ ಮತ್ತು ಪರೀಕ್ಷಿಸುವುದು
- ಸರ್ವರ್ ವೈಶಿಷ್ಟ್ಯಗಳು (ಸಂಪನ್ಮೂಲಗಳು, ಪ್ರಾಂಪ್ಟ್ ಗಳು ಮತ್ತು ಉಪಕರಣಗಳು) ಸೃಷ್ಟಿಸಿ ಬಳಸುವುದು
- ಸಂಕೀರ್ಣ ಕಾರ್ಯಗಳಿಗೆ ಪರಿಣಾಮಕಾರಿಯಾದ MCP ವರ್ಕ್ಫ್ಲೋಗಳನ್ನು ವಿನ್ಯಾಸಗೊಳಿಸುವುದು
- ಪ್ರದರ್ಶನ ಮತ್ತು ನಂಬಿಕೆಯತೆಗೆ MCP ಅನುಷ್ಠಾನಗಳನ್ನು tốiಮೈಸೇಶನ್ ಮಾಡುವುದು

## ಮುಖ್ಯ ಸರ್ವರ್ ವೈಶಿಷ್ಟ್ಯಗಳು

MCP ಸರ್ವರ್‌ಗಳು ಕೆಳಗಿನ ವೈಶಿಷ್ಟ್ಯಗಳ ಯಾವುದೇ ಸಂಯೋಜನೆಯನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಬಹುದು:

### ಸಂಪನ್ಮೂಲಗಳು

ಉಪಯೋಗಿಸುವವರಿಗೂ ಅಥವಾ AI ಮಾದರಿಗೂ ಸನ್ನಿವೇಶ ಮತ್ತು ಡೇಟಾ ಒದಗಿಸುವ ಸಂಪನ್ಮೂಲಗಳು:

- ದಾಖಲೆ ಸಂಗ್ರಹಗಳಾಗಿರುವRepositories
- ಜ್ಞಾನ ಆಧಾರಗಳು
- ರಚಿಸಿದ ಡೇಟಾ ಮೂಲಗಳು
- ಫೈಲ್ ವ್ಯವಸ್ಥೆಗಳು

### ಪ್ರಾಂಪ್ಟ್‌ಗಳು

ಬಳಕೆದಾರರಿಗಾಗಿ ಟೆಂಪ್ಲೇಟು ಮಾಡಿದ ಸಂದೇಶಗಳು ಮತ್ತು ವರ್ಕ್ಫ್ಲೋಗಳು:

- ಪೂರ್ವನಿರ್ಧಿಷ್ಟ ಸಂಭಾಷಣಾ ಟೆಂಪ್ಲೇಟುಗಳು
- ಮಾರ್ಗದರ್ಶಿಸಿದ ಸಂವಾದ ಮಾದರಿಗಳು
- ವಿಶಿಷ್ಟ ಸಂವಾದ ರಚನೆಗಳು

### ಉಪಕರಣಗಳು

AI ಮಾದರಿ ಕಾರ್ಯಾಚರಣೆಗಾಗಿ ಫಂಕ್ಷನ್‌ಗಳು:

- ಡೇಟಾ ಸಂಸ್ಕರಣಾ ಉಪಕರಣಗಳು
- ಹೊರಗಿನ API ಸಂಯೋಜನೆಗಳು
- ಲೆಕ್ಕಾಚಾರ ಸಾಮರ್ಥ್ಯಗಳು
- ಹುಡುಕಾಟ ಕಾರ್ಯಕ್ಷಮತೆ

## ಮಾದರಿ ಅನುಷ್ಠಾನಗಳು: C# ಅನುಷ್ಠಾನ

ಅಧಿಕೃತ C# SDK ರೆಪೊಸಿಟರಿಯಲ್ಲಿ MCP ಯ ವಿಭಿನ್ನ ಅಂಶಗಳನ್ನು ತೋರಿಸುವ ಅನೇಕ ಮಾದರಿ ಅನುಷ್ಠಾನಗಳಿವೆ:

- **ಮೂಲ MCP ಕ್ಲೈಂಟ್**: ಹೇಗೆ MCP ಕ್ಲೈಂಟ್ ಸೃಷ್ಟಿ ಮಾಡಿ, ಉಪಕರಣಗಳನ್ನು ಕರೆ ಮಾಡಬಹುದು ಎಂಬ ಸರಳ ಉದಾಹರಣೆ
- **ಮೂಲ MCP ಸರ್ವರ್**: ಮೂಲ ಉಪಕರಣ ನೋಂದಣಿ ಇರುವ ಕನಿಷ್ಠ ಸರ್ವರ್ ಅನುಷ್ಠಾನ
- **ಉನ್ನತ MCP ಸರ್ವರ್**: ಉಪಕರಣ ನೋಂದಣಿ, ದೃಢೀಕರಣ ಮತ್ತು ದೋಷ ನಿರ್ವಹಣೆ ಒಳಗೊಂಡ ಸಂಪೂರ್ಣ ವೈಶಿಷ್ಟ್ಯದ ಸರ್ವರ್
- **ASP.NET ಸಂಯೋಜನೆ**: ASP.NET ಕೋರ್ ಸಹ ಸಂಯೋಜಿಸುವ ಉದಾಹರಣೆಗಳು
- **ಉಪಕರಣ ಅನುಷ್ಠಾನ ಮಾದರಿಗಳು**: ವಿಭಿನ್ನ ಜಟಿಲತೆ ಮಟ್ಟಗಳೊಂದಿಗೆ ಉಪಕರಣಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸುವ ವಿವಿಧ ಮಾದರಿಗಳು

MCP C# SDK ಈಗ ಪ್ರೀವ್ಯೂನಲ್ಲಿ ಇದೆ ಮತ್ತು API ಗಳು ಬದಲಾಗಬಹುದು. SDK ಮುನ್ನಡೆಯುವಂತೆ ಈ ಬ್ಲಾಗ್ ಅನ್ನು ನಿರಂತರವಾಗಿ ನವೀಕರಿಸಲಾಗುತ್ತದೆ.

- ಮೇಲ್ವಿಚಾರಣೆ
- ಲೋಡ್ ಸಮತೋಲನ
- ಭದ್ರತೆ

## ದೂರಸ್ಥ MCP ಸರ್ವರ್ ಅನ್ನು Azure ಗೆ ನಿಯೋಜನೆ

ನಾವು ಹೀಗಾದಲ್ಲಿ ಮೊದಲಿಗೆ ಉಲ್ಲೇಖಿಸಿದ ಮಾದರಿಯನ್ನು ನಿಯೋಜಿಸಬಹುದು ಎಂಬುದನ್ನು ನೋಡೋಣ:

1. ರೆಪೊ ಕ್ಲೋನ್ ಮಾಡಿ

    ```bash
    git clone https://github.com/Azure-Samples/remote-mcp-apim-functions-python.git
    cd remote-mcp-apim-functions-python
    ```

2. `Microsoft.App` ಸಂಪನ್ಮೂಲ ಪೂರೈಕೆದಾರರನ್ನು ನೋಂದಾಯಿಸಿ.

   - ನೀವು Azure CLI ಬಳಿಸುತ್ತಿದ್ದರೆ, `az provider register --namespace Microsoft.App --wait` چلಿಸಿ.
   - ನೀವು Azure PowerShell ಬಳಿಸುತ್ತಿದ್ದರೆ, `Register-AzResourceProvider -ProviderNamespace Microsoft.App` ಚಲಿಸಿ. ನಂತರ ಇಠುತ್ತ ಸಮಯದ ನಂತರ ` (Get-AzResourceProvider -ProviderNamespace Microsoft.App).RegistrationState` 를 ಪರಿಶೀಲಿಸಿ ನೋಂದಣಿ ಪೂರ್ಣವಾಯಿತೇ ಎಂದು.

3. ಈ [azd](https://aka.ms/azd) ಕಮಾಂಡ್ ಅನ್ನು ನಡೆಯಿಸಿ api ನಿರ್ವಹಣಾಪ್ರದರ್ಶಕ, ಫಂಕ್ಷನ್ ಅಪ್ಲಿಕೇಶನ್ (ಕೋಡ್ ಒಳಗೊಂಡ) ಮತ್ತು ಬಾಕಿ ಎಲ್ಲಾ ಅಗತ್ಯ Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ಪ್ರावೀ뉴스 ಮಾಡಿ

    ```shell
    azd up
    ```

    ಈ ಕಮಾಂಡ್ ಇರೋ ಎಲ್ಲಾ ಕ್ಲೌಡ್ ಸಂಪನ್ಮೂಲಗಳನ್ನು Azure ನಲ್ಲಿ ನಿಯೋಜಿಸಬೇಕು

### MCP ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಬಳಸಿ ನಿಮ್ಮ ಸರ್ವರ್ ಪರೀಕ್ಷೆ

1. **ಹೊಸ ಟರ್ಮಿನಲ್ ವಿಂಡೋದಲ್ಲಿ**, MCP ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಅನ್ನು ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಿ ಮತ್ತು ಚಾಲನೆ ಮಾಡಿ

    ```shell
    npx @modelcontextprotocol/inspector
    ```

    ನೀವು ಈ ರೀತಿಯ ಇಂಟರ್ಫೇಸ್ ಅನ್ನು ನೋಡಬೇಕು:

    ![Node ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಸಂಪರ್ಕ](../../../translated_images/kn/connect.141db0b2bd05f096.webp)

2. ಸಿ.ಟಿಆರ್‌ಎಲ್ ಕ್ಲಿಕ್ ಮಾಡಿ MCP ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು URL ನಿಂದ ಲೋಡ್ ಮಾಡಿ (ಉದಾಹರಣೆಗೆ: [http://127.0.0.1:6274/#resources](http://127.0.0.1:6274/#resources))
3. ಸಂಚಾರ ಪ್ರಕಾರವನ್ನು `SSE` ಗೆ ಸೆಟ್ ಮಾಡಿ
4. URL ಅನ್ನು ನಿಮ್ಮ ಎಪಿಐ ನಿರ್ವಹಣಾ SSE ಎಂಡ್‌ಪಾಯಿಂಟ್‌ಗೆ ಸೆಟ್ ಮಾಡಿ, ಇದು `azd up` ನಂತರ ಪ್ರದರ್ಶಿಸಲ್ಪಡುತ್ತದೆ ಮತ್ತು **ಸಂಬಂಧಿಸಿ**:

    ```shell
    https://<apim-servicename-from-azd-output>.azure-api.net/mcp/sse
    ```

5. **ಟೂಲ್ ಪಟ್ಟಿಮಾಡಿ**. ಒಂದು ಉಪಕರಣ ಕ್ಲಿಕ್ ಮಾಡಿ **ಟೂಲ್ ಓಡಿಸಿ**. 

ಎಲ್ಲಾ ಹಂತಗಳು ಯಶಸ್ವಿಯಾಗಿ ನಡೆದಿದ್ದರೆ, ನೀವು ಈಗ MCP ಸರ್ವರ್ ಗೆ ಸಂಪರ್ಕ ಹೊಂದಿದ್ದೀರಿ ಮತ್ತು ಒಂದು ಉಪಕರಣವನ್ನು ಕರೆ ಮಾಡಬಲ್ಲಿದ್ದೀರಿ.

## Azure ಗಾಗಿ MCP ಸರ್ವರ್‌ಗಳು

[Remote-mcp-functions](https://github.com/Azure-Samples/remote-mcp-functions-dotnet): ಈ ರೆಪೊ ಸೆರಿಗಳು Python, C# .NET ಅಥವಾ Node/TypeScript ಬಳಸಿ Azure Functions ಮೂಲಕ ತನ್ನದೇ ಆದ ದೂರಸ್ಥ MCP ( ಮಾದರಿ ಸಂದರ್ಭ ಪ್ರೋಟೋಕಾಲ್ ) ಸರ್ವರ್‌ಗಳನ್ನು ರಚಿಸಿ ನಿಯೋಜಿಸಲು ಶೀಘ್ರಪ್ರಾರಂಭ ಪಾಠ್ಯಕ್ರಮ.

ಈ ಮಾದರಿ ಸಂಪೂರ್ಣ ಪರಿಹಾರವನ್ನು ಒದಗಿಸುತ್ತದೆ, ಇದರಿಂದ ಅಭಿವೃದ್ಧಿ ದಾರರು:

- ಸ್ಥಳೀಯವಾಗಿ ನಿರ್ಮಿಸಿ ಮತ್ತು ಓಡಿಸಿ: ಸ್ಥಳೀಯ ಯಂತ್ರದಲ್ಲಿ MCP ಸರ್ವರ್ ಅನ್ನು ಅಭಿವೃದ್ಧಿಪಡಿಸಿ ಮತ್ತು ಡಿಬಗ್ ಮಾಡಿ
- Azure ಗೆ ನಿಯೋಜಿಸಿ: ಸರಳ azd up ಕಮಾಂಡ್ ಮೂಲಕ ಕ್ಲೌಡ್‌ಗೆ ಸುಲಭ ನಿಯೋಜನೆ
- ಕ್ಲೈಂಟ್ ಗಳಿಂದ ಸಂಪರ್ಕಿಸಿ:‌‌ MCP ಸರ್ವರ್ ಗೆ ವಿವಿಧ ಕ್ಲೈಂಟ್ ಗಳಿಂದ ಸಂಪರ್ಕ ಹೊಂದಿ, ಇದರಲ್ಲಿ VS Code ನ Copilot ಏಜೆಂಟ್ ಮೋಡ್ ಮತ್ತು MCP ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಟೂಲ್ ಸೇರಿವೆ

### ಮುಖ್ಯ ವೈಶಿಷ್ಟ್ಯಗಳು

- ವಿನ್ಯಾಸದಿಂದ ಭದ್ರತೆ: MCP ಸರ್ವರ್ ಗುಪ್ತಕೇಳು ಮತ್ತು HTTPS ಬಳಸಿ ಸುರಕ್ಷಿತವಾಗಿರುತ್ತದೆ
- ಪ್ರಾಮಾಣಿಕರಣ ಆಯ್ಕೆಗಳು: ನಿರ್ಮಿತ ಅಡ್ಮಿನ್ ಮತ್ತು/ಅಥವಾ API ನಿರ್ವಹಣೆ ಮೂಲಕ OAuth ಬೆಂಬಲ
- ನೆಟ್ವರ್ಕ್ ಪ್ರತ್ಯೇಕತೆ: Azure ವರ್ಚುವಲ್ ನೆಟ್ವರ್ಕ್‌ಗಳು (VNET) ಮೂಲಕ ನೆಟ್‌ವರ್ಕ್ ಪ್ರತ್ಯೇಕತೆ ಅನುಮತಿಸುತ್ತದೆ
- ಸರ್ವರ್ ಲೆಸ್ ವಾಸ್ತುಶಿಲ್ಪ: Azure Functions ಬಳಸಿ ಸ್ಕೇಲೇಬಲ್ ಹಾಗೂ ಘಟನೆ ಆಧಾರಿತ ಕಾರ್ಯನಿರ್ವಹಣೆ
- ಸ್ಥಳೀಯ ಸುಧಾರಿತ ಅಭಿವೃದ್ಧಿ: ಸಂಪೂರ್ಣ ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿ ಮತ್ತು ಡಿಬಗ್ ಬೆಂಬಲ
- ಸರಳ ನಿಯೋಜನೆ: Azure ಗೆ ಸಮರ್ಪಕ ನಿಯೋಜನೆ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಸರಳೀಕರಿಸಲಾಗಿದೆ

ರೆಪೊ ಎಲ್ಲಾ ಅಗತ್ಯ ಸಂರಚನಾ ಕಡತಗಳು, ಮೂಲ ಕೋಡ್ ಮತ್ತು ಮೂಲಸೌಕರ್ಯ ವ್ಯಾಖ್ಯಾನಗಳನ್ನು ಸೇರಿಟ್ಟು, ಉತ್ಪಾದನೆ-ಸಿದ್ಧ MCP ಸರ್ವರ್ ಅನುಷ್ಠಾನದೊಂದಿಗೆ ತ್ವರಿತವಾಗಿ ಪ್ರಾರಂಭಿಸಲು ಸಹಕರಿಸುತ್ತದೆ.

- ಡಿಬಗ್ಗಿಂಗ್ ಮತ್ತು ಪರೀಕ್ಷೆ ಪ್ರಕ್ರಿಯೆ MCP ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ ನಂಬಿಕೆಯತೆಯ ಪ್ರಮುಖ ಅಂಶ
- ಪುನಃಬಳಕೆದಾರ ಪ್ರಾಂಪ್ಟ್ ಟೆಂಪ್ಲೇಟುಗಳು ಏಕರೂಪ AI ಸಂವಾದಗಳನ್ನು ಸಾಧ್ಯವನ್ನಾಗಿಸುತ್ತವೆ
- ಉತ್ತಮವಾಗಿ ವಿನ್ಯಾಸಗೊಳ್ಳಲಾದ ವರ್ಕ್ಫ್ಲೋಗಳು ಹಲವಾರು ಉಪಕರಣಗಳನ್ನು ಬಳಸಿ ಸಂಕೀರ್ಣ ಕಾರ್ಯಗಳನ್ನು ನಿಯಂತ್ರಿಸಬಹುದು
- MCP ಪರಿಹಾರಗಳ ಅನುಷ್ಠಾನ ಭದ್ರತೆ, ಪ್ರದರ್ಶನ ಮತ್ತು ದೋಷ ನಿರ್ವಹಣೆಯ ಪರಿಗಣನೆಗಳ ಅಗತ್ಯವಿದೆ

## ಅಭ್ಯಾಸ

ನಿಮ್ಮ ಕ್ಷೇತ್ರದಲ್ಲಿ ನೈಜ ಜಗತ್ತಿನ ಸಮಸ್ಯೆಗೆ ಪ್ರಾಯೋಗಿಕ MCP ವರ್ಕ್ಫ್ಲೋ ವಿನ್ಯಾಸ ಮಾಡಿ:

1. ಈ ಸಮಸ್ಯೆ ಪರಿಹರಿಸಲು ಉಪಯುಕ್ತವಾಗಬಹುದಾದ 3-4 ಉಪಕರಣಗಳನ್ನು ಗುರುತಿಸಿ
2. ಈ ಉಪಕರಣಗಳು ಹೇಗೆ ಪರಸ್ಪರಕ್ರಿಯೆ ಮಾಡುತ್ತವೆ ಎಂಬುದು ತೋರಿಸುವ ವರ್ಕ್ಫ್ಲೋ ಡಯಾಗ್ರಾಮ್ ರಚಿಸಿ
3. ಪ್ರೀತಿಪಾತ್ರ ಭಾಷೆಯನ್ನು ಬಳಸಿ ಒಂದರಲ್ಲಿ ಮೂಲ ಪ್ರಕಾರದ ಉಪಕರಣವನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಿ
4. ಮಾದರಿಗೆ ನಿಮ್ಮ ಉಪಕರಣವನ್ನು ಪರಿಣಾಮಕಾರಿಯಾಗಿ ಬಳಸಲು ಸಹಾಯ ಮಾಡುವ ಪ್ರಾಂಪ್ಟ್ ಟೆಂಪ್ಲೇಟ್ ಸೃಷ್ಟಿಸಿ
