# ಪ್ರಾಯೋಗಿಕ ಅನುಷ್ಠಾನ

[![ನಿಜವಾದ ಸಾಧನಗಳು ಮತ್ತು ವರ್ಕ್‌ಫ್ಲೋಗಳೊಂದಿಗೆ MCP ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿರ್ಮಿಸುವುದು, ಪರೀಕ್ಷಿಸುವುದು ಮತ್ತು ನಿಯೋಜಿಸುವುದು ಹೇಗೆ](../../../translated_images/kn/05.64bea204e25ca891.webp)](https://youtu.be/vCN9-mKBDfQ)

_(ಈ ಪಾಠದ ವೀಡಿಯೊವನ್ನು ವೀಕ್ಷಿಸಲು ಮೇಲಿನ ಚಿತ್ರವನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ)_

ಪ್ರಾಯೋಗಿಕ ಅನುಷ್ಠಾನವು ಮಾದರಿ_Context_ಪ್ರೋಟೋಕಾಲ್ (MCP) ಶಕ್ತಿಯು ಸ್ಪಷ್ಟವಾಗುತ್ತದೆ. MCP ರ ಹಿಂದೆ ಇರುವ ಸಿದ್ಧಾಂತ ಮತ್ತು ವಾಸ್ತುಶಿಲ್ಪವನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು ಮಹತ್ವಪೂರ್ಣವಾದರೂ, ನಿಜವಾದ ಮೌಲ್ಯವು ಈ ತತ್ವಗಳನ್ನು ಉಪಯೋಗಿಸಿ ನಿಜ ಜೀವನಸಮಸ್ಯೆಗಳನ್ನು ಪರಿಹರಿಸುವ ಪರಿಹಾರಗಳನ್ನು ನಿರ್ಮಿಸಲು, ಪರೀಕ್ಷಿಸಲು ಮತ್ತು ನಿಯೋಜಿಸಲು ಸಾಧ್ಯವಾಗುವಾಗ ಬಹಿರಂಗವಾಗುತ್ತದೆ. ಈ ಅಧ್ಯಾಯವು ವೈಚಾರಿಕ ಜ್ಞಾನ ಮತ್ತು ಹ್ಯಾಂಡ್-ಆನ್ ಅಭಿವೃದ್ಧಿಗಳ ನಡುವಿನ ಗ್ಯಾಪ್ ಅನ್ನು ಭರ್ತಿಮಾಡುತ್ತದೆ, MCP ಆಧಾರಿತ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಜೀವಂತವಾಗಿ ತರುವ ಪ್ರಕ್ರಿಯೆಯಲ್ಲಿ ನಿಮಗೆ ಮಾರ್ಗದರ್ಶನ ನೀಡುತ್ತದೆ.

ನೀವು ಬುದ್ಧಿವಂತ ಸಹಾಯಕರು, ವ್ಯಾಪಾರ ವರ್ಕ್ಫ್ಲೋಗಳಲ್ಲಿ AI ಸಂಯೋಜನೆಯು ಅಥವಾ ಡೇಟಾ ಪ್ರಕ್ರಿಯೆಗೆ ಕಸ್ಟಮ್ ಸಾಧನಗಳ ನಿರ್ಮಾಣವನ್ನು ಮಾಡುತ್ತಿರುವಿರಾ, MCP ಲವಚಿಕೆಯ ಆದಾರವನ್ನು ಒದಗಿಸುತ್ತದೆ. ಅದರ ಭಾಷಾ-ನಿರಪೇಕ್ಷ ವಿನ್ಯಾಸ ಮತ್ತು ಜನಪ್ರಿಯ ಕಾರ್ ಪ್ರೋಗ್ರಾಮಿಂಗ್ ಭಾಷೆಗಳಗಾಗಿ ಅಧಿಕೃತ SDK ಗಳು ಹೆಚ್ಚುವರಿ ಅಭಿವೃದ್ಧಿಪಡಿಸುವವರಿಗೆ ಪಲುತಾಗಿದೆ. ಈ SDK ಗಳನ್ನು ಉಪಯೋಗಿಸುವುದರಿಂದ, ನೀವು ವೇಗವಾಗಿ ಪ್ರೊಟೋಟೈಪ್ ಮಾಡಿ, ಪುನರಾವರ್ತಿ ಮಾಡಿ ಮತ್ತು ನಿಮ್ಮ ಪರಿಹಾರಗಳನ್ನು ವಿಭಿನ್ನ ವೇದಿಕೆ ಮತ್ತು ಪರಿಸರಗಳಲ್ಲಿ ವ್ಯಾಪಿಸಿಕೊಳ್ಳಬಹುದು.

ಮುಂದಿನ ವಿಭಾಗಗಳಲ್ಲಿ, ನೀವು MCP ಅನ್ನು C#, Java with Spring, TypeScript, JavaScript ಮತ್ತು Python ನಲ್ಲಿ ಹೇಗೆ ಅನ್ವಯಿಸುವುದನ್ನು ಪ್ರದರ್ಶಿಸುವ ಪ್ರಾಯೋಗಿಕ ಉದಾಹರಣೆಗಳು, ಮಾದರಿ ಕೋಡ್ ಮತ್ತು ನಿಯೋಜನೆ ತಂತ್ರಗಳನ್ನು ಕಾಣಬಹುದು. ನೀವು ನಿಮ್ಮ MCP ಸರ್ವರ್‌ಗಳನ್ನು ಡಿಬಗ್ ಮತ್ತು ಪರೀಕ್ಷಿಸುವುದನ್ನು, API ಗಳನ್ನು ನಿರ್ವಹಿಸುವುದನ್ನು ಮತ್ತು Azure ಯನ್ನು ಬಳಸಿ ಕ್ಲೌಡಿಗೆ ಪರಿಹಾರಗಳನ್ನು ನಿಯೋಜಿಸುವುದನ್ನು ಸಹ ಕಲಿಯುತ್ತೀರಿ. ಈ ಹ್ಯಾಂಡ್-ಆನ್ ಸಂಪನ್ಮೂಲಗಳು ನಿಮ್ಮ ಕಲಿಕೆಯನ್ನು ವೇಗವಾಗಿ ಮಾಡುತ್ತವೆ ಮತ್ತು ಬಲವಾದ, ಉತ್ಪಾದನೆ-ಸಿದ್ಧ MCP ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಆತ್ಮವಿಶ್ವಾಸದಿಂದ ನಿರ್ಮಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತವೆ.

## ಅವಲೋಕನ

ಈ ಪಾಠವು MCP ಅನುಷ್ಠಾನದ ಪ್ರಾಯೋಗಿಕ ಅಂಶಗಳ ಮೇಲೆ ಕೆಂದ್ರಿತವಾಗಿದೆ ಮತ್ತು ವಿವಿಧ ಪ್ರೋಗ್ರಾಮಿಂಗ್ ಭಾಷೆಗಳಲ್ಲಿನ MCP SDK ಗಳು ಹೇಗೆ ಉಪಯೋಗಿಸುವುದನ್ನು ಪರಿಶೀಲಿಸುತ್ತದೆ. ನಾವು C#, Java with Spring, TypeScript, JavaScript ಮತ್ತು Python ನಲ್ಲಿ MCP SDK ಗಳನ್ನು ಉಪಯೋಗಿಸಿ ಬಲವಾದ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿರ್ಮಿಸುವ ವಿಧಾನಗಳನ್ನು, MCP ಸರ್ವರ್‌ಗಳನ್ನು ಡಿಬಗ್ ಮತ್ತು ಪರೀಕ್ಷಿಸುವುದು, ಹಾಗೂ ಪುನರಾವರ್ತಿಸಬಹುದಾದ ಸಂಪನ್ಮೂಲಗಳು, ಪ್ರಾಂಪ್ಟ್‌ಗಳು ಮತ್ತು ಸಾಧನಗಳನ್ನು ಸೃಷ್ಟಿಸುವುದನ್ನು ಅನ್ವೇಷಣೆ ಮಾಡುತ್ತೇವೆ.

## ಕಲಿಕೆಯ ಗುರಿಗಳು

ಈ ಪಾಠದ ಕೊನೆಯಲ್ಲಿ, ನೀವು ಕೆಳಕಂಡ ಕಾರ್ಯಗಳನ್ನು ಮಾಡಬಲ್ಲಿರಿ:

- ವಿವಿಧ ಪ್ರೋಗ್ರಾಮಿಂಗ್ ಭಾಷೆಗಳಲ್ಲಿ ಅಧಿಕೃತ SDK ಗಳನ್ನು ಉಪಯೋಗಿಸಿ MCP ಪರಿಹಾರಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸುವುದು
- MCP ಸರ್ವರ್‌ಗಳನ್ನು ವ್ಯವಸ್ಥಿತವಾಗಿ ಡಿಬಗ್ ಮತ್ತು ಪರೀಕ್ಷಿಸುವುದು
- ಸರ್ವರ್ ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು (ಸಂಪನ್ಮೂಲಗಳು, ಪ್ರಾಂಪ್ಟ್‌ಗಳು ಮತ್ತು ಸಾಧನಗಳು) ರಚಿಸಿ ಮತ್ತು ಉಪಯೋಗಿಸುವುದು
- ಸಂಕೀರ್ಣ ಕಾರ್ಯಗಳಿಗೆ ಪರಿಣಾಮಕಾರಿಯಾದ MCP ವರ್ಕ್ಫ್ಲೋವನ್ನು ರೂಪಿಸುವುದು
- ಕಾರ್ಯಕ್ಷಮತೆ ಮತ್ತು ನಂಬಿಕೆಗೆ MCP ಅನುಷ್ಠಾನಗಳನ್ನು ಸೂಕ್ತಗೊಳಿಸುವುದು

- [ಪೋಟಿವೇದ್ಯತೆ ಮತ್ತು ದೊಡ್ಡ ಫಲಿತಾಂಶ ಸೆಟ್‌ಗಳು](./pagination/README.md) - ಸಾಧನಗಳು, ಸಂಪನ್ಮೂಲಗಳು ಮತ್ತು ದೊಡ್ಡ ಡೇಟಾಸೆಟ್‌ಗಳಿಗೆ ಸರ್ಸರ್ ಆಧಾರಿತ ಪೇಜಿನೇಶನ್ ಅನ್ನು ಕೈಗಾರಿಕೆ ಮಾಡುವುದು

## ಕೋರ್ ಸರ್ವರ್ ವೈಶಿಷ್ಟ್ಯಗಳು

MCP ಸರ್ವರ್‌ಗಳು ಕೆಳಕಂಡ ವೈಶಿಷ್ಟ್ಯಗಳ ಯಾವುದೇ ರೂಪಸಂಯೋಜನೆಯನ್ನೂ ಅನುಷ್ಠಾನಗೊಳಿಸಬಹುದು:

### ಸಂಪನ್ಮೂಲಗಳು

ಸಂಪನ್ಮೂಲಗಳು ಬಳಕೆದಾರ ಅಥವಾ AI ಮಾದರಿಗೆ ಒದಗಿಸುವಿಕೆಯ ನೆಲೆ ಮತ್ತು ಡೇಟಾವನ್ನು ಒದಗಿಸುತ್ತವೆ:

- ಡಾಕ್ಯುಮೆಂಟ್ ಸಂಗ್ರಹಣೆ
- ಜ್ಞಾನ ಮೂಲಗಳು
- ರಚನೆಗೊಂಡ ಡೇಟಾ ಮೂಲಗಳು
- ಫೈಲ್ ಸಿಸ್ಟಂಗಳು

### ಪ್ರಾಂಪ್ಟ್‌ಗಳು

ಪ್ರಾಂಪ್ಟ್‌ಗಳು ಬಳಕೆದಾರರಿಗಾಗಿ ಟೆಂಪ್ಲೇಟು ಮಾಡಿದ ಸಂದೇಶಗಳು ಮತ್ತು ವರ್ಕ್ಫ್ಲೋಗಳು:

- ಪೂರ್ವನಿಗದಿತ ಸಂಭಾಷಣೆಯ ಟೆಂಪ್ಲೇಟುಗಳು
- ಮಾರ್ಗದರ್ಶನ ನೀಡುವ ಸಂವಹನ ಮಾದರಿಗಳು
- ವಿಶೇಷ ಸಂವಾದ ರಚನೆಗಳು

### ಸಾಧನಗಳು

ಸಾಧನಗಳು AI ಮಾದರಿ ಕಾರ್ಯಗತಗೊಳಿಸುವ ಕಾರ್ಯಾಚರಣೆಗಳು:

- ಡೇಟಾ ಪ್ರಕ್ರಿಯೆ ಸಾಧನಗಳು
- ಹೊರಗಿನ API ಸಂಯೋಜನೆಗಳು
- ಲೆಕ್ಕಾಚಾರ ಸಾಮರ್ಥ್ಯಗಳು
- ಹುಡುಕಾಟ ಕಾರ್ಯಕ್ಷಮತೆ

## ಮಾದರಿ ಅನುಷ್ಠಾನಗಳು: C# ಅನುಷ್ಠಾನ

ಅಧಿಕೃತ C# SDK ರಿಪೋಜಿಟರಿಯಲ್ಲಿ MCP ನ ವಿವಿಧ ಅಂಶಗಳನ್ನು ತೋರುವ ಅನೇಕ ಮಾದರಿ ಅನುಷ್ಠಾನಗಳು ಇವೆ:

- **ಮೂಲ MCP ಕ್ಲೈಂಟ್**: MCP ಕ್ಲೈಂಟ್ ರಚಿಸುವುದು ಮತ್ತು ಸಾಧನಗಳನ್ನು ಕರೆ ಮಾಡುವ ಸಾದಾ ಉದಾಹರಣೆ
- **ಮೂಲ MCP ಸರ್ವರ್**: ಮೂಲ ಸಾಧನ ನೋಂದಣಿಯೊಂದಿಗೆ ಕನಿಷ್ಠ ಸರ್ವರ್ ಅನುಷ್ಠಾನ
- **ವಿಸ್ತೃತ MCP ಸರ್ವರ್**: ಸಾಧನ ನೋಂದಣಿ, ಪರಿಶೀಲನೆ ಮತ್ತು ದೋಷ ನಿರ್ವಹಣೆ ಸೇರಿದಂತೆ ಪೂರ್ಣ ವೈಶಿಷ್ಟ್ಯಗಳ ಸರ್ವರ್
- **ASP.NET ಸಂಯೋಜನೆ**: ASP.NET ಕೋರ್ ಜೊತೆ ಸಂಯೋಜನೆ ತೋರಿಸುವ ಉದಾಹರಣೆಗಳು
- **ಸಾಧನ ಅನುಷ್ಠಾನ ಮಾದರಿಗಳು**: ವಿವಿಧ ಸಾಥೀತ್ವದ ಮಟ್ಟಗಳ ಸಾಧನಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸುವ ವಿವಿಧ ಮಾದರಿಗಳು

MCP C# SDK ಪೂರ್ವವೀಕ್ಷಣೆಯಲ್ಲಿದ್ದು API ಗಳು ಬದಲಾಗಬಹುದು. SDK ಬದਲੇದಂತೆ ನಾವು ಈ ಬ್ಲಾಗ್‌ ಅನ್ನು ನಿರಂತರವಾಗಿ ನವೀಕರಿಸುವೆವು.

## API ನಿರ್ವಹಣೆ

Azure API ನಿರ್ವಹಣೆ MCP ಸರ್ವರ್‌ಗಳನ್ನು ಸುರಕ್ಷಿತಗೊಳಿಸುವ ಉತ್ತಮ ಉತ್ತರವಾಗಿದೆ. ಆಲೋಚನೆ ಎಂದರೆ ನಿಮ್ಮ MCP ಸರ್ವರ್ ಮುಂದೆ Azure API ನಿರ್ವಹಣೆಯನ್ನು ಇರಿಸಿ ಮತ್ತು ನೀವು ಬಯಸುವಂತಹ ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ಅದು ನಿರ್ವಹಿಸುವುದಾಗಿದೆ, ಉದಾಹರಣೆಗೆ:

- ದರ ನಿಗ್ರಹಣೆ
- ಟೋಕನ್ ನಿರ್ವಹಣೆ
- ಮೇಲ್ವಿಚಾರಣೆ
- ಲೋಡ್ ಜೊತೆಗೆ ಸಮತೋಲನ
- ಭದ್ರತೆ

## ರಿಮೋಟ್ MCP ಸರ್ವರ್ ಅನ್ನು Azure ಗೆ ನಿಯೋಜಿಸುವುದು

ಹಿಂದೆ ಉಲ್ಲೇಖಿಸಿದ ಮಾದರಿಯನ್ನು ನಿಯೋಜಿಸಲು ಶ್ರಮಿಸುವುದೆ ನೋಡೋಣ:

1. ರೆಪೊ ಕ್ಲೋನ್ ಮಾಡಿ

    ```bash
    git clone https://github.com/Azure-Samples/remote-mcp-apim-functions-python.git
    cd remote-mcp-apim-functions-python
    ```

1. `Microsoft.App` ಸಂಪನ್ಮೂಲ ಒದಗಿಸುವವರನ್ನು ನೋಂದಾಯಿಸಿ.

   - ನೀವು Azure CLI ಬಳಕೆಮಾಡುತ್ತಿದ್ದರೆ, `az provider register --namespace Microsoft.App --wait` ಕಾರ್ಯನಿರ್ವಹಿಸಿ.
   - ನೀವು Azure PowerShell ಬಳಕೆಮಾಡುತ್ತಿದ್ದರೆ, `Register-AzResourceProvider -ProviderNamespace Microsoft.App` ಕಾರ್ಯನಿರ್ವಹಿಸಿ. ನಂತರ ಸಮಯ ಕೊಟ್ಟು `(Get-AzResourceProvider -ProviderNamespace Microsoft.App).RegistrationState` ಚಾಲನೆ ಮಾಡಿರಿ ನೋಂದಣಿ ಪೂರೈಸಿದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಲು.

1. ಈ [azd](https://aka.ms/azd) ಆಜ್ಞೆಯನ್ನು ಚಲಾಯಿಸಿ API ನಿರ್ವಹಣೆ ಸೇವೆ, ಫಂಕ್ಷನ್ ಅಪ್ಲಿಕೇಶನ್(ಕೋಡ್ ಸಹಿತ) ಮತ್ತು ಇತರ ಅಗತ್ಯ Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ಪ್ರಾಂಜಲಗೊಳಿಸಲು

    ```shell
    azd up
    ```

    ಈ ಆಜ್ಞೆಗಳು ಮಿಂಚಿನವರು ಎಲ್ಲಾ ಕ್ಲೌಡ್ ಸಂಪನ್ಮೂಲಗಳನ್ನು Azure ನಲ್ಲಿ ನಿಯೋಜಿಸಬೇಕು

### MCP ಇನ್ಸ್ಪೆಕ್ಟರ್ ಮೂಲಕ ನಿಮ್ಮ ಸರ್ವರ್ ಅನ್ನು ಪರೀಕ್ಷಿಸುವುದು

1. **ಹೊಸ ಟರ್ಮಿನಲ್ ವಿಂಡೋದಲ್ಲಿ**, MCP ಇನ್ಸ್ಪೆಕ್ಟರ್ ಅನ್ನು ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಿ ಮತ್ತು ಚಾಲನೆ ಮಾಡಿ

    ```shell
    npx @modelcontextprotocol/inspector
    ```

    ನಿಮಗೆ ಕೆಳಗಿನಂತಿರುವ ಅಂತರ್ಜಾಲ ಕಾಣಿಸಬೇಕು:

    ![Node ಇನ್ಸ್ಪೆಕ್ಟರ್‌ಗೆ ಸಂಪರ್ಕಿಸುವುದು](../../../translated_images/kn/connect.141db0b2bd05f096.webp)

1. MCP ಇನ್ಸ್ಪೆಕ್ಟರ್ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್ URL (ಉದಾಹರಣೆಗೆ [http://127.0.0.1:6274/#resources](http://127.0.0.1:6274/#resources)) ನಲ್ಲಿ CTRL ಕ್ಲಿಕ್ ಮಾಡಿ ಲೋಡ್ ಮಾಡಿ
1. ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಪ್ರಕಾರವನ್ನು `SSE` ಗೆ ಸمراءಿಸಿ
1. URL ಅನ್ನು ನಿಮ್ಮ ಚಲಾಯುತ್ತಿರುವ API ನಿರ್ವಹಣೆ SSE ಅಂತಿಮ ಬಿಂದುಗೆ ಸೆಟ್ ಮಾಡಿ ಮತ್ತು **ಸಂಪರ್ಕಿಸಿ**:

    ```shell
    https://<apim-servicename-from-azd-output>.azure-api.net/mcp/sse
    ```

1. **ಸಾಧನಗಳನ್ನು ಪಟ್ಟಿ ಮಾಡಿ**. ಸಾಧನದ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ ಮತ್ತು **ಸಾಧನವನ್ನು ಚಾಲನೆ ಮಾಡಿ**.

ಎಲ್ಲಾ ಹಂತಗಳು ಯಶಸ್ವಿಯಾಗಿ ನಡೆದಿದ್ದರೆ, ನೀವು ಈಗ MCP ಸರ್ವರ್‌ಗೆ ಸಂಪರ್ಕ ಹೊಂದಿರುವಿರಿ ಮತ್ತು ಸಾಧನವನ್ನು ಕರೆ ಮಾಡಲು ಸಾಧ್ಯವಾಗಿದೆ.

## Azure ಗೆ MCP ಸರ್ವರ್‌ಗಳು

[Remote-mcp-functions](https://github.com/Azure-Samples/remote-mcp-functions-dotnet): ಈ ರೆಪೊಗಳ ಸಮೂಹವು Python, C# .NET ಅಥವಾ Node/TypeScript ಬಳಸಿ Azure ಫಂಕ್ಷನ್‌ಗಳೊಂದಿಗೆ ಕಸ್ಟಮ್ ರಿಮೋಟ್ MCP ಸರ್ವರ್‌ಗಳನ್ನು ನಿರ್ಮಿಸಲು ಮತ್ತು ನಿಯೋಜಿಸಲು ಕ್ವಿಕ್‌ಸ್ಟಾರ್ಟ್ ಟೆಂಪ್ಲೇಟುಗಳಾಗಿವೆ.

ಮಾದರಿಗಳು ಅಭಿವೃದ್ಧಿಪಡಿಸುವವರಿಗೆ ಪೂರ್ಣ ಪರಿಹಾರವನ್ನು ಒದಗಿಸುತ್ತವೆ, ಅದು:

- ಸ್ಥಳೀಯವಾಗಿ ನಿರ್ಮಿಸಿ ಮತ್ತು ಚಾಲನೆ: ಸ್ಥಳೀಯ ಯಂತ್ರದ ಮೇಲೆ MCP ಸರ್ವರ್ ಅನ್ನು ಅಭಿವೃದ್ಧಿಪಡಿಸಿ ಮತ್ತು ಡಿಬಗ್ ಮಾಡುವುದು
- Azure ಗೆ ನಿಯೋಜನೆ: ಸರಳ `azd up` ಆಜ್ಞೆಯಿಂದ ಕ್ಲೌಡಿಗೆ ಸುಲಭವಾಗಿ ನಿಯೋಜನೆ
- ಗ್ರಾಹಕರಿಂದ ಸಂಪರ್ಕ: VS Code ರ Copilot ಏಜೆಂಟ್ ಮೋಡ್ ಮತ್ತು MCP ಇನ್ಸ್ಪೆಕ್ಟರ್ ಟೂಲ್ ಸೇರಿದಂತೆ ವಿವಿಧ ಗ್ರಾಹಕರಿಂದ MCP ಸರ್ವರ್‌ಗೆ ಸಂಪರ್ಕ

### ಪ್ರಮುಖ ವೈಶಿಷ್ಟ್ಯಗಳು

- ವಿನ್ಯಾಸದ ಮೂಲಕ ಭದ್ರತೆ: MCP ಸರ್ವರ್ ಕೀಲಿಗಳು ಮತ್ತು HTTPS ಬಳಸಿ ಸುರಕ್ಷಿತವಾಗಿದೆ
- ಪ್ರಮಾಣಿಕರಣ ಆಯ್ಕೆಗಳು: ನಿರ್ಮಿತಿದ/auth ಮತ್ತು/ಅಥವಾ API ನಿರ್ವಹಣೆ ಬಳಸಿ OAuth ಬೆಂಬಲ
- ನೇಟ್‌ವರ್ಕ್ ಐಸೋಲೇಷನ್: Azure ವರ್ಚುವಲ್ ನೆಟ್‌ವರ್ಕ್‌ಗಳ (VNET) ಬಳಸಿಕೊಂಡು ನೆಟ್‌ವರ್ಕ್ ಐಸೋಲೇಷನ್ ಅನುಮತಿಸುತ್ತದೆ
- ಸರ್ವರ್‌ಲೆಸ್ ವಾಸ್ತುಶಿಲ್ಪ: ಸ್ಕೇಲಬಲ್, ಈವೆಂಟ್ ಚಾಲಿತ ಕಾರ್ಯಗತಗೊಳಣೆಗಾಗಿ Azure ಫಂಕ್ಷನ್‌ಗಳ ಉಪಯೋಗ
- ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿ: ಸಮಗ್ರ ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿ ಮತ್ತು ಡಿಬಗ್ ಬೆಂಬಲ
- ಸರಳ ನಿಯೋಜನೆ: Azure ಗೆ ಸುಗಮ ನಿಯೋಜನೆ ಪ್ರಕ್ರಿಯೆ

ರಿಪೋಜಿಟರಿ ಉತ್ಪಾದನಾ ಸಿದ್ಧ MCP ಸರ್ವರ್ ಅನುಷ್ಠಾನವನ್ನು ತ್ವರಿತವಾಗಿ ಪ್ರಾರಂಭಿಸಲು ಅಗತ್ಯವಿರುವ ಎಲ್ಲಾ ಸಂರಚನಾ ಫೈಲ್‌ಗಳು, ಮೂಲ ಕೋಡ್ ಮತ್ತು ಮೂಲಸೌಕರ್ಯ ವ್ಯಾಖ್ಯಾನಗಳನ್ನು ಒಳಗೊಂಡಿದೆ.

- MCP SDK ಗಳು ಬಲವಾದ MCP ಪರಿಹಾರಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಲು ಭಾಷಾ ಪ್ರತ್ಯೇಕ ಸಾಧನಗಳನ್ನು ಒದಗಿಸುತ್ತವೆ
- ಡಿಬಗ್ ಮತ್ತು ಪರೀಕ್ಷಾ ಪ್ರಕ್ರಿಯೆ ನಂಬಿಕೆಯಂತ MCP ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ ಸೂಕ್ಷ್ಮವಾಗಿದೆ
- ಪುನರಾವರ್ತಿಸಬಹುದಾದ ಪ್ರಾಂಪ್ಟ್ ಟೆಂಪ್ಲೇಟುಗಳು ಏಐ ಸಂವಾದಗಳ ಸಮತೋಲನವನ್ನು ಅನುಮತಿಸುತ್ತವೆ
- ಚೆನ್ನಾಗಿ ರೂಪಿಸಲಾಗಿದ ವರ್ಕ್ಫ್ಲೋಗಳು ಹಲವು ಸಾಧನಗಳನ್ನು ಬಳಸಿ ಸಂಕೀರ್ಣ ಕಾರ್ಯಗಳನ್ನು ವ್ಯವಸ್ಥಿತಗೊಳಿಸಬಹುದು
- MCP ಪರಿಹಾರಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸುವಾಗ ಭದ್ರತೆ, ಕಾರ್ಯಕ್ಷಮತೆ ಮತ್ತು ದೋಷ ನಿರ್ವಹಣೆಯನ್ನು ಪರಿಗಣಿಸುವುದು ಅಗತ್ಯ

## ಅಭ್ಯಾಸ

ನಿಮ್ಮ ಕ್ಷೇತ್ರದಲ್ಲಿ ನಿಜ ಜೀವನದ ಸಮಸ್ಯೆಯನ್ನು ಸಂಬಂಧಿಸಿದ ಪ್ರಾಯೋಗಿಕ MCP ವರ್ಕ್ಫ್ಲೋ ವಿನ್ಯಾಸ ಮಾಡಿರಿ:

1. ಈ ಸಮಸ್ಯೆಯನ್ನು ಪರಿಹರಿಸಲು ಉಪಯುಕ್ತವಾಗುವ 3-4 ಸಾಧನಗಳನ್ನು ಗುರುತಿಸಿ
2. ಈ ಸಾಧನಗಳು ಹೇಗೆ ಪರಸ್ಪರ ಸಂವಹನ ಮಾಡಲಿವೆ ಎಂಬ ವರ್ಕ್ಫ್ಲೋ диаг್ರಾಮ್ ಸೃಷ್ಟಿಸಿ
3. ನಿಮ್ಮ ಇಚ್ಚಿತ ಭಾಷೆಯಲ್ಲಿ ಒಂದೇ ಸಾಧನದ ಮೂಲ ಆವೃತ್ತಿಯನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಿ
4. ಮಾದರಿಯನ್ನು ನಿಮ್ಮ ಸಾಧನವನ್ನು ಪರಿಣಾಮಕಾರಿಯಾಗಿ ಬಳಸಲು ಸಹಾಯ ಮಾಡುವ ಪ್ರಾಂಪ್ಟ್ ಟೆಂಪ್ಲೇಟನ್ನು ರಚಿಸಿ
