# ব্যবহারিক বাস্তবায়ন

[![কিভাবে বাস্তব সরঞ্জাম এবং ওয়ার্কফ্লো দিয়ে MCP অ্যাপ তৈরি, পরীক্ষা এবং ডিপ্লয় করবেন](../../../translated_images/bn/05.64bea204e25ca891.webp)](https://youtu.be/vCN9-mKBDfQ)

_(এই পাঠের ভিডিও দেখার জন্য উপরের ছবিতে ক্লিক করুন)_

ব্যবহারিক বাস্তবায়ন হলো যেখানে মডেল কনটেক্সট প্রোটোকল (MCP)-এর ক্ষমতা স্পষ্ট হয়। MCP-এর পেছনের তত্ত্ব এবং স্থাপত্য বোঝা গুরুত্বপূর্ণ হলেও, প্রকৃত মূল্য আসে যখন আপনি এই ধারণাগুলি ব্যবহার করে বাস্তব সমস্যার সমাধান করার জন্য সমাধান তৈরি, পরীক্ষা এবং ডিপ্লয় করেন। এই অধ্যায়টি ধারণাগত জ্ঞানের এবং হাতে কলমে উন্নয়নের মধ্যে সেতুবন্ধন স্থাপন করে, আপনাকে MCP-ভিত্তিক অ্যাপ্লিকেশনসমূহ জীবন্ত করার প্রক্রিয়ায় পথপ্রদর্শন করে।

আপনি হোক বুদ্ধিমান সহকারী তৈরি করছেন, ব্যবসায়িক কাজের প্রবাহে AI ইন্টিগ্রেট করছেন, অথবা ডেটা প্রক্রিয়াকরণের জন্য কাস্টম টুল তৈরি করছেন, MCP একটি নমনীয় ভিত্তি প্রদান করে। এর ভাষা-নিরপেক্ষ নকশা এবং জনপ্রিয় প্রোগ্রামিং ভাষার জন্য অফিসিয়াল SDK-গুলি এটি বিস্তৃত ডেভেলপারদের জন্য সহজলভ্য করে তোলে। এই SDK-গুলি ব্যবহার করে, আপনি দ্রুত প্রোটোটাইপ তৈরি, পুনরাবৃত্তি এবং বিভিন্ন প্ল্যাটফর্ম ও পরিবেশে আপনার সমাধানগুলি স্কেল করতে পারবেন।

পরবর্তী অংশগুলোতে আপনি ব্যবহারিক উদাহরণ, নমুনা কোড এবং ডিপ্লয়মেন্ট কৌশল পাবেন যা C#, Java with Spring, TypeScript, JavaScript, এবং Python-এ MCP বাস্তবায়নের প্রদর্শন করে। এছাড়াও আপনি MCP সার্ভার ডিবাগ এবং পরীক্ষা করা, API গুলি পরিচালনা করা, এবং Azure ব্যবহার করে ক্লাউডে সমাধান ডিপ্লয় করা শিখবেন। এই হাতে কলমের সম্পদগুলি আপনার শেখার গতি বৃদ্ধি করবে এবং আত্মবিশ্বাসের সাথে দৃঢ়, প্রোডাকশন-রেডি MCP অ্যাপ্লিকেশন তৈরি করতে সাহায্য করবে।

## সংক্ষিপ্ত বিবরণ

এই পাঠ MCP বাস্তবায়নের ব্যবহারিক দিকগুলোর উপর কেন্দ্রীভূত যা একাধিক প্রোগ্রামিং ভাষায় প্রযোজ্য। আমরা দেখব কিভাবে MCP SDK-গুলি C#, Java with Spring, TypeScript, JavaScript, এবং Python-এ ব্যবহার করে দৃঢ় অ্যাপ্লিকেশন তৈরি করা যায়, MCP সার্ভার ডিবাগ এবং পরীক্ষা করা যায়, এবং পুনঃব্যবহারযোগ্য রিসোর্স, প্রম্পট এবং টুলস তৈরি করা যায়।

## শেখার লক্ষ্য

এই পাঠ শেষে, আপনি সক্ষম হবেন:

- বিভিন্ন প্রোগ্রামিং ভাষায় অফিসিয়াল SDK ব্যবহার করে MCP সমাধান বাস্তবায়ন করতে
- MCP সার্ভারগুলোকে উপায়ক্রমে ডিবাগ এবং পরীক্ষা করতে
- সার্ভার ফিচার (রিসোর্স, প্রম্পট, এবং টুল) তৈরি এবং ব্যবহার করতে
- জটিল কাজের জন্য কার্যকর MCP ওয়ার্কফ্লো ডিজাইন করতে
- পারফরম্যান্স এবং নির্ভরযোগ্যতার জন্য MCP বাস্তবায়ন অপ্টিমাইজ করতে

## মূল সার্ভার ফিচারসমূহ

MCP সার্ভারগুলি এই ফিচারগুলির যে কোনো সংমিশ্রণ বাস্তবায়ন করতে পারে:

### রিসোর্সসমূহ

রিসোর্স ব্যবহারকারী বা AI মডেলের ব্যবহারের জন্য প্রসঙ্গ এবং তথ্য প্রদান করে:

- ডকুমেন্ট রিপোজিটরি
- জ্ঞানভিত্তিক সংগ্রহস্থল
- কাঠামোবদ্ধ ডেটা উৎস
- ফাইল সিস্টেম

### প্রম্পটসমূহ

প্রম্পট হলো ব্যবহারকারীদের জন্য টেমপ্লেটকৃত মেসেজ এবং ওয়ার্কফ্লো:

- পূর্বনির্ধারিত সংলাপ টেমপ্লেট
- নির্দেশিত ইন্টারঅ্যাকশন প্যাটার্ন
- বিশেষায়িত ডায়লগ গঠন

### টুলস

টুল হলো AI মডেলের কার্যকর করার জন্য ফাংশনসমূহ:

- ডেটা প্রক্রিয়াকরণ উপযোগী সরঞ্জাম
- বহির্গত API সমন্বয়
- গণনামূলক ক্ষমতা
- অনুসন্ধান কার্যকারিতা

## নমুনা বাস্তবায়ন: C# বাস্তবায়ন

অফিসিয়াল C# SDK রিপোজিটরিটি MCP-এর বিভিন্ন দিক প্রদর্শনকারী কয়েকটি নমুনা বাস্তবায়ন ধারণ করে:

- **বেসিক MCP ক্লায়েন্ট**: কিভাবে একটি MCP ক্লায়েন্ট তৈরি করে টুল কল করা যায় তা দেখানোর সরল উদাহরণ
- **বেসিক MCP সার্ভার**: মৌলিক টুল নিবন্ধনের সঙ্গে একটি ন্যূনতম সার্ভার বাস্তবায়ন
- **অ্যাডভান্সড MCP সার্ভার**: টুল নিবন্ধন, প্রমাণীকরণ এবং ত্রুটি হ্যান্ডলিংসহ পূর্ণাঙ্গ সার্ভার
- **ASP.NET ইন্টিগ্রেশন**: ASP.NET Core-র সঙ্গে ইন্টিগ্রেশনের উদাহরণ
- **টুল বাস্তবায়ন প্যাটার্ন**: বিভিন্ন জটিলতার টুল বাস্তবায়ন প্যাটার্ন

MCP C# SDK এখন প্রিভিউতে রয়েছে এবং API পরিবর্তন হতে পারে। SDK আপডেট হওয়ার সাথে সাথে এই ব্লগ নিয়মিত হালনাগাদ করা হবে।

### প্রধান বৈশিষ্ট্যসমূহ

- [C# MCP Nuget ModelContextProtocol](https://www.nuget.org/packages/ModelContextProtocol)
- আপনার [প্রথম MCP সার্ভার নির্মাণ](https://devblogs.microsoft.com/dotnet/build-a-model-context-protocol-mcp-server-in-csharp/)।

## API পরিচালনা

Azure API Management হলো MCP সার্ভারগুলোকে নিরাপদ করার জন্য দুর্দান্ত উপায়। ধারণাটি হলো আপনার MCP সার্ভারের সামনে একটি Azure API Management ইনস্ট্যান্স স্থাপন করা, যা নীচের বৈশিষ্ট্য গুলো পরিচালনা করে:

- রেট সীমাবদ্ধকরণ
- টোকেন পরিচালনা
- পর্যবেক্ষণ
- লোড ব্যালেন্সিং
- নিরাপত্তা

### Azure নমুনা

এখানে একটি Azure নমুনা, অর্থাৎ [MCP সার্ভার তৈরি এবং Azure API Management দিয়ে নিরাপদকরণ](https://github.com/Azure-Samples/remote-mcp-apim-functions-python)।

নিচের ছবিতে অথরাইজেশন ফ্লো কিভাবে ঘটে দেখুন:

![APIM-MCP](https://github.com/Azure-Samples/remote-mcp-apim-functions-python/blob/main/mcp-client-authorization.gif?raw=true)

উপরের ছবিতে এই ঘটনাগুলো ঘটে:

## Remote MCP সার্ভার Azure-এ ডিপ্লয় করা

চলুন দেখি আগের উল্লেখিত নমুনাটি ডিপ্লয় করা যায় কি না:

1. রিপো ক্লোন করুন

    ```bash
    git clone https://github.com/Azure-Samples/remote-mcp-apim-functions-python.git
    cd remote-mcp-apim-functions-python
    ```

1. `Microsoft.App` রিসোর্স প্রদানকারী নিবন্ধন করুন।

   - Azure CLI ব্যবহার করলে চালান: `az provider register --namespace Microsoft.App --wait`.
   - Azure PowerShell ব্যবহার করলে চালান: `Register-AzResourceProvider -ProviderNamespace Microsoft.App`। পরে কিছু সময় পর `(Get-AzResourceProvider -ProviderNamespace Microsoft.App).RegistrationState` চালিয়ে নিবন্ধন সম্পূর্ণ হয়েছে কিনা দেখুন।

1. এই [azd](https://aka.ms/azd) কমান্ড চালান যা API ম্যানেজমেন্ট সার্ভিস, কোডসহ ফাংশন অ্যাপ এবং অন্যান্য প্রয়োজনীয় Azure রিসোর্সগুলো প্রস্তুত করবে

    ```shell
    azd up
    ```

    এই কমান্ডগুলো Azure-তে সব ক্লাউড রিসোর্স ডিপ্লয় করবে

### MCP Inspector দিয়ে আপনার সার্ভার পরীক্ষা

1. **নতুন টার্মিনাল উইন্ডোতে**, MCP Inspector ইনস্টল ও চালান

    ```shell
    npx @modelcontextprotocol/inspector
    ```

    আপনাকে এই রকম একটি ইন্টারফেস দেখতে হবে:

    ![Connect to Node inspector](../../../translated_images/bn/connect.141db0b2bd05f096.webp)

1. অ্যাপে প্রদর্শিত URL (যেমন [http://127.0.0.1:6274/#resources](http://127.0.0.1:6274/#resources)) থেকে MCP Inspector ওয়েব অ্যাপটি CTRL ক্লিক করে লোড করুন
1. ট্রান্সপোর্ট টাইপ `SSE` সেট করুন
1. চালু থাকা আপনার API Management SSE এন্ডপয়েন্টের URL দিন এবং **Connect** করুন:

    ```shell
    https://<apim-servicename-from-azd-output>.azure-api.net/mcp/sse
    ```

1. **List Tools**। একটি টুলে ক্লিক করে **Run Tool** করুন।

যদি সব ধাপ সফল হয়, আপনি এখন MCP সার্ভারে সংযুক্ত এবং একটি টুল কল করতে সক্ষম হবেন।

## Azure-র জন্য MCP সার্ভারগুলো

[Remote-mcp-functions](https://github.com/Azure-Samples/remote-mcp-functions-dotnet): এই রিপোজিটরিজগুলো Azure Functions দিয়ে Python, C# .NET অথবা Node/TypeScript ব্যবহার করে কাস্টম রিমোট MCP (Model Context Protocol) সার্ভার নির্মাণ ও ডিপ্লয়ের জন্য দ্রুত শুরু করা টেমপ্লেট।

স্যাম্পল সমূহ একটি পূর্ণাঙ্গ সমাধান প্রদান করে, যা ডেভেলপারদের সক্ষম করে:

- স্থানীয়ভাবে তৈরি ও চালানো: MCP সার্ভার স্থানীয় মেশিনে ডেভেলপ এবং ডিবাগ করা
- Azure-তে ডিপ্লয় করা: সহজে একটি `azd up` কমান্ড দিয়ে ক্লাউডে ডিপ্লয়
- ক্লায়েন্ট থেকে সংযোগ: ভিন্ন ক্লায়েন্ট যেমন VS Code-এর Copilot এজেন্ট মোড এবং MCP Inspector টুল থেকে MCP সার্ভারে সংযোগ স্থাপন

### প্রধান বৈশিষ্ট্যসমূহ

- নকশায় নিরাপত্তা: MCP সার্ভার কী এবং HTTPS ব্যবহার করে সুরক্ষিত
- প্রমাণীকরণ বিকল্প: বিল্ট-ইন অথ এবং/অথবা API Management ব্যবহার করে OAuth সাপোর্ট
- নেটওয়ার্ক বিচ্ছিন্নকরণ: Azure ভার্চুয়াল নেটওয়ার্ক (VNET) ব্যবহার করে নেটওয়ার্ক বিচ্ছিন্নকরণ অনুমতি
- সার্ভারহীন স্থাপত্য: Azure Functions দ্বারা স্কেলেবল, ইভেন্ট-চালিত এক্সিকিউশন
- স্থানীয় উন্নয়ন: ব্যাপক স্থানীয় ডেভেলপমেন্ট ও ডিবাগিং সাপোর্ট
- সহজ ডিপ্লয়মেন্ট: Azure-এ সরলীকৃত ডিপ্লয়মেন্ট প্রক্রিয়া

রিপোজিটরিতে সব প্রয়োজনীয় কনফিগারেশন ফাইল, সোর্স কোড, এবং অবকাঠামো সংজ্ঞা রয়েছে যা একটি প্রোডাকশন-রেডি MCP সার্ভার বাস্তবায়নে দ্রুত শুরু করতে সহায়ক।

- [Azure Remote MCP Functions Python](https://github.com/Azure-Samples/remote-mcp-functions-python) - Python সহ Azure Functions ব্যবহার করে MCP-এর নমুনা বাস্তবায়ন

- [Azure Remote MCP Functions .NET](https://github.com/Azure-Samples/remote-mcp-functions-dotnet) - C# .NET সহ Azure Functions ব্যবহার করে MCP-এর নমুনা বাস্তবায়ন

- [Azure Remote MCP Functions Node/Typescript](https://github.com/Azure-Samples/remote-mcp-functions-typescript) - Node/TypeScript সহ Azure Functions ব্যবহার করে MCP-এর নমুনা বাস্তবায়ন।

## মূল বিষয়গুলো

- MCP SDK গুলো ভাষাভিত্তিক সরঞ্জাম প্রদান করে দৃঢ় MCP সমাধান বাস্তবায়নের জন্য
- ডিবাগিং এবং পরীক্ষণ প্রক্রিয়া MCP অ্যাপ্লিকেশনগুলোর নির্ভরযোগ্যতার জন্য অপরিহার্য
- পুনঃব্যবহারযোগ্য প্রম্পট টেমপ্লেটগুলি ধারাবাহিক AI ইন্টারঅ্যাকশন সুবিধা দেয়
- ভাল ডিজাইনকৃত ওয়ার্কফ্লো বিভিন্ন টুল ব্যবহার করে জটিল কাজসমূহ পরিকল্পনা করতে পারে
- MCP সমাধান বাস্তবায়নের সময় নিরাপত্তা, পারফরম্যান্স এবং ত্রুটি হ্যান্ডলিং বিবেচনা করা প্রয়োজন

## অনুশীলন

আপনার ডোমেইনে একটি বাস্তব জগতের সমস্যার সমাধান সম্পর্কিত একটি ব্যবহারিক MCP ওয়ার্কফ্লো ডিজাইন করুন:

1. এই সমস্যার সমাধানে সহায়ক ৩-৪ টি টুল সনাক্ত করুন
2. এই টুলগুলো কিভাবে ইন্টারঅ্যাক্ট করে তা দেখানোর জন্য একটি ওয়ার্কফ্লো ডায়াগ্রাম তৈরি করুন
3. পছন্দের ভাষায় একটি টুলের মৌলিক সংস্করণ বাস্তবায়ন করুন
4. একটি প্রম্পট টেমপ্লেট তৈরি করুন যা মডেলকে আপনার টুল কার্যকরভাবে ব্যবহারে সহায়তা করবে
