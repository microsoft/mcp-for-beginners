# বাস্তবায়ন

[![কিভাবে MCP অ্যাপস তৈরি, পরীক্ষা, এবং বাস্তবায়ন করবেন বাস্তবিক টুলস এবং ওয়ার্কফ্লো ব্যবহার করে](../../../translated_images/bn/05.64bea204e25ca891.webp)](https://youtu.be/vCN9-mKBDfQ)

_(এই পাঠের ভিডিও দেখার জন্য উপরের ছবিতে ক্লিক করুন)_

বাস্তবায়ন হল এমন একটি স্থান যেখানে মডেল কন্টেক্সট প্রোটোকল (MCP) এর শক্তি স্পষ্ট হয়ে ওঠে। MCP এর পেছনে সুত্র ও আর্কিটেকচার বোঝা গুরুত্বপূর্ণ হলেও, প্রকৃত মূল্য তখনই পাওয়া যায় যখন আপনি এই ধারণাগুলো ব্যবহার করে বাস্তব বিশ্ব সমস্যাগুলোর সমাধান করার জন্য সমাধান তৈরি, পরীক্ষা, এবং বাস্তবায়ন করেন। এই অধ্যায় ধারণাগত জ্ঞান এবং হাতে কলমে উন্নয়নের মধ্যে সেতুবন্ধনের কাজ করে, আপনাকে MCP-ভিত্তিক অ্যাপ্লিকেশন জীবন্ত করে তোলার প্রক্রিয়ায় নির্দেশনা দেয়।

আপনি যদি বুদ্ধিমান সহকারী তৈরি করেন, AI-কে ব্যবসায়িক ওয়ার্কফ্লোতে সংহত করেন, অথবা ডেটা প্রক্রিয়াকরণের জন্য কাস্টম টুল তৈরি করেন, MCP নমনীয় একটি ভিত্তি প্রদান করে। এর ভাষা-নিরপেক্ষ নকশা এবং জনপ্রিয় প্রোগ্রামিং ভাষার জন্য অফিসিয়াল SDK গুলো অনেক ধরনের ডেভেলপারদের কাছে অ্যাক্সেসযোগ্য করে তোলে। এই SDK গুলো ব্যবহার করে আপনি দ্রুত প্রোটোটাইপ তৈরি, পুনরাবৃত্তি এবং আপনার সমাধান গুলো বিভিন্ন প্ল্যাটফর্ম এবং পরিবেশে স্কেল করতে পারেন।

পরবর্তী অংশগুলোতে, আপনি পাবেন বাস্তব উদাহরণ, নমুনা কোড, এবং বাস্তবায়ন কৌশল যা দেখাবে কিভাবে MCP কে C#, Java with Spring, TypeScript, JavaScript, এবং Python এ বাস্তবায়ন করা যায়। আপনি শিখবেন কিভাবে MCP সার্ভারগুলি ডিবাগ ও পরীক্ষা করবেন, API গুলো পরিচালনা করবেন, এবং Azure ব্যবহার করে ক্লাউডে সমাধানগুলি বাস্তবায়ন করবেন। এই হাতে কলমের রিসোর্সগুলি আপনার শেখাকে ত্বরান্বিত করবে এবং আপনাকে আত্মবিশ্বাসের সাথে মজবুত, প্রোডাকশন-রেডি MCP অ্যাপ্লিকেশন তৈরি করতে সাহায্য করবে।

## সারাংশ

এই পাঠ MCP বাস্তবায়নের ব্যবহারিক দিকগুলোকে বিভিন্ন প্রোগ্রামিং ভাষায় ফোকাস করে। আমরা দেখব কিভাবে MCP SDK গুলো C#, Java with Spring, TypeScript, JavaScript, এবং Python এ ব্যবহার করে মজবুত অ্যাপ্লিকেশন তৈরি করা যায়, MCP সার্ভার ডিবাগ ও পরীক্ষা করা যায়, এবং পুনর্ব্যবহারযোগ্য রিসোর্স, প্রম্পট, এবং টুল তৈরি করা যায়।

## শেখার উদ্দেশ্য

এই পাঠের শেষে আপনি সক্ষম হবেন:

- বিভিন্ন প্রোগ্রামিং ভাষায় অফিসিয়াল SDK ব্যবহার করে MCP সমাধান বাস্তবায়ন করতে
- MCP সার্ভারগুলি সুনির্দিষ্টভাবে ডিবাগ ও পরীক্ষা করতে
- সার্ভার ফিচার (রিসোর্স, প্রম্পট, এবং টুল) তৈরি ও ব্যবহার করতে
- জটিল কাজের জন্য কার্যকর MCP ওয়ার্কফ্লো ডিজাইন করতে
- পারফরমেন্স এবং নির্ভরযোগ্যতার জন্য MCP বাস্তবায়নগুলি অপটিমাইজ করতে

## মূল সার্ভার ফিচারগুলো

MCP সার্ভারগুলো নিম্নলিখিত ফিচারগুলোর যেকোনো কম্বিনেশন বাস্তবায়ন করতে পারে:

### রিসোর্স

রিসোর্স ব্যবহারকারী বা AI মডেলের জন্য প্রাসঙ্গিক তথ্য এবং ডেটা সরবরাহ করে:

- ডকুমেন্ট রিপোজিটরিজ
- জ্ঞানের ভিত্তি
- কাঠামোবদ্ধ ডাটা উৎস
- ফাইল সিস্টেম

### প্রম্পট

প্রম্পট হলো ব্যবহারকারীদের জন্য টেমপ্লেট করা বার্তা এবং ওয়ার্কফ্লো:

- পূর্বনির্ধারিত কথোপকথন টেমপ্লেটগুলো
- পরিচালিত ইন্টারঅ্যাকশন প্যাটার্নগুলো
- বিশেষায়িত ডায়ালগ কাঠামো

### টুল

টুল হলো AI মডেলের জন্য কার্যকরযোগ্য ফাংশন:

- ডেটা প্রক্রিয়াকরণ উপযোগী টুলস
- বহিরাগত API সংহতকরণ
- গণনার সক্ষমতা
- অনুসন্ধান কার্যকারিতা

## নমুনা বাস্তবায়ন: C# বাস্তবায়ন

অফিসিয়াল C# SDK রেপোজিটরিতে MCP এর বিভিন্ন দিক দেখানো কয়েকটি নমুনা বাস্তবায়ন রয়েছে:

- **মৌলিক MCP ক্লায়েন্ট**: একটি MCP ক্লায়েন্ট তৈরি এবং টুল কল করার সহজ উদাহরণ
- **মৌলিক MCP সার্ভার**: বেসিক টুল নিবন্ধনের minimalist সার্ভার বাস্তবায়ন
- **উন্নত MCP সার্ভার**: সম্পূর্ণ ফিচারসহ সার্ভার, টুল নিবন্ধন, প্রমাণীকরণ এবং ত্রুটি পরিচালনা সহ
- **ASP.NET ইন্টিগ্রেশন**: ASP.NET Core সাথে ইন্টিগ্রেশন দেখানো উদাহরণ
- **টুল বাস্তবায়ন প্যাটার্ন**: বিভিন্ন জটিলতার সঙ্গে টুল বাস্তবায়নের প্যাটার্নগুলো

MCP C# SDK এখনো প্রিভিউ পর্যায়ে রয়েছে এবং API গুলো পরিবর্তিত হতে পারে। SDK আপডেট হওয়ার সঙ্গে সঙ্গে আমরা এই ব্লগ নিয়মিত আপডেট করবো।

## API ব্যবস্থাপনা

Azure API Management MCP সার্ভারকে নিরাপদ করার জন্য একটি চমৎকার উত্তর। ধারণাটি হল একটি Azure API Management ইন্সট্যান্স MCP সার্ভারের সামনে রাখা এবং এটি নিয়ন্ত্রণ ও পরিচালনার নিম্নলিখিত বৈশিষ্ট্যগুলো সামলানো:

- রেট সীমাবদ্ধকরণ
- টোকেন ব্যবস্থাপনা
- মনিটরিং
- লোড ব্যালান্সিং
- সুরক্ষা

### Azure নমুনা

এখানে একটি Azure নমুনা দেওয়া হলো, অর্থাৎ [MCP সার্ভার তৈরি করা এবং Azure API Management দিয়ে এটি সুরক্ষিত করা](https://github.com/Azure-Samples/remote-mcp-apim-functions-python)।

নীচের ছবিতে দেখুন কিভাবে অনুমোদন প্রক্রিয়া ঘটছে:

![APIM-MCP](https://github.com/Azure-Samples/remote-mcp-apim-functions-python/blob/main/mcp-client-authorization.gif?raw=true)

উপরের ছবিতে নিম্নলিখিত ঘটে:

- Microsoft Entra ব্যবহার করে প্রমাণীকরণ/অনুমোদন সম্পন্ন হয়।
- Azure API Management একটি গেটওয়ে হিসাবে কাজ করে এবং পলিসি ব্যবহার করে ট্র্যাফিক পরিচালনা ও নিয়ন্ত্রণ করে।
- Azure Monitor বিশ্লেষণের জন্য সমস্ত অনুরোধ লগ করে।

## রিমোট MCP সার্ভার Azure-এ স্থাপন

আগে উল্লেখ করা নমুনাটি কি আমরা স্থাপন করতে পারি দেখি:

1. রেপো ক্লোন করুন

    ```bash
    git clone https://github.com/Azure-Samples/remote-mcp-apim-functions-python.git
    cd remote-mcp-apim-functions-python
    ```

1. `Microsoft.App` রিসোর্স প্রোভাইডার রেজিস্টার করুন।

   - যদি Azure CLI ব্যবহার করেন, তাহলে রান করুন `az provider register --namespace Microsoft.App --wait`।
   - যদি Azure PowerShell ব্যবহার করেন, তাহলে রান করুন `Register-AzResourceProvider -ProviderNamespace Microsoft.App`। কিছু সময় পর `(Get-AzResourceProvider -ProviderNamespace Microsoft.App).RegistrationState` রান করে দেখে নিন রেজিস্ট্রেশন সম্পূর্ণ হয়েছে কিনা।

1. এই [azd](https://aka.ms/azd) কমান্ড রান করুন api প্রসেস ম্যানেজমেন্ট সার্ভিস, ফাংশন অ্যাপ(কোড সহ) এবং অন্য সব প্রয়োজনীয় Azure রিসোর্স provision করতে।

    ```shell
    azd up
    ```

    এই কমান্ডগুলি Azure উপর সব ক্লাউড রিসোর্স ডিপ্লয় করবে।

### MCP Inspector দিয়ে আপনার সার্ভার পরীক্ষা করা

1. একটি **নতুন টার্মিনাল উইন্ডোতে**, MCP Inspector ইনস্টল করুন এবং চালান

    ```shell
    npx @modelcontextprotocol/inspector
    ```

    আপনি এমন একটি ইন্টারফেস দেখতে পাবেন:

    ![Connect to Node inspector](../../../translated_images/bn/connect.141db0b2bd05f096.webp)

1. MCP Inspector ওয়েব অ্যাপ লোড করতে URL এর উপরে CTRL ক্লিক করুন (যেমন [http://127.0.0.1:6274/#resources](http://127.0.0.1:6274/#resources))
1. ট্রান্সপোর্ট টাইপ `SSE` সিলেক্ট করুন
1. URL সিলেক্ট করুন আপনার চলছে এমন API Management SSE endpoint যেটি `azd up` এর পর প্রদর্শিত হয় এবং **Connect** করুন:

    ```shell
    https://<apim-servicename-from-azd-output>.azure-api.net/mcp/sse
    ```

1. **List Tools**। একটি টুলে ক্লিক করে **Run Tool** করুন।  

যদি সব ধাপ ঠিকমতো কাজ করে, তাহলে আপনি MCP সার্ভারের সাথে সংযুক্ত হয়েছেন এবং একটি টুল কল করতে পেরেছেন।

## Azure এর জন্য MCP সার্ভার

[Remote-mcp-functions](https://github.com/Azure-Samples/remote-mcp-functions-dotnet): এই রেপোজিটরি সেট MCP সার্ভার তৈরি ও ডিপ্লয় করার দ্রুত সূচনা টেমপ্লেট সরবরাহ করে Azure Functions ব্যবহার করে Python, C# .NET অথবা Node/TypeScript দিয়ে।

স্যাম্পলগুলো ডেভেলপারদের জন্য একটি সম্পূর্ণ সমাধান প্রদান করে যা:

- স্থানীয়ভাবে বিল্ড ও রান করা: একটি MCP সার্ভার স্থানীয় মেশিনে ডেভেলপ ও ডিবাগ করা
- Azure তে ডিপ্লয় করা: সহজে ক্লাউডে স্থাপন করার জন্য একটি সরল azd up কমান্ড
- ক্লায়েন্ট থেকে সংযোগ করা: MCP সার্ভারের সাথে সংযোগ বিভিন্ন ক্লায়েন্ট থেকে, যেমন VS Code এর Copilot এজেন্ট মোড এবং MCP Inspector টুল

### মূল ফিচার

- নিরাপত্তা ডিজাইনের মধ্যে: MCP সার্ভার HTTPS এবং কী ব্যবহার করে সুরক্ষিত
- প্রমাণীকরণ বিকল্প: বিল্ট-ইন অথ এবং/অথবা API Management ব্যবহার করে OAuth সাপোর্ট করে
- নেটওয়ার্ক বিচ্ছিন্নকরণ: Azure Virtual Networks (VNET) ব্যবহার করে নেটওয়ার্ক বিচ্ছিন্নতা প্রদান
- সার্ভারহীন আর্কিটেকচার: স্কেলযোগ্য, ইভেন্ট-চালিত এক্সিকিউশনের জন্য Azure Functions ব্যবহার
- স্থানীয় ডেভেলপমেন্ট: ব্যাপক স্থানীয় ডেভেলপমেন্ট এবং ডিবাগিং সাপোর্ট
- সহজ ডিপ্লয়মেন্ট: Azure এ স্থাপনের জন্য স্ট্রীমলাইন করা প্রক্রিয়া

রেপোজিটরিতে সমস্ত প্রয়োজনীয় কনফিগারেশন ফাইল, সোর্স কোড, এবং অবকাঠামোর ডেফিনিশন রয়েছে যা দ্রুত উৎপাদন-সক্ষম MCP সার্ভার বাস্তবায়নের জন্য প্রস্তুতি নেয়।

- [Azure Remote MCP Functions Python](https://github.com/Azure-Samples/remote-mcp-functions-python) - Azure Functions ব্যবহার করে Python এ MCP এর নমুনা বাস্তবায়ন

- [Azure Remote MCP Functions .NET](https://github.com/Azure-Samples/remote-mcp-functions-dotnet) - Azure Functions ব্যবহার করে C# .NET এ MCP এর নমুনা বাস্তবায়ন

- [Azure Remote MCP Functions Node/Typescript](https://github.com/Azure-Samples/remote-mcp-functions-typescript) - Azure Functions ব্যবহার করে Node/TypeScript এ MCP এর নমুনা বাস্তবায়ন

## মূল takeaway

- MCP SDK গুলো ভাষা-নির্দিষ্ট টুল সরবরাহ করে যা শক্তিশালী MCP সমাধান বাস্তবায়নের জন্য প্রয়োজনীয়
- ডিবাগ ও পরীক্ষা প্রক্রিয়া নির্ভরযোগ্য MCP অ্যাপ্লিকেশনের জন্য অত্যন্ত গুরুত্বপূর্ণ
- পুনর্ব্যবহারযোগ্য প্রম্পট টেমপ্লেটগুলি AI ইন্টারঅ্যাকশনগুলোতে ধারাবাহিকতা যোগ করে
- ভালো ডিজাইনকৃত ওয়ার্কফ্লো জটিল কাজগুলো একাধিক টুল ব্যবহার করে কার্যকরভাবে সংগঠিত করতে পারে
- MCP সমাধান বাস্তবায়নে সুরক্ষা, পারফরমেন্স, এবং ত্রুটি পরিচালনার বিষয়গুলো বিবেচনা করা প্রয়োজন

## ব্যায়াম

আপনার ডোমেইনে একটি বাস্তব MCP ওয়ার্কফ্লো ডিজাইন করুন যা একটি বাস্তবজীবনের সমস্যা সমাধান করে:

1. ৩-৪টি টুল চিহ্নিত করুন যা এই সমস্যার সমাধানে উপকারী হবে
2. একটি ওয়ার্কফ্লো ডায়াগ্রাম তৈরি করুন যা দেখায় এই টুলগুলো কিভাবে ইন্টারঅ্যাকশন করে
3. পছন্দের ভাষায় একটি টুলের মৌলিক সংস্করণ বাস্তবায়ন করুন
4. একটি প্রম্পট টেমপ্লেট তৈরি করুন যা মডেলকে আপনার টুল কার্যকরভাবে ব্যবহারে সাহায্য করবে
