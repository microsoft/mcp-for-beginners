# পরিবর্তনলিপি: MCP ফর বিগিনার্স কারিকুলাম

এই নথিটি মডেল কনটেক্সট প্রোটোকল (MCP) ফর বিগিনার্স কারিকুলামে করা সমস্ত গুরুত্বপূর্ণ পরিবর্তনের রেকর্ড হিসেবে কাজ করে। পরিবর্তনগুলি উল্টো কালানুক্রমিক ক্রমে (নতুন পরিবর্তন প্রথমে) নথিভুক্ত করা হয়েছে।

## ডিসেম্বর ১৮, ২০২৫

### সিকিউরিটি ডকুমেন্টেশন আপডেট - MCP স্পেসিফিকেশন ২০২৫-১১-২৫

#### MCP সিকিউরিটি বেস্ট প্র্যাকটিসেস (02-Security/mcp-best-practices.md) - স্পেসিফিকেশন ভার্সন আপডেট
- **প্রোটোকল ভার্সন আপডেট**: সর্বশেষ MCP স্পেসিফিকেশন ২০২৫-১১-২৫ (২৫ নভেম্বর, ২০২৫ প্রকাশিত) রেফারেন্সে আপডেট করা হয়েছে
  - সমস্ত স্পেসিফিকেশন ভার্সন রেফারেন্স ২০২৫-০৬-১৮ থেকে ২০২৫-১১-২৫ এ আপডেট করা হয়েছে
  - ডকুমেন্ট তারিখ রেফারেন্স আগস্ট ১৮, ২০২৫ থেকে ডিসেম্বর ১৮, ২০২৫ এ আপডেট করা হয়েছে
  - নিশ্চিত করা হয়েছে যে সমস্ত স্পেসিফিকেশন URL বর্তমান ডকুমেন্টেশনের দিকে নির্দেশ করে
- **কনটেন্ট ভ্যালিডেশন**: সর্বশেষ মানদণ্ডের বিরুদ্ধে সিকিউরিটি বেস্ট প্র্যাকটিসের ব্যাপক যাচাই
  - **মাইক্রোসফট সিকিউরিটি সলিউশনস**: প্রম্পট শিল্ডস (পূর্বে "জেলব্রেক রিস্ক ডিটেকশন"), আজুর কনটেন্ট সেফটি, মাইক্রোসফট এন্ট্রা আইডি, এবং আজুর কী ভল্টের বর্তমান টার্মিনোলজি ও লিঙ্ক যাচাই করা হয়েছে
  - **OAuth 2.1 সিকিউরিটি**: সর্বশেষ OAuth সিকিউরিটি বেস্ট প্র্যাকটিসের সাথে সামঞ্জস্য নিশ্চিত করা হয়েছে
  - **OWASP স্ট্যান্ডার্ডস**: LLMs এর জন্য OWASP টপ ১০ রেফারেন্সগুলি বর্তমান আছে তা যাচাই করা হয়েছে
  - **আজুর সার্ভিসেস**: সমস্ত মাইক্রোসফট আজুর ডকুমেন্টেশন লিঙ্ক এবং বেস্ট প্র্যাকটিস যাচাই করা হয়েছে
- **স্ট্যান্ডার্ডস সামঞ্জস্য**: সমস্ত রেফারেন্সকৃত সিকিউরিটি স্ট্যান্ডার্ড বর্তমান হিসেবে নিশ্চিত করা হয়েছে
  - NIST AI রিস্ক ম্যানেজমেন্ট ফ্রেমওয়ার্ক
  - ISO 27001:2022
  - OAuth 2.1 সিকিউরিটি বেস্ট প্র্যাকটিসেস
  - আজুর সিকিউরিটি এবং কমপ্লায়েন্স ফ্রেমওয়ার্কস
- **ইমপ্লিমেন্টেশন রিসোর্সেস**: সমস্ত ইমপ্লিমেন্টেশন গাইড লিঙ্ক এবং রিসোর্স যাচাই করা হয়েছে
  - আজুর API ম্যানেজমেন্ট অথেন্টিকেশন প্যাটার্নস
  - মাইক্রোসফট এন্ট্রা আইডি ইন্টিগ্রেশন গাইডস
  - আজুর কী ভল্ট সিক্রেটস ম্যানেজমেন্ট
  - ডেভসেকঅপস পাইপলাইনস এবং মনিটরিং সলিউশনস

### ডকুমেন্টেশন কোয়ালিটি অ্যাসিউরেন্স
- **স্পেসিফিকেশন কমপ্লায়েন্স**: নিশ্চিত করা হয়েছে যে সমস্ত বাধ্যতামূলক MCP সিকিউরিটি রিকোয়ারমেন্টস (MUST/MUST NOT) সর্বশেষ স্পেসিফিকেশনের সাথে সামঞ্জস্যপূর্ণ
- **রিসোর্স কারেন্সি**: সমস্ত বাইরের লিঙ্ক মাইক্রোসফট ডকুমেন্টেশন, সিকিউরিটি স্ট্যান্ডার্ডস, এবং ইমপ্লিমেন্টেশন গাইডসের সাথে যাচাই করা হয়েছে
- **বেস্ট প্র্যাকটিসেস কাভারেজ**: অথেন্টিকেশন, অথরাইজেশন, AI-নির্দিষ্ট হুমকি, সাপ্লাই চেইন সিকিউরিটি, এবং এন্টারপ্রাইজ প্যাটার্নসের ব্যাপক কাভারেজ নিশ্চিত করা হয়েছে

## অক্টোবর ৬, ২০২৫

### গেটিং স্টার্টেড সেকশন সম্প্রসারণ – অ্যাডভান্সড সার্ভার ব্যবহার ও সিম্পল অথেন্টিকেশন

#### অ্যাডভান্সড সার্ভার ব্যবহার (03-GettingStarted/10-advanced)
- **নতুন অধ্যায় যুক্ত**: অ্যাডভান্সড MCP সার্ভার ব্যবহারের ব্যাপক গাইড পরিচয় করানো হয়েছে, যা নিয়মিত এবং লো-লেভেল সার্ভার আর্কিটেকচার উভয়ই কভার করে।
  - **নিয়মিত বনাম লো-লেভেল সার্ভার**: উভয় পদ্ধতির জন্য পাইথন এবং টাইপস্ক্রিপ্টে বিস্তারিত তুলনা ও কোড উদাহরণ।
  - **হ্যান্ডলার-ভিত্তিক ডিজাইন**: স্কেলেবল, নমনীয় সার্ভার ইমপ্লিমেন্টেশনের জন্য হ্যান্ডলার-ভিত্তিক টুল/রিসোর্স/প্রম্পট ম্যানেজমেন্টের ব্যাখ্যা।
  - **প্র্যাকটিক্যাল প্যাটার্নস**: বাস্তব জীবনের পরিস্থিতি যেখানে লো-লেভেল সার্ভার প্যাটার্নগুলি উন্নত ফিচার এবং আর্কিটেকচারের জন্য উপকারী।

#### সিম্পল অথেন্টিকেশন (03-GettingStarted/11-simple-auth)
- **নতুন অধ্যায় যুক্ত**: MCP সার্ভারে সিম্পল অথেন্টিকেশন ইমপ্লিমেন্ট করার ধাপে ধাপে গাইড।
  - **অথ কনসেপ্টস**: অথেন্টিকেশন বনাম অথরাইজেশন এবং ক্রেডেনশিয়াল হ্যান্ডলিং স্পষ্ট ব্যাখ্যা।
  - **বেসিক অথ ইমপ্লিমেন্টেশন**: পাইথন (স্টারলেট) এবং টাইপস্ক্রিপ্ট (এক্সপ্রেস) এ মিডলওয়্যার-ভিত্তিক অথেন্টিকেশন প্যাটার্নস, কোড স্যাম্পলসহ।
  - **অ্যাডভান্সড সিকিউরিটিতে অগ্রগতি**: সিম্পল অথ দিয়ে শুরু করে OAuth 2.1 এবং RBAC এ উন্নতির নির্দেশনা, উন্নত সিকিউরিটি মডিউল রেফারেন্সসহ।

এই সংযোজনগুলি আরও শক্তিশালী, সুরক্ষিত এবং নমনীয় MCP সার্ভার ইমপ্লিমেন্টেশন তৈরির জন্য ব্যবহারিক, হাতে-কলমে গাইড প্রদান করে, মৌলিক ধারণাগুলিকে উন্নত প্রোডাকশন প্যাটার্নের সাথে সংযুক্ত করে।

## সেপ্টেম্বর ২৯, ২০২৫

### MCP সার্ভার ডাটাবেস ইন্টিগ্রেশন ল্যাবস - ব্যাপক হাতে-কলমে শেখার পথ

#### 11-MCPServerHandsOnLabs - নতুন সম্পূর্ণ ডাটাবেস ইন্টিগ্রেশন কারিকুলাম
- **সম্পূর্ণ ১৩-ল্যাব শেখার পথ**: প্রোডাকশন-রেডি MCP সার্ভার তৈরি করার জন্য PostgreSQL ডাটাবেস ইন্টিগ্রেশন সহ ব্যাপক হাতে-কলমে কারিকুলাম যুক্ত
  - **বাস্তব জীবনের ইমপ্লিমেন্টেশন**: এন্টারপ্রাইজ-গ্রেড প্যাটার্ন প্রদর্শনকারী Zava রিটেইল অ্যানালিটিক্স ইউজ কেস
  - **গঠনমূলক শেখার অগ্রগতি**:
    - **ল্যাব ০০-০৩: ভিত্তি** - পরিচিতি, কোর আর্কিটেকচার, সিকিউরিটি ও মাল্টি-টেন্যান্সি, পরিবেশ সেটআপ
    - **ল্যাব ০৪-০৬: MCP সার্ভার নির্মাণ** - ডাটাবেস ডিজাইন ও স্কিমা, MCP সার্ভার ইমপ্লিমেন্টেশন, টুল ডেভেলপমেন্ট  
    - **ল্যাব ০৭-০৯: উন্নত ফিচার** - সেমান্টিক সার্চ ইন্টিগ্রেশন, টেস্টিং ও ডিবাগিং, VS কোড ইন্টিগ্রেশন
    - **ল্যাব ১০-১২: প্রোডাকশন ও বেস্ট প্র্যাকটিস** - ডিপ্লয়মেন্ট স্ট্র্যাটেজি, মনিটরিং ও অবজারভেবিলিটি, বেস্ট প্র্যাকটিস ও অপটিমাইজেশন
  - **এন্টারপ্রাইজ প্রযুক্তি**: FastMCP ফ্রেমওয়ার্ক, PostgreSQL সহ pgvector, আজুর OpenAI এম্বেডিংস, আজুর কন্টেইনার অ্যাপস, অ্যাপ্লিকেশন ইনসাইটস
  - **উন্নত ফিচার**: রো লেভেল সিকিউরিটি (RLS), সেমান্টিক সার্চ, মাল্টি-টেন্যান্ট ডেটা অ্যাক্সেস, ভেক্টর এম্বেডিংস, রিয়েল-টাইম মনিটরিং

#### টার্মিনোলজি স্ট্যান্ডার্ডাইজেশন - মডিউল থেকে ল্যাবে রূপান্তর

#### কেস স্টাডিজ (09-CaseStudy/) - ইকোসিস্টেম ডেভেলপমেন্ট ফোকাস
- **README.md**: ব্যাপক GitHub MCP রেজিস্ট্রি কেস স্টাডি সহ বড় সম্প্রসারণ
  - **GitHub MCP রেজিস্ট্রি কেস স্টাডি**: সেপ্টেম্বর ২০২৫ এ GitHub এর MCP রেজিস্ট্রি লঞ্চ বিশ্লেষণ করে নতুন ব্যাপক কেস স্টাডি
    - **সমস্যা বিশ্লেষণ**: বিচ্ছিন্ন MCP সার্ভার ডিসকভারি ও ডিপ্লয়মেন্ট চ্যালেঞ্জের বিস্তারিত পরীক্ষা
    - **সমাধান আর্কিটেকচার**: GitHub এর কেন্দ্রীভূত রেজিস্ট্রি পদ্ধতি ও এক-ক্লিক VS কোড ইনস্টলেশন
    - **ব্যবসায়িক প্রভাব**: ডেভেলপার অনবোর্ডিং ও উৎপাদনশীলতায় পরিমাপযোগ্য উন্নতি
    - **কৌশলগত মূল্য**: মডুলার এজেন্ট ডিপ্লয়মেন্ট ও ক্রস-টুল ইন্টারঅপারেবিলিটির উপর ফোকাস
    - **ইকোসিস্টেম ডেভেলপমেন্ট**: এজেন্টিক ইন্টিগ্রেশনের জন্য ভিত্তিমূলক প্ল্যাটফর্ম হিসেবে অবস্থান
  - **উন্নত কেস স্টাডি স্ট্রাকচার**: সমস্ত সাতটি কেস স্টাডি ধারাবাহিক ফরম্যাটিং ও ব্যাপক বর্ণনা সহ আপডেট
    - আজুর AI ট্রাভেল এজেন্টস: মাল্টি-এজেন্ট অর্কেস্ট্রেশন জোর
    - আজুর DevOps ইন্টিগ্রেশন: ওয়ার্কফ্লো অটোমেশন ফোকাস
    - রিয়েল-টাইম ডকুমেন্টেশন রিট্রিভাল: পাইথন কনসোল ক্লায়েন্ট ইমপ্লিমেন্টেশন
    - ইন্টারঅ্যাকটিভ স্টাডি প্ল্যান জেনারেটর: Chainlit কথোপকথন ওয়েব অ্যাপ
    - ইন-এডিটর ডকুমেন্টেশন: VS কোড ও GitHub Copilot ইন্টিগ্রেশন
    - আজুর API ম্যানেজমেন্ট: এন্টারপ্রাইজ API ইন্টিগ্রেশন প্যাটার্নস
    - GitHub MCP রেজিস্ট্রি: ইকোসিস্টেম ডেভেলপমেন্ট ও কমিউনিটি প্ল্যাটফর্ম
  - **ব্যাপক উপসংহার**: সাতটি কেস স্টাডি জুড়ে MCP ইমপ্লিমেন্টেশনের বিভিন্ন মাত্রা হাইলাইট করে পুনর্লিখিত উপসংহার
    - এন্টারপ্রাইজ ইন্টিগ্রেশন, মাল্টি-এজেন্ট অর্কেস্ট্রেশন, ডেভেলপার উৎপাদনশীলতা
    - ইকোসিস্টেম ডেভেলপমেন্ট, শিক্ষামূলক অ্যাপ্লিকেশন শ্রেণীবিভাগ
    - আর্কিটেকচারাল প্যাটার্নস, ইমপ্লিমেন্টেশন স্ট্র্যাটেজি, এবং বেস্ট প্র্যাকটিসের উন্নত অন্তর্দৃষ্টি
    - MCP কে পরিপক্ক, প্রোডাকশন-রেডি প্রোটোকল হিসেবে জোর দেওয়া হয়েছে

#### MCP কাস্টম ট্রান্সপোর্টস (05-AdvancedTopics/mcp-transport/) - নতুন উন্নত ইমপ্লিমেন্টেশন গাইড
- **README.md**: কাস্টম MCP ট্রান্সপোর্ট মেকানিজমের সম্পূর্ণ ইমপ্লিমেন্টেশন গাইড
  - **আজুর ইভেন্ট গ্রিড ট্রান্সপোর্ট**: ব্যাপক সার্ভারলেস ইভেন্ট-চালিত ট্রান্সপোর্ট ইমপ্লিমেন্টেশন
    - C#, টাইপস্ক্রিপ্ট, এবং পাইথন উদাহরণ সহ আজুর ফাংশনস ইন্টিগ্রেশন
    - স্কেলেবল MCP সলিউশনের জন্য ইভেন্ট-চালিত আর্কিটেকচার প্যাটার্নস
    - ওয়েবহুক রিসিভার এবং পুশ-ভিত্তিক মেসেজ হ্যান্ডলিং
  - **আজুর ইভেন্ট হাবস ট্রান্সপোর্ট**: উচ্চ-থ্রুপুট স্ট্রিমিং ট্রান্সপোর্ট ইমপ্লিমেন্টেশন
    - কম বিলম্বের পরিস্থিতির জন্য রিয়েল-টাইম স্ট্রিমিং সক্ষমতা
    - পার্টিশনিং স্ট্র্যাটেজি এবং চেকপয়েন্ট ম্যানেজমেন্ট
    - মেসেজ ব্যাচিং এবং পারফরম্যান্স অপটিমাইজেশন
  - **এন্টারপ্রাইজ ইন্টিগ্রেশন প্যাটার্নস**: প্রোডাকশন-রেডি আর্কিটেকচারাল উদাহরণ
    - একাধিক আজুর ফাংশনস জুড়ে বিতরণকৃত MCP প্রসেসিং
    - একাধিক ট্রান্সপোর্ট টাইপ সংযুক্ত করে হাইব্রিড ট্রান্সপোর্ট আর্কিটেকচার
    - মেসেজ টেকসইতা, নির্ভরযোগ্যতা, এবং ত্রুটি হ্যান্ডলিং স্ট্র্যাটেজি
  - **সিকিউরিটি ও মনিটরিং**: আজুর কী ভল্ট ইন্টিগ্রেশন এবং অবজারভেবিলিটি প্যাটার্নস
    - ম্যানেজড আইডেন্টিটি অথেন্টিকেশন এবং সর্বনিম্ন প্রিভিলেজ অ্যাক্সেস
    - অ্যাপ্লিকেশন ইনসাইটস টেলিমেট্রি এবং পারফরম্যান্স মনিটরিং
    - সার্কিট ব্রেকারস এবং ফল্ট টলারেন্স প্যাটার্নস
  - **টেস্টিং ফ্রেমওয়ার্কস**: কাস্টম ট্রান্সপোর্টের জন্য ব্যাপক টেস্টিং স্ট্র্যাটেজি
    - টেস্ট ডাবলস এবং মকিং ফ্রেমওয়ার্ক সহ ইউনিট টেস্টিং
    - আজুর টেস্ট কন্টেইনারস সহ ইন্টিগ্রেশন টেস্টিং
    - পারফরম্যান্স এবং লোড টেস্টিং বিবেচনা

#### কনটেক্সট ইঞ্জিনিয়ারিং (05-AdvancedTopics/mcp-contextengineering/) - উদীয়মান AI শাখা
- **README.md**: উদীয়মান ক্ষেত্র হিসেবে কনটেক্সট ইঞ্জিনিয়ারিং এর ব্যাপক অনুসন্ধান
  - **কোর নীতিমালা**: সম্পূর্ণ কনটেক্সট শেয়ারিং, অ্যাকশন ডিসিশন সচেতনতা, এবং কনটেক্সট উইন্ডো ম্যানেজমেন্ট
  - **MCP প্রোটোকল সামঞ্জস্য**: কিভাবে MCP ডিজাইন কনটেক্সট ইঞ্জিনিয়ারিং চ্যালেঞ্জ মোকাবেলা করে
    - কনটেক্সট উইন্ডো সীমাবদ্ধতা এবং প্রগ্রেসিভ লোডিং স্ট্র্যাটেজি
    - প্রাসঙ্গিকতা নির্ধারণ এবং গতিশীল কনটেক্সট রিট্রিভাল
    - মাল্টি-মোডাল কনটেক্সট হ্যান্ডলিং এবং সিকিউরিটি বিবেচনা
  - **ইমপ্লিমেন্টেশন পদ্ধতি**: সিঙ্গল-থ্রেডেড বনাম মাল্টি-এজেন্ট আর্কিটেকচার
    - কনটেক্সট চাংকিং এবং অগ্রাধিকার নির্ধারণ কৌশল
    - প্রগ্রেসিভ কনটেক্সট লোডিং এবং কম্প্রেশন স্ট্র্যাটেজি
    - স্তরভিত্তিক কনটেক্সট পদ্ধতি এবং রিট্রিভাল অপটিমাইজেশন
  - **মাপজোক ফ্রেমওয়ার্ক**: কনটেক্সট কার্যকারিতা মূল্যায়নের উদীয়মান মেট্রিক্স

  - হোস্ট এখন স্পষ্টভাবে AI অ্যাপ্লিকেশন হিসাবে সংজ্ঞায়িত যা একাধিক MCP ক্লায়েন্ট সংযোগ সমন্বয় করে
  - ক্লায়েন্টকে প্রোটোকল সংযোগকারী হিসাবে বর্ণনা করা হয়েছে যা এক-এক সার্ভার সম্পর্ক বজায় রাখে
  - সার্ভার স্থানীয় বনাম দূরবর্তী স্থাপনার দৃশ্যপট সহ উন্নত করা হয়েছে
- **প্রিমিটিভ পুনর্গঠন**: সার্ভার এবং ক্লায়েন্ট প্রিমিটিভের সম্পূর্ণ পুনর্গঠন
  - সার্ভার প্রিমিটিভ: রিসোর্স (ডেটা সোর্স), প্রম্পট (টেমপ্লেট), টুল (এক্সিকিউটেবল ফাংশন) বিস্তারিত ব্যাখ্যা ও উদাহরণ সহ
  - ক্লায়েন্ট প্রিমিটিভ: স্যাম্পলিং (LLM সম্পূর্ণতা), এলিসিটেশন (ব্যবহারকারী ইনপুট), লগিং (ডিবাগিং/পর্যবেক্ষণ)
  - বর্তমান ডিসকভারি (`*/list`), রিট্রিভাল (`*/get`), এবং এক্সিকিউশন (`*/call`) পদ্ধতি প্যাটার্ন সহ আপডেট করা হয়েছে
- **প্রোটোকল স্থাপত্য**: দুই-স্তর স্থাপত্য মডেল পরিচিতি
  - ডেটা স্তর: JSON-RPC 2.0 ভিত্তি জীবনচক্র ব্যবস্থাপনা এবং প্রিমিটিভ সহ
  - পরিবহন স্তর: STDIO (স্থানীয়) এবং স্ট্রিমেবল HTTP SSE (দূরবর্তী) পরিবহন প্রক্রিয়া
- **সুরক্ষা ফ্রেমওয়ার্ক**: স্পষ্ট ব্যবহারকারী সম্মতি, ডেটা গোপনীয়তা সুরক্ষা, টুল এক্সিকিউশন নিরাপত্তা, এবং পরিবহন স্তর সুরক্ষা সহ ব্যাপক সুরক্ষা নীতিমালা
- **যোগাযোগ প্যাটার্ন**: প্রোটোকল বার্তা আপডেট করা হয়েছে আরম্ভ, আবিষ্কার, এক্সিকিউশন, এবং বিজ্ঞপ্তি প্রবাহ প্রদর্শনের জন্য
- **কোড উদাহরণ**: বর্তমান MCP SDK প্যাটার্ন প্রতিফলিত করতে বহু-ভাষার উদাহরণ (.NET, Java, Python, JavaScript) রিফ্রেশ করা হয়েছে

#### সুরক্ষা (02-Security/) - ব্যাপক সুরক্ষা পুনর্গঠন  
- **মানদণ্ড সামঞ্জস্য**: MCP স্পেসিফিকেশন 2025-06-18 সুরক্ষা প্রয়োজনীয়তার সাথে সম্পূর্ণ সামঞ্জস্য
- **প্রমাণীকরণ বিবর্তন**: কাস্টম OAuth সার্ভার থেকে বহিরাগত পরিচয় প্রদানকারী প্রতিনিধিত্ব (Microsoft Entra ID) বিবর্তন নথিভুক্ত
- **AI-নির্দিষ্ট হুমকি বিশ্লেষণ**: আধুনিক AI আক্রমণ ভেক্টরগুলোর উন্নত কাভারেজ
  - বাস্তব উদাহরণ সহ বিস্তারিত প্রম্পট ইনজেকশন আক্রমণ দৃশ্যপট
  - টুল পয়জনিং প্রক্রিয়া এবং "রাগ পুল" আক্রমণ প্যাটার্ন
  - প্রসঙ্গ উইন্ডো পয়জনিং এবং মডেল বিভ্রান্তি আক্রমণ
- **Microsoft AI সুরক্ষা সমাধান**: Microsoft সুরক্ষা ইকোসিস্টেমের ব্যাপক কাভারেজ
  - AI Prompt Shields উন্নত সনাক্তকরণ, স্পটলাইটিং, এবং ডেলিমিটার কৌশল সহ
  - Azure Content Safety ইন্টিগ্রেশন প্যাটার্ন
  - সরবরাহ শৃঙ্খল সুরক্ষার জন্য GitHub Advanced Security
- **উন্নত হুমকি প্রশমন**: বিস্তারিত সুরক্ষা নিয়ন্ত্রণ
  - MCP-নির্দিষ্ট আক্রমণ দৃশ্যপট এবং ক্রিপ্টোগ্রাফিক সেশন আইডি প্রয়োজনীয়তা সহ সেশন অপহরণ
  - MCP প্রক্সি দৃশ্যপটে বিভ্রান্ত ডেপুটি সমস্যা স্পষ্ট সম্মতি প্রয়োজনীয়তা সহ
  - বাধ্যতামূলক যাচাই নিয়ন্ত্রণ সহ টোকেন পাসথ্রু দুর্বলতা
- **সরবরাহ শৃঙ্খল সুরক্ষা**: ফাউন্ডেশন মডেল, এমবেডিং পরিষেবা, প্রসঙ্গ প্রদানকারী, এবং তৃতীয় পক্ষের API সহ AI সরবরাহ শৃঙ্খল সম্প্রসারিত কাভারেজ
- **ফাউন্ডেশন সুরক্ষা**: শূন্য বিশ্বাস স্থাপত্য এবং Microsoft সুরক্ষা ইকোসিস্টেম সহ এন্টারপ্রাইজ সুরক্ষা প্যাটার্নের উন্নত ইন্টিগ্রেশন
- **সম্পদ সংগঠন**: ধরণের (অফিসিয়াল ডকস, মানদণ্ড, গবেষণা, Microsoft সমাধান, বাস্তবায়ন গাইড) দ্বারা শ্রেণীবদ্ধ ব্যাপক সম্পদ লিঙ্ক
