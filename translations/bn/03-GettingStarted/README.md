## Getting Started  

[![Build Your First MCP Server](../../../translated_images/bn/04.0ea920069efd979a.webp)](https://youtu.be/sNDZO9N4m9Y)

_(এই পাঠের ভিডিও দেখতে উপরের ছবিতে ক্লিক করুন)_

এই অংশে বেশ কয়েকটি পাঠ রয়েছে:

- **1 আপনার প্রথম সার্ভার**, এই প্রথম পাঠে, আপনি শিখবেন কীভাবে আপনার প্রথম সার্ভার তৈরি করবেন এবং ইন্সপেক্টর টুল দিয়ে এটি পরীক্ষা করবেন, যা আপনার সার্ভার টেস্ট এবং ডিবাগ করার গুরুত্বপূর্ণ পদ্ধতি, [পাঠে যান](01-first-server/README.md)

- **2 ক্লায়েন্ট**, এই পাঠে আপনি শিখবেন কীভাবে একটি ক্লায়েন্ট লিখবেন যা আপনার সার্ভারের সাথে সংযোগ করতে সক্ষম হবে, [পাঠে যান](02-client/README.md)

- **3 এলএলএম সহ ক্লায়েন্ট**, ক্লায়েন্ট লেখার একটি আরও ভালো উপায় হলো এতে একটি এলএলএম যোগ করা যাতে এটি আপনার সার্ভারের সাথে "আলোচনা" করতে পারে কী করা উচিত, [পাঠে যান](03-llm-client/README.md)

- **4 ভিজ্যুয়াল স্টুডিও কোডে সার্ভার GitHub Copilot এজেন্ট মোড ব্যবহার**। এখানে আমরা দেখতে যাচ্ছি কীভাবে MCP সার্ভার ভিজ্যুয়াল স্টুডিও কোডের মধ্যে থেকে চালানো যায়, [পাঠে যান](04-vscode/README.md)

- **5 stdio ট্রান্সপোর্ট সার্ভার** stdio ট্রান্সপোর্ট স্থানীয় MCP সার্ভার থেকে ক্লায়েন্টের কমিউনিকেশনের জন্য সুপারিশকৃত স্ট্যান্ডার্ড, যা নিরাপদ সাবপ্রসেস-ভিত্তিক যোগাযোগ এবং প্রসেস আইসোলেশন প্রদান করে [পাঠে যান](05-stdio-server/README.md)

- **6 MCP এর সাথে HTTP স্ট্রিমিং (স্ট্রিমেবল HTTP)**। আধুনিক HTTP স্ট্রিমিং ট্রান্সপোর্ট সম্পর্কে জানুন (রিমোট MCP সার্ভারের জন্য প্রস্তাবিত পদ্ধতি [MCP স্পেসিফিকেশন 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/basic/transports/#streamable-http) অনুযায়ী), অগ্রগতির বিজ্ঞপ্তি, এবং কীভাবে স্কেলেবল, বাস্তব-সময়ের MCP সার্ভার এবং ক্লায়েন্ট স্ট্রিমেবল HTTP ব্যবহার করে নির্মাণ করবেন। [পাঠে যান](06-http-streaming/README.md)

- **7 VSCode এর জন্য AI টুলকিট ব্যবহার** MCP ক্লায়েন্ট এবং সার্ভার পরীক্ষা ও ব্যবহার করার জন্য [পাঠে যান](07-aitk/README.md)

- **8 পরীক্ষা**। এখানে আমরা বিশেষ করে শিখব কিভাবে বিভিন্ন উপায়ে আমাদের সার্ভার এবং ক্লায়েন্ট পরীক্ষা করা যায়, [পাঠে যান](08-testing/README.md)

- **9 ডিপ্লয়মেন্ট**। এই অধ্যায়ে আমরা MCP সমাধানগুলি ডিপ্লয় করার বিভিন্ন উপায় দেখব, [পাঠে যান](09-deployment/README.md)

- **10 উন্নত সার্ভার ব্যবহার**। এই অধ্যায় উন্নত সার্ভার ব্যবহারের বিষয়গুলি কভার করে, [পাঠে যান](./10-advanced/README.md)

- **11 অথেন্টিকেশন**। এই অধ্যায়ে আপনি শিখবেন কীভাবে সহজ অথ যোগ করবেন, বেসিক অথ থেকে JWT ও RBAC ব্যবহার করা পর্যন্ত। এখানে শুরু করার পর অধ্যায় ৫ এ উন্নত বিষয় এবং অধ্যায় ২ এর সুপারিশকৃত নিরাপত্তা ব্যবস্থা সম্পর্কে দেখার জন্য উৎসাহিত হোন, [পাঠে যান](./11-simple-auth/README.md)

- **12 MCP হোস্টস**। জনপ্রিয় MCP হোস্ট ক্লায়েন্ট যেমন Claude Desktop, Cursor, Cline, এবং Windsurf কনফিগার ও ব্যবহার করুন। ট্রান্সপোর্ট প্রকার এবং সমস্যা নির্ণয় শিখুন, [পাঠে যান](./12-mcp-hosts/README.md)

- **13 MCP ইন্সপেক্টর**। MCP ইন্সপেক্টর টুল ব্যবহার করে আপনার MCP সার্ভারগুলি ইন্টারঅ্যাকটিভভাবে ডিবাগ এবং পরীক্ষা করুন। টুল, রিসোর্স এবং প্রোটোকল মেসেজ সমস্যা নির্ণয় শিখুন, [পাঠে যান](./13-mcp-inspector/README.md)

- **14 স্যাম্পলিং**। MCP সার্ভার তৈরি করুন যা MCP ক্লায়েন্টদের সাথে এলএলএম সংক্রান্ত কাজগুলোতে সহযোগিতা করবে। [পাঠে যান](./14-sampling/README.md)

- **15 MCP অ্যাপস**। MCP সার্ভার তৈরি করুন যা UI নির্দেশাবলীর সাথেও প্রতিক্রিয়া দেয়, [পাঠে যান](./15-mcp-apps/README.md)

Model Context Protocol (MCP) একটি মুক্ত প্রোটোকল যা LLMs এর জন্য প্রাসঙ্গিক তথ্য সরবরাহের জন্য অ্যাপ্লিকেশনগুলোকে স্ট্যান্ডার্ডাইজ করে। MCP কে ভাবুন যেমন একটি USB-C পোর্ট AI অ্যাপ্লিকেশনের জন্য – এটি AI মডেলকে বিভিন্ন ডেটা সোর্স এবং টুলের সাথে সংযোগ করার জন্য একটি মানসম্মত উপায় প্রদান করে।

## শেখার লক্ষ্যসমূহ

এই পাঠের শেষে আপনি সক্ষম হবেন:

- C#, Java, Python, TypeScript এবং JavaScript এর জন্য MCP ডেভেলপমেন্ট পরিবেশ স্থাপন করতে
- কাস্টম বৈশিষ্ট্য (রিসোর্স, প্রম্পট, এবং টুল) সহ মৌলিক MCP সার্ভার তৈরি ও ডিপ্লয় করতে
- MCP সার্ভারের সাথে সংযোগ স্থাপনের জন্য হোস্ট অ্যাপ্লিকেশন তৈরি করতে
- MCP বাস্তবায়ন পরীক্ষা এবং ডিবাগ করতে
- সাধারণ সেটআপ চ্যালেঞ্জ এবং তাদের সমাধান বুঝতে
- আপনার MCP বাস্তবায়নগুলোকে জনপ্রিয় LLM সার্ভিসগুলোর সাথে সংযুক্ত করতে

## আপনার MCP পরিবেশ সেটআপ করা

MCP নিয়ে কাজ শুরু করার আগে, আপনার ডেভেলপমেন্ট পরিবেশ প্রস্তুত করা এবং মৌলিক ওয়ার্কফ্লো বোঝা গুরুত্বপূর্ণ। এই অংশ আপনাকে MCP দিয়ে সহজে কাজ শুরু করার জন্য প্রাথমিক সেটআপ ধাপগুলো নির্দেশ করবে।

### পূর্ব শর্ত

MCP ডেভেলপমেন্টে হাত দেওয়ার আগে নিম্নলিখিতগুলি নিশ্চিত করুন:

- **ডেভেলপমেন্ট পরিবেশ**: আপনার পছন্দের ভাষার জন্য (C#, Java, Python, TypeScript, অথবা JavaScript)
- **আইডিই/এডিটর**: Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm, অথবা যেকোন আধুনিক কোড এডিটর
- **প্যাকেজ ম্যানেজারস**: NuGet, Maven/Gradle, pip, অথবা npm/yarn
- **এপিআই কী**: আপনি যদি আপনার হোস্ট অ্যাপ্লিকেশনে কোনো AI সার্ভিস ব্যবহার করতে চান তাহলে

### অফিসিয়াল SDK গুলো

আগামী অধ্যায়গুলোতে আপনি Python, TypeScript, Java এবং .NET ব্যবহার করে সমাধানগুলো দেখবেন। নিচে সব অফিসিয়াল সমর্থিত SDK রয়েছে।

MCP বিভিন্ন ভাষার জন্য অফিসিয়াল SDK সরবরাহ করে (যা [MCP স্পেসিফিকেশন 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/) এর সাথে সামঞ্জস্যপূর্ণ):

- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk) - Microsoft এর সহযোগিতায় রক্ষণাবেক্ষণ
- [Java SDK](https://github.com/modelcontextprotocol/java-sdk) - Spring AI এর সহযোগিতায় রক্ষণাবেক্ষণ
- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) - অফিসিয়াল TypeScript বাস্তবায়ন
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk) - অফিসিয়াল Python বাস্তবায়ন (FastMCP)
- [Kotlin SDK](https://github.com/modelcontextprotocol/kotlin-sdk) - অফিসিয়াল Kotlin বাস্তবায়ন
- [Swift SDK](https://github.com/modelcontextprotocol/swift-sdk) - Loopwork AI এর সহযোগিতায় রক্ষণাবেক্ষণ
- [Rust SDK](https://github.com/modelcontextprotocol/rust-sdk) - অফিসিয়াল Rust বাস্তবায়ন
- [Go SDK](https://github.com/modelcontextprotocol/go-sdk) - অফিসিয়াল Go বাস্তবায়ন

## মূল বিষয়বস্তু

- MCP ডেভেলপমেন্ট পরিবেশ স্থাপন ভাষা-নির্দিষ্ট SDK দিয়ে সহজ
- MCP সার্ভার তৈরি করতে স্পষ্ট স্কিমা সহ টুল তৈরি এবং নিবন্ধন করা দরকার
- MCP ক্লায়েন্ট সার্ভার এবং মডেলের সাথে সংযোগ করে উন্নত কার্যক্ষমতা অর্জন করে
- পরীক্ষণ এবং ডিবাগিং MCP বাস্তবায়নে বিশ্বাসযোগ্যতার জন্য অপরিহার্য
- ডিপ্লয়মেন্ট বিকল্প স্থানীয় ডেভেলপমেন্ট থেকে ক্লাউড-ভিত্তিক সমাধান পর্যন্ত বিস্তৃত
