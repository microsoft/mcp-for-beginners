# MCP سیکیورٹی: AI سسٹمز کے لیے جامع تحفظ

[![MCP Security Best Practices](../../../translated_images/ur/03.175aed6dedae133f9d41e49cefd0f0a9a39c3317e1eaa7ef7182696af7534308.png)](https://youtu.be/88No8pw706o)

_(اس سبق کی ویڈیو دیکھنے کے لیے اوپر تصویر پر کلک کریں)_

سیکیورٹی AI سسٹم ڈیزائن کی بنیاد ہے، اسی لیے ہم اسے اپنے دوسرے سیکشن کے طور پر ترجیح دیتے ہیں۔ یہ مائیکروسافٹ کے [Secure Future Initiative](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) کے **Secure by Design** اصول کے مطابق ہے۔

ماڈل کانٹیکسٹ پروٹوکول (MCP) AI سے چلنے والی ایپلیکیشنز میں طاقتور نئی صلاحیتیں لاتا ہے جبکہ روایتی سافٹ ویئر خطرات سے آگے بڑھ کر منفرد سیکیورٹی چیلنجز بھی پیش کرتا ہے۔ MCP سسٹمز کو قائم شدہ سیکیورٹی خدشات (محفوظ کوڈنگ، کم از کم مراعات، سپلائی چین سیکیورٹی) اور AI مخصوص نئے خطرات جیسے پرامپٹ انجیکشن، ٹول پوائزننگ، سیشن ہائی جیکنگ، کنفیوزڈ ڈیپٹی حملے، ٹوکن پاس تھرو کمزوریاں، اور متحرک صلاحیت کی ترمیم کا سامنا ہوتا ہے۔

یہ سبق MCP نفاذ میں سب سے اہم سیکیورٹی خطرات کا جائزہ لیتا ہے—جس میں توثیق، اجازت، ضرورت سے زیادہ اجازتیں، بالواسطہ پرامپٹ انجیکشن، سیشن سیکیورٹی، کنفیوزڈ ڈیپٹی مسائل، ٹوکن مینجمنٹ، اور سپلائی چین کمزوریاں شامل ہیں۔ آپ ان خطرات کو کم کرنے کے لیے قابل عمل کنٹرولز اور بہترین طریقے سیکھیں گے، اور مائیکروسافٹ کے حل جیسے Prompt Shields، Azure Content Safety، اور GitHub Advanced Security کا استعمال کرتے ہوئے اپنے MCP تعیناتی کو مضبوط کریں گے۔

## سیکھنے کے مقاصد

اس سبق کے اختتام تک، آپ قابل ہوں گے:

- **MCP مخصوص خطرات کی شناخت**: MCP سسٹمز میں منفرد سیکیورٹی خطرات کو پہچاننا، جن میں پرامپٹ انجیکشن، ٹول پوائزننگ، ضرورت سے زیادہ اجازتیں، سیشن ہائی جیکنگ، کنفیوزڈ ڈیپٹی مسائل، ٹوکن پاس تھرو کمزوریاں، اور سپلائی چین خطرات شامل ہیں
- **سیکیورٹی کنٹرولز کا اطلاق**: مؤثر تخفیف کے اقدامات نافذ کرنا، جن میں مضبوط توثیق، کم از کم مراعات تک رسائی، محفوظ ٹوکن مینجمنٹ، سیشن سیکیورٹی کنٹرولز، اور سپلائی چین کی تصدیق شامل ہیں
- **مائیکروسافٹ سیکیورٹی حل کا فائدہ اٹھانا**: MCP ورک لوڈ کی حفاظت کے لیے Microsoft Prompt Shields، Azure Content Safety، اور GitHub Advanced Security کو سمجھنا اور تعینات کرنا
- **ٹول سیکیورٹی کی توثیق**: ٹول میٹا ڈیٹا کی توثیق کی اہمیت کو پہچاننا، متحرک تبدیلیوں کی نگرانی کرنا، اور بالواسطہ پرامپٹ انجیکشن حملوں کے خلاف دفاع کرنا
- **بہترین طریقوں کا انضمام**: قائم شدہ سیکیورٹی بنیادی اصولوں (محفوظ کوڈنگ، سرور ہارڈننگ، زیرو ٹرسٹ) کو MCP مخصوص کنٹرولز کے ساتھ ملا کر جامع تحفظ فراہم کرنا

# MCP سیکیورٹی آرکیٹیکچر اور کنٹرولز

جدید MCP نفاذ میں تہہ دار سیکیورٹی طریقے درکار ہیں جو روایتی سافٹ ویئر سیکیورٹی اور AI مخصوص خطرات دونوں کو حل کریں۔ تیزی سے ترقی پذیر MCP وضاحت اپنے سیکیورٹی کنٹرولز کو بہتر بناتی رہتی ہے، جس سے انٹرپرائز سیکیورٹی آرکیٹیکچرز اور قائم شدہ بہترین طریقوں کے ساتھ بہتر انضمام ممکن ہوتا ہے۔

[Microsoft Digital Defense Report](https://aka.ms/mddr) کی تحقیق سے پتہ چلتا ہے کہ **رپورٹ شدہ خلاف ورزیوں کا 98% مضبوط سیکیورٹی حفظان صحت سے روکا جا سکتا ہے**۔ سب سے مؤثر تحفظ کی حکمت عملی بنیادی سیکیورٹی طریقوں کو MCP مخصوص کنٹرولز کے ساتھ جوڑتی ہے—ثابت شدہ بنیادی سیکیورٹی اقدامات مجموعی سیکیورٹی خطرے کو کم کرنے میں سب سے زیادہ مؤثر ہیں۔

## موجودہ سیکیورٹی منظرنامہ

> **نوٹ:** یہ معلومات MCP سیکیورٹی معیارات کی عکاسی کرتی ہے جو **18 دسمبر 2025** تک ہیں۔ MCP پروٹوکول تیزی سے ترقی کر رہا ہے، اور مستقبل کے نفاذ میں نئی توثیقی پیٹرنز اور بہتر کنٹرولز شامل ہو سکتے ہیں۔ تازہ ترین رہنمائی کے لیے ہمیشہ موجودہ [MCP Specification](https://spec.modelcontextprotocol.io/)، [MCP GitHub repository](https://github.com/modelcontextprotocol)، اور [security best practices documentation](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) کا حوالہ دیں۔

### MCP توثیق کی ترقی

MCP وضاحت نے توثیق اور اجازت کے طریقہ کار میں نمایاں ترقی کی ہے:

- **اصل طریقہ کار**: ابتدائی وضاحتوں میں ڈویلپرز کو اپنی مرضی کے مطابق توثیقی سرورز بنانے کی ضرورت تھی، جبکہ MCP سرورز OAuth 2.0 اجازت سرورز کے طور پر کام کرتے تھے جو صارف کی توثیق براہ راست سنبھالتے تھے
- **موجودہ معیار (2025-11-25)**: اپ ڈیٹ شدہ وضاحت MCP سرورز کو بیرونی شناخت فراہم کنندگان (جیسے Microsoft Entra ID) کو توثیق تفویض کرنے کی اجازت دیتی ہے، جس سے سیکیورٹی کی حالت بہتر ہوتی ہے اور نفاذ کی پیچیدگی کم ہوتی ہے
- **ٹرانسپورٹ لیئر سیکیورٹی**: مقامی (STDIO) اور ریموٹ (Streamable HTTP) کنکشنز کے لیے مناسب توثیقی پیٹرنز کے ساتھ محفوظ ٹرانسپورٹ میکانزم کی بہتر حمایت

## توثیق اور اجازت کی سیکیورٹی

### موجودہ سیکیورٹی چیلنجز

جدید MCP نفاذ کو کئی توثیق اور اجازت کے چیلنجز کا سامنا ہے:

### خطرات اور حملے کے راستے

- **غلط ترتیب دی گئی اجازت منطق**: MCP سرورز میں اجازت کے غلط نفاذ سے حساس ڈیٹا ظاہر ہو سکتا ہے اور رسائی کنٹرولز غلط طریقے سے لاگو ہو سکتے ہیں
- **OAuth ٹوکن کا سمجھوتہ**: مقامی MCP سرور کے ٹوکن کی چوری حملہ آوروں کو سرورز کی نقل کرنے اور نیچے کی خدمات تک رسائی کی اجازت دیتی ہے
- **ٹوکن پاس تھرو کمزوریاں**: ٹوکن کی غلط ہینڈلنگ سیکیورٹی کنٹرولز کو بائی پاس اور جوابدہی کے خلا پیدا کرتی ہے
- **ضرورت سے زیادہ اجازتیں**: زیادہ مراعات یافتہ MCP سرور کم از کم مراعات کے اصول کی خلاف ورزی کرتے ہیں اور حملے کی سطح کو بڑھاتے ہیں

#### ٹوکن پاس تھرو: ایک سنگین اینٹی پیٹرن

موجودہ MCP اجازت کی وضاحت میں **ٹوکن پاس تھرو واضح طور پر ممنوع ہے** کیونکہ اس کے سنگین سیکیورٹی نتائج ہیں:

##### سیکیورٹی کنٹرولز کی بائی پاسنگ
- MCP سرورز اور نیچے کی API اہم سیکیورٹی کنٹرولز (ریٹ لمٹنگ، درخواست کی توثیق، ٹریفک مانیٹرنگ) نافذ کرتے ہیں جو مناسب ٹوکن کی توثیق پر منحصر ہیں
- کلائنٹ سے API تک براہ راست ٹوکن کا استعمال ان ضروری حفاظتی اقدامات کو بائی پاس کرتا ہے، جس سے سیکیورٹی آرکیٹیکچر کمزور ہوتا ہے

##### جوابدہی اور آڈٹ کے مسائل  
- MCP سرورز کلائنٹس کو اپ اسٹریم جاری کردہ ٹوکن استعمال کرنے والوں میں فرق نہیں کر سکتے، جس سے آڈٹ ٹریلز ٹوٹ جاتے ہیں
- نیچے کی ریسورس سرور لاگز درخواستوں کے اصل MCP سرور درمیانی افراد کی بجائے گمراہ کن ماخذ دکھاتے ہیں
- واقعہ کی تحقیقات اور تعمیل کے آڈٹ بہت مشکل ہو جاتے ہیں

##### ڈیٹا چوری کے خطرات
- غیر توثیق شدہ ٹوکن دعوے چوری شدہ ٹوکن کے ساتھ بدنیتی پر مبنی افراد کو MCP سرورز کو ڈیٹا چوری کے لیے پراکسی کے طور پر استعمال کرنے کی اجازت دیتے ہیں
- اعتماد کی حد کی خلاف ورزیاں غیر مجاز رسائی کے نمونے پیدا کرتی ہیں جو مطلوبہ سیکیورٹی کنٹرولز کو بائی پاس کرتی ہیں

##### کثیر سروس حملے کے راستے
- متعدد خدمات کی طرف سے قبول شدہ سمجھوتہ شدہ ٹوکنز مربوط سسٹمز میں افقی حرکت کی اجازت دیتے ہیں
- جب ٹوکن کے ماخذ کی تصدیق نہیں کی جا سکتی تو خدمات کے درمیان اعتماد کے مفروضے ٹوٹ سکتے ہیں

### سیکیورٹی کنٹرولز اور تخفیف

**اہم سیکیورٹی ضروریات:**

> **لازمی:** MCP سرورز **کسی بھی ایسے ٹوکن کو قبول نہیں کریں گے جو واضح طور پر MCP سرور کے لیے جاری نہ کیے گئے ہوں**

#### توثیق اور اجازت کے کنٹرولز

- **سخت اجازت کا جائزہ**: MCP سرور کی اجازت کی منطق کا جامع آڈٹ کریں تاکہ صرف مطلوبہ صارفین اور کلائنٹس حساس وسائل تک رسائی حاصل کر سکیں
  - **نفاذ گائیڈ**: [Azure API Management as Authentication Gateway for MCP Servers](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)
  - **شناخت کا انضمام**: [Using Microsoft Entra ID for MCP Server Authentication](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **محفوظ ٹوکن مینجمنٹ**: [Microsoft کے ٹوکن کی توثیق اور لائف سائیکل بہترین طریقے](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens) نافذ کریں
  - ٹوکن کے آڈینس دعوے MCP سرور کی شناخت سے میل کھاتے ہوں
  - مناسب ٹوکن گردش اور میعاد ختم ہونے کی پالیسیاں نافذ کریں
  - ٹوکن ری پلے حملوں اور غیر مجاز استعمال کو روکیں

- **محفوظ ٹوکن اسٹوریج**: ٹوکن کو آرام اور منتقلی دونوں میں انکرپشن کے ساتھ محفوظ کریں
  - **بہترین طریقے**: [Secure Token Storage and Encryption Guidelines](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### رسائی کنٹرول کا نفاذ

- **کم از کم مراعات کا اصول**: MCP سرورز کو صرف مطلوبہ فعالیت کے لیے کم از کم اجازتیں دیں
  - مراعات کے باقاعدہ جائزے اور اپ ڈیٹس تاکہ مراعات کی زیادتی روکی جا سکے
  - **مائیکروسافٹ دستاویزات**: [Secure Least-Privileged Access](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **رول پر مبنی رسائی کنٹرول (RBAC)**: باریک بینی سے رول تفویض کریں
  - مخصوص وسائل اور کارروائیوں کے لیے رولز کو محدود کریں
  - وسیع یا غیر ضروری اجازتوں سے گریز کریں جو حملے کی سطح کو بڑھائیں

- **مسلسل اجازت مانیٹرنگ**: جاری رسائی آڈٹ اور نگرانی نافذ کریں
  - اجازت کے استعمال کے نمونوں کی نگرانی کریں تاکہ غیر معمولی سرگرمیوں کا پتہ چل سکے
  - ضرورت سے زیادہ یا غیر استعمال شدہ مراعات کو فوری طور پر درست کریں

## AI مخصوص سیکیورٹی خطرات

### پرامپٹ انجیکشن اور ٹول مینیپولیشن حملے

جدید MCP نفاذ پیچیدہ AI مخصوص حملے کے راستوں کا سامنا کرتے ہیں جنہیں روایتی سیکیورٹی اقدامات مکمل طور پر حل نہیں کر سکتے:

#### **بالواسطہ پرامپٹ انجیکشن (کراس ڈومین پرامپٹ انجیکشن)**

**بالواسطہ پرامپٹ انجیکشن** MCP فعال AI سسٹمز میں سب سے سنگین کمزوریوں میں سے ایک ہے۔ حملہ آور بیرونی مواد—دستاویزات، ویب صفحات، ای میلز، یا ڈیٹا ذرائع—میں بدنیتی پر مبنی ہدایات شامل کرتے ہیں جو AI سسٹمز بعد میں جائز کمانڈز کے طور پر عمل کرتے ہیں۔

**حملے کے منظرنامے:**
- **دستاویز پر مبنی انجیکشن**: پروسیس کی گئی دستاویزات میں چھپی ہوئی بدنیتی پر مبنی ہدایات جو غیر ارادی AI کارروائیاں شروع کرتی ہیں
- **ویب مواد کا استحصال**: متاثرہ ویب صفحات جن میں شامل پرامپٹس AI کے رویے کو متاثر کرتے ہیں جب وہ سکریپ کیے جاتے ہیں
- **ای میل پر مبنی حملے**: ای میلز میں بدنیتی پر مبنی پرامپٹس جو AI اسسٹنٹس کو معلومات افشا کرنے یا غیر مجاز کارروائیاں کرنے پر مجبور کرتے ہیں
- **ڈیٹا سورس آلودگی**: متاثرہ ڈیٹا بیس یا APIs جو AI سسٹمز کو آلودہ مواد فراہم کرتے ہیں

**حقیقی دنیا کا اثر**: یہ حملے ڈیٹا چوری، پرائیویسی کی خلاف ورزی، نقصان دہ مواد کی تخلیق، اور صارف کے تعاملات میں مداخلت کا باعث بن سکتے ہیں۔ تفصیلی تجزیہ کے لیے دیکھیں [Prompt Injection in MCP (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/)۔

![Prompt Injection Attack Diagram](../../../translated_images/ur/prompt-injection.ed9fbfde297ca877c15bc6daa808681cd3c3dc7bf27bbbda342ef1ba5fc4f52d.png)

#### **ٹول پوائزننگ حملے**

**ٹول پوائزننگ** MCP ٹولز کی میٹا ڈیٹا کو نشانہ بناتی ہے، جو LLMs کے ٹول کی وضاحتوں اور پیرامیٹرز کی تشریح کے طریقے کا استحصال کرتی ہے تاکہ عمل درآمد کے فیصلے کیے جائیں۔

**حملے کے طریقے:**
- **میٹا ڈیٹا میں مداخلت**: حملہ آور ٹول کی وضاحتوں، پیرامیٹر کی تعریفوں، یا استعمال کی مثالوں میں بدنیتی پر مبنی ہدایات شامل کرتے ہیں
- **غیر مرئی ہدایات**: ٹول میٹا ڈیٹا میں چھپے ہوئے پرامپٹس جو AI ماڈلز کے ذریعے پروسیس ہوتے ہیں لیکن انسانی صارفین کے لیے نظر نہیں آتے
- **متحرک ٹول ترمیم ("رگ پلز")**: صارفین کی منظوری کے بعد ٹولز کو بدنیتی پر مبنی کارروائیاں کرنے کے لیے تبدیل کیا جاتا ہے بغیر صارف کی آگاہی کے
- **پیرامیٹر انجیکشن**: ٹول پیرامیٹر اسکیموں میں بدنیتی پر مبنی مواد شامل کرنا جو ماڈل کے رویے کو متاثر کرتا ہے

**میزبان سرور کے خطرات**: ریموٹ MCP سرورز میں خطرات بڑھ جاتے ہیں کیونکہ ٹول کی تعریفیں صارف کی ابتدائی منظوری کے بعد اپ ڈیٹ کی جا سکتی ہیں، جس سے پہلے محفوظ ٹولز بدنیتی پر مبنی بن سکتے ہیں۔ جامع تجزیہ کے لیے دیکھیں [Tool Poisoning Attacks (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks)۔

![Tool Injection Attack Diagram](../../../translated_images/ur/tool-injection.3b0b4a6b24de6befe7d3afdeae44138ef005881aebcfc84c6f61369ce31e3640.png)

#### **اضافی AI حملے کے راستے**

- **کراس ڈومین پرامپٹ انجیکشن (XPIA)**: پیچیدہ حملے جو متعدد ڈومینز کے مواد کا استعمال کرتے ہوئے سیکیورٹی کنٹرولز کو بائی پاس کرتے ہیں
- **متحرک صلاحیت کی ترمیم**: ٹول کی صلاحیتوں میں حقیقی وقت میں تبدیلیاں جو ابتدائی سیکیورٹی جائزوں سے بچ جاتی ہیں
- **کانٹیکسٹ ونڈو پوائزننگ**: بڑے کانٹیکسٹ ونڈوز کو مانیپولیٹ کرنے والے حملے جو بدنیتی پر مبنی ہدایات چھپاتے ہیں
- **ماڈل کنفیوژن حملے**: ماڈل کی حدود کا استحصال کر کے غیر متوقع یا غیر محفوظ رویے پیدا کرنا

### AI سیکیورٹی خطرے کا اثر

**اعلی اثرات:**
- **ڈیٹا چوری**: حساس انٹرپرائز یا ذاتی ڈیٹا تک غیر مجاز رسائی اور چوری
- **پرائیویسی کی خلاف ورزیاں**: ذاتی شناختی معلومات (PII) اور کاروباری رازوں کا انکشاف  
- **سسٹم کی مداخلت**: اہم سسٹمز اور ورک فلو میں غیر ارادی تبدیلیاں
- **اسناد کی چوری**: توثیقی ٹوکنز اور سروس اسناد کا سمجھوتہ
- **افقی حرکت**: وسیع نیٹ ورک حملوں کے لیے سمجھوتہ شدہ AI سسٹمز کا استعمال

### مائیکروسافٹ AI سیکیورٹی حل

#### **AI Prompt Shields: انجیکشن حملوں کے خلاف جدید تحفظ**

مائیکروسافٹ **AI Prompt Shields** براہ راست اور بالواسطہ پرامپٹ انجیکشن حملوں کے خلاف متعدد سیکیورٹی تہوں کے ذریعے جامع دفاع فراہم کرتے ہیں:

##### **بنیادی حفاظتی میکانزم:**

1. **جدید شناخت اور فلٹرنگ**
   - مشین لرننگ الگورتھمز اور NLP تکنیکیں بیرونی مواد میں بدنیتی پر مبنی ہدایات کا پتہ لگاتی ہیں
   - دستاویزات، ویب صفحات، ای میلز، اور ڈیٹا ذرائع کا حقیقی وقت میں تجزیہ
   - جائز اور بدنیتی پر مبنی پرامپٹ پیٹرنز کی سیاق و سباق کی سمجھ

2. **اسپاٹ لائٹنگ تکنیکیں**  
   - قابل اعتماد سسٹم ہدایات اور ممکنہ طور پر متاثرہ بیرونی ان پٹ کے درمیان فرق کرتی ہیں
   - متن کی تبدیلی کے طریقے جو ماڈل کی مطابقت کو بڑھاتے ہیں اور بدنیتی پر مبنی مواد کو الگ کرتے ہیں
   - AI سسٹمز کو مناسب ہدایت کی درجہ بندی برقرار رکھنے اور انجیکٹڈ کمانڈز کو نظر انداز کرنے میں مدد دیتی ہیں

3. **ڈیلیمیٹر اور ڈیٹا مارکنگ سسٹمز**
   - قابل اعتماد سسٹم پیغامات اور بیرونی ان پٹ متن کے درمیان واضح حد بندی
   - خاص مارکرز جو قابل اعتماد اور غیر قابل اعتماد ڈیٹا ذرائع کے درمیان حد بندی کو نمایاں کرتے ہیں
   - واضح علیحدگی ہدایت کی الجھن اور غیر مجاز کمانڈ کے نفاذ کو روکتی ہے

4. **مسلسل خطرے کی انٹیلی جنس**
   - مائیکروسافٹ ابھرتے ہوئے حملے کے نمونوں کی نگرانی کرتا ہے اور دفاع کو اپ ڈیٹ کرتا ہے
   - نئی انجیکشن تکنیکوں اور حملے کے راستوں کے لیے فعال خطرے کی تلاش
   - بدلتے ہوئے خطرات کے خلاف مؤثر رہنے کے لیے باقاعدہ سیکیورٹی ماڈل اپ ڈیٹس

5. **Azure Content Safety انضمام**
   - Azure AI Content Safety سوٹ کا حصہ
   - جیل بریک کوششوں، نقصان دہ مواد، اور سیکیورٹی پالیسی کی خلاف ورزیوں کے لیے اضافی شناخت
   - AI ایپلیکیشن کے اجزاء میں متحدہ سیکیورٹی کنٹرولز

## جدید MCP سیکیورٹی خطرات

### سیشن ہائی جیکنگ کمزوریاں

**سیشن ہائی جیکنگ** ریاستی MCP نفاذ میں ایک سنگین حملے کا راستہ ہے جہاں غیر مجاز فریق جائز سیشن شناخت کنندگان حاصل کر کے کلائنٹس کی نقل کرتے ہیں اور غیر مجاز کارروائیاں انجام دیتے ہیں۔

#### **حملے کے منظرنامے اور خطرات**

- **سیشن ہائی جیک پرامپٹ انجیکشن**: چوری شدہ سیشن IDs کے ساتھ حملہ آور سرورز میں بدنیتی پر مبنی واقعات انجیکٹ کرتے ہیں جو سیشن اسٹیٹ شیئر کرتے ہیں، ممکنہ طور پر نقصان دہ کارروائیاں شروع کرتے ہیں یا حساس ڈیٹا تک رسائی حاصل کرتے ہیں
- **براہ راست نقل**: چوری شدہ سیشن IDs حملہ آوروں کو توثیق کو بائی پاس کرتے ہوئے براہ راست MCP سرور کالز کرنے کی اجازت دیتے ہیں، انہیں جائز صارفین کے طور پر سمجھا جاتا ہے
- **سمجھوتہ شدہ ریزیوم ایبل اسٹریمز**: حملہ آور درخواستوں کو قبل از وقت ختم کر سکتے ہیں، جس سے جائز کلائنٹس کو ممکنہ طور پر بدنیتی پر مبنی مواد کے ساتھ دوبارہ شروع کرنا پڑتا ہے

#### **سیشن مینجمنٹ کے لیے سیکیورٹی کنٹرولز**

**اہم ضروریات:**
- **تصدیق کی توثیق**: MCP سرورز جو اجازت نامہ نافذ کرتے ہیں **ضروری ہے** کہ وہ تمام آنے والی درخواستوں کی تصدیق کریں اور **کسی بھی صورت میں** سیشنز پر توثیق کے لیے انحصار نہ کریں  
- **محفوظ سیشن جنریشن**: کرپٹوگرافک طور پر محفوظ، غیر متعین سیشن آئی ڈیز استعمال کریں جو محفوظ رینڈم نمبر جنریٹرز کے ذریعے بنائے گئے ہوں  
- **صارف مخصوص بائنڈنگ**: سیشن آئی ڈیز کو صارف مخصوص معلومات کے ساتھ باندھیں جیسے کہ `<user_id>:<session_id>` تاکہ کراس یوزر سیشن کے غلط استعمال کو روکا جا سکے  
- **سیشن لائف سائیکل مینجمنٹ**: مناسب میعاد ختم ہونے، گردش، اور منسوخی کو نافذ کریں تاکہ کمزوری کے مواقع محدود ہوں  
- **ٹرانسپورٹ سیکیورٹی**: تمام مواصلات کے لیے لازمی HTTPS تاکہ سیشن آئی ڈی کی چوری کو روکا جا سکے  

### الجھے ہوئے ڈپٹی کا مسئلہ

**الجھے ہوئے ڈپٹی کا مسئلہ** اس وقت پیش آتا ہے جب MCP سرورز کلائنٹس اور تیسرے فریق کی خدمات کے درمیان توثیق کے پراکسی کے طور پر کام کرتے ہیں، جو جامد کلائنٹ آئی ڈی کے غلط استعمال کے ذریعے اجازت نامہ بائی پاس کے مواقع پیدا کرتا ہے۔

#### **حملے کے طریقہ کار اور خطرات**

- **کوکی پر مبنی رضامندی کی بائی پاس**: پچھلی صارف کی توثیق رضامندی کوکیز بناتی ہے جنہیں حملہ آور جعلی اجازت نامہ درخواستوں کے ذریعے تیار کردہ ری ڈائریکٹ یو آر آئی کے ساتھ استحصال کرتے ہیں  
- **اجازت نامہ کوڈ کی چوری**: موجودہ رضامندی کوکیز اجازت نامہ سرورز کو رضامندی کے اسکرینز چھوڑنے پر مجبور کر سکتی ہیں، کوڈز کو حملہ آور کے کنٹرول والے اینڈ پوائنٹس پر ری ڈائریکٹ کرتی ہیں  
- **غیر مجاز API رسائی**: چوری شدہ اجازت نامہ کوڈز ٹوکن کے تبادلے اور صارف کی نقل کرنے کی اجازت دیتے ہیں بغیر واضح منظوری کے  

#### **تدارکی حکمت عملیاں**

**لازمی کنٹرولز:**  
- **واضح رضامندی کی ضروریات**: MCP پراکسی سرورز جو جامد کلائنٹ آئی ڈیز استعمال کرتے ہیں **ضروری ہے** کہ ہر متحرک طور پر رجسٹرڈ کلائنٹ کے لیے صارف کی رضامندی حاصل کریں  
- **OAuth 2.1 سیکیورٹی نفاذ**: تمام اجازت نامہ درخواستوں کے لیے موجودہ OAuth سیکیورٹی بہترین طریقہ کار بشمول PKCE (پروف کی برائے کوڈ ایکسچینج) پر عمل کریں  
- **سخت کلائنٹ کی توثیق**: ری ڈائریکٹ یو آر آئیز اور کلائنٹ شناخت کنندگان کی سخت توثیق نافذ کریں تاکہ استحصال کو روکا جا سکے  

### ٹوکن پاس تھرو کی کمزوریاں  

**ٹوکن پاس تھرو** ایک واضح اینٹی پیٹرن کی نمائندگی کرتا ہے جہاں MCP سرورز کلائنٹ ٹوکنز کو بغیر مناسب توثیق کے قبول کرتے ہیں اور انہیں نیچے کی طرف APIز کو بھیج دیتے ہیں، جو MCP اجازت نامہ وضاحتوں کی خلاف ورزی ہے۔

#### **سیکیورٹی کے مضمرات**

- **کنٹرول کی بائی پاسنگ**: کلائنٹ سے API تک براہ راست ٹوکن کا استعمال اہم ریٹ لمٹنگ، توثیق، اور نگرانی کنٹرولز کو بائی پاس کرتا ہے  
- **آڈٹ ٹریل کی خرابی**: اوپر سے جاری کردہ ٹوکنز کلائنٹ کی شناخت ناممکن بنا دیتے ہیں، حادثاتی تحقیقات کی صلاحیت کو توڑ دیتے ہیں  
- **پراکسی کی بنیاد پر ڈیٹا چوری**: غیر توثیق شدہ ٹوکنز حملہ آوروں کو سرورز کو غیر مجاز ڈیٹا تک رسائی کے لیے پراکسی کے طور پر استعمال کرنے کی اجازت دیتے ہیں  
- **اعتماد کی حد کی خلاف ورزیاں**: نیچے کی طرف خدمات کے اعتماد کے مفروضے ٹوکن کے ماخذ کی تصدیق نہ ہونے پر متاثر ہو سکتے ہیں  
- **کئی خدمات پر حملے کی توسیع**: متعدد خدمات میں قبول شدہ متاثرہ ٹوکنز افقی حرکت کی اجازت دیتے ہیں  

#### **ضروری سیکیورٹی کنٹرولز**

**ناقابلِ مذاکرہ تقاضے:**  
- **ٹوکن کی توثیق**: MCP سرورز **کبھی بھی** ایسے ٹوکن قبول نہ کریں جو واضح طور پر MCP سرور کے لیے جاری نہ کیے گئے ہوں  
- **ناظرین کی تصدیق**: ہمیشہ ٹوکن کے ناظرین کے دعووں کی تصدیق کریں کہ وہ MCP سرور کی شناخت سے میل کھاتے ہوں  
- **مناسب ٹوکن لائف سائیکل**: مختصر مدت کے ایکسیس ٹوکنز کو محفوظ گردش کے طریقوں کے ساتھ نافذ کریں  


## AI سسٹمز کے لیے سپلائی چین سیکیورٹی

سپلائی چین سیکیورٹی روایتی سافٹ ویئر انحصار سے آگے بڑھ کر پورے AI ماحولیاتی نظام کو شامل کر چکی ہے۔ جدید MCP نفاذ کو تمام AI سے متعلقہ اجزاء کی سختی سے تصدیق اور نگرانی کرنی چاہیے، کیونکہ ہر ایک ممکنہ کمزوری متعارف کراتا ہے جو نظام کی سالمیت کو متاثر کر سکتی ہے۔

### AI سپلائی چین کے توسیع شدہ اجزاء

**روایتی سافٹ ویئر انحصار:**  
- اوپن سورس لائبریریاں اور فریم ورکس  
- کنٹینر امیجز اور بنیادی نظام  
- ترقیاتی آلات اور بلڈ پائپ لائنز  
- انفراسٹرکچر کے اجزاء اور خدمات  

**AI مخصوص سپلائی چین عناصر:**  
- **بنیادی ماڈلز**: مختلف فراہم کنندگان سے پری ٹرینڈ ماڈلز جن کی اصلیت کی تصدیق ضروری ہے  
- **ایمبیڈنگ سروسز**: بیرونی ویکٹرائزیشن اور معنوی تلاش کی خدمات  
- **سیاق و سباق فراہم کرنے والے**: ڈیٹا ذرائع، علم کے ذخائر، اور دستاویزات کے ذخیرے  
- **تیسرے فریق APIز**: بیرونی AI خدمات، ML پائپ لائنز، اور ڈیٹا پروسیسنگ اینڈ پوائنٹس  
- **ماڈل آرٹیفیکٹس**: وزن، کنفیگریشنز، اور فائن ٹیونڈ ماڈل کی اقسام  
- **ٹریننگ ڈیٹا ذرائع**: ماڈل کی تربیت اور فائن ٹیوننگ کے لیے استعمال ہونے والے ڈیٹا سیٹس  

### جامع سپلائی چین سیکیورٹی حکمت عملی

#### **جزو کی تصدیق اور اعتماد**  
- **اصل کی تصدیق**: تمام AI اجزاء کی اصلیت، لائسنسنگ، اور سالمیت کی تصدیق کریں قبل از انضمام  
- **سیکیورٹی جائزہ**: ماڈلز، ڈیٹا ذرائع، اور AI خدمات کے لیے کمزوری اسکین اور سیکیورٹی جائزے کریں  
- **ساکھ کا تجزیہ**: AI سروس فراہم کنندگان کے سیکیورٹی ریکارڈ اور طریقہ کار کا جائزہ لیں  
- **تعمیل کی تصدیق**: یقینی بنائیں کہ تمام اجزاء تنظیمی سیکیورٹی اور ضابطہ کار کی ضروریات کو پورا کرتے ہیں  

#### **محفوظ تعیناتی پائپ لائنز**  
- **خودکار CI/CD سیکیورٹی**: خودکار تعیناتی پائپ لائنز میں سیکیورٹی اسکیننگ کو شامل کریں  
- **آرٹیفیکٹ سالمیت**: تمام تعینات شدہ آرٹیفیکٹس (کوڈ، ماڈلز، کنفیگریشنز) کے لیے کرپٹوگرافک تصدیق نافذ کریں  
- **مرحل وار تعیناتی**: ہر مرحلے پر سیکیورٹی کی تصدیق کے ساتھ تدریجی تعیناتی حکمت عملی استعمال کریں  
- **معتبر آرٹیفیکٹ ریپوزیٹریز**: صرف تصدیق شدہ، محفوظ آرٹیفیکٹ رجسٹریز اور ریپوزیٹریز سے تعینات کریں  

#### **مسلسل نگرانی اور ردعمل**  
- **انحصار اسکیننگ**: تمام سافٹ ویئر اور AI اجزاء کے انحصار کی کمزوریوں کی مسلسل نگرانی  
- **ماڈل کی نگرانی**: ماڈل کے رویے، کارکردگی میں تبدیلی، اور سیکیورٹی انومالیز کا مسلسل جائزہ  
- **سروس صحت کی نگرانی**: بیرونی AI خدمات کی دستیابی، سیکیورٹی واقعات، اور پالیسی تبدیلیوں کی نگرانی  
- **خطرے کی ذہانت کا انضمام**: AI اور ML سیکیورٹی خطرات کے لیے مخصوص خطرے کی معلومات کو شامل کریں  

#### **رسائی کنٹرول اور کم از کم مراعات**  
- **جزو کی سطح کی اجازتیں**: ماڈلز، ڈیٹا، اور خدمات تک رسائی کاروباری ضرورت کی بنیاد پر محدود کریں  
- **سروس اکاؤنٹ مینجمنٹ**: کم سے کم مطلوبہ اجازتوں کے ساتھ مخصوص سروس اکاؤنٹس نافذ کریں  
- **نیٹ ورک تقسیم**: AI اجزاء کو الگ تھلگ کریں اور خدمات کے درمیان نیٹ ورک رسائی محدود کریں  
- **API گیٹ وے کنٹرولز**: بیرونی AI خدمات تک رسائی کو کنٹرول اور مانیٹر کرنے کے لیے مرکزی API گیٹ وے استعمال کریں  

#### **واقعہ کا ردعمل اور بحالی**  
- **تیز ردعمل کے طریقہ کار**: متاثرہ AI اجزاء کی پیچنگ یا تبدیلی کے لیے قائم شدہ عمل  
- **اسناد کی گردش**: راز، API کیز، اور سروس اسناد کی گردش کے لیے خودکار نظام  
- **رول بیک صلاحیتیں**: AI اجزاء کے پچھلے معروف درست ورژنز پر جلدی واپس جانے کی صلاحیت  
- **سپلائی چین خلاف ورزی کی بحالی**: اوپر کی طرف AI سروس کے متاثر ہونے پر مخصوص ردعمل کے طریقہ کار  

### مائیکروسافٹ سیکیورٹی ٹولز اور انضمام

**GitHub Advanced Security** جامع سپلائی چین تحفظ فراہم کرتا ہے جس میں شامل ہیں:  
- **راز اسکیننگ**: ذخائر میں اسناد، API کیز، اور ٹوکنز کی خودکار دریافت  
- **انحصار اسکیننگ**: اوپن سورس انحصار اور لائبریریوں کے لیے کمزوری کا جائزہ  
- **CodeQL تجزیہ**: سیکیورٹی کمزوریوں اور کوڈنگ مسائل کے لیے جامد کوڈ تجزیہ  
- **سپلائی چین بصیرت**: انحصار کی صحت اور سیکیورٹی کی صورتحال کی بصیرت  

**Azure DevOps اور Azure Repos انضمام:**  
- مائیکروسافٹ ترقیاتی پلیٹ فارمز میں بغیر رکاوٹ سیکیورٹی اسکیننگ انضمام  
- AI ورک لوڈز کے لیے Azure پائپ لائنز میں خودکار سیکیورٹی چیکس  
- محفوظ AI جزو تعیناتی کے لیے پالیسی نفاذ  

**مائیکروسافٹ کے اندرونی طریقہ کار:**  
مائیکروسافٹ تمام مصنوعات میں وسیع سپلائی چین سیکیورٹی طریقہ کار نافذ کرتا ہے۔ [The Journey to Secure the Software Supply Chain at Microsoft](https://devblogs.microsoft.com/engineering-at-microsoft/the-journey-to-secure-the-software-supply-chain-at-microsoft/) میں ثابت شدہ طریقوں کے بارے میں جانیں۔  


## بنیادی سیکیورٹی بہترین طریقے

MCP نفاذ آپ کی تنظیم کی موجودہ سیکیورٹی پوزیشن کو وراثت میں لیتے اور اس پر تعمیر کرتے ہیں۔ بنیادی سیکیورٹی طریقوں کو مضبوط کرنا AI سسٹمز اور MCP تعیناتیوں کی مجموعی سیکیورٹی کو نمایاں طور پر بہتر بناتا ہے۔

### بنیادی سیکیورٹی اصول

#### **محفوظ ترقیاتی طریقے**  
- **OWASP تعمیل**: [OWASP Top 10](https://owasp.org/www-project-top-ten/) ویب ایپلیکیشن کمزوریوں سے حفاظت  
- **AI مخصوص تحفظات**: [OWASP Top 10 for LLMs](https://genai.owasp.org/download/43299/?tmstv=1731900559) کے لیے کنٹرول نافذ کریں  
- **محفوظ راز مینجمنٹ**: ٹوکنز، API کیز، اور حساس کنفیگریشن ڈیٹا کے لیے مخصوص والٹس استعمال کریں  
- **اینڈ ٹو اینڈ انکرپشن**: تمام ایپلیکیشن اجزاء اور ڈیٹا فلو میں محفوظ مواصلات نافذ کریں  
- **ان پٹ کی توثیق**: تمام صارف ان پٹس، API پیرامیٹرز، اور ڈیٹا ذرائع کی سخت توثیق  

#### **انفراسٹرکچر کی سختی**  
- **کثیر عنصر توثیق**: تمام انتظامی اور سروس اکاؤنٹس کے لیے لازمی MFA  
- **پیچ مینجمنٹ**: آپریٹنگ سسٹمز، فریم ورکس، اور انحصار کے لیے خودکار، بروقت پیچنگ  
- **شناخت فراہم کنندہ انضمام**: انٹرپرائز شناخت فراہم کنندگان (Microsoft Entra ID، Active Directory) کے ذریعے مرکزی شناخت مینجمنٹ  
- **نیٹ ورک تقسیم**: MCP اجزاء کی منطقی علیحدگی تاکہ افقی حرکت کی صلاحیت محدود ہو  
- **کم از کم مراعات کا اصول**: تمام نظام کے اجزاء اور اکاؤنٹس کے لیے کم سے کم مطلوبہ اجازتیں  

#### **سیکیورٹی نگرانی اور دریافت**  
- **جامع لاگنگ**: AI ایپلیکیشن سرگرمیوں بشمول MCP کلائنٹ-سرور تعاملات کی تفصیلی لاگنگ  
- **SIEM انضمام**: انومالی کی دریافت کے لیے مرکزی سیکیورٹی معلومات اور ایونٹ مینجمنٹ  
- **رویے کا تجزیہ**: نظام اور صارف کے رویے میں غیر معمولی پیٹرنز کی شناخت کے لیے AI سے چلنے والی نگرانی  
- **خطرے کی ذہانت**: بیرونی خطرے کی معلومات اور کمپرو مائز کے اشارے (IOCs) کا انضمام  
- **واقعہ کا ردعمل**: سیکیورٹی واقعہ کی دریافت، ردعمل، اور بحالی کے لیے واضح طریقہ کار  

#### **زیرو ٹرسٹ آرکیٹیکچر**  
- **کبھی اعتماد نہ کریں، ہمیشہ تصدیق کریں**: صارفین، آلات، اور نیٹ ورک کنکشنز کی مسلسل تصدیق  
- **مائیکرو-سیگمنٹیشن**: انفرادی ورک لوڈز اور خدمات کو الگ کرنے والے باریک نیٹ ورک کنٹرولز  
- **شناخت مرکزیت والی سیکیورٹی**: نیٹ ورک مقام کی بجائے تصدیق شدہ شناختوں پر مبنی سیکیورٹی پالیسیاں  
- **مسلسل خطرے کا جائزہ**: موجودہ سیاق و سباق اور رویے کی بنیاد پر متحرک سیکیورٹی پوزیشن کا جائزہ  
- **مشروط رسائی**: رسائی کنٹرول جو خطرے کے عوامل، مقام، اور آلہ کے اعتماد کی بنیاد پر ایڈجسٹ ہوتے ہیں  

### انٹرپرائز انضمام کے نمونے

#### **مائیکروسافٹ سیکیورٹی ماحولیاتی نظام انضمام**  
- **Microsoft Defender for Cloud**: جامع کلاؤڈ سیکیورٹی پوزیشن مینجمنٹ  
- **Azure Sentinel**: AI ورک لوڈ کی حفاظت کے لیے کلاؤڈ-نیٹو SIEM اور SOAR صلاحیتیں  
- **Microsoft Entra ID**: مشروط رسائی پالیسیوں کے ساتھ انٹرپرائز شناخت اور رسائی مینجمنٹ  
- **Azure Key Vault**: ہارڈویئر سیکیورٹی ماڈیول (HSM) کی حمایت کے ساتھ مرکزی راز مینجمنٹ  
- **Microsoft Purview**: AI ڈیٹا ذرائع اور ورک فلو کے لیے ڈیٹا گورننس اور تعمیل  

#### **تعمیل اور گورننس**  
- **ضابطہ کار کی ہم آہنگی**: یقینی بنائیں کہ MCP نفاذ صنعت مخصوص تعمیل کی ضروریات (GDPR، HIPAA، SOC 2) کو پورا کرتے ہیں  
- **ڈیٹا کی درجہ بندی**: AI سسٹمز کے ذریعے پروسیس کیے جانے والے حساس ڈیٹا کی مناسب درجہ بندی اور ہینڈلنگ  
- **آڈٹ ٹریلز**: ضابطہ کار کی تعمیل اور عدالتی تحقیقات کے لیے جامع لاگنگ  
- **پرائیویسی کنٹرولز**: AI سسٹم آرکیٹیکچر میں پرائیویسی بذریعہ ڈیزائن اصولوں کا نفاذ  
- **تبدیلی مینجمنٹ**: AI سسٹم میں تبدیلیوں کے سیکیورٹی جائزوں کے لیے رسمی عمل  

یہ بنیادی طریقے ایک مضبوط سیکیورٹی بنیاد قائم کرتے ہیں جو MCP مخصوص سیکیورٹی کنٹرولز کی تاثیر کو بڑھاتے ہیں اور AI سے چلنے والی ایپلیکیشنز کے لیے جامع تحفظ فراہم کرتے ہیں۔  

## اہم سیکیورٹی نکات

- **سطح وار سیکیورٹی نقطہ نظر**: بنیادی سیکیورٹی طریقوں (محفوظ کوڈنگ، کم از کم مراعات، سپلائی چین کی تصدیق، مسلسل نگرانی) کو AI مخصوص کنٹرولز کے ساتھ ملائیں تاکہ جامع تحفظ حاصل ہو  
- **AI مخصوص خطرات کا منظرنامہ**: MCP سسٹمز منفرد خطرات کا سامنا کرتے ہیں جن میں پرامپٹ انجیکشن، ٹول زہر آلودگی، سیشن ہائی جیکنگ، الجھے ہوئے ڈپٹی کے مسائل، ٹوکن پاس تھرو کی کمزوریاں، اور ضرورت سے زیادہ اجازتیں شامل ہیں جن کے لیے مخصوص تدارک ضروری ہیں  
- **تصدیق اور اجازت میں عمدگی**: بیرونی شناخت فراہم کنندگان (Microsoft Entra ID) کا استعمال کرتے ہوئے مضبوط توثیق نافذ کریں، مناسب ٹوکن کی توثیق کو نافذ کریں، اور کبھی بھی ایسے ٹوکن قبول نہ کریں جو واضح طور پر آپ کے MCP سرور کے لیے جاری نہ کیے گئے ہوں  
- **AI حملے کی روک تھام**: Microsoft Prompt Shields اور Azure Content Safety کو تعینات کریں تاکہ بالواسطہ پرامپٹ انجیکشن اور ٹول زہر آلودگی کے حملوں سے دفاع کیا جا سکے، جبکہ ٹول میٹا ڈیٹا کی تصدیق اور متحرک تبدیلیوں کی نگرانی کریں  
- **سیشن اور ٹرانسپورٹ سیکیورٹی**: کرپٹوگرافک طور پر محفوظ، غیر متعین سیشن آئی ڈیز استعمال کریں جو صارف کی شناخت سے منسلک ہوں، مناسب سیشن لائف سائیکل مینجمنٹ نافذ کریں، اور کبھی بھی توثیق کے لیے سیشنز کا استعمال نہ کریں  
- **OAuth سیکیورٹی بہترین طریقے**: متحرک طور پر رجسٹرڈ کلائنٹس کے لیے واضح صارف رضامندی، PKCE کے ساتھ مناسب OAuth 2.1 نفاذ، اور سخت ری ڈائریکٹ یو آر آئی کی توثیق کے ذریعے الجھے ہوئے ڈپٹی حملوں کو روکیں  
- **ٹوکن سیکیورٹی اصول**: ٹوکن پاس تھرو اینٹی پیٹرنز سے بچیں، ٹوکن ناظرین کے دعووں کی تصدیق کریں، محفوظ گردش کے ساتھ مختصر مدت کے ٹوکن نافذ کریں، اور واضح اعتماد کی حدیں برقرار رکھیں  
- **جامع سپلائی چین سیکیورٹی**: تمام AI ماحولیاتی نظام کے اجزاء (ماڈلز، ایمبیڈنگز، سیاق و سباق فراہم کرنے والے، بیرونی APIز) کو روایتی سافٹ ویئر انحصار کی طرح سخت سیکیورٹی کے ساتھ سنبھالیں  
- **مسلسل ارتقاء**: تیزی سے بدلتی ہوئی MCP وضاحتوں کے ساتھ تازہ ترین رہیں، سیکیورٹی کمیونٹی کے معیارات میں حصہ ڈالیں، اور پروٹوکول کی ترقی کے ساتھ موافق سیکیورٹی پوزیشنز برقرار رکھیں  
- **مائیکروسافٹ سیکیورٹی انضمام**: MCP تعیناتی کے تحفظ کے لیے Microsoft کے جامع سیکیورٹی ماحولیاتی نظام (Prompt Shields، Azure Content Safety، GitHub Advanced Security، Entra ID) کا فائدہ اٹھائیں
