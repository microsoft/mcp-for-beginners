# عملی نفاذ

[![MCP ایپلیکیشنز کو حقیقی آلات اور ورک فلو کے ساتھ کیسے بنائیں، ٹیسٹ کریں، اور تعینات کریں](../../../translated_images/ur/05.64bea204e25ca891.webp)](https://youtu.be/vCN9-mKBDfQ)

_(اس سبق کی ویڈیو دیکھنے کے لیے اوپر تصویر پر کلک کریں)_

عملی نفاذ وہ جگہ ہے جہاں ماڈل کانٹیکسٹ پروٹوکول (MCP) کی طاقت محسوس کی جا سکتی ہے۔ اگرچہ MCP کے پیچھے نظریہ اور فن تعمیر کو سمجھنا اہم ہے، حقیقی قدر اس وقت سامنے آتی ہے جب آپ ان تصورات کو استعمال کرتے ہوئے ایسی حل بنیے، ٹیسٹ کیجیے، اور تعینات کیجیے جو حقیقی دنیا کے مسائل کو حل کرتے ہیں۔ یہ باب نظریاتی علم اور عملی ترقی کے درمیان پل کا کام کرتا ہے، اور آپ کو MCP پر مبنی ایپلیکیشنز کو زندگی میں لانے کے عمل میں رہنمائی فراہم کرتا ہے۔

چاہے آپ ذہین اسسٹنٹس تیار کر رہے ہوں، کاروباری ورک فلو میں AI کو شامل کر رہے ہوں، یا ڈیٹا پراسیسنگ کے لیے کسٹم آلات بنا رہے ہوں، MCP ایک لچکدار بنیاد فراہم کرتا ہے۔ اس کی زبان سے آزاد ڈیزائن اور مقبول پروگرامنگ زبانوں کے لیے سرکاری SDKs اسے وسیع پیمانے پر ڈویلپرز کے لیے قابل رسائی بناتے ہیں۔ ان SDKs کا استعمال کرتے ہوئے، آپ جلدی سے پروٹوٹائپ کر سکتے ہیں، تکرار کر سکتے ہیں، اور اپنے حل کو مختلف پلیٹ فارمز اور ماحول میں بڑھا سکتے ہیں۔

درج ذیل حصوں میں، آپ کو عملی مثالیں، نمونہ کوڈ، اور تعیناتی کی حکمت عملی ملے گی جو دکھاتی ہیں کہ C#، جاوا (Spring کے ساتھ)، ٹائپ اسکرپٹ، جاوا اسکرپٹ، اور پائتھون میں MCP کو کیسے نافذ کیا جائے۔ آپ یہ بھی سیکھیں گے کہ MCP سرورز کو کیسے ڈی بگ اور ٹیسٹ کیا جائے، APIs کا انتظام کیسے کیا جائے، اور Azure کا استعمال کرتے ہوئے کلاؤڈ میں حل کیسے تعینات کیے جائیں۔ یہ عملی وسائل آپ کے سیکھنے کو تیز کرنے اور بااعتماد MCP ایپلیکیشنز بنانے میں مدد کے لیے تیار کیے گئے ہیں۔

## جائزہ

یہ سبق MCP کی مختلف پروگرامنگ زبانوں میں عملی نفاذ کے پہلوؤں پر توجہ دیتا ہے۔ ہم دیکھیں گے کہ C#، جاوا (Spring کے ساتھ)، ٹائپ اسکرپٹ، جاوا اسکرپٹ، اور پائتھون میں MCP SDKs کو کیسے استعمال کیا جائے تاکہ مضبوط ایپلیکیشنز بنائی جا سکیں، MCP سرورز کو ڈی بگ اور ٹیسٹ کیا جا سکے، اور دوبارہ قابل استعمال وسائل، پرامپٹس، اور آلات بنائے جا سکیں۔

## سیکھنے کے مقاصد

اس سبق کے اختتام پر آپ قاد ہوں گے:

- مختلف پروگرامنگ زبانوں میں سرکاری SDKs کا استعمال کرتے ہوئے MCP حل نافذ کریں
- MCP سرورز کو نظام کے تحت ڈی بگ اور ٹیسٹ کریں
- سرور فیچرز (وسائل، پرامپٹس، اور آلات) تخلیق اور استعمال کریں
- پیچیدہ کاموں کے لیے موثر MCP ورک فلو ڈیزائن کریں
- کارکردگی اور اعتماد کے لحاظ سے MCP نافذ کرنے کو بہتر بنائیں

## بنیادی سرور فیچرز

MCP سرورز ان فیچرز کے کسی بھی مجموعے کو نافذ کر سکتے ہیں:

### وسائل

وسائل صارف یا AI ماڈل کے استعمال کے لیے سیاق و سباق اور ڈیٹا مہیا کرتے ہیں:

- دستاویزات کے ذخائر
- معلوماتی بنیادیں
- منظم ڈیٹا کے ذرائع
- فائل سسٹمز

### پرامپٹس

پرامپٹس صارفین کے لیے سانچے والے پیغامات اور ورک فلو ہیں:

- پیشگی طے شدہ گفتگو کے سانچے
- رہنمائی شدہ تعامل کے پیٹرنز
- مخصوص ڈائیلاگ اسٹرکچرز

### آلات

آلات AI ماڈل کے لیے انجام دینے کے فنکشنز ہیں:

- ڈیٹا پراسیسنگ کے ٹولز
- خارجی API انضمامات
- کمپیوٹیشنل صلاحیتیں
- تلاش کی فعالیت

## نمونہ نفاذ: C# نفاذ

سرکاری C# SDK ریپوزیٹری میں MCP کے مختلف پہلوؤں کو ظاہر کرنے والی کئی نمونہ نفاذ شامل ہیں:

- **بنیادی MCP کلائنٹ**: ایک آسان مثال جو دکھاتی ہے کہ MCP کلائنٹ کیسے بنایا جائے اور آلات کو کال کریں
- **بنیادی MCP سرور**: بنیادی ٹول رجسٹریشن کے ساتھ کم از کم سرور نفاذ
- **اعلیٰ درجے کا MCP سرور**: مکمل فیچرز والا سرور جس میں ٹول رجسٹریشن، تصدیق، اور خرابیوں کا انتظام شامل ہے
- **ASP.NET انضمام**: ASP.NET Core کے ساتھ انضمام کی مثالیں
- **آلات نفاذ پیٹرنز**: مختلف پیچیدگی کی سطحوں کے ساتھ آلات نافذ کرنے کے مختلف پیٹرنز

MCP C# SDK پیشگی ورژن میں ہے اور APIs میں تبدیلی آ سکتی ہے۔ ہم اس بلاگ کو SDK کی ترقی کے ساتھ مسلسل اپ ڈیٹ کرتے رہیں گے۔

## ریموٹ MCP سرور کو Azure پر تعینات کریں

آئیے دیکھتے ہیں کہ کیا ہم پہلے ذکر کیے گئے نمونے کو تعینات کر سکتے ہیں:

1. ریپو کلون کریں

    ```bash
    git clone https://github.com/Azure-Samples/remote-mcp-apim-functions-python.git
    cd remote-mcp-apim-functions-python
    ```

1. `Microsoft.App` ریسورس پرووائیڈر رجسٹر کریں۔

   - اگر آپ Azure CLI استعمال کر رہے ہیں، تو چلائیں `az provider register --namespace Microsoft.App --wait`۔
   - اگر آپ Azure PowerShell استعمال کر رہے ہیں، تو چلائیں `Register-AzResourceProvider -ProviderNamespace Microsoft.App`۔ پھر کچھ دیر بعد چلائیں `(Get-AzResourceProvider -ProviderNamespace Microsoft.App).RegistrationState` تاکہ چیک کریں کہ رجسٹریشن مکمل ہوئی یا نہیں۔

1. یہ [azd](https://aka.ms/azd) کمانڈ چلائیں تاکہ API مینجمنٹ سروس، function app (کوڈ کے ساتھ) اور دیگر مطلوبہ Azure وسائل فراہم کیے جا سکیں۔

    ```shell
    azd up
    ```

    یہ کمانڈز تمام کلاؤڈ وسائل Azure پر تعینات کر دیں گی۔

### MCP انسپکٹر کے ساتھ اپنے سرور کی جانچ کریں

1. ایک **نئے ٹرمینل ونڈو** میں MCP انسپکٹر انسٹال اور چلائیں

    ```shell
    npx @modelcontextprotocol/inspector
    ```

    آپ کو ایک ایسا انٹرفیس دیکھنا چاہیے:

    ![Node inspector سے کنیکٹ کریں](../../../translated_images/ur/connect.141db0b2bd05f096.webp)

1. CTRL کلک کریں تاکہ MCP انسپکٹر ویب ایپ اس URL سے لوڈ ہو (مثلاً [http://127.0.0.1:6274/#resources](http://127.0.0.1:6274/#resources))
1. ٹرانسپورٹ کی قسم `SSE` پر سیٹ کریں
1. اپنا چل رہا API مینجمنٹ SSE اینڈ پوائنٹ جو `azd up` کے بعد ظاہر ہوتا ہے، URL فیلڈ میں ڈالیں اور **کنیکٹ** کریں:

    ```shell
    https://<apim-servicename-from-azd-output>.azure-api.net/mcp/sse
    ```

1. **لسٹ ٹولز**۔ کسی ٹول پر کلک کریں اور **ٹول چلائیں**۔

اگر تمام مراحل کامیاب رہے، تو آپ کو اب MCP سرور سے جڑ جانا چاہیے اور آپ ایک ٹول کال کر پائے ہیں۔

## Azure کے لیے MCP سرورز

[Remote-mcp-functions](https://github.com/Azure-Samples/remote-mcp-functions-dotnet): یہ ریپوزیٹریز کی ایک سیریز ہے جو Azure Functions کے ساتھ Python، C# .NET یا Node/TypeScript استعمال کرتے ہوئے کسٹم ریموٹ MCP (ماڈل کانٹیکسٹ پروٹوکول) سرورز بنانے اور تعینات کرنے کے لیے فوری آغاز کے سانچے ہیں۔

یہ نمونے ایک مکمل حل پیش کرتے ہیں جو ڈویلپرز کو اجازت دیتا ہے:

- لوکل طور پر بنائیں اور چلائیں: مقامی مشین پر MCP سرور تیار اور ڈی بگ کریں
- Azure پر تعینات کریں: کلاؤڈ میں تعیناتی آسانی سے `azd up` کمانڈ کے ساتھ کریں
- کلائنٹس سے کنیکٹ ہوں: مختلف کلائنٹس جیسے VS Code کے کو-پائلٹ ایجنٹ موڈ اور MCP انسپکٹر ٹول سے MCP سرور سے جڑیں

### کلیدی خصوصیات

- ڈیزائن کے اعتبار سے سیکیورٹی: MCP سرور کو کیز اور HTTPS کے ذریعے محفوظ کیا گیا ہے
- توثیق کے اختیارات: بلٹ ان اوتھ اور/یا API مینجمنٹ کے ذریعے اوتھ سپورٹ
- نیٹ ورک الگ تھلگ: Azure ورچوئل نیٹ ورکس (VNET) کے ذریعے نیٹ ورک ایزولیشن کی اجازت
- سرورلیس فن تعمیر: Azure Functions کو اسکالیبل، ایونٹ پر مبنی عمل کے لیے استعمال کرتا ہے
- لوکل ترقی: مکمل لوکل ترقی اور ڈی بگنگ کی حمایت
- آسان تعیناتی: Azure پر تعیناتی کے لیے آسان عمل

ریپوزیٹری میں تمام ضروری کنفیگریشن فائلز، ماخذ کوڈ، اور انفراسٹرکچر کی تعریفیں شامل ہیں تاکہ آپ جلدی سے پروڈکشن کے لیے تیار MCP سرور نفاذ کے ساتھ آغاز کر سکیں۔

- [Azure Remote MCP Functions Python](https://github.com/Azure-Samples/remote-mcp-functions-python) - Python کے ساتھ Azure Functions استعمال کرتے ہوئے MCP کا نمونہ نفاذ

- [Azure Remote MCP Functions .NET](https://github.com/Azure-Samples/remote-mcp-functions-dotnet) - C# .NET کے ساتھ Azure Functions استعمال کرتے ہوئے MCP کا نمونہ نفاذ

- [Azure Remote MCP Functions Node/Typescript](https://github.com/Azure-Samples/remote-mcp-functions-typescript) - Node/TypeScript کے ساتھ Azure Functions استعمال کرتے ہوئے MCP کا نمونہ نفاذ۔

## کلیدی نکات

- MCP SDKs مخصوص زبانوں کے لیے مضبوط MCP حل نافذ کرنے کے آلات فراہم کرتے ہیں
- ڈی بگنگ اور ٹیسٹنگ کا عمل قابل اعتماد MCP ایپلیکیشنز کے لیے انتہائی اہم ہے
- دوبارہ قابل استعمال پرامپٹ ٹیمپلیٹس مسلسل AI تعاملات کو ممکن بناتے ہیں
- اچھے ڈیزائن کردہ ورک فلو پیچیدہ کاموں کو متعدد آلات کے ذریعے منظم کر سکتے ہیں
- MCP حل نافذ کرنے میں سیکیورٹی، کارکردگی، اور خرابیوں کے انتظام پر غور ضروری ہے

## مشق

ایک عملی MCP ورک فلو ڈیزائن کریں جو آپ کے شعبے میں کسی حقیقی دنیا کے مسئلے کو حل کرے:

1. 3-4 ایسے آلات کی شناخت کریں جو اس مسئلے کا حل فراہم کرنے میں مددگار ہوں
2. ایک ورک فلو خاکہ بنائیں جو ظاہر کرے کہ یہ آلات کس طرح باہم تعامل کرتے ہیں
3. اپنی پسندیدہ زبان استعمال کرتے ہوئے ان میں سے کسی ایک ٹول کا بنیادی ورژن نافذ کریں
4. ایک پرامپٹ ٹیمپلیٹ بنائیں جو ماڈل کو آپ کے ٹول کو مؤثر طریقے سے استعمال کرنے میں مدد دے
