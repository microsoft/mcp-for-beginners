# Пример: Основной хост

Справочная реализация, показывающая, как создать приложение хоста MCP, которое подключается к серверам MCP и отображает пользовательские интерфейсы инструментов в защищённой песочнице.

Этот базовый хост также можно использовать для тестирования MCP-приложений во время локальной разработки.

## Ключевые файлы

- [`index.html`](../../../../../../../03-GettingStarted/15-mcp-apps/ext-apps/examples/basic-host/index.html) / [`src/index.tsx`](../../../../../../../03-GettingStarted/15-mcp-apps/ext-apps/examples/basic-host/src/index.tsx) - React UI хост с выбором инструмента, вводом параметров и управлением iframe
- [`sandbox.html`](../../../../../../../03-GettingStarted/15-mcp-apps/ext-apps/examples/basic-host/sandbox.html) / [`src/sandbox.ts`](../../../../../../../03-GettingStarted/15-mcp-apps/ext-apps/examples/basic-host/src/sandbox.ts) - Внешний iframe-прокси с проверкой безопасности и двунаправленной передачей сообщений
- [`src/implementation.ts`](../../../../../../../03-GettingStarted/15-mcp-apps/ext-apps/examples/basic-host/src/implementation.ts) - Основная логика: подключение к серверу, вызов инструментов и настройка AppBridge

## Начало работы

```bash
npm install
npm run start
# Откройте http://localhost:8080
```

По умолчанию приложение-хост попытается подключиться к серверу MCP по адресу `http://localhost:3001/mcp`. Вы можете настроить это поведение, установив переменную окружения `SERVERS` с JSON-массивом URL серверов:

```bash
SERVERS='["http://localhost:1234/mcp", "http://localhost:5678/mcp"]' npm run start
```

## Архитектура

В этом примере используется паттерн двух iframe для безопасной изоляции пользовательского интерфейса:

```
Host (port 8080)
  └── Outer iframe (port 8081) - sandbox proxy
        └── Inner iframe (srcdoc) - untrusted tool UI
```

**Зачем нужны два iframe?**

- Внешний iframe работает на отдельном источнике (порт 8081), что предотвращает прямой доступ к хосту
- Внутренний iframe получает HTML через `srcdoc` и ограничен атрибутами песочницы
- Сообщения проходят через внешний iframe, который проверяет и ретранслирует их в обе стороны

Такая архитектура гарантирует, что даже если код пользовательского интерфейса инструмента вредоносный, он не сможет получить доступ к DOM приложения-хоста, куки или контексту JavaScript.
