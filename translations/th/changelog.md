# บันทึกการเปลี่ยนแปลง: หลักสูตร MCP สำหรับผู้เริ่มต้น

เอกสารนี้ทำหน้าที่เป็นบันทึกของการเปลี่ยนแปลงที่สำคัญทั้งหมดที่ทำกับหลักสูตร Model Context Protocol (MCP) สำหรับผู้เริ่มต้น การเปลี่ยนแปลงจะถูกบันทึกในลำดับย้อนกลับตามเวลา (การเปลี่ยนแปลงล่าสุดก่อน)

## 18 ธันวาคม 2025

### การอัปเดตเอกสารความปลอดภัย - ข้อกำหนด MCP 2025-11-25

#### MCP Security Best Practices (02-Security/mcp-best-practices.md) - การอัปเดตเวอร์ชันข้อกำหนด
- **การอัปเดตเวอร์ชันโปรโตคอล**: อัปเดตเพื่ออ้างอิงข้อกำหนด MCP ล่าสุด 2025-11-25 (เผยแพร่วันที่ 25 พฤศจิกายน 2025)
  - อัปเดตการอ้างอิงเวอร์ชันข้อกำหนดทั้งหมดจาก 2025-06-18 เป็น 2025-11-25
  - อัปเดตการอ้างอิงวันที่ในเอกสารจาก 18 สิงหาคม 2025 เป็น 18 ธันวาคม 2025
  - ตรวจสอบให้แน่ใจว่าลิงก์ข้อกำหนดทั้งหมดชี้ไปยังเอกสารปัจจุบัน
- **การตรวจสอบเนื้อหา**: ตรวจสอบอย่างละเอียดเกี่ยวกับแนวทางปฏิบัติที่ดีที่สุดด้านความปลอดภัยตามมาตรฐานล่าสุด
  - **โซลูชันความปลอดภัยของ Microsoft**: ตรวจสอบคำศัพท์และลิงก์ปัจจุบันสำหรับ Prompt Shields (เดิมชื่อ "การตรวจจับความเสี่ยง Jailbreak"), Azure Content Safety, Microsoft Entra ID และ Azure Key Vault
  - **ความปลอดภัย OAuth 2.1**: ยืนยันความสอดคล้องกับแนวทางปฏิบัติที่ดีที่สุดด้านความปลอดภัย OAuth ล่าสุด
  - **มาตรฐาน OWASP**: ตรวจสอบความถูกต้องของการอ้างอิง OWASP Top 10 สำหรับ LLMs ว่ายังคงเป็นปัจจุบัน
  - **บริการ Azure**: ตรวจสอบลิงก์เอกสาร Microsoft Azure ทั้งหมดและแนวทางปฏิบัติที่ดีที่สุด
- **การสอดคล้องกับมาตรฐาน**: ยืนยันว่ามาตรฐานความปลอดภัยที่อ้างอิงทั้งหมดเป็นปัจจุบัน
  - กรอบการจัดการความเสี่ยง AI ของ NIST
  - ISO 27001:2022
  - แนวทางปฏิบัติที่ดีที่สุดด้านความปลอดภัย OAuth 2.1
  - กรอบความปลอดภัยและการปฏิบัติตามของ Azure
- **ทรัพยากรการใช้งาน**: ตรวจสอบลิงก์คู่มือการใช้งานและทรัพยากรทั้งหมด
  - รูปแบบการตรวจสอบสิทธิ์ Azure API Management
  - คู่มือการผสาน Microsoft Entra ID
  - การจัดการความลับ Azure Key Vault
  - ท่อ DevSecOps และโซลูชันการตรวจสอบ

### การประกันคุณภาพเอกสาร
- **การปฏิบัติตามข้อกำหนด**: ตรวจสอบให้แน่ใจว่าข้อกำหนดความปลอดภัย MCP ทั้งหมดที่บังคับใช้ (MUST/MUST NOT) สอดคล้องกับข้อกำหนดล่าสุด
- **ความทันสมัยของทรัพยากร**: ตรวจสอบลิงก์ภายนอกทั้งหมดไปยังเอกสาร Microsoft, มาตรฐานความปลอดภัย และคู่มือการใช้งาน
- **ความครอบคลุมของแนวทางปฏิบัติที่ดีที่สุด**: ยืนยันความครอบคลุมอย่างครบถ้วนของการตรวจสอบสิทธิ์ การอนุญาตภัยคุกคามเฉพาะ AI ความปลอดภัยห่วงโซ่อุปทาน และรูปแบบองค์กร

## 6 ตุลาคม 2025

### การขยายส่วนเริ่มต้น – การใช้งานเซิร์ฟเวอร์ขั้นสูง & การตรวจสอบสิทธิ์ง่ายๆ

#### การใช้งานเซิร์ฟเวอร์ขั้นสูง (03-GettingStarted/10-advanced)
- **เพิ่มบทใหม่**: แนะนำคู่มือครบถ้วนสำหรับการใช้งานเซิร์ฟเวอร์ MCP ขั้นสูง ครอบคลุมทั้งสถาปัตยกรรมเซิร์ฟเวอร์ปกติและระดับต่ำ
  - **เซิร์ฟเวอร์ปกติ vs ระดับต่ำ**: เปรียบเทียบอย่างละเอียดพร้อมตัวอย่างโค้ดใน Python และ TypeScript สำหรับทั้งสองแนวทาง
  - **การออกแบบแบบใช้ Handler**: อธิบายการจัดการเครื่องมือ/ทรัพยากร/พรอมต์แบบใช้ handler สำหรับการใช้งานเซิร์ฟเวอร์ที่ปรับขนาดและยืดหยุ่นได้
  - **รูปแบบการใช้งานจริง**: กรณีใช้งานจริงที่รูปแบบเซิร์ฟเวอร์ระดับต่ำมีประโยชน์สำหรับฟีเจอร์และสถาปัตยกรรมขั้นสูง

#### การตรวจสอบสิทธิ์ง่ายๆ (03-GettingStarted/11-simple-auth)
- **เพิ่มบทใหม่**: คู่มือทีละขั้นตอนสำหรับการใช้งานการตรวจสอบสิทธิ์ง่ายๆ ในเซิร์ฟเวอร์ MCP
  - **แนวคิดการตรวจสอบสิทธิ์**: อธิบายชัดเจนเกี่ยวกับการตรวจสอบสิทธิ์กับการอนุญาต และการจัดการข้อมูลรับรอง
  - **การใช้งาน Basic Auth**: รูปแบบการตรวจสอบสิทธิ์แบบ middleware ใน Python (Starlette) และ TypeScript (Express) พร้อมตัวอย่างโค้ด
  - **การก้าวสู่ความปลอดภัยขั้นสูง**: แนวทางเริ่มต้นด้วยการตรวจสอบสิทธิ์ง่ายๆ และก้าวสู่ OAuth 2.1 และ RBAC พร้อมการอ้างอิงโมดูลความปลอดภัยขั้นสูง

การเพิ่มเนื้อหาเหล่านี้ให้คำแนะนำเชิงปฏิบัติสำหรับการสร้างการใช้งานเซิร์ฟเวอร์ MCP ที่แข็งแกร่ง ปลอดภัย และยืดหยุ่นมากขึ้น เชื่อมโยงแนวคิดพื้นฐานกับรูปแบบการผลิตขั้นสูง

### การปรับปรุงกรณีศึกษา - การผสาน GitHub MCP Registry

#### กรณีศึกษา (09-CaseStudy/) - เน้นการพัฒนาระบบนิเวศ
- **README.md**: ขยายใหญ่พร้อมกรณีศึกษาครบถ้วนเกี่ยวกับ GitHub MCP Registry
  - **กรณีศึกษา GitHub MCP Registry**: กรณีศึกษาครบถ้วนใหม่ที่วิเคราะห์การเปิดตัว GitHub MCP Registry ในเดือนกันยายน 2025
    - **การวิเคราะห์ปัญหา**: การตรวจสอบอย่างละเอียดเกี่ยวกับการค้นหาและการปรับใช้เซิร์ฟเวอร์ MCP ที่กระจัดกระจาย
    - **สถาปัตยกรรมโซลูชัน**: แนวทาง registry ศูนย์กลางของ GitHub พร้อมการติดตั้ง VS Code ด้วยคลิกเดียว
    - **ผลกระทบทางธุรกิจ**: การปรับปรุงที่วัดผลได้ในการเริ่มต้นใช้งานและประสิทธิภาพของนักพัฒนา
    - **คุณค่าทางกลยุทธ์**: เน้นการปรับใช้ตัวแทนแบบโมดูลและการทำงานร่วมกันข้ามเครื่องมือ
    - **การพัฒนาระบบนิเวศ**: การวางตำแหน่งเป็นแพลตฟอร์มพื้นฐานสำหรับการผสานตัวแทน
  - **โครงสร้างกรณีศึกษาที่ปรับปรุง**: อัปเดตกรณีศึกษาทั้งเจ็ดเรื่องด้วยรูปแบบที่สม่ำเสมอและคำอธิบายครบถ้วน
    - ตัวแทนการเดินทาง AI ของ Azure: เน้นการประสานงานตัวแทนหลายตัว
    - การผสาน Azure DevOps: เน้นการทำงานอัตโนมัติของเวิร์กโฟลว์
    - การดึงเอกสารแบบเรียลไทม์: การใช้งานไคลเอนต์คอนโซล Python
    - ตัวสร้างแผนการศึกษาแบบโต้ตอบ: เว็บแอป Chainlit แบบสนทนา
    - เอกสารในตัวแก้ไข: การผสาน VS Code และ GitHub Copilot
    - การจัดการ API ของ Azure: รูปแบบการผสาน API ระดับองค์กร
    - GitHub MCP Registry: การพัฒนาระบบนิเวศและแพลตฟอร์มชุมชน
  - **บทสรุปครบถ้วน**: เขียนใหม่ส่วนบทสรุปเน้นกรณีศึกษาทั้งเจ็ดที่ครอบคลุมมิติการใช้งาน MCP หลายด้าน
    - การผสานระดับองค์กร การประสานงานตัวแทนหลายตัว ประสิทธิภาพนักพัฒนา
    - การพัฒนาระบบนิเวศ การประยุกต์ใช้ทางการศึกษา
    - ข้อมูลเชิงลึกที่เพิ่มขึ้นเกี่ยวกับรูปแบบสถาปัตยกรรม กลยุทธ์การใช้งาน และแนวทางปฏิบัติที่ดีที่สุด
    - เน้น MCP ในฐานะโปรโตคอลที่ครบถ้วนและพร้อมใช้งานจริง

- **การจัดระเบียบทรัพยากร**: การจัดหมวดหมู่ที่ครอบคลุมของเอกสารอย่างเป็นทางการ, มาตรฐานความปลอดภัย และคู่มือการใช้งาน
- **ตัวบ่งชี้ภาพ**: การทำเครื่องหมายที่ชัดเจนของข้อกำหนดที่บังคับใช้เทียบกับแนวปฏิบัติที่แนะนำ


#### แนวคิดหลัก (01-CoreConcepts/) - การปรับปรุงใหม่ทั้งหมด
- **อัปเดตเวอร์ชันโปรโตคอล**: อัปเดตเพื่ออ้างอิง MCP Specification 2025-06-18 ปัจจุบันพร้อมการกำหนดเวอร์ชันตามวันที่ (รูปแบบ YYYY-MM-DD)
- **การปรับปรุงสถาปัตยกรรม**: ขยายคำอธิบายของ Hosts, Clients และ Servers ให้สะท้อนรูปแบบสถาปัตยกรรม MCP ปัจจุบัน
  - Hosts ถูกกำหนดอย่างชัดเจนว่าเป็นแอปพลิเคชัน AI ที่ประสานการเชื่อมต่อ MCP client หลายรายการ
  - Clients อธิบายว่าเป็นตัวเชื่อมโปรโตคอลที่รักษาความสัมพันธ์แบบหนึ่งต่อหนึ่งกับเซิร์ฟเวอร์
  - Servers ปรับปรุงด้วยสถานการณ์การปรับใช้แบบท้องถิ่นและระยะไกล
- **การปรับโครงสร้าง Primitive**: ปรับโครงสร้างใหม่ทั้งหมดของ server และ client primitives
  - Server Primitives: แหล่งข้อมูล (data sources), Prompts (เทมเพลต), Tools (ฟังก์ชันที่สามารถรันได้) พร้อมคำอธิบายและตัวอย่างโดยละเอียด
  - Client Primitives: การสุ่มตัวอย่าง (LLM completions), การกระตุ้น (user input), การบันทึก (debugging/monitoring)
  - อัปเดตด้วยรูปแบบวิธีการค้นหา (`*/list`), ดึงข้อมูล (`*/get`), และการเรียกใช้งาน (`*/call`) ปัจจุบัน
- **สถาปัตยกรรมโปรโตคอล**: แนะนำโมเดลสถาปัตยกรรมสองชั้น
  - ชั้นข้อมูล: พื้นฐาน JSON-RPC 2.0 พร้อมการจัดการวงจรชีวิตและ primitives
  - ชั้นขนส่ง: STDIO (ท้องถิ่น) และ HTTP แบบสตรีมพร้อม SSE (ระยะไกล) เป็นกลไกขนส่ง
- **กรอบความปลอดภัย**: หลักการความปลอดภัยที่ครอบคลุมรวมถึงความยินยอมของผู้ใช้ที่ชัดเจน, การปกป้องความเป็นส่วนตัวของข้อมูล, ความปลอดภัยการรันเครื่องมือ และความปลอดภัยชั้นขนส่ง
- **รูปแบบการสื่อสาร**: อัปเดตข้อความโปรโตคอลเพื่อแสดงการเริ่มต้น, การค้นหา, การดำเนินการ และการแจ้งเตือน
- **ตัวอย่างโค้ด**: ปรับปรุงตัวอย่างหลายภาษา (.NET, Java, Python, JavaScript) ให้สะท้อนรูปแบบ MCP SDK ปัจจุบัน

#### ความปลอดภัย (02-Security/) - ปรับโครงสร้างความปลอดภัยอย่างครอบคลุม  
- **การสอดคล้องมาตรฐาน**: สอดคล้องเต็มที่กับข้อกำหนดความปลอดภัย MCP Specification 2025-06-18
- **วิวัฒนาการการพิสูจน์ตัวตน**: บันทึกวิวัฒนาการจากเซิร์ฟเวอร์ OAuth แบบกำหนดเองสู่การมอบหมายผู้ให้บริการตัวตนภายนอก (Microsoft Entra ID)
- **การวิเคราะห์ภัยคุกคามเฉพาะ AI**: ขยายความครอบคลุมของเวกเตอร์การโจมตี AI สมัยใหม่
  - สถานการณ์การโจมตีฉีด prompt โดยละเอียดพร้อมตัวอย่างจริง
  - กลไกการวางยาพิษเครื่องมือและรูปแบบการโจมตี "rug pull"
  - การวางยาพิษหน้าต่างบริบทและการโจมตีความสับสนของโมเดล
- **โซลูชันความปลอดภัย AI ของ Microsoft**: ครอบคลุมระบบนิเวศความปลอดภัย Microsoft อย่างครบถ้วน
  - AI Prompt Shields พร้อมการตรวจจับขั้นสูง, การเน้น และเทคนิคตัวคั่น
  - รูปแบบการผสานรวม Azure Content Safety
  - GitHub Advanced Security สำหรับการปกป้องห่วงโซ่อุปทาน
- **การบรรเทาภัยคุกคามขั้นสูง**: การควบคุมความปลอดภัยโดยละเอียดสำหรับ
  - การแฮ็กเซสชันพร้อมสถานการณ์การโจมตีเฉพาะ MCP และข้อกำหนด session ID ทางคริปโตกราฟี
  - ปัญหา Confused Deputy ในสถานการณ์พร็อกซี MCP พร้อมข้อกำหนดความยินยอมอย่างชัดเจน
  - ช่องโหว่การส่งผ่านโทเค็นพร้อมการควบคุมการตรวจสอบที่บังคับใช้
- **ความปลอดภัยห่วงโซ่อุปทาน**: ขยายความครอบคลุมห่วงโซ่อุปทาน AI รวมถึงโมเดลพื้นฐาน, บริการ embeddings, ผู้ให้บริการบริบท และ API ของบุคคลที่สาม
- **ความปลอดภัยพื้นฐาน**: การผสานรวมที่ปรับปรุงกับรูปแบบความปลอดภัยระดับองค์กรรวมถึงสถาปัตยกรรม zero trust และระบบนิเวศความปลอดภัย Microsoft
- **การจัดระเบียบทรัพยากร**: จัดหมวดหมู่ลิงก์ทรัพยากรอย่างครอบคลุมตามประเภท (เอกสารอย่างเป็นทางการ, มาตรฐาน, งานวิจัย, โซลูชัน Microsoft, คู่มือการใช้งาน)

### การปรับปรุงคุณภาพเอกสาร
- **วัตถุประสงค์การเรียนรู้ที่มีโครงสร้าง**: ปรับปรุงวัตถุประสงค์การเรียนรู้ด้วยผลลัพธ์ที่เฉพาะเจาะจงและปฏิบัติได้
- **การอ้างอิงข้าม**: เพิ่มลิงก์ระหว่างหัวข้อความปลอดภัยและแนวคิดหลักที่เกี่ยวข้อง
- **ข้อมูลปัจจุบัน**: อัปเดตการอ้างอิงวันที่และลิงก์ข้อกำหนดทั้งหมดให้เป็นมาตรฐานปัจจุบัน
- **แนะแนวการใช้งาน**: เพิ่มแนวทางการใช้งานที่เฉพาะเจาะจงและปฏิบัติได้ในทั้งสองส่วน
