# การใช้งานเชิงปฏิบัติ

[![วิธีสร้าง ทดสอบ และปล่อยใช้งานแอป MCP ด้วยเครื่องมือและเวิร์กโฟว์จริง](../../../translated_images/th/05.64bea204e25ca891.webp)](https://youtu.be/vCN9-mKBDfQ)

_(คลิกที่ภาพด้านบนเพื่อดูวิดีโอบทเรียนนี้)_

การใช้งานเชิงปฏิบัติคือจุดที่พลังของ Model Context Protocol (MCP) กลายเป็นรูปธรรม แม้ว่าการเข้าใจทฤษฎีและสถาปัตยกรรมเบื้องหลัง MCP จะเป็นสิ่งสำคัญ แต่คุณค่าที่แท้จริงเกิดขึ้นเมื่อคุณนำแนวคิดเหล่านี้ไปใช้สร้าง ทดสอบ และปล่อยใช้งานโซลูชันที่แก้ไขปัญหาในโลกจริง บทนี้จะเชื่อมช่องว่างระหว่างความรู้เชิงแนวคิดกับการพัฒนาจริง นำทางคุณผ่านกระบวนการนำแอปพลิเคชันที่ใช้ MCP มาสู่ชีวิต

ไม่ว่าคุณจะกำลังพัฒนาผู้ช่วยอัจฉริยะ, ผสานรวม AI เข้ากับเวิร์กโฟว์ทางธุรกิจ หรือสร้างเครื่องมือที่กำหนดเองสำหรับการประมวลผลข้อมูล MCP ให้พื้นฐานที่ยืดหยุ่น การออกแบบที่ไม่ผูกมัดกับภาษาใดภาษาหนึ่งและ SDK อย่างเป็นทางการสำหรับภาษาการเขียนโปรแกรมยอดนิยมทำให้เข้าถึงได้กับนักพัฒนาหลากหลายประเภท โดยใช้ประโยชน์จาก SDK เหล่านี้ คุณสามารถสร้างต้นแบบอย่างรวดเร็ว, ปรับปรุง, และปรับขนาดโซลูชันของคุณบนแพลตฟอร์มและสภาพแวดล้อมต่างๆ

ในส่วนต่อไปนี้ คุณจะพบตัวอย่างเชิงปฏิบัติ โค้ดตัวอย่าง และกลยุทธ์ในการปล่อยใช้งานที่แสดงให้เห็นวิธีใช้งาน MCP ด้วย C#, Java กับ Spring, TypeScript, JavaScript และ Python คุณยังจะได้เรียนรู้วิธีดีบักและทดสอบเซิร์ฟเวอร์ MCP, จัดการ API และปล่อยใช้งานบนคลาวด์ด้วย Azure ทรัพยากรเชิงปฏิบัติเหล่านี้ถูกออกแบบมาเพื่อเร่งการเรียนรู้ของคุณและช่วยให้คุณสร้างแอปพลิเคชัน MCP ที่มั่นคงและพร้อมใช้งานจริงได้อย่างมั่นใจ

## ภาพรวม

บทเรียนนี้เน้นด้านการใช้งานเชิงปฏิบัติของ MCP ในหลายภาษาการเขียนโปรแกรม เราจะสำรวจวิธีใช้ SDK ของ MCP ใน C#, Java กับ Spring, TypeScript, JavaScript และ Python เพื่อสร้างแอปพลิเคชันที่มั่นคง, ดีบักและทดสอบเซิร์ฟเวอร์ MCP, และสร้างทรัพยากร, คำสั่งเรียก (prompts) และเครื่องมือที่ใช้ซ้ำได้

## วัตถุประสงค์การเรียนรู้

เมื่อจบบทเรียนนี้ คุณจะสามารถ:

- ใช้โซลูชัน MCP โดยใช้ SDK อย่างเป็นทางการในหลายภาษาการเขียนโปรแกรม
- ดีบักและทดสอบเซิร์ฟเวอร์ MCP อย่างเป็นระบบ
- สร้างและใช้ฟีเจอร์เซิร์ฟเวอร์ (ทรัพยากร, คำสั่งเรียก, และเครื่องมือ)
- ออกแบบเวิร์กโฟว์ MCP ที่มีประสิทธิภาพสำหรับงานซับซ้อน
- ปรับแต่งการใช้งาน MCP ให้เหมาะสมกับประสิทธิภาพและความน่าเชื่อถือ

## ฟีเจอร์หลักของเซิร์ฟเวอร์

เซิร์ฟเวอร์ MCP สามารถใช้งานฟีเจอร์ใดๆ ต่อไปนี้รวมกันได้:

### ทรัพยากร

ทรัพยากรให้ข้อมูลและบริบทสำหรับผู้ใช้หรือโมเดล AI ใช้:

- คลังเอกสาร
- ฐานความรู้
- แหล่งข้อมูลที่มีโครงสร้าง
- ระบบไฟล์

### คำสั่งเรียก (Prompts)

คำสั่งเรียกเป็นข้อความแม่แบบและเวิร์กโฟว์สำหรับผู้ใช้:

- แม่แบบบทสนทนาที่กำหนดไว้ล่วงหน้า
- รูปแบบการโต้ตอบที่มีการแนะนำ
- โครงสร้างบทสนทนาเฉพาะทาง

### เครื่องมือ

เครื่องมือคือฟังก์ชันที่โมเดล AI เรียกใช้งานได้:

- ยูทิลิตี้สำหรับประมวลผลข้อมูล
- การผสานรวม API ภายนอก
- ความสามารถในการคำนวณ
- ฟังก์ชันการค้นหา

## ตัวอย่างการใช้งาน: การใช้งาน C#

ที่เก็บ SDK C# อย่างเป็นทางการมีตัวอย่างการใช้งานหลายชิ้นที่แสดงแง่มุมต่างๆ ของ MCP:

- **ไคลเอนต์ MCP พื้นฐาน**: ตัวอย่างง่ายแสดงวิธีการสร้างไคลเอนต์ MCP และเรียกใช้เครื่องมือ
- **เซิร์ฟเวอร์ MCP ขั้นพื้นฐาน**: การใช้งานเซิร์ฟเวอร์ขั้นต่ำที่ลงทะเบียนเครื่องมือพื้นฐาน
- **เซิร์ฟเวอร์ MCP ขั้นสูง**: เซิร์ฟเวอร์ครบฟีเจอร์พร้อมการลงทะเบียนเครื่องมือ, การพิสูจน์ตัวตน, และการจัดการข้อผิดพลาด
- **การผสานรวม ASP.NET**: ตัวอย่างแสดงการผสานรวมกับ ASP.NET Core
- **รูปแบบการใช้งานเครื่องมือ**: รูปแบบต่างๆ สำหรับการใช้งานเครื่องมือด้วยความซับซ้อนระดับต่างๆ

SDK C# MCP ยังอยู่ในสถานะพรีวิว และ API อาจเปลี่ยนแปลงได้ เราจะอัปเดตบล็อกนี้อย่างต่อเนื่องเมื่อตัว SDK พัฒนา

## การปล่อยใช้งาน Remote MCP Server บน Azure

มาดูว่าคุณสามารถปล่อยใช้งานตัวอย่างที่กล่าวไว้ข้างต้นได้ไหม:

1. โคลน repo

    ```bash
    git clone https://github.com/Azure-Samples/remote-mcp-apim-functions-python.git
    cd remote-mcp-apim-functions-python
    ```

1. ลงทะเบียนผู้ให้บริการทรัพยากร `Microsoft.App`

   - ถ้าคุณใช้ Azure CLI ให้รันคำสั่ง `az provider register --namespace Microsoft.App --wait`
   - ถ้าคุณใช้ Azure PowerShell ให้รัน `Register-AzResourceProvider -ProviderNamespace Microsoft.App` แล้วรัน `(Get-AzResourceProvider -ProviderNamespace Microsoft.App).RegistrationState` หลังจากนั้นสักครู่เพื่อตรวจสอบว่าการลงทะเบียนเสร็จสิ้นหรือไม่

1. รันคำสั่ง [azd](https://aka.ms/azd) นี้เพื่อจัดเตรียมบริการ api management, function app (พร้อมโค้ด) และทรัพยากร Azure อื่นๆ ที่จำเป็นทั้งหมด

    ```shell
    azd up
    ```

    คำสั่งนี้จะปล่อยใช้งานทรัพยากรคลาวด์ทั้งหมดบน Azure

### การทดสอบเซิร์ฟเวอร์ของคุณด้วย MCP Inspector

1. ใน **หน้าต่างเทอร์มินัลใหม่** ติดตั้งและรัน MCP Inspector

    ```shell
    npx @modelcontextprotocol/inspector
    ```

    คุณควรเห็นอินเทอร์เฟซลักษณะดังนี้:

    ![เชื่อมต่อกับ Node inspector](../../../translated_images/th/connect.141db0b2bd05f096.webp)

1. กด CTRL คลิกเพื่อโหลดเว็บแอป MCP Inspector จาก URL ที่แอปแสดง (เช่น [http://127.0.0.1:6274/#resources](http://127.0.0.1:6274/#resources))
1. ตั้งประเภทการส่งข้อมูลเป็น `SSE`
1. ตั้ง URL ไปที่จุดเอนด์พอยต์ API Management SSE ที่รันอยู่ซึ่งแสดงหลัง `azd up` และ **เชื่อมต่อ**

    ```shell
    https://<apim-servicename-from-azd-output>.azure-api.net/mcp/sse
    ```

1. **แสดงเครื่องมือ** คลิกที่เครื่องมือและ **รันเครื่องมือ**

ถ้าทุกขั้นตอนทำงานได้ดี ตอนนี้คุณควรเชื่อมต่อกับเซิร์ฟเวอร์ MCP ได้และเรียกใช้เครื่องมือได้สำเร็จ

- สร้างและรันในเครื่อง: พัฒนาและดีบักเซิร์ฟเวอร์ MCP บนเครื่องท้องถิ่น
- ปล่อยไปยัง Azure: ปล่อยใช้งานบนคลาวด์ได้ง่ายด้วยคำสั่ง azd up
- เชื่อมต่อจากไคลเอนต์: เชื่อมต่อกับเซิร์ฟเวอร์ MCP จากไคลเอนต์ต่างๆ รวมทั้งโหมดตัวแทน Copilot ของ VS Code และเครื่องมือ MCP Inspector

### ฟีเจอร์สำคัญ

- ความปลอดภัยโดยออกแบบ: เซิร์ฟเวอร์ MCP ได้รับการปกป้องด้วยคีย์และ HTTPS
- ตัวเลือกการพิสูจน์ตัวตน: รองรับ OAuth ผ่านการพิสูจน์ตัวตนในตัวและ/หรือ API Management
- การแยกเครือข่าย: รองรับการแยกเครือข่ายด้วย Azure Virtual Networks (VNET)
- สถาปัตยกรรมไร้เซิร์ฟเวอร์: ใช้ Azure Functions สำหรับการทำงานที่สเกลได้และขับเคลื่อนด้วยเหตุการณ์
- การพัฒนาในเครื่อง: รองรับการพัฒนาและดีบักในเครื่องอย่างครบถ้วน
- การปล่อยใช้งานง่าย: กระบวนการปล่อยใช้งานสู่ Azure อย่างง่ายดาย

ที่เก็บนี้มีไฟล์กำหนดค่าที่จำเป็นทั้งหมด, โค้ดต้นฉบับ, และนิยามโครงสร้างพื้นฐานเพื่อเริ่มต้นใช้งานการใช้งานเซิร์ฟเวอร์ MCP ที่พร้อมใช้งานจริงได้อย่างรวดเร็ว

- [Azure Remote MCP Functions Python](https://github.com/Azure-Samples/remote-mcp-functions-python) - ตัวอย่างการใช้งาน MCP โดยใช้ Azure Functions กับ Python
- [Azure Remote MCP Functions .NET](https://github.com/Azure-Samples/remote-mcp-functions-dotnet) - ตัวอย่างการใช้งาน MCP โดยใช้ Azure Functions กับ C# .NET
- [Azure Remote MCP Functions Node/Typescript](https://github.com/Azure-Samples/remote-mcp-functions-typescript) - ตัวอย่างการใช้งาน MCP โดยใช้ Azure Functions กับ Node/TypeScript

## ประเด็นสำคัญ

- SDK ของ MCP ให้เครื่องมือเฉพาะภาษาสำหรับการสร้างโซลูชัน MCP ที่มั่นคง
- กระบวนการดีบักและทดสอบมีความสำคัญสำหรับแอป MCP ที่เชื่อถือได้
- แม่แบบคำสั่งเรียกใช้ซ้ำช่วยให้ปฏิสัมพันธ์กับ AI มีความสม่ำเสมอ
- เวิร์กโฟว์ที่ออกแบบดีช่วยประสานงานงานซับซ้อนโดยใช้เครื่องมือหลายตัว
- การใช้งานโซลูชัน MCP ต้องพิจารณาด้านความปลอดภัย, ประสิทธิภาพ, และการจัดการข้อผิดพลาด

## แบบฝึกหัด

ออกแบบเวิร์กโฟว์ MCP เชิงปฏิบัติที่แก้ปัญหาในโลกจริงในโดเมนของคุณ:

1. ระบุเครื่องมือ 3-4 ตัวที่เป็นประโยชน์สำหรับแก้ปัญหานี้
2. สร้างแผนภาพเวิร์กโฟว์แสดงการทำงานร่วมกันของเครื่องมือเหล่านี้
3. สร้างเวอร์ชันพื้นฐานของเครื่องมือหนึ่งตัวโดยใช้ภาษาที่คุณชื่นชอบ
4. สร้างแม่แบบคำสั่งเรียก (prompt) ที่ช่วยให้โมเดลใช้เครื่องมือของคุณอย่างมีประสิทธิภาพ
