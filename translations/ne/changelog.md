# परिवर्तन विवरण: MCP for Beginners पाठ्यक्रम

यो दस्तावेजले Model Context Protocol (MCP) for Beginners पाठ्यक्रममा गरिएका सबै महत्वपूर्ण परिवर्तनहरूको अभिलेखको रूपमा काम गर्दछ। परिवर्तनहरू उल्टो कालानुक्रमिक क्रममा (नयाँ परिवर्तनहरू पहिले) दस्तावेज गरिएको छ।

## डिसेम्बर १८, २०२५

### सुरक्षा दस्तावेज अद्यावधिक - MCP विशिष्टता २०२५-११-२५

#### MCP सुरक्षा उत्तम अभ्यासहरू (02-Security/mcp-best-practices.md) - विशिष्टता संस्करण अद्यावधिक
- **प्रोटोकल संस्करण अद्यावधिक**: नवीनतम MCP विशिष्टता २०२५-११-२५ (नोभेम्बर २५, २०२५ मा जारी) लाई सन्दर्भित गर्न अद्यावधिक गरियो
  - सबै विशिष्टता संस्करण सन्दर्भहरू २०२५-०६-१८ बाट २०२५-११-२५ मा अद्यावधिक गरियो
  - दस्तावेज मिति सन्दर्भहरू अगस्ट १८, २०२५ बाट डिसेम्बर १८, २०२५ मा अद्यावधिक गरियो
  - सबै विशिष्टता URL हरू वर्तमान दस्तावेजतर्फ संकेत गर्छन् भनी प्रमाणित गरियो
- **सामग्री प्रमाणीकरण**: नवीनतम मापदण्डहरू विरुद्ध सुरक्षा उत्तम अभ्यासहरूको व्यापक प्रमाणीकरण
  - **Microsoft सुरक्षा समाधानहरू**: Prompt Shields (पहिले "Jailbreak risk detection"), Azure Content Safety, Microsoft Entra ID, र Azure Key Vault का लागि वर्तमान शब्दावली र लिंकहरू प्रमाणित गरियो
  - **OAuth 2.1 सुरक्षा**: नवीनतम OAuth सुरक्षा उत्तम अभ्यासहरूसँग मेल खाने पुष्टि गरियो
  - **OWASP मापदण्डहरू**: LLMs का लागि OWASP Top 10 सन्दर्भहरू वर्तमान छन् भनी प्रमाणित गरियो
  - **Azure सेवाहरू**: सबै Microsoft Azure दस्तावेज लिंकहरू र उत्तम अभ्यासहरू प्रमाणित गरियो
- **मापदण्ड मेल**: सबै सन्दर्भित सुरक्षा मापदण्डहरू वर्तमान छन् भनी पुष्टि गरियो
  - NIST AI जोखिम व्यवस्थापन फ्रेमवर्क
  - ISO 27001:2022
  - OAuth 2.1 सुरक्षा उत्तम अभ्यासहरू
  - Azure सुरक्षा र अनुपालन फ्रेमवर्कहरू
- **कार्यान्वयन स्रोतहरू**: सबै कार्यान्वयन मार्गदर्शक लिंकहरू र स्रोतहरू प्रमाणित गरियो
  - Azure API व्यवस्थापन प्रमाणीकरण ढाँचाहरू
  - Microsoft Entra ID एकीकरण मार्गदर्शकहरू
  - Azure Key Vault गोप्य व्यवस्थापन
  - DevSecOps पाइपलाइनहरू र अनुगमन समाधानहरू

### दस्तावेज गुणस्तर आश्वासन
- **विशिष्टता अनुपालन**: सबै अनिवार्य MCP सुरक्षा आवश्यकताहरू (MUST/MUST NOT) नवीनतम विशिष्टतासँग मेल खान्छन् भनी सुनिश्चित गरियो
- **स्रोत वर्तमानता**: Microsoft दस्तावेज, सुरक्षा मापदण्डहरू, र कार्यान्वयन मार्गदर्शकहरूका सबै बाह्य लिंकहरू प्रमाणित गरियो
- **उत्तम अभ्यास कभर**: प्रमाणीकरण, प्राधिकरण, AI-विशिष्ट खतरा, आपूर्ति श्रृंखला सुरक्षा, र उद्यम ढाँचाहरूको व्यापक कभर पुष्टि गरियो

## अक्टोबर ६, २०२५

### सुरु गर्ने खण्ड विस्तार – उन्नत सर्भर प्रयोग र सरल प्रमाणीकरण

#### उन्नत सर्भर प्रयोग (03-GettingStarted/10-advanced)
- **नयाँ अध्याय थपियो**: उन्नत MCP सर्भर प्रयोगको लागि व्यापक मार्गदर्शक परिचय गराइयो, जसले नियमित र कम-स्तर सर्भर वास्तुकलाहरू दुवै समेट्छ।
  - **नियमित बनाम कम-स्तर सर्भर**: दुवै दृष्टिकोणका लागि Python र TypeScript मा विस्तृत तुलना र कोड उदाहरणहरू।
  - **ह्यान्डलर-आधारित डिजाइन**: स्केलेबल, लचिलो सर्भर कार्यान्वयनका लागि ह्यान्डलर-आधारित उपकरण/स्रोत/प्रॉम्प्ट व्यवस्थापनको व्याख्या।
  - **व्यावहारिक ढाँचाहरू**: उन्नत सुविधाहरू र वास्तुकलाका लागि कम-स्तर सर्भर ढाँचाहरू लाभदायक हुने वास्तविक परिदृश्यहरू।

#### सरल प्रमाणीकरण (03-GettingStarted/11-simple-auth)
- **नयाँ अध्याय थपियो**: MCP सर्भरहरूमा सरल प्रमाणीकरण कार्यान्वयन गर्ने चरण-द्वारा-चरण मार्गदर्शक।
  - **प्रमाणीकरण अवधारणाहरू**: प्रमाणीकरण र प्राधिकरणको स्पष्ट व्याख्या, र प्रमाणपत्र व्यवस्थापन।
  - **मूलभूत प्रमाणीकरण कार्यान्वयन**: Python (Starlette) र TypeScript (Express) मा मिडलवेयर-आधारित प्रमाणीकरण ढाँचाहरू, कोड नमूनासहित।
  - **उन्नत सुरक्षा तर्फ प्रगति**: सरल प्रमाणीकरणबाट OAuth 2.1 र RBAC तर्फ बढ्ने मार्गदर्शन, उन्नत सुरक्षा मोड्युलहरूको सन्दर्भसहित।

यी थपहरूले अधिक बलियो, सुरक्षित, र लचिलो MCP सर्भर कार्यान्वयनहरू निर्माण गर्न व्यावहारिक, हात-मा-हात मार्गदर्शन प्रदान गर्छन्, आधारभूत अवधारणाहरूलाई उन्नत उत्पादन ढाँचासँग जोड्दै।

- **वास्तुकला परिमार्जन**: वर्तमान MCP वास्तुकला ढाँचाहरू प्रतिबिम्बित गर्न Hosts, Clients, र Servers को वर्णन सुधारियो
  - Hosts अब स्पष्ट रूपमा AI अनुप्रयोगहरू जसले धेरै MCP क्लाइन्ट कनेक्शनहरू समन्वय गर्छन् भनेर परिभाषित
  - Clients प्रोटोकल कनेक्टरहरू जसले एक-देखि-एक सर्भर सम्बन्ध कायम राख्छन् भनेर वर्णन
  - Servers स्थानीय र दूरस्थ परिनियोजन परिदृश्यहरूसहित सुधारियो
- **प्रिमिटिभ पुनर्संरचना**: सर्भर र क्लाइन्ट प्रिमिटिभहरूको पूर्ण पुनर्निर्माण
  - सर्भर प्रिमिटिभहरू: स्रोतहरू (डेटा स्रोतहरू), प्रॉम्प्टहरू (टेम्प्लेटहरू), उपकरणहरू (कार्यन्वयन योग्य कार्यहरू) विस्तृत व्याख्या र उदाहरणहरूसहित
  - क्लाइन्ट प्रिमिटिभहरू: नमूना (LLM पूर्णता), elicitation (प्रयोगकर्ता इनपुट), लगिङ (डिबगिङ/अनुगमन)
  - वर्तमान डिस्कभरी (`*/list`), पुनःप्राप्ति (`*/get`), र कार्यान्वयन (`*/call`) विधि ढाँचाहरू सहित अद्यावधिक
- **प्रोटोकल वास्तुकला**: दुई-स्तरीय वास्तुकला मोडेल परिचय
  - डेटा तह: JSON-RPC 2.0 आधारभूत जीवनचक्र व्यवस्थापन र प्रिमिटिभहरूसहित
  - ट्रान्सपोर्ट तह: STDIO (स्थानीय) र Streamable HTTP SSE (दूरस्थ) ट्रान्सपोर्ट मेकानिजमहरू
- **सुरक्षा फ्रेमवर्क**: स्पष्ट प्रयोगकर्ता सहमति, डेटा गोपनीयता सुरक्षा, उपकरण कार्यान्वयन सुरक्षा, र ट्रान्सपोर्ट तह सुरक्षा सहित व्यापक सुरक्षा सिद्धान्तहरू
- **सञ्चार ढाँचाहरू**: प्रारम्भ, खोज, कार्यान्वयन, र सूचना प्रवाह देखाउन प्रोटोकल सन्देशहरू अद्यावधिक
- **कोड उदाहरणहरू**: वर्तमान MCP SDK ढाँचाहरू प्रतिबिम्बित गर्न बहुभाषी उदाहरणहरू (.NET, Java, Python, JavaScript) ताजा गरियो

#### सुरक्षा (02-Security/) - व्यापक सुरक्षा पुनर्निर्माण  
- **मापदण्ड अनुरूपता**: MCP Specification 2025-06-18 सुरक्षा आवश्यकतासँग पूर्ण अनुरूपता
- **प्रमाणीकरण विकास**: कस्टम OAuth सर्भरहरूबाट बाह्य पहिचान प्रदायक प्रतिनिधित्व (Microsoft Entra ID) सम्मको विकास दस्तावेजीकरण
- **AI-विशिष्ट खतरा विश्लेषण**: आधुनिक AI आक्रमण भेक्टरहरूको विस्तारित समेट
  - वास्तविक उदाहरणहरूसहित विस्तृत प्रॉम्प्ट इन्जेक्शन आक्रमण परिदृश्यहरू
  - उपकरण विषाक्तता संयन्त्रहरू र "रग पुल" आक्रमण ढाँचाहरू
  - सन्दर्भ विन्डो विषाक्तता र मोडेल भ्रम आक्रमणहरू
- **Microsoft AI सुरक्षा समाधानहरू**: Microsoft सुरक्षा पारिस्थितिकी तन्त्रको व्यापक समेट
  - उन्नत पत्ता लगाउने, स्पटलाइटिङ, र डिलिमिटर प्रविधिहरूसहित AI Prompt Shields
  - Azure Content Safety एकीकरण ढाँचाहरू
  - आपूर्ति श्रृंखला संरक्षणका लागि GitHub Advanced Security
- **उन्नत खतरा न्यूनीकरण**: विस्तृत सुरक्षा नियन्त्रणहरू
  - MCP-विशिष्ट आक्रमण परिदृश्यहरू र क्रिप्टोग्राफिक सत्र ID आवश्यकतासहित सत्र अपहरण
  - स्पष्ट सहमति आवश्यकतासहित MCP प्रोक्सी परिदृश्यहरूमा Confused Deputy समस्या
  - अनिवार्य प्रमाणीकरण नियन्त्रणहरूसहित टोकन पासथ्रू कमजोरिहरू
- **आपूर्ति श्रृंखला सुरक्षा**: आधारभूत मोडेलहरू, एम्बेडिङ सेवा, सन्दर्भ प्रदायकहरू, र तेस्रो-पक्ष API सहित AI आपूर्ति श्रृंखला समेट विस्तार
- **आधार सुरक्षा**: शून्य ट्रस्ट वास्तुकला र Microsoft सुरक्षा पारिस्थितिकी तन्त्र सहित उद्यम सुरक्षा ढाँचासँग सुधारिएको एकीकरण
- **स्रोत संगठन**: प्रकार अनुसार वर्गीकृत व्यापक स्रोत लिंकहरू (आधिकारिक कागजातहरू, मापदण्डहरू, अनुसन्धान, Microsoft समाधानहरू, कार्यान्वयन मार्गनिर्देशन)

### कागजात गुणस्तर सुधारहरू
- **संरचित सिकाइ उद्देश्यहरू**: विशिष्ट, कार्यान्वयन योग्य परिणामहरूसहित सिकाइ उद्देश्यहरू सुधारियो
- **क्रस-रेफरेन्सहरू**: सम्बन्धित सुरक्षा र मुख्य अवधारणा विषयहरूबीच लिंकहरू थपियो
- **हालको जानकारी**: सबै मिति सन्दर्भहरू र विशिष्टता लिंकहरू वर्तमान मापदण्डहरूमा अद्यावधिक
- **कार्यान्वयन मार्गनिर्देशन**: दुवै खण्डहरूमा विशिष्ट, कार्यान्वयन योग्य मार्गनिर्देशन थपियो

## जुलाई १६, २०२५

### README र नेभिगेसन सुधारहरू
- README.md मा पाठ्यक्रम नेभिगेसन पूर्ण रूपमा पुनःडिजाइन गरियो
- `<details>` ट्यागहरूलाई थप पहुँचयोग्य तालिका-आधारित ढाँचाले प्रतिस्थापन गरियो
- नयाँ "alternative_layouts" फोल्डरमा वैकल्पिक लेआउट विकल्पहरू सिर्जना गरियो
- कार्ड-आधारित, ट्याब्ड-शैली, र एकर्डियन-शैली नेभिगेसन उदाहरणहरू थपियो
- सबै नवीनतम फाइलहरू समावेश गर्न रिपोजिटरी संरचना खण्ड अद्यावधिक गरियो
- "यस पाठ्यक्रम कसरी प्रयोग गर्ने" खण्ड स्पष्ट सिफारिसहरूसहित सुधारियो
- MCP विशिष्टता लिंकहरू सही URL तर्फ अद्यावधिक गरियो
- पाठ्यक्रम संरचनामा Context Engineering खण्ड (5.14) थपियो

### अध्ययन मार्गदर्शक अद्यावधिकहरू
- वर्तमान रिपोजिटरी संरचनासँग मेल खाने गरी अध्ययन मार्गदर्शक पूर्ण रूपमा पुनर्लेखन गरियो
- MCP क्लाइन्ट र उपकरणहरू, र लोकप्रिय MCP सर्भरहरूका लागि नयाँ खण्डहरू थपियो
- सबै विषयहरूलाई सही रूपमा प्रतिबिम्बित गर्न Visual Curriculum Map अद्यावधिक गरियो
- सबै विशेषज्ञ क्षेत्रहरू समेट्न उन्नत विषयहरूको वर्णन सुधारियो
- वास्तविक उदाहरणहरू प्रतिबिम्बित गर्न केस स्टडी खण्ड अद्यावधिक गरियो
- यो व्यापक परिवर्तन लग थपियो

### समुदाय योगदानहरू (06-CommunityContributions/)
