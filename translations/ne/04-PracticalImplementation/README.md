# व्यावहारिक कार्यान्वयन

[![वास्तविक उपकरणहरू र कार्यप्रवाहहरूसँग MCP एपहरू कसरी निर्माण, परीक्षण, र कार्यान्वयन गर्ने](../../../translated_images/ne/05.64bea204e25ca891.webp)](https://youtu.be/vCN9-mKBDfQ)

_(यो पाठको भिडियो हेर्न माथिको छविमा क्लिक गर्नुहोस्)_

व्यावहारिक कार्यान्वयन त्यही हो जहाँ मोडेल कन्टेक्स्ट प्रोटोकल (MCP) को शक्ति स्पर्श गर्न सकिने हुन्छ। MCP पछाडि थ्योरी र आर्किटेक्चर बुझ्नु महत्त्वपूर्ण भएता पनि, वास्तविक मूल्य तब आउँछ जब तपाईं यी अवधारणाहरूलाई प्रयोग गरेर वास्तविक विश्वका समस्याहरू समाधान गर्ने समाधानहरू निर्माण, परीक्षण, र कार्यान्वयन गर्नुहुन्छ। यो अध्याय वैचारिक ज्ञान र व्यावहारिक विकास बीचको दूरीलाई पूर्ति गर्छ, तपाईंलाई MCP-आधारित अनुप्रयोगहरूलाई जीवन्त बनाउनको लागि प्रक्रिया मार्फत मार्गदर्शन गर्दै।

तपाईं बुद्धिमान सहायकहरू विकास गर्दै हुनुहुन्छ, व्यापार कार्यप्रवाहहरूमा AI समाकलन गर्दै हुनुहुन्छ वा डाटा प्रशोधनका लागि अनुकूलित उपकरणहरू निर्माण गर्दै हुनुहुन्छ, MCP ले लचिलो आधार प्रदान गर्दछ। यसको भाषा-स्वतन्त्र डिजाइन र लोकप्रिय प्रोग्रामिङ भाषाहरूका लागि आधिकारिक SDK हरूले विभिन्न विकासकर्ताहरूका लागि पहुँचयोग्य बनाउँछ। यी SDK हरूको प्रयोग गरेर, तपाईं छिटो प्रोटोटाइप बनाउन, पुनरावृति गर्न, र विभिन्न प्लेटफर्म र वातावरणहरूमा आफ्ना समाधानहरू मापन गर्न सक्नुहुन्छ।

अगली खण्डहरूमा, तपाईंले C#, Java with Spring, TypeScript, JavaScript, र Python मा MCP कसरी कार्यान्वयन गर्ने भन्ने व्यावहारिक उदाहरणहरू, नमूना कोड, र कार्यान्वयन रणनीतिहरू भेट्नुहुनेछ। साथै तपाईंले आफ्नो MCP सर्भरहरू कसरी डिबग र परीक्षण गर्ने, API हरू व्यवस्थापन गर्ने, र Azure प्रयोग गरेर समाधानहरू क्लाउडमा तैनाथ गर्ने तरिका सिक्नुहुनेछ। यी व्यावहारिक स्रोतहरूले तपाईंको सिकाइलाई तिब्र बनाउने र तपाईंलाई आत्मविश्वासका साथ बलियो, उत्पादन-तयार MCP अनुप्रयोगहरू निर्माण गर्न मद्दत गर्ने उद्देश्य राख्छन्।

## अवलोकन

यो पाठले विभिन्न प्रोग्रामिङ भाषाहरूमा MCP कार्यान्वयनका व्यावहारिक पक्षहरूमा केन्द्रित छ। हामी C#, Java with Spring, TypeScript, JavaScript, र Python मा MCP SDK हरू कसरी प्रयोग गर्ने, बलियो अनुप्रयोगहरू निर्माण गर्ने, MCP सर्भरहरू डिबग र परीक्षण गर्ने, र पुनः प्रयोगयोग्य स्रोतहरू, प्रम्प्टहरू, र उपकरणहरू सिर्जना गर्ने बारे अन्वेषण गर्नेछौं।

## सिक्ने उद्देश्यहरू

यस पाठको अन्त्यसम्म, तपाईं सक्षम हुनुहुनेछ:

- विभिन्न प्रोग्रामिङ भाषाहरूमा आधिकारिक SDK हरू प्रयोग गरी MCP समाधानहरू कार्यान्वयन गर्न
- MCP सर्भरहरू व्यवस्थित रूपमा डिबग र परीक्षण गर्न
- सर्भर सुविधाहरू (स्रोतहरू, प्रम्प्टहरू, र उपकरणहरू) सिर्जना र प्रयोग गर्न
- जटिल कार्यहरूका लागि प्रभावकारी MCP कार्यप्रवाहहरू डिजाइन गर्न
- प्रदर्शन र विश्वसनीयताको लागि MCP कार्यान्वयनहरू अनुकूलित गर्न

## मूल सर्भर सुविधाहरू

MCP सर्भरहरूले यी सुविधाहरूको कुनै पनि संयोजन कार्यान्वयन गर्न सक्छन्:

### स्रोतहरू

स्रोतहरूले प्रयोगकर्ता वा AI मोडेलको प्रयोगको लागि संदर्भ र डाटा प्रदान गर्दछन्:

- कागजात भण्डारहरू
- ज्ञान भण्डारहरू
- संरचित डाटा स्रोतहरू
- फाइल प्रणालीहरू

### प्रम्प्टहरू

प्रम्प्टहरू प्रयोगकर्ताहरूका लागि टेम्प्लेट गरिएको सन्देशहरू र कार्यप्रवाहहरू हुन्:

- पूर्वनिर्धारित संवाद टेम्प्लेटहरू
- मार्गदर्शन गरिएको अन्तरक्रिया ढाँचाहरू
- विशेष संवाद संरचनाहरू

### उपकरणहरू

उपकरणहरू AI मोडेलले कार्यान्वयन गर्नका लागि कार्यहरू हुन्:

- डाटा प्रशोधन सुविधाहरू
- बाह्य API समाकलनहरू
- गणनात्मक क्षमताहरू
- खोज कार्यक्षमता

## नमुना कार्यान्वयन: C# कार्यान्वयन

आधिकारिक C# SDK रिपोजिटरीमा MCP का विभिन्न पक्षहरू प्रदर्शन गर्ने धेरै नमुना कार्यान्वयनहरू छन्:

- **मूल MCP क्लाइन्ट**: कसरी MCP क्लाइन्ट सिर्जना गर्ने र उपकरणहरू कल गर्ने प्रदर्शन गर्ने सामान्य उदाहरण
- **मूल MCP सर्भर**: आधारभूत उपकरण दर्ता सहितको न्यूनतम सर्भर कार्यान्वयन
- **उन्नत MCP सर्भर**: उपकरण दर्ता, प्रमाणीकरण र त्रुटि व्यवस्थापन सहित पूर्ण-विशेषतायुक्त सर्भर
- **ASP.NET एकीकरण**: ASP.NET Core सँग एकीकरण देखाउने उदाहरणहरू
- **उपकरण कार्यान्वयन ढाँचाहरू**: विभिन्न जटिलता स्तरहरूसँग उपकरण कार्यान्वयनका विभिन्न ढाँचाहरू

MCP C# SDK पूर्वावलोकन अवस्थामा छ र API हरू परिवर्तन हुन सक्छन्। हामी SDK विकाससँगै यो ब्लग निरन्तर अपडेट गर्नेछौं।

## Remote MCP Server लाई Azure मा तैनाथ गर्नुहोस्

अगाडि चर्चा गरेका नमूना तैनाथ गर्न सक्छौं कि भएन हेरौं:

1. रिपो क्लोन गर्नुहोस्

    ```bash
    git clone https://github.com/Azure-Samples/remote-mcp-apim-functions-python.git
    cd remote-mcp-apim-functions-python
    ```

1. `Microsoft.App` स्रोत प्रदायक दर्ता गर्नुहोस्।

   - यदि तपाईं Azure CLI प्रयोग गर्दै हुनुहुन्छ भने, `az provider register --namespace Microsoft.App --wait` चलाउनुहोस्।
   - यदि तपाईं Azure PowerShell प्रयोग गर्दै हुनुहुन्छ भने, `Register-AzResourceProvider -ProviderNamespace Microsoft.App` चलाउनुहोस्। त्यसपछि केही समयपछि `Get-AzResourceProvider -ProviderNamespace Microsoft.App).RegistrationState` चलाएर दर्ता पूरा भए नभएको जाँच गर्नुहोस्।

1. यो [azd](https://aka.ms/azd) आदेश चलाउनुहोस् जसले api व्यवस्थापन सेवा, function app (कोड सहित), र अन्य आवश्यक Azure स्रोतहरू प्रोभिजन गर्छ

    ```shell
    azd up
    ```

    यसले Azure मा सबै क्लाउड स्रोतहरू तैनाथ गर्नुपर्छ

1. URL तपाईंको चलिरहेको API Management SSE अन्तिम बिन्दु जसले `azd up` चलाएपछि देखिन्छ त्यहाँ राख्नुहोस् र **Connect** गर्नुहोस्:

    ```shell
    https://<apim-servicename-from-azd-output>.azure-api.net/mcp/sse
    ```

1. **सूची उपकरणहरू**। कुनै उपकरणमा क्लिक गरी **रन टूल** गर्नुहोस्।

यदि सबै चरणहरू काम गरे भने, तपाईं MCP सर्भरसँग जडान हुनुभयो र उपकरण कल गर्न सक्षम हुनुभयो।

## Azure का लागि MCP सर्भरहरू

[Remote-mcp-functions](https://github.com/Azure-Samples/remote-mcp-functions-dotnet): यी रिपोजिटरीहरू Python, C# .NET वा Node/TypeScript प्रयोग गरी Azure Functions मार्फत कस्टम रिमोट MCP (Model Context Protocol) सर्भरहरू बनाउन र तैनाथ गर्नका लागि तिब्र सुरु हुने ढाँचा हुन्।

नमूनाहरूले विकासकर्ताहरूलाई यस्ता पूर्ण समाधानहरू प्रदान गर्दछ:

- स्थानीय रूपमा निर्माण र चलाउनुहोस्: स्थानीय मेसिनमा MCP सर्भर विकास र डिबग गर्नुहोस्
- Azure मा तैनाथ गर्नुहोस्: सादा azd up आदेशको साथ सजिलो क्लाउडमा तैनाथीकरण
- क्लाइन्टबाट जडान गर्नुहोस्: विभिन्न क्लाइन्टहरू जस्तै VS Code को Copilot एजेन्ट मोड र MCP Inspector उपकरणबाट MCP सर्भरमा जडान गर्नुहोस्

### प्रमुख सुविधाहरू

- डिजाइनबाट सुरक्षा: MCP सर्भर कुञ्जीहरू र HTTPS मार्फत सुरक्षित हुन्छ
- प्रमाणीकरण विकल्पहरू: बिल्ट-इन auth र/वा API Management प्रयोग गरी OAuth समर्थन गर्दछ
- नेटवर्क पृथकीकरण: Azure Virtual Networks (VNET) प्रयोग गरी नेटवर्क पृथकीकरण सक्षम गर्दछ
- सर्भरलेस आर्किटेक्चर: Azure Functions को प्रयोगले लचीला, घटना-प्रेरित कार्यान्वयन सक्षम पार्दछ
- स्थानीय विकास: समग्र स्थानीय विकास र डिबग समर्थन
- सजिलो तैनाथीकरण: Azure मा सुव्यवस्थित तैनाथीकरण प्रक्रिया

रिपोजिटरीले सबै आवश्यक कन्फिगरेसन फाइलहरू, स्रोत कोड, र पूर्वाधार परिभाषाहरू समावेश गर्छ जसले उत्पादन-तयार MCP सर्भर कार्यान्वयन सुरु गर्न तुरुन्त मद्दत गर्छ।

- [Azure Remote MCP Functions Python](https://github.com/Azure-Samples/remote-mcp-functions-python) - Azure Functions मार्फत Python प्रयोग गरी MCP को नमुना कार्यान्वयन

- [Azure Remote MCP Functions .NET](https://github.com/Azure-Samples/remote-mcp-functions-dotnet) - Azure Functions मार्फत C# .NET प्रयोग गरी MCP को नमुना कार्यान्वयन

- [Azure Remote MCP Functions Node/Typescript](https://github.com/Azure-Samples/remote-mcp-functions-typescript) - Azure Functions मार्फत Node/TypeScript प्रयोग गरी MCP को नमुना कार्यान्वयन।

## मुख्य लिइने कुरा

- MCP SDK हरूले विशिष्ट भाषाहरूको लागि बलियो MCP समाधान कार्यान्वयन गर्न उपकरणहरू प्रदान गर्छन्
- डिबग र परीक्षण प्रक्रिया विश्वसनीय MCP अनुप्रयोगहरूको लागि अत्यावश्यक छ
- पुनः प्रयोगयोग्य प्रम्प्ट टेम्प्लेटहरूले निरन्तर AI अन्तरक्रियाहरू सक्षम पार्दछन्
- राम्रो डिजाइन गरिएको कार्यप्रवाहहरूले विभिन्न उपकरणहरूको प्रयोग गरेर जटिल कार्यहरू संयोजित गर्न सक्छन्
- MCP समाधान कार्यान्वयन गर्दा सुरक्षा, प्रदर्शन, र त्रुटि व्यवस्थापन विचार गर्नु पर्छ

## अभ्यास

तपाईंको क्षेत्रमा वास्तविक विश्व समस्या समाधान गर्ने व्यावहारिक MCP कार्यप्रवाह डिजाइन गर्नुहोस्:

1. यस समस्याको समाधानका लागि ३-४ उपयोगी उपकरणहरू पहिचान गर्नुहोस्
2. यी उपकरणहरू कसरी अन्तरक्रिया गर्छन् भनेर देखाउने कार्यप्रवाह आरेख सिर्जना गर्नुहोस्
3. आफ्नो प्राथमिक भाषामा एउटा उपकरणको आधारभूत संस्करण कार्यान्वयन गर्नुहोस्
4. मोडेललाई तपाईंको उपकरण प्रभावकारी रूपमा प्रयोग गर्न मद्दत गर्ने प्रम्प्ट टेम्प्लेट सिर्जना गर्नुहोस्
