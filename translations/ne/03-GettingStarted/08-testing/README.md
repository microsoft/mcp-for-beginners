## परीक्षण र डिबगिंग

तपाईंले आफ्नो MCP सर्भर परीक्षण गर्न सुरु गर्नु अघि, उपलब्ध उपकरणहरू र डिबगिंगका लागि उत्तम अभ्यासहरू बुझ्नु आवश्यक छ। प्रभावकारी परीक्षणले तपाइँको सर्भर अपेक्षित रूपमा व्यवहार गर्छ भन्ने सुनिश्चित गर्दछ र समस्या छिटो पत्ता लगाउन र समाधान गर्न मद्दत गर्दछ। तलको खण्डले तपाइँको MCP कार्यान्वयनलाई मान्य गर्ने सिफारिस गरिएका तरिकाहरूलाई देखाउँछ।

## अवलोकन

यो पाठले कसरी उपयुक्त परीक्षण विधि छान्ने र सबैभन्दा प्रभावकारी परीक्षण उपकरण बारेमा छ।

## सिकाइ उद्देश्यहरू

यस पाठको अन्त्यसम्म, तपाईं सक्षम हुनुहुनेछ:

- परीक्षणका विभिन्न तरिकाहरू वर्णन गर्न।
- आफ्नो कोडलाई प्रभावकारी रूपमा परीक्षण गर्न विभिन्न उपकरणहरू प्रयोग गर्न।


## MCP सर्भरहरू परीक्षण गर्दै

MCP ले तपाईँलाई आफ्नो सर्भरहरू परीक्षण र डिबग गर्न मद्दत गर्ने उपकरणहरू प्रदान गर्छ:

- **MCP Inspector**: एक कमाण्ड लाइन उपकरण जुन CLI उपकरण र भिजुअल उपकरण दुवैको रूपमा चलाउन सकिन्छ।
- **म्यानुअल परीक्षण**: तपाइँ curl जस्ता उपकरण प्रयोग गरेर वेब अनुरोधहरू चलाउन सक्नुहुन्छ, तर HTTP चलाउन सक्ने कुनै पनि उपकरणले काम गर्छ।
- **यूनिट परीक्षण**: तपाइँ आफ्नो मनपर्ने परीक्षण फ्रेमवर्क प्रयोग गरेर सर्भर र क्लाइन्ट दुबैको सुविधाहरू परीक्षण गर्न सक्नुहुन्छ।

### MCP Inspector को प्रयोग

हामीले यस उपकरणको प्रयोग अघिल्लो पाठहरूमा वर्णन गरिसकेका छौं तर यो एक उच्च स्तरको कुराकानी गरौं। यो Node.js मा निर्माण गरिएको उपकरण हो र तपाइँ यसलाई `npx` executable बोलाएर प्रयोग गर्न सक्नुहुन्छ जसले यो उपकरणलाई अस्थायी रूपमा डाउनलोड र इन्स्टल गर्छ र तपाइँको अनुरोध चलाएपछि आफैं सफा हुन्छ।

[MCP Inspector](https://github.com/modelcontextprotocol/inspector) ले तपाईलाई मद्दत गर्छ:

- **सर्भर क्षमताहरू पत्ता लगाउनुहोस्**: उपलब्ध स्रोतहरू, उपकरणहरू, र संकेतहरू स्वचालित रूपमा पत्ता लगाउनुहोस्
- **उपकरण कार्यान्वयन परीक्षण गर्नुहोस्**: विभिन्न प्यारामिटरहरू प्रयास गर्नुहोस् र वास्तविक समयमा प्रतिक्रिया हेर्नुहोस्
- **सर्भर मेटाडाटा हेर्नुहोस्**: सर्भर जानकारी, स्किमाहरू, र कन्फिगरेसनहरू निरीक्षण गर्नुहोस्

उपकरणको साधारण चलाउने तरिका यसप्रकार छ:

```bash
npx @modelcontextprotocol/inspector node build/index.js
```

माथिको कमाण्डले MCP र यसको भिजुअल इन्टरफेस सुरु गर्छ र तपाइँको ब्राउजरमा स्थानीय वेब इन्टरफेस खोल्छ। तपाइँले आफ्नो दर्ता गरिएको MCP सर्भरहरू, तिनका उपलब्ध उपकरणहरू, स्रोतहरू, र संकेतहरू देखाउने एक ड्यासबोर्ड देख्नुहुनेछ। इन्टरफेसले तपाइँलाई अन्तरक्रियात्मक रूपमा उपकरण कार्यान्वयन परीक्षण गर्न, सर्भर मेटाडाटा निरीक्षण गर्न, र वास्तविक-समय प्रतिक्रियाहरू हेर्न अनुमति दिन्छ, जसले तपाइँको MCP सर्भर कार्यान्वयनहरूलाई मान्य र डिबग गर्न सजिलो बनाउँछ।

यसले कस्तो देखिन्छ यहाँ छ: ![Inspector](../../../../translated_images/ne/connect.141db0b2bd05f096.webp)

यस उपकरणलाई CLI मोडमा पनि चलाउन सकिन्छ, यस अवस्थामा तपाइँले `--cli` विशेषता थप्नुहुन्छ। यहाँ "CLI" मोडमा उपकरण चलाएको उदाहरण छ जसले सर्भरमा उपलब्ध सबै उपकरणहरू सूचीबद्ध गर्छ:

```sh
npx @modelcontextprotocol/inspector --cli node build/index.js --method tools/list
```

### म्यानुअल परीक्षण

सर्भर क्षमताहरू परीक्षण गर्न Inspector उपकरण चलाउने बाहेक, अर्को समान तरिका HTTP चलाउन सक्ने क्लाइन्ट चलाउनु हो, उदाहरणका लागि curl।

curl को साथ, तपाइँ HTTP अनुरोधहरू प्रयोग गरी सिधै MCP सर्भर परीक्षण गर्न सक्नुहुन्छ:

```bash
# उदाहरण: टेस्ट सर्भर मेटाडाटा
curl http://localhost:3000/v1/metadata

# उदाहरण: उपकरण कार्यान्वयन गर्नुहोस्
curl -X POST http://localhost:3000/v1/tools/execute \
  -H "Content-Type: application/json" \
  -d '{"name": "calculator", "parameters": {"expression": "2+2"}}'
```

माथिल्लो curl प्रयोगबाट, तपाइँले देख्न सक्नुहुन्छ कि तपाइँले POST अनुरोध प्रयोग गरेर उपकरणलाई यसको नाम र प्यारामिटरहरू सहितको प्यालोडको माध्यमले बोलाउनुहुन्छ। आफूलाई सबैभन्दा उपयुक्त विधि प्रयोग गर्नुहोस्। CLI उपकरणहरू सामान्यतया छिटो प्रयोग गर्न र स्क्रिप्ट गर्न उपयुक्त हुन्छन् जुन CI/CD वातावरणमा उपयोगी हुन सक्छ।

### यूनिट परीक्षण

तपाइँका उपकरण र स्रोतहरूको लागि यूनिट परीक्षणहरू बनाउनुहोस् ताकि तिनीहरू अपेक्षित रूपमा काम गर्छन् भनेर सुनिश्चित गर्न सकियोस्। यहाँ केही परीक्षण कोडको उदाहरण छ।

```python
import pytest

from mcp.server.fastmcp import FastMCP
from mcp.shared.memory import (
    create_connected_server_and_client_session as create_session,
)

# सम्पूर्ण मोड्युललाई असिन्क परीक्षणहरूको लागि मार्क गर्नुहोस्
pytestmark = pytest.mark.anyio


async def test_list_tools_cursor_parameter():
    """Test that the cursor parameter is accepted for list_tools.

    Note: FastMCP doesn't currently implement pagination, so this test
    only verifies that the cursor parameter is accepted by the client.
    """

 server = FastMCP("test")

    # दुइटा टेस्ट उपकरणहरू सिर्जना गर्नुहोस्
    @server.tool(name="test_tool_1")
    async def test_tool_1() -> str:
        """First test tool"""
        return "Result 1"

    @server.tool(name="test_tool_2")
    async def test_tool_2() -> str:
        """Second test tool"""
        return "Result 2"

    async with create_session(server._mcp_server) as client_session:
        # कर्सर प्यारामिटर बिना परीक्षण गर्नुहोस् (छोडिएको)
        result1 = await client_session.list_tools()
        assert len(result1.tools) == 2

        # कर्सर=None सँग परीक्षण गर्नुहोस्
        result2 = await client_session.list_tools(cursor=None)
        assert len(result2.tools) == 2

        # कर्सरलाई स्ट्रिङको रूपमा परीक्षण गर्नुहोस्
        result3 = await client_session.list_tools(cursor="some_cursor_value")
        assert len(result3.tools) == 2

        # खाली स्ट्रिङ कर्सरसँग परीक्षण गर्नुहोस्
        result4 = await client_session.list_tools(cursor="")
        assert len(result4.tools) == 2
    
```

माथिको कोडले निम्न गर्दछ:

- pytest फ्रेमवर्क प्रयोग गर्छ जसले तपाइँलाई फङ्शनहरू रूपमा परीक्षणहरू सिर्जना गर्न र assert कथनहरू प्रयोग गर्न अनुमति दिन्छ।
- दुई फरक उपकरणहरूसहित MCP सर्भर सिर्जना गर्छ।
- निश्चित सर्तहरू पूरा भए कि छैनन् पुष्टि गर्न `assert` कथन प्रयोग गर्छ।

[पूर्ण फाइल यहाँ हेर्नुहोस्](https://github.com/modelcontextprotocol/python-sdk/blob/main/tests/client/test_list_methods_cursor.py)

माथिको फाइलका आधारमा, तपाइँ आफ्नो सर्भर परीक्षण गर्न सक्नुहुन्छ र क्षमताहरू ठीक प्रकारले सिर्जना भएका छन् कि छैनन् पुष्टि गर्न सक्नुहुन्छ।

सबै मुख्य SDK हरूमा समान परीक्षण अनुभागहरू हुन्छन् त्यसैले तपाइँले आफ्नो रोजेको रनटाइम अनुसार समायोजन गर्न सक्नुहुन्छ।
