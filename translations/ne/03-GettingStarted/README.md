## शुरुवात गर्दै  

[![Build Your First MCP Server](../../../translated_images/ne/04.0ea920069efd979a.webp)](https://youtu.be/sNDZO9N4m9Y)

_(यस पाठको भिडियो हेर्न माथिको तस्बिरमा क्लिक गर्नुहोस्)_

यो सेक्सनमा धेरै पाठहरू समावेश छन्:

- **1 तपाईंको पहिलो सर्भर**, यस पहिलो पाठमा, तपाईंले तपाईंको पहिलो सर्भर कसरी सिर्जना गर्ने र निरीक्षक उपकरणबाट परीक्षण गर्ने र डिबग गर्ने विधि सिक्नु हुनेछ, [पाठमा](01-first-server/README.md)

- **2 क्लाएन्ट**, यस पाठमा, तपाईंले क्लाएन्ट कसरी लेख्ने जसले तपाईंको सर्भरमा जडान गर्न सक्छ भन्ने सिक्नु हुनेछ, [पाठमा](02-client/README.md)

- **3 एलएलएमसहित क्लाएन्ट**, क्लाएन्ट लेख्ने अझ राम्रो तरिका भनेको एलएलएम थप गरेर जसले तपाईंको सर्भरसँग "नेगोसिएट" गर्न सक्छ, [पाठमा](03-llm-client/README.md)

- **4 Visual Studio Code मा GitHub Copilot Agent मोड प्रयोग गरी सर्भर उपभोग गर्ने**। यहाँ, हामी Visual Studio Code भित्रबाट हाम्रो MCP सर्भर सञ्चालन गर्ने हेरिरहेका छौं, [पाठमा](04-vscode/README.md)

- **5 stdio ट्रान्सपोर्ट सर्भर** stdio ट्रान्सपोर्ट स्थानीय MCP सर्भर-देखि-क्लाएन्ट सञ्‍चारको सिफारिस गरिएको मानक हो, जुन सुरक्षित subprocess-आधारित संचार र निर्मित प्रक्रियाको अलगाव प्रदान गर्दछ [पाठमा](05-stdio-server/README.md)

- **6 MCP सँग HTTP स्ट्रिमिङ (Streamable HTTP)**। आधुनिक HTTP स्ट्रिमिङ ट्रान्सपोर्ट (दूरस्थ MCP सर्भरहरूको लागि सिफारिस गरिएको तरिका [MCP स्पेसिफिकेसन 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/basic/transports/#streamable-http) अनुसार), प्रगति सूचनाहरू, र कसरी स्केलेबल, वास्तविक समयमा MCP सर्भरहरू र क्लाएन्टहरू Streamable HTTP प्रयोग गरी कार्यान्वयन गर्ने सिक्नुहोस्। [पाठमा](06-http-streaming/README.md)

- **7 VSCode लागि AI उपकरण किटको प्रयोग** तपाईंका MCP क्लाएन्ट र सर्भरहरू उपभोग र परीक्षण गर्न [पाठमा](07-aitk/README.md)

- **8 परीक्षण**। यहाँ हामी मुख्यतया कसरी हाम्रा सर्भर र क्लाएन्टलाई विभिन्न तरिकाले परीक्षण गर्ने तिर ध्यान केन्द्रित गर्नेछौं, [पाठमा](08-testing/README.md)

- **9 परिचालन**। यस अध्यायले तपाईंको MCP समाधानहरू परिचालन गर्ने विभिन्न तरिकाहरू हेर्नेछ, [पाठमा](09-deployment/README.md)

- **10 उन्नत सर्भर प्रयोग**। यस अध्यायले उन्नत सर्भर प्रयोग समेट्छ, [पाठमा](./10-advanced/README.md)

- **11 प्रमाणीकरण**। यस अध्यायले सरल प्रमाणीकरण थप्ने तरिका समेट्छ, बेसिक अथा देखि JWT र RBAC प्रयोग सम्म। तपाईंलाई यहाँबाट सुरु गर्न र पछि अध्याय 5 मा उन्नत विषयहरू हेर्न र अध्याय 2 मा सिफारिस गरिएका सुरक्षात्मक सुदृढीकरणहरू गर्न प्रोत्साहित गरिन्छ, [पाठमा](./11-simple-auth/README.md)

- **12 MCP होस्टहरू**। Claude Desktop, Cursor, Cline, र Windsurf सहित सामान्य MCP होस्ट क्लाएन्टहरूको कन्फिगरेसन र प्रयोग गर्नुहोस्। ट्रान्सपोर्ट प्रकारहरू र समस्या समाधान जान्नुहोस्, [पाठमा](./12-mcp-hosts/README.md)

- **13 MCP निरीक्षक**। MCP निरीक्षक उपकरण प्रयोग गरी अन्तरक्रियात्मक रूपमा MCP सर्भरहरू डिबग र परीक्षण गर्नुहोस्। समस्या समाधान उपकरणहरू, स्रोतहरू, र प्रोटोकल सन्देशहरू सिक्नुहोस्, [पाठमा](./13-mcp-inspector/README.md)

- **14 नमुना लिइरहनु**। LLM सम्बन्धित कार्यहरूमा MCP क्लाएन्टहरूसँग सहकार्य गर्ने MCP सर्भरहरू सिर्जना गर्नुहोस्। [पाठमा](./14-sampling/README.md)

- **15 MCP एपहरू**। UI निर्देशनहरू पनि जवाफ दिने MCP सर्भरहरू बनाउनुहोस्। [पाठमा](./15-mcp-apps/README.md)

मोडेल कन्क्टेक्स्ट प्रोटोकल (MCP) एउटा खुला प्रोटोकल हो जसले अनुप्रयोगहरूलाई LLM हरूलाई सन्दर्भ दिन कसरी गर्ने भन्ने मानकीकृत गर्दछ। MCP लाई AI अनुप्रयोगहरूको लागि USB-C पोर्ट जस्तो सोच्नुहोस् - यसले AI मोडेलहरूलाई विभिन्न डाटा स्रोतहरू र उपकरणहरूसँग जोड्ने मानकीकृत तरिका प्रदान गर्दछ।

## सिक्ने उद्देश्यहरू

यस पाठको अन्त्यसम्म, तपाईं सक्षम हुनुहुनेछ:

- C#, Java, Python, TypeScript, र JavaScript मा MCP विकास वातावरण सेटअप गर्न
- अनुकूलित सुविधाहरू (स्रोतहरू, प्रॉम्प्टहरू, र उपकरणहरू) सहित आधारभूत MCP सर्भरहरू निर्माण र परिचालन गर्न
- MCP सर्भरहरूमा जडान हुने होस्ट अनुप्रयोगहरू सिर्जना गर्न
- MCP कार्यान्वयनहरू परीक्षण र डिबग गर्न
- साझा सेटअप चुनौतीहरू र तिनका समाधानहरू बुझ्न
- तपाईंका MCP कार्यान्वयनहरूलाई सामान्य LLM सेवाहरूमा जडान गर्न

## तपाईंको MCP वातावरण सेटअप गर्ने

MCP मा काम सुरु गर्नु अघि, तपाईंको विकास वातावरण तयार पार्नु र आधारभूत कार्यप्रवाह बुझ्नु आवश्यक छ। यो सेक्सनले तपाईंलाई सुरुमा आवश्यक सेटअपका चरणहरू मार्फत मार्गदर्शन गर्नेछ ताकि MCP सँग सहज सुरुवात गर्न सकियोस्।

### पूर्वापेक्षाहरू

MCP विकासमा डुब्नुअघि, सुनिश्चित गर्नुहोस् तपाईंसँग छ:

- **विकास वातावरण**: तपाईंको रोजेको भाषा (C#, Java, Python, TypeScript, वा JavaScript) का लागि
- **IDE/संपादक**: Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm, वा कुनै आधुनिक कोड संपादक
- **प्याकेज म्यानेजरहरू**: NuGet, Maven/Gradle, pip, वा npm/yarn
- **API कुञ्जीहरू**: तपाईंको होस्ट अनुप्रयोगहरूमा प्रयोग गर्ने AI सेवाहरूका लागि

### आधिकारिक एसडिकेहरू

आगामी अध्यायहरूमा तपाईंले Python, TypeScript, Java र .NET प्रयोग गरी बनाइएका समाधानहरू देख्नु हुनेछ। यहाँ सबै आधिकारिक रूपमा समर्थित एसडिकेहरू छन्।

MCP ले धेरै भाषाहरूको लागि आधिकारिक एसडिकेहरू प्रदान गर्दछ ([MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/) अनुसार):
- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk) - Microsoft सँग सहकार्यमा मर्मत गरिएको
- [Java SDK](https://github.com/modelcontextprotocol/java-sdk) - Spring AI सँग सहकार्यमा मर्मत गरिएको
- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) - आधिकारिक TypeScript कार्यान्वयन
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk) - आधिकारिक Python कार्यान्वयन (FastMCP)
- [Kotlin SDK](https://github.com/modelcontextprotocol/kotlin-sdk) - आधिकारिक Kotlin कार्यान्वयन
- [Swift SDK](https://github.com/modelcontextprotocol/swift-sdk) - Loopwork AI सँग सहकार्यमा मर्मत गरिएको
- [Rust SDK](https://github.com/modelcontextprotocol/rust-sdk) - आधिकारिक Rust कार्यान्वयन
- [Go SDK](https://github.com/modelcontextprotocol/go-sdk) - आधिकारिक Go कार्यान्वयन

## मुख्य सिकाइ

- भाषा-विशिष्ट एसडिकेहरूसँग MCP विकास वातावरण सेटअप गर्नु सरल छ
- MCP सर्भरहरू बनाउँदा स्पष्ट स्किमा रहेका उपकरणहरू सिर्जना र दर्ता गर्नुपर्छ
- MCP क्लाएन्टहरूले सर्भरहरू र मोडेलहरूसँग जडान गरि विस्तार गरिएको क्षमता प्रयोग गर्छन्
- परीक्षण र डिबग अत्यावश्यक MCP कार्यान्वयनहरूको विश्वसनीयताको लागि
- परिचालन विकल्पहरू स्थानीय विकासदेखि क्लाउड-आधारित समाधानसम्म फराकिलो छन्
