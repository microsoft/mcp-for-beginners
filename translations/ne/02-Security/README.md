# MCP सुरक्षा: एआई प्रणालीहरूको लागि व्यापक सुरक्षा

[![MCP Security Best Practices](../../../translated_images/ne/03.175aed6dedae133f9d41e49cefd0f0a9a39c3317e1eaa7ef7182696af7534308.png)](https://youtu.be/88No8pw706o)

_(यो पाठको भिडियो हेर्न माथिको तस्बिरमा क्लिक गर्नुहोस्)_

सुरक्षा एआई प्रणाली डिजाइनको आधारभूत पक्ष हो, त्यसैले हामी यसलाई हाम्रो दोस्रो खण्डको रूपमा प्राथमिकता दिन्छौं। यो Microsoft को **Secure by Design** सिद्धान्तसँग मेल खान्छ जुन [Secure Future Initiative](https://www.microsoft.com/security/blog/2025/04/17/microsofts-secure-by-design-journey-one-year-of-success/) बाट आएको हो।

Model Context Protocol (MCP) ले एआई-चालित अनुप्रयोगहरूमा शक्तिशाली नयाँ क्षमता ल्याउँछ भने यसले परम्परागत सफ्टवेयर जोखिमहरू भन्दा पर जान्ने अनौठा सुरक्षा चुनौतीहरू पनि प्रस्तुत गर्दछ। MCP प्रणालीहरूले स्थापित सुरक्षा चिन्ताहरू (सुरक्षित कोडिङ, न्यूनतम अधिकार, आपूर्ति श्रृंखला सुरक्षा) र नयाँ एआई-विशिष्ट खतरा जस्तै प्रॉम्प्ट इन्जेक्शन, उपकरण विषाक्तता, सत्र अपहरण, भ्रमित डिप्युटी आक्रमणहरू, टोकन पासथ्रू कमजोरीहरू, र गतिशील क्षमता संशोधनको सामना गर्छन्।

यस पाठले MCP कार्यान्वयनहरूमा सबैभन्दा महत्वपूर्ण सुरक्षा जोखिमहरू अन्वेषण गर्दछ—प्रमाणीकरण, प्राधिकरण, अत्यधिक अनुमति, अप्रत्यक्ष प्रॉम्प्ट इन्जेक्शन, सत्र सुरक्षा, भ्रमित डिप्युटी समस्या, टोकन व्यवस्थापन, र आपूर्ति श्रृंखला कमजोरीहरू समेट्दै। तपाईंले यी जोखिमहरूलाई कम गर्नका लागि कार्यान्वयन योग्य नियन्त्रणहरू र उत्तम अभ्यासहरू सिक्नुहुनेछ, साथै Microsoft समाधानहरू जस्तै Prompt Shields, Azure Content Safety, र GitHub Advanced Security प्रयोग गरेर आफ्नो MCP परिनियोजनलाई बलियो बनाउन सक्नुहुनेछ।

## सिकाइ उद्देश्यहरू

यस पाठको अन्त्यसम्म, तपाईं सक्षम हुनुहुनेछ:

- **MCP-विशिष्ट खतरा पहिचान गर्नुहोस्**: MCP प्रणालीहरूमा अनौठा सुरक्षा जोखिमहरू जस्तै प्रॉम्प्ट इन्जेक्शन, उपकरण विषाक्तता, अत्यधिक अनुमति, सत्र अपहरण, भ्रमित डिप्युटी समस्या, टोकन पासथ्रू कमजोरीहरू, र आपूर्ति श्रृंखला जोखिमहरू चिन्हित गर्नुहोस्
- **सुरक्षा नियन्त्रणहरू लागू गर्नुहोस्**: बलियो प्रमाणीकरण, न्यूनतम अधिकार पहुँच, सुरक्षित टोकन व्यवस्थापन, सत्र सुरक्षा नियन्त्रणहरू, र आपूर्ति श्रृंखला प्रमाणीकरण सहित प्रभावकारी न्यूनीकरणहरू कार्यान्वयन गर्नुहोस्
- **Microsoft सुरक्षा समाधानहरू प्रयोग गर्नुहोस्**: MCP कार्यभार सुरक्षा लागि Microsoft Prompt Shields, Azure Content Safety, र GitHub Advanced Security बुझ्नुहोस् र तिनीहरूलाई परिनियोजन गर्नुहोस्
- **उपकरण सुरक्षा प्रमाणित गर्नुहोस्**: उपकरण मेटाडाटा प्रमाणीकरणको महत्त्व, गतिशील परिवर्तनहरूको अनुगमन, र अप्रत्यक्ष प्रॉम्प्ट इन्जेक्शन आक्रमणहरू विरुद्ध सुरक्षा बुझ्नुहोस्
- **उत्तम अभ्यासहरू समाहित गर्नुहोस्**: स्थापित सुरक्षा आधारभूत सिद्धान्तहरू (सुरक्षित कोडिङ, सर्भर कडा बनाउने, शून्य विश्वास) लाई MCP-विशिष्ट नियन्त्रणहरूसँग संयोजन गरी व्यापक सुरक्षा सुनिश्चित गर्नुहोस्

# MCP सुरक्षा वास्तुकला र नियन्त्रणहरू

आधुनिक MCP कार्यान्वयनहरूले परम्परागत सफ्टवेयर सुरक्षा र एआई-विशिष्ट खतरा दुवैलाई सम्बोधन गर्ने तहगत सुरक्षा दृष्टिकोणहरू आवश्यक पर्छ। तीव्र रूपमा विकास हुँदै गरेको MCP विशिष्टता यसको सुरक्षा नियन्त्रणहरूलाई परिपक्व बनाउँदैछ, जसले उद्यम सुरक्षा वास्तुकलाहरू र स्थापित उत्तम अभ्यासहरूसँग राम्रो एकीकरण सक्षम पार्दछ।

[Microsoft Digital Defense Report](https://aka.ms/mddr) बाट गरिएको अनुसन्धानले देखाउँछ कि **रिपोर्ट गरिएका ९८% उल्लङ्घनहरू बलियो सुरक्षा स्वच्छताले रोक्न सकिन्छ**। सबैभन्दा प्रभावकारी सुरक्षा रणनीतिले आधारभूत सुरक्षा अभ्यासहरूलाई MCP-विशिष्ट नियन्त्रणहरूसँग संयोजन गर्दछ—प्रमाणित आधारभूत सुरक्षा उपायहरू समग्र सुरक्षा जोखिम घटाउन सबैभन्दा प्रभावकारी रहन्छन्।

## वर्तमान सुरक्षा परिदृश्य

> **सूचना:** यो जानकारी MCP सुरक्षा मापदण्डहरूलाई **१८ डिसेम्बर २०२५** सम्म प्रतिबिम्बित गर्दछ। MCP प्रोटोकल तीव्र रूपमा विकास हुँदैछ, र भविष्यका कार्यान्वयनहरूले नयाँ प्रमाणीकरण ढाँचाहरू र सुधारिएको नियन्त्रणहरू प्रस्तुत गर्न सक्छन्। सधैं वर्तमान [MCP Specification](https://spec.modelcontextprotocol.io/), [MCP GitHub repository](https://github.com/modelcontextprotocol), र [security best practices documentation](https://modelcontextprotocol.io/specification/2025-11-25/basic/security_best_practices) लाई नवीनतम मार्गनिर्देशनका लागि सन्दर्भ गर्नुहोस्।

### MCP प्रमाणीकरणको विकास

MCP विशिष्टताले प्रमाणीकरण र प्राधिकरणमा उल्लेखनीय विकास गरेको छ:

- **मूल दृष्टिकोण**: प्रारम्भिक विशिष्टताले विकासकर्ताहरूलाई कस्टम प्रमाणीकरण सर्भरहरू कार्यान्वयन गर्न आवश्यक पारेको थियो, जहाँ MCP सर्भरहरूले OAuth 2.0 प्राधिकरण सर्भरको रूपमा प्रयोगकर्ताको प्रमाणीकरण सिधै व्यवस्थापन गर्थे
- **वर्तमान मापदण्ड (२०२५-११-२५)**: अद्यावधिक विशिष्टताले MCP सर्भरहरूलाई बाह्य पहिचान प्रदायकहरू (जस्तै Microsoft Entra ID) लाई प्रमाणीकरण प्रतिनिधित्व गर्न अनुमति दिन्छ, जसले सुरक्षा स्थिति सुधार्छ र कार्यान्वयन जटिलता घटाउँछ
- **ट्रान्सपोर्ट लेयर सुरक्षा**: स्थानीय (STDIO) र दूरस्थ (Streamable HTTP) जडानहरूका लागि उचित प्रमाणीकरण ढाँचासहित सुरक्षित ट्रान्सपोर्ट संयन्त्रहरूको सुधारिएको समर्थन

## प्रमाणीकरण र प्राधिकरण सुरक्षा

### वर्तमान सुरक्षा चुनौतीहरू

आधुनिक MCP कार्यान्वयनहरूले प्रमाणीकरण र प्राधिकरणमा विभिन्न चुनौतीहरूको सामना गर्छन्:

### जोखिम र खतरा भेक्टरहरू

- **गलत कन्फिगर गरिएको प्राधिकरण तर्क**: MCP सर्भरहरूमा त्रुटिपूर्ण प्राधिकरण कार्यान्वयनले संवेदनशील डाटा खुलाउन सक्छ र पहुँच नियन्त्रणहरू गलत रूपमा लागू गर्न सक्छ
- **OAuth टोकन सम्झौता**: स्थानीय MCP सर्भर टोकन चोरीले आक्रमणकारीहरूलाई सर्भरको नक्कल गर्न र डाउनस्ट्रीम सेवाहरूमा पहुँच दिन सक्षम बनाउँछ
- **टोकन पासथ्रू कमजोरीहरू**: गलत टोकन ह्यान्डलिङले सुरक्षा नियन्त्रण बाइपास र जवाफदेहितामा खालि ठाउँहरू सिर्जना गर्छ
- **अत्यधिक अनुमति**: अत्यधिक अधिकार प्राप्त MCP सर्भरहरूले न्यूनतम अधिकार सिद्धान्त उल्लङ्घन गर्छन् र आक्रमण सतहहरू विस्तार गर्छन्

#### टोकन पासथ्रू: एक गम्भीर एन्टी-प्याटर्न

**टोकन पासथ्रू वर्तमान MCP प्राधिकरण विशिष्टतामा स्पष्ट रूपमा निषेध गरिएको छ** किनभने यसले गम्भीर सुरक्षा प्रभावहरू निम्त्याउँछ:

##### सुरक्षा नियन्त्रण बाइपास
- MCP सर्भरहरू र डाउनस्ट्रीम APIहरूले महत्वपूर्ण सुरक्षा नियन्त्रणहरू (दर सीमितीकरण, अनुरोध प्रमाणीकरण, ट्राफिक अनुगमन) लागू गर्छन् जुन उचित टोकन प्रमाणीकरणमा निर्भर हुन्छन्
- सिधा क्लाइन्ट-देखि-API टोकन प्रयोगले यी आवश्यक सुरक्षा उपायहरूलाई बाइपास गर्छ, जसले सुरक्षा वास्तुकलालाई कमजोर बनाउँछ

##### जवाफदेहिता र अडिट चुनौतीहरू  
- MCP सर्भरहरूले माथिल्लो तहबाट जारी टोकन प्रयोग गर्ने क्लाइन्टहरूलाई छुट्याउन सक्दैनन्, जसले अडिट ट्रेलहरू भंग गर्छ
- डाउनस्ट्रीम स्रोत सर्भर लगहरूले वास्तविक MCP सर्भर मध्यस्थकर्ताहरूको सट्टा भ्रमपूर्ण अनुरोध उत्पत्तिहरू देखाउँछन्
- घटना अनुसन्धान र अनुपालन अडिटहरू निकै कठिन हुन्छन्

##### डाटा चोरी जोखिमहरू
- अवैध टोकन दाबीहरूले चोरी गरिएका टोकनहरूसँग दुष्ट पक्षहरूलाई MCP सर्भरहरूलाई डाटा चोरीका लागि प्रोक्सीको रूपमा प्रयोग गर्न सक्षम बनाउँछ
- विश्वास सीमा उल्लङ्घनहरूले अनधिकृत पहुँच ढाँचाहरूलाई अनुमति दिन्छ जुन इच्छित सुरक्षा नियन्त्रणहरूलाई बाइपास गर्छ

##### बहु-सेवा आक्रमण भेक्टरहरू
- धेरै सेवाहरूले स्वीकार गरेका सम्झौता टोकनहरूले जडित प्रणालीहरूमा पार्श्वगत आन्दोलन सक्षम पार्छ
- टोकन उत्पत्तिहरू प्रमाणित गर्न नसक्दा सेवाहरू बीचको विश्वास मान्यताहरू उल्लङ्घन हुन सक्छ

### सुरक्षा नियन्त्रण र न्यूनीकरणहरू

**गम्भीर सुरक्षा आवश्यकताहरू:**

> **अनिवार्य**: MCP सर्भरहरूले **त्यस्ता कुनै पनि टोकनहरू स्वीकार गर्नु हुँदैन जुन स्पष्ट रूपमा MCP सर्भरका लागि जारी गरिएको छैनन्**

#### प्रमाणीकरण र प्राधिकरण नियन्त्रणहरू

- **कडा प्राधिकरण समीक्षा**: MCP सर्भर प्राधिकरण तर्कको व्यापक अडिट गर्नुहोस् ताकि मात्र इच्छित प्रयोगकर्ता र क्लाइन्टहरूले संवेदनशील स्रोतहरू पहुँच गर्न सकून्
  - **कार्यान्वयन मार्गदर्शन**: [Azure API Management as Authentication Gateway for MCP Servers](https://techcommunity.microsoft.com/blog/integrationsonazureblog/azure-api-management-your-auth-gateway-for-mcp-servers/4402690)
  - **पहिचान एकीकरण**: [Using Microsoft Entra ID for MCP Server Authentication](https://den.dev/blog/mcp-server-auth-entra-id-session/)

- **सुरक्षित टोकन व्यवस्थापन**: [Microsoft को टोकन प्रमाणीकरण र जीवनचक्र उत्तम अभ्यासहरू](https://learn.microsoft.com/en-us/entra/identity-platform/access-tokens) कार्यान्वयन गर्नुहोस्
  - टोकन दर्शक दाबीहरू MCP सर्भर पहिचानसँग मेल खान्छ कि छैन जाँच गर्नुहोस्
  - उचित टोकन रोटेशन र समाप्ति नीतिहरू लागू गर्नुहोस्
  - टोकन पुन:प्रयोग आक्रमण र अनधिकृत प्रयोग रोक्नुहोस्

- **सुरक्षित टोकन भण्डारण**: विश्राम र ट्रान्जिट दुवैमा इन्क्रिप्सनसहित टोकन भण्डारण सुरक्षित गर्नुहोस्
  - **उत्तम अभ्यासहरू**: [Secure Token Storage and Encryption Guidelines](https://youtu.be/uRdX37EcCwg?si=6fSChs1G4glwXRy2)

#### पहुँच नियन्त्रण कार्यान्वयन

- **न्यूनतम अधिकार सिद्धान्त**: MCP सर्भरहरूलाई मात्र आवश्यक न्यूनतम अनुमति दिनुहोस्
  - नियमित अनुमति समीक्षा र अद्यावधिकहरू गरी अधिकार विस्तार रोक्नुहोस्
  - **Microsoft कागजात**: [Secure Least-Privileged Access](https://learn.microsoft.com/entra/identity-platform/secure-least-privileged-access)

- **भूमिका-आधारित पहुँच नियन्त्रण (RBAC)**: सूक्ष्म भूमिका असाइनमेन्टहरू कार्यान्वयन गर्नुहोस्
  - भूमिकाहरूलाई विशिष्ट स्रोतहरू र कार्यहरूमा कडाइका साथ सीमित गर्नुहोस्
  - आक्रमण सतह विस्तार गर्ने व्यापक वा अनावश्यक अनुमति टाढा राख्नुहोस्

- **लगातार अनुमति अनुगमन**: पहुँच अडिटिङ र अनुगमन निरन्तर कार्यान्वयन गर्नुहोस्
  - अनुमति प्रयोग ढाँचाहरूमा असामान्यताहरू अनुगमन गर्नुहोस्
  - अत्यधिक वा अप्रयुक्त अधिकारहरू तुरुन्त सुधार गर्नुहोस्

## एआई-विशिष्ट सुरक्षा खतरा

### प्रॉम्प्ट इन्जेक्शन र उपकरण हेरफेर आक्रमणहरू

आधुनिक MCP कार्यान्वयनहरूले जटिल एआई-विशिष्ट आक्रमण भेक्टरहरूको सामना गर्छन् जुन परम्परागत सुरक्षा उपायहरूले पूर्ण रूपमा सम्बोधन गर्न सक्दैनन्:

#### **अप्रत्यक्ष प्रॉम्प्ट इन्जेक्शन (क्रस-डोमेन प्रॉम्प्ट इन्जेक्शन)**

**अप्रत्यक्ष प्रॉम्प्ट इन्जेक्शन** MCP-सक्षम एआई प्रणालीहरूमा सबैभन्दा गम्भीर कमजोरीहरूमध्ये एक हो। आक्रमणकारीहरूले दुष्ट निर्देशनहरू बाह्य सामग्रीहरू—कागजातहरू, वेब पृष्ठहरू, इमेलहरू, वा डाटा स्रोतहरू—भित्र लुकाउँछन् जुन एआई प्रणालीहरूले पछि वैध आदेशको रूपमा प्रक्रिया गर्छन्।

**आक्रमण परिदृश्यहरू:**
- **कागजात-आधारित इन्जेक्शन**: प्रशोधित कागजातहरूमा लुकेको दुष्ट निर्देशनहरूले अनपेक्षित एआई क्रियाकलापहरू ट्रिगर गर्छ
- **वेब सामग्री शोषण**: कम्प्रोमाइज्ड वेब पृष्ठहरूमा समावेश प्रॉम्प्टहरूले स्क्र्याप गर्दा एआई व्यवहारलाई नियन्त्रण गर्छ
- **इमेल-आधारित आक्रमणहरू**: इमेलहरूमा दुष्ट प्रॉम्प्टहरूले एआई सहायकहरूलाई जानकारी चुहाउन वा अनधिकृत क्रियाकलाप गर्न बाध्य पार्छ
- **डाटा स्रोत प्रदूषण**: कम्प्रोमाइज्ड डाटाबेस वा APIहरूले दूषित सामग्री एआई प्रणालीहरूलाई सेवा दिन्छ

**वास्तविक प्रभाव**: यी आक्रमणहरूले डाटा चोरी, गोपनीयता उल्लङ्घन, हानिकारक सामग्री उत्पादन, र प्रयोगकर्ता अन्तरक्रियाहरूको हेरफेर निम्त्याउन सक्छ। विस्तृत विश्लेषणका लागि हेर्नुहोस् [Prompt Injection in MCP (Simon Willison)](https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/)।

![Prompt Injection Attack Diagram](../../../translated_images/ne/prompt-injection.ed9fbfde297ca877c15bc6daa808681cd3c3dc7bf27bbbda342ef1ba5fc4f52d.png)

#### **उपकरण विषाक्तता आक्रमणहरू**

**उपकरण विषाक्तता** MCP उपकरणहरू परिभाषित गर्ने मेटाडाटालाई लक्षित गर्छ, जसले LLMs लाई उपकरण विवरण र प्यारामिटरहरू व्याख्या गर्ने तरिका मार्फत कार्यान्वयन निर्णयहरूमा प्रभाव पार्छ।

**आक्रमण संयन्त्रहरू:**
- **मेटाडाटा हेरफेर**: आक्रमणकारीहरूले उपकरण विवरण, प्यारामिटर परिभाषा, वा प्रयोग उदाहरणहरूमा दुष्ट निर्देशनहरू इन्जेक्ट गर्छन्
- **अदृश्य निर्देशनहरू**: उपकरण मेटाडाटामा लुकेको प्रॉम्प्टहरू जुन AI मोडेलहरूले प्रक्रिया गर्छन् तर मानव प्रयोगकर्ताहरूलाई देखिँदैन
- **गतिशील उपकरण संशोधन ("रग पुल्स")**: प्रयोगकर्ताहरूले स्वीकृत गरेका उपकरणहरू पछि दुष्ट क्रियाकलाप गर्न परिवर्तन गरिन्छन् बिना प्रयोगकर्ताको जानकारी
- **प्यारामिटर इन्जेक्शन**: उपकरण प्यारामिटर स्किमाहरूमा समावेश दुष्ट सामग्रीले मोडेल व्यवहारलाई प्रभाव पार्छ

**होस्ट गरिएको सर्भर जोखिमहरू**: दूरस्थ MCP सर्भरहरूले उपकरण परिभाषाहरू प्रयोगकर्ताको प्रारम्भिक स्वीकृतिपछि अपडेट गर्न सक्ने भएकाले पहिले सुरक्षित रहेका उपकरणहरू दुष्ट बन्न सक्छन्। विस्तृत विश्लेषणका लागि हेर्नुहोस् [Tool Poisoning Attacks (Invariant Labs)](https://invariantlabs.ai/blog/mcp-security-notification-tool-poisoning-attacks)।

![Tool Injection Attack Diagram](../../../translated_images/ne/tool-injection.3b0b4a6b24de6befe7d3afdeae44138ef005881aebcfc84c6f61369ce31e3640.png)

#### **थप एआई आक्रमण भेक्टरहरू**

- **क्रस-डोमेन प्रॉम्प्ट इन्जेक्शन (XPIA)**: बहु डोमेनका सामग्रीहरू प्रयोग गरी सुरक्षा नियन्त्रणहरू बाइपास गर्ने जटिल आक्रमणहरू
- **गतिशील क्षमता संशोधन**: उपकरण क्षमताहरूमा वास्तविक-समय परिवर्तनहरू जुन प्रारम्भिक सुरक्षा मूल्याङ्कनबाट बच्छन्
- **सन्दर्भ विन्डो विषाक्तता**: ठूलो सन्दर्भ विन्डोहरूलाई दुष्ट निर्देशनहरू लुकाउन हेरफेर गर्ने आक्रमणहरू
- **मोडेल भ्रम आक्रमणहरू**: मोडेल सीमाहरूको दुरुपयोग गरी अप्रत्याशित वा असुरक्षित व्यवहार सिर्जना गर्ने

### एआई सुरक्षा जोखिम प्रभाव

**उच्च प्रभाव परिणामहरू:**
- **डाटा चोरी**: संवेदनशील उद्यम वा व्यक्तिगत डाटामा अनधिकृत पहुँच र चोरी
- **गोपनीयता उल्लङ्घन**: व्यक्तिगत पहिचानयोग्य जानकारी (PII) र गोप्य व्यापार डाटाको खुलासा  
- **प्रणाली हेरफेर**: महत्वपूर्ण प्रणालीहरू र कार्यप्रवाहहरूमा अनपेक्षित संशोधनहरू
- **प्रमाणीकरण टोकन र सेवा प्रमाणपत्र चोरी**: प्रमाणीकरण टोकन र सेवा प्रमाणपत्रहरूको सम्झौता
- **पार्श्वगत आन्दोलन**: व्यापक नेटवर्क आक्रमणका लागि कम्प्रोमाइज्ड एआई प्रणालीहरूको प्रयोग

### Microsoft एआई सुरक्षा समाधानहरू

#### **AI Prompt Shields: इन्जेक्शन आक्रमणहरू विरुद्ध उन्नत सुरक्षा**

Microsoft **AI Prompt Shields** ले प्रत्यक्ष र अप्रत्यक्ष दुवै प्रॉम्प्ट इन्जेक्शन आक्रमणहरू विरुद्ध बहु सुरक्षा तहहरू मार्फत व्यापक सुरक्षा प्रदान गर्छ:

##### **मुख्य सुरक्षा संयन्त्रहरू:**

1. **उन्नत पहिचान र फिल्टरिङ**
   - मेसिन लर्निङ एल्गोरिदम र NLP प्रविधिहरूले बाह्य सामग्रीमा दुष्ट निर्देशनहरू पत्ता लगाउँछन्
   - कागजातहरू, वेब पृष्ठहरू, इमेलहरू, र डाटा स्रोतहरूको वास्तविक-समय विश्लेषण
   - वैध र दुष्ट प्रॉम्प्ट ढाँचाहरूको सन्दर्भगत बुझाइ

2. **स्पटलाइटिङ प्रविधिहरू**  
   - विश्वासयोग्य प्रणाली निर्देशनहरू र सम्भावित कम्प्रोमाइज्ड बाह्य इनपुटहरू बीच भेद गर्दछ
   - पाठ रूपान्तरण विधिहरू जसले मोडेल सान्दर्भिकता बढाउँछन् र दुष्ट सामग्री अलग गर्छन्
   - एआई प्रणालीहरूले उचित निर्देशन पदानुक्रम कायम राख्न र इन्जेक्ट गरिएका आदेशहरूलाई बेवास्ता गर्न मद्दत गर्छ

3. **डिलिमिटर र डाटामार्किङ प्रणालीहरू**
   - विश्वासयोग्य प्रणाली सन्देशहरू र बाह्य इनपुट पाठबीच स्पष्ट सीमा परिभाषित गर्दछ
   - विशेष मार्करहरूले विश्वासयोग्य र अविश्वसनीय डाटा स्रोतहरू बीच सीमाहरू हाइलाइट गर्छन्
   - स्पष्ट पृथक्करणले निर्देशन भ्रम र अनधिकृत आदेश कार्यान्वयन रोक्छ

4. **निरन्तर खतरा खुफिया**
   - Microsoft ले नयाँ आक्रमण ढाँचाहरू निरन्तर अनुगमन गर्छ र सुरक्षा उपायहरू अद्यावधिक गर्छ
   - नयाँ इन्जेक्शन प्रविधि र आक्रमण भेक्टरहरूको सक्रिय खोजी
   - विकासशील खतरा विरुद्ध प्रभावकारिता कायम राख्न नियमित सुरक्षा मोडेल अपडेटहरू

5. **Azure Content Safety एकीकरण**

   - जेलब्रेक प्रयासहरू, हानिकारक सामग्री, र सुरक्षा नीति उल्लङ्घनहरूको अतिरिक्त पहिचान
   - एआई अनुप्रयोग कम्पोनेन्टहरूमा एकीकृत सुरक्षा नियन्त्रणहरू

## उन्नत MCP सुरक्षा खतरा

### सत्र अपहरण कमजोरीहरू

**सत्र अपहरण** राज्ययुक्त MCP कार्यान्वयनहरूमा एक गम्भीर आक्रमण भेक्टर हो जहाँ अनधिकृत पक्षहरूले वैध सत्र पहिचानकर्ताहरू प्राप्त गरी क्लाइन्टको नक्कल गर्न र अनधिकृत क्रियाकलाप गर्न सक्छन्।

#### **आक्रमण परिदृश्य र जोखिमहरू**

- **सत्र अपहरण प्रॉम्प्ट इन्जेक्शन**: चोरी गरिएका सत्र ID भएका आक्रमणकारीहरूले सत्र अवस्था साझा गर्ने सर्भरहरूमा दुष्ट घटनाहरू इन्जेक्ट गर्छन्, जसले हानिकारक क्रियाकलापहरू ट्रिगर गर्न वा संवेदनशील डाटा पहुँच गर्न सक्छ
- **सिधा नक्कलीकरण**: चोरी गरिएका सत्र IDहरूले प्रमाणीकरण बाइपास गर्दै सिधा MCP सर्भर कलहरू सक्षम पार्छ, जसले आक्रमणकारीलाई वैध प्रयोगकर्ताको रूपमा व्यवहार गर्छ
- **सम्भावित पुनःसञ्चालन योग्य स्ट्रिमहरूमा सम्झौता**: आक्रमणकारीहरूले अनुरोधहरू समयभन्दा पहिले समाप्त पार्न सक्छन्, जसले वैध क्लाइन्टहरूलाई सम्भावित दुष्ट सामग्रीसहित पुनःसञ्चालन गर्न बाध्य पार्छ

#### **सत्र व्यवस्थापनका लागि सुरक्षा नियन्त्रणहरू**

**गम्भीर आवश्यकताहरू:**
- **प्राधिकरण प्रमाणीकरण**: प्राधिकरण कार्यान्वयन गर्ने MCP सर्भरहरूले सबै इनबाउन्ड अनुरोधहरू प्रमाणीकरण गर्न **अवश्य** गर्नुपर्छ र प्रमाणीकरणका लागि सेसनहरूमा निर्भर हुनु हुँदैन
- **सुरक्षित सेसन निर्माण**: क्रिप्टोग्राफिक रूपमा सुरक्षित, गैर-निर्धारित सेसन ID हरू प्रयोग गर्नुहोस् जुन सुरक्षित र्यान्डम नम्बर जेनेरेटरहरूसँग उत्पन्न गरिन्छ
- **प्रयोगकर्ता-विशिष्ट बाइन्डिङ**: सेसन ID हरूलाई प्रयोगकर्ता-विशिष्ट जानकारीसँग `<user_id>:<session_id>` जस्ता ढाँचाहरू प्रयोग गरी बाँध्नुहोस् ताकि क्रस-प्रयोगकर्ता सेसन दुरुपयोग रोक्न सकियोस्
- **सेसन जीवनचक्र व्यवस्थापन**: जोखिमको समयावधि सीमित गर्न उचित समाप्ति, रोटेशन, र अमान्यकरण कार्यान्वयन गर्नुहोस्
- **ट्रान्सपोर्ट सुरक्षा**: सेसन ID हरूको अवरोध रोक्न सबै सञ्चारका लागि अनिवार्य HTTPS प्रयोग गर्नुहोस्

### भ्रमित डिप्युटी समस्या

**भ्रमित डिप्युटी समस्या** तब हुन्छ जब MCP सर्भरहरूले क्लाइन्ट र तेस्रो-पक्ष सेवाहरू बीच प्रमाणीकरण प्रोक्सीको रूपमा काम गर्छन्, जसले स्थिर क्लाइन्ट ID को दुरुपयोगबाट प्राधिकरण बाइपासको अवसर सिर्जना गर्छ।

#### **आक्रमण प्रक्रिया र जोखिमहरू**

- **कुकी-आधारित सहमति बाइपास**: पहिलेको प्रयोगकर्ता प्रमाणीकरणले सहमति कुकीहरू सिर्जना गर्छ जुन आक्रमणकारीहरूले बनाइएको रिडिरेक्ट URI हरू सहित दुष्ट प्राधिकरण अनुरोधहरू मार्फत दुरुपयोग गर्छन्
- **प्राधिकरण कोड चोरी**: अवस्थित सहमति कुकीहरूले प्राधिकरण सर्भरहरूलाई सहमति स्क्रिनहरू छोड्न बाध्य पार्न सक्छ, जसले कोडहरू आक्रमणकारी-नियन्त्रित अन्तबिन्दुहरूमा रिडिरेक्ट गर्छ  
- **अनधिकृत API पहुँच**: चोरी गरिएका प्राधिकरण कोडहरूले टोकन विनिमय र प्रयोगकर्ता नक्कलीकरण अनुमति दिन्छन् बिना स्पष्ट स्वीकृतिका

#### **निवारण रणनीतिहरू**

**अनिवार्य नियन्त्रणहरू:**
- **स्पष्ट सहमति आवश्यकताहरू**: स्थिर क्लाइन्ट ID प्रयोग गर्ने MCP प्रोक्सी सर्भरहरूले प्रत्येक गतिशील रूपमा दर्ता गरिएको क्लाइन्टको लागि प्रयोगकर्ता सहमति लिनु **अवश्य** छ
- **OAuth 2.1 सुरक्षा कार्यान्वयन**: सबै प्राधिकरण अनुरोधहरूका लागि PKCE (प्रूफ की फर कोड एक्सचेन्ज) सहित वर्तमान OAuth सुरक्षा सर्वोत्तम अभ्यासहरू पालना गर्नुहोस्
- **कडा क्लाइन्ट प्रमाणीकरण**: दुरुपयोग रोक्न रिडिरेक्ट URI र क्लाइन्ट पहिचानकर्ताहरूको कडाइका साथ प्रमाणीकरण गर्नुहोस्

### टोकन पासथ्रू कमजोरीहरू  

**टोकन पासथ्रू** एक स्पष्ट एन्टी-प्याटर्न हो जहाँ MCP सर्भरहरूले क्लाइन्ट टोकनहरू उचित प्रमाणीकरण बिना स्वीकार्छन् र तिनीहरूलाई डाउनस्ट्रीम API हरूमा अग्रेषित गर्छन्, जसले MCP प्राधिकरण विनिर्देशहरूको उल्लंघन गर्छ।

#### **सुरक्षा प्रभावहरू**

- **नियन्त्रण बाइपास**: सिधा क्लाइन्ट-देखि-API टोकन प्रयोगले महत्वपूर्ण दर सीमितीकरण, प्रमाणीकरण, र अनुगमन नियन्त्रणहरू बाइपास गर्छ
- **अडिट ट्रेल भ्रष्टाचार**: माथिल्लो स्तरबाट जारी टोकनहरूले क्लाइन्ट पहिचान असम्भव बनाउँछ, जसले घटना अनुसन्धान क्षमताहरू तोड्छ
- **प्रोक्सी-आधारित डाटा चोरी**: अमान्य टोकनहरूले दुष्ट पात्रहरूलाई सर्भरहरूलाई अनधिकृत डाटा पहुँचका लागि प्रोक्सीको रूपमा प्रयोग गर्न सक्षम बनाउँछ
- **विश्वास सीमा उल्लंघन**: टोकन उत्पत्तिको प्रमाणीकरण गर्न नसक्दा डाउनस्ट्रीम सेवाहरूको विश्वास मान्यताहरू उल्लंघन हुन सक्छन्
- **बहु-सेवा आक्रमण विस्तार**: धेरै सेवाहरूमा स्वीकार गरिएका कम्प्रोमाइज्ड टोकनहरूले पार्श्वगत आन्दोलन सक्षम पार्छ

#### **आवश्यक सुरक्षा नियन्त्रणहरू**

**अपरिहार्य आवश्यकताहरू:**
- **टोकन प्रमाणीकरण**: MCP सर्भरहरूले स्पष्ट रूपमा MCP सर्भरका लागि जारी नभएका टोकनहरू स्वीकार्नु हुँदैन
- **दर्शक प्रमाणीकरण**: सधैं टोकन दर्शक दाबीहरू MCP सर्भरको पहिचानसँग मेल खान्छ कि छैन जाँच गर्नुहोस्
- **उचित टोकन जीवनचक्र**: छोटो अवधिका पहुँच टोकनहरू सुरक्षित रोटेशन अभ्यासहरूसँग कार्यान्वयन गर्नुहोस्


## AI प्रणालीहरूको लागि आपूर्ति श्रृंखला सुरक्षा

आपूर्ति श्रृंखला सुरक्षा परम्परागत सफ्टवेयर निर्भरताहरू भन्दा पर विकसित भई सम्पूर्ण AI पारिस्थितिकी तन्त्र समेट्छ। आधुनिक MCP कार्यान्वयनहरूले सबै AI-संबन्धित कम्पोनेन्टहरू कडाइका साथ प्रमाणीकरण र अनुगमन गर्नुपर्छ, किनकि प्रत्येकले प्रणाली अखण्डतामा सम्भावित कमजोरीहरू ल्याउन सक्छ।

### विस्तारित AI आपूर्ति श्रृंखला कम्पोनेन्टहरू

**परम्परागत सफ्टवेयर निर्भरताहरू:**
- खुला स्रोत पुस्तकालयहरू र फ्रेमवर्कहरू
- कन्टेनर इमेजहरू र आधार प्रणालीहरू  
- विकास उपकरणहरू र निर्माण पाइपलाइनहरू
- पूर्वाधार कम्पोनेन्टहरू र सेवाहरू

**AI-विशिष्ट आपूर्ति श्रृंखला तत्वहरू:**
- **फाउन्डेशन मोडेलहरू**: विभिन्न प्रदायकहरूबाट पूर्व-प्रशिक्षित मोडेलहरू जसको उत्पत्तिको प्रमाणीकरण आवश्यक छ
- **एम्बेडिङ सेवा**: बाह्य भेक्टराइजेशन र सेम्यान्टिक खोज सेवा
- **सन्दर्भ प्रदायकहरू**: डाटा स्रोतहरू, ज्ञान आधारहरू, र कागजात भण्डारहरू  
- **तेस्रो-पक्ष API हरू**: बाह्य AI सेवा, ML पाइपलाइनहरू, र डाटा प्रशोधन अन्तबिन्दुहरू
- **मोडेल कलाकृतिहरू**: तौलहरू, कन्फिगरेसनहरू, र फाइन-ट्यून गरिएको मोडेल भेरियन्टहरू
- **प्रशिक्षण डाटा स्रोतहरू**: मोडेल प्रशिक्षण र फाइन-ट्यूनिङका लागि प्रयोग गरिएका डेटासेटहरू

### व्यापक आपूर्ति श्रृंखला सुरक्षा रणनीति

#### **कम्पोनेन्ट प्रमाणीकरण र विश्वास**
- **उत्पत्ति प्रमाणीकरण**: सबै AI कम्पोनेन्टहरूको उत्पत्ति, लाइसेन्सिङ, र अखण्डता एकीकरण अघि प्रमाणीकरण गर्नुहोस्
- **सुरक्षा मूल्याङ्कन**: मोडेलहरू, डाटा स्रोतहरू, र AI सेवाहरूको लागि कमजोरी स्क्यान र सुरक्षा समीक्षा गर्नुहोस्
- **प्रतिष्ठा विश्लेषण**: AI सेवा प्रदायकहरूको सुरक्षा ट्र्याक रेकर्ड र अभ्यासहरूको मूल्याङ्कन गर्नुहोस्
- **अनुपालन प्रमाणीकरण**: सबै कम्पोनेन्टहरूले संगठनात्मक सुरक्षा र नियामक आवश्यकताहरू पूरा गर्छन् भनी सुनिश्चित गर्नुहोस्

#### **सुरक्षित परिनियोजन पाइपलाइनहरू**  
- **स्वचालित CI/CD सुरक्षा**: स्वचालित परिनियोजन पाइपलाइनहरूमा सुरक्षा स्क्यानिङ समावेश गर्नुहोस्
- **कलाकृति अखण्डता**: सबै परिनियोजित कलाकृतिहरू (कोड, मोडेलहरू, कन्फिगरेसनहरू) को क्रिप्टोग्राफिक प्रमाणीकरण कार्यान्वयन गर्नुहोस्
- **चरणबद्ध परिनियोजन**: प्रत्येक चरणमा सुरक्षा प्रमाणीकरणसहित प्रगतिशील परिनियोजन रणनीतिहरू प्रयोग गर्नुहोस्
- **विश्वसनीय कलाकृति भण्डारहरू**: प्रमाणीकरण गरिएका, सुरक्षित कलाकृति रजिस्ट्री र भण्डारहरूबाट मात्र परिनियोजन गर्नुहोस्

#### **लगातार अनुगमन र प्रतिक्रिया**
- **निर्भरता स्क्यानिङ**: सबै सफ्टवेयर र AI कम्पोनेन्ट निर्भरताहरूको लागि निरन्तर कमजोरी अनुगमन
- **मोडेल अनुगमन**: मोडेल व्यवहार, प्रदर्शन विचलन, र सुरक्षा असामान्यताहरूको निरन्तर मूल्याङ्कन
- **सेवा स्वास्थ्य ट्र्याकिङ**: बाह्य AI सेवाहरूको उपलब्धता, सुरक्षा घटनाहरू, र नीति परिवर्तनहरूको अनुगमन
- **खतरा बुद्धिमत्ता एकीकरण**: AI र ML सुरक्षा जोखिमहरूका लागि विशिष्ट खतरा फिडहरू समावेश गर्नुहोस्

#### **पहुँच नियन्त्रण र न्यूनतम विशेषाधिकार**
- **कम्पोनेन्ट-स्तर अनुमति**: व्यवसाय आवश्यकताका आधारमा मोडेलहरू, डाटा, र सेवाहरूमा पहुँच सीमित गर्नुहोस्
- **सेवा खाता व्यवस्थापन**: न्यूनतम आवश्यक अनुमति सहित समर्पित सेवा खाता कार्यान्वयन गर्नुहोस्
- **नेटवर्क विभाजन**: AI कम्पोनेन्टहरू अलग राख्नुहोस् र सेवाहरू बीच नेटवर्क पहुँच सीमित गर्नुहोस्
- **API गेटवे नियन्त्रण**: बाह्य AI सेवाहरूमा पहुँच नियन्त्रण र अनुगमनका लागि केन्द्रित API गेटवेहरू प्रयोग गर्नुहोस्

#### **घटना प्रतिक्रिया र पुन: प्राप्ति**
- **छिटो प्रतिक्रिया प्रक्रिया**: कम्प्रोमाइज्ड AI कम्पोनेन्टहरूका लागि प्याचिङ वा प्रतिस्थापनका स्थापित प्रक्रिया
- **प्रमाणीकरण कुञ्जी रोटेशन**: गोप्य, API कुञ्जीहरू, र सेवा प्रमाणीकरण कुञ्जीहरूको स्वचालित रोटेशन प्रणालीहरू
- **रोलब्याक क्षमता**: AI कम्पोनेन्टहरूको पहिलेको ज्ञात-सही संस्करणमा छिटो फर्कन सक्ने क्षमता
- **आपूर्ति श्रृंखला उल्लंघन पुन: प्राप्ति**: माथिल्लो स्तर AI सेवा कम्प्रोमाइजहरूमा प्रतिक्रिया दिने विशेष प्रक्रिया

### माइक्रोसफ्ट सुरक्षा उपकरणहरू र एकीकरण

**GitHub Advanced Security** ले समग्र आपूर्ति श्रृंखला सुरक्षा प्रदान गर्दछ जसमा समावेश छन्:
- **गोप्य स्क्यानिङ**: भण्डारहरूमा प्रमाणीकरण, API कुञ्जीहरू, र टोकनहरूको स्वचालित पत्ता लगाउने
- **निर्भरता स्क्यानिङ**: खुला स्रोत निर्भरताहरू र पुस्तकालयहरूको कमजोरी मूल्याङ्कन
- **CodeQL विश्लेषण**: सुरक्षा कमजोरी र कोडिङ समस्याहरूको स्थिर कोड विश्लेषण
- **आपूर्ति श्रृंखला अन्तर्दृष्टि**: निर्भरता स्वास्थ्य र सुरक्षा स्थिति दृश्यता

**Azure DevOps र Azure Repos एकीकरण:**
- माइक्रोसफ्ट विकास प्लेटफर्महरूमा सहज सुरक्षा स्क्यानिङ एकीकरण
- AI कार्यभारहरूको लागि Azure पाइपलाइनहरूमा स्वचालित सुरक्षा जाँचहरू
- सुरक्षित AI कम्पोनेन्ट परिनियोजनका लागि नीति कार्यान्वयन

**माइक्रोसफ्ट आन्तरिक अभ्यासहरू:**
माइक्रोसफ्टले सबै उत्पादनहरूमा व्यापक आपूर्ति श्रृंखला सुरक्षा अभ्यासहरू कार्यान्वयन गर्दछ। [The Journey to Secure the Software Supply Chain at Microsoft](https://devblogs.microsoft.com/engineering-at-microsoft/the-journey-to-secure-the-software-supply-chain-at-microsoft/) मा प्रमाणित दृष्टिकोणहरू जान्नुहोस्।


## आधारभूत सुरक्षा सर्वोत्तम अभ्यासहरू

MCP कार्यान्वयनहरूले तपाईंको संगठनको विद्यमान सुरक्षा अवस्थाबाट उत्तराधिकार लिन्छ र त्यसलाई विस्तार गर्छ। आधारभूत सुरक्षा अभ्यासहरूलाई सुदृढ पार्दा AI प्रणालीहरू र MCP परिनियोजनहरूको समग्र सुरक्षा उल्लेखनीय रूपमा बढ्छ।

### मुख्य सुरक्षा आधारहरू

#### **सुरक्षित विकास अभ्यासहरू**
- **OWASP अनुपालन**: [OWASP Top 10](https://owasp.org/www-project-top-ten/) वेब अनुप्रयोग कमजोरीहरूबाट सुरक्षा गर्नुहोस्
- **AI-विशिष्ट सुरक्षा**: [OWASP Top 10 for LLMs](https://genai.owasp.org/download/43299/?tmstv=1731900559) का लागि नियन्त्रणहरू कार्यान्वयन गर्नुहोस्
- **सुरक्षित गोप्य व्यवस्थापन**: टोकन, API कुञ्जीहरू, र संवेदनशील कन्फिगरेसन डाटाका लागि समर्पित भल्टहरू प्रयोग गर्नुहोस्
- **एन्ड-टु-एन्ड इन्क्रिप्सन**: सबै अनुप्रयोग कम्पोनेन्ट र डाटा प्रवाहहरूमा सुरक्षित सञ्चार कार्यान्वयन गर्नुहोस्
- **इनपुट प्रमाणीकरण**: सबै प्रयोगकर्ता इनपुट, API प्यारामिटरहरू, र डाटा स्रोतहरूको कडाइका साथ प्रमाणीकरण गर्नुहोस्

#### **पूर्वाधार कडा बनाउने**
- **बहु-कारक प्रमाणीकरण**: सबै प्रशासनिक र सेवा खाताहरूका लागि अनिवार्य MFA
- **प्याच व्यवस्थापन**: अपरेटिङ सिस्टम, फ्रेमवर्क, र निर्भरताहरूको लागि स्वचालित, समयमा प्याचिङ  
- **पहिचान प्रदायक एकीकरण**: एंटरप्राइज पहिचान प्रदायकहरू (Microsoft Entra ID, Active Directory) मार्फत केन्द्रित पहिचान व्यवस्थापन
- **नेटवर्क विभाजन**: MCP कम्पोनेन्टहरूको तार्किक अलगाव जसले पार्श्वगत आन्दोलन सम्भावना सीमित गर्छ
- **न्यूनतम विशेषाधिकार सिद्धान्त**: सबै प्रणाली कम्पोनेन्ट र खाताहरूका लागि न्यूनतम आवश्यक अनुमति

#### **सुरक्षा अनुगमन र पत्ता लगाउने**
- **व्यापक लगिङ**: AI अनुप्रयोग गतिविधिहरूको विस्तृत लगिङ, जसमा MCP क्लाइन्ट-सर्भर अन्तरक्रियाहरू समावेश छन्
- **SIEM एकीकरण**: असामान्यताहरू पत्ता लगाउन केन्द्रित सुरक्षा सूचना र घटना व्यवस्थापन
- **व्यवहार विश्लेषण**: प्रणाली र प्रयोगकर्ता व्यवहारमा असामान्य ढाँचाहरू पत्ता लगाउन AI-संचालित अनुगमन
- **खतरा बुद्धिमत्ता**: बाह्य खतरा फिडहरू र कम्प्रोमाइज सूचकहरू (IOCs) को एकीकरण
- **घटना प्रतिक्रिया**: सुरक्षा घटना पत्ता लगाउने, प्रतिक्रिया दिने, र पुन: प्राप्ति गर्ने राम्रो परिभाषित प्रक्रिया

#### **शून्य विश्वास वास्तुकला**
- **कहिल्यै विश्वास नगर्नुहोस्, सधैं प्रमाणीकरण गर्नुहोस्**: प्रयोगकर्ता, उपकरण, र नेटवर्क जडानहरूको निरन्तर प्रमाणीकरण
- **माइक्रो-सेग्मेन्टेसन**: व्यक्तिगत कार्यभार र सेवाहरू अलग गर्ने सूक्ष्म नेटवर्क नियन्त्रणहरू
- **पहिचान-केंद्रित सुरक्षा**: नेटवर्क स्थानको सट्टा प्रमाणीकरण गरिएको पहिचानमा आधारित सुरक्षा नीतिहरू
- **निरन्तर जोखिम मूल्याङ्कन**: वर्तमान सन्दर्भ र व्यवहारमा आधारित गतिशील सुरक्षा अवस्था मूल्याङ्कन
- **सशर्त पहुँच**: जोखिम कारक, स्थान, र उपकरण विश्वासको आधारमा अनुकूलित पहुँच नियन्त्रणहरू

### एंटरप्राइज एकीकरण ढाँचाहरू

#### **माइक्रोसफ्ट सुरक्षा पारिस्थितिकी तन्त्र एकीकरण**
- **Microsoft Defender for Cloud**: समग्र क्लाउड सुरक्षा अवस्था व्यवस्थापन
- **Azure Sentinel**: AI कार्यभार सुरक्षाका लागि क्लाउड-नेटिभ SIEM र SOAR क्षमता
- **Microsoft Entra ID**: सशर्त पहुँच नीतिहरू सहित एंटरप्राइज पहिचान र पहुँच व्यवस्थापन
- **Azure Key Vault**: हार्डवेयर सुरक्षा मोड्युल (HSM) समर्थन सहित केन्द्रित गोप्य व्यवस्थापन
- **Microsoft Purview**: AI डाटा स्रोतहरू र कार्यप्रवाहहरूको लागि डाटा शासन र अनुपालन

#### **अनुपालन र शासन**
- **नियमक समायोजन**: MCP कार्यान्वयनहरूले उद्योग-विशिष्ट अनुपालन आवश्यकताहरू (GDPR, HIPAA, SOC 2) पूरा गर्ने सुनिश्चित गर्नुहोस्
- **डाटा वर्गीकरण**: AI प्रणालीहरूले प्रशोधन गर्ने संवेदनशील डाटाको उचित वर्गीकरण र ह्यान्डलिङ
- **अडिट ट्रेलहरू**: नियामक अनुपालन र फरेन्सिक अनुसन्धानका लागि व्यापक लगिङ
- **गोपनीयता नियन्त्रणहरू**: AI प्रणाली वास्तुकलामा गोपनीयता-डिजाइन सिद्धान्तहरूको कार्यान्वयन
- **परिवर्तन व्यवस्थापन**: AI प्रणाली परिमार्जनहरूको सुरक्षा समीक्षा लागि औपचारिक प्रक्रिया

यी आधारभूत अभ्यासहरूले एक बलियो सुरक्षा आधार सिर्जना गर्छन् जसले MCP-विशिष्ट सुरक्षा नियन्त्रणहरूको प्रभावकारिता बढाउँछ र AI-चालित अनुप्रयोगहरूको लागि समग्र सुरक्षा प्रदान गर्छ।

## मुख्य सुरक्षा सन्देशहरू

- **स्तरीय सुरक्षा दृष्टिकोण**: आधारभूत सुरक्षा अभ्यासहरू (सुरक्षित कोडिङ, न्यूनतम विशेषाधिकार, आपूर्ति श्रृंखला प्रमाणीकरण, निरन्तर अनुगमन) लाई AI-विशिष्ट नियन्त्रणहरूसँग संयोजन गरी समग्र सुरक्षा सुनिश्चित गर्नुहोस्

- **AI-विशिष्ट खतरा परिदृश्य**: MCP प्रणालीहरूले विशिष्ट जोखिमहरू सामना गर्छन् जस्तै प्रॉम्प्ट इन्जेक्सन, उपकरण विषाक्तता, सेसन हाइज्याकिङ, भ्रमित डिप्युटी समस्या, टोकन पासथ्रू कमजोरीहरू, र अत्यधिक अनुमति जसका लागि विशेष निवारण आवश्यक छ

- **प्रमाणीकरण र प्राधिकरण उत्कृष्टता**: बाह्य पहिचान प्रदायकहरू (Microsoft Entra ID) प्रयोग गरी बलियो प्रमाणीकरण कार्यान्वयन गर्नुहोस्, उचित टोकन प्रमाणीकरण लागू गर्नुहोस्, र कहिल्यै MCP सर्भरका लागि स्पष्ट रूपमा जारी नभएका टोकनहरू स्वीकार नगर्नुहोस्

- **AI आक्रमण रोकथाम**: Microsoft Prompt Shields र Azure Content Safety प्रयोग गरी अप्रत्यक्ष प्रॉम्प्ट इन्जेक्सन र उपकरण विषाक्तता आक्रमणहरूबाट सुरक्षा गर्नुहोस्, उपकरण मेटाडाटा प्रमाणीकरण गर्नुहोस् र गतिशील परिवर्तनहरूको अनुगमन गर्नुहोस्

- **सेसन र ट्रान्सपोर्ट सुरक्षा**: प्रयोगकर्ता पहिचानसँग बाँधिएका क्रिप्टोग्राफिक रूपमा सुरक्षित, गैर-निर्धारित सेसन ID हरू प्रयोग गर्नुहोस्, उचित सेसन जीवनचक्र व्यवस्थापन कार्यान्वयन गर्नुहोस्, र प्रमाणीकरणका लागि कहिल्यै सेसनहरू प्रयोग नगर्नुहोस्

- **OAuth सुरक्षा सर्वोत्तम अभ्यासहरू**: गतिशील रूपमा दर्ता गरिएका क्लाइन्टहरूको लागि स्पष्ट प्रयोगकर्ता सहमति, PKCE सहित उचित OAuth 2.1 कार्यान्वयन, र कडा रिडिरेक्ट URI प्रमाणीकरणमार्फत भ्रमित डिप्युटी आक्रमणहरू रोक्नुहोस्  

- **टोकन सुरक्षा सिद्धान्तहरू**: टोकन पासथ्रू एन्टी-प्याटर्नहरूबाट बच्नुहोस्, टोकन दर्शक दाबीहरू प्रमाणीकरण गर्नुहोस्, सुरक्षित रोटेशनसहित छोटो अवधिका टोकनहरू कार्यान्वयन गर्नुहोस्, र स्पष्ट विश्वास सीमाहरू कायम राख्नुहोस्

- **व्यापक आपूर्ति श्रृंखला सुरक्षा**: सबै AI पारिस्थितिकी तन्त्र कम्पोनेन्टहरू (मोडेलहरू, एम्बेडिङहरू, सन्दर्भ प्रदायकहरू, बाह्य API हरू) लाई परम्परागत सफ्टवेयर निर्भरताहरू जस्तै कडा सुरक्षा मापदण्डका साथ व्यवहार गर्नुहोस्

- **निरन्तर विकास**: छिटो विकास भइरहेका MCP विनिर्देशहरूसँग अद्यावधिक रहनुहोस्, सुरक्षा समुदायका मापदण्डहरूमा योगदान दिनुहोस्, र प्रोटोकल परिपक्व हुँदै जाँदा अनुकूल सुरक्षा अवस्थाहरू कायम राख्नुहोस्

- **माइक्रोसफ्ट सुरक्षा एकीकरण**: Microsoft को समग्र सुरक्षा पारिस्थितिकी तन्त्र (Prompt Shields, Azure Content Safety, GitHub Advanced Security, Entra ID) लाई MCP परिनियोजन सुरक्षाका लागि उपयोग गर्नुहोस्
