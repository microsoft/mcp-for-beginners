# Παράδειγμα: Βασικός Φιλοξενητής

Μια υλοποίηση αναφοράς που δείχνει πώς να δημιουργήσετε μια εφαρμογή φιλοξενίας MCP που συνδέεται με διακομιστές MCP και αποδίδει τα περιβάλλοντα χρήστη εργαλείων σε ένα ασφαλές περιβάλλον sandbox.

Αυτός ο βασικός φιλοξενητής μπορεί επίσης να χρησιμοποιηθεί για τη δοκιμή εφαρμογών MCP κατά την τοπική ανάπτυξη.

## Κύρια Αρχεία

- [`index.html`](../../../../../../../03-GettingStarted/15-mcp-apps/ext-apps/examples/basic-host/index.html) / [`src/index.tsx`](../../../../../../../03-GettingStarted/15-mcp-apps/ext-apps/examples/basic-host/src/index.tsx) - Φιλοξενούμενο UI React με επιλογή εργαλείου, εισαγωγή παραμέτρων και διαχείριση iframe
- [`sandbox.html`](../../../../../../../03-GettingStarted/15-mcp-apps/ext-apps/examples/basic-host/sandbox.html) / [`src/sandbox.ts`](../../../../../../../03-GettingStarted/15-mcp-apps/ext-apps/examples/basic-host/src/sandbox.ts) - Εξωτερικός proxy iframe με επαλήθευση ασφάλειας και αμφίδρομη διαβίβαση μηνυμάτων
- [`src/implementation.ts`](../../../../../../../03-GettingStarted/15-mcp-apps/ext-apps/examples/basic-host/src/implementation.ts) - Κεντρική λογική: σύνδεση διακομιστή, κλήση εργαλείων, και ρύθμιση AppBridge

## Ξεκινώντας

```bash
npm install
npm run start
# Ανοίξτε το http://localhost:8080
```

Κατά προεπιλογή, η εφαρμογή φιλοξενίας θα προσπαθήσει να συνδεθεί με διακομιστή MCP στη διεύθυνση `http://localhost:3001/mcp`. Μπορείτε να διαμορφώσετε αυτή τη συμπεριφορά ορίζοντας τη μεταβλητή περιβάλλοντος `SERVERS` με έναν πίνακα JSON που περιέχει διευθύνσεις URL διακομιστών:

```bash
SERVERS='["http://localhost:1234/mcp", "http://localhost:5678/mcp"]' npm run start
```

## Αρχιτεκτονική

Αυτό το παράδειγμα χρησιμοποιεί μοτίβο διπλού iframe sandbox για ασφαλή απομόνωση UI:

```
Host (port 8080)
  └── Outer iframe (port 8081) - sandbox proxy
        └── Inner iframe (srcdoc) - untrusted tool UI
```

**Γιατί δύο iframes;**

- Το εξωτερικό iframe λειτουργεί σε ξεχωριστό origin (θύρα 8081) αποτρέποντας άμεση πρόσβαση στον φιλοξενητή
- Το εσωτερικό iframe λαμβάνει HTML μέσω `srcdoc` και είναι περιορισμένο από τα χαρακτηριστικά sandbox
- Τα μηνύματα ρέουν μέσω του εξωτερικού iframe που τα επαληθεύει και τα διαβιβάζει αμφίδρομα

Αυτή η αρχιτεκτονική διασφαλίζει ότι ακόμα και αν ο κώδικας UI του εργαλείου είναι κακόβουλος, δεν μπορεί να έχει πρόσβαση στο DOM, στα cookies ή στο πλαίσιο JavaScript της εφαρμογής φιλοξενίας.
