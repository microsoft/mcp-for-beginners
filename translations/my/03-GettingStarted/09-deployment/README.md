# MCP ဆာဗာများ တပ်ဆင်ခြင်း

သင်၏ MCP ဆာဗာကို တပ်ဆင်ခြင်းဖြင့် မိမိ ဒေသတွင်းပတ်ဝန်းကျင်ပြင်ပမှ အခြားသူများအတွက် ၎င်း၏ တူလ်များနှင့် အရင်းအမြစ်များကို အသုံးပြုနိုင်စေပါသည်။ သင်၏ အလျင်မြန်မှု၊ ယုံကြည်စိတ်ချရမှု၊ နှင့် စီမံခန့်ခွဲရလွယ်ကူမှုလိုအပ်ချက်အပေါ် မူတည်၍ အမျိုးမျိုးသော တပ်ဆင်မှု များကို စဉ်းစားနိုင်ပါသည်။ အောက်တွင် သို့တပ်ဆင်ခြင်းဆိုင်ရာ လမ်းညွှန်ချက်များကို ဒေသတွင်း၊ container များတွင်နှင့် cloud သို့ တပ်ဆင်ခြင်းအပေါ် ရယူနိုင်ပါသည်။

## အနှစ်ချုပ်

ဤသင်ခန်းစာတွင် သင့် MCP Server အက်ပ်ကို မည်သို့ တပ်ဆင်ရမည်ကို ဖော်ပြထားသည်။

## သင်ယူရမည့် ရည်ရွယ်ချက်များ

ဤသင်ခန်းစာပြီးဆုံးသည့်အထိ သင်သည် အောက်ပါအရာများကို လုပ်ဆောင်နိုင်ပါလိမ့်မည်-

- မတူညီသော တပ်ဆင်မှုနည်းလမ်းများကို ခန့်မှန်းသုံးသပ်နိုင်ခြင်း။
- သင့်အက်ပ်ကို တပ်ဆင်နိုင်ခြင်း။

## ဒေသတွင်း ဖွံ့ဖြိုးမှုနှင့် တပ်ဆင်ခြင်း

သင့်ဆာဗာကို အသုံးပြုသူများ၏ စက်ပေါ်တွင် ပြေးဆွဲ၍ အသုံးပြုရန် ရည်ရွယ်ပါက အောက်ပါအဆင့်များကို လိုက်နာနိုင်ပါသည်-

1. **ဆာဗာကို ဒေါင်းလုဒ်လုပ်ပါ**။ ဆာဗာကို မရေးသားထားပါက ပထမဆုံး စက်သို့ ဒေါင်းလုဒ်လုပ်ပါ။
1. **ဆာဗာလုပ်ငန်းစဉ်ကို စတင်ပါ**: သင့် MCP ဆာဗာ အက်ပ်ကို ပြေးဆွဲပါ

SSE အတွက် (stdio type ဆာဗာမှာ မလိုအပ်ပါ)

1. **ကွန်ရက် ဆက်သွယ်မှုကို ပြင်ဆင်ပါ**: ဆာဗာကို မျှော်မှန်းထားသော port ပေါ်တွင် ရရှိနိုင်ရန် သေချာစေပါ
1. **ဖောက်သည်များကို ချိတ်ဆက်ပါ**: `http://localhost:3000` ကဲ့သို့ ဒေသတွင်း ချိတ်ဆက်မှု URL များကို အသုံးပြုပါ

## Cloud တပ်ဆင်ခြင်း

MCP ဆာဗာများကို cloud ပလက်ဖောင်းအမျိုးမျိုးသို့ တပ်ဆင်နိုင်သည်-

- **Serverless Functions**: အလွန် သေးငယ်၍ လွယ်ကူသော MCP ဆာဗာများကို serverless functions အဖြစ် တပ်ဆင်ပါ
- **Container Services**: Azure Container Apps, AWS ECS, သို့မဟုတ် Google Cloud Run ကဲ့သို့သော ဝန်ဆောင်မှုများကို အသုံးပြုပါ
- **Kubernetes**: MCP ဆာဗာများကို Kubernetes cluster များတွင် တပ်ဆင်ပြီး အမြဲရရှိနိုင်မှုအတွက် စီမံခန့်ခွဲပါ

### ဥပမာ- Azure Container Apps

Azure Container Apps သည် MCP ဆာဗာများ တပ်ဆင်ခြင်းကို ထောက်ပံ့သည်။ ၎င်းကို များစွာ တိုးတက်နေဆဲဖြစ်ပြီး ယနေ့တွင် SSE ဆာဗာများကိုသာ ထောက်ပံ့ပေးသည်။

ဒီလိုလုပ်ဆောင်နိုင်ပါတယ်-

1. repo တစ်ခုကို Clone လုပ်ပါ-

  ```sh
  git clone https://github.com/anthonychu/azure-container-apps-mcp-sample.git
  ```

1. အဲဒါကို ဒေသတွင်း စမ်းသပ်ရန် ပြေးဆွဲပါ-

  ```sh
  uv venv
  uv sync

  # လိန်နပ်စ်/မက်အိုအက်စ်
  export API_KEYS=<AN_API_KEY>
  # ဝင်းဒိုးပ်စ်
  set API_KEYS=<AN_API_KEY>

  uv run fastapi dev main.py
  ```

1. ဒေသတွင်း စမ်းသပ်ရန် အတွက် *.vscode* ဖိုလ်ဒါတွင် *mcp.json* ဖိုင်တစ်ခု ဖန်တီးပြီး အောက်ပါ အကြောင်းအရာကို ထည့်ပါ-

  ```json
  {
      "inputs": [
          {
              "type": "promptString",
              "id": "weather-api-key",
              "description": "Weather API Key",
              "password": true
          }
      ],
      "servers": {
          "weather-sse": {
              "type": "sse",
              "url": "http://localhost:8000/sse",
              "headers": {
                  "x-api-key": "${input:weather-api-key}"
              }
          }
      }
  }
  ```

  SSE ဆာဗာ စတင်ပြီးနောက် JSON ဖိုင်ထဲရှိ play အိုင်ကွန်ကို နှိပ်နိုင်ပြီး ဆာဗာပေါ်ရှိ tools များကို GitHub Copilot မှ ဖမ်းယူလိုက်သည်ကို တွေ့နိုင်ပါသည်၊ Tool အိုင်ကွန်ကို ကြည့်ပါ။

1. တပ်ဆင်ရန် အောက်ပါ command ကို လည်ပတ်ပါ-

  ```sh
  az containerapp up -g <RESOURCE_GROUP_NAME> -n weather-mcp --environment mcp -l westus --env-vars API_KEYS=<AN_API_KEY> --source .
  ```

ဒါဖြင့် ဒေသတွင်းတွင် တပ်ဆင်ခြင်း၊ Azure သို့ တပ်ဆင်ခြင်းတို့ကို အဆင့်ဆင့် ပြုလုပ်နိုင်ပါပြီ။
